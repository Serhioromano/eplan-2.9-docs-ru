*[Действия]: Операцией считается функциональность программы, которая выполняется программой "Eplan" и влияние которой заметно для пользователя. Некоторые операции можно вставлять в ленту в качестве команды, некоторые операции можно автоматически выполнять с помощью функции "Автоматизированная обработка", некоторые операции можно применять при вызове Eplan с помощью командной строки.
*[Свойства]: Признаки со значением или настройкой, которые присвоены объекту. Свойство может иметь разные значения и/или настройки для различных объектов одного типа. Свойства влияют на объект, но не на работу системы. Свойства идентифицируются с помощью номеров и могут выводиться в формах и т.д. В программе свойства подразделены на различные категории (например, 'данные', 'настройки', 'форматы'), которые могут выводиться на просмотр независимо друг от друга. В зависимости от текущей группы пользователей просмотреть и обработать можно разные свойства.
*[Фильтр]: Фильтры создают по определенным правилам подмножества из множества. Фильтры ограничивают, например, количество просматриваемых элементов в списке или дереве.
*[Автозаполнение]: В настройках перевода, специфических для пользователя, можно установить автоматическое добавление переводимых слов при вводе текста. Регистр не обязательно должен совпадать. Если вы принимаете предлагаемое слово (например, щелчком мыши, клавишей[Enter]или переходом в другое поле), то регистр отображаемого текста корректируется с учетом регистра в словаре.
*[Панели инструментов]: Панели инструментов состоят из кнопок, посредством которых можно выполнять операции, предусмотренные в программе. Панели инструментов и их вид могут быть настроены пользователем. Кроме того, пользователь может создавать свои собственные панели инструментов.
*[Главное окно]: Главное окно программы является ее рабочим пространством; вы можете изменять его размер и положение. В главном окне можно создавать и изменять схемы соединений, списки и обзоры, вызывать диалоговые окна и обрабатывать их объекты, а также присоединять панели инструментов.
*[Настройки]: Настройки оказывают влияние на поведение свойств объекта, а также на поведение программы; они идентифицируются посредством имени (см.Схемы) и не могут выводиться в формах и т.д. С помощью настроек можно адаптировать программу под ваши требования и задачи.
*[Сохранить]: Как только во время обработки проекта выбирается, открывается или создается форма, рамка и т. д, которая еще не использовалась в проекте, она сохраняется. "Сохранить" означает создание в проекте копии, к которой происходит обращение в процессе дальнейшей работе. Основные данные можно синхронизировать как специфические для проекта и в зависимости от настройки проекта, при открытии проекта, а также при резервировании/восстановлении с пулом основных данных. Если определенные основные данные, необходимые для обработки проектов, отсутствуют в пуле основных данных, их можно найти с помощью опции "Дополнить проект" и при необходимости автоматически сохранить. Если отсутствующие основные данные не могут быть дополнительно загружены из пула основных данных, появляется соответствующее сообщение.
*[Функции]: Функция является объектом, которому присвоено определение функции (см.Определения функций). Если функция размещается на схеме соединений (т.е. представлена символом на схеме соединений), то она понимается также как условное обозначение. Каждое устройство может содержать одну или несколько функций. Так, например, контактор состоит из катушки и нескольких контактов.
*[Соединения]: Соединение является участком, посредством которого от источника к цели транспортируется информация, электроэнергия, жидкость или воздух (давление). Соединение существует между двумя выводами устройства. Между двумя целями может существовать несколько соединений (например, удвоение с помощью перекрестных ссылок).
*[Символы]: Символ – это графика для представления функций (см.Функции). Логические данные обычно сохраняются в определениях функций. Символ может содержать определенные логические настройки, например для выводов устройств. (Объекты в интерфейсе пользователя, наоборот, представлены пиктограммами.)
*[Данные проекта]: Данные проекта – это все данные, которые содержатся в проекте.
*[Основные данные]: К основным данным относятся, например, библиотеки символов, рамки, формы, макросы, базовые проекты, данные изделий, словарь для переводов или база данных прав Eplan. В Eplan различаются основные данные проекта и основные данные системы.
*[Сеть]: Сеть в Eplan является подмножеством сигнала. Сеть состоит из выводов устройства, непосредственно связанных друг с другом, и соединениями между этими выводами. Внутри сети могут также находиться соединенные с сетью функции (т.е. на функции включено свойство Соединенный с сетью).
*[Схемы соединений]: Схема соединений — расположение (т.е. группа) выводов устройств. Схема соединений может быть присвоена изделию и определяет пространственное расположение и специфические характеристики соединений.
*[Сетка]: Для упрощения работы при размещении элементов имеется возможность использования сетки. При это точки вставки и точки элементов позиционируются по точкам сетки. При работе в трехмерном пространстве листа используется линейный растр. В этом случае точки вставки лежат на точках дуг окружностей линий. Возможная раздельная настройка сетки для 2-мерного и 3-мерного видов.
*[Навигаторы]: Навигаторы предлагают особый ракурс данных проекта, при этом отображаются все соответствующие данные открытого проекта, например обзор всех кабелей, всех штекеров или всех соединений. Эти диалоговые окна можно присоединять и оставлять открытыми во время обработки проекта. Для обработки отдельных проектных данных из навигатора можно вызвать диалоговое окно обработки (как правило, через всплывающее меню).
*[Присоединить]: Присоединить означает закрепить стыкуемые ("dockable") окна на рамке главной программы.
*[Формы]: Формы отображают информацию или результаты прогонов отчета в графической форме. С помощью форм устанавливается, какие свойства выдаются в какой последовательности, в каком лэйауте и т.д.
*[Пространство листа]: Пространство листа представляет собой трехмерную область отображения, показывающую размещенное устройство. Пространство листа обеспечивает представление и обработку данных в трехмерном режиме. Пространство листа не зависит от страницы проекта.
*[Объекты]: Объекты – это компоненты проекта, например, сам проект, страницы проекта, условные обозначения или функции, отдельные элементы на странице проекта. Объекты имеют свойства, которые можно обрабатывать или изменять.
*[Изделия]: Изделие является экономическим элементом, который охватывает как коммерческие, так и технические данные. Изделие не должно содержать технические функции, если оно не относится к устройству с соответствующими функциями. Посредством изделий также определяются устройства, при этом изделию присваиваются соответствующие шаблоны функций.
*[Разделитель]: С помощью разделителей главные идентификаторы отделяются от подидентификаторов. В Eplan практически для всех блоков идентификаторов можно выбрать точку или другой разделитель.
*[Кабели]: В общем под кабелем понимается объединение нескольких электрических или оптических проводников, образующих одно физическое целое. Кабель служит для передачи электрической энергии, данных, сигналов, света и т. д. Исключение: в энергетике приняты также одножильные кабели, напр., 1 х 240 мм2. У кабелей с несколькими жилами, как правило, отдельные жилы электрически изолированы. Внутри внешней и внутренней изоляции одновременно могут существовать неизолированные, но проведенные отдельно экранирования.В Eplan кабелем называется объединение нескольких кабельных соединений. Этими кабельными соединениями могут быть жилы, а также световоды, шланги и т.д.В базе данных изделий под "кабелем" подразумевается изделие кабеля.В графическом редакторе под "кабелем" подразумевается устройство (напр., "W2") и собранные при помощи линий определения кабеля соединения.
*[Экранирования]: Экранирования позволяют предотвратить или сократить проникновение или излучение электрических и /или магнитных полей.Экранирования внутри кабелей могут быть представлены в разной форме. Внешнее или общее экранирование охватывает все жилы кабеля. Экранирование может быть односторонним, двухсторонним или непроложенным.Наряду с общим экранированием существует экранирование отдельных жил и сразу нескольких жил. Типичными являются жилы с попарным экранированием. В одном кабеле могут встречаться физически разные формы экранов. Эти внутренние экраны также могут быть односторонними, двухсторонними или непроложенными.
*[Сортировка страниц]: Сортировка страниц показывает, как страница отчетов сортируется в имеющуюся структуру страниц.
*[Обозначения клемм]: Обозначение клеммы предназначено для идентификации клеммы; в нем могут использоваться цифры и любые другие символы, например "5", "PE", "+" и т.д.
*[Устройства]: Устройстваявляются логическими взаимодействующими единицами электротехники, Fluid-техники или технологии производственных процессов. Устройства обозначаются с помощью обозначения устройства (ОУ), например: M1, K1, X1, XS1, W1.Присвоив одно изделие (или несколько), устройство получает данные о своих функциях и в результате определяется как устройство. Наконец, устройство — это предварительно определенный набор функций (с заранее определенными (идентифицированными) и со свободными свойствами).Механические устройстваимеют определение функции из области раздела "Механика". Механические устройства по аналогии с логическими (напр., электротехническими) устройствами помечаются определенным обозначением устройства (ОУ), но не имеют выводов устройства. Механические устройства можно разместить в виде трехмерных размещений изделий в пространстве листа.
*[Неразмещенные функции]: 'Неразмещенные' функции — это необозначенные части устройств. Неразмещенные устройства используются в проекте, но не видны на схеме соединений.
*[Место установки]: Блок идентификаторов "Место установки" обозначает физическое место установки устройств или технического оборудования, например, электрошкафа, пульта управления и т.д. Возможный предшествующий знак в Eplan "+".
*[Пиктограммы]: Пиктограмма — это небольшой графический символ, который представляет объект (например, страницу, файл, функцию) в интерфейсе пользователя. Пиктограммы отображаются в диалоговых окнах (например, в древовидном представлении навигаторов), чтобы показать вид и/или состояние объекта.
*[Fluid-Техника]: Понятие Fluid-техники охватывает все процессы, в которых энергия передается посредством газа или жидкости.
*[Обозначение устройства]: На всех устройствах в промышленных системах, установках и оборудовании должно быть нанесено уникальное обозначение. Это обозначение устройства (ОУ) также должно быть внесено на соответствующем символе на схеме соединений. Перед обозначением устройства в качестве предшествующего знака ставится "-". В качествеполногообозначения устройства его можно расширить блоками идентификаторов "Функциональное присвоение" (предшествующий знак "=="), "Функциональное назначение" (предшествующий знак "="), "Место сборки" (предшествующий знак "++") и "Место установки" (предшествующий знак "+").
*[Функциональное назначение]: Блок идентификаторов "Функциональное назначение" обозначает установки, подустановки и техническое оборудование. Возможный предшествующий знак в Eplan — "=".
*[Обозначения]: Обозначение – это идентифицирующий текст (например, обозначение вывода устройства).
*[Сегменты]: В предварительном планировании "Сегмент" является единым понятием для структурных сегментов и объектов планирования. Структурные сегменты описывают часть структуры проекта, объекты планирования определяют часть установки и описывают устройство.
*[DXF]: DXF обозначает "Drawing eXchange Format" или "Drawing Interchange File Format" и представляет собой формат ASCII или двоичный формат файлов компании Autodesk для файлов чертежей AutoCAD. Этот формат можно использовать для экспорта чертежей AutoCAD в другие приложения или для импорта чертежей из других приложений. В настоящее время DXF является стандартом для обмена данными в системах проектирования и конструирования.
*[DWG]: DWG обозначает формат "DraWinG". Это стандартный файловый формат компании Autodesk для сохранения векторной графики в AutoCAD.
*[Комментарии]: В свойство "Комментарий" можно ввести любой дополнительный текст.
*[Рамки]: Рамка определяет логическую структуру схемы соединений, т.е. с помощью присвоения рамки к странице устанавливаются заголовок, размер листа и разделение на строки и столбцы. Для этого дополнительно имеются специальные тексты, тексты строк и столбцов.
*[Условное обозначение]: Условное обозначение – это графический элемент для отображения определенной функции (см.Функции). Условное обозначение состоит из функции и символа. При этом функция содержит логические данные, а символ – графические. Условное обозначение содержит обозначения устройства, обозначения выводов устройства и т. п.
*[Структурный идентификатор]: Каждому блоку идентификаторов присваивается определенный идентификатор, который состоит из последовательности букв и / или цифр. Этот идентификатор называется "Структурный идентификатор".
*[Конфигурации]: Конфигурация – это группа настроек, которые не сохраняются как схема, например, конфигурация столбцов в навигаторе страниц.
*[Точки определения потенциала]: Точки определения потенциала служат для того, чтобы присваивать одному или нескольким проходящим соединениям свойства потенциала, отличающиеся от тех, которые предварительно установлены в проекте, на выводе потенциала или в точке разрыва.
*[Схемы]: Схема — это группа настроек, которые сохраняются под определенным именем. Они служат шаблонами для различных настроек программы (например, фильтров, настроек отображения, опций). Существуют пользовательские и предварительно определенные схемы.
*[Уровни]: В многоуровневых клеммах "уровнями" обозначают различные "этажи", на которых расположены отдельные клеммы в корпусе клеммника.
*[Создать]: В Eplan термин "Создать" означает генерировать новый объектвручную(например, проект, страницу и т. д.). В меню есть запись "Создать". Даже если объекты генерируются на основе уже чего-то существующего, это все равно обозначает "создание" (например, базовые проекты создаются на основе существующего проекта).
*[Определено пользователем]: Блок идентификаторов "Определено пользователем" в Eplan позволяет задавать определенные структуры проектов. Этот блок идентификаторов можно использовать в комбинации со всеми блоками идентификаторов, которые определены по DIN. В Eplan возможен любой знак (предварительно установлен "#").
*[Макросы]: Макросы – это любые фрагменты из страниц или проектов, которые могут быть сохранены для дальнейшего использования. Благодаря установленным макросам не требуется создавать эти фрагменты каждый раз заново, особенно при большом их количестве.
*[Отсоединить]: Отсоединить означает открепить стыкуемые ("dockable") окна от рамки главной программы.
*[Редактор символов]: Редактор символов – это редактор для создания и обработки символов и библиотек символов.
*[Рабочие области]: Рабочая область Eplan охватывает размеры и позиции главного окна Eplan, редакторов, центра вставки и "присоединяемых" диалоговых окон. Для "присоединяемых" диалоговых окон и для центра вставки сохраняются также настройки позиции окна (например, присоединение, всплывающее окно и т. д.). В рабочую область сохраняются также настройки ленты с определенными пользователем вкладками и группами команд. Рабочие области можно создавать в любое время и использовать их в дальнейшем.
*[Базы данных изделий]: База данных изделий содержит технические и коммерческие данные изделия. В Eplan различают собственную базу данных изделий Eplan, которую можно обрабатывать в базе данных изделий, и внешние базы данных изделий (базы данных посторонних организаций). Внешние базы данных можно использовать дополнительно. В качестве примера можно назвать решение PMS, подключение к которому реализуется через соответствующий интерфейс.
*[Схемы сверления]: Схемы сверления представляют собой производственную документацию, присвоенную отдельному изделию. Схемы сверления содержат указания относительно вида и положения монтажных отверстий и отверстий в функциональных элементах электрошкафов.
*[Проекты]: В Eplan схемы соединений и относящиеся к ним документы, такие как списки и обзоры, создаются внутри проектов как страницы. Проект - это база данных, в которой кроме страниц проекта сохранены и все использованные в проекте основные данные (символы, рамки, формы, данные изделий и т. д.) Учет и систематизация проектов осуществляется в дополнительной базе данных, называемой "Управление проектами" (см.База данных управления проектами).
*[Элементы формата]: Элементы формата – это составные части форматов. Например, формат нумерации может содержать элементы формата "Буквенное обозначение", "Счетчик", "Страница", "Столбец" и "Строка".
*[Библиотеки символов]: Библиотека символов — это коллекция символов. Символы библиотеки символов чаще всего относятся к определенной группе или соответствуют определенному стандарту. Библиотеки символов могут быть загружены в проект или записываться в пуле основных данных.
*[Узлы]: Узлом является коллекция изделий, которые относятся кодномуустройству (например, кнопочный переключатель с замыкающим контактом, соответствующим кронштейном и нажимной кнопкой). Узел имеет собственный номер изделия и в свою очередь может сам содержать узлы (подузлы).
*[Переменные пути]: Переменная пути — это последовательность символов, используемая в Eplan для указания конкретных сведений, таких как имя дисковода, каталога и / или файла. Значения некоторых переменных задаются во время установки и могут быть частично изменены в настройках каталога; значения других переменных пути относятся к текущему проекту (каталогу проекта, свойствам проекта и т. д.) или к указанию времени, заданному на панели управления (дата и время). Можно также получить доступ к значениям переменных окружения.
*[Свойства проекта]: Свойства проекта – это информация, которая описывает проект.
*[Ссылки изделия]: Устройство может иметь несколько ссылок изделия с соответствующими указаниями о количестве. Идентификация ссылки изделия с изделием осуществляется через номер изделия.
*[Схемы выводов устройства]: Схема подключения представляет целевую информацию о выводе устройства. В этом смысле вывод устройства представляет собой, например, клемму по отношению к клеммнику, контакт штекера по отношению к штекеру и т. д. Если под выводами устройства подразумеваются клеммы, то они отображаются на схеме подключения клемм. Если речь идет от контактах штекера, то они отображаются на схеме подключения штекера. Если речь идет об устройствах общего назначения, то они отображаются на схеме подключения устройства. Если речь идет о конечных точках соединений, то они отображаются на схеме подключения кабеля.
*[Загрузить]: Для наличия и использования сценариев в Eplan (см.Сценарии) необходимо их загрузить.
*[Опции]: Опции являются частями проекта, которые можно показать или скрыть по своему усмотрению. Опция состоит из одного или нескольких фрагментов, т.е. из фрагментов схемы соединений, страниц или неразмещенных функций и соединений. Все фрагменты опции включаются и выключаются только все вместе.
*[Настройки для всего проекта]: Настройки для всего проекта действуют в рамках всего проекта, например нумерация зон.
*[Проектные настройки]: Настройки, действительные для определенного проекта, являются специфическими для проекта.
*[Эксклюзивный доступ]: Это право обеспечивает доступ к проекту только (исключительно) одному пользователю. Проект не может быть открыт двумя пользователями одновременно.
*[Add-ons]: Дополнительные модули — это наборы функций или блоки данных, расширяющие стандартный комплект поставки варианта продукта Eplan дополнительными функциями или данными, не входящими в этот вариант продукта. Имеются в виду дополнительные модули, как предоставленные Eplan, так и созданные пользователем самостоятельно. Наборы функций дополнительных модулей Eplan в большинстве случаев также доступны в качестве самостоятельной версии. Например, Eplan Fluid доступен в качестве самостоятельной версии, но его также можно приобрести как дополнительный модуль к Eplan Electric P8 Professional.
*[Управление правами]: С помощью управления правами системный администратор присваивает пользователям права доступа к файлам и настройкам.
*[Определения кабеля]: Кабели представлены в Eplan посредством определений кабеля, они могут быть представлены графически с помощью линии определения кабеля или экранирования.
*[Группы проектов]: Группы проектов упрощают одновременную работу с несколькими проектами. Присвоив проект определенной группе проектов, можно указать для него конкретный цвет контура в навигаторах данных проекта для лучшего распознавания. Кроме того, для входящих в группу проектов можно отображать дополнительные навигаторы, например второй навигатор страниц.
*[Проекты макросов]: Проект макросов – это определенный вид проектов, который используется для управления и упрощенного создания макросов. Этот вид проекта определяется в свойствах проекта. По сравнению с проектами схем соединений проекты макросов имеют ограниченные логические функции (например, отсутствуют соединения для всех страниц).
*[Номера свойств]: Номер свойства – это идентифицирующий номер для какого-либо свойства или настройки в Eplan.
*[Язык проекта]: Используемый в проекте язык называется языком проекта. Язык проекта может быть одним из языков словаря, а также может быть языком, который не содержится в словаре. Пример: Проект переведен на испанский и отправлен клиенту. Но в словаре клиента есть только английский и немецкий языки. В данном случае язык проекта не является языком словаря.
*[Базовые проекты]: Базовые проекты являются шаблонами для новых проектов; они могут содержать настройки проекта, проектные данные, страницы и основные данные. Базовые проекты имеют расширение имени файла zw9. При создании проекта на основе базового в новый проект из базового переносятся как настройки проектов, данные проектов, страницы, так и все сохраненные и исходные основные данные, на которые идет ссылка.
*[Файл проекта]: Файл проекта представляет собой проект и содержит ссылку на соответствующий каталог с данными проекта.
*[Ревизии]: Ревизия обозначает в Eplan коллекцию изменений в пределах проекта, которые сохраняются под определенным именем — именем ревизии.
*[Внешние документы]: Внешние документы — это документы, загруженные в проект с помощью типа страницы "Внешний документ". При этом ссылка на внешний документ указывается в свойствах страницы.
*[Присоединенные документы]: Присоединенными называются внешние документы, вставленные на страницы схемы соединений при помощи гиперссылок или внесенные в используемые изделия. Чтобы присоединенные документы могли быть распечатаны, файлы этих документов должны находиться на локальном или на сетевом диске, к которому у вас есть доступ.
*[ГОСТ]: ГОСТ – это российский стандарт.
*[Основные данные проекта]: Используемые и сохраненные в проекте основные данные называются основными данными проекта.
*[Слои]: Один слой — это "пленка", на которой находятся графические элементы (соответствует Layer в AutoCAD). Все видимые слои составляют чертеж. В графической обработке слои используются для скрытия элементов, которые не интересуют пользователя на данный момент.
*[Выгрузить]: Сценарии (см.Сценарии) выгружаются, если они больше не нужны в Eplan.
*[База данных управления проектами]: Вся информация, которая выводится на просмотр в управлении проектами, находится в базе данных управления проектами.
*[Основные данные системы]: Основные данные системы – это основные данные для нескольких проектов, которые хранятся в централизованном пуле основных данных. Эти основные данные можно изменять с помощью функции обработки основных данных.
*[Контрольные прогоны]: Eplan позволяет выполнить логическую проверку страниц схемы соединений, что очень помогает при проектировании. Для этого доступны отдельно вызываемые контрольные прогоны, которые позволяют проверять выбранные страницы или все страницы проекта на наличие в них недостатков или ошибок. Сообщения, сгенерированные при контрольном прогоне, сохраняются так называемым управлением сообщениями в базе данных и дополнительно отображаются в диалоговом окне сообщений.
*[Отчеты]: Отчеты представляют собой запрос данных проекта. Отчеты генерируются автоматически и выводятся на страницах или в файлах экспорта. Сюда относятся, к примеру, спецификации клеммников, списки обозначений устройств и содержания.
*[Предшествующий знак]: Каждому блоку идентификаторов присваиваются определенные предшествующие знаки. Предшествующие знаки до блоков идентификаторов "Функциональное присвоение" и "Определено пользователем" фиксированы и не изменяются. Могут использоваться следующие предшествующие знаки: "==" ("#")("≠") Функциональное присвоение, "=" Функциональное назначение, "++" Место сборки, "+" Место установки, "&" Вид документа, "Своб." Определено пользователем.
*[Маркировка продукции]: Маркировка продукции соответствует блоку идентификаторов для устройства (предшествующий знак "-"), она может дополнительно использоваться в структуре проекта (см.Структура проекта) в качестве элемента структурирования.
*[Схемы идентификаторов]: В Eplan структура проекта определяется с помощью схем идентификаторов. Для страниц и устройств можно выбирать независимые друг от друга схемы. Также можно составлять и применять собственные, определенные пользователем схемы идентификаторов.
*[Полное ОУ]: Полное ОУ (обозначение устройства) содержит описывающие и идентифицирующие структурные идентификаторы.
*[Видимое ОУ]: Видимое ОУ – это часть обозначения устройства, которая отображается на схеме соединений.
*[Вид документа]: Блок идентификаторов "Вид документа" служит для обозначения отдельных видов документов для их различия по информационному содержанию. Этот блок идентификаторов используется в структурах проектов согласно системе обозначения для электростанций (KKS). В Eplan встречаются следующие предшествующие знаки: "&" или " пробел ".
*[Блоки идентификаторов]: В "блоках идентификаторов " объединяется взаимосвязанная информация об обозначении страниц, устройств и функций. В Eplan имеются следующие идентификаторы для структурирования проектов:Функциональное присвоение Функциональное назначение Место сборки Место установки Вид документа Определено пользователем.
*[Блоки]: Блок – это группа выбранных объектов, свойства которых требуется обработать. Выбор может включать объекты того же или разных типов.
*[Место сборки]: Блок идентификаторов "Место сборки" обозначает топографическое место установки устройств или технического оборудования, например, здание, цех, помещение и т.д. Возможный предшествующий знак в Eplan – "++".
*[Обозначения местоположения]: Обозначения местоположения соответствуют черным ящикам только с точки зрения графических свойств и могут иметь ОУ, однако не являются устройствами. Кроме того, не допускается наличие у них номера изделия. Все, что находится в пределах обозначения местоположения, присваивается другому блоку идентификаторов, не идентичному указанному в свойствах страницы. В этом случае условные обозначения находятся в таком же отношении к обозначению местоположения, как в других случаях — к странице.
*[Черные ящики]: Черные ящики используются для оотображения устройств, для которых отсутствуют символы.
*[Определения клеммников]: Определение клеммника идентифицирует клеммник и содержит обозначение устройства клеммника. С помощью определения клеммника возможно также создавать клеммники без клемм.
*[Штекеры]: В качестве штекера обозначается комбинация нескольких контактов штекеров. Штекер может содержать только штыри, только гнезда или оба вида контактов. При раздельном управлении штырями и гнездами штекер содержит, как правило, только штыри, а обратный эквивалент с гнездами (в Eplan обозначен так же, как штекер) встраивается в устройство. Если осуществляется общее управление штекерными штырями и гнездами, штекер в этом случае рассматривается как один элемент. Штекер – это главная функция; к нему могут задаваться определения устройств и их изделия. Контакты штекера — это вспомогательная функции.
*[Определения штекеров]: Определение штекера идентифицирует штекер и, как правило, содержит обозначение устройства этого штекера. С помощью определения штекера можно создать штекер без штекерных контактов.
*[Кабельные соединения]: Кабельное соединение – это соединение, которое может быть составным элементом кабеля. Кабельное соединение обычно представляет собой жилу или, в случае гибридных кабелей, световод или шланг.Чтобы соединение в Eplan могло быть присвоено кабелю, свойству этого соединения Соединение: Принадлежностьдолжно быть присвоено значение "Кабель".
*[Классы сообщений]: Класс сообщений используется для подразделения сообщений контрольного прогона на общие тематические блоки, например на клеммы, штекеры, ПЛК и т.д.
*[Специфическая для проекта информация]: Информация, которая действительна только для этого проекта, является специфической для проекта.
*[Структура проекта]: В системе Eplan под "структурой проекта" понимается объединение всех использованных в проекте структур идентификаторов для объектов, страниц, устройств и функций. Все объекты проекта (страницы, устройства и функции) должны быть обозначены и должны образовывать в рамках проекта иерархическую структуру. Иерархическая структура проекта облегчает идентификацию и локализацию отдельных объектов внутри проекта.Проекты структурируются с помощью блоков идентификаторов, которые можно определить в Eplan с помощью схем.
*[Подидентификатор]: С помощью данных по стандарту DIN определенные блоки идентификаторов в Eplan могут иметь один или несколько подидентификаторов. Подидентификатор всегда подчинен главному идентификатору (см.Главный идентификатор). Подидентификаторы образуют иерархию с соответствующим главным идентификатором.
*[Главные идентификаторы]: Главные идентификаторы структуры проекта не зависят друг от друга. Главные идентификаторы упоминаются в проекте только один раз, но могут использоваться в различных местах. Изменение главного идентификатора влияет на все выбранные и нижестоящие объекты во всем проекте.
*[Функциональное присвоение]: Блок идентификаторов "Функциональное присвоение" отражает связь между установками, техническим оборудованием и устройствами. В Eplan встречаются следующие предшествующие знаки "==", или "#", или "≠".
*[Номер установки]: Блок идентификаторов "Номер установки" используется для структурирования устройств Fluid. Эта идентификация необходима в том случае, если проект содержит несколько установок.
*[Разделы]: Раздел определяет, какой метод используется или должен использоваться. В Eplan проводится различие между такими разделами, как гидравлика, пневматика, охлаждение, смазка, смазочно-охлаждающая жидкость и газовая техника на основе Fluid-техники, а также электротехника, механика или технология производственных процессов. На функциях из Fluid-техники соответствующий раздел Fluid-техники можно выбрать из списка свойства. В Соединениях раздел определяется автоматически по целям подсоединенных функций.

*[Устройства общего назначения]: Устройства общего назначения – это все устройства, не имеющие отдельных вкладок в диалоговом окне "Свойства" для обработки свойств, например предохранители, двигатели, контакторы и т.д. К остальным (специального назначения) устройствам относятся, например, клеммы, штекеры и кабели; для них имеются специальные возможности обработки и отчетов.
*[Варианты изделий]: Вариант изделия представляет собой набор технических свойств изделий. Изделие может иметь различные технические характеристики (напр., конфигурируемые контакторы). Для каждой характеристики имеется вариант, который управляется через однозначное обозначение.
*[Сети соединенных сегментов]: Пространство листа:Сеть соединенных сегментов состоит из соединенных друг с другом сегментов маршрутизации, кривых, соединительных отверстий для проводов и областей маршрутизации. В сети соединенных сегментов маршрутизируются кабели и соединения.Топология:Сеть соединенных сегментов состоит из соединенных друг с другом сегментов, точек маршрутизации и функций топологии. В сети соединенных сегментов маршрутизируются кабели, шланги, трубы и соединения.
*[Топология]: Под топологией в Eplan понимается область внутри двухмерной обработки схемы соединений, в которой планируются сети соединенных сегментов и маршрутизируются соединения. При этом можно маршрутизировать любые соединения, кабели, шланги и трубы. Задайте для этих соединений трассы маршрутизации внутри топологии (см.Трассы маршрутизации), по которым должны проходить соединения.
*[Маршрутизируемые соединения]: Маршрутизируемые соединения — это соединения с видом представления "Трехмерный чертеж монтажных поверхностей", которые проложены в пространстве листа или соединения с видом представления "Топология", которые проложены в топологии. Маршрутизируемое соединение имеет все свойства, которые есть у многополюсного соединения, и дополнительно обладает информацией о своей трассе маршрутизации.
*[Определения сегментов]: Определение сегмента определяет вид сегмента (т. е. сегмента структуры или объекта планирования) так же, как определение функции устанавливает вид функции. Определение сегмента представляет собой предварительное определение находящихся в распоряжении системы сегментов; оно определяет, какие сегменты могут использоваться в проекте и каково стандартное поведение этих сегментов.
*[Вспомогательные функции]: Вспомогательные функции представляют функции устройств, представленных распределенно (например, контакты катушки).
*[Главные функции]: Главная функция представляет устройство для распределенно представленных устройств (например катушка контактора). Только главная функция может содержать, например определение устройства для устройства. Поэтому на одно устройство приходится только одна главная функция.
*[Конфигурации ПЛК]: Конфигурация ПЛК — объединение всех данных файла обмена (файла конфигурации ПЛК) из программы конфигурации ПЛК. Конфигурация ПЛК содержит информацию о том, какие элементы сети связаны между собой, какие карты ПЛК (тип) располагаются на каких разъемах, а также какие адреса располагаются на каких картах ПЛК.
*[Контуры]: Контур обозначает геометрический контур или линию контура тела. В технике ЧУ контур является управляемой программным обеспечением длиной пути пускового режима станка машины ЧУ, который, например, генерирует фрезерованный прямоугольный фрагмент в монтажной плате.
*[Выводы устройства]: Выводами устройства являются, например, место резьбового вывода устройства штыря в штекере, вывод устройства ПЛК, места резьбовых выводов устройств U/V/W на двигателе, конец кабельного соединения. Логические данные выводов устройства содержатся в определении функции. Выводы обозначены номерами по порядку от 1 до n и графически представлены с помощью точек выводов устройства на символе.
*[Блоки ПЛК]: Блок ПЛК служит для отображения устройств ПЛК. Графическое представление соответствует представлению черного ящика, однако имеются функциональные различия.
*[Контакты штекеров]: Контакт штекера представляет собой точку контакта. В Eplan со штекерным контактом можно работать как с одной функцией (комбинация штырей и гнезд), так и с обеими функциями по отдельности (одна для штыря, другая — для гнезда).
*[Размещенные устройства]: Размещенное устройство является условным обозначением и представляет собой часть устройства, начерченную на одной странице.
*[Объекты-заполнители]: Объекты-заполнители могут быть присвоены многим объектам. Свойства этих объектов можно присвоить переменным. Объект-заполнитель представляется в Eplan со специальным символом-"якорем" и хранится на отдельном слое EPLAN322, "Символьная графика.Макрос.Объект заполнителя".
*[Рамки макросов]: Рамка макроса – это вспомогательное средство для генерирования и обработки макросов в проектах макросов, а также для дополнительной обработки макросов в проектах схем соединений. Полной функциональностью рамок макросов можно воспользоваться только в проектах макросов: с помощью рамок макросов в таких проектах определяются контуры и данные автоматически генерируемых макросов.
*[Идентификаторы сортировки]: Идентификаторы сортировки существуют для точек разрыва, для контактов штекера и для клемм. Идентификатор сортировки определяет последовательность, в которой выполняется сортировка точек разрыва внутри проекта. Идентификатор сортировки присваивается в Eplan автоматически и задается последовательностью точек разрыва в диалоговом окне их сортировки, но может изменяться вручную. Стандартный идентификатор сортировки – 100, т.е. сортировка производится только по странице, строке и столбцу. "0" в этом поле определяет точку разрыва как источник. В случае одинакового идентификатора сортировки точки разрыва сортируются в той последовательности, в которой они представлены в схеме соединений.Для контактов штекера идентификатор сортировки определяет последовательность контактов внутри штекера.Для клемм идентификатор сортировки определяет последовательность клемм внутри группы клемм. Последовательность групп клемм внутри клеммника определяется положением устройства (см.Положения устройства).
*[Выводы устройства ПЛК]: Вывод устройства ПЛК соответствует выводу устройства, но содержит дополнительную информацию (напр., символический адрес). Вывод устройства ПЛК вносится в схему соединений как обычная клемма. Однако вывод устройства ПЛК характеризуют только свойства вывода устройства, т. к. это не отдельный функциональный элемент, а подсоединение к карте ПЛК. Обозначение вывода устройства всегда присваивается карте ПЛК.
*[Точки определения сети]: Точка определения сети служит для того, чтобы задать свойства сети, а также определить сетевые соединения. В точке определения сети сохраняется, какой вывод устройства сети соединен с каким другим выводом устройства сети. Точка определения сети может содержать любое количество шлейфов.
*[Символы соединения]: Символы соединения используются для отображения профиля соединений. К ним относятся уголки, тройники и перекрестные соединения.
*[Размещения изделий]: Размещение изделия представляет собой изделие, размещенное на странице компоновки электрошкафа. На число штук изделия возможно одно размещение.
*[Обозначения выводов устройства]: Обозначение вывода устройства позволяет идентифицировать отдельные выводы какой-либо функции. Можно использовать буквенно-цифровые обозначения.
*[Адреса ПЛК]: Адрес ПЛК – это физический (реальный) адрес ПЛК. Адрес ПЛК всегда присвоен блоку управления ПЛК. Адрес ПЛК не зависит от обозначения вывода устройства.
*[Виды представления]: Размещенные функции различаются в многополюсных, однополюсных, графических и внешних функциях, в парных перекрестных ссылках и обзорных функциях. Это различие обозначается как вид представления. Дополнительно для функций и макросов, которые должны быть размещены в блок-схемах рабочего процесса, доступен вид представления "Функциональный". Для объектов в пространстве листа имеется вид представления "Трехмерный чертеж монтажных поверхностей", а для объектов в топологии — вид представления "Топология".Функция может использоваться в проекте одновременно в различных видах представления, однако каждый вид представления можно использовать только один раз для каждой функции. Вид представления влияет на действие функции в отчетах.
*[Выводы шины]: Выводы устройства ПЛК с определениями функций категории "Кабельный вывод сети / шины" в Eplan сокращенно называются выводами шины. Такие выводы устройства по умолчанию соединяют сети и передают сигнал. Они используются для проектирования выводов шины и выводов сети. Вывод шины представляет собой штекер со всеми контактами штекера и в базе данных изделий изображается только в видеодногошаблона функции.
*[Словари]: Словарь – это набор ключевых слов с текстами перевода на одном или нескольких иностранных языках.
*[Форматы]: Форматы используются для обозначения объектов по единой схеме. Например, форматы применяются для маркировочных этикеток и нумерации устройств. Формат состоит из одного или нескольких элементов формата (см.Элементы формата).
*[Системные сообщения]: Системные сообщения – это сообщения, касающиеся системы. Каждому системному сообщению присваивается шестизначный номер сообщения.
*[Генерировать]: В Eplan термин "Генерировать" означаетавтоматическоесоздание объектов (например, соединений) или отчетов (например, спецификаций клеммника, спецификаций кабелей, перечня материалов и т.д.).
*[Рабочие станции]: Под "станцией" понимается рабочая станция или ПЛК пользователя.ПЛК:Рабочие станции являются подмножествами устройств ПЛК внутри проекта конфигурации (см.Проекты конфигураций).
*[Перекрестные ссылки]: Перекрестная ссылка обеспечивает надежный поиск условного обозначения на множестве страниц. По этой причине перекрестная ссылка должна обозначать как минимум искомую страницу. Дополнительно она может содержать данные о столбце и строке для ориентации в пределах страницы.
*[Определения функций]: Определение функции является предварительно созданным определением имеющихся в распоряжении системы функций. С помощью определения функции указывается, какие функции могут быть использованы в проекте и каковы стандартные характеристики этих функций. Таким образом, определения функций определяют, как объект будет обрабатываться при генерации отчетов (например, контакты в образе контакта, клеммы в спецификация клеммника и т. п.). Управление определениями функций осуществляется в библиотеке определений функций (см.Библиотеки определений функций). Символам приписаны определения функций, которые автоматически заносятся в условное обозначение при размещении символов. Пользователь может ввести в изделия определения функций, сделав, таким образом, доступным выбор устройства.
*[Типы объектов]: Тип объекта объединяет группу объектов, свойства которых могут быть обработаны в общем диалоговом окне "Свойства". Свойства объектов типа объекта могут различаться. Типы объектов – это, например, устройства общего назначения, клеммы, кабели и тексты.
*[Функциональные тексты]: Функциональный текст относится к любому адресу. Этот текст можно вводить вручную на адресе ПЛК или автоматически копировать из зоны или с подключенного устройства.
*[Зоны]: Рамки подразделяются на несколько зон, чтобы быстрее находить размещенные условные обозначения. Положение зоны (горизонтально или вертикально) зависит от стандартов.
*[Шаблоны функций]: Шаблоны функций используются в качестве образцов для тех функций, которые присутствуют в проекте. Сами шаблоны не используются в проекте и не имеют графического представления в схеме соединений. Для шаблона функции нельзя ввести какую-либо дополнительную информацию, это возможно только для функции или условного обозначения. Шаблоны функций задаются посредством данных устройства (изделий). Т. е. шаблоны функции не являются частью проекта, а относятся к изделию.
*[Принадлежности]: При выборе устройства изделия разделяются на главные изделия и принадлежности (см.Главное изделие). В базе данных изделий с помощью соответствующей настройки задается, идет ли для изделия речь о принадлежности.
*[Закрытые проекты]: Закрытый проект защищен от записи и сохраняется с расширением имени файлов *.elr. В Eplan любые проекты закрываются и защищаются от записи, даже если они не содержат информацию по ревизии.
*[Исходные проекты]: Исходный проект — это защищенная от записи копия проекта, созданная в определенный момент времени. Исходный проект сохраняется с расширением имени файла *.elr и, таким образом, является закрытым проектом. Изначальный проект *.elk остается при генерировании исходным проектом. Дополнительно генерируется копия проекта как закрытый проект *.elr. Вы работаете в исходном проекте дальше и в определенный момент выполняете сравнение с исходным проектом.
*[Страницы отчетов]: Страница отчета представляет собой страницу проекта с результатом отчета.
*[Блоки отчетов]: Блок отчетов это совокупность связанных друг с другом отчетов. Все отчеты, относящиеся к одному блоку отчетов, имеют одинаковую начальную страницу.
*[Функциональные элементы]: Функциональный элемент — это часть устройства, которая не может быть разделена на дальнейшие части, например, двигатель, клемма, штекер, вспомогательный блок, кабель. Функциональному элементу может быть присвоен номер изделия; кроме того, он может быть представлен графически на монтажной плате. Функциональные элементы определяются через понятие изделия: функциональный элемент – это изделие, которому посредством шаблонов функций (см.Шаблоны функций) присвоены определенные технические свойства.
*[Группы пользователей]: Группа пользователей – это совокупность пользователей, зарегистрированная в управлении правами, которой были присвоены определенные права доступа.
*[Образ контакта]: Образ контакта представляет собой специальную форму представления перекрестных ссылок, показывающую все контакты устройства. В образе контакта также отображаются неразмещенные функции и функции устройства (шаблоны функций).
*[Сообщения контрольного прогона]: Сообщения контрольного прогона генерируются в ходе проверки во время обработки проекта или при выполнении отдельно запущенного контрольного прогона и выводятся в диалоговом окне управления сообщениями.
*[Компоновка электрошкафа]: Тип страниц "Компоновка электрошкафа" используется для чертежа компоновки электрошкафа (в более ранних версиях Eplan назывался "Оснащение монтажной платы").
*[Функциональная схема автоматизации]: Тип страницы "Функциональная схема автоматизации" используется для планов ТС и функциональных схем автоматизации в предварительном планировании.
*[Блоки AutoCAD]: Блоки AutoCAD представляют собой символы в файле DXF/DWG.
*[Макросы окон]: Макрос окна – это макрос, который содержит определенную область какой-либо страницы или все объекты, находящиеся на странице. Однако такой макрос не содержит данные заголовка страницы. Макросы окон при вставке добавляются к курсору и их можно произвольно позиционировать в направлении X и Y.
*[Спецификации]: Спецификации – это специфические для проекта списки, которые анализируют данные об изделиях, причем учитываются устройства с изделиями, устройства без изделий и покупные изделия.
*[Тексты свойств]: Тексты свойств используются с целью представления свойств условного обозначения для соответствующего символа в схеме соединений. При создании символа для каждого из них указывается, какие тексты свойств по умолчанию отображаются при вставке этого символа.
*[Модули]: Модуль является коллекцией изделий, относящихся к нескольким вложенным устройствам. В базе данных изделий рядом с каждым изделием модуля указано, к какому устройству модуля принадлежит данное изделие. Модулю присвоен собственный номер изделия. Он может содержать изделия, узлы и другие модули.
*[Языки отображения]: Языки отображения представляют собой языки, на которых переведенный текст проекта отображается на страницах соответствующего проекта.
*[Отверстия]: Отверстия представляют собой круглую форму монтажных отверстий разного диаметра. Отверстия генерируются вручную по шаблонам сверления или автоматическим сверлильным станком с ЧУ.
*[Видимая область страницы]: Видимая область страницы отображается в графическом редакторе и выводится при печати или преобразовании в формат PDF. Эта область определяется в зависимости от того, присвоена ли странице рамка: если страница не имеет рамки, то видимую область образуют все элементы, находящиеся на странице. Если страница имеет рамку, то видимую область образуют только графические элементы, находящиеся в рамке. Элементы, расположенные вне рамки, игнорируются. Также всегда игнорируются тексты, т. к. используемые специальные тексты иногда могут выходить за края рамки.
*[Технология производственных процессов]: Технология производственных процессов занимается техническим и экономическим осуществлением всех процессов, в которых изменяется вид, свойства и состав материалов. В частности, подразделмеханическаятехногогия производственных процессов посвящена процессам преобразования материалов, основанным на механическом воздействии. Четыре основные группы процессов — измельчение и агломерация, а также смешивание и разделение (фильтры, сита). Противоположностью этому являетсятермическаятехнология производственных процессов, важнейшими процессами которой являются дистилляция и описываемые подобными методами процессы экстракции и абсорбции.
*[Центр вставки]: Центр вставки — это диалоговое окно, с помощью которого можно вставлять разные объекты, такие как символы, устройства, а также макросы окон и символов из центрального хранилища непосредственно на страницы проекта или в пространства листов. Диалоговое окно представлено в виде всплывающего окна на правом краю графического редактора.
*[Редакторы]: В редакторах (графический редактор, редактор символов, редактор форм и т.д.) создаются и редактируются кнопки, символы, формы и другие объекты. Эти диалоговые окна можно присоединить и сгруппировать в одном окне вместе с другими диалоговыми окнами.
*[Макросы символов]: Макросы символов упоминаются в качестве дополнения к библиотекам символов. Таким образом, вы можете, например, объединить в одном объекте несколько отдельных символов или объектов, которые составляют логическую единицу. Макросы символов создаются аналогично макросам окна (см.Макросы окон), однако сохраняются с другим расширением файла в том же каталоге.
*[Диалоговое окно Свойства]: В этом диалоговом окне можно редактировать свойства размещенной или неразмещенной функции.
*[Варианты символа]: Для каждого символа имеется восемь вариантов; дополнительно у некоторых символов имеется специальный вариант для представления образа контакта. У символов соединений, кроме того, имеется еще один вариант для представления точки соединения. Варианты символа имеют общие свойства, то есть у них всегда одинаковое количество выводов устройства и они отражают одинаковую функцию.
*[Обрезать]: Обрезка является функциональностью изменения чертежей. Элементы обрезаются у определенных точек, удлиняются или укорачиваются.
*[Закруглить]: "Закруглить углы" — функциональность изменения чертежей. В 2-мерном виде округляется переход между двумя элементами чертежа (линии, круги, дуги окружности).
*[Закругления]: При проектировании изделий закругление реализуется касательной дугой окружности, создаваемой между двумя линейными элементами или между линейным элементом и дугой окружности.
*[Прерывания]: Прерывание – представляет собой невидимый символ без выводов устройства, который прерывает линию автоматического соединения и тем самым предотвращает автоматическое соединение.
*[Фаски]: При проектировании изделий фаска обозначает соединение между двумя линейными непараллельными элементами через прямую. При этом исходные элементы обрезаются (укорачиваются или удлиняются). Различают симметричную и несимметричную фаску. О симметричной фаске речь идет тогда, когда исходные элементы обрезаются на одинаковую величину (ширина фаски). В 3-мерном виде генерируют равномерные скосы на ребрах объектов, углах или внутренних отверстиях (например, отверстия для сверления).
*[Смещение исходной точки]: С помощью смещения исходной точки можно определить опциональную исходную точку координат. На новую исходную точку указывает небольшое координатное перекрестие. В строке состояния указание координат меняется с X / Y на DX / DY (графическая система координат) или с RX / RY в DRX / DRY (логическая система координат), и значения указываются относительно исходной точки.
*[Следующие формы]: Для определенных видов отчетов (например, список обозначений устройств, схема кабельных соединений, перечень клеммников и т.д.) возможно указание следующей формы. Эта форма выводится при формировании отчетов после текущей формы. Последующая форма может быть вставлена либо на той же странице, что и текущая форма, либо на следующей странице; это определяется с помощью точки вставки последующей формы.
*[Специальные тексты]: Специальные тексты – это свойства, которые размещаются в формах, рамках или на страницах схемы соединений. При этом речь может идти о свойствах проектов, страниц или устройств. Эти свойства заполняются соответствующими данными непосредственно при вставке.
*[Разъемы]: Карты ПЛК, как правило, скрыты в каркасе. При этом каждой карте присвоен определенный разъем. Разъемы задаются в Eplan в форме цифр, букв или буквенно-цифровых последовательностей.
*[Отслеживание цели]: Модель отслеживания цели описывает внутреннюю логику символа, т. е. между какими выводами устройств в символе проходит соединение.
*[Групповая обработка]: Под групповой обработкой понимается одновременное изменение свойств нескольких объектов одного типа.
*[Присоединенные тексты]: Присоединенные тексты сохраняют их исходный цвет, размер, вид шрифта или стиль шрифта. однако применяют угол и выравнивание, соответствующие основному тексту. При перемещении эти тексты также ведут себя как блок.
*[Точки обозначения соединения]: Точка обозначения соединения служит для присвоения свойств соединений, проходящих под этой точкой. Настроенные здесь свойства имеют преимущества перед свойствами соединений, предварительно настроенными для потенциала или в настройках проекта. Вся информация, которую могут содержать соединения (включая графическое отображение соединения), может быть определена посредством точки определения соединения.
*[Монтажные платы]: 2D:Монтажная плата – это логическая область на графической странице с типом "Компоновка электрошкафа". Эта область определяется посредством графически изменяемого объекта схемы соединений, который во многом подобен черному ящику.3D:Монтажная плата – это 3D-размещение изделия в пространстве листа. Монтажная плата обладает монтажными поверхностями, на которых можно разместить следующие трехмерные размещения изделий. Если для монтажной поверхности активировано свойство Механическая обработка, размещенные на ней объекты генерируют механические обработки (отверстия), которые могут быть переданы на машины ЧУ для изготовления.
*[Функциональные тексты зон]: Функциональный текст зоны можно произвольно размещать в пределах зоны и анализировать для генерирования перекрестных ссылок ПЛК, спецификации клеммников, прочих спецификаций. Если у условного обозначения нет собственного функционального текста, при создании отчетов используется функциональный текст из зоны.
*[Ломаные линии]: "Ломаной линией" называется взаимосвязанная последовательность сегментов линий. Если начальная и конечная точка ломаной линии соединены друг с другом, речь идет о замкнутой ломаной линии. Ломаная линия в Eplan создается и обрабатывается как единичный объект.
*[Кабельные каналы]: Кабельные каналы – это механические функциональные элементы в электрошкафу. В кабельных каналах упорядоченно группируются провода и кабели, проложенные к подсоединенным устройствам.
*[Выводы потенциала]: Вывод потенциала отображается на схеме так же, как "обычная" клемма. Тем не менее, вывод потенциала описывается только свойствами вывода устройства, так как он не является отдельным функциональным элементом (например, точка соединения с землей на клеммнике).
*[Описания выводов устройства]: Описание вывода устройства содержит дополнительное текстовое описание этого вывода.
*[Структуры сети]: Структура сети – это отображение конфигураций ПЛК (см.Конфигурации ПЛК) на страницах схем соединений, наглядно показывающее, какие элементы сети связаны между собой. Обычно структура сети изображается на однополюсных страницах схемы соединений.
*[Разводки]: Разводка – это совокупность электротехнических соединений, изображенных на схеме соединений, которая представляет собой линии маршрутизации внутри узла или электрошкафа. К ним относятся провода, кабельные соединения, проволочные перемычки, шины.
*[Обзоры установок]: Перечень установок — это структурированный обзор установки со всеми узлами и соответствующими соединениями (кабелями, шлангами, трубами). Часто при этом представляются структурные идентификаторы установки.
*[Обозначения соединений]: Обозначение соединения указывает на конкретное соединение. Обозначение соединений может использоваться в таблицах соединений. Обозначение не ограничивается цифрами, оно может содержать буквы и определенные специальные символы.
*[Жгуты]: Жгут – это совокупность соединений, объединенных между собой. Жгут может содержать кабели, отдельные провода, шланги и/или трубы.
*[Таблицы соединений]: Таблица соединений содержит информацию о технологических контурах, которые при определенных условиях должны блокироваться. "Блокировка" означает намеренное блокирование устройства другим устройством. Блокировка должна предотвращать нежелательные состояния, процессы и события. Таким образом должно предотвращаться выполнение ошибочных действий, которые могут нанести ущерб окружающей среде, техническому оборудованию или здоровью людей.
*[Автоматическое соединение]: Выводы устройства условных обозначений или элементов соединений, которые находятся точно друг напротив друга по горизонтали или по вертикали, соединяются автоматически. Данное действие обозначается как автоматическое соединение, а генерированные таким образом линии соединений называются линиями автоматического соединения. При вставке и перемещении символов отображается предварительный просмотр линий автоматического соединения. Вы можете разделить линию автоматического соединения посредством вставки "Прерывания".
*[Обозначения выводов жгутов]: Обозначение выводов жгута определяет соединение внутри жгута соединений. Это обозначение встречается на одном жгуте дважды: на выводе устройства, который входит в жгут, и на выводе устройства, который выходит из жгута.
*[Провода]: Под "проводом" понимается отдельное электропроводящее соединение, которое на всем своем протяжении может одновременно иметь только один потенциал (формы модуляции исключены).
*[Предварительный выбор изделий]: Предварительный список изделий представляет собой список изделий, который Вы составляете и сохраняете под определенным именем . Этот список можно использовать в качестве фильтра при выборе изделия. Предварительный выбор изделий сохраняется для конкретного проекта.
*[Легенды электрошкафа]: Легенда электрошкафа – это отчет, относящийся к функции, т.е. ОУ монтажной платы должно быть однозначным. Легенда электрошкафа указывает на взаимосвязи между изделиями на схеме соединений, позицию изделия в спецификации и графических объектах или размещениях изделий монтажной платы. При этом все размещения изделий имеют порядковый номер. Легенду электрошкафа можно создать в виде страницы отчета или вставить ее в виде "свободного размещения" на страницу типа "Компоновка электрошкафа".
*[Группа продуктов]: Отдельные части подразделяются на группы продуктов, например контакторы, конденсаторы, измерительные устройства, клапаны и т.п.
*[Маркеры ревизий]: если объекты в одной ревизии изменились, они отображаются в схеме соединений с помощью графических выделений. Изменения свойств отображаются дополнительно с помощью текста маркировки, указанного в свойстве "Маркер ревизии (из отслеживания изменений)" или "Маркер ревизии (из сравнения свойств)" (см. "Тексты маркировки ревизии). В качестве маркера ревизии обозначаются как графические маркеры в схеме соединений, так и текстовые маркеры в свойствах.
*[Тексты маркировки]: Стандартная форма маркировки функциональных элементов является номером позиции в соответствии с лежащей в основе проекта спецификацией. В качестве альтернативы можно переключиться на отображение обозначений устройств.
*[Блок-схемы рабочего процесса]: Блок-схемы рабочего процесса являются простыми графическими представлениями, которые наглядно представляют рабочий процесс установки, машины или устройства. В Eplan можно создавать как простые функциональные диаграммы с ориентацией на текст или символ (см.Функциональные диаграммы), так и диаграммы GRAFCET.
*[Функциональные диаграммы]: Функциональными диаграммами являются простые диаграммы путь-время или путь-шаг, которые наглядно представляют рабочий процесс установки, машины или устройства. В Eplan можно создавать как простые функциональные диаграммы, ориентированные на текст, с размещенными текстами свойств, так и функциональные диаграммы, ориентированные на символы, со вставленными символами устройства.
*[GRAFCET]: Диаграммы GRAFCET — это представления управления рабочим процессом, для которых используется язык спецификации GRAFCET (GRAphe Fonctionnel de Commande Etapes/Transitions) по системе стандартов DIN EN 60848.
*[Макрос изделия]: Макрос изделия представляет собой макрос, присвоенный определенному изделию в управлении изделиями. Если такое изделие сохранено на главной функции, то можно разместить макрос изделия (например, из навигатора).
*[Шаги GRAFCET]: Шаг GRAFCET является элементом диаграмм GRAFCET (см.GRAFCET). Шаг — это статус части рабочего процесса. Для идентификации шаги должны иметь обозначение. Для представления шага в диаграмме рекомендуется использовать квадрат.
*[Действия GRAFCET]: Действие GRAFCET является элементом диаграмм GRAFCET (см.GRAFCET). Одному шагу можно присвоить одно или несколько действий. Действие отображает, что должно происходить с переменной. Подобным образом действие представлено прямоугольником.Переменные можно наглядно объединять в таблице переменных. Связь с устройствами Eplan осуществляется дополнительным размещением обозначения устройства в данной таблице переменных.
*[Прохождения GRAFCET]: Прохождение GRAFCET является элементом диаграмм GRAFCET (см.GRAFCET). Прохождение — это переход от одного шага к следующему. В диаграмме прохождение представлено по большей части линией в правом углу для связи между двумя шагами. Прохождению можно присвоить имя, оно располагается слева и должно стоять в скобках. Справа рядом с прохождением стоит условие прохождения.
*[Макросы страниц]: Макрос страницы – это макрос, который состоит из одной или нескольких "настоящих" страниц, включая данные заголовка страницы.
*[Сервер SQL]: SQL-серверы представляют собой реляционные системы управления базами данных. Данные сохраняются в реляционной базе данных в виде таблиц и могут запрашиваться и обрабатываться посредством языка запроса SQL (Structured Query Language).
*[Набор макросов]: Набор макросов — это своего рода временное хранилище для всех макросов окон и символов, хранящихся в текущем настроенном каталоге макросов. Набор макросов предназначен для оптимизации рабочего процесса при вставке макросов с помощью центра вставки.
*[Ключевые слова]: Ключевое слово – это запись в словаре.
*[Диалоговые окна настроек]: В соответствующем диалоговом окне настроек можно определить все настройки для выбранной области.
*[Покупные изделия]: Покупные изделия непосредственно присваиваются проекту и не относятся к какому-либо устройству. Сюда относятся, напр., изделия для комплектующих деталей, общих установочных материалов и т.д.
*[Гидравлика]: В гидравлических установках энергия переносится жидкостью под давлением по кругу, направляется и регулируется. Часто в качестве рабочей среды используется масло.
*[Пневматика]: В пневматических установках энергия переносится сжатым воздухом по кругу, направляется и регулируется.
*[Номера символов]: Номер символа указывает на позицию символа в пределах библиотеки символов.
*[Обозначения контакта штекера]: Обозначение контакта штекера идентифицирует контакт штекера; можно использовать как числовые, так и произвольные значения, такие как 5, А3, и т.п.
*[Потенциалы]: Потенциал соответствует определенному уровню напряжения в определенный момент времени, он одинаков впереди и позади предохранителя (стандартно). Границы потенциала определяются источником и потребителем. Физические потенциалы по предварительной настройке заканчиваются на потребителе, трансформаторе или преобразователе.
*[Перекрестные ссылки оборудования]: Условные обозначения одинаковых устройств автоматически получают перекрестные ссылки, если они содержат одинаковые обозначения устройства. Чтобы получить безупречный отчет, в перекрестных ссылках оборудования необходимо отметить соответствующие условные обозначения как главную функцию или как одну или несколько вспомогательных функций.
*[Смещение]: Перемещение адресов накопителей, элементов CAD и графики на задаваемое значение называется смещением.
*[Типы изделий]: Согласно графическому представлению изделий имеются следующие различные типы изделий: отдельные части, узлы и модули.
*[Линий определения кабеля]: Определения кабеля служат для графического представления кабелей в схеме соединений. Обычно эти линии проходят ортогонально относительно рассеченных ими линий автоматического соединения (тех, которые образуют кабельные соединения), но их можно начертить и под углом.
*[Несущие шины]: Несущие шины представляют собой конструктивные функциональные элементы в электротехнических и механических проектах и предназначены для крепления к ним устройств. Несущая шина обычно закреплена на монтажной плате или профиле шкафа.
*[Резервуары]: В технологии производственных процессов резервуары относятся к группе аппаратов. Резервуары могут составлять часть установки и вставляться под слой сегментов структуры. Резервуары могут содержать технологические контуры и объекты планирования.
*[Сегменты маршрутизации]: Сегмент маршрутизации – это раздел сети соединенных сегментов. Сегменты маршрутизации – это вставленные в ручном или автоматическом режиме маршруты, вдоль которых можно маршрутизировать соединения. Сегменты маршрутизации могут (но не обязательно) быть выполнены из одного материала и могут иметь изделие.Пространство листа:При автоматическом генерировании сети соединенных сегментов в кабельных каналах автоматически создаются сегменты маршрутизации. Сегменты маршрутизации можно также создать вручную, чтобы расширить сеть соединенных сегментов вне кабельных каналов.Графически сегменты маршрутизации обозначаются синей линией с точками захвата в начале, конце и середине.
*[Точки маршрутизации]: Точки маршрутизации — это особые места в сети соединенных сегментов, такие как кривые, отверстия в стенах, смещение, отверстия в электрошкафу и т.д. Точка маршрутизации служит для перехода соединения к цели, как место разветвления на несколько сегментов или как место соединения сегментов.
*[Сигналы]: Сигнал в Eplan — это подмножество потенциала. Сигнал представляет собой имя некоторого количества соединений, которые непосредственно связаны друг с другом. Сигнал заканчивается на функциях, разделяющих сигнал (т.е. на функции, у которой установлено свойство С разделением сигнала).
*[Главное изделие]: При выборе устройств изделия разделяются на главные изделия и принадлежности (см.Принадлежности). Обычно главные изделия содержат шаблоны функций для указания технических данных, а принадлежности, напротив, нет. В базе данных изделий главному изделию на вкладке Принадлежностиприсваиваются возможные элементы принадлежностей.
*[Определения устройств]: Определение устройства сохраняется в изделии и состоит из набора x шаблонов функций. В каждом шаблоне функции наряду с определением функции указаны идентифицирующие свойства функции (например, обозначение вывода устройства).
*[Специфические для пользователя]: Настройки, действительные для определенного пользователя, специфические для пользователя.
*[Варианты применения]: Данное применение устройства будет обозначено как "Использование". Каждому варианту применения соответствует свой набор параметров, важных с точки зрения безопасности (см.Параметры, важные с точки зрения безопасности). То есть в зависимости от применения устройства должны использоваться различные параметры, важные с точки зрения безопасности. Различные варианты применения могут возникать, например, вследствие различного использования устройства. Например, привод можно использовать с тормозом или без него, что означает и применение различных параметров, важных с точки зрения безопасности.
*[Объекты планирования]: Объекты планирования определяют часть машины или установки в предварительном планировании, они описывают соответствующее устройство и относятся к функциям устройства. При этом устройство представляет функцию машины или установки (например, электродвигатель для ленточного конвейера).
*[Перекрестные ссылки точек разрыва]: Точки разрыва служат для постраничного представления соединений на схеме соединений. Перекрестные ссылки между точками разрыва создаются системой Eplan автоматически, причем перекрестные ссылки представляют ссылку на страницу в схеме соединений. Определяются два варианта перекрестных ссылок точек разрыва: перекрестные ссылки типа "звезда" и "цепь".
*[Парные перекрестные ссылки]: Данная перекрестная ссылка указывает на условное обозначение так называемого "контакта парной перекрестной ссылки" на его "подключенный" в схеме соединений обратный эквивалент. И наоборот: обратный эквивалент указывает на контакт парной перекрестной ссылки (см.Контакты парной перекрестной ссылки). Ссылки на главную функцию в данном случае нет.
*[Перекрестные ссылки монтажных плат]: Перекрестная ссылка монтажной платы указывает, исходя из главной функции на схеме соединений, на соответствующее изделие на монтажной плате (страница типа "Компоновка электрошкафа") и наоборот.
*[Карты ПЛК]: Карты ПЛК – это компоненты аппаратного обеспечения ПЛК (как правило, сменные). Карты ПЛК содержат, например, блок питания, центральное устройство (ЦПУ) или так называемые "Сигнальные узлы" (аналоговые или цифровые узлы входа или выхода). В Eplan реализация карт ПЛК осуществляется графически и логически при помощи блоков ПЛК.
*[Обзоры карт ПЛК]: На страницах обзоров карт ПЛК можно представить описания выводов устройства. В распределенных представлениях карт ПЛК обзоры облегчают поиск входов/выходов на схеме соединений. В обзорах показываются все перекрестные ссылки на выводы устройства ПЛК на схеме соединений.
*[Перекрестные ссылки ПЛК]: Перекрестные ссылки ПЛК относятся к обзорным перекрестным ссылкам. При этом между выводами устройства ПЛК на схеме соединений и выводами устройства ПЛК на страницах перечня создаются перекрестные ссылки.
*[Выдавливание]: В геометрии выдавливание обозначает увеличение измерений двухмерного элемента через параллельный перенос в пространстве. Например, линия выдавливается в плоскость.
*[Списки результатов]: При поиске понятий с помощью функции поиска, результат поиска отображается в списке результатов.
*[Обзоры модели]: Обзоры модели представляют собой стандартизованные виды, такие как виды оснащенных монтажных поверхностей. Обзоры моделей служат для отображения и создания чертежей. Дополнительные сведения, такие как указание размеров, тексты и т. д. для производства электрошкафов (оснащение электрошкафов), можно обозначить стандартными функциями платформы в обзорах моделей.
*[Трубопроводы]: Понятие "трубопроводы" используется в Eplan для соединений fluid-техники, определенных как устройства. В гидравлике и пневматике такими соединениями, к примеру, являются предварительно изготовленные трубы или шланги.
*[Свойства блока]: Свойства блока объединяют несколько прямых и / или косвенных свойств, Эти свойства можно форматировать и просматривать совместно.
*[Косвенные свойства]: Косвенные свойства – это свойства объекта, связанного с исходным объектом. Через эти свойства блока (см.Свойства) можно запрашивать в определенном объекте (исходном объекте) свойства другого, связанного с ним объекта.
*[Тексты заполнителей]: Тексты-заполнители подобны специальным текстам, однако они используются в формах, чтобы определить позиции и форматирование соответствующих свойств. При обработке формы тексты-заполнители автоматически заменяются на соответствующие значения конкретных объектов проекта, например, устройств, страниц, символов и т.д.
*[Проводной монтаж]: Проводной монтаж — это расположение проводов и кабелей, а также других компонентов (клемм, контактов штекеров и т. д.). Кабели связываются в кабельный жгут, например с помощью скоб, хомутов и т. д., комбинируются в проводной монтаж на специальном станке или формовочном шаблоне и скрепляются скобами между собой.
*[Сегменты структуры]: Сегменты структуры описывают часть структуры проекта в предварительном планировании. Один сегмент структуры может замещать один или несколько структурных идентификаторов (обычно функциональное назначение) в детальном проектировании.
*[Функции ТК]: Функция ТК описывает (под)функцию технологического контура. При этом имеется в виду функция измерения или функция потребления.
*[Язык диалогового окна]: Язык диалогового окна – это язык, на котором отображается интерфейс Eplan.
*[Языки перевода]: Язык перевода – это язык, на который переведен текст проекта.
*[Главные клеммы]: Главная клемма представляет группу клемм на клеммнике. Устройство может содержать любое количество главных клемм, но только одну главную функцию. Поэтому клеммник определяется как главная функция, а все клеммы — как главные клеммы. Если клемма определена как главная клемма, тогда она ведет себя аналогично главной функции. Например, для главной клеммы в навигаторе клеммников отображаются шаблоны функций, и можно выполнить выбор устройства.
*[Сборные шины]: Сборные шины представляют собой конструктивные функциональные элементы в электротехнических и механических проектах, они служат для приема устройств.
*[Распределитель проводов]: Распределители имеют несколько выводов устройства и служат для распределения или разветвления соединений. Во Fluid-технике и технологии производственных процессов распределители служат для распределения гидравлической среды в установке Fluid-техники. В электротехнике через распределители проводов осуществляется разветвление соединений или кабелей. В Eplan распределители графически представлены в виде логических символов соединения и управляются как устройства общего назначения. Т. е. распределителям могут быть присвоены данные устройства, символа, функции и изделия.
*[Соединитель]: Соединители служат для соединения соединений между собой. В Fluid-технике соединители служат для адаптации различных соединений (шланговых / трубных сочленений, поперечных редукций) в одной установке Fluid-техники. В Eplan соединители графически представлены в виде логических символов соединения и управляются как устройства общего назначения. Т. е. соединителям могут быть присвоены данные устройства, символа, функции и изделия.
*[Динамические символы]: Динамические символы имеют переменное расширение. Примерами динамических символов являются линия определения кабеля, экранирование, черный ящик или блок ПЛК.
*[Сетевые соединения]: Сетевые соединения представляют собой соединения, определяемые на точке определения сети. Соединения при этом определяются в форме шлейфов (см.Шлейфы).
*[Обработка концов проводов]: Обработка концов проводов указывает, каким образом необходимо выполнить подготовку концов кабелей с помощью сборочных машин (снять изоляцию, обрезать, обжать, формат конечной гильзы и т.п.).
*[Длина зачистки]: Длина зачистки обозначает длину, на которую удаляется изоляция кабеля. В случае соединения длина зачистки обозначает длину, на которую удаляется изоляция соединения. Эти данные потребуются при переносе на сборочные машины.
*[Группы соединений]: В Eplan группой соединений в Fluid-технике является объединение нескольких соединений групп (см.Соединения групп). Эти соединения групп могут быть шлангами, а также другими соединениями. В большинстве случаев речь идет о группах соединений шлангов. Группы соединений и соединения групп управляются аналогично кабелям и кабельным соединениям.
*[Использование имеющегося]: Использование имеющихся — это размещение или присвоение ранее не размещенной функции или функции, предопределенной с помощью шаблона функции. При этом речь не идет о создании новой функции.
*[Маршрутизировать]: Маршрутизация — это создание маршрутизируемых соединений с видом представления "Трехмерный чертеж монтажных поверхностей" (в пространстве листа) или с видом представления "Топология" (в топологии).
*[Жилы]: Понятие "Жила" обозначает составляющую кабеля. Жила – составляющий элемент кабеля, идентифицировать который без совокупности кабеля невозможно. Так как наряду с жилами кабель может также содержать другие соединения (например, световоды или трубки), в Eplan используется общее понятие "кабельное соединение" (см.Кабельные соединения).
*[Прямые соединения]: В Eplan в схеме соединений можно разместить отдельные символы штырей или гнезд штекера с так называемым "Прямым выводом устройства". Если эти символы соединяются таким образом, что выводы устройства "2" находятся друг напротив друга, то в рамках обозначенного штекерного соединения генерируется линия автоматического соединения. Этому соединению автоматически присваивается определение функции "Прямое соединение".
*[Точки маршрутизации структурного идентификатора]: Точки маршрутизации структурного идентификатора — это точки маршрутизации с определением функции "Точка маршрутизации структурного идентификатора топологии". В одной точке маршрутизации структурного идентификатора можно генерировать функции топологии вручную и автоматически. Точка маршрутизации структурного идентификатора представляет подключенные функции. С помощью точек маршрутизации структурного идентификатора можно прокладывать соединения между различными структурами обозначений, не размещая подключенные функции топологии.
*[Шлейфы]: Шлейф представляет собой список выводов устройства, которые соединены в ряд попарно друг с другом. Первый вывод устройства шлейфа является источником первого соединения. Второй вывод устройства шлейфа является целью первого соединения и одновременно источником второго соединения. Соответственно все остальные выводы устройства шлейфа всегда являются целью для предыдущего и источником для следующего соединения, пока не достигнут последний вывод устройства.
*[Свойства кабеля]: Кабели можно идентифицировать с помощью большого количества различных свойств. Для управления, определения и отчета по кабелям используется следующая информация: тип кабеля, число соединений, кодирование цвета соединений, поперечное сечение, экранирование, механическая конструкция и номер производителя и изделия.
*[Типы вывода устройства]: Тип вывода устройства указывает, какой вид соединения (см.Соединения) возможен на каком выводе устройства. Типы выводов соответствуют определениям функций соединения, например жила/провод, перемычка, световод, шланг, труба и т.д.
*[Изделие кабеля]: Изделия кабеля – это изделия, связанные с кабелем; они вводятся на линиях определения кабеля или экранах.
*[Заданные имена]: Эти имена задаются программой, если пользователь в операционной системе не указал в настройках имя для рабочей станции или компании.
*[Типы кабелей]: С помощью понятия "Тип кабеля" обозначается надгруппа какого-либо типа кабеля, к которой могут относиться различные кабели с различными свойствами и характеристиками. Тип кабеля описывает вид, характеристику и свойства соединения кабелей. При этом для обозначения типа кабеля используются, как правило, стандартные буквенные комбинации в сочетании со специфическими для производителя обозначениями.
*[Покупные кабельные изделия]: Покупные кабельные изделия – это специфические для проекта изделия, генерируемые при суммировании длины кабелей, просмотреть которые можно в спецификации изделий.
*[Обмен данными]: При обмене данными осуществляется обмен данными между различными программами, системами или устройствами. Обмен возможен в обоих направлениях.
*[Трассы маршрутизации]: Пространство листа:Трасса маршрутизации – это совокупность сегментов маршрутизации, соединительных отверстий для проводов и областей маршрутизации, через которые проходит соединение, проложенное в пространстве листа.Топология:Трасса маршрутизации – это совокупность сегментов и точек маршрутизации, через которые проходит соединение, маршрутизированное в сети соединенных сегментов (топологии).
*[Прокладывать]: Соединение кабельных соединений или экранирований с контактами штекера или клеммами обозначается как "прокладывание". В Eplan под термином "прокладывание" понимается присвоение соединений, определенных как кабельные соединения, соединениям какого-либо изделия кабеля.
*[Длина трассы]: Топология:Длина трассы – это длина сегмента маршрутизации или точки маршрутизации.
*[Размер присоединения]: Здесь можно указывать размер присоединения для болтовых соединений, соединений с помощью плоского штекера, ленточных выводов, шинных соединений, внутренней резьбы и наружной резьбы, например, диаметр болта ("3 мм) или присоединительную резьбу ("M3" или "G1/8").
*[Распределенные сегменты маршрутизации]: Распределенные сегменты маршрутизации – это сегменты маршрутизации с определением функции "Распределенный сегмент маршрутизации топологии", которые предназначены для представления сегмента маршрутизации в распределенном виде и соединяют по две точки маршрутизации на разных страницах топологии. С помощью распределенных сегментов маршрутизации можно соединить сети соединенных сегментов, которые находятся на разных высотных отметках (например, на разных этажах здания). На распределенных сегментах маршрутизации можно просматривать разность высотных отметок, которая указывает вертикальное расстояние между сетями соединенных сегментов.
*[Типы сегментов маршрутизации]: Для сегментов маршрутизации топологии можно присвоить определенный тип сегмента маршрутизации. Тип сегмента маршрутизации указывает на то, каким образом должна быть выполнена маршрутизация соединений или кабелей в данном сегменте маршрутизации (например, труба, кабельный канал, кабельный лоток и т. д.). При маршрутизации соединений типы сегментов маршрутизации, которые предварительно определены на сегментах маршрутизации, назначаются для проходящих кабелей и соединений и переносятся на них.
*[Каналы]: Канал объединяет несколько взаимосвязанных выводов устройства ПЛК (например, вывод устройства ПЛК и соответствующие точки подвода питания).
*[Степень заполнения]: Отображение степени заполнения создает сообщение о степени заполнения кабельных каналов на участке разделов и узлов. Благодаря этому пользователь знает, в каком месте на данный момент необходимо вручную изменить маршрутизацию, чтобы применить маршрутизацию через каналы с меньшей загруженностью.
*[Кабельные сборки]: Кабельная сборка — это совокупность проводов и кабелей (обычно с пересечениями), которые были связаны между собой либо затянуты в резиновую или пластиковую оболочку и которые можно подключить к электрической цепи. Кабельная сборка, как правило, гораздо проще и меньше проводного монтажа.
*[Библиотеки определений функций]: Библиотека определений функций - это коллекция определений функций. Библиотека определений функций упорядочена иерархически по разделу, области, категории, группе и определению функции. Эта группировка служит для быстрого поиска требуемого определения функции. В диалоговом окне выбора определения функции отображаются в структуре дерева согласно данной группировке.Раздел, например "Электротехника", подразделяется на различные области, например "Катушки и контакты" или "Двигатели".Категория определяет работу системы функции. Это обозначает также, что независимо от категории (например, контакты, защитные устройства, катушки, клеммы), в распоряжение функций представлены определенные свойства. Свойство Адрес ПЛК, например, существует только для функций ПЛК, а свойство Катушка: Напряжение— только для катушек.Группа задает основную функцию соответствующей категории. Так, например, функция "Замыкающий контакт (2 вывода устройства)" является общим определением и уточняется"силовым замыкающим контактом".
*[Клеммы потенциала]: Клеммы потенциала используются, чтобы применить потенциал. В большинстве случаев речь идет о многоуровневых клеммах, уровни которых связаны друг с другом и которые можно произвольно присвоить какому-либо потенциалу.
*[Распределенные клеммы]: Выводы устройств клемм можно представлять раздельно с помощью распределенных клемм. Для любой распределенной клеммы можно представить один или несколько выводов устройств клемм. Число выводов устройств определяется посредством выбора определений функций. Количество изображенных выводов устройства (от 0 до n) любой функции клеммы с условным обозначением может быть любым и зависит от заданных обозначений выводов устройства.
*[Положения устройств]: Положение устройства задает положение группы клемм внутри клеммника, к которому относится клемма. Все клеммы группы клемм имеют одинаковое положение устройства. Порядок клемм внутри группы клемм определяет идентификатор сортировки (см.Идентификаторы сортировки).
*[Направления выводов устройства]: Символы:Свойство на выводе устройства символа графически обозначается стрелкой. Стрелка указывает направление, откуда возможно автоматическое соединение (автоматическое соединение).Пространство листа:Направление вывода устройства указывает установленное посредством вектора направление, в котором маршрутизируемое соединение выходит из вывода функционального элемента.
*[Гнездо розетки]: Гнездо розетки — это гнездовая часть штекера (тип "мама"). Гнезда безопасны при прикосновении. Потенциалы, проводящие ток, должны всегда назначаться со стороны гнезда.
*[Каркас]: Каркас указывает имя ПЛК. Каждая карта, которая относится к ПЛК, должна иметь имя каркаса. Каркасы — это группировки ПЛК-устройств (расположены в определенной последовательности относительно друг друга в открытом корпусе или на монтажной шине), которые связаны между собой шиной задней стенки (задняя панель). Шина задней стенки служит для подачи напряжения и коммуникации связанных устройств между собой.
*[Программы конфигурации ПЛК]: Программа конфигурации ПЛК — это внешняя программа (инструмент) производителя ПЛК для конфигурации и программирования ПЛК.
*[AML]: AML – общепринятое сокращение, обозначающее AutomationML. AutomationML (Automation Markup Language) является незапатентованным форматом данных на базе XML для обмена инженерными данными между различными программами планирования / обработки.
*[Символические адреса]: Каждому адресу ПЛК можно присвоить один символический адрес. Символический адрес можно внести вручную, либо ОУ первой подсоединенной клеммы датчика или исполнительного элемента вносится автоматически.
*[Списки присвоений]: Список присвоений содержит адреса, символические адреса и функциональные тексты ПЛК. В зависимости от производителя список присвоений предоставляется в различных форматах.
*[Источники питания]: Источник питания – источник напряжения для датчиков или исполнительных элементов, которые подключаются к вводу/выводу устройства В/В ПЛК.
*[Группы функций]: Группы функций – это элементы группировки для изделий в управлении изделиями, которые можно использовать для фильтрации при выборе изделия.
*[Многополюсные представления]: В схемах соединений с многополюсным представлением используются условные обозначения с полным числом выводов устройств. Параллельно идущие соединения и кабельные соединения представляются по отдельности.
*[Распределение адресного пространства]: Распределение адресного пространства – это обзор диапазонов адресов всех карт ПЛК, назначенных одному и тому же ЦПУ. Адресное пространство — это количество байтов адреса, которые предоставляет ЦПУ. Диапазон адресов, занимаемый картой ПЛК в адресном пространстве, определяется в Eplan на карте ПЛК путем указания начального адреса и соответствующей длины данных. Карты ПЛК могут иметь отдельные диапазоны адресов для входов и выходов.
*[Проекты конфигураций]: Проекты конфигураций — это подмножества устройств ПЛК внутри проекта Eplan. Для определения проектов конфигураций имеется одноименное свойство блоков ПЛК и выводов шины. Все объекты, которым назначен один и тот же проект конфигураций, связаны друг с другом и представляют собой конфигурацию ПЛК. Возможен обмен данными определенного таким образом количества устройств с программами конфигурации ПЛК от различных производителей ПЛК.
*[Идентифицирующее ОУ]: Идентифицирующее ОУ содержит идентифицирующие структурные идентификаторы.
*[Возврат]: Возврат удаляет только графическое представление условного обозначения в схеме, в то время как данные неразмещенной функции остаются в базе данных. Напротив, при удалении все данные полностью убираются из базы данных.
*[Однополюсные представления]: Однополюсные представления – это упрощенные представления схем соединений, в которых используются условные обозначения с сокращенным числом выводов устройства (один или максимум два). Параллельно идущие соединения или кабельные соединения сокращаются соответственно до одной линии. Также допускается представление соединений и кабелей по жгутам. Однополюсное представление часто используется для обзоров установок.
*[Группы опций проекта]: Опциональная группа используется для группирования опций, которые в рамках проекта исключают друг друга. В опциональной группе можно всегда включить только одну опцию, другие опции при этом автоматически выключаются.
*[Нумерация соединений]: Автоматическое присвоение обозначений соединений называется в Eplan нумерацией соединений. Это также применимо в том случае, когда обозначение соединения состоит не только из номера.
*[Форматы нумерации]: Формат нумерации позволяет пронумеровать устройства в соответствии с единой схемой. Формат нумерации состоит из нескольких элементов формата и указывает на последовательность элементов формата. Для нумерации в режиме подключения и нумерации в автономном режиме используются разные форматы нумерации.
*[Энергоснабжение устройства]: Энергоснабжение устройства – это вывод устройства, с помощью которого устройство снабжается энергией (например, электроснабжение фоторелейного барьера или карты ПЛК).
*[Сохраненные изделия]: Сохраненные изделия являются составной частью проекта и передаются вместе с проектом. Эти изделия сохраняются в отдельной базе данных в пределах проекта, которую можно синхронизировать с центральной базой данных изделий Eplan.
*[Точки определения изделия]: С помощью точки определения изделия можно сохранить изделия, не присвоенные никакому устройству, на графике в схеме соединений. Каждой точке определения изделия (как и любой главной функции) могут быть присвоены максимум 50 изделий.
*[Резервы]: Резерные функциональные элементы — это элементы, не обозначенные на схеме соединений. Резервы при желании могут учитываться в выводе спецификаций. Все виды устройств могут управляться как резервы: Клеммы, кабели, контакторы, двигатели, ПЛК и остальные резервы не представлены ни в схеме соединений, ни на страницах обзора или монтажной плате.
*[Управление структурными идентификаторами]: Управление структурными идентификаторами – это редактор для обработки структурных идентификаторов, определенных в проекте.
*[Смазка]: При смазке смазочный материал под давлением дозировано подводится через распределитель смазки к трущимся участкам. В качестве смазочного вещества используется преимущественно масло и текучая консистентная смазка.
*[Список результатов ревизии]: Список результатов ревизии – это внутренний список, в котором сохраняются результаты сравнения свойств проектов. Этот список сохраняется в базе данных текущего проекта.
*[Временные исходные проекты]: Исходный проект (см.Исходные проекты) можно также создать временно. Такой проект защищен от записи и может быть удален в любой момент.
*[Имена символов]: Имя символа – это идентифицирующее обозначение символа, например буквенное обозначение согласно стандарту. К примеру, "S" может обозначать замыкающий контакт, а "SL" — силовой замыкающий контакт.
*[Точки вставки последующей формы]: Точка вставки последующей формы может быть размещена в другой форме, чтобы разместить на этом месте другой отчет. Благодаря этому можно генерировать блоки отчетов, состоящих из двух разных форм. Все страницы, сгенерированные для одного имени формы, образуют вместе блок отчетов.
*[Типы отчетов]: Типы отчетов указывают, какая группа схожей информации подлежит обработке. Типы отчетов установлены. Типу отчетов присваивается форма. Форма может быть также заменена.
*[Позиции в форме]: Позиция в форме указывает, какой по счету целью отображается клеммная цель в списках отчетов (A, B, C, D).
*[Динамические формы]: В динамических формах может изменяться графика. Например, может изменяться число строк, в результате чего будут отображаться только строки, заполненные данными, но не пустые строки. Другими словами, графика адаптируется к фактическому числу строк. В таких формах верхние колонтитулы, области данных и нижние колонтитулы могут быть установлены в границах одной страницы.
*[Фиксированная графика]: Есть схемы подключения, на которых на каждую клемму дается фиксированная "винтиковая графика". Здесь можно всегда заменить заполнитель произвольной, но фиксированной графикой. Это позволяет, например, автоматически генерировать сеточную разбивку формы.
*[Записи данных ЧУ]: ЧУ расшифровывается как "Числовое управление". Записи данных ЧУ содержат команды для управления станками. При этом команды считываются, преобразуются в производственный процесс, а затем выполняются.
*[Точки определения контура]: Точка определения контура представляет собой специальный объект, через который логическим элементам "Направляющая рейка", "Монтажные поверхности", "Монтажные поверхности ЧУ" и "Размер поля" контуров типа "Выдавить контур" присваиваются свойства.
*[Направляющие рейки]: Пользовательская шина может иметь на передней стороне несколько направляющих реек. Поэтому на графике шины контура размещаются другие логические шины, которые определяют направляющие рейки. Направляющие рейки указываются как точки текста с обозначением "Направляющая рейка". Точка размещения текста выдавливается в трехмерной шине до 'логической шины'. Графически логическая шина отображается при этом как тонкое тело.
*[Монтажные поверхности]: Монтажные поверхности — это поверхности функциональных элементов, на которых могут быть размещены другие компоненты. Монтажные поверхности обеспечивают привязку к точкам крепления шин, каналов или компонентов, которые должны быть на них размещены.
*[Базы данных покупных изделий]: База данных покупных изделий содержит все изделия, сохраненные в проекте Eplan; и является внутренней составной частью проекта Eplan. Изделия в этой базе данных могут синхронизироваться с данными базы данных изделий с помощью функции синхронизации.
*[ETIM]: ETIM означает Модель Электро Технической Информации, и представляет собой модель классификации изделий для электроиндустрии, которая в стандартной форме подробно и наглядно описывает свойства электротехнических изделий. Для этого изделия классифицируются в иерархически-плоской структуре, состоящей из классов изделий с описательными признаками. Это облегчает обзор и поиск отдельных изделий, каждое изделие присваивается одному классу продуктов и описывается в зависимости от класса продуктов, различных объективированных свойств продуктов, таких как размеры или потребление мощности.ETIM также означает стандартизированный формат обмена основных данных изделий. Крупные торговые организации электроиндустрии, организованные в объединение, договорились о едином формате обмена.ETIM является участником eCl@ss и наоборот. Благодаря этому гарантируется согласование обеих моделей классификации.
*[Подразделы]: Раздел может состоять из нескольких так называемых подразделов. Например, раздел "Охлаждение" может состоять из подразделов "Водяное охлаждение" и "Масляное охлаждение". Указание подразделов в Eplan Fluid в качестве дополнительных свойств функции позволяет генерировать очень подробные отчеты.
*[Текстуры]: Текстуры – это рисунки / образцы материалов интерфейса, такие как камень, дерево или стекло, которые применяются в программах рендеринга для поверхностей трехмерных моделей CAD.
*[Точки монтажа]: Точки монтажа определяют точки, поверхности или линии на трехмерных объектах, на которых можно разместить другие компоненты. Точки монтажа подходят для исключающего размещения соответствующей точки захвата на трехмерном объекте. Пример: Размещение центра отверстия несущей шины на центре отверстия основы. Таким образом, точка монтажа является точкой захвата.
*[Категории соединений]: Категория соединения точнее определяет тип вывода устройства соединения, например винтовое подключение, пружинная клемма, внутренняя резьба и т. д.
*[BMEcat]: BMEcat — это основанный на XML стандарт для передачи электронных каталогов продуктов. Формат обеспечивает стандартный обмен данными каталога, а также системами классификации продукции и используется, прежде всего, в немецкоязычном пространстве.
*[Категории сообщений]: Сообщения подразделяются в Eplan на следующие категории: ошибка, предупреждение и указание. Категории сообщений различаются по цвету и пиктограммам. Cистемные сообщения обладают категориями сообщений "Фатальная ошибка".
*[Отображения отверстий]: 2D:Отображения отверстий (2D) показывают данные, релевантные для ЧУ и отверстий, схемы сверления, а также геометрические контуры основных функциональных элементов и изделий в обзоре модели. Функциональные элементы и отверстия в отображениях отверстий можно снабдить указаниями размеров. Отображение отверстия (2D) представляет собой внутренний документ проекта, который может использоваться перед началом производства для синхронизации выведенных отчетов сверления с текущим состоянием обработки в проекте Eplan.3D:Отображения отверстий (3D) показывают в пространстве листа отверстия, генерируемые функциональными элементами из схемы сверления или размещенные вручную на основном функциональном элементе. *[Шасси]: Шасси представляет собой дополнительный функциональный элемент, добавляемый к размещенному электрошкафу.
*[Монтажная сетка]: Монтажные сетки присваиваются отдельным поверхностям трехмерного размещения изделия и при размещении других компонентов отображаются на этих поверхностях. Таким образом, размещение возможно теперь только в точках пересечения линий сетки. Монтажная сетка позволяет увидеть, например, на профилях электрошкафа образцы отверстий, специфических для производителя и типа, внутри которых при монтаже будет выполняться резьбовое соединение.

Таким образом, размер поля определяет величину монтажной поверхности, релевантной для ЧУ, с производственно-технической точки зрения. Поэтому важно, например, закладывается ли монтажная плата в автоматический сверлильный станок как монтажная деталь, или удаляется. С помощью этого размера машине сообщается о величине функционального элемента к обработке.

Соединения в областях маршрутизации не проходят по ходу сегмента маршрутизации, маршрутизация всегда выполняется посредством максимально короткого, прямого соединения. За счет этого можно, например, выполнить разводку на задней стороне монтажной платы. Никакие функциональные элементы не могут находиться в областях маршрутизации или вдаваться в них. Область маршрутизации связана с сетью соединенных сегментов через сегменты маршрутизации, заканчивающиеся на ее краях.

В соединительных отверстиях для проводов автоматически создаются сегменты маршрутизации, т.е. монтажные отверстия для проволоки являются составной частью сети соединенных сегментов.
*[Вектор нормали]: Вектор нормали интерфейса — это математический размер в расчете трехмерных поверхностей, который направлен вертикально к нижней поверхности, вверх или вниз. Вектор нормали определяет, ориентирована ли поверхность по направлению к наблюдателю или от него. В Eplan Pro Panel с помощью вектора нормали определяется, видна ли поверхность или закрыта другими поверхностями, прозрачна ли она или непрозрачна.
*[Оттенение]: Под "оттенением" в трехмерной графике понимают симуляцию интерфейса объекта, например через отображение заполненных цветом плоскостей. В трехмерных модулях Eplan отображаются электрошкаф и его функциональные элементы с заполненными цветом плоскостями.
*[Продольные отверстия]: Продольные отверстия представляют собой геометрические основные формы, состоящие из двух дуг окружности, соединенных друг с другом прямыми линиями. Продольные отверстия являются возможными производственными формами для отверстий для схем сверления.
*[Защита кабеля / шланга]: Защита кабеля или шланга предохраняет кабель и отдельные соединения или шланги от таких внешних воздействий, как тряска, зной, влажность, трение, протирание, излом и т.д. С этой целью часто используются спиральные шланги из пластмассы или металла, тканевые шланги, защитные трубы и т.д.
*[Фазные шины]: Устройства, как правило, связаны между собой с помощью одно- или многофазных шин для оптимизации модели разводки. Эти фазные шины поставляются изготовителем устройства как принадлежности для устройства. Для соединений имеется определение функции "Соединение фазовых шин". Соединение фазных шин является соединением, реализуемым через фазовую шину; оно имеет схожесть с мостовой перемычкой.
*[Точки модификации]: Точки модификации – это точки в сети соединенных сегментов (см.Сети соединенных сегментов), с помощью которых можно менять маршрутизацию соединений. Эти точки указываются при изменении маршрутизации вручную. Путем перемещения точки модификации в другую точку можно изменить трассу маршрутизации соединений. В одной развертке изображается только один функциональный элемент. Кроме внешней формы, на развертке представлены данные о положении мест изгиба. Для этого изображаются линии изгиба и концы изгиба. Далее на местах изгиба записывается информация об угле изгиба и направлении изгиба. Развертка представляет собой техническую документацию для гибочного станка, который выполняет последующие технологические операции на функциональном элементе-шине. Данные изготовления механической обработки медных функциональных элементов используют информацию в развертке и с ее помощью управляют производством на станке с ЧПУ.

Вдоль профиля формируются в основном только прямоугольные контуры; другие формы поперечного сечения присвоить нельзя. При размещении функционального элемента-шины он записывается в навигаторе пространства листа в ступени иерархии "медные жгуты" с названием жгута и номером. *[Установка]: Блок идентификаторов. Обозначает установки, подустановки и техническое оборудование. Возможный предшествующий знак в EPLAN — "=".
*[Описания выводов устройств]: Дополнительное текстовое описание вывода устройства.
*[Обозначения выводов устройств]: Идентифицирующий текст для графического размещения номера вывода устройства (номера по порядку от 1 до n).
*[Изделие]: Изделие является экономическим элементом. Оно содержит как экономические, так и технические данные. Изделие не должно содержать технические функции, если оно не относится к устройству с соответствующими функциями. Посредством изделий также определяются устройства, при этом изделию присваиваются соответствующие шаблоны функций.
*[Базы данных изделий проекта]: База данных, которая содержит все изделия, созданные в пределах проекта EPLAN, является неотъемлемой составной частью проекта EPLAN. Изделия в составе этой базы данных могут синхронизироваться через функцию обновления выбора изделий с данными остальных баз данных изделий.
*[Резервуар]: В технологии производственных процессов резервуары относятся к группе аппаратов. Они могут составлять часть установки и вставляться под слой сегментов структуры. Резервуары могут содержать технологические контуры и объекты планирования.
*[Границы схем сверления]: Вспомогательные средства для определения графических элементов чертежа, относящегося к схеме сверления. С помощью границ схем сверления задаются монтажные отверстия, данные по размещению и значения размеров соответствующих схем сверления.
*[Запретные зоны для сверления]: Запретные зоны для сверления закрывают затронутые области от определения координат сверления. Отверстия, лежащие внутри данных областей, не выводятся в схеме сверления или с данными ЧУ и поэтому не высверливаются.
*[Типы записи данных]: Тип записи данных указывает тип данных, относящихся к изделию или общих, например, отдельная часть, узел, кабель, корпус, список принадлежностей, схема сверления, клиент.
*[eCl@ss]: eCl@ss — стандартизированная межотраслевая система классификации для товарных групп и товарных признаков, а также услуг, которая имеет целью облегчить электронную торговлю классифицированными изделиями. Она была инициирована международными концернами и классифицирует большой спектр промышленных изделий. Как правило, для обмена данными по изделиям между поставщиками и покупателями система используется в электронных системах закупок, которые имеют доступ к каталогам многочисленных поставщиков.ETIM является участником eCl@ss и наоборот. Благодаря этому гарантируется согласование обеих моделей классификации.
*[Список результатов]: При поиске понятий с помощью функции поиска, результат поиска отображается в списке результатов.
*[Ид.]: Идентифицирующий номер объекта, свойства или сообщения. Номера свойств и номера сообщений обозначаются сокращенно как Ид.
*[Комментарий]: В свойство "Комментарий" можно ввести любой дополнительный текст.
*[Неразмещенные устройства]: Неразмещенные' функции — это необозначенные части устройств. Неразмещенные устройства используются в проекте, но не видны на схеме соединений.
*[Шаблоны проектов]: Проектные шаблоны являются шаблонами для новых проектов; они могут содержать настройки проекта, данные проекта и страницы. Шаблоны проектов имеют расширение имени файла *.ept. При создании проекта на основании шаблона проекта, все настройки проектов, данные проектов и страницы копируются из шаблона проектов в новый проект.
*[Всплывающие подсказки]: Небольшое текстовое окно, содержащее информацию, например, о кнопке на панели инструментов или объекте, размещенном в графическом редакторе.
*[Сборные шины (изогнутые)]: Медные функциональные элементы в различных исполнениях, которые различаются по форме, поперечному сечению и размеру и могут адаптироваться к текущей ситуации в электрошкафу. Сборные шины (изогнутые) определены:Изделием (шиной с заданной высотой и шириной или с макросом прямоугольного поперечного сечения)Профилем (путем, по которому проходит прямоугольная основная форма, чтобы сформировать шину)Радиусом сгиба для мест, в которых шину необходимо согнуть.
*[Условные обозначения]: Условное обозначение — это графический элемент для отображения определенной функции (см. Функции). Он состоит из функции и символа. При этом функция содержит логические данные, а символ – графические. Условное обозначение содержит обозначения устройства, обозначения выводов устройства и т. п.
*[Параметры, важные с точки зрения безопасности]: Различные стандарты и директивы (например, Директивы по машинному оборудованию 2006/42/EG) требуют выполнения расчета вероятности отказа машин и защитных функций (см. Защитные функции). По этой причине изготовители устройств снабжают свои устройства или компоненты важными с точки зрения безопасности параметрами. Так, например, параметр "MTBF" отображает частоту отказа устройства или выдачи им ошибочных результатов ("MTBF": английское сокращение для "Mean Time Between Failures" = "Среднее время между отказами").
*[Варианты]: Набор технических свойств изделий. Изделие может иметь различные технические характеристики (напр., конфигурируемые контакторы). Для каждой характеристики имеется вариант, который управляется через однозначное обозначение.
*[Закрепленный]: Прикрепленные окна нельзя перемещать из рамок главной программы.
*[Профиль]: Профиль — это путь, по которому проходит прямоугольный основной контур, формирующий изогнутую медную шину. Вдоль профиля формируются в основном только прямоугольные контуры; другие формы поперечного сечения присвоить нельзя. В редакторе контура создается профиль и сохраняется в виде файла Профиль, медь (*.fp1); причем разрешается использовать только линии и ломаные линии. Чертеж контура должен представлять собой одну сплошную линию; разветвления и перекрестные соединения недопустимы.
