# Диалоговое окно Сортировка страниц – <Тип отчета>

Вы открыли проект.

* Параметры > Настройки > Проекты > "Имя проекта" > Отчеты > Вывод в страницах. В столбце Сортировка страниц щелкните в строке групповой спецификации изделий и нажмите кнопку ++...++.
* Сервисные программы > Отчеты > Генерировать > [Настройки] > Вывод в страницах. В столбце Сортировка страниц щелкните в строке групповой спецификации изделий и нажмите кнопку ++...++.
* Сервисные программы > Отчеты > Генерировать. Выберите вкладку Шаблоны. Выделите шаблон отчета, щелкните рядом с полем Сортировка страниц в столбце Значение и нажмите кнопку ++...++.

В этом диалоговом окне задайте [Сортировка страниц для отчетов](formgeneratorgui_k_seitensortierung.md). Сортировка страниц указывает, для каких уровней структурного идентификатора сортировки страниц генерируются блоки отчетов. Можно также генерировать блоки отчетов независимо от условного обозначения-свойства Группа вывода.

!!! note "Замечание:"

    То, какие виды сортировки страниц доступны, зависит от типа отчетов. Для типа отчетов Перечень форм доступна, к примеру, только сортировка страниц "Общая".

Обзор основных элементов диалогового окна:

### Группировать по / в

Через раскрывающийся список определяется вид сортировки страниц.

* Из настроек: (недоступно в диалоговом окне Настройки). Активация этой опции означает вступление в силу заданных в настройках проекта параметров.
* Структурный идентификатор: если выбрана эта опция, укажите, поставив флажки внизу диалогового окна, какие блоки идентификаторов следует учитывать при создании отчета. Как минимум один флажок должен быть установлен. Если Вы выбрали несколько блоков идентификаторов, для каждой комбинации структурных идентификаторов генерируется один блок отчетов. Блоки отчетов сортируются в структуру страниц в соответствии со структурными идентификаторами.
* Общая: сортировка страниц Общая генерирует общий отчет для всего проекта. Если выбрана эта опция, то для всех уровней структурных идентификаторов будет выдаваться блок отчетов. В зависимости от количества данных проекта этот блок отчетов может содержать одну или несколько страниц отчетов.
Если выбраны дополнительные блоки идентификаторов, то, кроме общего отчета, будут генерироваться отдельные блоки отчетов для каждой комбинации структурных идентификаторов и на соответствующим месте сортироватся в структуру страниц. Для отчетов, относящихся к функции, настройки для блоков идентификаторов недоступны.
* Группа вывода: если выбрана эта опция, то блок отчетов генерируется каждый раз при изменении свойства условного обозначения Группа вывода. Комбинирование данной настройки с другими настройками в текущем диалоговом окне невозможно.
* Идентификатор объекта: если выбрана эта опция, то устанавливается только флажок Вид документа. Его установка открывает доступ к структурированию страниц по стандарту DIN EN 61355-1. При этом через идентификатор объекта определенному объекту (например, функции, месту или продукту) присваивается документ (т.е. страница проекта).

!!! example "Пример:"

    Для типа отчета "Содержание" выбрана только опция Общая. EPLAN генерирует для всего проекта содержание и блок отчетов. В этом примере блок отчетов содержит 9страницы отчетов.Если выбран параметр Структурный идентификатори установлены флажки Установкаи Место установки, то для каждой комбинации установки и места установки генерируется отдельное содержание, например для комбинаций=GB1+A1,=GD1+A2,=GL1+B2.Y1и т. д. Содержания сортируются в структуру страниц, т. е. содержание для=GB1+A1размещается подA1, содержание для=GL1+B2.Y1размещается подY1и т. д.

### Таблица Структура / Значение

Если флажок в одном из полей "Функциональное присвоение", "Установка" и т. д. установлен, то для каждого уровня структурных идентификаторов блока идентификаторов (например, функциональное присвоение) соответственно генерируется собственный блок отчетов. При этом глубину структуры можно задать в раскрывающемся списке в столбце Значение: здесь доступны значения "Полный" и, если имеются, "Главный идентификатор" и "Подидентификатор". Комбинирование этой настройки с настройкой Группа вывода невозможно. Для отчетов, относящихся к функции, дополнительная настройка Общее недоступна.

!!! example "Пример:"

    Для типа отчета "Содержание" вы установили только флажок Установкаи установили для глубины структуры значение "Полный". Для каждого уровня структурных идентификаторов блока идентификаторов "Установка" EPLAN генерирует содержание, например для=GB1,=GD1,=GD2и т. д. Отчет при этом генерируется в нескольких блоках отчетов. Содержания сортируются в структуре страниц под соответствующей установкой.

### Использовать источник/цель

Этот флажок устанавливается только для спецификаций кабеля, перечней кабелей, схем подключения кабеля и таблиц соединений.
Если флажок установлен, для сортировки страниц используется обозначение не устройства кабеля, а источников и целей. Кабели, источники и цели которых имеют различные структурные идентификаторы (например, места установки), выводятся в нескольких структурных идентификаторах.

### Использовать размещение

Если установлен этот флажок, то при составлении отчетов по устройствам в различных структурах проекта учитываются не структурные идентификаторы устройства, а структурные идентификаторы страницы, на которой они размещены.
