# Предварительное планирование: Устройства ПЛК

В предварительном планировании уже может быть известно, сколько выводов устройства ПЛК имеют устройства (объекты планирования, технологические контуры, функции ТК). [Данные ПЛК](planninggui_r_sps.md), предварительно определенные на объектах планирования, можно перенести из предварительного планирования в детальное планирование или использовать на устройствах в детальном планировании. При этом данные ПЛК идентифицируются с помощью символического адреса.

Определение однозначного символического адреса

На сегментах (сегментах структуры, объектах планирования и т. д.) можно ввести отдельный элемент символического адреса (свойство Символический адрес (отдельный элемент), ид. 44053). Эти отдельные элементы прикрепляются друг к другу на входах и выходах ПЛК объектов планирования, начиная с верхнего узла, и составляют автоматически найденный символический адрес. Таким образом на объектах предварительного планирования генерируется однозначный символический адрес.

В диалоговом окне "Свойства" объектов планирования отображается определенный таким образом символический адрес на вкладке ПЛК в столбце Символический адрес. В этом столбце не следует вводить данные вручную. На основе символических адресов становится возможным присвоение выводам устройства ПЛК в детальном планировании.

Подготовка макросов

Рекомендуется работать с подготовленными макросами, внесенными на объектах планирования. Такие макросы содержат все необходимые входы и выходы ПЛК. Дополнительно для каждого входа и выхода ПЛК макросы должны содержать объект-заполнитель, где свойство Символический адрес используется как переменная. Для этого определите переменную для символического адреса и внесите свойство Адрес ПЛК: Символический адрес (ид. 44024) в форме от <44024,1> до <44024,50>. При помощи индекса свойства различаются отдельные входы и выходы ПЛК, макс. 50 вводов.

Таким образом можно переносить значения из предварительного планирования на соответствующие выводы устройства ПЛК на схеме соединений: с помощью переменных значения символического адреса считываются из вкладки ПЛК объекта планирования, а (при размещении макроса) с помощью свойства Адрес ПЛК: Символический адрес (ид. 44024) переносятся на соответствующие выводы устройства ПЛК на схеме соединений.

!!! note "Замечание:"

    Для каждого входа и выхода ПЛК в макросе на вкладке ПЛК должна существовать строка. Для правильного присвоения данных последовательность строк должна соответствовать графической последовательности входов и выходов ПЛК в макросе.

Связь с детальным планированием

Детальное планирование создается путем перетаскивания мышью из макросов: объекты планирования перемещаются из навигатора предварительного планирования в навигатор страниц. Таким образом генерируются новые страницы схем соединений, на которых размещаются макросы, указанные на объектах планирования. При этом автоматически используются имеющиеся адреса ПЛК из предварительного планирования.

Если устройству ПЛК, связанному с объектом планирования, в детальном планировании присвоено изделие, шаблонам функций изделия можно присвоить поблочно имеющиеся выводы устройства ПЛК из предварительного планирования.

Для получения дополнительной информации по теме "Унаследование ПЛК данных предварительного планирования" и рекомендаций по данной теме обратитесь в службу поддержки EPLAN.
