# Адресация выводов устройства ПЛК

Выводам устройств ПЛК адреса можно присвоить вручную, впоследствии или автоматически.

* ***Адресация вручную*** позволяет присвоить адрес ПЛК отдельно каждому выводу устройства. Для этого введите адрес в диалоговом окне Свойства во вкладке Вывод устройства ПЛК в поле Адрес.
* ***Адресация впоследствии*** означает, что нескольким выводам устройства ПЛК с уже имеющимися адресами в ходе работы будут присвоены новые адреса. Для этого выберите пункты меню Данные проекта > ПЛК > Адресация.
* При ***автоматической адресации*** выводы устройства ПЛК автоматически содержат новый адрес при вставке на страницу схемы соединений или на страницу обзора.

При применении различных блоков управления ПЛК в проекте необходимо каждому ЦПУ присвоить схему для специфических для ПЛК настроек. В такой схеме определяются форматы для адресов ПЛК относительно типа ПЛК. В таком случае при последующей или автоматической адресации учитываются настройки соответствующей присвоенной ЦПУ схемы.

Если ЦПУ не присвоена схема для специфических для ПЛК настроек, применяется схема из настроек проекта. При автоматической адресации различных ЦПУ ***без*** присвоенной схемы могут быть предложены неверно сформатированные адреса.

При вставке или копировании выводов устройства ПЛК посредством выбора символов или макросов в ходе автоматической адресации также применяется схема из настроек проекта.

При работе в многопользовательском режиме рекомендуется отключить автоматическую адресацию выводов устройства ПЛК.

!!! note "Замечание:"

    Если при последующей адресации заданный начальный адрес не соответствует формату адреса, будет выведено сообщение. В этом случае начальный адрес и / или формат адреса необходимо соответствующим образом скорректировать.

### Выводы устройства ПЛК с фиксированными адресами

Если в проекте используются выводы устройства ПЛК с фиксированными адресами, то их, как правило, не следует изменять при последующей адресации или при вставке макросов. С помощью свойства Не учитывать при адресации для вывода устройства ПЛК можно задать, чтобы он не учитывался при адресации и сохранял таким образом свой первоначальный адрес.
При адресации такие выводы устройства ПЛК также не будут отображаться в предварительном просмотре результатов.

Следите за тем, чтобы свойство было установлено на каждом использованном виде представления вывода устройства ПЛК для унификации поведения видов представления при адресации.

### Карты ПЛК с различными диапазонами адресов

Для карт ПЛК с ***уже существующими входами и выходами*** можно задать два отдельных диапазона адресов и два отдельных начальных адреса для входов и выходов. Для этого на вкладке Блок ПЛК диалогового окна свойств предусмотрены свойства Начальный адрес карты ПЛК и Начальный адрес 2 карты ПЛК. Эти свойства можно использовать при адресации со [значением конфигурации](eservicesgui_d_formatzaehler.md).

!!! tip "Совет:"

    Для проверки расположения адресов вне диапазона адресов карты ПЛК или пересечения диапазонов адресов нескольких карт ПЛК доступны контрольные прогоны 004106 и 004107. Чтобы использовать эти контрольные прогоны, на картах ПЛК в дополнение к начальным адресам необходимо ввести свойства Устройство ПЛК: Длина данных (входы) и / или Устройство ПЛК: Длина данных (выходы). Диапазон адресов определяется в каждом случае исходя из начального адреса и длины данных. Длина данных указывает количество битов следующего свободного адреса на следующей карте ПЛК.

Кроме того, можно управлять устройствами ПЛК, которые состоят из нескольких интегрированных узлов и имеют несколько разных диапазонов адресов с разными начальными адресами. Для этого доступно до двенадцати подустройств ПЛК, которые определяются с помощью соответствующих свойств.
