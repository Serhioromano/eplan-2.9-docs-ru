# Вкладка Экспорт (Пользователь, DXF)

* Параметры > Настройки > Пользователь > Интерфейсы > Экспорт и импорт DXF / DWG. Выберите вкладку Экспорт.
* Страница > Экспортировать > DXF/DWG. Щелкните рядом с полем Схема на ++...++. Выберите вкладку Экспорт.

Определите на этой вкладке специальные настройки экспорта DXF / DWG.

Обзор основных элементов диалогового окна:

### Фильтр страниц

Из раскрывающегося списка выберите фильтр страниц, сгенерированный как схема в навигаторе страниц. Выбранный таким образом фильтр сохраняется в схеме экспорта DXF/DWG. Если сохраненный фильтр больше не существует, экспорт прерывается с соответствующим сообщением.

Выбранный фильтр активируется автоматически. Запись "- Не активировано -" отключает фильтр и приводит к тому, что данные отображаются в неотфильтрованном виде.

### Средство экспорта

В данном групповом поле задайте, на каком средстве экспорта должен выполняться экспорт:

* Носитель данных: При выборе этого параметра файл DXF/DWG выводится на носитель данных.
В поле Каталог вывода укажите место хранения. В качестве целевых для экспорта данных можно указать любой дисковод и подкаталог.
* Электронная почта: При выборе этого параметра файл DXF / DWG высылается по электронной почте. После запуска функции файл запаковывается посредством внутреннего архиватора, после чего запускается настроенный редактор электронной почты.
Из раскрывающегося списка Ограничение размера выберите максимальный размер для отдельных файлов, отправляемых по электронной почте. Вы можете установить значение от 1 до 10 Mб с шагом 1 Mб или выбрать запись "Неогранич.". EPLAN автоматически создаст то количество файлов для отправки, которое необходимо для отправки всех частей. Всем частям будет сопутствовать следующий текст: "<Имя файла> n/m". При этом "n" — это номер отдельного файла для отправки по электронной почте, а "m" — общее количество файлов.

!!! note "Замечание:"

    Для обмена электронными сообщениями на вашем компьютере должен быть установлен и настроен клиент электронной почты, поддерживающий MAPI. По этой же причине не может применяться почтовое приложение Windows 8®.

### Генерировать имена файлов

В этом поле определяется стандартный порядок присвоения имен файлам экспорта DXF/DWG:

* Из порядкового номера: если выбран этот параметр, EPLAN по умолчанию присваивает файлам DXF / DWG имена по образцу: <порядковый_номер>.dxf или <порядковый_номер>.dwg. если уже имеются файлы DXF / DWG, EPLAN начнет нумерацию со следующего доступного числа.
* Из имени страницы: если выбран этот параметр, EPLAN будет по умолчанию присваивать файлам DXF/DWG имена по образцу: <имя страницы>.dxf или <имя страницы>.dwg.
* Из свойств: если выбран этот параметр, имя файла составляется из свойств страниц и проекта, а также свободно определяемого разделителя. Используя кнопку ++...++, откройте диалоговое окно Формат: Свойство для конфигурации имени файла.

### Генерировать подкаталоги

В данном раскрывающемся списке определяется стандартный порядок присвоения имен подкаталогам экспорта DXF/DWG:

* Нет: Если выбран этот параметр, при экспорте из свойств не генерируются подкаталоги.

!!! note "Замечание:"

    По организационным причинам с помощью этой настройки генерируется специфический для проекта подкаталог, чтобы более ранние одноименные файлы экспорта не были случайно перезаписаны.

* Из дерева страниц: Если выбран данный параметр, то при экспорте автоматически генерируются подкаталоги, отражающие структуру проекта. Если, например, в проекте существует установка =Установка1, то подкаталог =Установка1 генерирует и экспортирует туда все соответствующие страницы.
* Из свойств: если выбран этот параметр, имя подкаталога составляется из свойств страниц и проекта, а также свободно определяемого разделителя. Используя кнопку ++...++, откройте диалоговое окно Формат: Свойство для конфигурации имени файла. Для группировки в имени каталога свойств проекта и страниц используйте здесь, например, элемент формата "Разделитель каталога".

### Файл типов линий

В этом файле определяется образец линий для копирования в AutoCAD. Файл не является составной частью объема поставки, он должен быть предоставлен программой САПР (например, файл ACAD.lin для AutoCAD).

С помощью пункта всплывающего меню Вставить переменную пути перейдите в диалоговое окно [Выбрать переменную пути](modaldialogsdb_d_pfadvariablen.md), из которого можно будет скопировать одну из доступных переменных пути.

### Файл стиля схемы

В этом файле определен стиль схемы для копирования в AutoCAD, если вы используете в AutoCAD "названные таблицы стилей схемы". Заданный здесь стиль схемы будет восприниматься при чтении экспортированных файлов в AutoCAD и учитываться при печати как значение по умолчанию. Файл не является составной частью объема поставки, он должен быть предоставлен программой САПР.

С помощью пункта всплывающего меню Вставить переменную пути перейдите в диалоговое окно [Выбрать переменную пути](modaldialogsdb_d_pfadvariablen.md), из которого можно будет скопировать одну из доступных переменных пути.

!!! note "Замечание:"

    * Файлы ctb ***не*** записываются в файл экспорта с указанием полного пути. Чтобы проанализировать файл стиля схемы в AutoCAD, он должен быть скопирован в специальный каталог AutoCAD для стилей схем (если речь не идет о файле ctb, который входит в установочный пакет AutoCAD). По умолчанию это каталог C:\Users\\<username\>\AppData\Roaming\Autodesk\AutoCAD 2015\R20.0\enu\Plotters\Plot Styles, причем для \<username\> должно использоваться имя пользователя Windows, а для AutoCAD 2015\R20.0 — соответствующий номер версии установленной системы AutoCAD; запись enu ("***EN***glish ***U***SA") может быть заменена другим идентификатором языка.
    Или можно внести в AutoCAD стандартный каталог для файлов стиля схемы в параметрах (Параметры > Файлы > Путь для файла поддержки принтера > Путь поиска для таблицы стилей схем).
    * Если для файла стиля схемы выбран файл monochrome.ctb, это ***не*** означает, что экспорт выполняется в монохромном режиме; это означает только то, что в AutoCAD отображение и вывод на печать будут выполняться в соответствии с настройками в файле monochrome.ctb ******, которые можно деактивировать в AutoCAD с помощью настроек для отображения и вывода на печать.
