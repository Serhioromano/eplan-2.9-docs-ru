# Вкладка <Отображаемое имя сегмента структуры>

Вы открыли проект.

* Вы сгенерировали сегмент структуры в навигаторе предварительного планирования или вставили его в Графическом редакторе. Выберите в диалоговом окне Свойства ++...++ вкладку <Отображаемое имя сегмента структуры>.
* Вы выделили сегмент структуры в навигаторе предварительного планирования или в Графическом редакторе. Обработать > Свойства (или Всплывающее меню > Свойства). Выберите вкладку <Отображаемое имя сегмента структуры>.
* Вы сгенерировали шаблон сегмента структуры в навигаторе шаблонов сегментов. Выберите в диалоговом окне Свойства ++...++ вкладку <Отображаемое имя сегмента структуры>.

На этой вкладке указываются свойства выделенного сегмента структуры.

Если в диалоговом окне свойств записи в полях отличаются от стандартных значений присвоенного шаблона сегмента, они выделяются [желтым цветом фона](planninggui_k_segmentvorlagen.md).

Обзор основных элементов диалогового окна:

### Обозначение

Введите здесь имя сегмента структуры. Указанное здесь обозначение по умолчанию отображается в навигаторе предварительного планирования на сегменте структуры.

### Полное обозначение

В этом поле отображается обозначение сегмента структуры, включая обозначения вышестоящих сегментов структуры.

### Описание

Введите здесь описательный текст для сегмента структуры. Этот текст по умолчанию отображается в навигаторе предварительного планирования вместе с обозначением на сегменте структуры. Если для сегмента структуры внесен структурный идентификатор, то в дальнейшем описание в управлении структурными идентификаторами копируется в виде описания структур присвоенного структурного идентификатора.

### Шаблон сегмента

В этом поле отображается шаблон сегмента, присвоенный сегменту. С помощью кнопки ++...++ откройте диалоговое окно Выбрать шаблон сегмента, чтобы выбрать шаблон. Отображаются только шаблоны сегментов, подходящие к определению текущего сегмента. При замене шаблона сегмента значения свойств нового шаблона сегмента переносятся на сегмент и отображаются в диалоговом окне Свойства. Значения, измененные вручную, сохраняются.

С помощью кнопки ![](images/all_delete_as.png){: .ui-icon } (Удалить) из сегмента можно удалить ссылку на шаблон сегмента. Определенные в шаблоне сегмента значения свойств удаляются из сегмента. Значения, измененные вручную, сохраняются.

### Структурный идентификатор

В таблице — в зависимости от структуры проекта — отображаются и могут обрабатываться структурные идентификаторы текущего сегмента структуры.

* Структура: Здесь перечислены блоки идентификаторов, соответствующие структуре проекта. В соседнем столбце отображаются соответствующие предшествующие знаки.
* Видимый: В этих полях можно обрабатывать структурные идентификаторы текущего сегмента структуры. Для этого в соответствующем поле необходимо указать структурный идентификатор или щелкнуть по кнопке ++...++, чтобы выбрать структурный идентификатор при помощи диалогового окна <Структурный идентификатор> (<Имя проекта>).
* Полный: В этих полях отображаются структурные идентификаторы текущего сегмента структуры, включая вышестоящие сегменты структуры. Вышестоящие сегменты структуры здесь не могут обрабатываться, они обусловлены иерархией сегмента структуры в структуре дерева навигатора предварительного планирования.

!!! note "Замечание:"

    * Здесь ***невозможно*** обработать предшествующие знаки и разделители между отдельными структурными идентификаторами; эти знаки и разделители обусловлены настройками диалогового окна Расширенные структуры проекта, которое вы можете открыть, нажав на вкладке Структура свойств проекта кнопку ++Другие++.
    * В каждое из этих полей вводится только структурный идентификатор (без разделителей или подидентификаторов). В противном случае структуру не удастся правильно вывести в отчет. Для создания подструктур создайте новый сегмент структуры.

!!! tip "Совет:"

    * Занимайтесь сначала только одним блоком идентификаторов (например, установкой) и не пытайтесь планировать все блоки идентификаторов одновременно.
    * Вы можете также оставлять поля для структурных идентификаторов пустыми и использовать такие сегменты структуры только для структурирования предварительного планирования.
