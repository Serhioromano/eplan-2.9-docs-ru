# Диалоговое окно Восстановить <Основные данные>

Проект > Восстановить > "Основные данные"

В диалоговых окнах Восстановить <Основные данные> определяется, каким образом и куда будут восстановлены резервированные основные данные. Диалоговые окна имеют одинаковую для всех основных данных структуру.

Посредством диалоговых окон можно восстановить следующие основные данные:

* Базовые проекты
* Шаблоны проектов
* Библиотеки символов
* Рамки
* Контуры
* Формы
* Макросы
* Данные изделия
* Словари
* Настройки вне зависимости от проекта.

Обзор основных элементов диалогового окна:

### Каталог для резервирования

Выберите каталог, в котором находятся основные данные для резервирования. С помощью кнопки ++...++ открывается диалоговое окно выбора и задается требуемый каталог путем выбора файла резервирования с соответствующим типом файла.

С помощью пункта всплывающего меню Вставить переменную пути перейдите в диалоговое окно [Выбрать переменную пути](modaldialogsdb_d_pfadvariablen.md), из которого можно будет скопировать одну из доступных переменных пути.

### [Отобразить содержимое]

Предварительно выделив в списке <Основные данные> определенный файл, при помощи этой кнопки можно отобразить [содержимое файла резервирования](bakrestoredlggui_d_inhaltsicherungsdatei.md).

### Основные данные: <Основные данные>

В этом списке перечислены основные данные, расположенные в выбранном каталоге. Выделите файлы для восстановления.

### Описание

В этом поле можно просмотреть информативный текст, заданный при резервировании основных данных.

### Целевой каталог

Укажите каталог, в который будут сохранены восстановленные основные данные. При помощи ++...++ можно открыть диалоговое окно для выбора каталога и перейти в нем к нужному каталогу.

С помощью пункта всплывающего меню Вставить переменную пути перейдите в диалоговое окно [Выбрать переменную пути](modaldialogsdb_d_pfadvariablen.md), из которого можно будет скопировать одну из доступных переменных пути.

### Подкаталог

В этом поле отображается подкаталог, расположенный под стандартным каталогом, в котором находились основные данные на момент резервирования. Если подкаталога не было, то не отображается ничего.

### Восстановить в подкаталоге

!!! warning "Предупреждение:"

    Этот флажок недоступен, если резервирование основных данных осуществлялось в "Каталог основных данных", а также если, например, библиотеки символов были сохранены в каталоге $(MD_SYMBOLS). Если при резервировании подкаталог каталога символов используется в качестве каталога основных данных, то этот подкаталог сохраняется при резервировании и предлагается при восстановлении, т. е. флажок может быть установлен.

Если этот флажок установлен, основные данные будут восстановлены в исходном подкаталоге, отображаемом в поле Подкаталог.
Если флажок не установлен, то основные данные будут восстановлены в каталоге, отображаемом в поле Целевой каталог.

При необходимости использования подкаталога, который не существует, появится запрос о его создании.
