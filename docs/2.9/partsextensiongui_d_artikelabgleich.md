# Диалоговое окно Синхронизация изделий — <Имя проекта>

Вы открыли проект, изделия которого необходимо синхронизировать; вы ***не*** открыли редактор для обработки основных данных (редактор рамок, редактор форм и т. д.). Сервисные программы > Изделие > Синхронизировать текущий проект.

В этом диалоговом окне установите, какие данные и в каком направлении должны быть синхронизированы. Учтите при этом, что ссылки на изделия не выполняются, они сохраняются в проекте, а также копируются. Тем самым, осуществляется сохранение в ***проекте*** использованных основных данных.

Обзор основных элементов диалогового окна:

### Сохраненные изделия

Список изделий, используемых в проекте, содержит следующие записи:

* Статус: Сведения о статусе, например "Более новый" или "Идентичный"
* Номер изделия: Номер изделия из базы данных изделий
* Вариант: Номер варианта изделия
* Последнее изменение: Дата последнего изменения изделия в проекте.

Записи сортируются по столбцам в алфавитно-числовом порядке, при этом сортировка по умолчанию производится по записи Номер изделия. Если щелкнуть столбец Вариант, то все записи отображаются отсортированными по номеру варианта.

### Изделие системы

Список изделий системы содержит следующие записи:

* Статус: Сведения о статусе, например "Не сохранено" или "Идентичный"
* Номер изделия: Номер изделия из базы данных изделий
* Вариант: Номер варианта изделия
* Последнее изменение: Дата последнего изменения изделия в базе данных изделий.

Эти записи также сортируются по столбцам в алфавитно-числовом порядке, при этом сортировка по умолчанию производится по записи Номер изделия. Если щелкнуть столбец Вариант, то все записи отображаются отсортированными по номеру варианта.

### Всплывающее меню

Команда |  Значение
---|---
Выделить все |  Выделяет все элементы списка.
Подобрать ширину столбца |  Все столбцы так подогнаны к таблице по ширине, что и заголовок, и содержимое столбцов полностью читаемы.
Открыть гиперссылку |  Дает возможность открыть внешние документы или графический файл, если они были присвоены изделию в базе данных изделий.
Конфигурировать представление |  Открывает диалоговое окно Конфигурировать представление, в котором можно задать, какие свойства должны отображаться в списке.

### Кнопки со стрелками

Используйте клавиши со стрелками для копирования в проект изделий системы или отдельных изделий, сохраненных в проекте, в основные данные системы. Многократный выбор возможен в обоих списках.

В окне указаний EPLAN отображает информацию о том, сколько сохраненных изделий / изделий системы были обновлены; здесь также отображается информация в случае, если сохраненные изделия и изделия системы идентичны.

### Фильтр

В этом раскрывающемся списке отображаются все доступные фильтры. Выбранный фильтр активируется автоматически. Пункт "-Не активировано-" отключает фильтр и приводит к тому, что данные отображаются в неотфильтрованном виде.

* Отобразить различия: Выберите этот фильтр, чтобы отображались только различные данные как изделий проекта, так и изделий системы, независимо от того, какие данные более ранние или поздние.
* Основные данные проекта: Выберите этот фильтр, чтобы отображались только сохраненные в проекте изделия, которые являются более ранними, чем соответствующие изделия системы.
* Основные данные системы: Выберите этот фильтр, чтобы отображались только изделия системы, которые являются более ранними, чем соответствующие изделия, сохраненные в проекте.

### [Дополнительно]

Пункт меню |  Значение
---|---
Обновить текущий проект |  Заменяет устаревшие сохраненные в проекте изделия на новые изделия системы.
Дополнить текущую систему |  Дополняет в систему (так называемый "банк данных изделий") изделие, которое имеется в проекте, но отсутствует в системе.

!!! note "Замечание:"

    * Если при синхронизации новая версия объектов основных данных заменяется на более старую (идентификация по записи в Посл. изменение), выводится соответствующее указание, и вы можете прервать обновление.
    * При сравнении основных данных допускаются имена файлов с прописными и строчными буквами, но обрабатываются они одинаково. То есть имя файла ABC.XYZ идентично с abc.xyz.
    * Если ***Изделие*** имеется в проекте, но отсутствует в системе, можно выполнить синхронизацию проекта в системе, в ходе которой изделие будет создано в системе.
    Если изделие имеется в проекте и в системе, и изделие в проекте более позднее, чем изделие системы, то данные не переносятся в систему. По этой причине при выделении по отдельности кнопка ![](images/all_arrowright_as.png){: .ui-icon } становится недоступной; тем не менее, функция [Дополнительно] > Дополнить текущую систему остается активной, поскольку она действует на все диалоговое окно.
