# Диалоговое окно Объединить маршрутизируемые соединения

Сгенерированы маршрутизируемые соединения. В навигаторе соединений выделены маршрутизируемые соединения. Выберите пункты меню Данные проекта > Соединения > Объединить.

В этом диалоговом окне выберите схему с настройками для автоматического объединения маршрутизируемых соединений. Каждая схема объединенных проводов может включать в себя несколько дополнительных схем для фильтрации, сортировки и критериев объединения проводов. Вы можете создать новые схемы и обработать имеющиеся. Кроме этого, в диалоговом окне установите общие настройки для объединения, не учитываемые в схеме.

При автоматическом объединении номера объединенных проводов записываются в свойства маршрутизируемых соединений Объединенные провода и Группа объединенных проводов.

Доступны следующие элементы диалогового окна:

### Настройки

* Схема объединенных проводов: Выберите в левом столбце таблицы схему, которую вы хотите обработать.

С помощью кнопки ++...++ открывается диалоговое окно Настройки: Объединить, в котором вы можете настроить фильтры и критерии для объединения по этой схеме.

При помощи расположенной выше [панели инструментов](schemeconfig_m_schemaschaltflaechen.md) вы можете сделать новые записи в списке или удалить их. Кроме этого, можно изменить последовательность схем. В соответствии с установленной таким образом последовательностью выполняется последовательное генерирование объединенных проводов со сквозной нумерацией для каждой схемы.
* Описание: В этом поле введите текст описания для схемы.

Схемы объединенных проводов в таблице обрабатываются последовательно сверху вниз. Провода, распознанные схемой, объединяются соответствующим образом. Номера объединенных проводов, присваиваемые вышестоящей схемой, при этом больше не перезаписываются, если один и тот же провод необходимо еще раз объединить в последующей схеме.

!!! example "Пример:"

    ***Расположение схемы объединенных проводов:***Верхняя схема:Критерий фильтра: цвет / номер соединения = RD (красный)Вторая схема:Критерий фильтра: цвет / номер соединения = BK (черный)Третья схема:Критерий фильтра = нет фильтра.В данном расположении с тремя схемами объединенных проводов критерии для сортировки (поперечное сечение / диаметр соединения) и для новых объединенных проводов (поперечное сечение / диаметр соединения) одинаковы для всех схем.***Результат:***Сначала объединяются все красныепроводапо поперечному сечению (например, номер пучка1).После этого также в соответствии с поперечным сечением объединяются все черные провода (например, номер объединенных проводов2,3).В последнюю очередь в соответствии с поперечным сечением объединяются все остальные провода, кроме красного и черного (например, номер объединенных проводов4,5,6).

### Начальное значение

Номера объединенных проводов генерируются с заданной величиной шага от начального значения. Стандартное значение: 1. Объединенным проводам присваиваются номера, которые отсутствуют в проекте. Пропуски заполняются.

### Величина шага

Определяет интервал, с учетом которого объединенным проводам присваивается номер. Стандартное значение: 1.

### Максимальное количество проводов в объединенных проводах

Максимальное количество проводов в объединенных проводах — это число проводов, которое может объединить в группу машина по сборке проводов. При превышении заданного числа автоматически генерируются новые объединенные провода. Если настройка остается пустой, объединенные провода могут включать в себя неограниченное число проводов.

### Максимальная сумма поперечного сечения жил

Максимальная сумма поперечных сечений жил — это предельное для данной машины значение для суммы поперечных сечений всех проводов. При превышении заданной суммы автоматически генерируются новые объединенные провода. Если эта настройка остается пустой, то сумма поперечных сечений жил не ограничена.

### Объединить шлейфы с приоритетом

Если поставлен этот флажок, маршрутизируемые соединения в сети, содержащей более одного соединения, объединяются прежде всех остальных маршрутизируемых соединений в группу объединенных проводов. В этом случае расположенное под ним поле Начиная с числа проводов активируется, и можно задать число проводов, начиная с которого должны объединяться шлейфы.

### Начиная с числа проводов

После установки флажка Объединить шлейфы с приоритетом в этом поле можно задать число проводов, начиная с которого должны объединяться шлейфы; при значении 0 (стандартное значение) ограничения отсутствуют. Если в цепочке жил нет требуемого числа проводов, то соединения объединяются согласно установленным критериям объединения.

Если настройка Объединить шлейфы с приоритетом не активирована, то число проводов игнорируется программой.

### Применить ко всему проекту

При помощи этого флажка выбор для объединения распространяется на весь проект в зависимости от выбора объединяемых соединений.

* Если не выбрано ни одно соединение, этот флажок устанавливается и убрать его нельзя.
* Если выбрано одно или несколько соединений, флажок не устанавливается, и его можно установить.

### Предварительный просмотр результата

Если установлен этот флажок, перед копированием новых номеров объединения проводов откроется диалоговое окно Объединить соединения: Предварительный просмотр результата. В этом диалоговом окне приводятся все соединения, которые будут изменены. Новые номера объединений проводов можно подтвердить, нажав на клавишу ++OK++, или изменить вручную.
