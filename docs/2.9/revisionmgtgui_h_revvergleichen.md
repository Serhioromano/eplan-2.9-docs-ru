# Сравнить свойства проектов

Указывается разница между двумя проектами во время сравнения определенных свойств между собой. В диалоговом окне Сравнить свойства проектов укажите, какие проекты следует сравнить между собой. Свойства, которые следует сравнить, указываются в диалоговом окне Настройки: Сравнение свойств проектов. если значение свойств для двух проектов совпадает, разница между ними не определяется.

Результат сравнения записывается в список результатов ревизии. Этот список сохраняется в базе данных текущего проекта.

Условия:

* Вы открыли проект, в каталоге проекта имеется исходный проект (или любой другой проект, с которым следует сравнить текущий проект).
* Задано максимальное число сообщений, отображаемых в диалоговом окне Результаты сравнения свойств — <Имя проекта> (в диалоговом окне Настройки: ревизия (Сравнение свойств проектов) в поле Максимальное число отображенных результатов).

1. Выделите в навигаторе страниц нужный проект и выберите пункты меню Сервисные программы > Управление ревизиями > Сравнение свойств проектов > Сравнить проекты.
2. В диалоговом окне Сравнить свойства проектов установите флажок Отобразить только соответствующие исходные проекты.

!!! info "Для сведения:"

    При этом в списке отображаются только проекты для сравнения выделенного проекта с защитой от записи. В противном случае в списке отображаются все проекты из каталога проектов (за исключением выделенного).

!!! note "Замечание:"

    Если вы отмените защиту от записи соответствующего исходного проекта, то он перестанет отображаться как 'соответствующий исходный проект', а будет отображаться в списке всех проектов.

3. Выберите в списке Сравнить с, с каким проектом для сравнения следует сравнить текущий открытый проект.
4. В поле Настройки для сравнения свойств выберите схему, определяющую настройки сравнения. (Если вы еще не указали схему, будет предложена только стандартная схема. В этом случае создайте новую схему, в которой вы определите настройки сравнения.)
5. В поле Текст маркеров ревизий введите текст, которым следует обозначить изменения ревизии в свойствах.
6. Установите флажок Для удаленных объектов записать маркер ревизий обратно в исходный проект, если удаленные объекты необходимо обозначить в сравнении проектов. Исходные проекты при этом не должны быть защищены от записи.
7. Установите флажок Альтернативная идентификация, если для идентификации объектов должны использоваться не внутренние идентификаторы объектов, а возможные ключи (определенные идентифицирующие свойства). Это целесообразно применять к автоматически сгенерированным проектам, поскольку при каждой генерации внутренние идентификаторы объектов задаются заново.
8. Щелкните по кнопке ++ОК++.

!!! info "Для сведения:"

    Выполняется сравнение, и его результат записывается в список результатов ревизии.

!!! info "Для сведения:"

    Открывается диалоговое окно Результаты сравнения свойств – <Имя проекта>, представляющее обзор результатов.

!!! info "Для сведения:"

    Изменения отображаются в проекте.

!!! note "Замечание:"

    Помните, что такое сравнение в больших проектах может занять относительно много времени; это также зависит от количества свойств, выбранных для сравнения. Поэтому имеет смысл выполнять такой процесс ночью или на отдельном накопителе.

!!! example "Пример:"

    В одном проекте произведены многочисленные изменения клеммников, после чего обновлены соответствующиеотчеты(перечень клеммников,спецификацииклеммников и т. д.). При помощи сравнения свойств проектов необходимо найти изменения, внесенные в отчеты, и сделать их видимыми.Измените схему для настроек сравнения. Выберите из раскрывающегося списка Категориязапись Тексты заполнителей. В слое "Данные" выделите свойство Содерж. текстаи при помощи кнопки со стрелкой переместите его в поле Сравнивать. Для записи "Тексты заполнителей" установите флажок Учитывать удаленные и новые объекты. Полностью исключите из сравнения проектов все остальныетипы объектов; кроме того, снимите для этих типов объектов флажок Учитывать удаленные и новые объекты.После сравнения свойств проектов изменения в спецификациях клеммников, схемах подключения клемм и т. д. будут обозначены соответствующими маркерами ревизий.
