# Диалоговое окно Настройки: Вывод в страницах

Вы открыли проект.

* Параметры > Настройки > Проекты > "Имя проекта" > Отчеты > Вывод в страницах.
* Сервисные программы > Отчеты > Генерировать > [Настройки] > Вывод в страницах.

В этом диалоговом окне установите настройки для вывода отчетов в страницы.

Обзор основных элементов диалогового окна:

### Панель инструментов

Через панель инструментов можно изменить последовательность типов отчетов в столбце Тип отчета. Если несколько отчетов обновить вместе, то данная последовательность влияет на ***Последовательность генерирования отчетов***. При генерировании отчетов из нескольких шаблонов отчетов, то данная последовательность может иметь влияние на ***Последовательность страниц отчетов*** в навигаторе страниц. Это происходит, если оба шаблона отчетов имеют одну начальную страницу или первый сгенерированный отчет состоит из нескольких страниц, и поэтому имена страниц уже заданы.

    Если необходимо изменить последовательность отчетов типа "Перечень клеммников", "Перечень штекеров" и / или "Перечень кабелей", обязательно убедитесь, что начальные страницы для соответствующих шаблонов отчетов (свойство Начальная страница блока отчетов) настроены правильно, то есть можно приступать к сортировке (в качестве альтернативного варианта действий предлагается снять флажок Подстраница для соответствующих типов отчетов).

### Тип отчета

В этом столбце перечислены все доступные в EPLAN типы отчетов. Существуют типы отчетов для [перечней отчетов](formgeneratorgui_k_auswertungsuebersicht.md) и типов отчетов для [отчетов, относящихся к функции](formgeneratorgui_k_funktionsbezogen.md). Используя кнопки со стрелками на панели инструментов, можно задать последовательность типов отчетов.

!!! note "Замечание:"

    Следите за тем, чтобы тип отчета 'Содержание' ***всегда*** выводился последним! Если определенные отчеты не должны выводиться в содержании, необходимо настроить соответствующие фильтры, чтобы исключить их.

### Форма

Чтобы присвоить выбранному типу отчетов (другую) форму, в этом столбце откройте раскрывающийся список и щелкните по "Поиск". В диалоговом окне Выбрать форму настроен каталог, который занесен в настройки как каталог форм. В поле Тип файла автоматически отображается тип формы, подходящий для типа отчетов. Выделите нужную форму и щелкните по ++Открыть++.

### Сортировка страниц

В этом столбце в зависимости от типа отчета укажите [Сортировка страниц для отчетов](formgeneratorgui_k_seitensortierung.md). Сортировка страниц указывает, для каких уровней структурного идентификатора сортировки страниц генерируются блоки отчетов. Также можно генерировать блок отчетов для каждой группы вывода. Щелкните поле таблицы и кнопку ++...++. В дополнительном диалоговом окне определите необходимые блоки идентификаторов или условное обозначение-свойство Группа вывода. Выбранные блоки идентификаторов или условное обозначение-свойство вносятся в столбец Сортировка страниц. Для каждого уровня структурного идентификатора выбранного блока идентификаторов или для каждой группы вывода генерируется собственный блок отчетов.

!!! note "Замечание:"

    То, какие виды сортировки страниц доступны, зависит от типа отчетов. Для типа отчетов "Перечень форм"доступна, к примеру, только сортировка страниц "Общая".

### Частичный вывод

Выберите из этого раскрывающегося списка форму для частичного вывода. Эта настройка используется только для определенных типов отчетов, таких как содержание, перечень кабелей и др. Если выбрана форма для частичного вывода, то генерируется полный вывод с установленной в поле Форма формой и, например, относящийся к установке вывод генерируется с выбранной здесь формой.

### Объединить

Эта функциональность действительна для динамических форм. Если флажок установлен, то объединяются отчеты нескольких функций на одной странице отчетов.

### Мин. число строк отчета

Для форм, у которых имеется флажок Объединить, это значение устанавливает, какое минимальное число строк отчета функции может отображаться на одной странице отчетов. Эта настройка влияет на то, в каком месте вводится разбивка строки.

### Подстраница

Подстраницами являются страницы, для которых в навигаторе страниц EPLAN создает остальные уровни сегментации. Такие страницы имеют в именах страниц, например, формат 1.1, 1.a, или 1.A. Если флажок установлен и, например, в столбце Символ выбран параметр 'Числовой', генерируются главные и подстраницы, относящиеся к блокам отчетов.

Если отчет состоит из блока отчетов, как, например, спецификация изделий, то первая станица блока отчетов начинается на главной странице и все другие страницы выводятся на подстраницы. Если отчет состоит из нескольких блоков страниц, как, например, спецификация клеммника, который генерирует блок отчетов на клеммник, то каждая первая страница блока отчетов начинается на главной странице. Все остальные страницы блока отчетов выводятся на подстраницы.

Если в блоке отчетов только одна страница, то в таком случае подстраница не генерируется. В зависимости от столбца Символ можно задать, что каждая первая страница блока отчетов начинается прямо с подстраницы. Если флажок не установлен, EPLAN не генерирует подстраницы.

    Для объединенных отчетов с подстраницами генерируется только ***один*** блок отчетов, который содержит все страницы.

### Символ

Если установлен флажок Подстраница, укажите в этом столбце представление подстраниц в пределах имени страницы. При этом подстраница всегда разделяется точкой перед записью для главной страницы. Определите также, будет ли начальная страница отчета начинаться непосредственно с подстраницы.

* Алфавитный, строчн.: Подстраница представлена строчными буквами, например "1.а".
* Алфавитный, прописн.: Подстраница представлена прописными буквами, например "1.A".
* Цифровой: Подстраница представлена цифрами, например "1.1".
* Алфавитный, строчн. (со стр. 1): Начальная страница отчета начинается непосредственно на подстранице. Подстраница представлена буквами, например "1.а".
* Алфавитный, прописн. (со стр. 1): Начальная страница отчета начинается непосредственно на подстранице. Подстраница представлена заглавными буквами, например "1.А".
* Цифровой (со страницы 1): Начальная страница отчета начинается непосредственно на подстранице. Подстраница представлена цифрами, например "1.1".

!!! example "Пример:"

    Выберите параметр Алфавитный, строчн. Создайте отчет из трех страниц, и введите2в качестве начальной страницы отчета. Результат:22.a2.bНачальная страница начинается на странице2. Для последующих страниц отчета EPLAN создаст подстраницы2.aи2.b.Выберите параметр Алфавитный, строчн. (со стр. 1). Создайте отчет из трех страниц, и введите2в качестве начальной страницы отчета. Результат:2.a2.b2.cНачальная страница начинается непосредственно с подстраницы2.a. Для последующих страниц отчета EPLAN создаст подстраницы2.bи2.c.

### Свободные страницы

Значение в этом столбце указывает, сколько страниц остаются свободными между страницей проекта, использованной в последний раз, и новой страницей отчета.

### Округлить

В связи со значением для поля Свободные страницы эта настройка определяет страницу, на которой кратное должно быть округлено.

!!! example "Пример:"

    Свободные страницы=4,Округлить=10Последняя страница проекта Страница отчета47604550

### Скрыть структурный идентификатор при совпадении

Эта настройка позволяет сократить отображение ОУ в страницах отчета.

* Проверить относительно страницы: Если структурные идентификаторы или элементы структурных идентификаторов выводимых устройств соответствуют структурным идентификаторам страницы, то эти структурные идентификаторы не отображаются в ОУ. Если структурные идентификаторы устройств отличаются от структурных идентификаторов страницы, тогда эти структурные идентификаторы представлены в ОУ.

!!! example "Пример:"

    Вы выбрали опцию Проверить относительно страницы.Структурные идентификаторы страницы ОУ в данных проекта ОУ в отображении страницы отчета=A.B+O.1=A.B+O.1-X1-X1=A.B+O.1=A.C+O.2-X1.C+.2-X1

* Проверить относительно объекта заголовка: Эта настройка доступна для [отчетов, относящихся к функции](formgeneratorgui_k_funktionsbezogen.md), например спецификации клеммника. Если структурные идентификаторы или элементы структурных идентификаторов выводимых устройств соответствуют структурным идентификаторам верхнего колонтитула, тогда структурные идентификаторы не просматриваются в ОУ.
* Проверить все относительно страницы (включая данные заголовка): Эта настройка доступна для [отчетов, относящихся к функции](formgeneratorgui_k_funktionsbezogen.md), например спецификации клеммника. Если структурные идентификаторы или элементы структурных идентификаторов выводимых устройств соответствуют структурным идентификаторам страницы, тогда структурные идентификаторы не отображаются также и в верхнем колонтитуле.
* Нет: ОУ устройств отображается на страницах отчета со структурными идентификаторами, включая подидентификаторы.

!!! example "Пример:"

    Вы выбрали опцию Нет.Структурный идентификатор в верхнем колонтитуле ОУ в данных проекта ОУ в отображении страницы отчета=A.B+O.1=A.B+O.1-X1=A.B+O.1-X1=A.B+O.1=A.C+O.2-X1=A.C+O.2-X1

### Синхронизировать

Если этот флажок установлен, то при открытии проекта соответствующая форма автоматически синхронизируется с пулом основных данных, если в диалоговом окне Настройки: Общее (в меню Параметры > Настройки > Проекты > "Имя проекта" > Управление > Общее) установлен флажок Синхронизировать основные данные проекта при открытии.
