# Проверка проекта

Часто участники проекта, например заказчик, посылают проект на определенной стадии доработки другим участникам проекта, например поставщикам. Последние вносят необходимые изменения и посылают доработанную версию обратно.

С помощью проверки проекта вы можете удостовериться, что проект соответствует вашим требованиям и удовлетворяет заранее принятым стандартам. При сравнении [проекта для сравнения и проекта с замечаниями](processreceiptcontrol_k_projekte.md) вы можете проверить следующие параметры:

* Нет ли в проекте ошибок? В проекте нет ошибок, если все положенные контрольные прогоны прошли без сообщений.
* Не изменились ли специфические для проекта настройки?
* Не изменились ли свойства данных проекта?
* Были ли соблюдены стандарты проектирования?

Если состояние проекта не соответствует заданным требованиям, будет показано сообщение, в котором перечисляются пункты несоответствия.

!!! note "Замечание:"

    Если проект для сравнения был создан при помощи другой программы, нежели проект с замечаниями, настройки и изменения, касающиеся дополнительной функциональности, могут не упоминаться в сообщении.
