# Основные данные: Управлять макросами

Макросы разрешены в проекте, пока они введены. В таком случае данные не отличаются от отдельного элемента, вставленного вручную. При этом все данные сохраняются со ссылкой на макрос (в том числе текстов на иностранных языках).

Макрос указывается в имени библиотек(и) символов, из которой (которых) генерируется (отмечается) макрос. Благодаря этому предварительное отображение макроса может использовать нужную библиотеку символов для индикации.

На уровне операционной системы макросы являются отдельными файлами, которые в качестве примера можно отправить заказчикам.

Условие:

Вы открыли проект.

### Управлять макросами с данными изделия

если в макросе содержатся изделия они сохраняются ***не*** автоматически. Для того, чтобы произвести это дополнительно, действуйте следующим образом:

!!! warning "Предупреждение:"

    Обратите внимание, что этим действием все другие данные изделий, связанные с проектом переписываются основными данными.

1. Выберите пункты меню Сервисные программы > Изделие > Обновить текущий проект.

!!! info "Для сведения:"

    Изделия в макросах синхронизируются с изделиями основных данных, т. е. данные базы данных изделий переносятся в макросы.

!!! tip "Совет:"

    Для целенаправленной синхронизации отдельных изделий в проекте можно использовать диалоговое окно Синхронизация изделий — <Имя проекта>. Для этого выберите пункты меню Сервисные программы > Изделие > Синхронизировать текущий проект.

### Управлять макросами с рамками и формами

если в макросе содержится рамка или форма, то эти основные данные также сохраняются ***не*** автоматически. Для того, чтобы произвести это дополнительно, действуйте следующим образом:

1. Выберите пункты меню Сервисные программы > Основные данные > Дополнить текущий проект.

!!! info "Для сведения:"

    Проект ищет недостающие основные данные, содержащиеся в макросе, после чего автоматически их сохраняет.
