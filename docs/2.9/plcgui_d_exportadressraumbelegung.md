# Диалоговое окно Экспортировать распределение адресного пространства

Вы открыли проект. Данные проекта > ПЛК > Адреса / списки присвоения > [Дополнительно] > Экспортировать распределение адресного пространства.

В этом диалоговом окне экспортируется распределение адресного пространства ЦПУ. Экспортируется не зависящий от языка список, который указывает, какие адреса используются определенной картой ПЛК. Экспортируются только те карты ПЛК, на которых [Диапазон адресов](devicetaggui_r_spskarte.md) задан путем внесения начального адреса и соответствующей длины данных.

Список распределения адресного пространства имеет такую структуру:

* Структура строки: "Номер байта", "ОУ (карта входа)", "ОУ (карта выхода)"
Пример (отдельные карты ПЛК для входов и выходов): "0","=A1+O1-K1","=A1+O1-K2"
Пример (карта ПЛК имеет как входы, так и выходы): "0","=A1+O1-K1","=A1+O1-K1"
* Если карта ПЛК использует несколько байтов, строка повторяется.
* Если байт не используется, запись для ОУ пустая.
* Вывод списка заканчивается последним используемым байтом.

!!! note "Замечание:"

    Вы можете экспортировать только один отдельный список для каждого процессора (ЦПУ). Поэтому в диалоговом окне Адреса / списки присвоений настройте фильтр так, чтобы отображались только адреса ПЛК ***одного*** ЦПУ.

Обзор основных элементов диалогового окна:

### Специфические для ПЛК настройки

В этом поле отображается схема, задающая формат адресов ПЛК. Схема находится на основе ЦПУ, содержащихся в выборе, и здесь ее нельзя изменить. Если ЦПУ не присвоена схема, используется схема, заданная в настройках проекта. Над полем дополнительно отображаются проект конфигураций и имя станции, которым присвоено ЦПУ.

### Имя файла

В этом поле введите имя, под которым сохраняется список распределения адресного пространства. С помощью [...] открывается диалоговое окно выбора файлов, в котором вы можете выбрать список распределения адресного пространства, который нужно перезаписать или переименовать. Список можно сохранить как файл CSV или TXT.

С помощью пункта всплывающего меню Вставить переменную пути перейдите в диалоговое окно [Выбрать переменную пути](modaldialogsdb_d_pfadvariablen.md), из которого можно будет скопировать одну из доступных переменных пути.
