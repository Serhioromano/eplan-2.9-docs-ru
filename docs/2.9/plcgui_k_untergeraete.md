# Подустройства ПЛК

Некоторые устройства ПЛК состоят из нескольких интегрированных узлов, каждый из которых имеет собственные диапазоны адресов и начальные адреса. Такое устройство, например, может состоять из внутреннего узла ЦПУ, внутреннего узла ввода-вывода и внутреннего узла счетчика, но при этом будет иметь только один номер изделия. Такие интегрированные модули в устройстве ПЛК могут отображаться в EPLAN подустройствами ПЛК — для этого доступно до двенадцати подустройств ПЛК. Подустройства ПЛК учитываются при обмене данными ПЛК в формате AutomationML. Данные устройства для главного устройства и связанных подустройств ПЛК настраиваются производителем в базе данных изделий один раз для каждого устройства.

Каждое подустройство ПЛК определяется соответствующими свойствами ("x" в последующем обозначает номер подустройства ПЛК от 1 до 12).

Необходимые свойства

* Подустройство ПЛК x: Имя: Имя устройства логического функционального узла.
* Подустройство ПЛК x: На позиции / разъем: Указывает, на каком разъеме / в какой позиции на карте ПЛК находится это подустройство ПЛК.
* Подустройство ПЛК x: Начальный адрес (входы): Начальный адрес для входов подустройства ПЛК x.
* Подустройство ПЛК x: Начальный адрес (выходы): Начальный адрес для выходов подустройства ПЛК x.

Свойства Подустройство ПЛК x: Имя и Подустройство ПЛК x: На позиции / разъем идентифицируют подустройство ПЛК и ***должны*** быть заполнены на блоке ПЛК в проекте. Путем выбора изделия эти свойства заполняются со значениями из базы данных изделий. С помощью свойств Подустройство ПЛК x: Начальный адрес (входы) и Подустройство ПЛК x: Начальный адрес (выходы) можно определить соответствующие начальные адреса для подустройств ПЛК на блоке ПЛК.

Подустройства ПЛК экспортируются, если они имеют хотя бы начальный адрес, вывод шины, вывод устройства В/В или идентификацию устройства.

На выводе устройства ПЛК в свойстве (вывод устройства В/В или вывод шины) Подустройство ПЛК: Индекс указывается, к какому подустройству ПЛК относится вывод устройства. Так, например, значение индекса "3" означает, что вывод устройства ПЛК относится к подустройству ПЛК 3. Эта информация необходима для автоматической адресации и обмена данными ПЛК в формате AutomationML. Свойство также может указывать в шаблонах функций изделий и переносить на выводы устройства ПЛК при выборе устройства или при вставке устройств.

!!! note "Замечание:"

    Свойства для начального адреса доступны только на блоке ПЛК в проекте, но не в базе данных изделий.

Дополнительные свойства

* Подустройство ПЛК x: Длина данных (входы): требуемая длина в адресном пространстве для входов подустройства ПЛК x. Информация представлена в битах.
* Подустройство ПЛК x: Длина данных (выходы): требуемая длина в адресном пространстве для выходов подустройства ПЛК x. Информация представлена в битах.

Свойства для длины данных используются только для контрольных прогонов и вывода загрузки адресного пространства.

Из длины данных вместе с начальным адресом определяется диапазон адресов подустройства ПЛК. Обратите внимание, что имеются карты ПЛК, например, технологические модули, которые занимают всего несколько входов/выходов, но при этом требуют большего диапазона адресов, чем занятые в данный момент. Всегда вводите здесь максимальную необходимую длину для диапазона адресов входов или выходов.

Специальные свойства

Перечисленные ниже свойства используются только для идентификации подустройств ПЛК, которые управляются как ***автономные устройства в программе конфигурации ПЛК***.

* Подустройство ПЛК x: Обозначение типа ПЛК: обозначение типа ПЛК для подустройства ПЛК x карты ПЛК.
* Подустройство ПЛК: Описание устройства: Индекс в файле: ссылка на устройство в файле описания устройства, который введен на блоке ПЛК (главном устройстве) в свойстве Описание устройства: Имя файла.

### Идентификация устройства / присвоение изделий подустройств ПЛК

#### Идентификация через основное устройство

Для подустройств ПЛК, которые ***в программе конфигурации ПЛК рассматриваются как интегрированные устройства***, [идентификация устройств](plcgui_k_austauschbuskonfig.md) и [присвоение изделий](plcgui_k_importbuskonfig.md) при импорте файлов конфигурации ПЛК осуществляется исключительно с помощью обозначения типа ПЛК или файла описания устройства, который вводится в блок ПЛК основного устройства. В этом случае свойства Подустройство ПЛК x: Обозначение типа ПЛК и Подустройство ПЛК x: описание устройства: индекс в файле пусты.

#### Идентификация с помощью нескольких обозначений типа ПЛК или нескольких индексов в файле описания устройства

Для подустройств ПЛК, которые ***в программе конфигурации ПЛК рассматриваются как автономные устройства***, идентификация устройства реализуется с помощью свойств Подустройство ПЛК x: Обозначение типа ПЛК или Подустройство ПЛК x: описание устройства: индекс в файле.

!!! note "Замечание:"

    Идентификация устройства должна выполняться аналогичным образом для главного устройства и связанных с ним подустройств ПЛК. Если, например, на блоке ПЛК указано обозначение типа ПЛК, то подустройства ПЛК также должны определяться с использованием обозначения типа ПЛК.

Свойство Подустройство ПЛК x: Обозначение типа ПЛК необходимо заполнить в подустройствах ПЛК, если обозначение типа ПЛК также введено на основном устройстве. Данные необходимо вводить строго в той же форме, в которой они использовались в каталоге аппаратного обеспечения производителя.

Свойство Подустройство ПЛК x: Описание устройства: индекс в файле следует заполнять для подустройств ПЛК, если они идентифицируются с помощью файла описания устройства и соответствующего индекса. Файл описания устройства указывается на блоке ПЛК (главном устройстве). Индексы файлов описания устройств в AML-файле дополнительно имеют префиксный контекст (например, M/1000). Этот контекст всегда необходимо указывать для подустройства ПЛК при проектировании:

* D/ (Device)
* R/ (Rack)
* DAP/ (Device Access Point)
* M/ (Module)
