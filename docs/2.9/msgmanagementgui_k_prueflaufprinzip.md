# Контрольные прогоны

Сообщения, сгенерированные во время контрольного прогона, сохраняются в базе данных сообщений и выводятся в диалоговом окне Управление сообщениями — <Имя проекта>. При этом для каждого сообщения выводится и краткое описание проблемы. С помощью справочной системы можно вызвать подробное описание причины ошибки и получить информацию о методах ее устранения.

Проверяемые критерии можно настроить самостоятельно и сохранить в виде схемы. Кроме того, схемы можно экспортировать, чтобы затем использовать их для других проектов.

При этом осуществляется проверка данных устройств, например, неполнота определений (клемм, штекеров, кабелей, устройств, ПЛК и т. д.), или наличие многократно использованных (не связанные перекрестными ссылками) или несуществующих устройств. Помимо этого, можно выполнить проверки функционирования, например проверки коротких замыканий, кольцевых соединений, неполных перекрестных ссылок или точек разрывов, неверных определений потенциалов и т. д. Также можно проверить сохраненные в базе данных изделий основные данные изделий.

### Виды проверки

Для контрольных прогонов можно настроить так называемые виды проверки. Вид проверки указывает на то, как и какие критерии проверки должны применяться. Управление сообщениями имеет следующие настройки видов проверки:

* ***Без:*** Контрольный прогон не выполняется, и сообщения не выводятся.
* ***Автономно:*** Сообщения записываются в управление сообщениями не во время обработки, а лишь после выполнения контрольного прогона. Если в выбранной схеме контрольных прогонов включены контрольные прогоны, касающиеся соединений, то перед выполнением автономного контрольного прогона осуществляется автоматическое обновление соединений (как для отдельных страниц, так и для всего проекта).
* ***Подкл. / автоном.:*** Сообщения записываются в управление сообщениями сразу же при их появлении (то есть "online"). Для этого должно быть открыто диалоговое окно Управление проектами — <Имя проекта>. Обработанные сообщения не удаляются автоматически, они убираются из управления сообщениями при выполнении контрольного прогона (т. е. "в автономном режиме"). Однако для сохранения наглядности при обработке сообщения можно помечать как обработанные и фильтровать по этому свойству.
* ***Предотвращение ошибок:*** При выполнении изменения, нарушающего установленный критерий проверки, изменение отменяется после копирования данных. Кроме того, для текста контрольного прогона также выводится страница, ОУ и позиция неверной функции.
* ***Относ. к модулю:*** Помимо управления сообщениями, другие модули выполняют проверки и выводят сообщения в диалоговом окне управления сообщениями (например, при автоматическом генерировании кабелей или при адресации входов и / или выходов устройств ПЛК). Эти проверки отображаются в диалоговом окне Настройки: Сообщения и контр. прогоны в столбце Вид проверки с записью "Относ. к модулю". Если относящаяся к модулям проверка не выполняется, то для вида проверки выберите настройку "Нет".
