# Диалоговое окно Резервировать <Основные данные>

Проект > Резервировать > "Основные данные"

В диалоговых окнах Резервировать <Основные данные> укажите, каким образом и куда будут резервироваться выбранные основные данные. Диалоговые окна имеют одинаковую для всех основных данных структуру.

Посредством диалоговых окон можно резервировать следующие основные данные:

* Базовые проекты
* Шаблоны проектов
* Библиотеки символов
* Рамки
* Контуры
* Формы
* Макросы
* Данные изделия
* Словари
* Настройки вне зависимости от проекта.

!!! note "Замечание:"

    Графика рамок или форм резервируется только в том случае, если она загружена в рамке или форме. Графика со ссылками не может быть резервирована и восстановлена.

Обзор основных элементов диалогового окна:

### Каталог

Выберите каталог, в котором находятся основные данные для резервирования. Каталог основных данных по умолчанию предварительно установлен (как задано в Параметры > Настройки > Пользователь > Управление > Каталоги). При помощи ++...++ можно открыть диалоговое окно для выбора каталога и перейти в нем к нужному каталогу.

С помощью пункта всплывающего меню Вставить переменную пути перейдите в диалоговое окно [Выбрать переменную пути](modaldialogsdb_d_pfadvariablen.md), из которого можно будет скопировать одну из доступных переменных пути.

### Основные данные: <Основные данные>

В этой таблице перечислены основные данные, расположенные в выбранном каталоге. Для сортировки основных данных по возрастанию, по убыванию или в первоначальной последовательности щелкните по заголовку столбца. Выделите основные данные для резервирования. Если ничего не выделено, резервироваться будут все основные данные.

### Резервировать подкаталоги

Если этот флажок установлен, в процесс резервирования будут включаться и основные данные подкаталогов.

### Имя файла резервирования

Укажите имя файла, в который будут резервироваться выбранные основные данные. Нажав ++...++, откройте диалоговое окно Выбор файла резервирования и перейдите к требуемому каталогу или создайте новый.

### Описание

В этом поле можно ввести информативный текст (напр., указание места хранения), который сохраняется вместе с основными данными и отображается в EPLAN при резервировании и восстановлении.

### Носитель для резервирования

В данном поле указывается, будут ли основные данные сохранены на носитель данных (жесткий диск, сменный диск и т. д.) или отправлены по электронной почте:

* Носитель данных: при выборе этой опции основные данные резервируются на носителе данных.
В поле Целевой каталог укажите место накопления. В качестве целевых для резервирования данных можно указать любой дисковод и подкаталог.
Нажав ++...++, откройте диалоговое окно Выбрать каталог и перейдите к нужному каталогу или создайте новый.
С помощью пункта всплывающего меню Вставить переменную пути перейдите в диалоговое окно [Выбрать переменную пути](modaldialogsdb_d_pfadvariablen.md), из которого можно будет скопировать одну из доступных переменных пути.
* Электронная почта: при выборе этой опции основные данные будут отправлены по электронной почте. После запуска функции выбранные основные данные запаковывается посредством внутреннего архиватора, после чего запускается настроенный редактор электронной почты.
Из раскрывающегося списка Ограничение размера выберите максимальный размер для отдельных файлов, отправляемых по электронной почте. Вы можете установить значение от 1 до 10 Mб с шагом 1 Mб или выбрать запись "Неогранич.". EPLAN автоматически создаст то количество файлов для отправки по электронной почте, которое необходимо для отправки всех частей. Всем частям будет сопутствовать следующий текст: "<Имя файла резервирования> n/m". При этом "n" — это номер отдельного файла для отправки по электронной почте, а "m" — общее количество файлов.
