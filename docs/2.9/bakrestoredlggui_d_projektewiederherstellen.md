# Диалоговое окно Восстановить проект

* Вы открыли Навигатор страниц. Проект > Восстановить > Проект.
* Вы открыли окно управления проектами. [Организовать] > Восстановить.

В этом диалоговом окне указывается, каким образом и где будет восстанавливаться резервированный проект.

Обзор основных элементов диалогового окна:

### Каталог для резервирования

Выберите каталог, в котором находятся проекты для резервирования. С помощью кнопки ++...++ открывается диалоговое окно выбора и задается требуемый каталог путем выбора файла резервирования с соответствующим типом файла.

С помощью пункта всплывающего меню Вставить переменную пути перейдите в диалоговое окно [Выбрать переменную пути](modaldialogsdb_d_pfadvariablen.md), из которого можно будет скопировать одну из доступных переменных пути.

### [Отобразить содержимое]

Предварительно выделив в списке Проекты определенный проект, при помощи этой кнопки можно отобразить [Содержание файла резервирования](bakrestoredlggui_d_inhaltsicherungsdatei.md).

### Проекты

В этом списке перечислены проекты, расположенные в выбранном каталоге. Выделите проекты для восстановления.

### Описание

В этом поле можно просмотреть информативный текст, заданный при резервировании проекта.

### Целевой каталог

Укажите каталог, в который будет сохранен восстановленный проект. При помощи ++...++ можно открыть диалоговое окно для выбора каталога и перейти в нем к нужному каталогу.

С помощью пункта всплывающего меню Вставить переменную пути перейдите в диалоговое окно [Выбрать переменную пути](modaldialogsdb_d_pfadvariablen.md), из которого можно будет скопировать одну из доступных переменных пути.

### Подкаталог

В этом поле отображается подкаталог, расположенный под стандартным каталогом, в котором находился проект на момент резервирования. Если подкаталог отсутствовал, ничего не отображается.

### Восстановить в подкаталоге

Установите этот флажок, если проект должен быть восстановлен в исходном подкаталоге. Если при резервировании данных сохраненных каталогов нет, флажок недоступен.
При необходимости использования подкаталога, который не существует, появится запрос о его создании.

### Имя проекта

В этом поле отображается имя проекта, выделенного в списке Проекты. Чтобы не перезаписывать уже существующий проект, следует задать другое имя.
