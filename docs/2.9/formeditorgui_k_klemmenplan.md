# Формуляры для спецификаций клеммников и штекеров

В формах для спецификаций клеммников и штекеров можно интегрировать одну или несколько таблиц кабелей. Из таблицы кабелей видно, какое кабельное соединение связывает клемму или контакт штекера с целевым устройством; при этом кабельные соединения сортируются по кабелям.

Обычно одна форма содержит две таблицы кабелей, например, левую для внешних, а правую –для внутренних выводов устройства клемм / контактов штекера. Также могут существовать таблицы кабелей для определенной позиции в форме (A, B, C или D), а также таблицы кабелей, в которых две позиции в форме соединены (обычно "A и C" или "B и D"). Позиции в форме можно вносить в соответствующие клеммы или контакты штекеров (на вкладке Соединения диалогового окна "Свойства").

!!! example "Пример:"

    Следующая спецификация клеммника содержит две таблицы кабелей. Слева отображаются внешние, а справа внутренниевыводы устройстваклемм.

### Создание таблицы кабелей

Таблица кабелей состоит из верхнего колонтитула и области данных. В верхнем колонтитуле отображаются свойства кабелей, в области данных отображаются свойства кабельных соединений. Обычно выравнивание верхнего колонтитула аналогично выравниванию области данных. Однако есть возможность отобразить верхний колонтитул поперек области данных (повернуть на 90°); в этом случае необходимо активировать свойство формы Таблица кабелей: Верхний колонтитул поперек области данных.

!!! example "Пример:"

    Верхний колонтитул (1) параллельно области данных (2)Верхний колонтитул (1) поперек области данных (2)

#### Задать интервалы

Интервалы для области данных указываются в зависимости от выравнивания формы: Положительные значения проходят параллельно выравниванию формы, отрицательные — в противоположном направлении.

Если выравнивание верхнего колонтитула аналогично выравниванию области данных, достаточно задать все интервалы с помощью области данных (свойства Таблица кабелей: Интервал области данных ...). Интервалы столбцов или строк в верхнем колонтитуле переносятся из области данных.

Если верхний колонтитул находится поперек области данных, интервалы для верхнего колонтитула и области данных задаются отдельно. Для верхнего колонтитула можно использовать свойства Таблица кабелей: Интервал верхнего колонтитула ..., а для области данных — свойства Таблица кабелей: Интервал области данных ..., причем существуют отдельные свойства для различных позиций как для внешних, так и для внутренних выводов устройства.

#### Отображение свойств кабелей и кабельных соединений

Для отображения свойств кабелей в верхнем колонтитуле таблицы кабелей можно использовать заполнитель Таблицы кабелей — верхний колонтитул... Если выравнивание верхнего колонтитула аналогично выравниванию области данных, выберите свойства только с помощью заполнителя Таблицы кабелей — верхний колонтитул. Если верхний колонтитул повернут против области данных, используйте заполнитель Таблицы кабелей — верхний колонтитул, внешн., Таблицы кабелей — верхний колонтитул, внутр. или Таблицы кабелей — верхний колонтитул над позицией в форме.

Для отображения свойств в области данных существует соответствующий заполнитель Таблицы кабелей — область данных внешн., Таблицы кабелей — область данных внутр. и Таблицы кабелей — область данных над позицией в форме.

Если просмотр определенных свойств кабельных соединений не требуется, но надо увидеть, подключено ли кабельное соединение к клемме или контакту штекера, то кабельные соединения, а также отдельные соединения можно отобразить в спецификации клеммника с помощью замещающего символа. Замещающий символ устанавливается в диалоговом окне Настройки: Отображение / вывод (доступно в меню Параметры > Настройки > Проекты > "Имя проекта" > Отчеты > Отображение / вывод).

### Отображение отдельных соединений в таблице кабелей

В таблице кабелей также отображаются отдельные соединения, т. е. соединения, которые не являются кабельными соединениями кабеля. Это указывается в форме, где для использующихся текстов-заполнителей Таблица кабелей —область данных... устанавливается флажок Отобразить все соединения.

Отдельные соединения записываются в область данных таблицы кабелей в направлении, противоположном направлению записи кабельных соединений, т. е. при построении таблицы кабелей по столбцам запись кабельных соединений начинается с первого столбца, а отдельных соединений — с последнего столбца. Если, например, кабельные соединения вносятся справа налево, то отдельные соединения вносятся слева направо.

!!! example "Пример:"

    Для таблицы кабелей определены восемь столбцов. При составлении отчета первый обнаруженный кабель появляется в первом столбце справа (красный1), второй —во втором столбце справа (красный2) и т. д. Первое обнаруженное отдельное соединение появляется в первом столбце слева (зеленый1), второе —в первом столбце слева (зеленый2) и т. д. Если свободных столбцов больше не осталось, начинается новая страница отчета.В этом примере в верхнем колонтитуле таблицы кабелей отображается ОУ кабеля (текст-заполнитель Таблицы кабелей — верхний колонтитул/обозначение устройства (полностью)) и тип кабеля (текст-заполнитель Таблица кабелей —Верхний колонтитул/тип кабеля). Для отдельных соединений вместо типа кабеля отображаются свойства Поперечное сечение / диаметр соединения,Единица поперечного сечения / диаметр соединенияи Цвет / номер соединения. Эта замена выполняется автоматически, причем новый текст-заполнитель можно не задавать.В области данных для кабельных соединений отображается Цвет / номер соединения (текст-заполнитель Таблицы кабелей — область данных внешн. / цвет / номер соединения); отдельные соединения отображаются с помощью заменяющего символа '*'.

#### Правила отображения отдельных соединений в таблице кабелей

В таблице кабелей, как правило, отображаются только такие соединения, которые отличаются от предварительно установленных по умолчанию значений проекта. Соединения, при которых значения свойств Поперечное сечение / диаметр соединения (ID 31002), Единица поперечного сечения / диаметр соединения (ID 31059), Цвет / номер соединения (ID 31004) и Примечание (ID 20045) соответствуют предварительно установленным по умолчанию значениям из настроек проекта, не отображаются.

Для отдельных соединений в верхнем колонтитуле таблицы кабелей отдельные заполнители не указываются. Если область данных содержит свойство Цвет / номер соединения, в верхнем колонтитуле автоматически отображаются свойства Поперечное сечение / диаметр соединения, Единица поперечного сечения / диаметр соединения и Примечание. Свойства с одинаковыми значениями объединяются при этом в один столбец.

Отдельные соединения отображаются при помощи заменяющего символа, и свойство Цвет / номер соединения отображается дополнительно в верхнем колонтитуле.
