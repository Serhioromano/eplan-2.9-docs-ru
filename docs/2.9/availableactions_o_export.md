# Операция: export

**Параметр** |  **Описание**
---|---
TYPE  |  Вид задачи, которую должна выполнить операция:
PXFPROJECT: Экспортировать проект в формате EPJ.
GRAPHICPROJECT: Экспортировать проект в графическом формате (TIF, GIF, PNG, JPG).
GRAPHICPAGE: Экспортировать страницы в графическом формате (TIF, GIF, PNG, JPG).
DXFPROJECT: Экспортировать проект в формате DXF.
DWGPROJECT: Экспортировать проект в формате DWG.
PDFPROJECT: Экспортировать проект в формате PDF. - Устаревший формат, используйте PDFPROJECTSCHEME!
DXFPAGE: Экспортировать страницы в формате DXF.
DWGPAGE: Экспортировать страницы в формате DWG.
PDFPAGE: Экспортировать одну страницу в формате PDF. - Устаревший формат, используйте PDFPAGESSCHEME!
PDFPAGES: Экспортировать несколько страниц в формате PDF. - Устаревший формат, используйте PDFPAGESSCHEME!
PDFPROJECTSCHEME: Экспортировать проект в формате PDF.
PDFPAGESSCHEME: Экспортировать страницы в формате PDF. Используется вместе с параметром PAGENAMEn (например, /PAGENAME1:=EB3+ET1/2 /PAGENAME2:=EB3+ET1/4 /PAGENAME3:=EB3+ET1/7 и т. д.). DXFDWGPROJECTSCHEME: Экспортировать проект в формате DXF или DWG. Все настройки считываются из схемы, в том числе формат DXF или DWG. DXFDWGPPAGESSCHEME: Экспортировать страницы в формате DXF или DWG. Все настройки считываются из схемы, в том числе формат DXF или DWG. Используется вместе с параметром PAGENAMEn (например, /PAGENAME1:=AP+ST1/2 /PAGENAME2:=AP+ST1/4 /PAGENAME3:=AP+ST1/7 и т. д.).
PROJECTNAME  |  Имя проекта с полным путем файла (является опцией).
Если не задано, то выбранный проект используется, когда операция вызывается через интерфейс пользователя (напр., через сценарий или панель инструментов). При вызове из командной строки Windows следует определить PROJECTNAME или сначала следует использовать ProjectAction. В противном случае отобразится системное сообщение.
EXPORTFILE  |  Если параметр TYPE имеет значение PXFPROJECT, то вместе с EXPORTFILE указывается имя экспортируемого проекта (является опцией). Значение по умолчанию: имя проекта. Система автоматически добавляет расширение имени файла.
Если параметр TYPE установлен на PDFPROJECTSCHEME или PDFPAGESSCHEME, укажите вместе с этим параметром имя PDF-файла (требуется: полное имя и путь файла).
EXPORTMASTERDATA  |  Указывает, должны ли экспортироваться основные данные (необязательно). По умолчанию = 1 (Да). Этот параметр оказывает влияние, только когда параметр TYPE имеет следующее значение: PXFPROJECT.
EXPORTCONNECTIONS  |  Указывает, должны ли экспортироваться соединения (необязательно). По умолчанию = 0 (Нет). Этот параметр оказывает влияние, только когда параметр TYPE имеет следующее значение: PXFPROJECT.
PAGENAME  |  Имя экспортируемой страницы (необязательно). Этот параметр оказывает влияние, только когда параметр TYPE имеет одно из следующих значений: DXFPAGE, DXFPROJECT, DWGPAGE, DWGPROJECT, GRAPHICPAGE, GRAPHICPROJECT.
PAGENAMEn  |  Имена экспортируемых страниц (является опцией). При этом n — это номер, например: /PAGENAME1:=EB3+ET1/2 /PAGENAME2:=EB3+ET1/4 /PAGENAME3:=EB3+ET1/7 и т. д. Эти параметры оказывают влияние, только когда параметр TYPE имеет одно из следующих значений: DXFPAGE, DWGPAGE, PDFPAGES, PDFPAGESSCHEME.
DESTINATIONPATH  |  Целевой каталог. Важность данного параметра.
Для GRAPHICPAGE, GRAPHICPROJECT: каталог, в котором сохраняются графические файлы. Если этого каталога еще нет, то он создается. Под этим каталогом создается новый каталог с именем проекта, в котором сохраняются отдельные графические файлы.
Для DXFPAGE, DXFPROJECT, DWGPAGE, DWGPROJECT: каталог, в который экспортируются файлы (необязательно). Значение по умолчанию — каталог проекта. При совместном применении с параметром "PAGENAMEn" показатель DESTINATIONPATH игнорируется и используется целевой каталог из схемы.
FORMAT  |  Формат вывода: Поддерживаемые форматы экспорта:

* BMP: доступная глубина цвета: 1, 4, 8, 16, 24 и 32
* TIF: доступная глубина цвета: 1, 4, 8, 24 и 32
* GIF: глубина цвета нерелевантна
* PNG: доступная глубина цвета: 1, 4, 8, 16, 24 и 32
* JPG: доступная глубина цвета: 1, 4, 8, 16, 24 и 32

Данный параметр необязателен. Значение по умолчанию: TIF. Этот параметр оказывает влияние, только когда параметр TYPE имеет одно из следующих значений: GRAPHICPAGE, GRAPHICPROJECT.
COLORDEPTH  |  Глубина цвета рисунка. Возможные значения — 1, 8, 16, 24 и 32. Данный параметр необязателен. Значение по умолчанию: 24. Этот параметр оказывает влияние, только когда параметр TYPE имеет одно из следующих значений: GRAPHICPAGE, GRAPHICPROJECT.
IMAGEWIDTH  |  Ширина рисунка в пикселах. Высота рассчитывается автоматически исходя из габаритов страницы. Данный параметр необязателен. Значение по умолчанию: 80. Этот параметр оказывает влияние, только когда параметр TYPE имеет одно из следующих значений: GRAPHICPAGE, GRAPHICPROJECT.
IMAGECOMPRESSION  |  Тип сжатия при выводе в формате Tiff. Данный параметр не влияет на другие форматы вывода. Для сжатия CCITT3-, CCITT4- и RLE- глубина цвета всегда составляет 1, т. е. создается двоичный рисунок. Данный параметр необязателен. Возможные значения LZW, RLE, CCITT3, CCITT4, NONE. Значение по умолчанию: NONE. Этот параметр оказывает влияние, только когда параметр TYPE имеет одно из следующих значений: GRAPHICPAGE, GRAPHICPROJECT.
BLACKWHITE  |  Вывод — черно-белый. Замечание: Это не влияет на формат или размер рисунка, т. е. файлы не уменьшаются. Значение по умолчанию: 1 (черно-белый). Этот параметр также является опцией и оказывает влияние, только когда параметр TYPE имеет одно из следующих значений: GRAPHICPAGE, GRAPHICPROJECT. Параметр можно также использовать для PDFPROJECTSCHEME и PDFPAGESSCHEME, где 1 = черно-белый, 0 = цветной и 2 = оттенки серого.
USEPAGEFILTER  |  Определяет, должны ли использоваться только отфильтрованные страницы или все страницы проекта (необязательно). Этот параметр используется только вместе со значениями GRAPHICPAGE, DXFPAGE и DWGPAGE параметра TYPE, если параметры PAGENAME или PAGENAMEn не используются.
Значение по умолчанию: 0.
EXPORTSCHEME  |  Схема для экспорта DXF / DWG, а также PDF (дополнительно). Если данный параметр отсутствует или пуст (''), то используется последняя использованная схема экспорта DXF / DWG или PDF. Этот параметр оказывает влияние, только когда параметр TYPE имеет одно из следующих значений: DXFPAGE, DXFPROJECT, DWGPAGE, DWGPROJECT, PDFPROJECTSCHEME, PDFPAGESSCHEME.
USEZOOM  |  Если значение составляет 1, то при переходе со страниц навигации к компонентам в файле PDF используется окно фрагмента. Тогда компоненты отображаются в окне фрагмента по центру.
ZOOMLEVEL  |  Если значение USEZOOM установлено на 1, задайте в ZOOMLEVEL требуемое масштабирование в 'мм'. При этом введенное здесь значение соответствует высоте фрагмента изображения, который должен появиться на экране после перехода в увеличенном виде. Меньшее значение (например, 20 мм) приводит соответственно к сильному увеличению представления соответствующего условного обозначения на экране. Можно вводить значения от 1 до 3500.
Если значение USEZOOM составляет 0, то ZOOMLEVEL не влияет на результат.
USESIMPLELINK  |  Если значение составляет 1, то в файле PDF создается только простая связь. Если значение составляет 0, то для всех компонентов в файле PDF доступны "трехходовые переходы".
FASTWEBVIEW  |  Если значение составляет 1, то в файле PDF активируется быстрое веб-отображение.
READONLYEXPORT  |  Если значение составляет 1, то файл PDF будет защищен от записи.
USEPRINTMARGINS  |  Если значение равно 1, при экспорте PDF используются поля для печати (является опцией). Значение по умолчанию: 0. Этот параметр оказывает влияние, только когда параметр TYPE имеет одно из следующих значений: PDFPAGE, PDFPAGES und PDFPROJECT.
TARGET  |  Указывает средство экспорта. Параметр может иметь следующие значения: "Disk", "EMail", "FromSettings". При написании значений регистр можно не учитывать. Значение по умолчанию: 'Disk'. Если ничего не указано, экспорт выполняется на носитель данных.
Если настроено значение 'FromSettings', с помощью настроек (схема) устанавливается цель экспорта (каталог вывода). Другими словами, в этом случае определяют настройки, выполняется экспорт через электронную почту или на носитель данных. Если настроено значение 'Disk', каталог вывода устанавливается параметром DESTINATIONPATH.
Этот параметр оказывает влияние, только когда параметр TYPE имеет одно из следующих значений: DXFPROJECT, DXFPAGE, DWGPROJECT, DWGPAGE. Если страницы в формате DXF или DWG (/TYPE:DXFPAGE, /TYPE:DWGPAGE) экспортируются с параметром PAGENAMEn, для параметра TARGET автоматически устанавливается значение "FromSettings".
LANGUAGE  |  Код языка. Задает язык вывода для экспорта. Замечание: При вводе этого параметра необходимо учитывать регистр. Правильные коды языков: en_US, de_DE и т. д. Этот параметр оказывает влияние, только когда параметр TYPE имеет одно из следующих значений: DXFPROJECT, DXFPAGE, DWGPROJECT, DWGPAGE.
SPLITSIZE  |  С помощью значения этого параметра можно перезаписать настройку Ограничение размера. При этом для параметра TARGET должно быть установлено значение 'EMail'. Этот параметр оказывает влияние, только когда параметр TYPE имеет одно из следующих значений: DXFPROJECT, DXFPAGE, DWGPROJECT, DWGPAGE.
EXPORTMODEL  |  Булев параметр. Если значение равно 1, то в проекте экспортируются также трехмерные модели пространств листа. Обратите внимание на то, что в схеме экспорта есть и другие настройки, которые влияют на экспорт трехмерных моделей. Этот параметр оказывает влияние, только когда параметр TYPE имеет одно из следующих значений: PDFPROJECTSCHEME, PDFPAGESSCHEME.

!!! note "Замечание:"

    * Если страница указана в параметре PAGENAME, то экспортируется только эта страница, а параметр USEPAGEFILTER игнорируется.
    * Если страница в параметре PAGENAME не указана, то определяются страницы проекта. Если параметр USEPAGEFILTER установлен на 1 (то есть активирован фильтр страниц), то в навигаторе страниц экспортируются только отфильтрованные страницы. Если параметр USEPAGEFILTER не используется или установлен на 0, экспортируются все страницы проекта.
    * Если параметр TYPE = DXFPAGE, DXFPROJECT, DWGPAGE или DWGPROJECT и параметр TARGET не установлен, то цель вывода копируется из настроек, а параметр DESTINATIONPATH игнорируется.
