# Конфигурировать представления в виде списка и структуры дерева

В ***представлении в виде списка*** стандартно выводится много информации о соответствующих объектах. Для вывода действительно необходимой информации по устройствам, изделиям, соединениям и т. д. каждый список можно сконфигурировать отдельно. При этом в диалоговом окне выбора свойств определите столбцы, которые будут представлены в виде списка.

В ***представлениях структуры дерева*** большинства навигаторов и диалоговых окон для изделий на экран может выводиться дополнительная информация для быстрой идентификации объектов и изделий. Представление этой дополнительной информации можно также сконфигурировать.

Конфигурация с выводимой информацией (свойствами) сохраняется в виде схемы как для представления в виде списка, так и для представления структуры дерева и в любой момент доступна для повторного использования или обработки.

!!! example "Пример:"

    За счетконфигурациипредставления структуры дерева возможно следующее:***Навигатор страниц:***Отображение несущественной дополнительной информации (напр., описание страницы) можно подавлять.***Навигаторы данных проекта:***Для уровней структуры дерева***устройств***(пиктограмма) и***функций***(напр., пиктограмма) можно выводить дополнительную информацию, например функциональный текст или технический параметр.На***функциях***с присвоенным объектом планирования или технологическим контуром можно отображатьобозначенияэтих сегментов предварительного планирования. После номеров изделий можно отобразить другую информацию, например описание изделий избазы данных изделий.***Навигатор макросов:***В представлении структуры дерева навигатора макросов можно, например, отобразить тип макроса и вид представления подготовленных макросов. После генерируемых макросов страницы можно отобразить описание страницы.***Диалоговое окно "Выбор ОУ":***Представление структуры дерева можно расширить для каждого объекта отображением дополнительной информации, например функционального текста.***База данных изделий, выбор изделия и навигатор основных данных изделий:***В представлениях структуры дерева для изделий, помимо обозначения изделия, также можно отобразить другиесвойствав дополнение к номеру изделия (напр.,Номер ERP,Номер типаи т. д.). Для вариантов и других уровней структуры дерева (список принадлежностей, схемы сверления и т. д.) можно не конфигурировать представление.

### Конфигурировать представления в виде списка

Условие:

Вы открыли проект и находитесь в представлении навигатора в виде списка, в представлении диалогового окне Выбор символа в виде списка, в таблице данных диалогового окна Синхронизировать устройства.

1. Выберите пункт всплывающего меню Конфигурировать представление.

!!! info "Для сведения:"

    В диалоговом окне Конфигурировать представление в поле Конфигурация отображаются все свойства, используемые на данный момент для представления в виде списка.

2. В поле Схема выберите уже имеющуюся схему или создайте новую.
3. В поле Конфигурация щелкните ![](images/all_new_as.png){: .ui-icon } (Создать), чтобы к выбранной схеме добавить свойство для конфигурации.
4. В диалоговом окне Выбор свойств выберите требуемое свойство. Возможен многократный выбор.

!!! info "Для сведения:"

    Вставленные свойства добавляются в конце списка.

5. Используя кнопки со стрелками, задайте последовательность, в которой будут отображаться свойства.
6. Щелкните по кнопке ++OK++.

!!! info "Для сведения:"

    Закрывается диалоговое окно Конфигурировать представление, и представление обновляется.

7. При необходимости выровняйте каждый столбец по ширине.

Заданная ширина столбцов представления в виде списка в навигаторах автоматически сохраняется как при закрытии диалогового окна навигатора, так и при выборе пункта всплывающего меню Конфигурировать представление. Таким образом однажды заданная ширина столбцов сохраняется и восстанавливается при повторном выборе схемы для конфигурации представления.

!!! note "Замечание:"

    * Заданная ширина столбцов представления в виде списка сохраняется в диалоговой памяти отдельно для каждого пользователя. Это означает, что даже при обмене данными схемы конфигурации для представления в виде списка каждый пользователь сам для себя определяет ширину столбцов.
    * Определенные свойства ***всегда*** отображаются. Такие свойства ***невозможно*** удалить из списка в поле Конфигурация. Таким образом, например, свойство Имя проекта отображается в представлении в виде списка всегда в виде столбца, если открыто более одного проекта или, наоборот, не открыт ни один проект.

### Конфигурировать представление структуры дерева для навигаторов

Условие:

Вы открыли проект и находитесь в представлении структуры дерева определенных навигаторов.

1. Выберите пункт всплывающего меню Конфигурировать представление.

!!! info "Для сведения:"

    В таблице Конфигурация диалогового окна Конфигурировать представление перечисляются объекты (проекты, страницы, устройства, функции, структурные идентификаторы и т. д.), которые можно отобразить в соответствующем дереве.

2. В поле Схема выберите уже имеющуюся схему или создайте новую.
3. Щелкните в столбце Формат блока для объекта и с помощью кнопки ++...++ перейдите в диалоговое окно Формат, чтобы изменить форматирование уже выбранного свойства или добавить следующие свойства для соответствующего объекта.

!!! info "Для сведения:"

    Форматирование свойств для объектов представления структуры дерева выполняется так же, как и [определение свойств блока](blockproperties_h_blockeigenschaftendefinieren.md).

4. После того как в диалоговом окне Формат будут добавлены элементы формата (свойства, разделители), будет определена их последовательность или будет обработан формат уже выбранных элементов формата, щелкните ++OK++.

!!! info "Для сведения:"

    На этом конфигурация представления структуры дерева для этого объекта завершена. В диалоговом окне Конфигурировать представление соответствующая ячейка столбца Формат блока с предварительным просмотром заданного формата заполнена.

5. Чтобы при необходимости сконфигурировать представление структуры дерева для других объектов, повторите приведенные выше шаги.
6. Щелкните по кнопке ++OK++.

!!! info "Для сведения:"

    Закрывается диалоговое окно Конфигурировать представление, и представление обновляется.

!!! note "Замечание:"

    Определенные свойства ***всегда*** отображаются. Например, свойство Имя проекта также отображается в представлении структуры дерева, если для столбца Формат блока не задан формат.
