# Диалоговое окно Адреса / списки присвоения

Вы открыли проект. Данные проекта > ПЛК > Адреса / Списки присвоений.

В этом диалоговом окне управляются все входы и выходы использованных в проекте блоков управления ПЛК. Чтобы можно было различать несколько блоков управления ПЛК в составе одного проекта, адреса присваиваются одному процессу (т.е. ЦПУ). В списке затем выводится, к какому процессору относится вывод устройства. ЦПУ идентифицируется путем указания имени ЦПУ. Полное имя ЦПУ состоит из содержаний свойств Проект конфигураций и Имя станции, а также идентификатора ЦПУ в форме [Проект конфигураций].[Имя станции].[Идентификатор ЦПУ].

* Вы можете определить новые адреса, вставляя пустые строки и заполняя их данными.
* Новые символические адреса можно определить, вставляя пустые строки и заполняя их данными. При этом необходимо дополнительно заполнить как минимум тип данных.
* С помощью команд "Копировать" и "Вставить" можно обмениваться данными с таблицей Excel.
* Кроме того, можно импортировать списки присвоений и синхронизировать со схемой соединений, а также создавать новые списки присвоений.

Обзор основных элементов диалогового окна:

### Фильтр

В этом раскрывающемся списке отображаются все доступные фильтры. Выбранный фильтр активируется автоматически. Запись "- Не активировано -" отключает фильтр и приводит к тому, что данные отображаются в неотфильтрованном виде. С помощью кнопки ++...++ откройте диалоговое окно [Фильтр](modaldialogsdb_d_filternnach.md). Здесь можно создать, обработать, удалить, скопировать, экспортировать, импортировать фильтр и управлять им.

Во всплывающем меню раскрывающегося списка Фильтр содержатся следующие записи:

* Выключить: Этот пункт меню доступен, если фильтр установлен: Сбрасывает настройку фильтра до записи "- Не активировано -".
* Активировать <Имя фильтра>: Этот пункт меню доступен, если для настройки фильтра установлено значение "- Не активировано -": Повторно активирует последний активный фильтр.

Таким образом можно быстро переключаться между неотфильтрованным и отфильтрованным в соответствии с требованиями пользователя представлениями.

При помощи фильтра вы можете ограничить отображение адресов ПЛК одним (или несколькими) определенным ЦПУ.

!!! tip "Совет:"

    При помощи пункта всплывающего меню Создать фильтр для ЦПУ в таблице вы можете автоматически сгенерировать из данных выделенной строки фильтр для отображения адресов ПЛК. При этом в качестве критерия фильтра используются значения свойств Проект конфигураций (косвен.), Имя рабочей станции ПЛК (косвен.) и ЦПУ (косвен.).

* * *

Область таблицы "Данные ПЛК из схемы соединений"

В таблице в области Данные ПЛК из схемы соединений выводятся все адреса в соответствии с выбранным фильтром с указанием следующих свойств:

* Проект конфигураций (косвен.): Показывает проект конфигураций, которому присвоен вывод устройства. Этот столбец можно дополнительно вывести через диалоговое окно Конфигурировать представление.
* Имя рабочей станции ПЛК (косвен.): Показывает рабочую станцию, которой присвоен вывод устройства. Этот столбец можно дополнительно вывести через диалоговое окно Конфигурировать представление.
* ЦПУ (косвен.): Здесь выводится процессор, которому присвоен вывод устройства (если на блоке ПЛК указано ЦПУ).
* Адрес ПЛК: Здесь выводится адрес вывода устройства ПЛК.
* Тип данных: Здесь отображается тип данных.
* Символический адрес: Показывает символический адрес, который был вручную записан на выводе устройства ПЛК (если таковой имеется).
* Символический адрес (автоматич.): Отображает содержание введенного вручную символического адреса или, при его отсутствии, определенный символический адрес.
* Символический адрес (определен): Отображает символический адрес, определенный посредством отслеживания цели.
* Символический адрес: Группа: Показывает присвоение символического адреса группе. Эта группа может использоваться для группировки диапазонов символических адресов в списке присвоений, например, входов, выходов, адресов (отказоуст.) и т. д.
* Функциональный текст: Показывает функциональный текст, который был вручную введен на выводе устройства ПЛК (если таковой имеется).
* Функциональный текст (автоматич.): Отображает автоматический функциональный текст. Если отсутствует введенный вручную функциональный текст, здесь отображается функциональный текст, определенный посредством отслеживания цели.
* ОУ (идентифицирующее): Здесь выводится идентифицирующее обозначение вывода устройства ПЛК. Это поле заполняется автоматически при размещении вывода устройства.
* Обозначение вывода устройства (с обозначением штекера (автоматически)): Составное свойство из обозначения штекера (автоматически) и обозначения вывода устройства ПЛК. Это поле заполняется автоматически при размещении вывода устройства.
* Направление: Указывает на направление (вход, выход, не определено) адреса.
* Выделено для удаления: Этот столбец можно дополнительно вывести через диалоговое окно Конфигурировать представление. В нем отображается, отмечен ли какой-либо адрес для удаления.
* Имя (идентифицирующее): Идентифицирующее имя функции, состоит из идентифицирующего обозначения устройства, а также (для выводов устройства) из обозначения вывода устройства.
* Предварительное планирование: Выводит объект планирования, которому присвоен адрес ПЛК. Этот столбец можно дополнительно вывести через диалоговое окно Конфигурировать представление.
* Обозначение канала: Отображает обозначение канала ПЛК, которому присвоен вывод устройства ПЛК. Этот столбец можно дополнительно вывести через диалоговое окно Конфигурировать представление. Отображение обозначения канала позволяет распознавать присвоение функциональных текстов каналу.

* * *

Область таблицы "Новые данные ПЛК / Из списка присвоений"

В области Новые данные ПЛК / Из списка присвоений можно изменить определенные данные ПЛК. Если вы импортировали список присвоений, в этой таблице показываются импортированные данные, и вы можете принять их или изменить. Если старые и новые значения отличаются, соответствующие строки выделяются цветом. При этом новые значения обозначаются желтым цветом, а старые — серым фоном. Можно изменять следующие свойства:

* Адрес ПЛК: Здесь выводится адрес вывода устройства ПЛК. Адрес можно изменять вручную или при помощи новой адресации.
* Тип данных: Здесь отображается тип данных; можно выбирать другую запись из раскрывающегося списка или перезаписать запись.
* Символический адрес: Здесь выводится автоматический символический адрес. Можно перезаписать запись; затем изменение переносится в ручной символический адрес.
* Символический адрес: Группа: Это свойство может использоваться для группировки диапазонов символических адресов в списке присвоений, например, входов, выходов, адресов (отказоуст.) и т. д. Для этого введите здесь произвольно выбираемое обозначение. Свойство используется при обмене данными ПЛК в формате AutomationML.
* Функциональный текст: Здесь выводится автоматический функциональный текст. Можно перезаписать запись; затем изменение переносится в ручной функциональный текст. Если функциональный текст был определен из функционального текста зоны или из текста функции датчика или исполнительного элемента, изменение текста функции зоны или датчика / исполнительного элемента исправляется.
* Выделено для удаления: Этот столбец можно дополнительно вывести через диалоговое окно Конфигурировать представление. В нем отображается, отмечен ли какой-либо адрес для удаления. Снятие этого флажка позволяет отменить данное обозначение.

### Всплывающее меню

Всплывающее меню дает доступ, в зависимости от типа поля (например, дата, целое число, многоязычный), к пунктам меню, при помощи которых вы можете по необходимости, например, влиять на представление таблиц или обрабатывать значения в полях. Обзор пунктов этого всплывающего меню вы можете найти в разделе [Пункты всплывающего меню](userinterface_m_kontextmenu.md).

Дополнительно здесь представлены следующие пункты всплывающего меню, специфические для данного диалогового окна:

Пункт меню |  Значение
---|---
Вставить строку |  Вставляет перед выделенной строкой новую строку.
Удалить строку |  Удаляет выделенные строки. Установленные в проекте адреса не могут удаляться.
Адресация |  Открывает диалоговое окно Присвоить адреса ПЛК повторно.
Генерировать с устройством |  Генерирует неразмещенные устройства для выделенных адресов при помощи выбора устройства.
Оставить старое значение |  После изменения содержимого ячейки копирует старое значение.
Сохранить новые строки в проекте |  Переносит старый функциональный текст с имеющимися новыми строками в поле нового функционального текста, если указанные функциональные тексты идентичны (включая новые строки и пробелы). Так можно восстановить разделение на строки, удаленное при импорте списка присвоений. Содержащиеся в проекте возможные переводы также сохраняются.
Создать фильтр для ЦПУ |  Из данных выделенной строки автоматически генерирует фильтр для отображения адресов ПЛК. При этом в качестве критерия фильтрации используются значения свойств Проект конфигураций (косвен.), Имя раб. станции (косвенно) и ЦПУ (косвенно).

### [Дополнительно]

Пункт меню |  Значение
---|---
Импортировать список присвоений |  Импортирует список присвоений. При помощи пунктов подменю задайте, должен ли импорт адресов ПЛК осуществляться в текущее отображаемое ЦПУ или в новое ЦПУ:

* Отображаемое ЦПУ: Открывает диалоговое окно Импортировать / синхронизировать список присвоений.
* Новое ЦПУ: Открывает диалоговое окно Новое ЦПУ, в котором можно создать новое ЦПУ. После этого открывается диалоговое окно Импортировать / синхронизировать список присвоений.

Экспортировать список присвоений |  Открывает диалоговое окно Экспортировать список присвоений.
Экспортировать распределение адресного пространства |  Открывает диалоговое окно Экспортировать распределение адресного пространства и позволяет экспортировать распределение адресного пространства ЦПУ в файл в виде списка.
Настройки |  Открывает диалоговое окно Настройки: ПЛК.

!!! note "Замечание:"

    Вы можете экспортировать или импортировать всякий раз только отдельный список для каждого процессора (ЦП). Поэтому настройте фильтр таким образом, чтобы отображались только адреса ПЛК ***одного*** ЦПУ.
