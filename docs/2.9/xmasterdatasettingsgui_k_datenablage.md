# Основные данные: Архив данных

В архиве основных данных необходимо, как правило, различать архив ***Основные данные проекта*** и архив ***Основные данные системы***.

### Архив данных Основные данные проекта

Как правило, на принцип работы для основных данных проекта ***есть ссылка***, причем основные данные сохраняются в проекте, для чего создается ***копия***, например библиотеки символов в проекте, которая используется в дальнейшем.
Основные данные проекта, таким образом, архивируются, связываясь с проектом. Они автоматически синхронизируются при открытии проекта, а также при резервировании / восстановлении с пулом основных данных, в случае если флажок Синхронизировать основные данные проекта при открытии (в меню Параметры > Настройки > Проекты > "Имя проекта" > Управление > Общее) установлен.

Так как при таком режиме работы необходимо учитывать немного больший объем данных, как правило необходимо, чтобы емкость жесткого диска была достаточной и требуемые основные данные занимали относительно немного места.

Ссылки относятся к связанным с проектом копиям основных данных, например, библиотек символов в базе данных проекта, рамок в каталоге проекта и т.д. Здесь не включены макросы и иностранные языки; последние сохраняются в схеме соединений в виде файлов или слов.

### Архив данных Основные данные системы

Макросы, формы и рамки сохраняются как отдельные файлы. Форматом является текстовый формат XML. Библиотеки символов, напротив, состоят из нескольких двоичных файлов в одной структуре каталога.

Отдельные файлы обрабатываются редактором символов, форм и рамок, причем EPLAN при сохранении в проекты всегда работает с отдельными файлами или с библиотекой.

С целью документирования, отдельные файлы могут считываться отдельно в так называемые проекты документации (Сервисные программы > Отчеты > Генерировать).

!!! note "Замечание:"

    ***Нет*** автоматически запускающихся функций для синхронизации между отдельными файлами и проектом основных данных, все операции необходимо запускать вручную. Это дает возможность, с одной стороны, структурировать файлы в каталогах и, с другой стороны, просто управлять отдельными файлами (обновление, копирование, отправка ...).
