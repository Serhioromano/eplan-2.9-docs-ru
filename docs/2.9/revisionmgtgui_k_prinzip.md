# Управление ревизиями: Принцип

Ревизии проектов служат для того, чтобы охватывать изменения, касающиеся какого-нибудь конкретного состояния проекта. Изменения можно пометить при помощи ***Отслеживания изменений*** в проекте для сравнения или использовать ***Сравнение свойств проектов***, чтобы сравнить текущий проект с проектом для сравнения.

Изменение всегда указывается для ***страницы*** или ***Пространства листа***, если распечатка страницы или пространства листа оптически отличается от последней версии. Изменение ***проекта*** указывается, если данные, видимые или невидимые, напр. свойства устройств или номера изделий, изменяются. Соответствующие страницы отчетов затем оптически отличаются. Вы можете самостоятельно настроить в управлении ревизиями, какие изменения данных можно распознать в проектах для сравнения в качестве нового статуса ревизии.

Ревизионные проекты, закрытые проекты и проекты для сравнения отображаются в навигаторе страниц так же, как другие проекты. Дополнительно статус проекта отображается пиктограммой в дереве:

Пиктограмма |  Значение
---|---

![](images/revisionmgtgui_graystamp_as.png){: .ui-icon } |  Ревизионный проект, открытый для обработки.

### Создать ревизионный проект

Ревизионный проект — это проект, открытый для обработки с отслеживанием изменений. Он создается через пункты меню Сервисные программы > Управление ревизиями > Отслеживание изменений > Генерировать ревизию. При этом первоначальный проект *.elk сохраняется с расширением имени файла *.ell.

### Создать исходный проект

Исходный проект — это защищенная от записи копия проекта, созданная в определенный момент времени. Он сохраняется с расширением имени файла *.elr и, таким образом, является закрытым проектом. Первоначальный проект *.elk при создании проекта для сравнения сохраняется. Дополнительно создается копия проекта в качестве закрытого проекта с расширением имени файла *.elr. Автор и дата создания проекта для сравнения сохраняются автоматически в качестве свойств проекта.

Исходный проект можно создать двумя способами:

* Вручную через пункты меню Сервисные программы > Управление ревизиями > Сравнение свойств проектов > Сгенерировать исходный проект. Вы работаете в исходном проекте дальше и в определенный момент выполняете сравнение с исходным проектом.
* Автоматически при создании ревизии к закрытому проекту. Для этого в настройках проекта в пунктах меню Параметры > Настройки > Проекты > "Имя проекта" > Управление > Ревизия (отслеживание изменений) нужно активировать опцию При новой ревизии закрытого проекта сгенерировать исходный проект.

### Обработка ревизионных проектов

В случае изменения страницы ревизионного проекта она обозначается как 'План'. Графически это указывается водяным знаком на странице, и странице присваивается свойство Страница в режиме плана. Это обозначение сохраняется, пока вы не закроете страницу.

Позиция водяного знака при этом может задаваться свободно. Для этого в редакторе рамок через пункты меню Вставить > Спец. текст > Водяной знак (управление ревизиями) разместите особый специальный текст, который автоматически размещается на соответствующем слое, и отформатируйте его. Если затем EPLAN должна активировать на странице этот водяной знак, то сначала ищется спец. текст и принимаются его свойства. Если форматирование не задано, то водяной знак размещается в середине страницы. (Текст, который должен применяться в виде водяных знаков, задайте в настройках в пунктах меню Параметры > Настройки > Проекты > "Имя проекта" > Управление > Ревизия (отслеживание изменений).)

Если в ревизионном проекте переместить курсор на измененный объект, на просмотр выводится всплывающая подсказка, в которой содержится информация о ревизии. Для облегчения поиска измененных объектов они отображаются на странице с помощью графического маркера.

#### Вставка страниц или макросов страницы с данными ревизии

Данными ревизии являются сведения, вводимые автоматически или вручную после закрытия страниц, проектов или областей обработки, например причина изменения, имя ревизии, индекс ревизии, автор ревизии, дата ревизии и т. д. Данные ревизии отображаются в свойствах страницы и в диалоговом окне Дополнительно обработать данные ревизии, а также выводятся в обзорах ревизии.

При копировании и вставке страниц из проектов ревизии или при вставке макросов страницы, содержащих данные ревизии, эти данные ревизии и маркеры ревизии можно сохранить или удалить. Требуемые действия задаются через настройку пользователя Сохранить данные и маркеры ревизии при вставке (путь меню: Параметры > Настройки > Пользователь > Представление > Общее).
