# Подчиненные формы

***Динамические формы*** в EPLAN предоставляют возможность определения "Подформ", к которым привязываются условия. Благодаря этому возникает так называемая "Главная форма" и присвоенная ей "Подчиненная форма". При этом подчиненные формы должны быть того же типа, что и главная форма.

Подчиненные формы доступны для всех отчетов, относящихся к функции, для которых можно определить фильтр функции, за исключением схем подключения. Т. е. следующие типы отчетов ***не*** поддерживают использование подчиненных форм:

* Перечень форм
* Схема подключения устройства
* Схема подключения кабеля
* Схема подключения клемм
* Перечень рамок
* Обзор ревизий
* Схема подключения штекера
* Титульный лист
* Схема кабельных соединений.

Подчиненные формы оцениваются только однократно; если у самой подчиненной формы есть другие подформы, то они не учитываются.

Подчиненные формы присваиваются объекту через подходящий фильтр, причем фильтр анализирует выбранные свойства.

Если в динамической форме определяются другие формы с условиями фильтрации, они анализируются следующим образом:
Для каждого объекта проверяется, подходит ли условие фильтрации одной или нескольким подформам. Все подходящие формы анализируются по порядку (последовательность в диалоговом окне), причем анализируются только динамические области из подформы. Свойства формы, графика вне областей и верхние колонтитулы игнорируются и далее используются из главной формы.
Если для объекта не найдено подформы, то данный объект анализируется с главной формой.

Если главная форма сохранена в проекте, то все подформы также сохраняются. Подформы также учитываются при синхронизации и при сжатии данных.
