# Определения трубопровода на функциональной схеме автоматизации

Определение трубопровода позволяет установить связь определенной последовательности условных обозначений, символов соединения и линий автоматического соединения на функциональной схеме автоматизации с трубопроводом.

### Вставить точки определения трубопроводов на функциональной схеме автоматизации

Для определения трубопроводов с одной стороны можно использовать объекты планирования (трубопроводы) ***из предварительного планирования***, которые потом размещаются из навигатора предварительного планирования на функциональной схеме автоматизации. При этом появится точка определения трубопровода со ссылкой на объект планирования (трубопровод). Точке определения трубопровода присваиваются данные объекта планирования (трубопровода).

Точки определения трубопроводов также можно вставлять на странице функциональной схемы автоматизации напрямую (без присвоения их объекту планирования, т. е. ***без предварительного планирования***). В этом случае элементы сначала не будут присваиваться объекту планирования. Но через некоторое время можно присвоить объект планирования (трубопровод) точке определения трубопровода.

С помощью вывода трубопровода можно разместить определение трубопровода в начале линии автоматического соединения. С выводом трубопровода объект планирования (трубопровод) также можно присвоить впоследствии.

### Способ действий

Для определения трубопровода разместите точку определения трубопровода на линии автоматического соединения на функциональной схеме автоматизации. Если объект планирования не присвоен, в диалоговом окне "Свойства" этой точки определения трубопровода задайте имя трубопровода, класс трубы, а также вещество, транспортируемое по этому трубопроводу. С помощью этой точки определения трубопровода можно отобразить трассу трубопровода на технологической схеме и обработать ее.

Если на функциональной схеме автоматизации содержатся устройства (теплообменники, запорные арматуры и т. д.), на которых изменяется состояние вещества, можно определить их состояния с помощью дополнительных точек определения трубопровода. Для этого на линиях автоматического соединения перед соответствующим устройством и после него разместите дополнительные точки определения трубопровода с данными соответствующего трубопровода (то же имя трубопровода, класс трубы и т. д.) и укажите требуемое состояние.

### Трасса и источник / цель трубопровода

Трасса трубопровода, т. е. относящиеся к трубопроводу соединения, определяется автоматически и задается путем передачи потенциала. При этом трасса трубопровода выходит за рамки условного обозначения, если на логической схеме выводов устройства функции задана подходящая передача потенциала.

Источник и цель определенного трубопровода определяются согласно тем же правилам, что действовали для соединений. Чтобы поменять местами источник и цель трубопровода, можно воспользоваться следующими вариантами:

* На связанном объекте планирования трубопровода в навигаторе предварительного планирования через пункты меню Данные проекта > Соединения > Заменить источник и цель:
Для точки определения трубопровода, которая в дереве навигатора размещена под объектом планирования трубопровода, направление потока в трубопроводе будет скорректировано на функциональной схеме автоматизации, если источник и цель на связанном объекте планирования трубопровода были указаны в предварительном планировании. Кроме того, исходные или целевые функции трубопровода на функциональной схеме автоматизации должны быть связаны с объектами планирования, которые соответствуют исходным или целевым сегментам на объекте планирования трубопровода.
* На точке определения трубопровода в навигаторе предварительного планирования или на функциональной схеме автоматизации с помощью свойства Трубопровод: Заменить источник и цель (ид. 33301):
Это возможно, только если источник и цель на связанном объекте планирования трубопровода не были полностью заданы в предварительном планировании.

Если источник и / или цель на функциональной схеме автоматизации не согласуются с источником и / или целью присвоенного трубопровода из предварительного планирования, это можно проверить при помощи контрольного прогона "028027".

### Выделить и обработать трассу трубопровода

Чтобы временно выделить цветом заданные трубопроводы на функциональной схеме автоматизации и при необходимости обработать трассу трубопроводов, можно использовать пункты меню Данные проекта > Соединения > Трасса трубопровода.

При выборе точки определения трубопровода с помощью пункта меню Трасса трубопровода направление потока в трубопроводе определяется по источнику и цели в точке определения трубопровода. Везде, где на концах участков соединения источник и цель не будут соответствовать направлению потока в трубопроводе, автоматически размещаются точки определения соединения, для которых активировано свойство Поменять источник и цель. Таким образом, направление потока переносится на весь трубопровод.

С помощью двух специальных символов из библиотеки символов SPECIAL направления потока трубопроводов также могут отображаться постоянно. Для получения дополнительной информации см. советы в разделе "[Выделить и обработать трассу трубопровода](planningri_h_rohrleitungmarkieren.md)".

### Прерывания трубопроводов

Трубопроводы на функциональной схеме автоматизации согласно стандарту DIN 28000-4:2014-07 отображаются с прерываниями, когда профиль пересекается с другим символом (например, резервуара) или трубопроводом. Чтобы вставить такие прерывания на функциональную схему автоматизации, в библиотеке символов SPECIAL доступен символ соединения PBR.

### Отчеты для трубопроводов на функциональной схеме автоматизации

Трубопроводы, определенные для проекта на функциональной схеме автоматизации, можно вывести с помощью типа отчета "Функциональная схема автоматизации: Обзор трубопроводов" в виде списка.

Все находящиеся на странице функциональной схемы автоматизации трубопроводы, вещества и классы труб в определенных типах отчетов могут размещаться вручную в качестве встроенных отчетов на соответствующей функциональной схеме автоматизации. Это возможно для следующих типов отчетов:

* Функциональная схема автоматизации: Обзор трубопровода
* Предварительное планирование: Обзор категории трубы
* Предварительное планирование: Обзор среды.

Чтобы в таком отчете учитывались только данные текущей страницы функциональной схемы автоматизации, необходимо активировать настройку Текущая страница в диалоговом окне Определить отчет для формы вывода "Размещение вручную".
