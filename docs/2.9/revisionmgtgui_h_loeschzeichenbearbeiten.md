# Обработать маркер удаления в проектах ревизии

Если вы создали ревизию проекта, то при активации настройки проекта При удалении сгенерировать маркер удаления и перечислить удаленные страницы удаленные объекты на страницах проекта ревизии помечаются маркерами ревизии и маркерами удаления. Свойства удаленных объектов сохраняются на маркерах удаления, где их можно просматривать.

Условие:

Вы открыли проект ревизии (расширение имени файла *.ell), содержащий маркеры удаления, и находитесь на странице схем соединений в графическом редакторе.

1. Выделите маркер удаления и переместите его в графический редактор.

!!! info "Для сведения:"

    Маркер удаления получает новый маркер ревизии. При этом цвет измененных объектов указывает пользователю на то, что речь идет о перемещенном объекте. Кроме того, текст маркера определяется заново и при необходимости обновляется.

Для маркера удаления по умолчанию отображается всплывающая подсказка с автоматически созданным текстом маркировки ревизии. Этот текст нельзя дополнительно обработать.

1. Выделите маркер удаления и выберите пункт всплывающего меню Обработать маркеры ревизии.
2. Измените текст маркера ревизии в диалоговом окне [Обработать маркеры ревизии](revisionlogmgtgui_d_markierungstexte.md).
3. Щелкните по кнопке ++OK++.

1. Выделите маркер удаления и выберите пункт всплывающего меню Удалить окончательно.

!!! info "Для сведения:"

    Сведения об удаленном объекте будут окончательно удалены из проекта.

!!! note "Замечание:"

    Если удалять маркеры ревизии и маркеры удаления удаленного объекта обычным способом в графическом редакторе (например, через пункт всплывающего меню Удалить), то удаленные объекты заменяются на идентичные новые объекты.

!!! tip "Совет:"

    Дополнительно имеется возможность удаления всех имеющихся в проекте маркеров ревизий и маркеров удаления, которые были созданы при отслеживании изменений. Для этого выберите Сервисные программы > Управление ревизиями > Отслеживание изменений > Удалить ревизию.

    При ревизии с ***отслеживанием изменений*** удаленные страницы рассматриваются как данные. Чтобы их удалить, в диалоговом окне Удалить ревизию установите флажок Данные ревизии из отслеживания изменений.
