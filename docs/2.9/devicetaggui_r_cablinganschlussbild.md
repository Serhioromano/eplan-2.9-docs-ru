# Вкладка Схема соединений (топология)

Вы открыли проект.

* Вы добавили функцию топологии в Графическом редакторе или сгенерировали в навигаторе топологии. В диалоговом окне Свойства ++...++ выберите вкладку Схема соединений.
* Вы выделили функцию топологии в Графическом редакторе или в навигаторе топологии. Обработать > Свойства. Выберите вкладку Схема соединений.

В этой вкладке показаны данные выводов устройства, которые присваиваются функции топологии через изделие. Данные выводов устройства могут быть определены на изделии в присвоенной схеме соединений или в шаблонах функций. Эти данные выводов устройства можно сохранять и изменять локально в функции топологии.

Обзор основных элементов диалогового окна:

### Схема

Выберите из раскрывающегося списка схему, которую требуется использовать. Или щелкните по кнопке ++...++, чтобы в диалоговом окне Настройки: Конфигурация столбцов обработать схему или создать новую. Схема определяет, какие данные вывода устройства и в каком порядке будут отображаться. Так можно скрыть ненужные столбцы.

### Локальная схема соединений

Если этот флажок установлен, данные выводов устройства сохраняются локально в функции топологии. Если изделию присвоена схема соединений или на изделии сохранены шаблоны функций, данные выводов устройства копируются, а защита от записи снимается. После этого данные выводов устройства можно обрабатывать в таблице. При синхронизации изделий изменения, вносимые вручную, остаются и не перезаписываются.

Если флажок снят, используются данные выводов устройства, которые определены на изделии. В этом случае данные выводов устройства защищены от записи. Вносить изменения можно только в поля Обозначение вывода устройства, Обозначение штекера и Интерфейс шины: Имя, а для клемм — также в поля Уровень и Внутр./внешн. индекс. Эти "идентифицирующие" поля используются для идентификации вывода устройства в базе данных изделий: Если значения в этих полях для вывода устройства, определенного на изделии, и вывода устройства, имеющегося в функции топологии, совпадают, речь идет об одном и том же выводе устройства.

### Таблица

В таблице в зависимости от выбранной схемы отображаются специфические данные выводов устройства, которые относятся к схеме соединений.

* Строка: Непрерывный номер строки.
* Статус: В этом столбце отображается, отличается ли схема соединений на функции топологии от схемы соединений присвоенного изделия. Пиктограммы расшифровываются следующим образом:

Кнопка |  Значение
---|---
![](images/all_statusok_as.png){: .ui-icon } |  Обозначает вывод устройства, который копируется из базы данных изделий без изменений. На локальной схеме соединений все выводы устройства имеют этот идентификатор.
![](images/all_statusnotok_as.png){: .ui-icon } |  Обозначает вывод устройства, определенный в изделии, но отсутствующий на функции топологии. Такой вывод устройства появляется, когда вы изменяете записи в "идентифицирующих полях" для какого-либо вывода устройства, скопированного из изделия. В этом случае первоначальный вывод устройства, определенный на изделии, отображается с таким идентификатором.
![](images/erroricon_error_systemmessages.png){: .ui-icon } |  Обозначает вывод устройства, определенный в функции топологии, но отсутствующий в изделии. Такой вывод устройства появляется, когда вы добавляете новый вывод устройства или изменяете записи в 'идентифицирующих' полях для какого-либо вывода устройства, скопированного из изделия.
* Обозначение вывода устройства: Введите здесь имя позиции вывода устройства.
* Обозначение штекера: Введите здесь ОУ штекера, с которым связан вывод устройства.
* Интерфейс шины: Имя: Для вывода шины введите здесь имя интерфейса шины.
* Уровень: Если вывод устройства представляет собой многоуровневую клемму, запишите сюда уровень.
* Внутренний / внешний индекс: Указывает для вывода устройства на схеме соединений, какой по счету внутренний или внешний вывод устройства он представляет.
* Позиция X / Позиция Y / Позиция Z: В обоих этих полях указывается позиция вывода устройства.
* Направление подсоединения: Выберите из раскрывающегося списка одно из возможных направлений вывода устройства "Автоматически", "Вверх" или "Вниз", "Влево" или "Вправо".
* Вектор X / Вектор Y / Вектор Z: В этих полях задаются соответствующие значения вектора выводов устройства для направления вывода устройства.
* Дополнительная длина: Введите здесь значение дополнительной длины позиции вывода устройства.
* Категория соединения: выберите из этого раскрывающегося списка значение по умолчанию, которое точнее определяет тип вывода устройства соединений, например винтовое подключение или пружинная клемма.
* Размер присоединения: В этом поле можно указать размер винта (например, M6) или накладки (например, 4,8x0,5) для винтовых подключений, резьбы, ниппелей и штекерных выводов.
* Мин. поперечное сечение / Макс. поперечное сечение: Укажите здесь значения для наименьшего и наибольшего поперечного сечения провода, которые можно подключить к выводу устройства.
* Макс. число соединений: Введите здесь максимальное число подключаемых проводов.
* Предписать двойную трубку: При помощи флажка укажите, следует ли использовать для вывода устройства двойные трубки, если к этому выводу устройства ведут два соединения.
* Мин. AWG / Макс. AWG: Укажите здесь значения для минимального и максимального поперечного сечения AWG, которые можно подключить к выводу устройства.

!!! note "Замечание:"

    Поля Направление вывода устройства, Мин. поперечное сечение / Макс. поперечное сечение и Предписать двойную трубку не учитываются при маршрутизации соединений в топологии.

### Всплывающее меню

Всплывающее меню дает доступ, в зависимости от типа поля (например, дата, целое число, многоязычный), к пунктам меню, при помощи которых вы можете по необходимости, например, влиять на представление таблиц или обрабатывать значения в полях. Обзор пунктов этого всплывающего меню вы можете найти в разделе [Пункты всплывающего меню](userinterface_m_kontextmenu.md).

Дополнительно здесь представлены следующие пункты всплывающего меню, специфические для данного диалогового окна:

Пункт меню |  Значение
---|---
Обновить схему соединений |  Копирует схему соединений из изделия. Дополнительно генерируются выводы устройств из многополюсных функций. Если сгенерированный вывод устройства совпадает с соответствующим выводом из схемы соединений, данные собираются в одной строке. В ином случае для вывода устройства отображаются две строки.
