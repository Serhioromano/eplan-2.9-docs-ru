# Импорт пользователей Windows

В средних или крупных организациях управление пользователями зачастую осуществляется через группы Active Directory. Каждый пользователь регистрируется на своей рабочей станции со своей учетной записью в Active Directory (имя пользователя Windows и пароль).

В управлении правами EPLAN появилась возможность импортировать этих пользователей и эти группы Windows из Active Directory. Если для такого пользователя была создана конфигурация в управлении правами EPLAN, он автоматически регистрируется в EPLAN, а программа становится доступной со сконфигурированным набором функций.

Чтобы импортированные из Windows пользователи могли войти в EPLAN, необходимо использовать рабочий режим Регистрация с именем пользователя Windows.

### Конфигурация прав

#### Управление группами и пользователями EPLAN в Active Directory

Отдельных пользователей EPLAN так же, как и группы EPLAN, можно указывать и конфигурировать прямо в Active Directory. В управление правами EPLAN необходимо импортировать только группы Active Directory. Обратите внимание, что импортированные группы Active Directory в управлении правами отображаются и управляются как пользователи. Детальное присвоение прав EPLAN выполняется за счет того, что каждой группе Active Directory присваивается одна или несколько групп EPLAN. Все члены этой группы Active Directory получают соответствующие права на обработку. Не нужно создавать и обрабатывать отдельного пользователя для каждого члена группы. Изменения в составе группы необходимо выполнять только в Active Directory.

Если пользователь является членом в нескольких группах Active Directory, то при регистрации в EPLAN он получает все права, которые присвоены этим группам.

#### Управление только пользователями EPLAN в Active Directory

Если группы EPLAN не должны управляться в Active Directory, то отдельных пользователей системы Windows необходимо импортировать из Active Directory в управление правами EPLAN. После этого присвоение пользователей системы Windows группам EPLAN выполняется в управлении правами. Действия с новыми и удаленными пользователями необходимо выполнять как в Active Directory, так и в управлении правами.
