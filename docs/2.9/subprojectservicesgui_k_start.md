# Частичные проекты

Объемные проекты часто обрабатываются командой специалистов из нескольких человек. Для этого варианта применения проекты на платформе EPLAN можно разделять на так называемые "[Области обработки](workingsection_k_start.md)". Это позволяет увеличить наглядность проекта и повышает скорость работы. Такой способ действий имеет смысл только в том случае, когда команда состоит исключительно из собственных сотрудников и имеет в распоряжении мощную сеть.

При привлечении к обработке объемных проектов сторонних лиц, имеющих доступ к сети компании, или при их обработке без сетевого соединения есть возможность разделить проекты на несколько частичных проектов с помощью функции Частичные проекты. После разделения сгенерированные частичные проекты в целях дальнейшей обработки можно сохранить на локальной рабочей станции с помощью резервирования данных или направить для обработки сторонним лицам.

Возможны следующие действия:

* В диалоговом окне Частичные проекты, которое можно использовать только для эксклюзивно открытых проектов, перечисляются все заданные в проекте области обработки; каждая область обработки представляет собой потенциальный частичный проект.

!!! note "Замечание:"

    При выборе пунктов меню Проект > Частичный проект выполняется внутренняя проверка того, открыт ли проект эксклюзивно. Если это не так, проект закрывается и открывается автоматически в эксклюзивном режиме. Если это невозможно, появится сообщение о том, что эксклюзивный доступ невозможен, и действие прерывается.

* Путем генерирования и выгрузки частичных проектов можно разделять объемные проекты таким образом, что они становятся доступны внешним обработчикам (с установленными вариантами программы EPLAN).
* Выгрузка частичного проекта на локальный носитель данных позволяет выполнять обработку определенной части проекта в том числе и без постоянного сетевого соединения с исходным проектом.
