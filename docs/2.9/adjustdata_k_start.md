# Синхронизация и корректировочный прогон

Синхронизация и корректировочные прогоны обеспечивают синхронизацию данных для всего проекта, что позволяет устранять возможные противоречия. Ниже перечислено, в каких случаях проводятся синхронизация и корректировочные прогоны:

* ***Устройства с повторяющимися / излишними главными функциями:*** Одно устройство может содержать только одну главную функцию. Для удаления из устройства повторяющихся / излишних главных функций можно одним действием для всего проекта преобразовать все повторяющиеся / излишние главные функции во вспомогательные.
* ***Излишние данные при разных видах представлений определенной функции:*** В EPLAN существует функция представления определенной функции в проекте в распределенном виде, например в качестве однополюсной функции, многополюсной функции, функции обзора, функции парной перекр. ссылки или функции функциональной схемы автоматизации. Кроме того, допускается многократное присутствие функции в одном виде представления, как, например, блок ПЛК, представляемый двумя многополюсными функциями на двух различных страницах. Свойства функции можно преобразовывать в любом виде представления. Для обработки свойств распределения представленных функций с целью унификации можно синхронизировать связанные друг с другом функции.
* ***Если вспомогательная функция должна быть главной:*** можно преобразовать вспомогательную функцию в главную. При этом исходная главная функция становится вспомогательной.
