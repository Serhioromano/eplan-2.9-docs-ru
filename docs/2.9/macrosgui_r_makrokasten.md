# Вкладка Рамка макроса

В проект макросов вставлена рамка макроса (Вставить > Блок / вывод устройства / монтажная плата > рамка макроса), либо выполнен двойной щелчок по уже имеющейся рамке макроса.

На этой вкладке указываются свойства рамки макроса, В проекте макросов с помощью рамок макросов указываются контуры и данные генерируемых макросов окон и / или символов.

!!! note "Замечание:"

    Большинство свойств рамки макроса можно обработать только в ***проекте макросов***. Для этого в поле Метод использования необходимо настроить запись "Определяющий" или "Не задан".

    В ***проекте схемы соединений*** или проекте макросов, в котором для метода использования установлена запись "Ссылающийся" или "Нижестоящий", большинство полей используются только для отображения данных и не подлежат обработке.

!!! tip "Совет:"

    Для присвоения ***нескольким макросам*** подготовленной рамки макроса используйте вкладку Определение макроса. Внесите в строки этой вкладки соответствующие имя и описание генерируемых макросов. Затем при автоматическом генерировании в нескольких макросах сохраняется одно и то же содержание (одна и та же рамка макроса).

Обзор основных элементов диалогового окна:

### Метод использования

Метод использования позволяет различать макросы и определяет, какие из функций Автоматически генерировать макросы и Обновить макросы могут быть выполнены, а какие — нет. Выберите требуемую запись из раскрывающегося списка:

* Не задан: Можно генерировать и обновлять. Это соответствует поведению в предыдущих версиях EPLAN (до версии 2.8). Эта запись доступна только из соображений совместимости.
* Определяющий: Учитывается только при генерировании (например, для подготовленных макросов в проекте макросов, из которых необходимо генерировать макросы).
* Ссылающийся: Учитывается только при обновлении (например, для макросов в проекте схемы соединений, которые может потребоваться обновить последними данными).
* Нижестоящий: Генерирование и обновление невозможны (например, для макросов, рамка которых используется как "внутренняя" рамка макроса во вложенных рамках макросов).

В дереве навигатора макросов подготовленные макросы объединяются в разные уровни иерархии в зависимости от метода использования.

### Имя

Введите здесь имя рамки макроса, при необходимости указав зону. Это имя будет использоваться для имени файла генерируемого макроса. В соответствии с этим макрос нельзя создать без введенного имени.

С помощью пункта всплывающего меню Вставить переменную пути перейдите в диалоговое окно [Выбрать переменную пути](modaldialogsdb_d_pfadvariablen.md), из которого можно будет скопировать одну из доступных переменных пути.

### Вид представления

Вид представления макроса служит для сортировки, т. е. она помогает в управлении вашими макросами. Она не влияет на функции внутри макроса, сохраняя соответствующий вид их представления. Таким образом, все макросы изделия могут содержаться в файле. Для каждого вида представления макроса можно создать до 26 вариантов. Вид представления предварительно настроен согласно выделенным функциям или согласно типу страниц. Выберите требуемую запись из раскрывающегося списка:

* Многополюсный: Для макросов, которые размещаются на многополюсных страницах схемы соединений.
* Многопол. Fluid: для макросов, которые размещаются на страницах схемы соединений для Fluid-Техники.
* Обзор: Для макросов, которые размещаются на страницах обзора.
* Парная перекрестная ссылка: Для макросов, которые используются для реализации парной перекрестной ссылки.
* Однополюсный: для макросов, которые размещаются на однополюсных страницах схемы соединений.
* Топология: Для макросов, которые размещаются на страницах топологии.
* Функциональная схема автоматизации: Для макросов, которые размещаются на страницах функциональной схемы автоматизации.
* Функциональный: Для макросов, которые размещаются на блок-схемах рабочего процесса.
* Компоновка электрошкафа: Для размещений изделий вида представления "Компоновка электрошкафа" или "Компоновка электрошкафа подробно".
* Трехмерный чертеж монтажных поверхностей: Для макросов, которые размещаются в пространствах листа.
* Предварительное планирование: Для макросов, которые размещаются на страницах предварительного планирования.
* Обзор функций (Fluid-Техника): Для макросов, которые размещаются в обзорах функций для Fluid-техники.
* Графика: Для макросов, которые содержат только графические условные обозначения. Графические условные обозначения не учитывается ни при прогонах отчетов, ни при контрольных прогонах, ни при образовании перекрестных ссылок, и ***не*** регистрируются как цель.

!!! note "Замечание:"

    Если функция имеет другие виды представлений, где общие свойства содержат различные данные, в зависимости от вида представления выносится решение о том, какие свойства будут учитываться при составлении отчета. Для этого в EPLAN существует установленная [последовательность использования видов представлений при общей обработке и генерации отчетов](reverseengineering_k_reihenfolge.md).

### Вариант

С помощью этого раскрывающего списка указывается обозначение варианта для данного создаваемого макроса. В качестве предварительной настройку установлен "Вариант A".

!!! note "Замечание:"

    Для автоматического создания вариантов макроса из проекта макросов сначала необходимо вручную создать эти варианты на страницах проекта. Затем каждый вариант снабжается рамкой макроса и всем рамкам указывается одинаковое имя с разными обозначениями вариантов. После автоматического создания макросов пери вставке соответствующего макроса можно будет выбрать один из требуемых вариантов.

### Версия

Это поле позволяет управлять версиями макросов, причем эта запись не имеет ничего общего с версиями, специфическими для EPLAN.

### Исходный проект

При автоматической генерации макросов из проекта макросов в это поле вводится имя исходного проекта.

### Источник / ссылка

Здесь можно указать, например, происхождение соответствующей схемы (например, Каталог Siemens).

### Описание

Текст, введенный в этом поле, становится текстом описания макроса, автоматически созданного из проекта макросов, и выводится в поле комментария просмотр при вставке соответствующего макроса.

### Всплывающее меню

Всплывающее меню дает доступ, в зависимости от типа поля (например, дата, целое число, многоязычный), к пунктам меню, при помощи которых вы можете по необходимости, например, влиять на представление таблиц или обрабатывать значения в полях. Обзор пунктов этого всплывающего меню вы можете найти в разделе [Пункты всплывающего меню](userinterface_m_kontextmenu.md).

### Присвоение объектов вручную

Обычно все объекты, находящиеся в рамке макроса, относятся к макросу.

Установите этот флажок, если, например, требуется также присвоить объекты, находящиеся вне рамки. Затем необходимо выделить все объекты, которые должны относится к макросу, и выбрать во всплывающем меню рамки макроса функцию Присвоить объекты рамке макроса. Если снова снять этот флажок, в макросе восстанавливается первоначальное присвоение объектов.
