# Порядок действий при размещении устройств

Описываемые ниже внутренние процессы всегда относятся к выбору, для которого инициируется размещение.

Определение подходящего макроса

Вначале из указанных на изделиях файлов макросов определяется наиболее подходящий макрос. При этом EPLAN учитывает только макросы целевого вида представления, т. е. вида представления открытой страницы, и выполняет поиск макроса, охватывающего как можно большее количество функций / шаблонов функций текущего выбора. Если таких подходящих макросов окажется несколько, то система выберет из них тот, который будет содержать как можно меньше других функций одного и того же устройства.

Определение подходящего устройства в макросе

Если в макросе содержатся несколько устройств, то из них определяется наиболее подходящий вариант, а в случае их совпадения — графически первый.

Присвоение функций или шаблонов функций проекта функциям из макроса

Для этого присвоения сравниваются свойства, которые могут иметь соответствующий шаблон функции. Первыми присваиваются те функции, которые еще не были размещены в целевом виде представления. Затем присваиваются спаренные функции с наивысшей степенью соответствия, а после них — с меньшим числом соответствий.

Отсюда следует, что спаренные функции с одинаковыми обозначениями выводов устройства имеют преимущество при присвоении. Затем также присваиваются те функции, которые содержат только подходящие определения функций.

Если после этого не для всех функций в макросе будет найден обратный эквивалент, то выбор распространится на все устройство, а оставшаяся часть будет присвоена.

Функции из макроса, которые по-прежнему останутся неприсвоенными, будут вставлены дополнительно. При наличии функций предварительно определенного устройства они получат то же ОУ, что и функции, которые удалось присвоить: Устройство не будет разделено.

Присвоение данных

Относительно присвоения данных между графическими данными и данными функции существует разница. При этом графические данные — это данные, влияющие на графику или зависящие от графического расположения (напр., символы, порядок свойств, свойство Перенос ОУ: Направление поиска и т. д.), в то время как данные функции являются данными, которые определяют или описывают определенную функцию.

Все данные функции, уже содержащиеся на функциях в проекте или на шаблонах функций, перезаписывают данные в макросе. Заполненные в макросе свойства при этом не перезаписываются пустыми свойствами; таким образом данные функции из макроса могут сохраняться.

Наоборот, вся графическая информация заимствуется из макроса.

Разместить взаимосвязанные функции в виде связки шаблонов

В базе данных изделий шаблоны взаимосвязанных функций можно объединить в одну логическую единицу. Для этого введите однозначный номер или обозначение в столбце Связка шаблонов (многополюсная) вкладки Шаблоны функций.

С помощью группирования в связку шаблонов все взаимосвязанные функции можно совместно разместить из навигатора устройств. Если одна из этих функций выделена в навигаторе, при размещении — вместе с макросом, указанным на изделии — все функции связки шаблонов размещаются одновременно. Такое совместное размещение возможно только для многополюсных страниц (тип страницы "Многополюсная схема соединений" и "Схема соединений, Fluid-Техника").

### Вставить макрос

Чтобы использовать макрос для добавления устройству в проекте других функций, необходимо вставить макрос в графическом редакторе и соответствующим образом настроить ОУ.
