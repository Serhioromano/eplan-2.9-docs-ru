# Диалоговое окно Настройки: страницы (проект)

Параметры > Настройки > Проекты > "имя проекта" > Управление > Страницы

В этом диалоговом окне указываются настройки для имен страниц, использованной рамки и нумерации зон.

Обзор основных элементов диалогового окна:

### Символы для подстраниц

Выбрав значение из раскрывающегося списка, укажите какие символы должны использоваться для подстраниц (строчные буквы, прописные буквы, числа); также можно указать, что подстраницы не используются.

### Мин. длина имени страницы

Здесь можно указать минимальное число символов для имени страницы. Стандартной настройкой является "0", т. е. имена страниц выводятся без ведущих нулей. Если ввести здесь другое число, имя страницы будет выводиться с соответствующим числом ведущих нулей.

!!! note "Замечание:"

    Если указать здесь число, не равное нулю, а позже вручную дополнить ведущие нули в имени страницы, то введенные вручную записи подавляются.

### Стандартная рамка

Укажите здесь, какая рамка по умолчанию присваивается при создании страницы.
Нажав кнопку "Поиск", откройте диалоговое окно для выбора файла и выберите рамку из любого каталога.

### Сортировка страниц вручную

Если этот флажок установлен, сортировка выполняется для таких операций, как перелистывание, печать, вывод перекрестных ссылок и т. д. в соответствии с указанной в диалоговом окне последовательностью Сортировка страниц вручную. Сортировка страниц вручную влияет только на представление в виде списка, представление в виде дерева сохраняется без изменений.

* * *

Групповое поле Нумерация зон

Здесь указывается, каким образом нумеруются зоны в проекте. Страницы внешних документов всегда игнорируются при нумерации зон.

* По страницам: При использовании этой настройки подсчет зон начинается на каждой странице заново. На каждой странице EPLAN начинает от настроенного начального значения и выполняет подсчет по восходящей до достижения последней зоны на странице.
* По проектам: При использовании этой настройки выполняется сквозная нумерация зон по возрастающей для всех страниц. На первой странице EPLAN начинает от настроенного начального значения и выполняет подсчет по восходящей до последней зоны на последней странице. (При этом учитываются только логические страницы.)
* На каждый структурный идентификатор: При использовании этой настройки зоны нумеруются по группам для каждого структурного идентификатора. В случае полностью пронумерованных проектов эта опция неактивна.

!!! note "Замечание:"

    Если установлена настройка По проектам или На каждый структурный идентификатор, то при вставке/копировании новых страниц появляется запрос о необходимости автоматического обновления зон.

Для настроек По проектам и На каждый структурный идентификатор дополнительно активны следующие флажки:

### Только логич. страницы

Если установлен этот флажок, нумеруются только зоны на логических страницах в соответствии с выбранной выше настройкой. Зоны на других страницах нумеруются постранично.

### Учитывать пропуски в номерах страниц

Если установлен этот флажок, то в случае пропусков в номерах страниц соответствующее количество обозначений зон остается свободным ("число отсутствующих страниц" x "число зон в стандартной рамке").

!!! example "Пример:"

    Если отсутствуют две страницы, а в стандартной рамке настроено 10 зон, свободными оставляются 2 x 10 = 20 обозначений зон.Страница Обозначения зон10, 1, 2, 3, 4, 5, 6, 7, 8, 9210, 11, 12, 13, 14, 15, 16, 17, 18, 19--540, 41, 42, 43, 44, 45, 46, 47, 48, 49

### Генерировать подномера для подстраниц

Если установлен этот флажок, на подстраницах добавляются подномера к обозначениям зон. В качестве основных номеров применяются номера соответствующей главной страницы.

!!! example "Пример:"

    В проекте имеются страницы 1, 1.A, 2, 2.A. Каждая страница имеет 10 зон. В этом случае зоны получаютобозначенияв соответствии со следующей таблицей:Страница Обозначения зон10, 1, 2, 3, ..., 91.A0.A, 1.A, 2.A, 3.A, ..., 9.A210, 11, 12, 13, ..., 192.A10.A, 11.A, 12.A, 13.A, ..., 19.A

### Сетка для типа страницы

В этом групповом поле можно настроить сетку для разных типов страниц. Настроенный здесь размер сетки будет использоваться при генерировании страниц, если не выделена ни одна страница такого же типа.

* Схема соединений электротехники: Размер сетки для типа страниц "Многополюсная схема соединений" и "Однополюсная схема соединений", стандартное значение равно 4 мм.
* Схема соединений Fluid-техники: Размер сетки для типа страниц "Схема соединений Fluid-техники", стандартное значение равно 2 мм.
* Функциональная схема автоматизации: Размер сетки для типа страниц "Функциональная схема автоматизации", стандартное значение равно 2,5 мм.
* Другие: Размер сетки для других типов страниц, стандартное значение равно 1 мм.

### Число / имена страниц на свойство

Здесь необходимо указать свойство, которое используется в качестве критерия для сравнения свойств Количество страниц в свойстве, Имена страниц в свойстве и Счетчик страниц в свойстве. Во время прогона отчета (для запуска перейдите в меню Страница > Число / имена страниц на свойство) подсчитываются страницы с одинаковым значением этого свойства, перечисляются их имена, а также определяется, о какой странице с одинаковым значением идет речь. Результаты записываются в свойства страницы Количество страниц в свойстве, Имена страниц в свойстве и Счетчик страниц в свойстве.

### Для генерации содержания автоматически выполнить "Число/имена страниц на свойство"

Если этот флажок установлен, то при генерации содержания автоматически запускается прогон отчета, который подсчитывает и перечисляет страницы с одинаковым значением вышеназванного свойства.
