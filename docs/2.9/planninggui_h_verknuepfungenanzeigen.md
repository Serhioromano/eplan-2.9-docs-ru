# Отобразить связи с объектами планирования соединений

С помощью связей в предварительном планировании становятся видимыми зависимости между различными сегментами. Чтобы в навигаторе предварительного планирования было видно, вводятся ли сегменты в качестве исходных или целевых сегментов для объектов планирования кабелей и трубопроводов, с помощью нового диалогового окна Представление в виде дерева можно отобразить связи с объектами планирования соединений.

Вид в представлении структуры дерева навигатора предварительного планирования может быть неудобным для просмотра. Ввиду этого в диалоговом окне Представление в виде дерева существует возможность показывать или скрывать объекты, отображаемые по умолчанию для дерева, такие как адреса ПЛК, обозначения местоположения, функции и т. д.

Условие:

* Открыт проект с уже существующими сегментами структуры и объектами планирования.
* Вы определили исходный и целевой сегменты для объектов планирования трубопроводов и/или кабелей.

1. Выберите следующие пункты меню: Данные проекта > Предварительное планирование > Навигатор
2. Выберите в дереве навигатора предварительного планирования пункты меню Всплывающее меню > Представление в виде дерева.
3. В диалоговом окне Представление в виде дерева установите флажок Связи с объектами планирования кабелей и трубопроводов.
4. При необходимости установите или снимите дополнительные флажки, чтобы показать или скрыть соответствующие объекты в дереве под сегментами.
5. Щелкните по кнопке ++OK++.

!!! info "Для сведения:"

    Связи с объектами планирования кабелей и трубопроводов отображаются под сегментами, введенными в качестве исходных или целевых сегментов для объектов планирования соединений в представлении структуры дерева навигатора предварительного планирования (пиктограмма: ![](images/planninggui_link_as.png){: .ui-icon }).

!!! info "Для сведения:"

    В соответствии с определением в диалоговом окне Представление в виде дерева под сегментами отображаются другие объекты (адреса ПЛК, обозначения местоположения, функции и т. д.).
