# Диалоговое окно Управление сообщениями — <Имя проекта>

Вы открыли проект. Данные проекта > Сообщения > Управление.

В этом диалоговом окне в таблице перечисляются сообщения, появившиеся после ошибок или несоответствий при проверке в автономном режиме или в режиме подключения.

Заголовки столбцов таблицы являются свободно конфигурируемыми. Через всплывающее меню для этой цели можно вызвать пункт меню Конфигурировать представление. В открывшемся следующем диалоговом окне задайте отображение нужных столбцов и определите их последовательность.

Сообщения можно сортировать, дважды щелкнув по заголовкам столбцов.

Если открыть предварительный просмотр графики (Вид > Предв. просмотр графики), то он покажет страницу, к которой относится сообщение, где будет выделена ошибочная функция.

Обзор основных элементов диалогового окна:

### Таблица

Помимо номера и текста сообщения, в таблице можно добавлять следующие столбцы:

* Статус: Отображает присвоенную категорию сообщения в форме пиктограммы.

* Категория: Отображает присвоенную категорию сообщения в сокращенной форме.
* Страница: Отображает страницу, на которой находится некорректный объект.
* Пространство листа: Отображает пространство листа, в котором находится некорректный объект.
* ОУ: Указывает ОУ устройства, в котором обнаружена ошибка.
* Номер изделия: Если при проверке базы данных изделий была найдена запись изделия с недостающими или недопустимыми данными, в этом поле отображается соответствующий номер изделия.
* Вариант изделия: В случае ошибки вместе с номером изделия дополнительно отображается вариант изделия.
* Источник: Указывает ОУ исходной функции, подключенной к устройству, на котором заявлена ошибка.
* Цель: Указывает ОУ целевой функции, подключенной к устройству, на котором заявлена ошибка.
* Выполнено: Установите этот флажок, если ошибка была устранена. Сообщения можно фильтровать по этому свойству, что позволяет наглядно представить уже устраненные и еще имеющиеся ошибки.
Кроме того, при следующем контрольном прогоне можно проверить только выполненные сообщения. Таким способом можно проверить, удачно ли прошли исправления в проекте.
* Создано: Отображает автора сообщения. В большинстве случаев сообщения создает EPLAN. Однако Вы также можете указывать сообщения с помощью интерфейса API.
* X / Y: Отображает координаты некорректного устройства. Значения координат округляются; десятичные разряды не выводятся.
* Генерирует с помощью вида проверки: Указывает тип возникновения сообщения (например, при проверке в автономном режиме или в режиме подключения).

### Всплывающее меню

Всплывающее меню дает доступ, в зависимости от типа поля (например, дата, целое число, многоязычный), к пунктам меню, при помощи которых вы можете по необходимости, например, влиять на представление таблиц или обрабатывать значения в полях. Обзор пунктов этого всплывающего меню вы можете найти в разделе [Пункты всплывающего меню](userinterface_m_kontextmenu.md).

Дополнительно здесь представлены следующие пункты всплывающего меню, специфические для данного диалогового окна:

Пункт меню |  Значение
---|---
Удалить отображение |  Удаляет из диалогового окна все выведенные сообщения. В этом случае вплоть до следующего обновления всего проекта будут выводиться только те сообщения, которые вызваны текущей обработкой.
Выполнить контр. прогон |  Открывает диалоговое окно Выполнить контр. прогон, в котором можно выполнить контрольный прогон на базе текущих или других настроек.
Проверять только выполненные сообщения |  Проверка всех сообщений, для которых установлен флажок Выполнено. При этом диалоговое окно Выполнить контр. прогон***не ***открывается.
Перейти к (перекрестная ссылка) |  Заносит перекрестные функции в список Перейти к и открывает его.
Перейти к (все виды представлений) |  Заносит все виды представлений функции (например, на странице схем соединений, странице обзора и странице отчета) в список Перейти к и открывает его.
Перейти к (графика) |  Выполняет переход к первому размещенному объекту, который содержит ссылку на выделенную функцию.
Перейти ко 2-й координате |  Позиционирует курсор на точке вставке некорректного объекта в графическом редакторе. Дважды щелкнув по объекту, можно вызвать диалоговое окно для обработки свойств данного объекта.
Синхронизировать выбор |  В открытых навигаторах выделяет объект, вызвавший сообщение.
Конфигурировать представление |  Открывает диалоговое окно Конфигурировать представление, в котором можно задать, какие свойства должны отображаться в списке.
Табличная обработка |  Открывает табличную обработку с возможностью обрабатывать свойства функций.
Свойства |  Открывает диалоговое окно Свойства (усл. обозначение): ++...++. Обеспечивает обработку функции.
Свойства (общие) |  Открывает диалоговое окно Свойства (общие): ++...++. Позволяет обрабатывать свойства устройства.

### Фильтр

С помощью кнопки ++...++ откройте диалоговое окно Фильтр: Сообщения, чтобы создать фильтр для сообщений.

Для просмотра только тех сообщений, которые соответствуют фильтру, установите в диалоговом окне флажок Активн..

### Выбор

По выполнении контрольных прогонов некорректные функции в навигаторах будут помечены восклицательным знаком. Выделите в навигаторе функцию и установите этот флажок, таким образом в управлении сообщениями будут отображаться только сообщения, относящиеся к этой функции. Если вы установите флажок, находясь в навигаторе страниц, то в диалоговом окне управления сообщениями отображаются только сообщения объектов на соответствующих страницах; если вы установите флажок из графического редактора, будут отображаться сообщения по выбранным в данный момент условным обозначениям, точкам определения соединений и точкам разрыва.

!!! note "Замечание:"

    * Флажок Выбор синхронизирует управление сообщениями с выбором в навигаторе страниц, в одном из навигаторов данных проекта или в графическом редакторе, только если в базе данных сообщений содержится ***не более 1000 сообщений***. Ограничьте при необходимости количество сообщений в настройках проекта (Параметры > Настройки > Проекты > 'Имя проекта' > Управление > Управление сообщениями, поле Макс. кол-во сообщений контрольного прогона).
    * В справочной системе содержится подробное описание всех сообщений контрольного прогона и указания по исправлению ошибок. Для вызова справки по определенному сообщению щелкните в строке по соответствующему сообщению и нажмите клавишу ++F1++.
