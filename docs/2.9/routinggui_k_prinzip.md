# Маршрутизируемые соединения: Принцип

Маршрутизация соединений проходит в несколько этапов:

* Определение соединений в схеме соединений, присвоение изделия соединению.
* Размещение трехмерных изделий участвующих объектов в пространстве листа.
* Определение сети соединенных сегментов, которая доступна для маршрутизации соединения.
* Создание маршрутизируемых соединений.
* Нахождение трассы маршрутизации (сумма сегментов маршрутизации, соединительных отверстий для проводов и областей маршрутизации, через которые проходит соединение, проложенное в пространстве листа).

Маршрутизация влияет на выделенные объекты, к которым подключены соединения. Можно выделить объекты в пространстве листа и во всех навигаторах данных проекта. В навигаторах можно отметить как отдельные, так и несколько соединений.

Если текущая сеть соединенных сегментов в пространстве листа не устарела, то перед маршрутизацией происходит автоматическое обновление сети соединенных сегментов.

Прокладываются многополюсные соединения, для которых существует размещенное трехмерное размещение изделия. При этом для многополюсных соединений рекомендуется задать изделие.

* Многополюсные соединения без присвоенного изделия генерируют маршрутизируемые соединения в виде линий, так как информация о внешнем диаметре отсутствует.
* Если многополюсное соединение имеет присвоенное изделие-провод, то маршрутизируемое соединение генерируется по цвету и диаметру согласно шаблону изделия и таким образом подготавливается к заказу или изготовлению проводов.

### Определение функций соединений

Маршрутизируются только соединения со следующими определениями функций:

* Жила / провод
* Соединение, общ.
* Шланг
* Труба
* Неэлектрическое соединение
* Световод
* Проволочная перемычка
* Технол. произв. процессов.

В навигаторе соединений отображаются маршрутизируемые соединения с пиктограммой ![](images/navigatorgui_mountingfunction_as.png){: .ui-icon } (функция, вид представления "Трехмерный чертеж монтажных поверхностей").

### Свойства маршрутизируемых соединений

При маршрутизации соединений указываются следующие свойства:

Свойство |  Значение
---|---
Длина |  Длина соединения от источника до цели. Длина рассчитывается с учетом всех пересекаемых сегментов и областей маршрутизации, включая дополнительную длину из схемы соединений, дополнительную длину для соединительных отверстий для проводов и радиус изгиба.
Трасса маршрутизации |  Пересекаемые сегменты, области маршрутизации и соединительные отверстия для проводов перечисляются здесь от источника до цели с указанием ОУ. Названия перечисляются через точку с запятой. Пустое свойство Пространство листа: Трасса маршрутизации означает немаршрутизированное соединение. При удалении маршрутизации это свойство очищается.
Трасса маршрутизации по умолчанию |  Свойство соединения Пространство листа: Трасса маршрутизации — по умолчанию задается вручную путем изменения трассы маршрутизации. После этого при маршрутизации этих соединений фильтры соединений не учитываются. Трасса маршрутизации по умолчанию имеет преимущество. Если, например, при недопустимом напряжении трасса маршрутизации по умолчанию указана в сегменте / области маршрутизации, то сегмент / область маршрутизации прокладывается без учета других фильтров соединений.
Обработка концов проводов, источник / цель |  Предусмотренная обработка концов проводов указывается в соответствии с настройками маршрутизируемых соединений.
Размер присоединения, источник / цель |  При маршрутизации источник и цель размера присоединения берутся из свойств размещения изделия (вкладка Схема соединений).
Направление подсоединения, источник / цель |  Направления подсоединений (например, 'вправо', 'вниз') определяются автоматически на основании выполненной маршрутизации соединения. Они отражают направление маршрутизации провода от устройств по трассе маршрутизации и содержат важную информацию для прокладчика.
Предписать двойную трубку (источник / цель) |  Это свойство берется из свойств размещения изделия (вкладка Схема соединений) или изделия (из присвоенной схемы соединения).
