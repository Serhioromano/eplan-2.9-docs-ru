# Применение клемм рукоятки

Главная клемма представляет группу клемм на клеммнике. Устройство может содержать любое количество главных клемм, но только одну главную функцию. Поэтому клеммник определяется как главная функция, а клеммы — как главные или второстепенные клеммы. Если клемма определена как главная клемма, тогда она ведет себя аналогично главной функции. Например, для главной клеммы в навигаторе клеммников отображаются шаблоны функций, и можно выполнить выбор устройства.

### Самостоятельные области клеммника

Использованием главных клемм клеммник разделяется на так называемые "самостоятельные области". Каждая такая самостоятельная область соответствует одному устройству с клеммами, которое может включать в себя несколько функций. Клеммники дополнительно идентифицируются с помощью свойства Клемма: Положение устройства. Обычно все клеммы одной самостоятельной области имеют одинаковое [Положение устройства](terminalgui_k_verwaltung.md), и в одной группе клемм есть только одна главная клемма. Клеммник может содержать произвольное число главных клемм и, следовательно, произвольное число самостоятельных областей. При вставке клеммы по умолчанию определяются как главные клеммы. Так для каждой клеммы образуется самостоятельная область. Клемму можно сделать вспомогательной, сняв флажок Главная клемма в диалоговом окне "Свойства" клеммы.

Самостоятельная область простирается от одной главной клеммы (включительно) до следующей главной клеммы (при одинаковом положении устройства). То есть она содержит одну главную и принадлежащие ей второстепенные клеммы. Если первая клемма на клеммнике не является главной, то первая область клеммника (т. е. область от начала клеммника до первой главной клеммы) задается через определение клеммника. Без главных клемм весь клеммник представляет собой одну область и задается полностью через определение клеммника. При отсутствии определения клеммника отсутствуют и шаблоны функций для первой области клеммника.

Свойство Основные клеммы не используются позволяет указать, что клеммник ***не*** подразделяется на самостоятельные области и задается только через определение клеммника. Тогда шаблоны функций добавленных к клеммам изделий не используются, но изделия выводятся в отчетах. Кроме того, при копировании и вставке клемм используются имеющиеся свободные шаблоны функций.

#### Определение многоуровневых клемм

Самостоятельные области позволяют определить многоуровневые клеммы. Многоуровневая клемма состоит из нескольких следующих друг за другом клемм ***одной области*** с расположенными соответственно друг на друге уровнями. При этом номер уровня может быть возрастающим или убывающим. Клеммы в различных областях не могут относиться к одной и той же многоуровневой клемме. Т. е. каждая многоуровневая клемма может содержать только одну главную клемму, остальные, принадлежащие ей клеммы должны быть второстепенными.

!!! example "Пример:"

    Клеммник состоит из определения клеммника и клемм1,2,3,4,5,6,7, причем клеммы3и6являются главными. Таким образом, в данном клеммнике выделяются три самостоятельные области:Область 1: Определение клеммника + клеммы1,2Область 2: Главная клемма3+ клеммы4,5Область 3: Главная клемма6+ клемма7В данном примере каждая область задает одну многоуровневую клемму. Уровни отображаются здесь в квадратных скобках (например,[1]для самого нижнего уровня).

### Присвоение шаблонов функций

Если на определении клеммника или на главных клеммах внесены изделия с сохраненными в них шаблонами функций, то эти изделия будут замещены функциями клеммы в пределах соответствующих самостоятельных областей. За пределами области присвоения не происходит, даже при наличии подходящих функций.

* Если ***Определение клеммника*** содержит изделие с шаблонами функций, и свойство Основные клеммы не используются активировано, то эти шаблоны функций задают последовательность клемм на клеммнике. Положение устройства и идентификатор сортировки клемм указывают только на то, в какой последовательности клеммы присваиваются шаблонам функций. Если определение клеммника не содержит шаблонов функций, то последовательность на клеммники определяют сами клеммы.
* Если какая-либо ***Главная клемма*** содержит изделие с шаблонами функций, то эти шаблоны будут замещены функциями главной клеммы и соответствующих второстепенных клемм. Положение устройства главных клемм задает самостоятельные области: одной главной клемме принадлежат все второстепенные клеммы с одинаковым положением устройства. При этом неважно, имеется ли на этой главной клемме свободный шаблон функции. В пределах одной самостоятельной области последовательность, в которой клеммы присваиваются шаблонам функций, задается идентификатором сортировки.

Клеммники рекомендуется составлять либо только из главных клемм (и принадлежащих им второстепенных клемм), либо абсолютно без главных клемм. Т. е. клеммник должен быть определен либо только через самостоятельные области главных клемм, либо только через определение клеммника. Смешанное строение возможно, но присвоение шаблонов функций при этом становится менее обозримым. В первом случае изделия следует заносить на главные клеммы, но не в определение клеммника. Шаблоны функций изделия будут замещены функциями главной клеммы и соответствующих второстепенных клемм. Во втором случае изделия следует заносить только в определение клеммника. Шаблоны функций изделий будут замещены функциями всех клемм клеммника.

!!! note "Замечание:"

    При работе с трехмерным навигатором монтажных поверхностей изделия следует заносить на главные клеммы (а не на определение клеммника). Только тогда клеммы будут видны в трехмерном навигаторе монтажных поверхностей, поскольку там отображаются только такие устройства, которым присвоены изделия. Это условие является одним из основным для обеспечения интерфейса к внешним программам конфигурации клемм.

#### Выбор устройства

При выборе устройства шаблоны функций присваиваются только в пределах одной самостоятельной области. Многоуровневые клеммы сохраняются и не разделяются.
