# Устройства: Принцип

При работе с устройствами устройство является исходной точкой. Для выполнения определенной задачи Вам нужно устройство (определенное устройство), функцию от которого Вы желаете разместить.

Проектирование с устройствами начинается с управления устройствами. Управление устройствами интегрировано в базу данных изделий; она содержит экономические и технические данные. Одному изделию можно присвоить как номер изделия, так и номер типа и номер для заказа. Эти номера дают разные возможности доступа к изделию. Если, например, для Вас привычнее работать с номерами типа, чем с номерами изделий, то Вы можете выбирать устройство, просто вводя номер типа.

На стадии конструирования используйте имеющиеся функции данного устройства. Это позволяет избегать ошибок и проводить проверки непротиворечивости.

При выборе устройств выбираются типы, подходящие для функции. Это можно делать или после проектирования в схеме соединений, или в ходе предварительного выбора, после которого на втором шаге происходит проектирование в схеме соединений.

### Модули

Модуль является коллекцией изделий, относящихся к нескольким вложенным устройствам. В базе данных изделий рядом с каждым изделием модуля указано, к какому устройству модуля принадлежит данное изделие. Модулю присвоен собственный номер изделия. Он может содержать изделия, узлы и другие модули.

### Шаблоны функций

Шаблоны функций вносятся в изделия. Они не имеют графического представления в схеме соединений и служат для определения устройств.

Шаблоны функций могут быть замещены "неразмещенными" или "размещенными" функциями, если идентифицирующие свойства шаблонов функций идентичны свойствам функций. В итоге шаблон функции присваивается функции. Для шаблонов функции нельзя ввести какие-либо дополнительные данные. Это возможно только для самой функции. Т. е. шаблоны функции не являются частью проекта, но относятся к изделию.

### Определение устройства

В EPLAN есть возможность генерирования "пула функций" для устройства. Доступный для контроля пул функций генерируется через выбор изделий или устройств. Из этого пула Вы можете выбирать и размещать функции во время проектирования. Функцию можно проверять на ошибки только в том случае, если ей было присвоено определение устройства.

Устройства предварительно определяются в базе данных изделий; в ней Вы можете закрепить определение устройства за любым изделием. При помощи шаблонов функций определяются идентифицирующие свойства (т.е., электротехнические функции и технические данные) каждой функции устройства. В качестве альтернативы можно закрепить за изделием символ или макрос, который будет размещаться при вставке этого изделия.

Определение устройства состоит из набора x шаблонов функций. В каждом шаблоне функции наряду с определением функции указаны идентифицирующие свойства функции (например, обозначение вывода устройства).

Определение устройства в основном сохраняется на изделие в главной функции. Напр., в случае с контактором — это катушка, в случае со штекерами — текст определения штекера и т.д. При удалении главной функции (или изделия) удаляется и определение устройства. Если копируется главная функция, то также копируется и определение устройства и т.д.
