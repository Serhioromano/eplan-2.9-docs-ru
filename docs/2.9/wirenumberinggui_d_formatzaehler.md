# Диалоговое окно Формат: Счетчик (нумерация соединений)

Вы открыли проект. Вы открыли диалоговое окно [Настройки: Нумерация соединений](wirenumberinggui_d_verbnumeinstellungen.md) (напр., через пункты меню Параметры > Настройки > Проекты > "Имя проекта" > Соединения > Нумерация соединений). В диалоговом окне Настройки: Нумерация соединений вы выбрали определенную пользователем схему и перешли во вкладку Обозначение. Вы выделили запись в поле Группы форматов и щелкнули ![](images/all_edit_as.png){: .ui-icon } (Обработать) или ![](images/all_new_as.png){: .ui-icon } (Создать).

В диалоговом окне Нумерация соединений: Форматы в списке Доступные элементы формата выберите элемент формата типа "Счетчик" и перенесите его кнопкой со стрелкой в список Выбранные элементы формата.

В этом диалоговом окне указывается форматирование для счетчика или подсчетчика в группе форматов.

Обзор основных элементов диалогового окна:

### Резерв/страница

На каждой странице вы можете указать резервное значение для номеров счетчиков. Разрешается использовать только числовые значения, ноль означает отсутствие резерва.

Исходя из последнего номера счетчика на странице, все номера счетчика будут зарезервированы до следующего многочлена указанного резервного значения. На следующей странице номера счетчиков будут начинаться с числа, увеличенного на соответствующее значение. Резервное значение, таким образом, задает максимальное число зарезервированных номеров счетчика.

!!! example "Пример:"

    Резерв/страница: 100Последний номер счетчика на странице: 230Начальный номер на следующей странице: 300Резерв/страница: 50Последний номер счетчика на странице: 230Начальный номер на следующей странице: 250

### Мин. число разрядов

Здесь указывается минимальное число разрядов для значения свойства. Более длинные значения выводятся полностью. Разрешено использовать только положительные числовые значения. При вводе нуля число разрядов является переменным.

### Заполнитель

Значение заполняется с помощью этого символа, если минимальное число разрядов больше, чем ширина значения. При переменном числе разрядов (Запись "0" в поле Мин. число разрядов) ввод заполнителя невозможен.

### Направление

Здесь задается направление, в котором нумеруются соединения. Возможные значения:

* По строкам: Нумерация выполняется в строках слева направо.
* По столбцам: Нумерация выполняется в столбцах сверху вниз.
* По логике рамки: Будет ли нумерация выполняться по строкам или по столбцам, определяется логикой рамки.

### Нумеровать по устройствам

Если установлен этот флажок, будут нумероваться соединения между двумя условными обозначениями в последовательности, в которой они представлены в схеме соединений, вместо последовательности, определенной в поле Направление. При этом также будут учитываться соединения, которые находятся между различными условными обозначениями, которые имеют одинаковое, но невидимое ОУ. Это относится к условным обозначениям, которые при [переносе ОУ](devicetaggui_k_bmkuebernahme.md) получают идентичную ОУ.

!!! note "Замечание:"

    Настройка Нумеровать по устройствам целесообразна только в том случае, если для нумерации выбран объем "Отдельные соединения". При большем объеме нумерации учитываются дополнительные правила нумерации соединений (напр., для сетей или потенциалов). Таким образом можно получить измененную последовательность.

### Использовать алфавитно-числов. счетчик

Если установлен этот флажок, применяется алфавитно-числовой счетчик. Если флажок снят, применяется числовой счетчик.

### Алфавитно-числовой счетчик

При использовании алфавитно-числового счетчика здесь необходимо определить символы, которые будут использоваться для счетчика. Разрешается использовать только буквы и цифры.

Алфавитно-числовые счетчики сортируются по символам слева направа (как в столбцах в Excel). Например, если предварительно заданы символы "Тех. марк.", то счет осуществляется следующим образом: A, K, S, AA, AK, AS, KA, ..., AAA, AAK, AAS, AKA ..., AAAA, AAAK, ...

### Сброс на одну структуру/страницу

Если установлен этот флажок, то счетчик начинается снова с начального значения, если, например, изменяется страница или структурный идентификатор. Через меню ++Настройки++ откройте диалоговое окно [Настройки: Счетчик области сброса](wirenumberinggui_d_zaehlerruecksetzbereich.md) и там задайте критерии, при которых счетчик сбрасывается. Таким образом, например, можно начать с начального значения для отдельной страницы или установки. Сброс счетчика целесообразен только в том случае, если обозначения соединений неидентичны (напр., отличаются по номеру страницы или структурному идентификатору для установки). Чтобы различать соединения с одинаковым именем, используйте подсчетчик.

Если флажок снят, выбранные для нумерации соединения нумеруются по возрастающей.

### Счетчик явл. подсчетчиком

Установите этот флажок, чтобы определить подсчетчик. Эта настройка определяет возможности настройки подсчетчика. Подсчетчик помогает различать соединения с одинаковым именем.

Если флажок снят, настройки подсчетчика становятся недоступными и отображаются серым цветом.

### Блокировать первый счетчик / вспомогательный счетчик

Если установлен этот флажок, первый счетчик или — если одновременно установлен флажок Счетчик явл. подсчетчиком — подсчетчик подавляется.

В случае со вспомогательным счетчиком в зависимости от выбранного объема всегда подавляется первый подсчетчик. Например, у каждого первого соединения на потенциал подсчетчик не отображается.

В случае со счетчиком, если область сброса счетчика не задана, подавляется только первый счетчик в проекте. Если для счетчика дополнительно установлен флажок Сброс на одну структуру / страницу, первый счетчик подавляется каждый раз, когда выполняется критерий для сброса (например, на страницу или на установку).

!!! example "Пример:"

    Вы хотите назначить обозначение соединения для каждого потенциала. Поэтому для соответствующейгруппы соединений(например, "Общие соединения") в диалоговом окне Нумерация соединений: Форматыв поле Объемвы выбрали запись "Потенциал / трубопровод". Кроме того, каждое отдельное соединение в потенциале должно содержать подсчетчик. Для этого в поле Выбранные форматывы определили группу форматов, которая состоит из счетчика, разделителя и подсчетчика.Для подсчетчика выберите в поле Объем для подсчетчиканастройку "Отдельное соединение". Кроме того, установите флажок Блокировать первый счетчик / вспомогательный счетчик.У каждого первого соединения на потенциал подсчетчик отображаться в этом случае не будет. Например, вместо "1.1 / 1.2 / 1.3" и "2.1 / 2.2 / 2.3" будут отображатьсяобозначениясоединений "1. / 1.2 / 1.3" и "2. / 2.2 / 2.3".Если для подсчетчика дополнительно установлен флажок Включая предыдущий разделитель, обозначения соединений будут, например, такими: "1 / 1.2 / 1.3" и "2 / 2.2 / 2.3".

### Включая предыдущий разделитель

Установите этот флажок, чтобы вместе с первым счетчиком или подсчетчиком подавлялся предыдущий разделитель.

### Объем для подсчетчика

Определите здесь объем соединений, для которых будет присваиваться подсчетчик.

!!! example "Пример:"

    Если вы хотите присваивать обозначения на каждый потенциал, а на каждуюсетьвнутри этого потенциала должен создаваться подномер, выберите настройку "Сеть".Если каждое отдельное соединение в потенциале должно получать подсчетчик, выберите настройку "Отдельное соединение".
