# Диалоговое окно Выбрать для использования имеющегося

Вы открыли проект. В диалоговом окне Адреса / списки присвоения импортирован или создан список присвоений.

* Выделено несколько выводов устройства ПЛК в графическом редакторе, в диалоговом окне навигатора ПЛК — <Имя проекта> или в диалоговом окне навигатора Устройствo — <Имя проекта>. Или вы выделили одну или несколько страниц либо проект в диалоговом окне навигатора Страницы — <имя проекта>. Данные проекта > ПЛК > Использование имеющихся адресов ПЛК поблочно.
* Открыто диалоговое окно Использ имеющихся адресов ПЛК поблочно. Щелкните в панели инструментов по кнопке ++...++.
* Открыто диалоговое окно Свойства ++...++ для одной или нескольких выводов устройства ПЛК. Во вкладке Вывод устройства ПЛК щелкните рядом с полем Адрес по кнопке ++...++.

В этом диалоговом окне выводятся адреса ПЛК текущего списка присвоений и их можно выбирать для использования. При этом можно использовать как свободные, так и установленные адреса.

Обзор основных элементов диалогового окна:

### Фильтр

В этом раскрывающемся списке отображаются все доступные фильтры. Выбранный фильтр активируется автоматически. Запись "- Не активировано -" отключает фильтр и приводит к тому, что данные отображаются в неотфильтрованном виде. С помощью кнопки ++...++ откройте диалоговое окно [Фильтр](modaldialogsdb_d_filternnach.md). Здесь можно создать, обработать, удалить, скопировать, экспортировать, импортировать фильтр и управлять им.

Во всплывающем меню раскрывающегося списка Фильтр содержатся следующие записи:

* Выключить: Этот пункт меню доступен, если фильтр установлен: Сбрасывает настройку фильтра до записи "- Не активировано -".
* Активировать <Имя фильтра>: Этот пункт меню доступен, если для настройки фильтра установлено значение "- Не активировано -": Повторно активирует последний активный фильтр.

Таким образом можно быстро переключаться между неотфильтрованным и отфильтрованным в соответствии с требованиями пользователя представлениями.

С помощью фильтра можно ограничивать отображение адресов ПЛК в определенной области, например все адреса определенного ЦПУ.

### Таблица

В таблице выводятся свободные адреса списка присвоений с указанием ЦПУ, символического адреса и функционального текста. Возможен многократный выбор адресов.

Если адрес уже занят, в столбце ОУ (идентифицирующее) будет отображено ОУ, а в столбце Обозначение вывода устройства (с обозначением штекера (автоматически)) — обозначение штекера (если имеется) и обозначение вывода устройства ПЛК.

### Всплывающее меню

Всплывающее меню дает доступ, в зависимости от типа поля (например, дата, целое число, многоязычный), к пунктам меню, при помощи которых вы можете по необходимости, например, влиять на представление таблиц или обрабатывать значения в полях. Обзор пунктов этого всплывающего меню вы можете найти в разделе [Пункты всплывающего меню](userinterface_m_kontextmenu.md).
