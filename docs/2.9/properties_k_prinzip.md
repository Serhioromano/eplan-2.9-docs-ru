# Свойства EPLAN: Основные положения

Каждому объекту в EPLAN присваиваются определенные свойства. Объекты подразделяются на различные классы объектов, например, "Проекты", "Формы", "Символы", "Функции", "Ссылки изделия" и т.д. Свойства обозначаются номерами свойств (IDs), при этом один и тот же номер может присутствовать в разных классах объектов Так, например, свойству "Описание определения функции" с номером \<20117\> назначены классы объектов "Функции", "Символы" и "Соединения".

В обзоре свойств EPLAN приводится следующая информация:

Запись |  Значение
---|---
Ид. |  Номер свойства (или Ид) однозначно идентифицирует свойство. EPLAN использует этот номер внутри себя. Задайте в пользовательских настройках, должен ли отображаться номер свойства в интерфейсе программы.
Обозначение |  Название свойства, отображаемое в интерфейсе программы.
Классы объектов |  Указывает, для каких объектов доступно свойство.
Описание |  Для большинства свойств здесь приводится поясняющий текст.
Категория |  Для просмотра в программе свойства подразделены на различные категории, т.е. на группы свойств, которые могут отображаться независимо друг от друга.
Тип поля |  Тип поля определяет разрешенные символы и формат, предусмотренные для свойства. Например, возможны следующие типы полей: "Булево (истина / ложь)", "Десятичное число", "Многоязычный текст".
Индекс (макс.) |  Указывает, возможен ли индекс для свойства, если да, то здесь указывается максимально возможное значение индекса. В этом случае в диалоговом окне выбора рядом со свойством отображается знак 'плюс'. Щелкните по этому символу, тогда раскроется иерархический список возможных индексированных свойств, и вы сможете выбрать необходимое. Индекс может применяться для того, чтобы обеспечить многократное отображение свойства, например, для отображения нескольких номеров изделия, присвоенных одному и тому же устройству. Для свойства "Номер изделия" \<20100\> индекс "[3]" выдает третий номер изделия.  Если свойство имеет индекс, то обычно значение индекса "[0]" означает, что будут выведены ***все*** значения свойства, т.е., например, все присвоенные номера изделия. В иерархической структуре диалогового окна выбора это соответствует уровню с символом 'плюс'; если вы пометите его, то будут выбраны совместно все соответствующие индексированные свойства.
R/O |  Указывает, защищено ли свойство от записи (Read-only). "Да" означает защищенное от записи свойство; "Нет" означает, что свойство может обрабатываться. В зависимости от типа поля свойства вы можете тогда вводить значения вручную, или выбирать из списка, или активировать флажок.
API-Define (API-имя) |  Имя, под которым свойство может быть запрошено через API-интерфейс.
