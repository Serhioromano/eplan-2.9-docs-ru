# Диалоговое окно Импортировать записи данных

Сервисные программы > Изделие > Управление. Через меню ++Дополнительно++ > Настройки была открыта база данных изделий. [Дополнительно] > Импортировать.

В этом диалоговом окне определяется, какие данные требуется импортировать и как следует обрабатывать идентичные записи данных при импорте.

Обзор основных элементов диалогового окна:

### Тип файла

Выберите требуемый тип файла из раскрывающегося списка. В зависимости от установленного здесь формата импорта кнопка ++...++ становится активной, и в последующем диалоговом окне можно указать более подробную обработку при импорте (например,относительно файла конфигурации).

!!! note "Замечание:"

    Выберите тип файла "VDMA 66413", чтобы импортировать XML-файлы с параметрами, важными с точки зрения безопасности, которые созданы в формате обмена VDMA. "VDMA" — сокращение от названия "Союз машиностроителей Германии" — это объединение для защиты экономических интересов в машиностроительной отрасли. Оно насчитывает около 3000 членов, предприятий преимущественно среднего бизнеса и считается самым влиятельным хозяйственным объединением в Германии.
    Для выполнения импорта необходимо выполнить еще одно простое присвоение. При этом нужно указать, какие идентифицирующие свойства в XML-файле соответствуют определенным свойствам изделий в вашей базе данных изделий. Присвоение выполняется в следующем диалоговом окне, в которое можно перейти из поля Тип файла, нажав на кнопку ++...++.

### Имя файла

Введите в это поле имя файла импорта или щелкните ++...++, чтобы выполнить интерактивный выбор (возможен многократный выбор). При этом расширение имени файла добавляется автоматически в зависимости от выбранного в поле Тип файла типа файла (например, .csv для всех типов файла 'CSV для ++...++ ').

С помощью пункта всплывающего меню Вставить переменную пути перейдите в диалоговое окно [Выбрать переменную пути](modaldialogsdb_d_pfadvariablen.md), из которого можно будет скопировать одну из доступных переменных пути.

### Присвоения полей

Выберите из этого раскрывающегося списка схему, заданную для структуры и произвольных свойств, в каких полях базы даннных считываются экспортированные данные. Благодаря этому облегчается обмен данными между клиентами, которые используют различные структуры.
Нажмите ++...++, чтобы открыть диалоговое окно Присвоения полей и там создать новую, определенную пользователем, схему.

### Обработка при импорте

С помощью опций в нижней части диалогового окна также указываются следующие настройки:

* Добавлять только новые записи данных: существующие записи данных не учитываются, а добавляются только новые.
* Обновить только имеющиеся записи данных: игнорирует новые записи данных и обновляет только различные данные уже существующих изделий; это необходимо, например, в случае изменения цен.
* Обновить имеющиеся записи данных и добавить новые: комбинирует оба указанных выше параметра.

!!! note "Замечание:"

    Обратите внимание на то, что при операции ***Обновить***, применяемой к атрибутам и свободным свойствам, данные дополняются, а не заменяются. Но для шаблонов функций существующие данные будут тем не менее заменены.
