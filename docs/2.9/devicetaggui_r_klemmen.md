# Вкладка Клемма

Вы открыли проект.

* В графическом редакторе уже выделена клемма. Обработать > Свойства. Выберите в диалоговом окне Свойства ++...++ вкладку Клемма.
* Вы выделили клемму в навигаторе клеммников или в навигаторе устройств. Пункт всплывающего меню Свойства или Свойства (общие).Выберите в диалоговом окне Свойства ++...++ вкладку Клемма.
* В навигаторе клеммников при помощи пункта всплывающего меню Генерировать клемму генерируется клемма.

В этой вкладке вы можете задать и обработать свойства клемм. Неизменяемые свойства недоступны для выбора.

В режиме обработки ***Свойства (общие)*** обрабатываются свойства для всех представлений клемм сразу. В этом случае отображаются дополнительные вкладки Определение клеммника (Устройство) и Изделие (Устройство) для соответствующей главной функции.

Обзор основных элементов диалогового окна:

### Видимое ОУ

Видимое ОУ будет показано в графическом редакторе рядом с условным обозначением. Если вы определили в настройках проекта, что устройство должно автоматически нумероваться при вставке символов, здесь вам будет автоматически предложено ОУ. Вы можете сохранить или переписать вручную это ОУ.

Кроме того, можно использовать [всплывающее меню или кнопку ++...++](devicetaggui_r_feldsichtbaresbmk.md), чтобы cгенерировать или выбрать ОУ.

### Полное ОУ

В этом поле отображается полное ОУ (т. е. структурный идентификатор текущего окружения + видимое ОУ) видимого ОУ / функционального элемента. С помощью кнопки ++...++ откройте диалоговое окно Полное ОУ. Здесь можно обработать структурный идентификатор и, например, выбрать другое место установки.

### Обозначение

Это поле показывает обозначение клеммы. На основании обозначения клеммы определяется позиция клеммы в клеммнике. введенное здесь значение сохраняется в свойстве Обозначение клеммы / контакта штекера.

!!! tip "Совет:"

    Если в поле Видимое ОУ ввести двоеточие, все символы после двоеточия автоматически перенесутся в поле Обозначение.

### Описание

Введите здесь соответствующее описание клеммы.

### Обозначение вывода устройства

В этом поле отображаются обозначения выводов устройства текущей функции, а также обозначения выводов устройств тех функций, в которые копируется ОУ текущей функции.

Обозначения выводов устройств для функции или для устройства можно изменить вручную или выбрать запись из раскрывающегося списка. Несколько обозначений выводов устройств разделяются знаком абзаца. При обработке нескольких обозначений выводов устройств вручную необходимо учитывать следующее:

* Отдельные обозначения выводов устройств можно изменить вводом значения на соответствующей позиции между знаками абзаца.
* При перезаписывании всей записи одним отдельным значением это значение будет присвоено первому выводу устройства. Остальные обозначения выводов устройств останутся без изменения.
* При удалении всей записи удаляются все обозначения выводов устройств.

!!! note "Замечание:"

    * Содержимое данного поля не сохраняется в отдельном свойстве. Чтобы отобразить обозначения выводов устройств на условном обозначении, можно воспользоваться свойством Обозначения выводов устройства (все) (ID 20038). Это свойство отображает все обозначения выводов устройств текущей функции. Однако это свойство не содержит отображаемые в поле Обозначение вывода устройства обозначения выводов устройств других функций.
    * При помощи опции Отображать обозначения выводов устройства в повернутом виде в настройках проекта (Параметры > Настройки > "Имя проекта" > Графическая обработка > Общее) вы можете задать, чтобы поля Обозначение клеммы / контакта штекера и Обозначение вывода устройства отображались в повернутом виде. Эта настройка учитывается только для порядка свойств "Стандартное значение по умолчанию".

### Описание вывода устройства

Введите здесь описание выводов устройств для текущей функции или устройства или выберите эти данные из раскрывающегося списка.

### Уровень

Здесь можно указать, является ли клемма многоуровневой клеммой. Многоуровневые клеммы состоят из нескольких расположенных друг над другом уровней.

Если вы введете значение > 0, то клемма будет определена как многоуровневая клемма. Значение "1" соответствует нижнему уровню.

При значении "0" клемма будет определена как "одиночная" клемма.

### Функциональный текст

Укажите здесь функциональный текст для функции / устройства.

Независимо от введенного здесь вручную функционального текста существуют другие (определяемые автоматически) функциональные тексты, которые находятся в таблице:

* Функциональный текст (автоматически): Отображает содержание введенного вручную функционального текста или, при его отсутствии, функциональный текст зоны. Если вы изменяете функциональный текст в режиме обработки ***Свойства (общие)***, его содержание переносится обратно в ту функцию, в которой функциональный текст был первоначально задан. При отображении этого свойства на странице проекта разрывы строк удаляются.
* Функциональный текст (общий): отображает (автоматический) функциональный текст функции или, при его отсутствии, (автоматический) функциональный текст соответствующей главной функции.

### Мостовая перемычка

Здесь можно указать позицию мостовой перемычки. Возможность автоматического образования мостовой перемычки задается в определении функции клеммы (дополнительное свойство Возможность мост. перемычки). В раскрывающемся списке доступны следующие опции:

* Автоматически: клеммы, которые располагаются рядом на одном клеммнике и связаны друг с другом каким-либо образом, автоматически соединяются с помощью мостовой перемычки.
* Автоматически, начало перемычки: начало мостовой перемычки, сгенерированной автоматически. Препятствует попаданию клемм, расположенных впереди сортировки, в сгенерированную мостовую перемычку.
* Автоматически, конец перемычки: конец мостовой перемычки, сгенерированной автоматически. Препятствует попаданию клемм, расположенных сзади сортировки, в сгенерированную мостовую перемычку.
* Без автоматических перемычек: Предотвращает автоматическое построение мостовой перемычки с обеих сторон.

!!! note "Замечание:"

    Вручную созданные мостовые перемычки можно задать в диалоговом окне Обработать клеммник. Настройки для мостовых перемычек сохраняются на клемме, которая представляет начало перемычки. Для этого клеммы были расширены свойствами Вручную созданные мостовые перемычки (внутр.) и Вручную созданные мостовые перемычки (внешн.).

### Определение функции

С помощью кнопки ++...++ откройте диалоговое окно Определения функций. Определения функций там уже отфильтрованы, и выводятся только клеммы. Выберите нужную клемму.

Это поле синхронизируется с полем Определение, находящимся на вкладке Данные символа / функции.

### Главная клемма

Если установлен данный флажок, то клемма является главной клеммой и ведет себя аналогично главной функции. Например, для главной клеммы в навигаторе клеммников отображаются шаблоны функций, и можно выполнить выбор устройства. Клеммник может содержать любое количество главных клемм.

Если флажок снят, то клемма является вспомогательной клеммой.

### Распределенная клемма

Если флажок установлен, то клемма является распределенной клеммой. Одна распределенная клемма может представлять один или несколько выводов устройства клеммы. Вкладка Соединения для распределенных клемм не отображается.

Если флажок снят, то клемма полная, т. е. все выводы устройств функции клеммы представлены на данной клемме.

[Групповое поле "Свойства"](devicetaggui_r_gruppenfeldeigenschaften.md)

Для клемм доступны, в частности, следующие свойства:

* Категория клеммы: данное свойство позволяет точно охарактеризовать клемму. Это может быть клемма, например проходная клемма, разъединительная клемма, клемма переключения, диодная клемма и т. д. Это свойство доступно в отчетах по клеммам и там его можно использовать для вывода графики клемм.
* Допускается одинаковое обозначение: при помощи этого свойства можно указать, чтобы EPLAN не выводил сообщения при наличии нескольких клемм с одинаковыми обозначениями. Это может быть целесообразно, например, для PE- / PEN-клемм.

!!! note "Замечание:"

    Если вы размещаете клеммы в нескольких видах представления, вам не следует устанавливать эту настройку. Определите каждую клемму однозначно. Иначе при обработке во всех представлениях возникнут проблемы.

* Клемма: Положение устройства: С помощью этого свойства идентифицируются группы клемм (напр., многоуровневые клеммы). Все клеммы группы клемм имеют одинаковое положение устройства. Внутри группы клемм идентификатор сортировки устанавливает порядок клемм.
