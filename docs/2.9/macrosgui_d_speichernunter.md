# Диалоговое окно Сохранить как

Вы открыли проект. Выделены требуемые элементы или страницы, которые необходимо сохранить в виде макроса.

* Обработать > Создать макрос окна / символа
* Страница > Макрос страницы > Создать
* Данные проекта > Предварительное планирование > Макрос предварительного планирования > Создать

С помощью этого диалогового окна можно сохранить ранее выделенные элементы или страницы как макрос.

Обзор основных элементов диалогового окна:

### Каталог

В этом поле выводится каталог для сохранения макроса с указанием дисковода и пути файла. Вы также можете при необходимости изменить эту настройку каталогов.

### Имя файла

Введите здесь имя файла макроса или же выберите его в интерактивном режиме, щелкнув по кнопке ++...++. В случае повторного сохранения одинаковых типов макроса предварительно настраиваются имя и другие настройки (Каталог, вариант и т. д.) последнего сохраненного макроса.

### Вид представления

Невозможно выбрать для макросов страниц и макросов предварительного планирования! Вид представления макроса служит для сортировки, т. е. она помогает в управлении вашими макросами. Она не влияет на функции внутри макроса, сохраняя соответствующий вид их представления. Таким образом, все макросы изделия могут содержаться в файле. Для каждого вида представления макроса можно создать до 26 вариантов. Вид представления предварительно настроен согласно выделенным функциям или согласно типу страниц. Выберите требуемую запись из раскрывающегося списка:

* Многополюсный: Для макросов, которые размещаются на многополюсных страницах схемы соединений.
* Многопол. Fluid: для макросов, которые размещаются на страницах схемы соединений для Fluid-Техники.
* Обзор: Для макросов, которые размещаются на страницах обзора.
* Парная перекрестная ссылка: Для макросов, которые используются для реализации парной перекрестной ссылки.
* Однополюсный: Для макросов, которые размещаются на однополюсных страницах схемы соединений.
* Топология: Для макросов, которые размещаются на страницах топологии.
* Функциональная схема автоматизации: Для макросов, которые размещаются на страницах функциональной схемы автоматизации.
* Функциональный: Для макросов, которые размещаются на блок-схемах рабочего процесса.
* Графика: Для макросов, которые содержат только графические условные обозначения. Графические условные обозначения не учитывается ни при прогонах отчетов, ни при контрольных прогонах, ни при образовании перекрестных ссылок, и ***не*** регистрируются как цель.
* Размещение изделия: Для макросов, которые размещаются на монтажных платах.
* Компоновка электрошкафа: Для макросов, размещаемых на страницах компоновки электрошкафа.
* Предварительное планирование: Для макросов, которые вставляются в навигатор предварительного планирования.

!!! note "Замечание:"

    Если функция имеет другие виды представлений, где общие свойства содержат различные данные, в зависимости от вида представления выносится решение о том, какие свойства будут учитываться при составлении отчета. Для этого в EPLAN существует установленная [последовательность использования видов представлений при общей обработке и генерации отчетов](reverseengineering_k_reihenfolge.md).

### Вариант

Невозможно выбрать для макросов страниц и макросов предварительного планирования! С помощью раскрывающего списка указывается обозначение варианта для создаваемого макроса. В качестве предварительной настройку установлен "Вариант A".

!!! note "Замечание:"

    Варианты угла и другие варианты макроса не генерируются автоматически, а должны быть созданы вручную на страница проекта, а затем сохранены с тем же именем макроса, но с другим обозначением варианта. Для каждого вида представления макроса можно создать до 26 вариантов. Если для макроса созданы различные варианты, при вставке можно выбрать один их них.

Если выбрать уже имеющийся вариант макроса, то после щелчка по кнопке ++Ok++ будет выведен дополнительный запрос, в котором требуется подтвердить перезапись старого варианта макроса.

### Описание

Введите здесь информативное описание файла макроса. При вставке макроса этот текст выводится под графическим просмотром в поле комментария.

### Всплывающее меню

Всплывающее меню дает доступ, в зависимости от типа поля (например, дата, целое число, многоязычный), к пунктам меню, при помощи которых вы можете по необходимости, например, влиять на представление таблиц или обрабатывать значения в полях. Обзор пунктов этого всплывающего меню вы можете найти в разделе [Пункты всплывающего меню](userinterface_m_kontextmenu.md).

### Учитывать масштаб страницы

Невозможно активировать для макросов страниц и макросов предварительного планирования! Если этот флажок установлен, при вставке макрос оптически адаптируется так, что первоначальный размер остается неизменным, но на странице отображается увеличенным или уменьшенным по отношению к настроенному масштабу.

Если этот флажок не установлен, в соответствии с масштабом страниц макрос увеличивается / уменьшается.

!!! example "Пример:"

    Следующие графические элементы сохраняются как макрос окнаtest.ema.Случай 1. Если флажок установлен, при сохраненииtest.emaмасштаб страницы, на которую вставляется макрос, составляет "1:5".Случай 2. Если флажок снят, при сохраненииtest.emaмасштаб страницы, на которую вставляется макрос, составляет "1:5".

### Число страниц

В случае макросов страницы здесь выводится число страниц, сохраненных в макросе.

Это поле доступно для обработки, если в навигаторе страниц выделена только одна страница или если курсор находится на открытой странице. Щелкните по стрелкам, чтобы изменить предварительную настройку "1". В этом случае страницы сохраняются в макрос, начиная с выбранной страницы и в соответствии с числом страниц.

### [Дополнительно]

Нажмите эту кнопку, чтобы выполнить следующие операции:

* Определить точку захвата: невозможно активировать для макросов страниц и макросов предварительного планирования! Точкой захвата называется точка, на которой "захватывается" курсор при вставке макроса. Обычно макрос захватывается "вверху, слева" у первой точки вставки.
Выберите эту опцию, если необходимо указать положение точки захвата. Диалоговое окно Сохранить как кратковременно закрывается. Переместите курсор в требуемую позицию и щелчком мыши определите место на странице проекта, где должна находиться точка захвата для макроса. После этого EPLАN вновь вызывает диалоговое окно Сохранить как и работу над макросом можно завершить.
Позиционирование точки захвата можно повторить в любое время. Ее положение в макросе при создании и позже при предварительном просмотре макроса обозначается специальным графическим символом.
* Присвоить данные изделия: Невозможно активировать для макросов предварительного планирования! В следующем диалоговом окне [Присвоение данных изделия](macrosgui_r_artikeldatenzuweisung.md) свойству объекта можно присвоить данные изделия.
