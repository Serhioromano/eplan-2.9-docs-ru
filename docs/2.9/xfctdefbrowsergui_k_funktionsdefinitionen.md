# Определения функции: Принцип

Определение функции определяет стандартные характеристики функции. Она определяет, как объект будет обрабатываться при генерации отчетов (например, перечисляются контакты в образе контакта, клеммы в спецификации клеммника и т.п.).

Кроме того, в определении функции указывается дальнейшая информация, например, является ли функция с разделением сигнала или нет. Эта логическая информация относится немного ко всей функции, но большей частью к выводам устройства.

Определение функций имеет такие свойства:

* ***Имя:*** У определения функции есть имя функции, например "Силовой замыкающий контакт", "Катушка АС" и т. д.
* ***Описание:*** Задайте техническую цель, для которой будет использоваться функция.
* ***Число выводов устройства:*** Нормальное определение функции имеет как минимум один вывод устройства. Существуют особые функции, у которых нет вывода устройства (например, черный ящик). Количество может быть переменным, тогда они заносятся здесь как"n". В этом случае не указывается подробная логическая информация для выводов устройства. Возможно только данные, которые действительны для всех выводов устройства ("n") и данные для последнего вывода устройства ("z"), если существует несколько выводов устройства.

Для каждого определения функции могут быть сохранены 10 предложений обозначения вывода устройства и 10 предложений описания вывода устройства.

На функции (т. е. на условном обозначении) можно произвольно настраивать логическую схему выводов устройства, независимо от того, что несет с собой определение функции. Введенные там свойства заносятся на условном обозначении и перезаписывают свойства определения функции.

Для изделий Вы можете записать определения функций и таким образом у Вас есть возможность выбора устройств. Затем система автоматически предлагает соответствующие символы, которые подходят к определениям функций, указанным на устройстве.

### Библиотека определений функций

Библиотека определений функций - это коллекция определений функций. Библиотека определений функций упорядочена иерархически по разделу, области, категории, группе и определению функции. Эта группировка служит для быстрого поиска требуемого определения функции. В диалоговом окне выбора определения функции отображаются в структуре дерева согласно данной группировке.

Библиотеки определений функций поставляются EPLAN для распространенного использования и актуальны для функциональных элементов, присутствующих на рынке, они не могут быть ***созданы*** или расширены Вами самими.

Слой |  Значение
---|---
Раздел |  Определите, для какой технической цели используется функция, например "Электротехника" или "Гидравлика".
Область |  Задайте области в пределах раздела, например "Катушки и контакты" или "Двигатели".
Категория |  Определите работу системы функции. Это обозначает также, что независимо от категории (например, контакты, защитные устройства, катушки, клеммы), в распоряжение функций представлены определенные свойства. Свойство Адрес ПЛК, например, существует только для функций ПЛК, а свойство Катушка: Напряжение — только для катушек.
Группа |  Задайте основную функцию соответствующей категории. Так, например, функция "Замыкающий контакт (2 вывода устройства)" является общим определением и уточняется"силовым замыкающим контактом".
Определение функции |  Определите специальный вид функции, например,"Силовой замыкающий контакт".

#### Общие определения функции

Во всех группах определений функции существуют общие определения функции, например "Клемма, общее, 2 вывода". Такое общее определение функции задает основополагающую функциональность функционального элемента. Все определения функций, находящиеся в пределах одной группы, могут меняться. Таким образом, сначала можно использовать общую функцию в проекте и только позднее задать более точное определение.

Общие определения функций предназначены для использования в проекте, если при проектировании еще не установлено точно, о каком изделии идет речь. При выборе устройств общие определения функции соответствуют всем точным определениям функции внутри одной группы. Т. е. при выборе устройства определение "Клемма, общее, 2 вывода" можно заменить как на "N-клемма, 2 вывода", так и на "PE-клемма, 2 вывода".

Если в базе данных изделий указываются шаблоны функции, следует использовать только точные определения функции.
