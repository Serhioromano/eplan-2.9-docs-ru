# Вкладка Шаблоны функций

Сервисные программы > Изделие > Управление. В дереве или списке выделено одно изделие. Выберите вкладку Шаблоны функций.

!!! note "Замечание:"

    Поскольку для модулей нельзя установить шаблоны функций, поля в этой вкладке в случае обработки модуля затенены.

Шаблон функции является важным элементом для синхронизации выбранных (через выбор устройства) устройств с запроектированными (по схеме соединений) устройствами и проверить их на правильность. В каждом шаблоне функции наряду с определением функции указаны идентифицирующие свойства функции (например, обозначение вывода устройства). Таким образом, здесь определяется, какие поля являются идентифицирующими для сравнения: сравнение выполняется именно для того, что предварительно было указано в шаблоне функции.

Вносите в шаблонах функций только точные определения функций. [Общие определения функции](xfctdefbrowsergui_k_funktionsdefinitionen.md), например "Клемма, общее, 2 вывода", не следует использовать в базе данных изделий.

!!! example "Пример:"

    Если указать для обозначений выводов устройства13¶14для определения функции "замыкающий контакт", то замыкающий контакт с обозначениями выводов устройства23¶24распознается как ошибка. Если определение функции остается пустым, то замыкающий контакт с обозначениями выводов устройства23¶24разрешен.

На этой вкладке через шаблон функции задается часть технических свойств ***варианта***; таким образом, шаблон функции получает данные ***функции***, причем функция является, например, контактом, кабельным соединением, штырем штекера и т. д. При этом данные для каждого изделия могут указываться ***многократно***, а именно один раз для каждого варианта.

!!! note "Замечание:"

    Информация о выводах устройства первого шаблона учитывается, только если для этого изделия предлагается несколько ***вариантов***. При размещении изделия выводы устройства копируются из шаблонов функций (при их наличии), если они полностью представлены на схеме соединений изделия или если на ней не определены выводы устройства. Таким образом, например, вспомогательные блоки в контакторах могут существовать в разных вариантах изделий, чтобы отображать различные обозначения выводов устройства ("13;14" и "23;24"). Это важно при ***генерировании маршрутизируемых соединений***.

    Для работы со схемами соединений в нескольких вариантах изделий предусмотрен следующий порядок действий:

1. На схеме соединений определены все выводы устройства, которые необходимо использовать во всех вариантах.
2. В самих вариантах новые выводы устройства в дополнение к выбранной схеме соединений не генерируются.

Обзор основных элементов диалогового окна:

### Выбор устройства (шаблоны функций)

В таблице установите данные, которые анализируются при ***выборе устройства***. Таблица содержит при этом строку для каждого определения функции варианта изделия. Последовательность столбцов не конфигурируется.

Поля, которые здесь отображаются рядом со столбцом Определение функции, зависят от выбранного определения функции.

!!! note "Замечание:"

    При этом столбцу Описание придается особое значение: занесенный в этот столбец (многоязычный) текст при выборе изделия / вставке устройства не переносится на функцию. Зато описание отображается в представлениях в виде дерева в навигаторах. Там описание заключается в квадратные скобки и стоит непосредственно за определением функции.

### Всплывающее меню

Всплывающее меню дает доступ, в зависимости от типа поля (например, дата, целое число, многоязычный), к пунктам меню, при помощи которых вы можете по необходимости, например, влиять на представление таблиц или обрабатывать значения в полях. Обзор пунктов этого всплывающего меню вы можете найти в разделе [Пункты всплывающего меню](userinterface_m_kontextmenu.md).

!!! note "Замечание:"

    Таблица переключается при выборе нескольких вариантов / изделий на "С защитой от записи", потом ее нельзя обрабатывать.
