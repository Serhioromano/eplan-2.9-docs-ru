# Использовать и отменять имеющиеся адреса ПЛК

Если проект содержит свободные адреса ПЛК, то их можно использовать и присваивать имеющимся в проекте выводам устройства.

На отдельном выводе устройства ПЛК в диалоговом окне Свойства ++...++ можно использовать один имеющийся адрес или снова отменить его. В диалоговом окне Использ. имеющихся адресов ПЛК поблочно имеется возможность одновременно использовать несколько адресов (также для различных размещений выводов устройства ПЛК).

### Использование отдельных адресов ПЛК

Условие:

Вы открыли диалоговое окно Свойства ++...++ для вывода устройства ПЛК.

1. Выберите вкладку <Категория функции> и щелкните рядом с полем Адрес по ++...++.

!!! info "Для сведения:"

    В диалоговом окне Выбрать для использования имеющегося отображаются все адреса ПЛК данного проекта.

2. Выберите в таблице требуемый адрес ПЛК.
3. Щелкните по кнопке ++OK++.

!!! info "Для сведения:"

    Адрес ПЛК переносится в диалоговое окно Свойства ++...++.

!!! tip "Совет:"

    При помощи фильтра вы можете ограничить отображение адресов ПЛК одним (или несколькими) ЦПУ. Фильтр активируется автоматически сразу после выбора из раскрывающегося списка Фильтр.

### Отменить отдельные адреса ПЛК

Условия:

* Вы открыли диалоговое окно Свойства ++...++ для вывода устройства ПЛК.
* Выбранный адрес вывода устройства ПЛК не указан ни в каких других представлениях этого вывода.

1. Выберите вкладку <Категория функции> и щелкните рядом с полем Адрес по кнопке ![](images/all_rebook_as.png){: .ui-icon }.

!!! info "Для сведения:"

    Адрес удаляется из этого поля и добавляется в текущий список присвоений как свободный адрес, но значения Функциональный текст (автоматич.) и Символический адрес (автоматич.) при этом сохраняются.

!!! info "Для сведения:"

    Функциональные тексты и / или символические адреса, заданные вручную, удаляются.

2. Щелкните рядом с полем Адрес по кнопке ++...++, чтобы использовать новый адрес, или оставьте это поле пустым, чтобы заполнить его адресом позднее.
3. Щелкните по кнопке ++ОК++.

!!! tip "Совет:"

    В режиме обработки ***Свойства (общие)*** предусмотрена возможность отменить адреса одновременно на всех выводах устройства. Пробелы в адресе при этом ***не*** учитываются, то есть адреса "E0.0" и "E 0.0" считаются идентичными друг другу.

### Использ. имеющихся адресов ПЛК поблочно

Условие:

Несколько выводов устройства ПЛК выделено в графическом редакторе, навигаторе ПЛК или в навигаторе устройств.

1. Выберите пункты меню Данные проекта > ПЛК > Использовать адреса поблочно.

!!! info "Для сведения:"

    Диалоговые окна Использ. имеющихся адресов ПЛК поблочно и Выбрать для использования имеющегося открываются автоматически, причем последнее находится на переднем плане и может быть сразу обработано.

2. Выделите адреса ПЛК, которые необходимо использовать, и щелкните по ++OK++.

!!! info "Для сведения:"

    В диалоговом окне Использ. имеющихся адресов ПЛК поблочно только что выбранные адреса ПЛК отображаются в таблице Из.

3. При помощи кнопок со стрелками переместите адреса ПЛК в правой таблице, чтобы присвоить их выводам устройства ПЛК в левой таблице Вкл.

!!! info "Для сведения:"

    При перемещении записи можно поместить только на другое свободное место; на невыделенных занятых местах изменение не происходит.

!!! info "Для сведения:"

    Если выбранные количества отличаются, следует откорректировать это, добавив ниже пустые строки.

4. Щелкните по кнопке ++OK++.

!!! info "Для сведения:"

    Адреса ПЛК используются и присваиваются выводам устройства ПЛК.

!!! info "Для сведения:"

    На точки подвода питания переносятся адрес, символический адрес и функциональный текст только в том случае, если эти свойства были уже заполнены предварительно значениями.

!!! info "Для сведения:"

    Если по выводу устройства ПЛК также выделены соответствующие точки подвода питания и представления на страницах обзора, то адрес используется совместно для всех представлений.

!!! tip "Совет:"

    При помощи фильтра вы можете ограничить отображение адресов ПЛК одним (или несколькими) ЦПУ. Фильтр активируется автоматически сразу после выбора из раскрывающегося списка Фильтр.
