# Управление перемычками

Перемычки служат для того, чтобы соединять между собой клеммы и клеммники. Производители клемм предлагают множество возможностей соединения для клемм. Клеммы, расположенные далеко друг от друга, соединяются, как правило, при помощи проводов. В этом случае применяются ***проволочные перемычки***. Если клеммы находятся близко друг от друга, применяются металлические перемычки, позволяющие также соединять их с соседними клеммами, чтобы распределить имеющийся потенциал по нескольким выводам. Эти перемычки обозначаются как ***мостовые перемычки***. Клеммы, приспособленные для мостовых перемычек, имеют, как правило, расположенный посередине паз, в который может вставляться металлическая перемычка. С помощью ***Вставные перемычки*** можно соединить соседние или близко расположенные клеммы. Вставные перемычки чаще всего представляют собой соединительные элементы в форме гребенки, которые прикручиваются к клеммному выводу. Обычно для прямого соединения расположенных рядом клемм используются мостовые и вставные перемычки, но они могут пропустить некоторые клеммы.
В технике электроснабжения часто используются переключаемые перемычки (***Перемычки***) и разъединительные клеммы.

### Типы перемычек и создание перемычек в EPLAN.

В EPLAN перемычки являются соединениями со специальными свойствами. Имеются различные типы перемычек:

* Перемычки (общее)
* Проволочная перемычка
* Вставная перемычка
* Мостовая перемычка
* Переключаемая перемычка.

Тип перемычки задается в свойствах соединения с помощью определения функции соединения. Последняя имеет преимущество перед настройками, которые вы можете определить на клеммах при создании перемычек. Соедините две соседние клеммы клеммника на схеме соединений, по умолчанию "Мостовая перемычка" настроена как определение соединения (при условии, что клеммы имеют возможность установки мостовых перемычек и не являются клеммами PE / PEN). Если соединяются две несоседние клеммы одного клеммника, то по умолчанию в качестве определения соединения устанавливается "Проволочная перемычка".

На самих клеммах в диалоговом окне Свойства ++...++ у вас есть возможность повлиять на создание мостовой перемычки через поле Мостовая перемычка. Здесь вы указываете, должна ли мостовая перемычка быть создана автоматически или создавать перемычку не нужно. По умолчанию, когда вы вставляете клемму в схему соединений, устанавливается создание перемычки "Автоматически".

!!! note "Замечание:"

    Имеет ли клемма возможность установки мостовых перемычек, отображается в диалоговом окне Логическая схема выводов устройства. В этом случае клемма имеет дополнительные выводы устройства функции с типом вывода "Мостовая перемычка". (Так задано в определении функции.) Количество возможных мостовых перемычек задается в строке Число мостовых перемычек.
    В диалоговом окне Обработать клеммник также можно определять возможность установки мостовых перемычек. В столбцах Мостовые перемычки (внешн.) и Мостовые перемычки (внутр.) графика (малая точка) указывает, доступен ли для клеммы хотя бы один внешний / внутренний вывод мостовой перемычки.

#### Автоматически создаваемые мостовые перемычки

Настройка мостовых перемычек "Автоматически" означает, что для связанных друг с другом соседних клемм одного клеммника системой EPLAN автоматически предусматривается мостовая перемычка. При условии, что клеммы имеют возможность установки мостовых перемычек и не являются клеммами PE / PEN. Кроме того, при автоматически создаваемых мостовых перемычках можно определять, является ли выделенная клемма началом или концом перемычки.

#### Создание мостовых перемычек вручную

Вручную созданные мостовые перемычки можно также генерировать между не соседствующими друг с другом клеммами. Для мостовых перемычек, определенных вручную необязательно, чтобы существовали соединения. Таким образом можно создавать мостовые перемычки даже в том случае, когда между соединяемыми клеммами находится еще несколько клемм, не участвующих в образовании перемычки.

Мостовые перемычки можно отдельно генерировать для внутренних и внешних выводов мостовых перемычек. Для этого в диалоговом окне Обработать клеммник предусмотрена кнопка ![](images/terminalgui_createsaddlejumpext_as.png){: .ui-icon } (Генерировать созданную вручную мостовую перемычку (внешн.)) и ![](images/terminalgui_createsaddlejumpint_as.png){: .ui-icon } (Генерировать созданную вручную мостовую перемычку (внутр.)).

Настройки для вручную созданных мостовых перемычек сохраняются на клемме, которая представляет начало перемычки. Для этого клеммы были расширены свойствами Вручную созданные мостовые перемычки (внутр.) (Ид. 20350) и Вручную созданные мостовые перемычки (внешн.) (Ид. 20351). В этих свойствах определяется "Гребенка перемычки". Для этого от начала перемычки указывается величина шага до следующих клемм с перемычками, а также величина шага до относящегося к ней уровня.

!!! example "Пример:"

    Вы вставили в схему соединений пять клемм, которые обладают возможностью установки мостовых перемычек и не соединены через линии автоматического соединения. В диалоговом окне Обработать клеммниксгенерируйте вручную созданную мостовую перемычку (внутр.) между клеммамиX1:1,X1:3иX1:5.Определенные таким образом мостовые перемычки сохраняются на клеммеX1:1(начало перемычки) в свойстве Вручную созданные мостовые перемычки (внутр.)со значением2/0;2/0. При этом число до косой черты указывает величину шага до следующей клеммы с перемычками, а число после косой черты — величину шага до следующего уровня с перемычками. Начиная с текущей клеммы, величина шага составит две клеммы, и для клеммыX1:3будет генерироваться вручную созданная мостовая перемычка. Затем отсчет продолжится от клеммыX1:3также с величиной шага в две клеммы, и вручную созданная мостовая перемычка будет генерироваться для клеммыX1:5. Т. к. уровень при этом не меняется, величина шага для уровня в обоих случаях равна0.Мостовые перемычки между различными уровнями В этом примере первые три клеммы образуют многоуровневую клемму. На рисунке в первом столбце показано обозначение клеммы, а во втором — уровень.При генерировании вручную созданной мостовой перемычки между клеммамиX1:2иX1:5она сохраняется на клеммеX1:2(начало перемычки) в свойстве Вручную созданные мостовые перемычки (внутр.)со значением2/-2. Многоуровневая клемма рассматривается как группа клемм, поэтому величина шага составляет2, несмотря на то, что фактически отсчитывается по три отдельные клеммы. Т. к. перемычка ведет от уровня2к уровню0, величина шага для уровня составляет здесь-2.

Вследствие того, что вручную созданные мостовые перемычки определяются на клемме, перемычки ориентированы на устройство. Длина перемычки задается в соответствии с изделием перемычки на клемме. После обработки других клемм из клеммника длина перемычки останется неизменной. Путем вставки клемм между двумя клеммами, соединенными перемычками вручную, мостовой перемычке задается другая цель.

Если для вручную созданной перемычки не найден конец, она отображается как неисправная в диалоговом окне Обработать клеммник. В нем представлен открытый конец перемычки. Перемычку можно корректировать или удалить.

!!! note "Замечание:"

    Автоматические или вручную созданные мостовые перемычки отличаются цветом графики (красный = автоматически, черный = вручную) в диалоговом окне Обработать клеммник в столбцах для отображения мостовых перемычек.

С помощью настройки пользователя Удалить ручную мостовую перемычку при копировании можно задать, будет ли информация о перемычке сохраняться или удаляться при копировании и вставке клемм в графическом редакторе.

#### Переключаемые перемычки

Переключаемая перемычка соединяет клемму со ***следующей*** клеммой с возможностью переключения. Переключаемые перемычки имеют два состояния подключения ("разомкнуто" и "замкнуто") и определяются отдельно для внутренней и внешней стороны клеммы. Для образования переключаемых перемычек используются выводы мостовых перемычек (если таковые имеются), либо выводы клемм. При этом для внешних переключаемых перемычек используются только внешние, а для внутренних — только внутренние выводы мостовых перемычек / клемм.

Настройка "Закрыто" влияет на логическую схему (соединений, потенциалов и т. д.). Если эта настройка выбрана для какой-либо клеммы, генерируется соединение со следующей клеммой посредством переключаемой перемычки. Настройка "Открыто" не влияет на логическую схему, она просто служит индикатором состояния подключения.

#### Разъединительные клеммы

Разъединительные клеммы соединяют или разъединяют внешнюю и внутреннюю сторону клеммы. На состояние разъединительной клеммы в EPLAN указывают свойства Блокировка переключения (ид. 20293) и Клемма открыта (ид. 20232). Отображение состояний подключения на схеме соединений может осуществляться напрямую с помощью этих свойств или через свойства блока.

* Свойство Блокировка переключения указывает на то, что клемма заблокирована и не может переключаться.
* Свойство Клемма открыта указывает на то, что клемма открыта. Если это свойство активировано, соединение между внутренней и внешней стороной клеммы разъединяется, т.е. прерывается дальнейшая передача потенциала внутри клеммы.
Это свойство также можно использовать для распределенных клемм. Чтобы распознать передачу потенциала внутри клеммы, необходимо включить настройку Автоматически связать родственные распределенные клеммы (в диалоговом окне [Настройки: Распределенные клеммы](terminalgui_d_projekteinstellungenklemmen.md)).

#### Правила автоматического создания перемычек

* Автоматически создаваемые мостовые перемычки создаются только между клеммами с одинаковым определением функции. Исключение составляет "Клемма, общ."; она может составлять мостовые перемычки со всеми клеммами. У многоуровневых клемм также должны совпадать уровни.
* Автоматически создаваемые мостовые перемычки могут создаваться только в том случае, если, по меньшей мере, на одном выводе устройства в определении функции возможна мостовая перемычка. Для этого на логической схеме выводов устройства значение свойства Число мостовых перемычек не должно равняться нулю.
* Если у обеих клемм имеется возможность установки мостовых перемычек, то выполняется генерирование. Клеммы должны прилегать непосредственно друг к другу и между ними не должно быть никаких последовательных принадлежностей.
* Для соединений между клеммами, по меньшей мере одна из которых не имеет возможности установки мостовой перемычки, справедливо: если имеются общие клеммы или одна общая и одна PE- / PEN-клемма, то будет создана ***проволочная перемычка***.
* Между двумя клеммами PE / PEN всегда генерируется ***прямое соединение***, а также не задаются цели. Возможность установки мостовых перемычек при этом не играет никакой роли.
* Между клеммами с ***прямыми выводами устройства*** автоматически генерируются прямые соединения, если прямые выводы устройства были соединены между собой на схеме соединений. Такие прямые соединения отображаются в диалоговом окне Обработать клеммник в столбцах Мостовые перемычки (внешн.) и Мостовые перемычки (внутр.). В отличие от мостовых перемычек выводы устройства при прямых соединениях отображаются в виде прямоугольников вместо кругов. Такой вид представления также действителен для отчетов.

!!! tip "Совет:"

    Свойства графически представленного соединения можно изменять с помощью точки определения соединения и из проволочной перемычки, например, создавать вставную перемычку.
