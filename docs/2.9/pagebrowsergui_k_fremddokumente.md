# Внешние документы

В проект можно интегрировать внешние документы (например, файлы MS Word или PDF). Использую тип страницы Внешн. документ эти файлы можно вывести в навигаторе страниц и в содержании. Независимо от типа файла документов для внешних документов используется только один тип страницы. В навигаторе страниц внешний документ выводится в виде символа, который присвоен этому типу файлов в операционной системе. Так, например, для документа MS Word используется обычный символ Word. Документ с неизвестным типом данных получает в дереве символ EPLAN.

Ссылка на внешний документ указывается в свойствах страницы. Если в поле вода отсутствует путь к документу, это означает, что документ находится в каталоге проекта. В этом случае текущий каталог проекта не указывается.

Имя документа можно ввести вручную или найти с помощью диалогового окна выбора файла. Если имя файла выбрано из текущего каталога проекта с помощью диалогового окна выбора файла, имя файла вводится в поле ввода без указания каталога проекта, в ином случае указывается путь к файлу.

### Управление и обработка

Управление страницами внешних документов осуществляется так же, как и для обычных страниц, для них используются и идентифицирующие свойства страницы. Все функции для обработки страниц, например сортировка, нумерация и т. д., могут использоваться и для страниц внешних документов. К примеру, внешнему документу можно назначить рамку, не содержащуюся в настройках проекта. Эта рамка не обрабатывается при обзоре и печати, но учитывается при экспорте в PDF.
Однако не все свойства обычной страницы имеет смысл указывать для этого типа страниц, например, невозможно ввести масштаб.

В содержании каждый внешний документ выводится только как одна страница независимо от того, сколько страниц имеет данный внешний документ в действительности.

Внешние документы можно открыть в навигаторе страниц, дважды щелкнув по ним; после этого запускается соответствующая программа и документ можно обработать. Однако такие документы нельзя открыть как обычные страницы с помощью перелистывания.

Во время копирования страниц между разными проектами внешние документы копируются в каталог целевого проекта, если документ находится в пути проекта.

### Печать

Печать внешних документов может осуществляться из EPLAN. При этом запускается программа, связанная с внешним документом, и ей передается команда печати. Настройки печати, например Обратный порядок печати, Число и Сортировка, не действуют для внешних документов. Кроме того, не выводятся никакие рамки, и номера страниц в проекте не увеличиваются. Внешние документы также не могут просматриваться в режиме предварительного просмотра перед печатью.

Страницы внешних документов в конце документа всегда выводятся в отдельном задании для печати. Если осуществляется печать внешних документов из нескольких проектов, они всегда печатаются по окончании всех документов.

!!! note "Замечание:"

    Документы, связанные при помощи гиперссылок, не выводятся на печать при выборе пункта меню Проект > Печать. Их можно распечатать, выбрав пункт меню Проект > Печать присоединенных документов, при условии, что файлы находятся на локальном диске или на сетевом диске, к которому у вас имеется доступ.

### Резервирование данных

Данные, находящиеся в каталоге проекта, можно включить в объем резервирования при выполнении данной операции. При создании страницы внешнего документа программа выводит вопрос, следует ли копировать документ в каталог документов проекта $(EPLAN)\Проекты\ ++...++ \<Имя проекта>\DOC.
