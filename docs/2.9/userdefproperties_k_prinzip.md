# Определенные пользователем свойства: Принцип

Определенные пользователем свойства можно создавать как для проекта, так и в базе данных изделий. Определенные пользователем свойства для проекта сохраняются как специфические для проекта и передаются вместе с ним. И, аналогично, определенные пользователем свойства для изделий хранятся в базе данных изделий.

Создание определенных пользователем свойств и управление ими осуществляется с помощью [диалогового окна конфигурации](eservicesgui_d_konfigeigenschaften.md). При этом определенные пользователем свойства, специфические для проекта, отображаются только в конфигурации для проекта. После сохранения в проекте определенных пользователем свойств для изделий посредством выбора изделия и / или устройства эти определенные пользователем свойства для изделий также будут отображаться в конфигурации для проекта.

### Особенности при автоматическом переводе

В диалоговом окне конфигурации с помощью соответствующей настройки Перевести значение свойства можно установить, должно ли учитываться значение определенного пользователем многоязычного свойства при автоматическом переводе проекта или при автоматическом переводе в базе данных изделий.

Для перевода отображаемых имен и значений по умолчанию в списках выбора доступны соответствующие настройки как в специфических для проекта настройках, так и в настройках для базы данных изделий. Для этого в диалоговых окнах [Настройки: Объем](translategui_r_einstellungumfang.md) и [Настройки: База данных изделий](translategui_r_artikelverwaltung.md) доступны настройки Определенные пользователем свойства: Отображаемое имя, Определенные пользователем свойства: Описание и Определенные пользователем свойства: Список выбора значений по умолчанию.

### Определенные пользователем свойства для базы данных изделий

Конфигурация определенных пользователем свойств в базе данных изделий выполняется с помощью пунктов меню ++Дополнительно++ > Конфигурировать свойства. С помощью вкладки Определенные пользователем свойства эти сконфигурированные свойства можно присвоить выбранному изделию.

Определенные пользователем свойства для изделий предоставляют, в частности, такие возможности:

* С полнотекстовым фильтром идентифицирующие имена и значения определенных пользователем свойств могут использоваться в качестве искомых понятий, а с фильтром по полю — как критерии фильтрации в базе данных изделий.
* Если впоследствии выбрать изделие или устройство, определенные пользователем свойства со значениями будут перенесены в соответствующую главную функцию.
* Определенные пользователем свойства для изделия учитываются при синхронизации данных изделий между базой данных изделий и проектом.
* Определенные пользователем свойства учитываются также при экспорте и импорте данных изделий с типами файлов EDZ и XML.
Если вы экспортируете всю базу данных или ее часть, то есть открываете диалоговое окно экспорта с помощью пунктов меню ++Дополнительно++ > Экспортировать, в диалоговом окне Экспортировать записи данных будет доступен тип записи данных "Конфигурация свойств". При экспорте отдельно выбранных изделий присвоенные определенные пользователем свойства также будут экспортированы, если используются упомянутые выше типы файлов.
