# Настройки: Принцип работы

Настройки в EPLAN для категорий Проекты, Пользователь, Рабочая станция и Фирма вводятся раздельно. Все настройки вводятся и изменяются независимо от категории в центральном месте программы и управляются в раздельных базах данных с расширением .xdb. Доступ к базам данных производится с помощью отдельного файла с расширением .xlk. Отдельные базы данных и файлы доступа сохраняются в общем каталоге конфигураций CFG локальной рабочей станции или при сетевой работе на дисководе сервера.

!!! note "Замечание:"

    Множество настроек можно найти в отдельном диалоговом окне "Настройки" конкретного модуля. Находящиеся там настройки имеют локальное значение, и их изменение приводит к тому, что соответствующие настройки для всего проекта перезаписываются.

### Проектные настройки

С помощью проектных настроек можно указать все свойства проекта. Проектные настройки разделяются по электротехническим функциям.
Все настройки проекта можно ввести в ***одном*** диалоговом окне, даже если настройки действуют для различных модулей.

Для проектных настроек в программе заданы стандартные значения, которые можно изменить и сохранить вместе с проектом. Программа загружает эти проектные настройки, когда вы или другой пользователь открывает проект. При этом неважно, на какой рабочей станции работает соответствующий пользователь. Определенные пользователем настройки управляются в базе данных с обозначением ПРОЕКТ.<Версия программы>.xdb. Доступ к базе данных выполняется с помощью файла ПРОЕКТ.<Версия программы>.xlk.

### Пользовательские настройки

С помощью пользовательских настроек можно определить рабочую среду пользователя. При этом можно настроить функциональность программы под ваш принцип работы или принцип работы другого пользователя.

Если пользователь регистрируется под своим именем на рабочей станции и запускает программу, она считывает имя пользователя из операционной системы. Независимо от того, на какой рабочей станции работает пользователь, программа загружает определенные пользователем настройки.

если несколько пользователей используют одинаковую рабочую станцию, программа предоставит только настройки соответствующего пользователя.

Определенные пользователем настройки управляются в базе данных с обозначением ПОЛЬЗОВАТЕЛЬ.<Версия программы>.<Имя пользователя>.xdb. Доступ к базе данных выполняется с помощью файла ПОЛЬЗОВАТЕЛЬ.<Версия программы>.<Имя пользователя>.xlk.

### Настройки станции

С помощью настроек станции можно создать рабочее место.
Информация о рабочих местах считывается программой из операционной системы. Настройки рабочей станции управляются в базе данных с обозначением РАБОЧАЯ СТАНЦИЯ.<Версия программы>.<Имя станции>.xdb. Доступ к базе данных выполняется с помощью файла РАБОЧАЯ СТАНЦИЯ.<Версия программы>.<Имя станции>.xlk.

Настройки станции сохраняются на рабочей станции. Каждый пользователь, работающий с этой станцией, может выполнять или изменять настройки.

### Настройки с учетом потребностей предприятия

С помощью этих настроек можно определить настройки, действующие во всем предприятии и для всех рабочих станций.
Настройки, относящиеся к фирме, управляются в базе данных с обозначением КОМПАНИЯ.<Версия программы>.<Название фирмы>.xdb. Доступ к базе данных выполняется с помощью файла КОМПАНИЯ.<Версия программы>.<Название фирмы>.xlk.
Информация о компании считывается программой из операционной системы. если вы создали в операционной системе имя компании, оно используется для базы данных. Если вы не указали название фирмы в операционной системе, программа использует имя COMPANY.Basic.xlk.
