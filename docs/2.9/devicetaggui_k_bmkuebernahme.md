# Перенос ОУ

Функции, не имеющие собственного видимого ОУ, могут копировать ОУ у другой функции. В этом есть следующие преимущества:

* Представление на странице становится обзорнее.
* Условные обозначения можно просто переименовывать, выполняя графические операции (Удалить, Переместить или Вставить из макроса).
* Так можно составить устройства из нескольких символов.

При переносе ОУ производится переход к следующей функции, находящейся по направлению поиска. При этом точка вставки — опознавательный признак: Для копирования из предыдущего столбца точки вставки должны находиться на той же высоте Y; для переноса из предыдущей строки точки вставки должны находиться на той же высоте X. Для этого на функции должно быть установлено свойство Перенос ОУ: Направление поиска. Направление поиска может быть "Согласно выравниванию рамки" или "Альтернативно выравниванию рамки".

Если найденная функция не содержит видимого ОУ, то отсюда производится переход к следующей функции (если у этой функции также указано направление поиска для переноса ОУ). Это происходит до тех пор, пока не будет найдено ОУ. Если ОУ невозможно найти или если функция не подходит для переноса ОУ, то вводится "пустое" ОУ и генерируется соответствующее сообщение.

Существует дополнительная настройка "Только блоков". В этом случае ОУ переносится только из функции охватывающего блока (черный ящик, блок ПЛК). Если функция находится не в блоке, перенос ОУ не осуществляется.

Копирование ОУ "Только блоков" можно задать также для отдельных страниц проекта, активировав в выбранной рамке свойство Копирование ОУ: Только блоков.

ОУ переносится также из одной "графической" и "внешней" функции.

### Направления поиска для копирования ОУ

По умолчанию направление поиска определяется по выравниванию рамки. Выравнивание рамки зависит от настройки ("Вертикально" или "Горизонтально"), установленной в свойстве Направление создания отчета рамки.

!!! example "Пример:"

    При работе с проектами по стандарту IEC свойство Направление создания отчетадля рамки выбирается, как правило, "Вертикально", при работе с проектами по стандарту NFPA - "Горизонтально".Если длясвойства Перенос ОУ: Направление поискавыбрана настройка "Согласно выравниванию рамки", то при работе с проектами по стандарту IEC ОУ копируется слева, а в проектах по стандарту NFPA — сверху.Если для свойства Перенос ОУ: Направление поискавыбрана настройка "Альтернативно выравниванию рамки", то при работе с проектами по стандарту IEC ОУ копируется сверху, а в проектах по стандарту NFPA — слева.

#### Направление поиска для стандарта ГОСТ

Свойство рамки Перенос ОУ: Направление поиска ОУ согласно ГОСТу позволяет соответствующим образом изменить направление поиска для переноса ОУ. Если это свойство активировано, то для рамок с "вертикальным" направлением генерации отчета поиск ОУ осуществляется слева направо, при "горизонтальном" направлении генерации отчета поиск ОУ осуществляется сверху вниз.

#### Таблица обзора направлений поиска

Следующая таблица показывает возможные направления поиска для копирования ОУ в зависимости от направления генерации отчета рамки. При этом предполагается, что свойство рамки Перенос ОУ: Направление поиска ОУ согласно ГОСТу для рамок по стандартам IEC и NFPA отключено, а для рамок по стандарту ГОСТ — включено. Альтернативное направление поиска применяется, когда для какой-либо функции в свойстве Перенос ОУ: Направление поиска выбрана настройка "Альтернативно выравниванию рамки".

Рамка |  Направление создания отчета |  Направление поиска |  Альтернативное направление поиска
---|---|---|---
IEC |  Вертикально |  Справа налево |  Снизу вверх
NFPA |  Горизонтально |  Снизу вверх |  Справа налево
ГОСТ |  Вертикально |  Слева направо |  Сверху вниз
ГОСТ |  Горизонтально |  Сверху вниз |  Слева направо

### Сохранение ОУ при перемещении

При перемещении условного обозначения удерживайте нажатой клавишу ++Shift++, чтобы сохранить первоначальное полное ОУ. Также сохраняются идентифицирующие обозначения устройства имеющихся на странице и затронутым этим действием условные обозначения. В результате такой операции может измениться видимое ОУ перемещенных, а также и других затронутых условных обозначений. Функции устройства при таком действии перемещения сохраняют свои связи.

Кроме того, перемещение с нажатой клавишей [Shift] влияет на свойства Обозначение штекера и Обозначение канала. При "нормальном" перемещении условного обозначения значения этого свойства остаются без изменений. При перемещении с нажатой клавишей [Shift] значения свойств Обозначение штекера и Обозначение канала подгоняются таким образом, что значения свойств Обозначение штекера (автоматически) и Обозначение канала (автоматич.) остаются равными друг другу.
