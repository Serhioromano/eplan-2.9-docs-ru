# Общая обработка на страницах отчетов

Функциональность ***Общая обработка*** позволяет обрабатывать с одной страницы отчета данные исходных объектов. Связь между отчетами и исходными объектами осуществляется через тексты-заполнители, которые размещены на страницах отчетов.

При общей обработке открывается соответствующее диалоговое окно для объекта, например диалоговое окно "Свойства" для исходного объекта "Функция" или окно управления структурными идентификаторами в случае структурных идентификаторов. После применения выполненного в окне изменения нажатием кнопки ++OK++ данные объекта обновляются на всех логических страницах и во всех представлениях. На самой странице отчета обновляется строка или, в зависимости от используемой формы, столбец, в котором находится измененный объект. Данные на других страницах отчета не изменяются, в данном случае при необходимости требуется выполнить обновление этих страниц.

На страницах отчетов с помощью указанного диалогового окна могут быть обработаны следующие виды объектов:

Объект источника |  Диалоговое окно обработки |  Страницы отчетов
---|---|---
Страница |  Свойства страницы |  Содержание
Функция |  Свойства (общие): ++...++ |  Например, список обозначений устройств, схема подключения устройства, спецификация клеммника
Соединение |  Свойства (общие): ++...++ |  Например, таблица соединений, спецификация кабеля
Изделие |  Обработать изделие |  Спецификация изделий, групповая спецификация изделий
Опция |  Опция |  Перечень опций
Объект-заполнитель |  Текст заполнитель |  Перечень объект-заполнителей
Структурный идентификатор |  Управление структурными идентификаторами |  Перечень структурных идентификаторов

Если на странице отчета выбираются разные типы исходных объектов (например, тексты-заполнители и устройства общего назначения), как и при групповой обработке, отображается вопрос, какой тип объекта следует обработать. После этого вызывается диалоговое окно обработки для выбранного типа объекта.

Если на страницах отчета ***не*** выполняется "общая" обработка, при двойном щелчке по объекту всегда открывается только диалоговое окно Свойства — текст заполнитель. Изменение текста не влияет на данные объекта, а ограничивается соответствующей страницей отчета. При дальнейшем обновлении страницы отчета это изменение будет утеряно.

### Общая обработка в списках результатов поиска

Общая обработка данных проекта также возможна с помощью ***списков результатов***. Для этого во всплывающем меню диалогового окна Результаты поиска имеется пункт меню Свойства (общие). Если функция выполняется для найденного текста-заполнителя, также открывается соответствующее диалоговое окно для объекта. Выполненные изменения снова обновляются как в исходном объекте на странице отчета, так и на логических страницах во всех представлениях.

### Прямая обработка в отчетах

Кроме того, для общей обработки данных проекта в отчетах можно использовать режим обработки ***Прямая обработка***. Если включен этот режим (Параметры > Прямая обработка), данные некоторых [свойств функций](inplaceeditinggui_k_eigenschaften.md) можно изменять непосредственно на страницах отчетов. Для этого после позиционирования курсора на соответствующем тексте заполнителе под точкой ставки выводится рамка обработки для прямого ввода. После обработки измененные данные записываются в объект и выполняется обновление текста-заполнителя.
