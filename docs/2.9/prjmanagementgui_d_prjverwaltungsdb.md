# Диалоговое окно Настройки: База данных управления проектами

Параметры > Настройки > Пользователь > Управление > База данных управления проектами.

В этом диалоговом окне задаются настройки для текущей базы данных управления проектами. Управление проектами может основываться на базе данных Access или на базе данных сервера SQL.

Обзор основных элементов диалогового окна:

### Access

Если вы выбрали данный параметр, перейдите при помощи клавиши ++…++ в диалоговое окно выбора файла и выберите там базу данных управления проектами (*.mdb). По умолчанию база данных управления проектами сохраняется в каталоге проекта (переменная пути "$(MD_PROJECTS)"). Однако вы можете выбрать и любой другой каталог.

С помощью пункта всплывающего меню Вставить переменную пути перейдите в диалоговое окно [Выбрать переменную пути](modaldialogsdb_d_pfadvariablen.md), из которого можно будет скопировать одну из доступных переменных пути.

Если это поле пусто, работайте без управления проектами.

### Сервер SQL

Выберите этот параметр и щелкните мышкой по кнопке ++...++, чтобы в диалоговом окне [Сервер SQL — Настройка](modaldialogs_d_sqlserver.md) определить настройки для использования сервера SQL.
Или выберите требуемую базу данных из раскрывающегося списка 15 последних использованных баз данных SLQ, при этом последние использованные базы находятся в верхней позиции. При открытии / генерации новой базы данных, которая отсутствует в списке, при необходимости из списка удаляется база данных, находящаяся на самой нижней позиции. С помощью пункта всплывающего меню Удалить записи можно полностью очистить список, в том числе базу данных, использующуюся в настоящий момент.

### (Создать)

Кнопка ![](images/all_new_as.png){: .ui-icon } (Создать) рядом с полями Access и Сервер SQL дает вам возможность создать базу данных управления проектами. В случае выбора параметра Access открывается диалоговое окно Новая база данных Access. В случае выбора параметра Сервер SQL открывается диалоговое окно Сгенерировать базу данных сервера SQL. После этого вновь созданная база данных заносится в настройки в качестве базы данных управления проектами и автоматически выбирается как таковая.
