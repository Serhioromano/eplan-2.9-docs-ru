# Автоматизация зданий

При автоматизации зданий согласно стандартам VDI 3814 и DIN EN ISO 16484-3 принято выводить информацию, требующуюся для планирования и исполнения систем автоматизации зданий и установок MSR, в списках функций. Для поддержки этих списков функций в EPLAN имеются специальные шаблоны проектов, формы, а также свойства для предварительного планирования.

### Списки функций

Форма для списков функций разделена на разделы и столбцы. Свойства, отображаемые в форме, рассортированы по разделам. В EPLAN предусмотренные стандартом свойства изображаются отчасти в виде фиксированных свойств EPLAN, а отчасти в виде определенных пользователем свойств. В разделе 1 представлены фиксированные свойства EPLAN для отображения входов и выходов ПЛК. В разделах со 2 и 8 содержатся определенные пользователем свойства. В шаблоне проекта для автоматизации зданий эти свойства уже предварительно определены и при использовании этого шаблона сразу же становятся доступны.

Приведенные ниже свойства EPLAN используются в разделе 1 формы для списков функций. Эти свойства получаются из вводов на вкладке ПЛК (в диалоговом окне "Свойства" объектов планирования). При этом комбинация записей в столбцах Направление, Тип данных и Счетчик задает, о каком виде входа и выхода ПЛК идет речь.

Раздел |  Столбец |  Функции ввода / вывода |  Свойство EPLAN
---|---|---|---
1 |  1 |  Бинарный вывод: Переключение / регулировка |  Суммарное число цифровых выходов ПЛК (ид. 44029)
1 |  2 |  Аналоговый вывод: Регулировка |  Суммарное число аналоговых выходов ПЛК (ид. 44031)
1 |  3 |  Бинарный ввод: Оповещение |  Суммарное число цифровых входов ПЛК (ид. 44028)
1 |  4 |  Бинарный ввод: Счет |  Общее количество числовых входов ПЛК ПЛК (ид. 44059)
1 |  5 |  Аналоговый ввод: Измерение |  Суммарное число аналоговых входов ПЛК (ид. 44030)

Для вывода списков устройств для автоматизации зданий согласно стандарту VDI 3814 используйте тип отчета "Предварительное планирование: Обзор объектов планирования" и присвойте ему форму F40_VDI3814.f40 или F40_VDI3814_A.f40.

### Списки функций в представлении, совместимом с сетевым протоколом BACnet

Списки функций для автоматизации зданий могут выводиться в представлении, совместимом с сетевым протоколом BACnet, в соответствии со стандартами VDI 3814 и DIN EN ISO 16484-5. В этом формате представления функции ТК описывают не только функции измерения и функции потребителя, но также выполняют функции ПО технологических контуров.

В представлении, совместимом с сетевым протоколом BACnet, для каждой программной функции (каждого входа и выхода ПЛК) в навигаторе предварительного планирования должна быть создана функция ТК, а функции ТК должны находиться под технологическими контурами. По умолчанию эти функции ТК имеют одну строку на вкладке ПЛК. Назначение адреса ПЛК не является обязательным.

Для вывода списков функций в представлении, совместимом с сетевым протоколом BACnet, в основных данных доступны некоторые формы (например, F40_VDI3814_1.f40, F40_VDI3814_1_en_US.f40 и т. д.).

### Графическое представление схем автоматизации

Схематический обзор (схема автоматизации) запланированной области установок можно начертить в проекте EPLAN на странице типа "Функциональная схема автоматизации". В этом обзоре можно представить количество используемых или необходимых входов/выходов на один объект планирования. Для этого используйте на размещенных в схеме автоматизации функциях порядок свойств, в котором для отображения выбраны свойства для суммарного числа входов и выходов ПЛК (например, Суммарное число цифровых выходов ПЛК (Ид. 44029)). Эта информация обычно отображается под чертежом в виде пути. В порядке свойств для свойств, которые выводят общее количество входов и выходов ПЛК, активируйте свойство отображения Ноль как знак пробела, если нулевые значения в списке общего количества отображать не нужно.

Чтобы получить такое отображение в виде пути, для соответствующих свойств необходимо изменить настройки отображения и выбрать автоматическую ординату Y. Для этого активируйте в порядке свойств свойство отображения "Координата Y автоматически (зона)". Значение для координаты Y (по отношению к нулевой точке страницы) задайте в свойстве Порядок свойств: Координата Y автоматически (зона) (Ид. 12062). Это свойство доступно на рамках и страницах.

Также возможно отображение по строкам (напр., для проектов по стандарту NFPA). Для этого активируйте в порядке свойств свойство отображения Координата X автоматически (зона). Значение для координаты X (по отношению к нулевой точке страницы) задайте в свойстве Порядок свойств: Координата X автоматически (зона) (Ид. 12063). Это свойство доступно на рамках и страницах.
