# База данных изделий: Модули

Модули обладают теми же свойствами, что и изделия, однако дополнительно содержат списки подизделий (т. н. позиций модулей). Эти подизделия идентифицируются с помощью нижестоящего ОУ или "идентификатора ОУ". Модуль тем самым является коллекцией изделий, которые относятся к нескольким общим вложенным устройствам (например, управление двигателя, которое встроено в 19"-стойку). Модули на платформе EPLAN предоставляют одну из опций, которая позволяет отображать сложные устройства.

Модуль может содержать изделия, узлы и другие модули. изделия, которые содержатся в модулях, имеют, кроме того, также информацию о вариантах изделия. Именно при выборе устройства или для отчетов в этом случае учитываются и технические данные вариантов.

### Способ действий

* ***Подготовить макросы для модулей:***
Приступая к определению в базе данных изделий специфических для изделия данных модуля, сначала необходимо создать макрос для использования этого модуля.
Рекомендуется подготовить макросы для модулей в проекте макросов. При использовании идентификатора ОУ эти идентификаторы должны быть уникальными внутри макроса / модуля. Идентификатор ОУ указывается на устройствах с помощью свойства Идентификатор ОУ. Учтите, что это свойство необходимо определять как на главных, так и на вспомогательных функциях, чтобы при выборе устройств также учитывались модули.
Элементы макроса обязательно должны находиться внутри черного ящика. Этот черный ящик становится составной частью макроса. Все относящиеся к модулю устройства должны вкладываться в макросе внутрь черного ящика. При создании макроса присвоение изделия не является обязательным, так как это выполняется автоматически при размещении изделия модуля в графическом редакторе.
* ***Создать модули в базе данных изделий:***
Модули создаются под уровнем иерархии "Главная группа продуктов" (по аналогии с отдельными деталями и узлами). На вкладке Модуль модулю присваиваются позиции, т. е. изделия, которые должны содержаться в нем. При этом необходимо присвоить каждому модулю соответствующий макрос (на вкладке Технические данные в поле Макрос).
Для идентификации устройства, находящегося в макросе, используется либо нижестоящее ОУ устройства, либо указанный на устройстве идентификатор ОУ. Благодаря использованию идентификатора ОУ используемые макросы и модули становятся независимыми от стандарта маркировки устройств и поэтому могут использоваться для различных стандартов.
* ***Применить модули в проектировании:***
Модули можно применять так же, как и устройства, и, например, вставлять их через пункты меню Вставить > Устройство.

### Механические модули

Механический модуль на трехмерном чертеже монтажных поверхностей представляет собой стандартный функциональный элемент, который один производитель получает от другого производителя как единое целое (например, цоколь электрошкафа, изготовленный из различных элементов, таких как уголок для цоколя, заглушка для цоколя, кабельная шина и т. д.). В базе данных изделий такой модуль создается под главной группой продуктов "Механика". В отличие от других модулей, механические модули могут содержать только изделия и не могут содержать других узлов или модулей.

Порядок действий с механическими модулями приблизительно соответствует порядку действий с другими модулями. Для размещения механического модуля на трехмерном чертеже монтажных поверхностей нужно создать 3D-макрос, который является стандартным функциональным элементом и в котором уже содержатся все элементы модуля. Для последующей нумерации ОУ элементы в 3D-макросе должны иметь идентификатор ОУ или подчиненное ОУ. Готовый 3D-макрос сохраняется на изделии модуля в базе данных изделий (на вкладке Монтажные данные в поле Графический макрос).

Механические модули можно разместить только как единое целое. В связи с этим на вкладке Модуль настройка Распределенное размещение модуля предустановлена на параметр "Разместить модульные изделия и элементы", и эту настройку нельзя изменить.

При размещении механических модулей в пространстве листа длина функциональных элементов с переменной длиной определяется из сохраненного макроса и вводится в данные ссылки изделия в свойство Подмножество / длина (ид. 31008). При размещении для механического модуля отображаются точки захвата, чтобы модуль можно было присоединить многократно.
