# Диалоговое окно Настройки: Совместимость

Вы открыли проект. Параметры > Настройки > Проекты > "Имя проекта" > Управление > Совместимость.

В этом диалоговом окне выполняются настройки, доступные исключительно по причинам совместимости с предыдущими версиями EPLAN. Рекомендуем для новых проектов ***не*** использовать эти настройки, так как они скоро перестанут поддерживаться EPLAN.

Обзор основных элементов диалогового окна:

### Копировать обозначение вывода устройства

Если этот флажок установлен, то в диалоговом окне "Свойства функции" (Вкладка <Категория функции>, поле Обозначение вывода устройства) дополнительно отображаются обозначения выводов устройства функций, которые копируют ОУ с текущей функции.

Если этот флажок снят, то в диалоговом окне "Свойства" будут отображаться только обозначения выводов устройства текущей функции. Обозначения выводов устройства не будут переноситься на другие функции.

### Отобразить увеличенные тексты

Если этот флажок установлен, тексты увеличиваются, как в предыдущих версиях EPLAN прибл. на 20 %. Рекомендуется использовать этот флажок ***только*** в том случае, если необходимо сохранить формат проектов, созданных с помощью предыдущих версий EPLAN. Учтите, что при этом длина текстов и межстрочные интервалы будут незначительно отличаться от "предыдущих версий".

Если этот флажок не установлен, отображение и распечатка текстов выполняются соответственно масштабу.

### Разрешить нестандартную настройку структуры 'Вышестоящий'

Если этот флажок установлен,то для определенных устройств (клеммников, штекеров, кабелей или точек разрыва) во вкладке Структура свойств проекта отображается настройка Вышестоящий. Эта настройка позволяет обрабатывать соответствующие устройства (без символа "−" в ОУ) как устройства общей нумерации. Для структуры проекта с заданными блоками идентификаторов (например, установка, место установки) эта настройка не соответствует стандартам и входит в программу только ради статистики.

Если флажок снят, то в свойствах этого проекта настройка Вышестоящ. отображаться не будет.

!!! note "Замечание:"

    * Начиная с версии 2.6, в новых проектах эта настройка больше ***не*** активируется.
      * Если настройка Вышестоящ. уже используется в старом проекте (до версии 2.6), то флажок Разрешить нестандартную настройку структуры 'Вышестоящий' установлен и в то же время заблокирован. В таком случае флажок можно снять только после того, как настройка Вышестоящ. больше не будет использоваться в проекте.

### Использовать вкладку 'Соединения' для клемм и контактов штекера

Если этот флажок установлен, в диалоговом окне Свойства клемм и контактов штекера отображается вкладка [Соединения](devicetaggui_r_verbindungenklemmen.md) и используются заданные в ней настройки. С помощью настроек в этой вкладке можно анализировать цели выводов клемм и контактов штекера отдельно от графики. При этом настройки в логической схеме выводов устройства клеммы / контакта штекера не меняются.

Использование вкладки Соединения также влияет на диалоговое окно Обработать клеммник. В виде с ориентацией на цель через пункт всплывающего меню Вывод устройства: Поменять внутр. / внешн. вы меняете не подключения, а цели клемм.

Если флажок снят, вкладка Соединения для клемм и контактов штекера становится скрытой и не используется. Клеммы / контакты штекера всегда анализируются в соответствии с настройками в логической схеме выводов устройства. Таким образом, отчеты всегда соответствуют проектированию на схеме соединений. Можно продолжить проектирование всех вариантов разводки при помощи оставшихся настроек и однозначной схемы соединений.

### Разрешить определять клеммы как главную функцию

Если этот флажок установлен, клеммам можно присвоить свойство Главная функция. Для этого в диалоговом окне Свойства клемм во вкладке Данные символа / функции доступен для установки флажок Главная функция. Это соответствует поведению в версиях EPLAN до версии 2.6.

Если флажок снят, клеммы не могут быть главными функциями. Согласно этому в диалоговом окне Свойства клемм во вкладке Данные символа / функции всегда снят и недоступен флажок Главная функция.

!!! note "Замечание:"

    Мы рекомендуем использовать для клемм только свойство Главная клемма, а свойство Главная функция ***не*** использовать.

### Отчеты: Заменить идентичные функциональные тексты

В страницах отчета может быть, что выводятся функции, у которых функциональный текст идентичен и поэтому отображается несколько раз. Если флажок установлен, то повторяющиеся функциональные тексты, стоящие друг под другом, заменяются замещающим символом. Замещающий символ задается с помощью настройки Заменить идентичные тексты на (в меню Настройки > Проекты > "Имя проекта" > Отчеты > Отображение / вывод).

!!! example "Пример:"

    Задайте в качестве замещающего символа символ повторения".Функциональный текст Позиция Двигатель=AP+0/1.2"=AP+ST1/2.15Генератор=AP+0/1.1"=AP+ST1/2.12

### Отображать имя интерфейса шины в целевой информации

Начиная с версии 2.8 для идентификации выводов шины используется комбинация имени интерфейса шины и обозначения штекера.

Если этот флажок установлен, везде, где отображается целевая информация, в дополнение к обозначению штекера отображается имя интерфейса шины выводов шины. Это, в частности, относится к отображению источника и цели соединений, представлению структуры дерева навигатора соединений, перекрестным ссылкам, отчетам, отображению целей в диалоговых окнах Обработать клеммник и Обработать штекер и т. д.

Если флажок снят, при отображении целевой информации будет показано только обозначение штекера. Это соответствует поведению в версиях EPLAN до версии 2.8.

### Обрабатывать распределенные клеммы при экспорте / импорте данных проводного монтажа раздельно

Если этот флажок установлен, при экспорте данных проводного монтажа распределенные клеммы экспортируются как отдельные строки. Это соответствует поведению в версиях EPLAN до версии 2.8.
Соответственно при импорте необходимо установить флажок, если в файле импорта содержатся отдельные строки для распределенных клемм.

Если этот флажок снят, при экспорте данных проводного монтажа распределенные клеммы объединяются. Это значит, что для каждой клеммы в файле экспорта имеется только одна строка.
Соответственно при импорте необходимо установить флажок, если в файле импорта для каждой клеммы содержится одна строка с объединенными данными распределенных клемм.
