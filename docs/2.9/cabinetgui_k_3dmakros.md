# 3D-макротехника: принцип

Макротехника в трехмерном чертеже монтажных поверхностей ориентируется в основном на принятый в EPLAN способ работы для организации, создания и использования макросов.

* Управление 3D-макросами осуществляется в проекте макросов.
* 3D-макросы сохранены и подготовлены в проекте макросов. Для наглядного отображения 3D-макросов и управления ими теперь можно использовать расширенные функции навигатора макросов. Они позволяют структурировать содержащиеся в 3D-проекте макросов 3D-макросы, исключая необходимость предварительного создания вспомогательной структуры или разделения макросов.
* Имеется возможность создать и использовать 3D-макрос окна (*.ema) и 3D-макрос символа (*.ems).
* 3D-макросы присваиваются в базе данных изделий одному из изделий. Представление изделия за счет этого будет детализировано и приближено к реальности. Содержимое графического макроса определяет графику. Если у изделия нет графического макроса, то графика будет представлена в виде параллелепипеда с указанными высотой, шириной и глубиной.

### Рекомендации по работе с 2D- и 3D-макросами

Для оптимальной эффективности вашей работы с платформой EPLAN двух- и трехмерные данные следует сохранять не вместе в одном файле макроса, а в отдельных файлах макроса. Тем самым вы избавляете пользователей, работающих с двухмерными данными, от необходимости каждый раз вести поиск среди множества трехмерных данных.

Мы рекомендуем сохранить файл макроса с трехмерными данными и видом представления 'Трехмерный чертеж монтажных поверхностей' в базе данных изделий на вкладке Монтажные данные в поле Графический макрос. Файл макроса с двухмерными данными и другими видами представления ("Многополюсные", "Обзор" и т. д.) следует, как и прежде, записывать на вкладке Технические данные базы данных изделий в поле Макрос. Различать файлы можно, например, по расширению имени (например, трехмерный макрос ABC.12345_3D.ema и соответствующий двухмерный макрос ABC.12345_2D.ema).

При всех действиях в трехмерном окружении, при которых используется макрос изделия (например, вставка устройства), сначала проверяется 'графический макрос изделия'. Если для изделия на вкладке Монтажные данные записан графический макрос и в нем содержится требуемый вид представления "Трехмерный чертеж монтажных поверхностей", то используется этот макрос. Если этого нет, то используется "технический макрос изделия", указанный на вкладке Технические данные. В трехмерном окружении на вкладке Монтажные данные в поле Графический макрос заданный чертеж контура используется с приоритетом перед заданным на вкладке Технические данные.

### 3D-макросы в пространстве листа

Для создания 3D-макросов в проекте макросов рамки макросов не используются. 3D-макросы всегда имеются в пространстве листа. Проект макросов имеет отдельное пространство листа для каждого макроса. При автоматическом генерировании 3D-макросов в проекте макросов для каждого пространства листа будет сгенерирован один макрос. Обзор модели 3D-макроса можно разместить на одной странице 2D. Из этого обзора всплывающего меню можно в случае необходимости переместиться в соответствующее пространство листа.

Для 3D-макроса имеются следующие свойства пространства листа:

* Макрос: имя
* Макрос: Описание
* Макрос: Вариант
* Макрос: версия
* Макрос: источник / ссылка

Эти свойства задаются вручную при сохранении макроса или автоматически при автоматическом создании.

!!! tip "Совет:"

    Для присвоения одного подготовленного пространства листа нескольким макросам используйте [вкладку Определение макроса](macrosgui_r_makrodefinition.md) диалогового окна Свойства. Внесите в строки этой вкладки соответствующие имя и описание генерируемых макросов. Затем при автоматическом генерировании в нескольких макросах сохраняется одно и то же содержимое (одно и то же пространство листа).

### Расположение и точка захвата 3D-макроса

Все 3D-макросы сохраняются в одном и том же положении (выравнивание по X/Y). Имеется возможность изменить впоследствии расположение функциональных элементов при помощи функций обработки. Для определения трехмерных точек захвата можно использовать функцию Обработать > Логика устройства > Точка захвата.

### Группирование 3D-макросов

3D-макрос (т. е. пространство листа в проекте макросов) может содержать несколько отдельных элементов, потому что несколько элементов было в импортируемом исходном файле. Хотя геометрическое суммирование элементов невозможно, функция Обработать > Графика > Объединить дает возможность так сгруппировать все компоненты пространства листа, что трехмерный макрос при использовании рассматривается (при отображении в навигаторе или при выборе) как ***один*** функциональный элемент.

!!! note "Замечание:"

    Объединение трехмерных тел не подходит для уменьшения объема данных в генерируемом макросе. Если импортированный файл STEP содержит несколько одинаковых или очень подробных частей, это может привести к тому, что объем данных после объединения существенно возрастет. Это имеет непосредственное влияние на размер файлов макросов, поскольку особенности системы не допускают их сжатия.

    Для достижения приемлемого размера файла макроса в таком случае следует отказаться от объединения импортированных одинаковых частей и генерировать макрос с импортированными отдельными частями.

### Объекты-заполнители в 3D-макросах

В 3D-макросы можно вставлять объекты-заполнители. Способ определения и характеристики аналогичны объектам-заполнителям в двухмерном макросе, однако имеются следующие отличия:

* Вместо свойств страниц можно выбрать свойства пространства листа, указав в них переменные.
* Объект-заполнитель отображается не на странице проекта в качестве якорного символа в графике, а в качестве [якорного ярлыка](cabinetgui_d_navigator.md) в структуре дерева навигатора пространства листа.
