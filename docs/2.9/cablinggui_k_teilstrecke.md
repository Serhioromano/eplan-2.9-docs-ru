# Соблюдение разности высотных отметок при маршрутизации

При планировании сетей соединенных сегментов в этих сетях может содержаться разность высотных отметок, так как, например, они распространяются по нескольким этажам здания. Чтобы соединить между собой сети соединенных сегментов, которые относятся к разным этажам здания, потребуются вертикальные сегменты маршрутизации. Эти вертикальные сегменты маршрутизации реализуются в EPLAN с помощью так называемых "Распределенных сегментов маршрутизации". При этом речь идет о сегментах маршрутизации, представленных в распределенном виде, которые соединяют между собой по две точки маршрутизации на разных страницах топологии.

### Распределенные сегменты маршрутизации

Распределенные сегменты маршрутизации — это сегменты маршрутизации с определением функции "Распределенный сегмент маршрутизации топологии". Они предназначены для представления сегмента маршрутизации в распределенном виде и соединяют по две точки маршрутизации между собой. С помощью распределенных сегментов маршрутизации можно соединить сети соединенных сегментов, которые находятся на разных высотных отметках (например, на разных этажах здания). Кроме того, распределенные сегменты маршрутизации можно использовать просто для того, чтобы провести сеть соединенных сегментов на другую страницу топологии (без разности высотных отметок).

С одним и тем же ОУ всегда должны быть только два распределенных сегмента маршрутизации, один из которых выполняет главную функцию, а другой — вспомогательную. Данные обоих распределенных сегментов маршрутизации синхронизируются между собой.

На распределенных сегментах маршрутизации можно просматривать разность высотных отметок, которая указывает вертикальное расстояние между сетями соединенных сегментов. Значение определяется из длины трассы сегмента маршрутизации. В связи с этим для распределенных сегментов маршрутизации доступны такие свойства:

* Топология: Разность высотных отметок (Ид. 20347): в этом свойстве отображается длина трассы вертикального сегмента маршрутизации с предшествующим знаком. Предшествующий знак "+" означает "Вверх", "-" означает "Вниз". Свойство заполняется только если для сегмента маршрутизации указано направление для разности высотных отметок.
* Топология: Направление для разности высотных отметок (Ид. 20348): задает направление для разности высотных отметок. Возможные значения: "Нет", "Вверх" и "Вниз".

### Способ действий

Чтобы соединить сети соединенных сегментов разных уровней по вертикальному сегменту маршрутизации, проект должен содержать не менее двух сетей соединенных сегментов с точками и сегментами маршрутизации. Сети соединенных сегментов обычно находятся на разных страницах топологии, но они также могут быть начерчены на одной и той же странице топологии. Для генерирования маршрутизируемого соединения необходимо выполнить такие рабочие шаги:

* ***Вставка распределенного сегмента маршрутизации в сеть соединенных сегментов нижнего уровня:*** с помощью окна выбора символов вставьте распределенный сегмент маршрутизации в точке маршрутизации, из которой должен исходить вертикальный сегмент маршрутизации. Для этого в библиотеке символов SPECIAL существуют символы CABDSU и CABDSD. В диалоговом окне Свойства данного распределенного сегмента маршрутизации введите ОУ и длину трассы, а затем выберите для свойства Топология: Направление для разности высотных отметок пункт "Вверх". Установите флажок Главная функция.
* ***Вставка распределенного сегмента маршрутизации в сеть соединенных сегментов верхнего уровня:*** с помощью окна выбора символов вставьте распределенный сегмент маршрутизации в точке маршрутизации, с которой соединен вертикальный сегмент маршрутизации. В диалоговом окне Свойства данного распределенного сегмента маршрутизации укажите то же ОУ, которое назначено для соотносящегося распределенного сегмента маршрутизации, а затем выберите для свойства Топология: Направление для разности высотных отметок пункт "Вниз". Оставьте поле Длина трассы пустым и не устанавливайте флажок Главная функция.
* ***Отображение разности высотных отметок:*** в диалоговом окне Свойства распределенных сегментов маршрутизации выберите на вкладке Отображение для обоих распределенных сегментов маршрутизации свойство Топология: Разность высотных отметок для отображения (если отображение еще не установлено в используемом символе).
* ***Маршрутизация соединений:*** при маршрутизации соединений через пункты меню Данные проекта > Топология > Маршрутизировать выполняется маршрутизация соединений между уровнями по вертикальному сегменту маршрутизации. Сегмент маршрутизации приводится в трассе маршрутизации (только один раз), а длина трассы сегмента маршрутизации учитывается при определении общей длины.

!!! tip "Совет:"

    Можно быстро вставлять распределенный сегмент маршрутизации путем выбора пунктов меню Вставить > Топология > Сегмент маршрутизации и последующего нажатия клавиши ++Backspace++. В открывшемся диалоговом окне Выбор символа символы уже отфильтрованы: отображаются только символы с категорией функции "Трасса маршрутизации топологии". Под категорией этой функции можно выбрать символ для распределенного сегмента маршрутизации топологии.
