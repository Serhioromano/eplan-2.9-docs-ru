# Диалоговое окно Настройки: Общие (Пользователь, Представление)

Параметры > Настройки > Пользователь > Отображение > Общ.

В этом диалоговом окне устанавливаются общие настройки пользователя для представления.

Обзор основных элементов диалогового окна:

### Использовать только один буфер обмена

Если этот флажок установлен, для вырезанных и скопированных объектов, а также для скопированных страниц может использоваться ***только один*** буфер обмена.

Если этот флажок снят, для вырезанных и скопированных объектов, а также для скопированных страниц может использоваться ***несколько*** буферов обмена. Если в этом случае копируются два и более объектов (или страниц), при вставке EPLaN открывает диалоговое окно Вставить. В этом диалоговом окне можно выбрать, какой из скопированных объектов (страниц) необходимо вставить.

### Не отображать диалоговое окно описания

Если этот флажок установлен, вырезанный или скопированный объект (или скопированная страница) сохраняется в буфер обмена без описания.

Если этот флажок снят, при копировании объекта или страницы открывается диалоговое окно Буфер обмена. В нем можно ввести описание объекта. Это описание выводится в диалоговом окне Вставить в качестве помощи во время выбора.

### Обновить отчеты при открытии страниц

Если этот флажок установлен, существующие страницы отчетов автоматически обновляются при открытии.

Если этот флажок снят, автоматическое обновление существующих страниц отчетов при открытии не выполняется.

### Обновлять отчеты при печати и экспорте

Если этот флажок установлен, существующие страницы отчетов автоматически обновляются при печати и экспорте. Если отчеты содержат ***шаблон***, то они также генерируются.

Если этот флажок не установлен, существующие страницы отчетов не обновляются автоматически при печати и экспорте.

### Обновить соединения при смене страницы

Если флажок установлен, то при смене страницы соединения (открытые. закрытые, пролистываемые и т. д.) автоматически обновляются. Соединения обновляются как на открытых страницах, так и на тех страницах, где соединения проводятся через точки разрыва. Дополнительно соединения в пределах страницы (также те, которые не заканчиваются точками разрыва) обновляются после каждого шага обработки. Обновление для всего проекта не выполняется.

Если флажок снят, то автоматическое обновление не будет выполняться во время обработки в графическом редакторе и при смене страницы.

### Обновлять соединения при маршрутизации

Если этот флажок установлен, соединения автоматически обновляются в пространстве листа или в топологии перед генерированием маршрутизируемых соединений.

Если этот флажок снят, соединения не будут автоматически обновляться при маршрутизации.

### Обновить соединения для всего проекта (вручную)

Эта настройка влияет только на ***ручное обновление соединений***.
Если этот флажок установлен, то соединения обновляются таким образом, как будто выбран весь проект.

Если флажок не установлен, то обновление выполняется только для текущего выбора (страниц, функций, ...).

### Всегда обновлять перекрестные ссылки

Если этот флажок установлен, вид в графическом редакторе создается заново при каждом шаге обработки. Благодаря этому перекр. ссылки точек разрыва на странице всегда обновляются автоматически.

Если флажок снят, автоматическое обновление не производится. В этом случае можно обновить перекрестные ссылки точек разрыва с помощью команды Вид > Начертить заново или клавиши ++F5++.

### Отображать полосы прокрутки

Если этот флажок установлен, в редакторах (***Графический редактор***, ***Редактор символов***, ***Редактор форм*** и т.д.) справа и / или внизу отображаются полосы прокрутки, при помощи которых можно перемещать видимый фрагмент.

Если этот флажок снят, полосы прокрутки в редакторах не отображаются.

### Синхронизировать навигаторы при открывании

В графическом редакторе, в области управления сообщениями или в диалоговом окне Результаты поиска выделенные объекты могут автоматически синхронизироваться при открытии навигаторов данных проекта, для этого пользователю не нужно выбирать пункт всплывающего меню Синхронизировать выбор.

Если этот флажок установлен и на открытой странице выделен один или несколько объектов, то при открывании навигаторов данных проекта выбранные объекты синхронизируются. Для быстрого открытия навигатора нажмите нужную кнопку в панели инструментов Навигаторы. Выделенные объекты остаются выделенными и в открывшемся навигаторе, если этот навигатор содержит эти объекты.

### Сохранить данные и маркеры ревизии при вставке

Если установлен этот флажок, то при вставке скопированных страниц или макросов страницы данные ревизии будут перенесены из исходного проекта в целевой. Кроме того, при копировании и вставке фрагментов схемы соединений, страниц или пространств листа, а также при вставке макросов вместе с ними копируются маркеры ревизии и вставляются в целевой проект.

Если флажок снят, ***ни*** данные, ***ни*** маркеры ревизии не будут вставляться в целевой проект.

!!! note "Замечание:"

    * ***Данные ревизии*** станут видимыми в целевом проекте только при наличии там ревизии с идентичным внутренним "номером ревизии" (в соответствии с порядком создания). Т. е. данные второй ревизии исходного проекта сортируются во вторую ревизию целевого проекта, независимо от имени и индексов ревизии в обоих проектах. Данные ревизии также вставляются в еще не прошедшие ревизию проекты, но становятся видимыми только после генерации соответствующей ревизии.
    * ***Маркеры ревизии*** будут видимыми в целевом проекте, только если активирована опция Вид > Маркеры ревизий.
