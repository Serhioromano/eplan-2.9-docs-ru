# Резервирование данных: Принцип

Для функций Резервировать и Восстановить соответствующий проект должен быть заблокирован. Поэтому резервирование в многопользовательском режиме невозможно; все пользователи должны закрыть проект. Программа закрывает все открытые окна, например графический редактор или навигаторы резервируемого проекта; окно управления проектами и навигатор страниц можно оставить открытыми. Основные данные тоже блокируются и могут резервироваться только в том случае, если они в данный момент не обрабатываются.

Одновременно можно резервировать несколько проектов. Только при отправке проектов по электронной почте за один раз можно отправить только один проект.

EPLAN поддерживает имена, соответствующие соглашению об универсальном назначении имен, по всем проектам. При этом речь идет об имени, которое соответствует соглашению об универсальном соглашении имен и используется для указания пути к данным: имена серверов и каталоги включаются в путь к данным, "псевдонимы" (сокращения команд) не используются. Путь к данным выглядит при этом следующим образом: \\\Имя сервера\Имя общей папки\Каталог\Подкаталог\Имя файла.abc

Зарезервированные данные, как правило, архивируются (при помощи программы 7-Zip). При упаковывании все данные проекта сохраняются. (Напротив, при сжатии проекта есть возможность убрать из проекта неиспользуемые данные.) Для файлов применяются следующие расширения имен файлов:

* Проекты: *.zw1
* Базовые проекты с основными данными: *.zx1
* Шаблоны проектов: *.zx2
* Библиотеки символов: *.zw2
* Рамки: *.zw3
* Формы: *.zw4
* Макросы: *.zw5
* Данные изделий: *.zw6
* Словари: *.zw7
* Данные пользователя, рабочей станции: *.zw8
* Контуры: *.zwa.

Резервированные данные можно открыть двойным щелчком по соответствующему файлу. При этом EPLAN автоматически будет запущен в режиме ***Восстановить***.

В EPLAN существует функция разделения резервированных данных на несколько небольших блоков данных, а также функция восстановления. Это особенно важно при отправке данных по электронной почте.

!!! note "Замечание:"

    * ***Запакованные*** проекты сохраняются в каталоге проектов с расширением имени файлов *.elp. Сами же запакованные файлы сохраняются с расширением имени файла *.zw0, что позволяет отличать их от ***резервированных*** проектов (*.zw1).
    * При отправке данных по электронной почте разделенные файлы резервирования будут различаться по дополнению от ".000" до максимально ".999" к имени файла. Для восстановления проекта эти файлы необходимо скопировать в общий каталог. В диалоговом окне Восстановить проект отображается только файл с дополнением к имени файла ".000". Поэтому для файлов резервирования не следует использовать имена, в которых после точки стоит трехзначное число.

!!! example "Пример:"

    Проект Мой_проект.elkсохраняется как Мой_проект.zw1.Библиотека символов Мои_символы.slkсохраняется как Мои_символы.zw2.При отправке данных по электронной почте для проекта Мой_проект.elkгенерируются файлы резервирования Мой_проект.000.elk,Мой_проект.001.elkи Мой_проект.002.elk. Для восстановления проекта скопируйте три файла резервирования в общий каталог и щелкните дважды по файлу Мой_проект.000.elk. В диалоговом окне Восстановить проектотображается только файл Мой_проект.000.elk, а остальные два файла резервирования невидимы.

### Архивирование данных на длительный срок

Для долгосрочного архивирования проектов используйте функцию архивирования EPLAN; при этом создается файл в формате ZIP.

Кроме этого, можно экспортировать проекты в следующие не зависящие от базы данных форматы:

* DXF / DWG
* Графический файл
* PDF.

Таким образом проанализировать заархивированные данные можно даже в том случае, если аппаратные платформы и программное обеспечение за несколько лет изменятся.

Можно экспортировать как весь проект, так и несколько отдельных страниц.

#### Повторная печать графики

Как правило, схемы соединений передаются заказчику в бумажном виде. Так как с течением времени бумага желтеет и изнашивается, существует необходимость печатать схемы повторно. Для такого вида долгосрочного архивирования можно экспортировать страницы проекта в распространенный графический формат, напр., TIFF, JPEG, BMP, DXF.

### Обмен данными через дискету / компакт-диск / электронную почту / сеть

Путем резервирования данных можно обмениваться данными с субподрядчиками, заказчиками или другими обработчиками. Все относящиеся к проекту данные (библиотеки символов, рамки, макросы и т. д.) сохраняются в выгруженном файле.

Можно также отправить проект по электронной почте.

При восстановлении проектов и данных проекта проверяется, существуют ли такие проекты / данные и можно ли их перезаписывать.
При восстановлении проектов / данных проекта в качестве целевого можно указать любой диск и каталог.

### Внешние документы и связанные данные

По желанию при резервировании и восстановлении данных можно учитывать внешние документы. При этом резервируются только документы, сохраненные в каталоге проекта (в подкаталоге \Документы). Документы из других каталогов в проекте не резервируются. При добавлении внешних документов определяется, следует ли копировать внешние документы в каталог проекта или же оставить в первоначальном месте хранения с добавлением ссылки на файл. Так посредством места хранения определяется, будет ли файл резервироваться при осуществлении резервирования данных. Аналогичная система применяется и для графических файлов и связей через гиперссылки. При этом графические файлы сохраняются в отдельном подкаталоге \Рисунки.

При восстановлении файлы вставляются в те каталоги, в которых они хранились на момент резервирования данных. Перед перезаписыванием данных на экран выводится предупреждение с указанием, какие данные являются более актуальными.

### Преобразование проектов при помощи символов не в коде ASCII в имена основных данных

Для проектов из прежних версий EPLAN, при которых в именах сохраненных основных данных (форм, рамок, изделий) встречаются "символы не в коде ASCII", существует функция корректировки. С помощью преобразования можно устранить проблемы, возникшие при передаче проектов между различными языковыми версиями операционной системы.

При восстановлении или открытии соответствующих проектов открывается запрос Обновить индексы поиска. Если щелкнуть [Да], устанавливаются индексы поиска, необходимые для доступа к основным данным, и соответствующие имена файлов правильно записываются в базу данных управления проектами. Затем открывается проект. Если щелкнуть [Нет], проект закроется.

!!! note "Замечание:"

    Проект, преобразованный таким образом, не следует редактировать в прежних версиях EPLAN. По причине преобразования имен основных данных в символы Unicode, основные данные больше не обнаруживаются в прежних версиях EPLAN.
