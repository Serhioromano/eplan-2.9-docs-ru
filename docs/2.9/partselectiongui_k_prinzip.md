# Поведение изделий при выборе устройств

Для выбора устройства большое значение имеет нижеописанное поведение изделий.

### Сохранение изделий

Изделия сохраняются в проекте в следующих местах:

* Изделия для комплектующих диталей, общих установочных материалов и т. д. можно присваивать непосредственно ***проекту***. Возможно применение до 1000 покупных изделий.
* Главной функции ***устройства*** можно присваивать до 50 номеров изделия.
* Через ***точку определения изделия*** (как у каждой главной функции) можно сохранить в одном месте схемы соединений до 50 изделий.

### Сохраненные в проекте изделия

Некоторые свойства изделия (напр., закупочная цена) необходимы для расчета в списках вывода, но не всегда они должны попадать заказчикам. Через диалоговое окно конфигурации можно установить, какие свойства сохраняются в проект. Затем эти свойства также применяются при дальнейшей передаче проекта. Несохраненные свойства изделий доступны только через обращение к базе данных изделий.

В диалоговом окне Удалить сохраненные свойства изделия можно быстро просмотреть, какие свойства изделий сохранены в проекте. Кроме того, можно удалить отдельные свойства сохраненных в проекте изделий.

В диалоговом окне Синхронизация изделий - <Имя проекта> можно выполнять синхронизацию сохраненных в проекте изделий с изделиями системы.

### Обновление данных устройства

Изделия и другие основные данные постоянно обновляются и становятся доступными, например, через EPLAN Data Portal. Это может привести к изменению данных изделий, хранящихся в проекте, в базе данных изделий. Для переноса измененных данных в проект доступны различные функции синхронизации изделий и данных устройств:

* Синхронизировать текущий проект
* Обновить текущий проект
* Дополнить текущий проект
* Обновить сохраненные изделия при открытии проекта
* Синхронизация данных после изменения изделия

Если изделия в проекте старее, чем в базе данных изделий, то новые изделия сохраняются в проекте при синхронизации данных изделий. Кроме того, более новые данные устройства переносятся на главные функции, использующие эти изделия.

При обновлении или пополнении проекта данные устройства обновляются для ***всех*** используемых изделий, а при синхронизации данных изделий обновляются только данные устройства для ***выбранного*** изделия. При этом соответствующие номера изделий и данные ссылки изделий остаются неизменными.

#### Выбор изделия и устройства — обновленное изделие

При выборе изделия и устройства для изделия, состояние которого в проекте более старое, чем в базе данных изделий, обновление данных устройства происходит на соответствующей главной функции. Однако, поскольку данные устройства не могут быть перенесены на все варианты использования изделия в проекте (например, если данные шаблона были изменены), в этом случае отображается запрос. Вы можете решить, нужно ли выбрать это обновленное изделие или другое изделие.

Чтобы в таком случае иметь возможность обновлять данные устройства на функциях в проекте, в навигаторе спецификаций доступен пункт всплывающего меню Обновить данные устройства.

### Изделия, сгенерированные из данных схемы соединений

Через команду Генерировать изделие можно сгенерировать изделия из данных схемы соединений. Это изделие сохраняется в проекте как сохраненное в проекте изделие. При этом в базе данных изделий сохраняется устройство, т. е. изделия и типовые для устройства данные сохраняются в базе данных изделий.

Это действие может выполняться только при условном обозначении главной функции. Оно доступно только в том случае, если внесена хотя бы ***одна*** ссылка изделия (с числом изделий 1) в ОУ.

Таким образом сгенерированное изделие содержит шаблоны функции для всех функций устройства.

Если команда для главной функции выполняется повторно, с помощью двухуровневого запроса можно определить, должно ли перезаписываться только уже содержащееся в проекте изделие или также должны обновляться основные данные изделия.

### Спецификация

Через сохраненное изделие можно генерировать спецификацию, которая содержит такие дополнительные данные, как "Описание", " Поставщик" и т. д. Такая спецификация может, например, затем передаваться заказчикам (с дополнительной информацией) без необходимости передачи всей базы данных изделий.

Спецификация изделий может выдаваться в различных отсортированных / сгруппированных списках вывода. При этом иерархически построенные группы изделий могут дополнительно разворачиваться или выдаваться до установленной глубины.

!!! note "Замечание:"

    Следует обратить внимание на то, что изделия с количеством = "0" выводятся всегда! Чтобы не выводить такие изделия, необходимо установить соответствующий фильтр.

### Компоновка электрошкафа

Элементы, сохраненные в схеме соединений на устройствах, определяют на монтажной плате механическую занимаемую площадь. В базе данных изделий (вкладка Монтажные данные) можно управлять присвоением данных заказов к механическим символам с определением размеров.

### Узлы

При присвоении изделий устройствам могут разворачиваться вложенные узлы. Отдельный элемент содержит при этом свою информацию, к какому узлу он относится.

В настройках пользователя Развернуть узел (Параметры > Настройки > Пользователь > Управление > Выбор изделия) указывается, разбиваются ли узлы при вставке и до какой степени. Дополнительно существует настройка проекта Развернуть узел (Параметры > Настройки > Проекты > "Имя проекта" > Отчеты > Изделие), с помощью которой указывается, разбиваются ли узлы в отчетах и каким образом. Если узлы были развернуты при вводе, их нельзя снова обрабатывать в отчете в качестве узла.

### Варианты изделий

Для поддержки многофункциональных реле в базе данных изделий могут создаваться так называемые варианты изделий.

Если к устройству добавляется новое изделие и в этом изделии находится шаблон функции, то этот шаблон функции добавляется в устройство. Если изделие удаляется из устройства, то также удаляются все шаблоны функции, которые относятся к этому изделию.
