# Диалоговое окно Сравнить свойства проектов

Вы открыли проект. Сервисные программы > Управление ревизиями > Сравнение свойств проектов > Сравнить проекты.

В этом новом диалоговом окне установите настройки для сравнения свойств проектов. При этом сравните определенные свойства текущего проекта с соответствующими свойствами исходного проекта. Обычно исходный проект является копией текущего проекта на более раннем этапе; тем не менее, можно также выбрать любой проект в качестве исходного проекта.

Обзор основных элементов диалогового окна:

### Выбранный проект

Здесь отображается текущий выделенный проект.

### Отобразить только соответствующие исходные проекты

если установлен флажок, в списке отображаются только исходные проекты, относящиеся к текущему проекту; в противном случае в списке отображаются все проекты, содержащиеся в каталоге проектов (за исключением выделенного).

!!! note "Замечание:"

    Если Вы отмените защиту от записи соответствующего исходного проекта, то он перестанет отображаться как "соответствующий исходный проект", а будет отображаться в списке всех проектов.

### Сравнить с

В списке отображаются проекты/исходные проекты, содержащиеся в каталоге проектов. Выберите, с каким проектом вы хотите сравнить текущий открытый проект.

### Настройки для сравнения свойств

Это поле показывает текущую схему для сравнения свойств проектов. С помощью кнопки ++...++ перейдите в диалоговое окно [Настройки: Сравнение свойств проектов](revisionmgtgui_d_vergleichseinstellungen.md). Там укажите, какие свойства следует сравнить между собой, и сохраните эти настройки в собственной схеме.

Выбранная схема активируется автоматически.
Если выбрана запись "-Не активировано-", используется стандартная схема. В этом случае сравниваются все свойства всех объектов. Это может занять больше времени при обработке больших проектов.

### Настройка структурного идентификатора

Это поле показывает текущую схему для замены структурного идентификатора. Если структурный идентификатор переименован, то при сравнении свойств проектов он обычно распознается как удаленный или вновь добавленный. Если отображать такие изменения нет необходимости, можно создать схему, которая задает, какие структурные идентификаторы были переименованы. При сравнении свойств проектов указанные структурные идентификаторы больше не считаются различными. Эта настройка имеет смысл только тогда, когда с помощью флажка Альтернативная идентификация активирована альтернативная идентификация объектов.

С помощью кнопки ++...++ перейдите в диалоговое окно [Настройки структурного идентификатора](revisionmgtcomparegui_d_einstellungenstrukturkennzeichen.md). Установите, какой структурный идентификатор в исходном проекте для сравнения заменен на структурный идентификатор в текущем проекте, и сохраните эти настройки в собственной схеме.

Выбранная схема активируется автоматически.
Если выбрана запись "-Не активировано-", ни одна из схем не будет использоваться. В этом случае переименованный структурный идентификатор распознается как удаленный и заново добавленный.

### Текст маркера ревизии

Введите здесь текст, который должен отображаться в качестве маркера ревизии.

### Сгенерировать маркер удаления для удаленных объектов

Если этот флажок установлен, то удаленные объекты (условные обозначения, тексты и т. д.) обозначаются на страницах проекта посредством маркера ревизии и маркера удаления. Свойства удаленных объектов сохраняются на маркерах удаления, где их можно просматривать.

Если флажок не установлен, маркеры удаления не отображаются.

### Для удаленных объектов записать маркер ревизии обратно в исходный проект

Если этот флажок установлен, для объектов, более не существующих в текущем проекте, в исходный проект вводится маркер ревизии. В исходном проекте объекты обозначаются как удаленные.

!!! note "Замечание:"

    Маркеры ревизии можно вводить только в не защищенный от записи исходный проект. В защищенном от записи исходном проекте отображается сообщение. До сравнения свойств необходимо удалить защиту от записи исходного проекта. Обратите внимание на то, что временные исходные проекты также имеют защиту от записи.

### Удалить все существующие маркеры ревизий

если установлен этот флажок, маркеры ревизии предыдущего сравнения свойств удаляются из проекта.

если флажок не установлен, сохраняются все маркеры ревизии, содержащиеся в проекте. Новые маркеры ревизии дополнительно вносятся в проект.

### Использовать активные фильтры из навигатора страниц

Если этот флажок установлен, то при сравнении свойств учитывается установленный в навигаторе страниц фильтр. Это означает, что отображенные страницы сравниваются только в навигаторе страниц. Скрытые страницы, как и неразмещенные объекты, не сравниваются.

Если этот флажок не установлен, сравниваются все страницы проекта, даже не отображенные в навигаторе страниц.

### Альтернативная идентификация

Если этот флажок установлен, объекты идентифицируются с помощью так называемого "возможного ключа". Возможный ключ – это минимальное количество свойств для однозначной идентификации объекта. Идентифицирующие свойства зафиксированы в EPLAN.

Если флажок не установлен, идентификация объекта выполняется с помощью внутреннего ид. объекта.

!!! note "Замечание:"

    При сравнении свойств автоматически сгенерированных проектов (например, с помощью EPLAN Engineering Configuration) этот флажок необходимо установить, чтобы при каждом генерировании внутренние ид. объектов задавались заново. Это означает, что при каждом генерировании возникает полностью новый проект, и все объекты в этом проекте получают произвольные новые ид. объектов. При этом сравнение свойств посредством идентификации объектов на основе ид. объектов невозможно.

### Собрать результаты сравнения

Если этот флажок установлен, список результатов ревизии не перезаписывается при каждом сравнении свойств проектов, а результаты заносятся в имеющийся список.

Если флажок снят, список результатов ревизии перезаписывается при каждом новом сравнении свойств проектов. Это настройка по умолчанию.

!!! note "Замечание:"

    Данный флажок снимается при каждом открытии диалогового окна и активируется по желанию пользователя перед запуском сравнения проектов.
