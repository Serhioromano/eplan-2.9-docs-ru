# База данных изделий: Основные положения

Для возможности активной работы с базой данных изделий важны нижеописанные основы:

### Принцип работы программы

Аналогично к другим исходным данным также для изделий в проекте сохраняется вся информация, необходимая для независимого использования проекта. Таким образом, всегда имеются ***две базы данных изделий*** (резервное хранение данных): центральная база данных изделий для всех проектов и база данных изделий, сохраненная в проект, которая содержит только установленные в проекте изделия.

В составе проекта обрабатываются изделия из базы данных изделий проекта, т. е., изделие, использованное 10 раз, сохраняется только один раз и обрабатывается в проекте 10 раз (через номер изделия). Измененные данные изделия могут также просто вводиться или синхронизироваться с центральной базой данных изделий.

### Структура окна базы данных изделий

Главное окно базы данных изделий имеет вертикальное разделение, аналогично Проводнику Windows. В левой половине окна в виде дерева или списка выводятся управляемые данные.

В качестве последнего уровня иерархии в дереве на экран выводятся отдельные номера изделий. При наличии более одного варианта изделия эти варианты изображаются под изделием.

Кроме дерева, данные могут отображаться в виде списка, причем объем отображаемых данных настраивается через всплывающее меню > Конфигурировать представление. При этом в списке представлены ***все*** изделия, но в нем нет производителей, поставщиков и клиентов, кроме того, выводятся только те свойства, которые имеют ***все*** изделия (т. е., например, номер изделия, группа продуктов и т.д.).

В правой половине диалогового окна выводится соответствующая информация об элементах, которые выделены в левой части окна. При этом имеются вкладки, которые доступны для ***всех*** типов изделий (например, Общее, Цены / прочее и т. д.), и вкладки, в которых имеются особые возможности ввода для определенного типа изделия (например, Модуль, Узел и т. д.).

### Индивидуализация структуры дерева

Чтобы обеспечить клиентский выбор изделий по технической информации и при этом не выполнять выбор по номеру изделия, можно адаптировать представление в виде дерева согласно вашим потребностям (так называемая "[Конфигурация дерева](partsmanagementgui_d_baumkonfiguration.md)"); например, можно создать собственные структурные узлы и сохранить определенные критерии для выбора проектирования под именем. Поиск изделия выполняется в структуре дерева на основе многоступенчатых критериев для описания устройства, а не по номеру изделия.
