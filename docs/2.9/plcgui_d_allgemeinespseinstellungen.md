# Диалоговое окно Настройки: ПЛК

Вы открыли проект.

* Параметры > Настройки > Проекты > "Имя проекта" > Устройство > ПЛК.
* Данные проекта > ПЛК > Адреса / Списки присвоений. В диалоговом окне Адреса / Списки присвоений выберите ++Дополнительно++ > Настройки.

В этом диалоговом окне установите проектные настройки для данных ПЛК.

Обзор основных элементов диалогового окна:

### При безрезультатном поиске символического адреса взять адрес

Если этот флажок установлен, (реальный) адрес копируется в символический адрес, когда поиск символического адреса не дает результата. Это происходит, если при отслеживании цели не найден датчик или исполнительный элемент. Подробную информацию см. в разделе "[Автоматическое присвоение символического адреса](plcgui_k_symbolischeadressen.md)".

Если этот флажок снят, поле символического адреса остается незаполненным, когда поиск символического адреса не дает результата.

### Одинаково обработать датчики и исполнительные элементы

При установке флажка отслеживание цели не различается между датчиками и исполнительными элементами. Поиск цели завершается при обнаружении датчика или исполнительного элемента.

При снятии флажка от входа разыскиваются только датчики и от выхода -- только исполнительные элементы.

### Расширенное копирование функционального текста зоны

Если этот флажок установлен, в выводах устройства ПЛК, для которых не были найдены функциональный текст зоны и запись в поле Функциональный текст (автоматически) связанного датчика или исполнительного элемента, производится поиск функционального текста зоны в имеющихся зонах. Найденный функциональный текст копируется в зависимости от направления вывода устройства слева или сверху.

Если этот флажок не установлен, поиск функционального текста зоны при связанном датчике или исполнительном элементе прерывается. По умолчанию флажок снят.

### Задать буквенное обозначение при импорте

Этот флажок относится только к импорту данных ПЛК в ***формате AutomationML***. Укажите здесь, как присваивать ОУ новым устройствам (в том числе устройствам, для которых в файле импорта отсутствует ОУ) при импорте. Имеющиеся в файле импорта ОУ всегда импортируются независимо от этой настройки и перезаписывают ОУ, которые есть в проекте. Устройства идентифицируются с помощью GUID AutomationML. Если ОУ в файле импорта пустое, а устройство уже содержится в EPLAN, это устройство сохранит свое ОУ.

Если этот флажок установлен, при импорте данных ПЛК буквенные обозначения автоматически присваиваются новым устройствам. При этом используется набор буквенных обозначений, заданных в [Настройках для online-режима нумерации](numerationgui_d_einstellnummerierung.md). Перед буквенным обозначением в качестве маркера устанавливается знак вопроса.

Если флажок снят, при импорте новые устройства не получают буквенного обозначения. Устройства нумеруются автоматически и дополнительно обозначаются вопросительным знаком.

### Макрос перехода для создания каркаса

Здесь задается макрос, используемый при генерации схемы соединений ПЛК и позволяющий графически обозначить переход к обзору каркаса. Этот файл макроса в варианте Е должен содержать графику для перехода, а в варианте F – графику для продвижения. Нажав кнопку ++...++, можно открыть диалоговое окно Выбрать макрос и выбрать там файл из любого каталога.

### Специфические для ПЛК настройки

Здесь установите настройки, которые зависят от типа ПЛК. Установленная здесь схема будет применяться в тех случаях, если ЦПУ не присвоено схем. Выберите имеющуюся схему из раскрывающегося списка. С помощью кнопки ++...++ откройте диалоговое окно Настройки: Специфические для ПЛК, в котором можно создать или обработать схему.
