# Функции топологии

Функция топологии — это обычная функция с видом представления "Топология". Для представления функции топологии существуют дополнительные библиотеки символов со специальными символами топологии. В библиотеках символов имеются символы топологии подходящего размера для масштабов страниц 1:1, 1:20, 1:50 и 1:100. На основании разных библиотек символов можно выбрать увеличенные символы топологии, которые еще можно различить на страницах с уменьшенным масштабом. Если символ имеет вывод устройства топологии, то при вставки этого символа автоматически присваивается вид представления "Топология".

Если размещается многополюсная функция с видом представления "Топология" (например, через пункты всплывающего меню Разместить > Топологияв навигаторе устройств), производится поиск и размещение символа с видом представления "Топология", соответствующего определению функции. Для этого в библиотеках символов топологии существует по символу для каждой категории функции.

Каждое устройство может содержать только ***одну*** функцию с видом представления "Топология". Исключением являются клеммники, которые могут содержать несколько функций топологии. В топологии размещается главная функция устройства, а вспомогательные функции управляются на схемах соединений.

### Выводы устройства топологии

Выводы устройства топологии служат для построения сети соединенных сегментов, они не оцениваются как нормальные выводы функции, и между ними также не устанавливается автоматическое соединение.

Обычно каждый символ топологии имеет только по одному выводу устройства топологии на вариант символа. Выводы устройства топологии и нормальные выводы устройств не должны смешиватся в одном символе. Даже если символ имеет только выводы устройства топологии, у функции топологии есть также и логические выводы устройства функции. Эти выводы функции видны, однако, только в схеме соединений функции.

Выводы устройства топологии соединяются сегментами маршрутизации. Маршрутизируемые соединения, напротив, подключаются к выводам устройства функции в схеме соединений функции топологии.

### Линий определения кабеля

Если линия определения кабеля вставляется на странице топологии, она автоматически получает вид представления "Топология" и определение функции "Кабель топологии". У линии определения кабеля отсутствуют выводы устройства топологии. Для соединений, отрезанных от линии определения кабеля, копирование ОУ не выполняется.

### Точки разрыва топологии

При помощи точки разрыва топологии можно продолжить сегмент маршрутизации на другой странице проекта. Точка разрыва топологии ведет себя как обычная точка разрыва.

Символ может иметь не более одного вывода устройства топологии, по умолчанию используется точка вставки как вывод устройства.

### Клеммники

В качестве функций топологии можно управлять только клеммниками, но не отдельными клеммами. Клеммы управляются на схемах соединений клеммников.

Один клеммник с видом представления "Топология" может встречаться в нескольких сетях соединенных сегментов, но только один раз внутри ***одной и той же*** сети соединенных сегментов.

Если клеммник встречается в нескольких сетях соединенных сегментов, то схема соединений автоматически расширяется при маршрутизации. В схему соединений на клеммнике принимаются только выводы устройства, связанные с функциями топологии внутри сети соединенных сегментов.

Схема соединений на клеммниках всегда локальна. При обновлении данных выводов устройств через действие Обновить схему соединений в диалоговом окне "Свойства" клеммника составляется схема соединений из всех клемм клеммника с их изделиями. После этого уже нет непосредственной связи с отдельными изделиями клемм.

Если клеммы не имеют обозначения вывода устройства, то при маршрутизации учитывается логическая схема вывода устройства (внутренний / внешний вывод устройства).

### Штекеры

Штекеры могут содержать несколько устройств в одном устройстве, при этом отдельные устройства будут отображаться по определениям штекеров. Таким образом, например, можно задать определение штекера для штырей и гнезд с одинаковым ОУ. В качестве функций топологии можно управлять только определениями штекера, но не отдельными контактами штекера. Контакты штекера управляются на схемах соединений определений штекеров. Для каждого устройства (т. е. каждого определения штекера) штекера требуется наличие соответствующей функции топологии.
