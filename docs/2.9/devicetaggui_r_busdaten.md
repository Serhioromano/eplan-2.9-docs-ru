# Вкладка Данные шины

Вы открыли проект.

* В графическом редакторе уже выделен кабельный вывод сети / шины. Обработать > Свойства. В диалоговом окне Свойства ++...++ выберите вкладку Данные шины.
* Вы выделили кабельный вывод сети / шины в навигаторе устройств или в одном из навигаторов ПЛК. Пункт всплывающего меню Свойства или Свойства (общие). В диалоговом окне Свойства ++...++ выберите вкладку Данные шины.

Для обозначенного вывода шины определите на этой вкладке свойства, которые этот вывод будет иметь внутри системы шины.

В режиме обработки ***Свойства (общие)*** вы изменяете свойства сразу для всех представлений функции, в этом случае во вкладке дополнительно отображается "(Устройство)".

Обзор основных элементов диалогового окна:

### Проект конфигураций

Если это поле пустое, проект конфигураций вывода шины автоматически определяется соответствующим блоком ПЛК (главная функция). Ввод требуется только в том случае, если проект конфигураций вывода шины отличается от проекта конфигураций соответствующего блока ПЛК.

Для отображения на вкладке Данные шины имеется свойство Проект конфигурации (автоматически, на выводах шин) (ид. 20581). Данное свойство выводит введенный вручную проект конфигураций на выводе шины или, если этот проект конфигураций пуст, выводит проект конфигураций соответствующего блока (главная функция) ПЛК.

### Физическая сеть: Имя

Введите здесь имя физической сети, к которой подключен вывод шины. Это имя внутри проекта конфигураций должно быть уникальным. Внутри физической сети можно указать только одну систему шин.

### Физическая сеть: Адрес шины / номер позиции

Введите здесь адрес шины соответствующего вывода шины. В зависимости от системы шин значение может быть простым числом, IP-адресом или комбинацией букв и цифр.

!!! note "Замечание:"

    В групповом поле Свойства сетевого интерфейса в свойстве Физическая сеть: Адрес шины / номер позиции 2 можно указать другой адрес вывода шины. Свойство используется для устройств ASI при обмене данными ПЛК в формате AutomationML.

### Система шин

Выберите из раскрывающегося списка систему шин, например Profibus, Ethernet, ASI и т. д.

В информационной области под этим полем отображается дополнительная информация о выбранной системе шин, связанной с поведением во время обмена данными ПЛК. При этом отображаются комбинации следующих записей:

* Неправильное соединение с конкретным портом: эти системы шин можно экспортировать с подробным соединением портов.
* Физическая сеть: эти системы шин формируют физическую сеть.
* Логическая сеть: для этих систем шин также экспортируется логическая сеть, указанная в свойстве Логическая сеть: Имя.

* * *

[Групповое поле Свойства сетевого интерфейса](devicetaggui_r_gruppenfeldeigenschaften.md)

Для выводов шины доступны, в частности, следующие свойства:

* Интерфейс шины: Имя: Имя интерфейса шины используется при группировке выводов шины для экспорта [систем шин на базе Ethernet](plcgui_k_spsanschluesse.md). При помощи этого имени родственные выводы шин объединяются в одну логическую единицу. Для этого используйте выводы шины типа "Кабельный вывод сети / шины, общее".
Имя интерфейса шины используется вместе с обозначением штекера при идентификации выводов шины.
* Интерфейс шины: Главный вывод шины: Обозначает вывод шины в интерфейсе как главный. Этот вывод шины представляет интерфейс шины и является носителем данных, необходимых для обмена данными (в т. ч. Физическая сеть: Адрес шины / номер позиции, MasterSystemID). При экспорте данные считываются с главного вывода шины и записываются в файл конфигурации ПЛК.
В интерфейсе можно указывать только один главный вывод шины, то есть один из выводов шины типа "Кабельный вывод сети / шины, общее".
* Игнорировать недостающий адрес шины: Эта настройка предусмотрена для устройств в системах шин, не требующих адреса шины. Если это свойство активировано на выводе шины, недостающий адрес шины игнорируется при выполнении контрольного прогона [004037](msg/messages_p_004037.md), а для этого вывода шины не выводится сообщение контрольного прогона.
* Логическая сеть: Имя: введите здесь имя логической сети, к которой относится вывод шины. Эта запись должна быть уникальной внутри физической сети.
При обмене данными ПЛК для ***систем IO*** укажите здесь имя контроллера IO для системы IO. Все блоки ПЛК с одинаковым именем логической сети относятся к одной системе IO. Система IO устанавливает, каким контроллером IO управляется каждая рабочая станция ("Устройство IO" в TIA Portal). Это похоже на определение принадлежности ЦПУ: в то время как принадлежность ЦПУ ориентирована на карты ПЛК, система IO ориентирована на станцию.
При использовании системы шин Profibus "Система IO" аналогична "Главной системе DP", "Контроллер IO" аналогичен "DP-главному", а "Устройство IO" — "DP-подчиненному".
* Логическая сеть: Вывод шины является главным: установите этот флажок, если вывод шины является главным логической сети. В системе IO (в системах шин на базе Ethernet) или в главной системе DP (в Profibus) всегда должен быть обозначен вывод шины, который является главным для присвоения.
