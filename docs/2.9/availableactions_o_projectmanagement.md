# Операция: projectmanagement

**Параметр** |  **Описание**
---|---
TYPE |  Вид выполняемой задачи:
READPROJECTINFO: Загрузить информацию о проекте из файла XML в проект.
ToggleSections: Если активирован выбор областей обработки, это значение позволяет включать и отключать временную обработку во всем проекте для зарегистрированного в данный момент пользователя. При этом проект должен быть открыт.
PUBLISHTODISK: Проект публикуется на носителе данных.
PUBLISHTOEMAIL: Проект публикуется в виде электронного сообщения.
CREATESNAPSHOTCOPY: Генерирует моментальную копию проекта.
EXPORTPROPERTYPLACEMENTSSCHEMAS: Экспортирует порядок свойств проекта.
IMPORTPROPERTYPLACEMENTSSCHEMAS: Импортирует порядок свойств в проект.
CORRECTPROJECTITEMS: Корректирует проект.
REORGANIZE: Реорганизует проект.
REORGANIZE: Реорганизует проект.
LOADDIRECTORY: Считывает каталог. Если не указаны дополнительные параметры, по умолчанию считывается стандартный каталог проекта $(MD_PROJECTS). Если указан параметр PROJECTSDIRECTORY, можно определить каталог для чтения. Параметр SCANSUBDIRECTORIES определяет, следует ли читать подкаталоги (0 означает "Нет", 1 означает "Да"). Если этот (второй) параметр не указан, также считываются подкаталоги.
PROJECTNAME |  Имя проекта с полным путем файла (является опцией).
Если не задано, то выбранный проект используется, когда операция вызывается через интерфейс пользователя (напр., через сценарий или панель инструментов). При вызове из командной строки Windows следует определить PROJECTNAME или сначала следует использовать ProjectAction. В противном случае отобразится системное сообщение.
FILENAME |  Полный путь файла и имя импортируемого XML-файла.
SCHEME |  Имя схемы. Действительно, только если параметр TYPE имеет следующие значения:
PUBLISHTODISK
PUBLISHTOEMAIL
CREATESNAPSHOTCOPY
CORRECTPROJECTITEMS
OVERWRITE |  Если импортированная схема уже существует для размещений свойств, этот параметр указывает на то, должна ли она перезаписываться. Возможные значения: 0: Нет (значение по умолчанию); 1: Да.
Действительно, только если параметр TYPE имеет следующее значение: IMPORTPROPERTYPLACEMENTSSCHEMAS.
EXTENDEDMODE |  Задает "расширенный режим" для реорганизации проектов. Возможные значения: 0: Нет (значение по умолчанию); 1: Да.
Действительно, только если параметр TYPE имеет следующее значение: REORGANIZE.

!!! example "Пример:"

    Загрузить в проект информацию о проекте:projectmanagement
    /TYPE:READPROJECTINFO
    /PROJECTNAME:"C:\Projects\EPLAN projects\ESS_Sample_Project.elk"
    /FILENAME:C:\Files\ProjectInfo.xmlВключение/отключение временной обработки во всем проекте:projectmanagement
    /TYPE:ToggleSections
    /PROJECTNAME:"C:\Projects\EPLAN projects\ESS_Sample_Project.elk"
