# Диалоговое окно Настройки: Общее (предварительное планирование)

Вы открыли проект. Параметры > Настройки > Проекты > "Имя проекта"> Предварительное планирование > Общ..

В этом диалоговом окне укажите, следует ли по умолчанию защищать функции, связанные с объектами планирования. Кроме того, вы определяете поведение при генерировании схем соединений из предварительного планирования.

Обзор основных элементов диалогового окна:

### Защита прибора при генерировании функций

Если этот флажок установлен, функции, связанные с объектами планирования, будут оборудованы автоматической защитой прибора. То есть при размещении и присвоении объектов планирования, а также при генерировании функций в объекте планирования соответствующим функциям автоматически присваивается свойство Защита прибора (ID 20475). Эта защита применяется в основном в отношении изделий, присвоенных функции, но она также предотвращает удаление функций.

Если флажок снят, то функции, связанные с объектами планирования, остаются незащищенными по умолчанию.

Групповое поле Генерировать схему соединений

Из предварительного планирования можно генерировать схемы соединений, перетаскивая мышью объекты планирования с присвоенными им макросами в навигатор страниц. При этом по выбору возможно размещение каждого макроса на отдельной странице или нескольких макросов на одной странице. Имя страницы автоматически определяется из структурного идентификатора вышестоящих сегментов объекта планирования.

### Защита от изменений при генерировании страниц

Если этот флажок установлен, страницы, генерированные из макросов, автоматически снабжаются защитой от изменений. Т. е. этим страницам присваивается свойство Защита от изменений (ид. 3014). В результате этого страницы и размещенные на них функции получают защиту от изменений, и впоследствии их уже нельзя обработать, удалить и т. д. Изменения через предварительное планирование с помощью функции Обновить детальное планирование возможны.

Если флажок снят, страницы, генерированные из макросов, остаются незащищенными и доступны для дальнейшей обработки.

### Учитывать только структурные идентификаторы сегментов структуры

Эта настройка актуальна, только если на объектах планирования (технологический контур / датчик / исполнительный элемент) были указаны [отдельные элементы](planninggui_k_prinzip.md) структурных идентификаторов, отличающиеся от структуры вышестоящих сегментов (напр., в свойство Место установки (отдельный элемент)).

Если этот флажок установлен, отдельные элементы объектов планирования ***не*** будут переноситься в структурные идентификаторы создаваемых страниц. Для структурных идентификаторов страниц имеют значение только структурные идентификаторы ***сегментов структуры***. В этом случае отдельные элементы структурных идентификаторов объекта планирования переносятся только на ту функцию, для которой на объекте планирования указан шаблон функции.

Если этот флажок снят, отдельные элементы объектов планирования переносятся в структурные идентификаторы создаваемых страниц. Для структурных идентификаторов страниц имеют значение ***все*** структурные идентификаторы сегментов структуры и объектов планирования. В результате отдельные элементы структурных идентификаторов объекта планирования также переносятся на все функции, размещенные на странице.

### Разместить на одной странице несколько макросов

Если этот флажок установлен, несколько макросов автоматически располагаются на странице, пока там есть место. После этого генерируется новая страница и там размещаются следующие макросы.
Если перетянуть объект планирования на уже имеющуюся страницу, он будет размещен там (при наличии места), в ином случае сгенерируется новая страница.

Если флажок снят, для каждого макроса генерируется отдельная страница.

### Расстояние до края

Если вы размещаете несколько макросов на одной странице, установите здесь расстояние, на котором макросы должны размещаться до боковой стенки.

### Расстояние между макросами

Если вы размещаете несколько макросов на одной странице, установите здесь расстояние между макросами.

### Разместить макросы друг под другом

Этот флажок доступен, только если была включена настройка Разместить на одной странице несколько макросов.

Если этот флажок установлен, несколько макросов автоматически размещаются на странице друг под другом. В этом случае действуют настройки Расстояние до края и Расстояние между макросами для размещения в направлении Y.

Если этот флажок снят, несколько макросов автоматически размещаются на странице друг за другом. В этом случае действуют настройки Расстояние до края и Расстояние между макросами для размещения в направлении X.

Групповое поле Окрасить условное обозначение и трубопровод в функц. схеме автоматизации

Вы можете выделить цветом резервуары, технологические контуры, трубопроводы и т. д., которые связаны с сегментами в предварительном планировании, на функциональной схеме автоматизации. Это позволяет сделать видимыми структуры из предварительного планирования, которые размещены на функциональной схеме автоматизации.

В навигаторе предварительного планирования задайте желаемый цвет в диалоговом окне 'Свойства' сегмента, используя свойство Цвет для функциональной схемы автоматизации.

### С присвоенным сегментом

Если этот флажок установлен, все размещенные условные обозначения и трубопроводы, связь которых находится в навигаторе предварительного планирования под сегментом с выбранным цветом (свойство Цвет для функциональной схемы автоматизации), будут выделены этим цветом на функциональной схеме автоматизации. В навигаторе предварительного планирования обозначение и описание такого сегмента также будут отображаться в этом цвете.

Если этот флажок снят, цвета не будут отображаться как на функциональной схеме автоматизации для объектов со связанными сегментами, так и для сегментов в навигаторе предварительного планирования.

### В трубопроводах (без присвоенного сегмента)

Этот флажок доступен, только если была включена настройка С присвоенным сегментом.

Если установить этот флажок, для выделенных цветом трубопроводов дополнительно выделяются цветом условные обозначения, которые относятся к трубопроводу, но не связаны с сегментом в предварительном планировании.
