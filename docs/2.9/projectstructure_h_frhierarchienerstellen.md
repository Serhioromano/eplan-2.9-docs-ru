# Определить пользовательскую структуру проекта

Наряду с предварительно заданными схемами обозначения для структуры проекта можно также создать определенные пользователем схемы и тем самым ввести собственные структуры проекта. Определяемая пользователем структура обозначения охватывает наряду с блоками идентификаторов "Функциональное присвоение", "Установка", "Место сборки", "Номер установки" и "Вид документа" также пользовательский блок идентификаторов, для которого можно свободно выбрать предшествующий знак.

Определяемая пользователем структура проекта, состоящая из пользовательской структуры страниц и устройств, может быть впоследствии изменена с помощью диалогового окна свойств проекта. При этом повторно изменять структуру страниц в проекте со страницами впоследствии можно только при определенных [условиях](projectstructure_k_hinterg.md). Назначение всех структур производится одинаковым образом.

Условие:

Вы создали проект и выделили его в навигаторе страниц.

1. Выберите пункты меню Проект > Свойства и переместите вкладку Структура на передний план.
2. Щелкните по ++...++ рядом с полем для структуры страниц или рядом с одним из полей для устройств.
3. В диалоговом окне Структура страниц / диалоговом окне Структура устройства с помощью раскрывающегося списка присвойте каждому блоку идентификаторов одно из трех свойств: "Идентифицирующ.", "Описывающ." или "Недоступно".
4. Если вы хотите изменить последовательность идентификаторов в структуре страниц, выделите в диалоговом окне Структура страниц в столбце Структура требуемый блок идентификаторов и передвиньте его на желаемую позицию с помощью кнопок с изображением стрелки.
5. Щелкните по кнопке ++OK++.

### Назначение определенных пользователем предшествующих знаков и разделителей

В определенной пользователем структуре проекта можно также назначить выбираемые и/или пользовательские предшествующие знаки и/или разделители для блоков идентификаторов. Тем не менее, это возможно не для всех блоков идентификаторов, поскольку некоторые знаки фиксированы. В следующей таблице дается разъяснение этой взаимосвязи:

Блок идентификаторов |  Предшествующий знак  |  Разделитель для подидентификатора
---|---|---
Функциональное присвоение |  ==
#
≠ |  .
==
Установка |  == |  .
==
Место сборки | ++|  .++ Место установки |  + |  .
+
Номер установки |  Определено пользователем |
Вид документа |  Пробел
& |
Определено пользователем |  Определено пользователем |  Определено пользователем

!!! note "Замечание:"

    Предшествующие знаки и/или разделители можно выбирать или самостоятельно определять, только если соответствующему блоку идентификаторов при создании новой схемы обозначения присваивается свойство "Идентифицирующ." или "Описывающ.".

Порядок действий:

Условие:

Вы создали новую схему для структуры страниц или устройств.

1. Во вкладке Структура диалогового окна Свойства проекта нажмите кнопку ++Другие++.
2. Выберите в диалоговом окне Расширенные структуры проекта для блоков идентификаторов Функциональное присвоение, Номер установки, Вид документа и Опред. пользователем предшествующий знак или введите там, где возможен свободный выбор, предшествующий знак по вашему выбору.
3. Выполните такие же действия с разделителем для подидентификаторов.
4. Щелкните по кнопке ++OK++.

!!! note "Замечание:"

    Следующие предшествующие знаки и идентификаторы ***нельзя*** использовать для блоков идентификаторов, потому что они предназначены для других целей:

* / разделитель перед именем страницы
* \ структура каталогов
* ? Заполнитель для автономной нумерации
* Разделитель для обозначения вывода устройства.
