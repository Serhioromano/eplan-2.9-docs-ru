# Диалоговое окно "Сервер SQL — Настройка" / Диалоговое окно "Генерировать базу данных сервера SQL"

* Параметры > Настройки > Пользователь > Перевод > Словарь. Выберите параметр Сервер SQL и щелкните по кнопке ++...++.
* Сервисные программы > Перевод > Обработать словарь. Выберите вкладку Управление. Выберите параметр Сервер SQL и щелкните по кнопке ++...++.
* Параметры > Настройки > Пользователь > Управление > База данных управления проектами. Выберите параметр Сервер SQL и щелкните по кнопке ++...++.

В этом диалоговом окне задаются предварительные настройки для сервера SQL, который должен использоваться для работы со словарем или с управлением проектами.

* Сервисные программы > Изделие > Управление > [Дополнительно] > Настройки. Выберите параметр Сервер SQL и щелкните по кнопке ++...++.
* Параметры > Настройки > Пользователь > Управление > Изделия. Выберите параметр Сервер SQL и щелкните по кнопке ++...++.
* Вы открыли проект и выделили главную функцию в графическом редакторе или навигаторе. Обработать > Свойства. Вы щелкнули на вкладке Изделие рядом с полем Источн. данных по кнопке ++...++. В диалоговом окне Настройки: Изделие (пользователь) выберите опцию Сервер SQL и щелкните по кнопке ++...++.

В этом диалоговом окне задайте предварительные настройки для сервера SQL, который должен использоваться для работы с базой данных изделий и / или выбором изделия. При этом настройки эксклюзивны, т. е. настройки, заданные для базы данных изделий, ***не*** действительны автоматически для выбора изделия.

* Сервисные программы > Перевод > Обработать словарь. Выберите вкладку Управление. Выберите параметр Сервер SQL. [Дополнительно] > Создать.
* Параметры > Настройки > Пользователь > Управление > База данных управления проектами. Выберите параметр Сервер SQL. Щелкните по кнопке ![](images/all_new_as.png){: .ui-icon } (Создать).

В этом диалоговом окне можно сгенерировать новую пустую базу данных сервера SQL.

!!! note "Замечание:"

    Для переключения словаря на сервер SQL платформа EPLAN поддерживает перечисленные в описании характеристик версии ***Microsoft SQL Server***. Это относится как к перенастройке базы данных изделий, так и к настройке внутреннего выбора изделия на сервере SQL.

Обзор основных элементов диалогового окна:

### Сервер

Укажите здесь имя сервера, на котором осуществляется управление базами данных.

### Регистрация

Выберите из раскрывающегося списка вид регистрации для сервера SQL:

* Проверка подлинности Windows: С помощью этой настройки можно ссылаться на сервер SQL сразу после регистрации в Windows в качестве пользователя; отдельный пароль для идентификации не требуется.
* Сервер SQL: С помощью этой настройки можно ссылаться на сервер SQL, только если вы указаны как авторизованный пользователь в обоих нижних полях. Для этого на сервере SQL необходимо создать базу данных прав и заполнить ее данными.

### Пользователь/ пароль

Эти поля активны только тогда, когда в окне Регистрация выбран параметр Сервер SQL!
Укажите здесь ваше имя и пароль, если необходимо защитить доступ на сервер SQL с помощью пароля. Чтобы гарантировать конфиденциальность, введенная строка символов представлена точками.

### База данных

Укажите имя базы данных, которая должна использоваться для словаря или для базы данных управления проектами. Или выберите из раскрывающегося списка имя существующей на указанном сервере базы данных.

!!! tip "Совет:"

    Чтобы перенести записи из существующей базы данных Access в базу данных сервера SQL, экспортируйте сначала данные из словаря Access, сгенерируйте новую базу данных сервера SQL, а затем импортируйте данные в новую базу данных.
