# Управление ревизиями

При последующей модификации имеющихся установок необходимо указывать информацию об изменениях. С помощью управления ревизиями можно вводить и документировать такие изменения; вы также можете возвращаться к более старым версиям проекта.

Для этого вы можете "закрепить" определенное состояние проекта и позже сравнить его с другим состоянием. Таким образом вы можете, например, выяснить, какие страницы изменились, чтобы отправить новое задание на печать. Или вы можете поставить разрешающую печать на измененных страницах.

Управление ревизиями предлагает следующие возможности:

* У вас есть возможность работать с ***Отслеживанием изменений*** или используя ***Сравнение свойств проектов***.
* Отслеживая изменения, вы создаете ревизию. При этом текущий проект становится ревизионным проектом, в котором вы продолжаете теперь работать. Все изменения в данном проекте ревизии регистрируются.
* При сравнении свойств проектов выполняется сравнение определенных свойств текущего проекта с соответствующими свойствами проекта для сравнения. При этом вы самостоятельно указываете, какие свойства проверяются на предмет изменений.
* Можно закрывать проекты в каждый из моментов времени, т. е. "закреплять" состояние проекта и дальше работать в новой ревизии проекта.
* В любой момент вы можете создать копию текущего состояния проекта в качестве исходного проекта.
* Изменения отображаются в проекте графически и текстово в качестве маркеров ревизии.
* Данные ревизии, объединенные при помощи отслеживания изменений в единый проект, можно обработать.
* Привязанные к одному проекту состояния ревизии можно вывести в обзорах ревизий. Их можно распечатать или ввести в текущий проект в качестве отдельных страниц отчета.
