# Создание подробного планирования из макросов

Если на объектах планирования сохранены макросы, из них можно создать подробное планирование. Для этого можно разместить эти макросы на уже имеющихся страницах проекта или автоматически генерировать страницы проекта, на которых будут размещены макросы.

!!! note "Замечание:"

    При размещении макросов, как правило, вместе с ними размещаются рамки макросов. С помощью рамки макроса идентифицируются содержащиеся в нем функции, объекты-заполнители и т. д., и при необходимости их можно обновить в подробном планировании.

### Размещение с выбором вида представления

В навигаторе предварительного планирования с помощью пункта всплывающего меню Разместить можно разместить макрос, записанный на объекте планирования, на имеющейся странице проекта. В открывшемся диалоговом окне Разместить устройство для макроса изделия выберите вид представления, в котором размещается макрос.

Если на объекте планирования не сохранены макросы, размещается сохраненный на изделии макрос изделия (при наличии), в противном случае с помощью открывшегося диалогового окна Разместить устройство может открыться диалоговое окно выбора макроса.

### Размещение путем перетаскивания мышью

Объекты планирования с присвоенными макросами можно перетащить из дерева навигатора предварительного планирования в навигатор страниц с помощью мыши. Если присвоен макрос страницы, по умолчанию генерируется новая страница схемы соединений. Если присвоен макрос окна, на каждый вид представления в макросе можно сгенерировать новую страницу схемы соединений. Имена страницы автоматически определяются из структурного идентификатора вышестоящих сегментов объекта планирования. В зависимости от настройки проекта Учитывать только структурные идентификаторы сегментов структуры отдельные элементы структурных идентификаторов, указанных на объекте планирования, дополнительно переносятся в имя страницы.

Также при многократном выборе объектов планирования по умолчанию для каждого макроса генерируется новая страница проекта.

Пользуясь настройкой проекта Разместить на одной странице несколько макросов, один или несколько макросов можно разместить на одной уже существующей или новой странице. В этом случае макросы размещаются на странице с учетом настроек для расстояния до края страницы и расстояния между макросами, пока на странице хватает места. После этого генерируется новая страница и там размещаются следующие макросы.

По умолчанию макросы размещаются друг за другом. С помощью настройки проекта Разместить макросы друг под другом можно задать, чтобы макросы размещались на странице друг под другом.

Если страница генерируется из макроса, который сохранен на объекте планирования, эта страница автоматически записывается на объекте планирования в список присвоенных страниц.

!!! tip "Совет:"

    С помощью свойства Макрос для предварительного планирования и функциональная схема автоматизации вы можете определить на сегментах макросы окон с видом представления "Предварительное планирование" и / или "Функциональная схема автоматизации". С помощью функции перетаскивания такой сегмент с присвоенным макросом окна можно разместить на странице проекта соответствующего типа страницы и тем самым [дополнить существующее предварительное планирование графическими представлениями и функциональными схемами автоматизации](planninggui_k_vorplanungerweitern.md).

### Обновление макросов

Для обновления размещенных на страницах проекта макросов или их данных имеются различные возможности:

* ***Разместить макросы заново:***
Если в предварительном планировании на объекте планирования сохраненный макрос заменяется другим, макрос, размещенный на странице проекта, можно поменять. Для этого в навигаторе предварительного планирования выберите пункт всплывающего меню Обновить детальное планирование, а в одноименном открывшемся диалоговом окне установите флажок Разместить макросы заново. Благодаря этому для выделенного в навигаторе предварительного планирования объекта стирается размещенный до этого макрос, а новый макрос размещается заново.
* ***Обновить объекты-заполнители:***
Объекты заполнителя в макросе можно использовать для того, чтобы переносить данные из свойств объектов планирования в размещенные функции детального планирования. Если в предварительном планировании были изменены данные объекта планирования (напр., выбран другой набор значений для объекта-заполнителя в макросе), с помощью обновления детального планирования также можно перенести измененные данные в размещенный в подробном планировании объект-заполнитель. Для этого в навигаторе предварительного планирования выберите пункт всплывающего меню Обновить детальное планирование, а в одноименном открывшемся диалоговом окне установите флажок Обновить наборы значений объектов-заполнителей.
* ***Обновить макрос в схеме соединений:***
Если данные макроса изменились (например, потому что в макросе были изменены условные обозначения), на странице схемы соединений с помощью пункта всплывающего меню Обновить макрос можно обновить макрос, относящийся к выделенной рамке макроса. Графическая информация, например порядок свойств или форматирование, попадает сюда через макрос и изменяется соответствующим образом; логическая информация при обновлении остается.
