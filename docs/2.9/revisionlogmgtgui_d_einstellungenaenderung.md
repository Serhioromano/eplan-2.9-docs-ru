# Диалоговое окно Настройки: Ревизия (отслеживание изменений)

Вы открыли проект. Проект не защищен от записи. Параметры > Настройки > Проекты > "Имя проекта" > Управление > Ревизия (Отслеживание изменений).

В этом диалоговом окне задаются настройки отслеживания изменений.

Обзор основных элементов диалогового окна:

### Текст водяного знака

Введите в это поле текст, которым измененные страницы обозначаются как "План". Этот текст отображается на соответствующих страницах водяными знаками.

### Шаблон для автоматического выделения

При определенных изменениях страницы и при изменении объектов EPLAN автоматически создает тексты описания, отображаемые в качестве маркера ревизии при печати и/или обзорах ревизий.

В этом поле укажите, какие свойства следует отображать в автоматическом тексте описания. Через [...] откройте диалоговое окно Формат, в котором можно определять свойства и их форматирование.

### Всегда запрашивать описание изменения страниц

Если этот флажок установлен, то при определенных изменениях страниц проекта ревизии отображается диалоговое окно Обработать маркеры ревизий, в котором можно обработать текст маркеров ревизии.

### Вывести проект как PDF при закрытии

если установлен этот флажок, при закрытии проекта автоматически сохраняется его копия в PDF-формате.

### Всегда запрашивать текст маркировки

Если установлен этот флажок, то при определенных изменениях в проекте ревизии отображается диалоговое окно Обработать маркеры ревизий, в котором можно обработать автоматически созданный текст маркеров ревизии.

Если этот флажок не установлен, то тексты маркеров ревизии автоматически вводятся на измененных объектах без отображения диалогового окна для обработки текста.

### При новой ревизии закрытого проекта сгенерировать исходный проект

Если установлен этот флажок, то при создании новой ревизии ***закрытого*** проекта дополнительно создается исходный проект. Исходный проект автоматически получает имя и сохраняется в каталоге проектов.

если установлен этот флажок, закрытый проект при создании ревизии становится *.ell-проектом. Изменения управляются с помощью данных ревизии.

### При удалении сгенерировать маркер удаления и перечислить удаленные страницы

Если этот флажок установлен, при удалении объектов (условных обозначений, текстов и т.д.) генерируются маркеры удаления, а удаленные объекты обозначаются на страницах проекта посредством маркера ревизии и маркера удаления. Свойства удаленных объектов сохраняются на маркерах удаления, где их можно просматривать.

Дополнительно сохраняются данные об удаленных страницах. Список удаленных страниц отображается в диалоговом окне Удаленные страницы.

Если этот флажок не установлен, то маркеры удаления не создаются и данные об удаленных страницах не сохраняются.

### При закрытии отдельно учитывать типы страниц

В этот групповом поле можно указать, чтобы определенные типы страниц при закрытии обрабатывались отдельно.

Отображенные в поле Всегда завершать типы страниц всегда учитываются при закрытии и снабжаются индексом ревизии, даже если на странице проекта на выполнялось изменений.

Отображенные в поле Никогда не завершать типы страниц никогда не учитываются при закрытии, независимо от того, производились изменения на странице проекта или нет.

Посредством кнопки, расположенной вверху обоих полей, можно вставлять требуемые типы страниц в соответствующее поле или удалять их из поля.

Кнопка |  Значение
---|---
![](images/all_new_as.png){: .ui-icon } (Создать) |  Откройте диалоговое окно Выбрать тип страницы, чтобы выбрать требуемые типы страниц.
![](images/all_delete_as.png){: .ui-icon } (Удалить) |  Удаляет выделенные типы страниц из поля.

Чтобы страницы с выбранным в поле Всегда завершать типом страниц учитывались при закрытии страниц, необходимо выделить эти страницы в навигаторе страниц до закрытия или при закрытии проекта. Если текущий индекс ревизии требует дополнительной нумерации, необходимо выбрать соответствующую схему нумерации в настройках фирмы.

!!! example "Пример:"

    Всегда завершать типы страниц Из образца проектаESS_Sample_Projectвы создали ревизию и на страницесхемы соединений=GB1+A1&EFS1/1внесли изменение с помощью описания Электропитание 400/230 В. Чтобы последний индекс ревизии также отображался на титульной странице, в диалоговом окне Настройки: Ревизия (отслеживание изменений)в поле Всегда завершатьвыберите тип страницы Титульный лист. Сначала выделите в навигаторе страниц страницусхемысоединений и титульный лист. После того как страницы с индексомEPLAN1закрыты, этот текст будет отображен также в рамке на титульном листе.Никогда не завершать типы страниц Вы произвели изменения на страницах схемы соединений проекта ревизии. Если при закрытии проекта установлен флажок Сгенерировать отчет при закрытии проекта, то страницы отчета обновляются при закрытии. Обновления учитываются как изменения и отображаются в приведенном выше обзоре ревизий. В случае объемных отчетов это может приводить к тому, что обзор ревизии становится неупорядоченным и трудночитаемым.Если обновленные страницы отчета не должны приводиться в обзоре ревизий, предварительно отмените завершение страниц отчета. Для этого в диалоговом окне Настройки: Ревизия (отслеживание изменений)в поле Никогда не завершатьвыберите соответствующие типы страниц (Спецификация изделий,Список обозначений устройстви т. д.). При закрытии проекта страницы отчета обновляются, но не приводятся в обзоре ревизий.

### Учитывать объекты в опциях при включении и выключении

Если этот флажок установлен, в ревизии записывается как изменение, так и переключение опций.

Если этот флажок не установлен, в ревизии не записывается переключение опций. Изменения опций записываются. Эта настройка имеет смысл для "основных проектов", в которых обрабатываются опции.

При сравнении проектов опции не учитываются. Сравниваются только функции, содержащиеся на данный момент в проекте, т. е. включенные опции.
