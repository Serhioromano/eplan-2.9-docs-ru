# Обозначение страниц согласно стандарту DIN EN 61355-1

Платформа EPLAN позволяет выполнить идентификацию страниц в соответствии со стандартом DIN EN 61355-1. В этом виде идентификации для структуризации используется так называемый "Идентификатор объекта". При этом через идентификатор объекта определенному объекту (например, функции, месту или продукту) присваивается документ (т.е. страница проекта).

!!! note "Замечание:"

    * Чтобы использовать идентификатор объекта для структурирования, в свойствах проекта должно быть также активировано расширенное обозначение ссылки (Флажок Расширенное обозначение ссылки на вкладке Структура).
    * Кроме того, необходимо указать, что идентификаторы объектов используются для структурирования страниц. Для этого при создании проекта в диалоговом окне Структура страниц поставьте для идентификатора объекта флажок Использовать для структурирования.
    * Идентификатор объекта используется ***только*** вместе с блоком идентификаторов Вид документа. Комбинация идентификаторов объектов и других блоков идентификаторов невозможна.
    * Во входящем в комплект поставки шаблоне проекта IEC_tpl002.ept для структурирования используется идентификатор объекта, а структура страницы отображается согласно DIN EN 61355-1.

При таком структурировании полное имя страницы образуется из идентификатора объекта, вида документа и имени страницы. Содержанием идентификатора объекта при этом может быть структурный идентификатор блока идентификаторов (например, место установки или маркировка продукции) или свободный текст.

!!! example "Пример:"

    Der vollständige Seitenname–K01.Q01&EFS/1setzt sich folgendermaßen zusammen:Идентификатор объекта:-K01.Q01, здесь маркировка продукции используется в качестве идентификатора объекта.Вид документа:EFSИмя страницы:1

Если флажок Использовать для структурирования установлен, поведение в отношении ввода структуры страницы и ее отчета переключается:

* В диалоговом окне Полное имя страницы отображается идентификатор объекта в таблице Структурный идентификатор. Вы можете обработать идентификатор объекта.
* В зависимости от настройки проекта Развернутое представление подструктур (в меню Параметры > Настройки > Проекты > "Имя проекта" > Отображение > Структура дерева (страницы)) идентификатор объекта в представлении структуры дерева отображается в развернутом виде, т. е. подструктуры отображаются в качестве отдельных узлов дерева.
* Идентификатор объекта является ***идентифицирующим***, т. е. страницы –K01.Q01&EFS/1 и –K01.Q02&EFS/1 обозначают две различные страницы.
* Всякий раз в качестве идентификатора объекта может использоваться только ***один*** аспект (функциональный аспект, аспект места или аспект продукции), т. е. ***один*** блок идентификаторов или свободный текст, например номер типа. При этом учитывается аспект функции для установки (предшествующий знак '='), аспект места для места установки (предшествующий знак '+') и аспект продукта для маркировки продукции (предшествующий знак '-'). Маркировка продукции является блоком идентификаторов для устройства.
* Идентификатор объекта подлежит проверке синтаксиса, как обычные структурные идентификаторы (если они используются как идентификаторы объектов). При вводе свободного текста проверка синтаксиса не выполняется. Тексты, которые не содержат предшествующих знаков или содержат несколько предшествующих знаков, автоматически распознаются как свободные тексты.
* Сортировка страниц ориентируется по сортировке в управлении структурными идентификаторами. Если свободный текст используется как идентификатор объекта, сортировка проводится в алфавитно-цифровом порядке.
* Если идентификатор в управлении структурными идентификаторами переименовывается, он изменяется также в идентификаторе объекта.
