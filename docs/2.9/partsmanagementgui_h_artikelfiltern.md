# Фильтровать изделия

Фильтрация данных является эффективной возможностью ограничения числа одновременно просматриваемых изделий и улучшения тем самым обзорности просмотра. Можно фильтровать двумя способами в базе данных изделий:

* на основе текста
* на основе полей.

Фильтр на основе текста может выводить в левой части окна только такие изделия, в которых любое поле содержит текст, установленный в фильтре. С помощью фильтра на основе полей можно выполнить поиск только в определенных полях записей данных (например, для поля Номер типа). При использовании этого фильтра проверяется, совпадает ли все содержимое поле с искомым текстом.

!!! note "Замечание:"

    Фильтрация записей данных возможна только в базах данных относительно изделия. Это значит, что в базах данных "Производитель / поставщик" и "Клиент" выполнение фильтрации невозможно!

Условия:

* Вы открыли базу данных изделий: Сервисные программы > Изделие > Управление
* Через меню ++Дополнительно++ > Настройки в диалоговом окне База данных изделий была открыта база данных изделий.

1. Щелкните по ++...++ в поле "Фильтр по полю".
2. Выберите в диалоговом окне Фильтр из раскрывающего списка схему и при необходимости доработайте ее или щелкните по ![](images/all_new_as.png){: .ui-icon } (Создать), чтобы создать новую схему.
3. Щелкните по ![](images/all_new_as.png){: .ui-icon } (Создать) в панели инструментов критериев фильтрации, чтобы вставить новый критерий.
4. Выберите в диалоговом окне Выбор критериев то свойство, которое должно быть использовано в качестве критерия фильтрации, или выберите запись "ИЛИ", чтобы задать логическую операцию ИЛИ.
5. Щелкните по кнопке ++OK++.
6. Повторите эти шаги, чтобы определить другие критерии фильтрации и ограничить выбор изделий. В случае выбора нескольких полей они логически соединяются друг с другом с помощью оператора И (стандарт), логическая операция ИЛИ может быть установлена с помощью выбора записи ИЛИ в диалоговом окне Выбор критериев.
7. Щелкните по кнопке ++OK++.

При использовании фильтра по полю проверяется, ***точно*** ли соответствует введенный текст содержимому поля данных. Если, например, выполняется фильтрация баз данных изделий по краткому названию производителя HER, то изделия, в соответствующем поле данных которых имеется запись HER1, найдены не будут.

!!! warning "Предупреждение:"

    Если в качестве критериев фильтрации используются данные с ***единицами измерения***, например "интервал установки" или "вес" (вкладка Монтажные данные), соответствующее значение, если оно быть использовано для фильтрации, следует указывать без единицы измерения. При фильтрации по ***Имени файла***, например, имени графического макроса, для соответствующего имени файла следует также указать путь файла и расширение имени файла.

    Кроме того, при выполнении фильтрации по ***внешнему документу*** следует иметь в виду, что после имени файла, выбранного с помощью кнопки ++...++ в поле Файл/гиперссылка, в этом поле может находиться еще один текст. (Его можно указать на вкладке [Документы](partsmanagementgui_r_dokumente.md) в поле Обозначение). Чтобы фильтр в этом случае функционировал, после имени файла должен находиться символ-заполнитель "*". Без этого заполнителя фильтровались бы только те изделия, при которых нет описательного текста. Так, например, при вводе "*Dokumentation_FES.123060_De.pdf*Test123*" будет выполнена фильтрация по содержимому поля "Dokumentation_FES.123060_De.pdf??_??@Test123;".

Как только начинает применяться фильтр, фильтруется вся база данных изделий, то есть число изделий сокращается, а имя схемы выводится в раскрывающемся списке слева рядом с кнопкой [...]. Но отчет фильтрации учитывает только изделия, которые содержат также все выбранные поля.

!!! example "Пример:"

    В базе данных изделий определен следующий фильтр:Критерий Изделие: Обозначение 1=Силовой выключатель Критерий Вид напряжения=Переменное.В этом случае будут найдены изделия, для которых в поле Обозначение 1введен текст Силовой выключатель,***а***в поле Вид напряженияуказано значение Переменное.

1. В поле Полнотекст. фильтр введите текст, по которому требуется выполнить поиск. При этом введенный текст может представлять собой только часть искомого понятия. Например, при фильтрации по тексту "Конденсатор" будут также показаны изделия, в тексте которых имеется вхождение "Силовой конденсатор".
Обратите внимание, что полнотекстовый фильтр в отображении ***Представление в виде дерева*** также будет использовать главные узлы "Список принадлежностей", "Размещение принадлежностей", "Схема сверления", "Схема соединений". На вкладках Список и Комбинация, наоборот, будут учитываться только данные изделия.
2. Щелкните по кнопке ![](images/all_search_as.png){: .ui-icon } (Поиск).

!!! info "Для сведения:"

    Отображение в дереве/в списке обновляется и отображает только изделия, одно из полей которых содержит введенный текст.

Искомый текст может быть расширен с помощью логических операторов (для синтаксиса со связями И / ИЛИ см. [Полнотекст. фильтр](partsmanagementgui_d_artverw.md)). При выходе из базы данных изделий фильр по тексту удаляется.
