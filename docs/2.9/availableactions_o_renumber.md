# Операция: Renumber

**Параметр** |  **Описание**
---|---
TYPE |  Вид задачи, которую должна выполнить операция:
DEVICES: Нумеровать устройства.
PAGES: Нумеровать страницы.
TERMINALS: Нумеровать клеммы.
CABLES: Нумеровать кабели.
CONNECTIONS: Нумеровать соединения.
PROJECTNAME  |  Имя проекта с полным путем файла (является опцией).
Если не задано, то выбранный проект используется, когда операция вызывается через интерфейс пользователя (напр., через сценарий или панель инструментов). При вызове из командной строки Windows следует определить PROJECTNAME или сначала следует использовать ProjectAction. В противном случае отобразится системное сообщение.
USESELECTION  |  Устанавливает, необходимо ли учитывать текущий выбор в интерфейсе пользователя при нумерации (необязательно, 0 = Нет, 1 = Да).
Значение по умолчанию: 0. Доступно для использования, только когда не используется параметр PROJECTNAME.
CONFIGSCHEME  |  Схема нумерации (является опцией).
Значение по умолчанию: последняя использованная схема нумерации. Доступно, только если параметр TYPE имеет следующие значения: DEVICES, TERMINALS, CABLES, CONNECTIONS.
STARTVALUE  |  Начальное значение для нумерации (является опцией).
Значение по умолчанию: 1. Доступно, только если параметр TYPE имеет следующие значения: DEVICES, PAGES, TERMINALS, CABLES.
STEPVALUE  |  Величина шага; значение, на которое увеличиваются показания счетчика (является опцией). Значение по умолчанию: 1. Доступно, только если параметр TYPE имеет следующие значения: DEVICES, PAGES, TERMINALS, CABLES.
POSTNUMERATE  |  Дополнительная нумерация устройств, обозначенных маркером "?" (необязательно, 0 = Нет, 1 = Да).
Значение по умолчанию: 0. Доступно, только если параметр TYPE имеет следующие значения: DEVICES, TERMINALS.
ALSONUMERATEDBYPLC  |  Соответствует настройке Также нумеровать с учетом влияния ПЛК (необязательно, 0 = Нет, 1 = Да).
Значение по умолчанию: 0. Доступно, только если параметр TYPE имеет следующие значения: DEVICES, TERMINALS.
PERMITSORTCHANGE  |  Соответствует настройке Разрешить изменение сортировки (необязательно, 0 = Нет, 1 = Да).
Значение по умолчанию: 0. Доступно, только если параметр TYPE имеет следующее значение: TERMINALS.
FILLGAPS  |  Соответствует настройке Заполнить пробелы (необязательно, 0 = Нет, 1 = Да).
Значение по умолчанию: 0. Доступно, только если параметр TYPE имеет следующее значение: TERMINALS.
IDENTIFIER  |  Соответствует настройке Группа (необязательно, 0 = Нет, 1 = Да).
Значение по умолчанию: 0. Если установлен этот параметр, буквенное обозначение в столбце Буквенное обозначение становится частью группы буквенных обозначений; при этом нумеруются все устройства, которые начинаются с этого буквенного обозначения (например, "K" в "KT44"). Если параметр не задан, выполняется нумерация всех устройств, которые содержат только это буквенное обозначение в ОУ (например, "K23").
Доступно, только если параметр TYPE имеет следующее значение: DEVICES.
STRUCTUREORIENTED  |  Соответствует настройке Нумерация с учетом структуры (необязательно, 0 = Нет, 1 = Да).
Значение по умолчанию: 0. Если этот параметр задан, выполняется нумерация для каждого структурного идентификатора. Если параметр не задан, нумеруются страницы.
Доступно, только если параметр TYPE имеет следующее значение: PAGES.
KEEPINTERVAL  |  Соответствует настройке Интервал не изменять (необязательно, 0 = Нет, 1 = Да).
Значение по умолчанию: 0. В комбинации с параметром STARTVALUE этот параметр указывает на то, что интервал выделенных страниц должен использоваться и для целевых страниц. Благодаря этому любое выделенное число страниц можно переместить на определенный интервал. Ввод величины шага в данном случае невозможен.
Доступно, только если параметр TYPE имеет следующее значение: PAGES.
KEEPTEXT  |  Соответствует настройке Текст не изменять (необязательно, 0 = Нет, 1 = Да).
Значение по умолчанию: 0. Если задан этот параметр, буквенная часть имени страницы не будет перезаписываться. Если параметр не задан, текстовая часть будет удалена.
Доступно, только если параметр TYPE имеет следующее значение: PAGES.
SUBPAGES  |  Соответствует настройке Подстраницы (является опцией).
Возможные значения:
0 = Удерживать: Имеющиеся подстраницы будут скопированы в целевые страницы без изменений.
1 = Нумеровать: Имеющиеся подстраницы будут заново пронумерованы с использованием начального значения "1" и величины шага "1". При каждой смене главной страницы нумерация подстраниц снова начинается со значения "1".
2 = Преобразовать в главные страницы: Подстраницы преобразуются в главные страницы и заново нумеруются.
Доступно, только если параметр TYPE имеет следующее значение: PAGES.
PREFIX  |  Соответствует настройке Префикс (необязательно, 0 = Нет, 1 = Да).
Значение по умолчанию: 0. Доступно, только если параметр TYPE имеет следующее значение: TERMINALS.
SUFFIX  |  Соответствует настройке Суффикс (необязательно, 0 = Нет, 1 = Да).
Значение по умолчанию: 0. Доступно, только если параметр TYPE имеет следующее значение: TERMINALS.
SEQUENCE  |  Соответствует настройке Последовательность (является опцией).
Возможные значения:
0 = Как сортировка: При этом используется отображаемая в навигаторе клеммников или штекеров последовательность сортировки клемм / контактов штекера. Выберите этот параметр, если вам необходимо пронумеровать имеющиеся клеммы / контакты штекеров в текущей последовательности.
1 = C ориент. на страницы.
2 = С ориент. на кабели.
3 = C ориент. на уровни.
Доступно, только если параметр TYPE имеет следующее значение: TERMINALS.
EXTENT  |  Соответствует настройке Объем (является опцией).
Возможные значения:
0 = Выделенные: Нумеруются только выделенные клеммы или контакты штекеров.
1 = Все из выбранных клеммников/штекеров: Нумеруются все клеммы выделенных клеммников или все контакты выделенных штекеров.
2 = Все из всех клеммников/штекеров: Нумеруются клеммы всех клеммников в проекте или контакты всех штекеров в проекте.
Доступно, только если параметр TYPE имеет следующее значение: TERMINALS.
POTENTIAL_N  |  Соответствует настройке N-клеммы / -контакты штекера (является опцией).
Возможные значения:
0 = Не изменять.
1 = Не изменять, соблюдать последовательность: N-клеммы при нумерации не изменяются, но учитываются при нумерации остальных клемм. Это означает, что обозначения для остальных клемм распределяются так, что N-клеммы нумеруются вместе.
2 = Нумеровать: Нумеруются все N-клеммы или контакты штекера.
Доступно, только если параметр TYPE имеет следующее значение: TERMINALS.
POTENTIAL_PE  |  Соответствует настройке PE-клеммы / -контакты штекера (является опцией).
Возможные значения:
0 = Не изменять.
1 = Не изменять, соблюдать последовательность: PE- / PEN-клеммы при нумерации не изменяются, но учитываются при нумерации остальных клемм. Это означает, что обозначения для остальных клемм распределяются так, что PE- / PEN-клеммы нумеруются вместе.
2 = Нумеровать: Нумеруются все PE- / PEN-клеммы или контакты штекера.
Доступно, только если параметр TYPE имеет следующее значение: TERMINALS.
POTENTIAL_SH  |  Соответствует настройке SH-клеммы / -контакты штекера (является опцией).
Возможные значения:
0 = Не изменять.
1 = Не изменять, соблюдать последовательность: SH-клеммы при нумерации не изменяются, но учитываются при нумерации остальных клемм. Это означает, что обозначения для остальных клемм распределяются так, что SH-клеммы нумеруются вместе.
2 = Нумеровать: Нумеруются все SH-клеммы или контакты штекера.
Доступно, только если параметр TYPE имеет следующее значение: TERMINALS.
MULTIPLETERMINALS  |  Соответствует настройке Многоуровневые клеммы (является опцией).
Возможные значения:
0 = Не изменять.
1 = Нумеровать одинаково: Одинаково обозначенные клеммы со свойством Допускается одинаковое обозначение получают одинаковый номер.
2 = Нумеровать по отдельности: Клеммы со свойством Допускается одинаковое обозначение получают собственный номер. Если многоуровневые клеммы перед нумерацией имели одинаковый номер, то после они имеют различные номера.
Доступно, только если параметр TYPE имеет следующее значение: TERMINALS.
KEEPALPHA  |  Соответствует настройке Обозначенные в алфавитном порядке (является опцией).
Возможные значения:
0 = Не изменять: Клеммы или контакты штекеров с символами алфавита в обозначении при нумерации игнорируются.
1 = Алфавитные элементы не изменять: Символы алфавита в обозначении клеммы или контакта штекера остаются. А первый цифровой элемент будет пронумерован заново. Если обозначение включает только алфавитные элементы, старое обозначение будет добавлено к новой нумерации. Следующие одна за другой клеммы с неодинаковыми счетчиками, но с одинаковым числовым элементом после нумерации получают одинаковый числовой элемент в счетчике.
2 = Нумеровать: Все клеммы / контакты штекера нумеруются заново. При этом старое обозначение перезаписывается.
Доступно, только если параметр TYPE имеет следующее значение: TERMINALS.
KEEPEXISTING  |  Соответствует настройке Имеющиеся ОУ кабеля не изменять (необязательно, 0 = Нет, 1 = Да).
Значение по умолчанию: 0. Если этот параметр задан, дополнительно нумеруются заново сгенерированные или скопированные кабели. ОУ уже пронумерованного кабеля сохраняется. Нумеруются только кабели со знаком "?" в буквенном обозначении.
Доступно, только если параметр TYPE имеет следующее значение: CABLES.
STARTVALUE_n  |  Начальное значение для нумерации групп соединений (является опцией).
Значение по умолчанию: 1. Доступно, только если параметр TYPE имеет следующее значение: CONNECTIONS.
STEPVALUE_n  |  Величина шага для нумерации групп соединений; значение, на которое увеличиваются показания счетчика (является опцией). Значение по умолчанию: 1. Доступно, только если параметр TYPE имеет следующее значение: CONNECTIONS.
GROUP_n  |  Группа соединений (является опцией). Доступно, только если параметр TYPE имеет следующее значение: CONNECTIONS.
OVERWRITE  |  Режим перезаписывания (является опцией).
Возможные значения:
0 = Все: Перезаписываются обозначения всех точек определения соединений, которые могут иметь обозначение соединения.
1 = Кроме 'установлено вручную': Обозначения соединений, которые имеют свойство Установлено вручную, перезаписываться не будут.
2 = Нет: Соединения, которые уже имеют обозначение, остаются без изменений и не обозначаются заново.
Доступно, только если параметр TYPE имеет следующее значение: CONNECTIONS.
AVOIDIDENTICALDESIGNATIONS  |  Соответствует настройке Избегать одинаковых названий (является опцией).
Возможные значения:
0 = Во всем проекте: Одинаковые обозначения избегаются во всем проекте, т. е. каждое соединение получает в пределах проекта уникальное обозначение соединения.
1 = При выборе: Каждое соединение, содержащееся в выборе, получает уникальное обозначение соединения. Обозначения соединения уникальны только в пределах выбора, кроме того, в проекте допускаются многократные обозначения.
2 = Нет: Одинаковые обозначения допускаются везде.
3 = На один счетчик области сброса (структура / страница): Одинаковые обозначения избегаются только в пределах области, определенной для сброса счетчика. Обозначения соединения уникальны только в счетчике сброса области, кроме того, в проекте допускаются многократные обозначения. Область сброса счетчика воздействует только на счетчик, но не на подсчетчик.
Доступно, только если параметр TYPE имеет следующее значение: CONNECTIONS.
VISIBILITY  |  Соответствует настройке Область видимости (является опцией).
Возможные значения:
0 = Показать все: При этой настройке также представляются видимыми обозначения, которые настроены на "невидимые".
1 = Не изменять: Обозначения отображаются так, как они определены, т. е. "невидимые" обозначения соединений не отображаются. Это настройка по умолчанию.
2 = Один раз на страницу и объем: При этой настройке соответственно на графически первом соединении включается видимость обозначения соединения и выключается при всех других соединениях того же диапазона.
Доступно, только если параметр TYPE имеет следующее значение: CONNECTIONS.
MARKASMANUAL  |  Соответствует настройке Выделить как 'установлено вручную' (является опцией). Доступно, только если параметр TYPE имеет следующее значение: CONNECTIONS.

!!! example "Пример:"

    Нумеровать устройства:Renumber
    /TYPE:DEVICES
    /STARTVALUE:1
    /STEPVALUE:1
    /USESELECTION:0
    /OMITNUMERATEDBYPLC:0
    /NUMERATECABLES:0
    /IDENTIFIER:X*
    /CONFIGSCHEME:"Kennbuchstabe Zähler"
    /POSTNUMERATE:0Нумеровать страницы:Renumber
    /TYPE:PAGES
    /STRUCTUREORIENTED:0
    /STARTVALUE:3
    /STEPVALUE:10
    /KEEPINTERVAL:1
    /KEEPTEXT:0
    /SUBPAGES:1
    /USESELECTION:0Нумеровать соединения:Renumber
    /TYPE:CONNECTIONS
    /CONFIGSCHEME: Potenzialorientiert
    /STARTVALUE_1:13
    /STEPVALUE_1:3
    /OVERWRITE:0
    /AVOIDIDENTICALDESIGNATIONS:0
    /VISIBILITY:1
    /MARKASMANUAL:1
    /USESELECTION:0
    /GROUP_1:1
