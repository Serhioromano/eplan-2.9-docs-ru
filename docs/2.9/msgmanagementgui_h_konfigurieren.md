# Конфигурировать контрольные прогоны

Условие:

Вы открыли проект.

1. Выберите следующие пункты меню: Данные проекта > Сообщения > Выполнить контр. прогон
2. Щелкните в диалоговом окне Выполнить контр. прогон рядом с полем Настройки по кнопке ++...++.
3. В диалоговом окне Настройки: Сообщения и контрольные прогоны выберите определенную пользователем схему из раскрывающегося списка или создайте аналогичную схему, используя [панель инструментов](schemeconfig_m_schemaschaltflaechen.md).
4. Выделите класс сообщений слева в списке.

!!! info "Для сведения:"

    В таблице, расположенной в правой части диалогового окна, выводятся возможные сообщения для этого класса.

5. Выберите в поле Категория в раскрывающемся списке, выводится ли сообщение как ошибка, предупреждение или замечание.
6. В поле Вид проверки выберите запись [Виды проверки](msgmanagementgui_k_prueflaufprinzip.md) из раскрывающегося списка.
7. Щелкните в поле Фильтр [...] и укажите в диалоговом окне Фильтр, на какие разделы должна распространяться проверка. Можно выбрать предварительно заданные схемы фильтров для разделов или определить собственные.
8. Щелкните по кнопке ++OK++.

!!! info "Для сведения:"

    Настройки сохраняются в выбранной схеме.

!!! note "Замечание:"

    * Обратите внимание на то, что не все вышеприведенные виды проверки выполнимы при каждой проверке, доступны лишь определенные виды проверки.
    * Некоторые полезные схемы с определенными конфигурациями контрольного прогона предлагаются EPLAN по умолчанию после установки программы.
