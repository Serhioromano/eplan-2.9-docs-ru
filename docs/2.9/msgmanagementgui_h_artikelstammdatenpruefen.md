# Проверить основные данные изделия

EPLAN предлагает возможность проверять созданные вами и сохраненные в базе данных изделий основные данные изделий на правильность и полноту. При этом требуется сначала создать проект, который будет служить "контрольным проектом". В контрольном проекте не должно быть страниц.

После настройки контрольного проекта мы рекомендуем действовать следующим образом:

* Сконфигурируйте управление сообщениями для отображения номера и варианта изделия.
* Выберите в контрольном проекте базу данных изделий для проверки.
* Создайте схему контрольного прогона для основных данных изделий.
* Задайте объем проверки и сохраните его в схеме.
* Укажите в навигаторе основных данных изделий, какие основные данные должны проверяться, и выполните контрольный прогон.

!!! note "Замечание:"

    Во время проверки основных данных изделий открытым должен быть только контрольный проект.

Условия:

* Вы открыли Навигатор страниц.
* Вы создали контрольный проект (без страниц) из любого шаблона проекта.
* Вы открыли контрольный проект, он отображается в навигаторе страниц.

### Отображать номер и вариант изделия в управлении сообщениями

1. Выберите пункты меню Данные проекта > Сообщения > Управление.

2. В диалоговом окне Управление сообщениями - <Имя проекта> выделите таблицу.
3. Выберите пункт всплывающего меню Конфигурировать представление.

4. Дополнительно установите флажок Номер изделия и Вариант изделия в диалоговом окне Конфигурировать представление.

!!! info "Для сведения:"

    Эта настройка позволяет отображать в диалоговом окне Управление сообщениями два дополнительных столбца, в которых выводится номер и вариант неверного или неполного изделия.

5. При необходимости измените позицию записи для обоих столбцов с помощью кнопок со стрелками.

6. Щелкните по кнопке ++OK++.

### Выбрать базу данных изделий для проверки

1. Выберите пункты меню Сервисные программы > Изделие > Управление.
2. В диалоговом окне База данных изделий щелкните ++Дополнительно++ и выберите пункт меню Настройки.
3. В открывшемся диалоговом окне выберите проверяемую базу данных изделий и щелкните ++Открыть++.

### Создать схему контрольных прогонов для основных данных изделий

1. Выберите пункты меню Данные проекта > Сообщения > Выполнить контр. прогон.
2. Щелкните в диалоговом окне Выполнить контр. прогон рядом с полем Настройки по кнопке ++...++.
3. На панели инструментов диалогового окна Настройки: Сообщения и контрольные прогоны щелкните по кнопке ![](images/all_new_as.png){: .ui-icon } (Создать), чтобы создать схему контрольного прогона, которая проверяет только основные данные изделий.
4. Присвойте схеме подходящее имя в диалоговом окне Новая схема, например Основные данные изделий, и сохраните при необходимости текст описания, например имя проверяемой базы данных изделий.
5. Щелкните по кнопке ++OK++.

!!! info "Для сведения:"

    Затем снова перейдите в диалоговое окно Настройки: Сообщения и контрольные прогоны, в котором предварительно настроена вновь созданная схема.

### Задать объем проверки и сохранить его в схеме

1. В диалоговом окне Настройки: Сообщения и контрольные прогоны выделите в списке слева все классы сообщений.
2. В диалоговом окне справа в столбце Вид проверки установите для произвольного сообщения контрольного прогона запись "Нет".
3. Выделите эту запись и скопируйте ее в буфер обмена.
4. Щелкните по столбцу Вид проверки.
5. С помощью комбинации клавиш [Ctrl] \+ [V] за один рабочий шаг присвойте всем сообщениям всех классов вид проверки "Нет".
6. Выделите класс сообщения "501 Основные данные изделия".
7. Выберите для всех сообщений контрольного прогона этого класса сообщений требуемый вид проверки ("Нет" или "Автономно").
8. При желании дополнительно определите в столбце Категория этого класса сообщений, как должно выводиться сообщение: как ошибка, предупреждение или указание.
9. На панели инструментов вверху и справа щелкните по кнопке ![](images/all_save_as.png){: .ui-icon } (Сохранить).
10. Подтвердите ввод.

### Задать основные данные изделий для проверки в навигаторе основных данных изделий и выполнить контрольный прогон

1. Выберите пункты меню Сервисные программы > Изделие > Навигатор основных данных изделий.

!!! info "Для сведения:"

    В этом диалоговом окне вы получите обзор изделий, содержащихся в проверяемой базе данных изделий.

2. Выделите в структуре дерева нужную ветвь с основными данными изделий, которые необходимо проверить.
3. Создайте при необходимости фильтр с определенными критериями, чтобы далее ограничить выбор основных данных изделий.
4. Выберите пункты меню Данные проекта > Сообщения > Выполнить контр. прогон.
5. Выберите в диалоговом окне Выполнить контр. прогон из раскрывающегося списка Настройки вновь созданную схему контрольного прогона и нажмите ++OK++.

!!! info "Для сведения:"

    Выполняется проверка основных данных изделий, содержащихся в базе данных изделий.

!!! info "Для сведения:"

    Результат отображается в диалоговом окне управления сообщениями (пункты меню: Данные проекта > Сообщения > Управление). Неверные или неполные изделия можно идентифицировать там на основе выведенного номера изделия. Через пункт всплывающего меню Свойства можно перейти прямо в базу данных изделий к заявленному изделию.

!!! info "Для сведения:"

    Основные данные изделий с неверной или неполной информацией отмечаются в навигаторе основных данных изделий и в базе данных изделий красным восклицательным знаком. При помощи свойства Сообщения в управлении сообщениями можно настроить фильтр по полю, с помощью которого вы сможете отобразить как в навигаторе основных данных изделий, так и в базе данных изделий только изделия, отмеченные восклицательным знаком.
