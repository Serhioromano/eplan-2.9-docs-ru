# Реорганизовать проекты

При реорганизации проекта удаленные данные окончательно удаляются из базы данных проекта. При этом база данных проектов уменьшается.

Условие:

Вы выделили проект в навигаторе страниц или в диалоговом окне Управление проектами.

1. Выберите в навигаторе страниц пункты меню Проект > Организовать > Реорганизовать или в управлении проектами пункты меню ++Организовать++ > Реорганизовать.
2. Не устанавливайте в диалоговом окне Реорганизовать проект флажок Расширенный режим и нажмите ++Да++.

!!! info "Для сведения:"

    Выполняется реорганизация проекта.

### Корректировать структуру проектов

Для реорганизации вам предлагается расширенный режим. Этот режим позволяет автоматически корректировать превышения длины поля в базе данных проекта, а также проводить внутренние проверки идентификаторов сортировки структур проекта и при необходимости автоматически корректировать их.

Превышения длины поля могут возникнуть при вводе очень длинных идентификаторов, номеров и обозначений. Это происходит потому, что большинство полей базы данных проекта имеет ограничение по длине. Превышение длины поля может привести к усечению введенных данных. Это может привести к ошибочным результатам.

Условие:

Вы выделили проект в навигаторе страниц или в диалоговом окне Управление проектами.

1. Выберите в навигаторе страниц пункты меню Проект > Организовать > Реорганизовать или в управлении проектами пункты меню ++Организовать++ > Реорганизовать.
2. Установите в диалоговом окне Реорганизовать проект флажок Расширенный режим.
3. Щелкните по кнопке ++OK++.

!!! info "Для сведения:"

    Выполняется реорганизация проекта.

!!! info "Для сведения:"

    Дублирующиеся изделия в проекте удаляются. Соединения удаляются, а затем генерируются заново и оптимизируются.

!!! info "Для сведения:"

    Дополнительно база данных проекта подвергается проверке на превышение длины поля. Если обнаруживается превышение длины поля, база данных еще раз переструктурируется, так, чтобы длина каждого поля базы данных соответствовала длине введенных данных (плюс 10% резерв). После этого новая длина поля сохраняется в проекте.
