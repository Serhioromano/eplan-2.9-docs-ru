# Создание схемы контрольных прогонов для проверки проекта

Определите в схеме контрольных прогонов для проверки проекта, какие контрольные прогоны выполняет EPLAN.

Условия:

* [Выбрать базу данных управления проектами](prjmanagementgui_h_projekteverwalten.md)
* [Считать каталоги и проекты](prjmanagementgui_h_verzeichniseinlesen.md)

1. Проект > Управление
2. Выделите проект.
3. Выберите ++Дополнительно++ > Проверить проект.
4. Щелкните рядом с флажком Выполнить контрольные прогоны по ++...++.
5. Щелкните рядом с полем Схема по кнопке ![](images/all_new_as.png){: .ui-icon } (Создать).
6. В диалоговом окне Новая схема в поле Имя введите информативное имя для новой схемы и щелкните по кнопке ++OK++.
7. Выделите в столбце Класс класс сообщения.

!!! info "Для сведения:"

    В таблице, расположенной в правой части диалогового окна, выводятся возможные сообщения для этого класса.

8. Определите для каждого сообщения в столбце Категория в раскрывающемся списке, выводится ли сообщение как ошибка, предупреждение или замечание.
9. Определите в раскрывающемся списке в столбце Вид проверки, следует ли выполнить проверку как в режиме подключения, так и в автономном режиме (настройка "Подкл. / автоном.") или только в автономном режиме (настройка "Автономно") или же соответствующую проверку не нужно выполнять (настройка "Нет").
10. Щелкните по кнопке (Сохранить) и по ++OK++.
