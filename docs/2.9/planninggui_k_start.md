# Предварительное планирование

Благодаря предварительному планированию можно осуществить первые мероприятия по планированию технических аспектов процесса проектирования уже на ранних стадиях на платформе EPLAN. В процессе проектирования машины / установки на некоторых отдельных этапах, начиная с первого чернового наброска и идеи, происходит постепенное уточнение и конкретизация концепции, пока в конце концов не будут созданы все необходимые документы и информация для изготовления машины.

В ходе фазы предварительного планирования разрабатываются концепции для технического объема машин/установок и оцениваются первые количественные основы. Целью является нахождение технически выгодных концепций и определение стандартных значений для последующего подробного планирования. На основе предварительного планирования позднее может осуществляться создание схемы соединений и детализация установки.

Типичные задачи предварительного планирования:

* Определение и описание областей машины или установки для разделения на смысловые структуры и блоки.
* Создание первых графических схем в качестве общей основы планирования.
* Определение "черновых заполнителей" для функций и функциональных элементов, которые на данный момент еще не могут быть определены в деталях.
* Определение и оценка первых количественных аспектов (приводов / датчиков, входов и выходов ПЛК и т. д.).
* Создание первых спецификаций для поддержки расчетов и определения комплектующих с длительными сроками поставки.

В предварительном планировании осуществляется общее структурирование машины / установки. Основным диалоговым окном системы предварительного планирования на платформе EPLAN является навигатор предварительного планирования. В этом диалоговом окне вы создаете и обрабатываете соединения между машинами / устройствами. При этом используемая в предварительном планировании структура может не совпадать с позднейшей структурой проекта. Эта структура скорее представляет собой структуру изготовления, которая отражает последовательность/ход изготовления. Однако она может и соответствовать позднейшей (функциональной) структуре проекта. В последующем подробном планировании предварительное планирование служит при создании схемы соединений в качестве шаблона.

Возможно следующее:

* В навигаторе предварительного планирования можно отображать и обрабатывать определенные в проекте сегменты структуры и объекты планирования, а также создать новые сегменты.
* С помощью связей можно в дереве навигатора предварительного планирования указывать путь к уже имеющимся сегментам в других структурах.
* Сохраненные на сегментах в предварительном планировании данные можно выводить в отчетах и, тем самым, использовать, например, в спецификациях и при рассчете цен и затрат.
* Вы можете определить конкретное изделие или только шаблоны функций.
* Можно также создать макросы предварительного планирования, содержащие некую область структуры.
* Объекту планирования можно присвоить макрос. Такой макрос обычно содержит объекты-заполнители. В объекте заполнителе можно выбрать свойства объекта планирования.
* Из предварительного планирования с помощью мыши можно создать подробное планирование (схему соединений).
* На условном обозначении схемы соединений могут отображаться свойства связанного объекта планирования.
* На условном обозначении схемы соединений можно обрабатывать свойства связанного объекта планирования.
* Предварительное планирование на странице типа "Предварительное планирование" можно также выполнить графически.
* Через настройки проекта можно защитить функции, если эти функции связаны с объектами планирования.
* Вы можете задать пользовательские определения сегментов и свойства и тем самым адаптировать предварительное планирование к своим потребностям.
