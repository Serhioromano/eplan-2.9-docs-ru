# Диалоговое окно Обозначить соединения

Вы выделили соединения в графическом редакторе или в диалоговом окне навигатора Соединения — <Имя проекта>. Или вы выделили одну или несколько страниц либо проект в диалоговом окне навигатора Страницы — <Имя проекта>. Данные проекта > Соединения > Нумерация > Обозначить.

В этом диалоговом окне вы можете обозначить точки определения соединения, которые могут иметь обозначение соединения.

После вызова пункта меню сначала автоматически генерируются новые соединения.

Обзор основных элементов диалогового окна:

### Настройки

Выберите из раскрывающегося списка схему, которая определяет настройки для нумерации соединений. С помощью кнопки ++...++ откройте диалоговое окно [Настройки: Нумерация соединений](wirenumberinggui_d_verbnumeinstellungen.md). Здесь вы можете определять настройки для нумерации и сохранять их в схеме.

### Начальное значение / величина шага

В таблице приводятся все группы соединений, которые определены в текущих настройках и содержат в своем формате счетчик и / или подсчетчик. Для этих групп счетчик будет начинаться с заданного начального значения и вести счет с указанным вами шагом. Оба значения можно изменить в таблице.

Если для какой-либо группы соединений установлен флажок Группа, то счетчики для группы соединений объединяются. В этом случае принимается единое начальное значение и величина шага для всех счетчиков / подсчетчиков группы.
Если флажок снят, для каждого счетчика / подсчетчика можно установить отдельное начальное значение и величину шага.

Последовательность счетчиков задается через последовательность групп формата в диалоговом окне Настройки: Нумерация соединений (вкладка Обозначение) и последовательность счетчиков в группах.

!!! note "Замечание:"

    Допускается также величина шага, равна "0", в этом случае показания счетчика не изменяются. Следующее свободное значение определяется в соответствии с настройкой в поле Избегать одинаковых названий.

### Перезаписать

В данном раскрывающемся списке задаются точки определения соединений, для которых необходимо перезаписать обозначение.

* Все: Перезаписываются обозначения всех точек определения соединений, которые могут иметь обозначение соединения.
* Кроме "установленных вручную": Обозначения соединений, которые имеют свойство Установлено вручную, перезаписываться не будут. Обозначения всех остальных точек определения соединений, которые могут иметь обозначение, будут перезаписаны. При этом анализируется свойство Установлено вручную параметра ***Соединение***, а не из содержащихся в выборе точек определения соединений.
* Нет: Соединения, которые уже имеют обозначение, остаются без изменений и не обозначаются заново. Всем остальным соединениям, которые не имеют обозначения или обозначения которых содержат один или несколько символов "?", присваиваются новые обозначения.

При настройках Кроме 'установлено вручную' и Нет записываются обозначения соединения. Соединения, добавленные к имеющемуся объему, копируют обозначение соединений из объема. При наличии нескольких обозначений соединения применяется графически первое обозначение и другие перезаписываются. Таким образом, например, соединения с одинаковым сигналом при дополнительной нумерации содержат те же обозначения соединения.

Точки определения соединений тех соединений, которые завершены через одну из данных настроек нумерации соединений, копируют обозначение соединений находящегося ниже соединения.

!!! note "Замечание:"

    Если Вы размещаете точку определения соединения вручную и присваиваете ей свойство Установлено вручную, свойство Установлено вручную присваивается всему соединению. Тем не менее остальные свойства соединения, в том числе Обозначение соединения, присваиваются соединению от ***графически первой*** точки обозначения соединения.

### Избегать одинаковых названий

В данном раскрывающемся списке задайте, допускаются ли многократные обозначения. Доступны следующие настройки.

* Нет: Многократные обозначения допускаются везде.
* При выборе: Каждое соединение, содержащееся в выборе, получает уникальное обозначение соединения. Обозначения соединения уникальны только в пределах выбора, кроме того, в проекте допускаются многократные обозначения.
* Во всем проекте: Многократных обозначений стараются избежать во всем проекте, т. е. каждое соединение получает в пределах проекта уникальное обозначение соединения.
* На один счетчик области сброса (структура / страница): Многократных обозначений стараются избежать только в пределах области, определенной для сброса счетчика. Обозначения соединения уникальны только в счетчике сброса области, кроме того, в проекте допускаются многократные обозначения. Область сброса счетчика воздействует только на счетчик, но ***не ***на подсчетчик.

!!! example "Пример:"

    Выбрана настройка На один счетчик области сброса (структура/страница), и счетчик области сброса определен так, что счетчик при изменении страницы сбрасывается. Тогда обозначения соединения в пределах страницы уникальны. Допускаются аналогичные обозначения соединения на различных страницах.

Если многократных обозначений нет, в форматах счетчиками или подсчетчиками не задаются уже существующие значения, но счетчик или подсчетчик выполняет пошаговое увеличение номера (только не в том же, по выбору полученном диапазоне). При этом учитываются все выбранные обозначения соединений, а также обозначения, которые представлены в той же нумерации в предыдущих группах соединений.

Если формат содержит счетчик и также подсчетчик, счетчик увеличивает номера пошагово, если нижний счетчик находится еще на своем самом низком значении. В противном случае подсчетчик увеличивает номера пошагово.

!!! example "Пример:"

    В следующем примере для нумерации была выбрана группа соединений "Общие соединения" с объемом "Сигнал". Формат для обозначений соединения включает счетчик и подсчетчик, разделенные точкой. На соединениях междуH1иH2и междуH3иH2уже имеются обозначения соединения. Теперь следует пронумеровать соединения междуH4иH6и междуH5иH6. При этом выделяется весь фрагментсхемы соединений.Соединение междуH4иH6обозначается002.1. Поскольку обозначение001.1уже имеется и является наименьшим значение подсчетчика (1), счетчик пошагово увеличивает номер.Соединение междуH5иH6обозначается002.3. Поскольку обозначение002.2уже имеется, подсчетчик пошагово увеличивает номер.

В пределах диапазона допускаются идентичные обозначения, если формат определен соответствующим образом и выбран весь диапазон. При этом элементы формата не должны исходить от отдельного соединения.

!!! example "Пример:"

    В следующем примере для нумерации была выбрана группа соединений "Общие соединения" с объемом "Сигнал". Формат для обозначения соединения содержит номер страницы и подсчетчик (также с объемом "Сигнал"), разделенные дефисом. Теперь следует пронумеровать все соединения.Поскольку все соединения относятся к одному сигналу, они имеют одинаковые обозначения1-1.

### Область видимости

Выберите из этого раскрывающегося списка, какие соединения должны отображаться:

* Не изменять: Обозначения отображаются так, как они определены, т.е. "невидимые" обозначения соединений не отображаются. Это настройка по умолчанию.
* Показать все: При этой настройке также представляются видимыми обозначения, которые настроены на "невидимые".
* Один раз на страницу и объем: При этой настройке соответственно на графически первом соединении включается видимость обозначения соединения и выключается при всех других соединениях того же диапазона.

### Выделить как "установлено вручную"

Если установлен этот флажок, всем точкам определения соединений, для которых в процессе нумерации соединений было определено обозначение соединения, присваивается свойство Установлено вручную.
