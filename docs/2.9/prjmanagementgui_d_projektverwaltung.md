# Диалоговое окно Управление проектами

Вы выбрали базу данных управления проектами. Проект > Управление.

В это диалоговое окно считываются каталоги проекта для управления несколькими проектами. При этом каталоги проекта могут находиться в вашей системе или в сетевой среде. Вы можете использовать свойства проекта и их значения в качестве критериев фильтрации для нахождения определенных проектов. Вы можете также просматривать и обрабатывать свойства проекта. Для данных проекта, свойств проекта и информации для проекта в управлении проектами имеются режимы просмотра и обработки.

Обзор основных элементов диалогового окна:

### Фильтр

В этом раскрывающемся списке отображаются все доступные фильтры. Выбранный фильтр активируется автоматически. Запись "- Не активировано -" отключает фильтр и приводит к тому, что данные отображаются в неотфильтрованном виде. С помощью кнопки ++...++ откройте диалоговое окно [Фильтр](modaldialogsdb_d_filternnach.md). Здесь можно создать, обработать, удалить, скопировать, экспортировать, импортировать фильтр и управлять им.

Во всплывающем меню раскрывающегося списка Фильтр содержатся следующие записи:

* Выключить: Этот пункт меню доступен, если фильтр установлен: Сбрасывает настройку фильтра до записи "- Не активировано -".
* Активировать <Имя фильтра>: Этот пункт меню доступен, если для настройки фильтра установлено значение "- Не активировано -": Повторно активирует последний активный фильтр.

Таким образом можно быстро переключаться между неотфильтрованным и отфильтрованным в соответствии с требованиями пользователя представлениями.

### Значение: <Свойство>

При помощи [быстрого ввода](modaldialogsdb_k_filter.md) в данном поле для определенного и активированного фильтра можно быстро изменить значение его критерия.

### Собственная рабочая станция / внешняя рабочая станция / сервер

В этой древовидной структуре отображаются все считанные каталоги и содержащиеся в них проекты. Вы можете отфильтровать вид при помощи определенной вами схемы фильтра. Следующие пиктограммы отображают место нахождения считанных каталогов:

Пиктограмма |  Значение |  описание
---|---|---
![](images/prjmanagementgui_userstation_as.png){: .ui-icon } |  Собственная рабочая станция |  Каталоги находятся в собственной системе. Они были считаны с локальным путем в базу управления проектами, например С:\Программы\EPLAN\EPLAN\Enterprise\6756\Проекты.
![](images/prjmanagementgui_serverstation_as.png){: .ui-icon } |  Сервер |  Каталоги находятся на сервере. Они были считаны через общую папку сервера, например \\\Сервер\EPLAN\Совместные проекты.
![](images/prjmanagementgui_notownuserstation_as.png){: .ui-icon } |  Внешняя рабочая станция |  Каталоги находятся в системе третьего лица. Каталоги не включены. Они были считаны с локальным путем в базу управления проектами, например D:\Программы\EPLAN\EPLAN\Enterprise\4567\Проекты.

Считанные проекты представляются в алфавитном порядке. Открытые проекты выделяются полужирным шрифтом. [Расширения имени файла и пиктограммы для проектов](projects_k_icons.md) наглядно показывают, о каком типе проекта идет речь.

### Всплывающее меню

Пункт меню |  Значение
---|---
Убрать из управления проектами |  Убирает выделенные проекты или узлы (включая все подкаталоги и содержащиеся в них проекты) из управления проектами.

Если вы выделили проект, всплывающее меню содержит дополнительные пункты меню, доступные также через меню под кнопками [Организовать] или ++Дополнительно++.

### [Организовать]

Некоторые пункты этого меню доступны и во всплывающем меню структуры дерева.

Пункт меню |  Значение
---|---
Открыть |  Этот пункт меню доступен, если выделен как минимум один проект. Проект открывается в диалоговом окне Страницы — <Имя проекта>. Имя проекта выделяется в управлении проектами полужирным шрифтом. Если вы работаете в многопользовательском режиме, открытый таким образом проект могут обрабатывать сразу несколько пользователей.
Открыть (эксклюз.) |  Этот пункт меню доступен ***только во всплывающем меню***, если выделен как минимум один проект. Открывает проект в режиме обработки "Эксклюзивно" в диалоговом окне Страницы — <Имя проекта>. Имя проекта выделяется в управлении проектами полужирным шрифтом. Проект можете обрабатывать только вы. Если вы работаете в многопользовательском режиме, другие пользователи не имеют доступа к открытому таким образом проекту.
Открыть (с защитой от записи) |  Этот пункт меню доступен ***только во всплывающем меню***, если выделен как минимум один проект. Проект открывается с защитой от записи в диалоговом окне Страницы — <Имя проекта>. Имя проекта выделяется в управлении проектами полужирным шрифтом. Если вы работаете в многопользовательском режиме, открытый таким образом проект могут обрабатывать другие пользователи.
Закрыть |  Этот пункт меню доступен, если выделен как минимум один проект. Проект закрывается в диалоговом окне Страницы — <Имя проекта>. Выделение имени проекта отменяется в управлении проектами.
Переместить |  Этот пункт меню доступен, если выделен как минимум один проект. Открывает диалоговое окно Выбрать каталог. Позволяет один или несколько проектов переместить в другой каталог.
Переименовать |  Этот пункт меню доступен, если выделен проект. Позволяет дать проекту другое имя.
Копировать |  Этот пункт меню доступен, если выделен как минимум один проект. Открывает диалоговое окно Копировать проект. Позволяет копировать один или несколько проектов. Скопированный (последний) проект автоматически открывается в диалоговом окне Страницы — <Имя проекта>.
Удалить |  Этот пункт меню доступен, если выделен как минимум один проект. Удаляет один или несколько проектов и копирует его/их в корзину, как в Windows.
Создать |  Открывает диалоговое окно Создать проект. Позволяет создать новый проект.
Импортировать |  Позволяет импортировать проект EPLAN (*.epj). Такие проекты имеют формат XML. При импорте есть возможность указать настройки для импорта.
Экспортировать |  Этот пункт меню доступен, если выделен как минимум один проект. Открывает диалоговое окно Выбрать каталог, чтобы определить целевой каталог. Позволяет экспортировать проект EPLAN как файл epj в формат XML.
Сжать |  Этот пункт меню доступен, если выделен как минимум один проект. Открывает диалоговое окно Сжать проект. Позволяет сжать проект. При этом в схеме можно определить, в каком объеме должен быть сжат проект.
Реорганизовать |  Этот пункт меню доступен, если выделен как минимум один проект. Позволяет реорганизовать базу данных проекта. Удаленные данные убираются из базы данных проектов окончательно. При этом база данных проектов уменьшается.
Автоматизированная обработка |  Этот пункт меню доступен, если выделен как минимум один проект. Открывает диалоговое окно Выполнить: Автоматизированная обработка. Позволяет автоматически выполнять последовательность операций для одного или нескольких проектов.
Запаковать  |  Этот пункт меню доступен, если выделен как минимум один проект. Упаковывает один или несколько проектов в одном каталоге. Расширение имени файла *.elk заменяется на *.elp.
Резервировать |  Этот пункт меню доступен, если выделен как минимум один проект. Открывает диалоговое окно Резервировать проекты. Позволяет выполнить архивирование одного или нескольких проектов, выгрузить для внешней обработки или дополнительно сохранить проекты.
Восстановить |  Позволяет проекты, которые до этого были выгружены, архивированы или для которых было выполнено резервное копирование, вновь считать в базу данных проектов.
Отправить по эл. почте |  Этот пункт меню доступен, если выделен как минимум один проект. Сжимает проект. Позволяет переслать проект по электронной почте.
Распаковать |  Этот пункт меню доступен, если выделен, по крайней мере, один ***запакованный*** проект. Позволяет распаковать проект.

### [Дополнительно]

Это меню доступно в режиме Вид. Для смены режима Вид нажмите кнопку ++Сохранить++. Некоторые пункты этого меню доступны и во всплывающем меню структуры дерева.

Пункт меню |  Значение
---|---
Обновить данные |  Обновляет Вид. При этом учитываются данные, которые вы или другой пользователь сохранил в базе данных управления проектами. Обновляются также результаты контр. прогонов.
Считать каталог |  Позволяет каталоги и все имеющиеся в них проекты считывать в управление проектами. Каталог может быть считан только в том случае, если он содержит как минимум один проект EPLAN PPE.
Вывести список проектов |  Выводит список XML ProjList.xml проектов, считанных в управление проектами.
Обработать каталог |  Открывает выделенный каталог на носителе данных.
Считать информацию проекта |  Этот пункт меню доступен, если выделен как минимум один проект. Открывает диалоговое окно Выбрать файл информ. проекта. Позволяет считать файл XML с информацией проекта.
Обновить основные данные проекта |  Этот пункт меню доступен, если выделен как минимум один проект. EPLAN проверяет, являются ли основные данные системы более приоритетными, чем основные данные проекта. Если это так и основные данные системы и проекта совместимы, основные данные системы перезаписывают основные данные проекта.
Проверить проект |  Этот пункт меню доступен, если выделен как минимум один проект. Открывает диалоговое окно Проверить проект. Позволяет выполнить проверку проекта. Настройки и свойства проекта для сравнения сравниваются с настройками и свойствами проекта с замечаниями. Вы также выполняете целенаправленные контрольные прогоны по проекту с замечаниями.
Вывести информацию о проекте |  Этот пункт меню доступен, если выделен как минимум один проект. Открывает диалоговое окно [Вывести информацию о проекте](prjmanagementgui_d_projektinfoausgeben.md). Позволяет отобразить информацию о выделенном проекте в виде краткого или общего обзора на экране.
Показать пользователей |  Этот пункт меню доступен, если выделен как минимум один проект. Открывает диалоговое окно Показать пользователей. Показывает, какие пользователи обрабатывают в настоящее время выделенный проект.
Создать базовый проект |  Этот пункт меню доступен, если выделен как минимум один проект. Открывает диалоговое окно Создать базовый проект. Позволяет создать Базовый проект EPLAN (*.zw9). [Базовые проекты](projects_k_basisprojekteerweitert.md) — шаблоны, содержащие данные, на основе которых можно создавать новые проекты.
Создать шаблон проекта |  Этот пункт меню доступен, если выделен как минимум один проект. Открывает диалоговое окно Создать шаблон проекта. Позволяет создать Шаблон проекта EPLAN (*.ept). [Шаблоны проектов](projects_k_projektvorlagen.md) — шаблоны, содержащие данные, на основе которых можно создавать новые проекты.
Считать данные клиента > Автор |  Этот пункт меню доступен, если выделен проект. Открывает диалоговое окно Выбрать адрес. Позволяет считать данные клиента из базы данных изделий в свойства проекта. При этом считываются данные адреса группы управления "Производитель / поставщик" в группе свойств "Автор". Считанные данные отображаются на вкладке Свойства.
Считать данные клиента > Клиент |  Этот пункт меню доступен, если выделен проект. Открывает диалоговое окно Выбрать адрес. Позволяет считать данные клиента из базы данных изделий в свойства проекта. При этом считываются данные адреса группы управления "Клиент" в группе свойств "Клиент". Считанные данные отображаются на вкладке Свойства.
Считать данные клиента > Заказчик |  Этот пункт меню доступен, если выделен проект. Открывает диалоговое окно Выбрать адрес. Позволяет считать данные клиента из базы данных изделий в свойства проекта. При этом считываются данные адреса группы управления "Клиент" в группе свойств "Заказчик". Считанные данные отображаются на вкладке Свойства.

!!! tip "Совет:"

    Если в дереве управления проектами выделить каталог, то все проекты, находящиеся в этом каталоге, считаются выбранными. В таком случае, например, доступно большинство пунктов меню, вызываемых с помощью кнопок [Организовать] и [Дополнительно]. Поэтому все проекты в этом каталоге можно обрабатывать совместно (например, Автоматизированная обработка).

### [Обработать]

Для переключения в ***режим обработки*** нажмите эту кнопку. В режиме обработки вы можете обрабатывать содержание на вкладках Свойства и Структура.

### [Сохранить]

Эта кнопка служит для сохранения в базе данных управления проектами изменений и обновлений во вкладках Свойства, Статистика, Структура и Статус в базе данных управления проектами. Управление проектами переключается в ***режим Вид***.
