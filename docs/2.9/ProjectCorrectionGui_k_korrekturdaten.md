# Корректируемые данные проекта

Для клемм могут быть выполнены следующие виды корректировки:

Корректировка |  Значение
---|---
Поменять число целей при неверных клеммах |  Если этот флажок установлен, корректируются клеммы, выводы которых представляют различное число целей и при которых фактическое число целей не соответствует числу, связанному с логической схемой выводов устройства. При этом в логической схеме выводов устройства данной клеммы настройки "Число целей" вывода 1 и вывода 2 меняются местами, если фактическое число целей вывода 1 соответствует настройке "Число целей" вывода 2 и наоборот.
Добавить отсутствующие определения клеммников |  Если активирован этот флажок, то для клеммников без предварительного определения создаются неразмещенные определения клеммников. Клеммники без предварительного определения появляются, например, если Вы, при вставке клемм, указываете ОУ, еще не существующее как определение клеммника, или Вы удаляете определение клеммника, для которого еще имеются клеммы.
Разрешить одинаковые обозначения |  Если этот флажок установлен, то одинаковые обозначения клемм, неоднократно присутствующие внутри одного клеммника, не будут рассматриваться как ошибки. У соответствующих функций будет установлено свойство Допускается одинаковое обозначение.
Корректировать созданные вручную мостовые перемычки |  Если этот флажок установлен, корректируются настройки для вручную создаваемых мостовых перемычек, которые были спроектированы в предыдущих версиях EPLAN (версия 2.4 или старше). Такие мостовые перемычки могут попадать в проект, например, при вставке макроса. Начиная с версии 2.5 настройки для вручную созданных мостовых перемычек сохраняются только на той клемме, которая представляет начало перемычки.

Для штекеров может быть выполнена следующая корректировка:

Корректировка |  Значение
---|---
Добавить отсутствующие определения штекеров |  Если активирован этот флажок, то для штекеров без предварительного определения создаются неразмещенные определения штекеров. При этом, в зависимости от контактов, имеющихся в штекере (то есть штыри, гнезда или их комбинации), создаются подходящие определения штекеров. В этом случае при необходимости генерируются два определения штекера на каждое ОУ. Штекеры без предварительного определения появляются, например, если вы, при вставке контактов штекера, указываете ОУ, еще не существующее как определение штекера, или вы удаляете определение штекера, для которого еще имеются контакты.

Для кабелей могут быть выполнены следующие виды корректировки:

Корректировка |  Значение
---|---
Перенести содержимое поля примечаний кабеля и экранирований в функциональный текст |  Если этот флажок установлен, для линий определения кабеля и экранирования содержимое свойства Примечание переносится в свойство Функциональный текст. При этом свойство просмотра "Примечание" применяется и к свойству "Функциональный текст". Это служит для корректировки старых проектов, в которых вместо функциональных текстов использовались и оценивались примечания.

!!! note "Замечание:"

    Содержимое поля примечания будет перенесено в функциональный текст только в том случае, если поле последнего пусто.

Убрать пробелы из единицы измерения поперечного сечения кабелей, экранирований и соединений |  Если этот флажок установлен, убираются пробелы перед и после единицы измерения поперечного сечения в поле Поперечное сечение / диаметр. Пробелы при единице измерения поперечного сечения могут мешать при поиске и использовании фильтров, так как, например, значения "1,5", " 1,5" и "1,5 " представляются не аналогичными.

!!! tip "Совет:"

    При помощи свойства Кабель / Группа соединений: Поперечное сечение / Диаметр с единицей измерения (Ид. 20070) единицу измерения поперечного сечения и единицу измерения можно вывести вместе. В настройках проекта определите, вставляется ли при этом пробел между значением и единицей измерения. В диалоговом окне Настройки: Кабели (доступно через пункты меню Параметры > Настройки > Проекты > "Имя проекта" > Устройство > Кабели) установите флажок Поперечные сечения с пробелом.

Для соединений могут быть выполнены следующие виды корректировки:

Корректировка |  Значение
---|---
Корректировать символы соединения за пределами определенной сети (активировать "Задать цель") |  Если этот флажок установлен, то для всех символов соединений (тройников, перекрестного соединения, перемычек), которые уже находятся за пределами определенной сети, вновь устанавливается флажок Задать цели. К примеру, эта функция может потребоваться при определении сетей для сетевых соединений, если точки определения сети были удалены вручную, а о перенастройке соответствующих символов соединений забыли.

!!! tip "Совет:"

    Для удаления из проекта всех точек определения сети можно воспользоваться сжатием проектов. Для этого в диалоговом окне Настройки: Сжатие на уровне иерархии Точки определения сети установите флажок Удалить все точки определения сети и корректировать символы соединения. При этом все соответствующие символы соединений перенастраиваются автоматически (устанавливается флажок Задать цели).

Для выводов устройств ПЛК могут быть выполнены следующие виды корректировки:

Корректировка |  Значение
---|---
Записать обратно обозначения выводов устройства |  Если этот флажок установлен, для всех блоков управления ПЛК, имеющихся в проекте, обозначения выводов устройства ПЛК на страницах обзора будут перенесены в соответствующие выводы устройства ПЛК на страницах схем соединений. Для этого необходимо, чтобы соответствующие выводы устройства ПЛК были присоединены к клеммам или контактам штекера. Соответствующие выводы устройства ПЛК обнаруживаются с помощью адреса ПЛК.
Выполнить обратную запись обозначений вывода устройства и перенести на подключенные контакты штекера / клеммы |

!!! note "Замечание:"

    Данная корректировка целесообразна, только если выводы устройства ПЛК и контакты штекера / клеммы графически выполнены отдельно. Контакты штекера и клеммы должны находиться внутри блока ПЛК и иметь то же ОУ, что и соответствующий вывод устройства ПЛК.

 Если этот флажок установлен, для всех блоков управления ПЛК, имеющихся в проекте, обозначения выводов устройства ПЛК на страницах обзора будут перенесены в соответствующие выводы устройства ПЛК на страницах схем соединений. Обозначения вывода устройства дополнительно переносятся на те контакты штекера и клеммы на схеме соединений, которые напрямую связаны с выводами устройства ПЛК. При этом обозначение штекера вывода устройства ПЛК переносится в видимое ОУ присоединенного контакта штекера / присоединенной клеммы, а обозначение вывода устройства ПЛК переносится в обозначение контакта штекера / обозначение клеммы.
Установить типы данных |  Если этот флажок установлен, на основе идентификатора для выводов устройства ПЛК автоматически устанавливается тип данных, если соответствующее поле не заполнено. Имеющиеся значения не перезаписываются.
Синхронизировать имена интерфейсов шины |  Если этот флажок установлен, имя интерфейса шины переносится на все виды представления того же вывода шины. Это значит, что значение свойства Интерфейс шины: Имя для выводов шины, которое введено в виде представления, переносится на все другие виды представления. Если на разных видах представлений находятся разные имена интерфейсов шины, значения остаются неизменными и имя интерфейса шины не переносится на другие виды представления.
Синхронизировать имена физических сетей |  Если этот флажок установлен, для выводов шины значение свойства Физическая сеть: Имя переносится с главного шины логической сети на остальные выводы шины (подчиненные) внутри той же логической сети. Если на выводах шины уже имеются записи для имени физической сети, они перезаписываются со значением соответствующего главного шины. Поиск соответствующих подчиненных шины выполняется по свойствам Логическая сеть: Имя и Проект конфигураций; соответственно оба значения должны совпадать для главного и подчиненного шины.

!!! tip "Совет:"

    Для безошибочного экспорта структур сети все выводы шины физической сети должны отображаться в однополюсном или многополюсном представлении, и они должны быть соединены между собой. С помощью контрольных прогонов 004103 и 004104 можно проверить, содержат ли главный и подчиненный шины одинаковое значение свойства Физическая сеть: Имя, и все ли выводы шины физической сети связаны между собой.

Удалить лишний проект конфигураций на выводах шины |  Если этот флажок установлен, проект конфигураций, введенный на выводах шины, удаляется, если он соответствует проекту конфигураций связанного блока ПЛК. Тогда проект конфигураций вывода шины будет определен связанным блоком ПЛК и автоматически обновится в случае изменений.
