# Вкладка Вывод устройства ПЛК

Вы открыли проект.

* В Графическом редакторе выделен вывод устройства ПЛК или символ канала. Обработать > Свойства. Выберите в диалоговом окне Свойства ++...++ вкладку Вывод устройства ПЛК.
* Вы выделили вывод устройства ПЛК или символ канала в Навигаторе устройств или в одном из Навигаторов ПЛК.Пункт всплывающего меню Свойства или Свойства (общие). Выберите в диалоговом окне Свойства ++...++ вкладку Вывод устройства ПЛК.

На этой вкладке указываются свойства выделенных выводах устройства ПЛК. Помимо свойств, которые имеют все функции, здесь также представлены другие специфические для ПЛК свойства.

В режиме обработки ***Свойства (общие)*** вы обрабатываете свойства для всех представлений выводов устройства ПЛК сразу; в этом случае появляются дополнительные вкладки Блок ПЛК (ОУ), Данные структуры ПЛК (ОУ) и Изделие (ОУ) для соответствующей главной функции.

Обзор основных элементов диалогового окна:

### Видимое ОУ

Видимое ОУ будет показано в графическом редакторе рядом с условным обозначением. Если вы определили в настройках проекта, что устройство должно автоматически нумероваться при вставке символов, здесь вам будет автоматически предложено ОУ. Вы можете сохранить или переписать вручную это ОУ.

Кроме того, можно использовать [всплывающее меню или кнопку ++...++](devicetaggui_r_feldsichtbaresbmk.md), чтобы cгенерировать или выбрать ОУ.

### Полное ОУ

В этом поле отображается полное ОУ (т. е. структурный идентификатор текущего окружения + видимое ОУ) видимого ОУ / функционального элемента. С помощью кнопки ++...++ откройте диалоговое окно Полное ОУ. Здесь можно обработать структурный идентификатор и, например, выбрать другое место установки.

### Обозначение вывода устройства

В этом поле отображаются обозначения выводов устройства текущей функции, а также обозначения выводов устройств тех функций, в которые копируется ОУ текущей функции. Чтобы различать одинаковые обозначения выводов устройств на одной карте ПЛК, необходимо дополнительно указать ОУ штекера в поле Обозначение штекера. Идентифицирующим для распознавания является ОУ карты ПЛК, обозначение штекера (для выводов шины: имя интерфейса шины + обозначение штекера) и обозначение вывода устройства.

Обозначения выводов устройств для функции или для устройства можно изменить вручную или выбрать запись из раскрывающегося списка. Несколько обозначений выводов устройств разделяются знаком абзаца. При обработке нескольких обозначений выводов устройств вручную необходимо учитывать следующее:

* Отдельные обозначения выводов устройств можно изменить вводом значения на соответствующей позиции между знаками абзаца.
* При перезаписывании всей записи одним отдельным значением это значение будет присвоено первому выводу устройства. Остальные обозначения выводов устройств останутся без изменения.
* При удалении всей записи удаляются все обозначения выводов устройств.

!!! note "Замечание:"

    Содержимое данного поля не сохраняется в отдельном свойстве. Чтобы отобразить обозначения выводов устройств на условном обозначении, можно воспользоваться свойством Обозначения выводов устройства (все) (ID 20038). Это свойство отображает все обозначения выводов устройств текущей функции. Однако это свойство не содержит отображаемые в поле Обозначение вывода устройства обозначения выводов устройств других функций.

### Описание вывода устройства

Введите здесь описание выводов устройств для текущей функции или устройства или выберите эти данные из раскрывающегося списка.

### Обозначение штекера

Введите здесь обозначение штекера (например, -X1), на котором находится вывод устройства или канал на карте ПЛК. Обозначение штекера можно, как и в случае с клеммами, скопировать слева (или сверху). В таблице свойств существует свойство Обозначение штекера (автоматически), в котором содержится автоматически полученное значение.

Если автоматическое присвоение обозначения штекера нежелательно для определенных выводов устройства ПЛК, его можно предотвратить с помощью свойства Обозначение штекера (автоматически): отменить поиск (ид. 20579).

Штекер карты ПЛК не отображается в Навигаторе как самостоятельный объект. Речь идет только об информации у выводов устройства или каналов ПЛК, которые учитываются при идентификации отдельного вывода устройства ПЛК или при образовании перекрестных ссылок.

При идентификации вывода устройства для выводов шины вместо обозначения штекера используется комбинация имени интерфейса шины и обозначения штекера. Свойство Интерфейс шины: Имя (с обозначением штекера) содержит полученное значение.

!!! note "Замечание:"

    * При обработке свойства Обозначение штекера в режиме обработки ***Свойства (общие)*** содержимое передается обратно в те представления, в которых обозначение штекера было введено первоначально.
    * Если предварительно не ввести здесь "-" (т. е., например, "X1"), ОУ кабеля все равно отобразит этот символ (например, "-W(-A1-X1/-S1)").

### Функциональный текст

Укажите здесь функциональный текст для функции / устройства.

Независимо от введенного здесь вручную функционального текста существуют другие (определяемые автоматически) функциональные тексты, которые находятся в таблице:

* Функциональный текст (автоматически): Отображает содержание введенного вручную функционального текста или, при его отсутствии, функциональный текст зоны. Если на выводе устройства ПЛК не будет найден функциональный текст зоны, на экране в соответствии с [отчетом функциональных текстов зоны](gededitgui_k_pfadtextauswertung.md) отображается свойство Функциональный текст (автоматически) связанного сенсора или исполнительного элемента. При отображении этого свойства на странице проекта разрывы строк удаляются.
* Функциональный текст (общий): отображает (автоматический) функциональный текст функции или, при его отсутствии, (автоматический) функциональный текст соответствующей главной функции.

!!! note "Замечание:"

    При обработке свойства Функциональный текст в режиме обработки ***Свойства (общие)*** содержимое передается обратно в те представления, в которых функциональный текст был введен первоначально.

### Адрес

Адрес может быть введен вручную или задан автоматически. Он не является идентифицирующим для вывода устройства ПЛК. В рамках одного ЦПУ адреса должны задаваться однозначно. ЦПУ однозначно идентифицируется путем ввода имени ЦПУ в формате [Проект конфигураций].[Имя станции].[Идентификатор ЦПУ].

При помощи кнопки ![](images/all_rebook_as.png){: .ui-icon } (Возврат) возможен возврат адресов. Это возможно только в тех случаях, когда адрес не указан ни в каких других представлениях вывода устройства (однополюсный, многополюсный и т. д.). В этом случае адрес заносится в актуальный список присвоений как свободный адрес, при этом сохраняются свойства Функциональный текст (автоматически) и Символический адрес (автоматически). Функциональные тексты и / или символические адреса, заданные вручную, удаляются. Если данный адрес указан в другом представлении вывода устройства, возврат его будет невозможен.

В режиме обработки ***Свойства (общие)*** возврат адреса осуществляется на всех выводах устройства одновременно. Пробелы в адресе при этом ***не*** учитываются, то есть адреса "E0.0" и "E 0.0" считаются идентичными друг другу.

### Символический адрес

Введите здесь символический адрес для вывода устройства ПЛК.

Независимо от введенного здесь символического адреса существуют другие (определяемые автоматически) свойства, которые находятся в таблице свойств в категории "Данные":

* Символический адрес (автоматический): Отображает содержание введенного вручную символического адреса или, при его отсутствии, определенный символический адрес.
* Символический адрес (определен): При помощи отслеживания цели ведется поиск подсоединенного датчика (при входе) или исполнительного элемента (при выходе), а его ОУ автоматически вводится как символический адрес. Если датчик и исполнительный элемент не найдены, вводится ОУ последней подсоединенной функции, на которой был прерван поиск. В случае определенного символического адреса при поиске датчика / исполнительного элемента анализируется информация логической схемы вывода устройства, а именно с какого вывода устройства функции необходимо продолжить поиск.

!!! note "Замечание:"

    При обработке свойства Символический адрес в режиме обработки ***Свойства (общие)*** содержимое передается обратно в те представления, в которых символический адрес был введен первоначально.

### Обозначение канала

Обозначение канала можно вносить вручную или задавать автоматически. Внутри карты ПЛК канал должен быть однозначным. Для точек подвода питания назначение, обычно, выполняется графически. Обозначение канала необходимо вносить вручную только в том случае, если невозможно графическое присвоение. В таблице свойств есть свойство Обозначение канала (автоматич.), которое содержит автоматически определенное обозначение.

Если автоматическое присвоение обозначения канала нежелательно для определенных выводов устройства ПЛК, его можно предотвратить с помощью свойства Обозначение канала (автоматически): отменить поиск (ид. 20578).

У точек подвода питания дополнительные поля данных для выводов устройства ПЛК не используются и не заполняются автоматически.

!!! note "Замечание:"

    При обработке свойства Обозначение канала в режиме обработки ***Свойства (общие)*** содержимое передается обратно в те представления, в которых обозначение канала было введено первоначально.

### Определение функции

В этом поле выводится текущее выбранное определение функции; щелкните по кнопке ++...++, чтобы при необходимости выбрать другое определение функции в диалоговом окне Определения функций.

Определения функций для вывода устройства ПЛК подразделяются на различные группы:

* Вывод устройства ПЛК, источник питания: Сортируется по типу потенциалов, для электропитания датчиков и исполнительных элементов.
* Вывод устройства ПЛК, В / В: Для аналоговых и дискретных вводов и выводов.
* Вывод устройства ПЛК, питание модуля: Сортируется по типу потенциалов, для электропитания карт ПЛК.
* Вывод устройства ПЛК, сетевой кабель / коммуникационная шина: Для вывода устройства абонентов шины или устройств в сети.
* Вывод устройства ПЛК, перемен: Для выводов устройства ПЛК с переменным числом выводов устройства.

!!! example "Пример:"

    На плате для вывода реле на одном контакте расположены два вывода устройства ПЛК. Первый вывод определен как выход с определением функции "Вывод устройства ПЛК, дискр. выход". Второму выводу присвоен тип потенциала "Неопределенный" с определением функции "Вывод устройства ПЛК, Источник питания (не определен)".

[Групповое поле "Свойства"](devicetaggui_r_gruppenfeldeigenschaften.md)

Для выводов устройства ПЛК доступны, в частности, следующие свойства:

* Тип данных: Введите тип данных вручную или выберите значение из раскрывающегося списка. В зависимости от установленного типа ПЛК в списке выводятся различные типы данных. Изменение установленного в проекте типа ПЛК не приводит к изменению типа данных, установленного для адреса.
* Проект конфигураций (косвен.): Выводит проект конфигураций для блока ПЛК на выводе устройства ПЛК.
* Имя рабочей станции ПЛК (косвен.): Выводит имя рабочей станции для соответствующего блока ПЛК на выводе устройства ПЛК.
* ЦПУ (косвен.): Выводит на выводе устройства ПЛК имя ЦПУ соответствующего блока ПЛК.
