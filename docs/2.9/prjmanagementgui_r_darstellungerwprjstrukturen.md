# Вкладка Представление (Расширенные структуры проекта)

* Вы открыли проект. Страница > Навигатор. Выделите проект. Выберите пункт всплывающего меню Свойства. Выберите вкладку Структура. Нажмите кнопку ++Другие++. Выберите в диалоговом окне Расширенные структуры проекта  вкладку Представление.
* Вы выбрали базу данных управления проектами. Вы считали каталоги и проекты. Проект > Управление. Выделите проект. Выберите вкладку Структура. Нажмите кнопку ++Другие++. Выберите в диалоговом окне Расширенные структуры проекта  вкладку Представление.

На этой вкладке установите предшествующий знак для структурных идентификаторов и разделитель для подыдентификаторов структуры проекта.

Обзор основных элементов диалогового окна:

Групповое поле Блоки идентификаторов

### Функциональное присвоение

В этих полях находятся предшествующий знак и разделитель для "Функционального присвоения". Поля доступны, если вы определили структурный идентификатор в пределах структуры проекта как минимум один раз как описывающий или идентифицирующий. В следующей таблице показаны возможные символы.

Предшествующий знак |  Разделитель для подидентификатора
---|---
***==***  |  ***==***
***#***  |  ***.***
***≠***  |

### Установка

В этих полях находятся предшествующий знак и разделитель для "Установки". Поля доступны, если вы определили структурный идентификатор в пределах структуры проекта как минимум один раз как описывающий или идентифицирующий. В следующей таблице показаны возможные символы.

Предшествующий знак |  Разделитель для подидентификатора

|  ***.***

### Место сборки

В этих полях находятся предшествующий знак и разделитель для "Место сборки". Поля доступны, если вы определили структурный идентификатор в пределах структуры проекта как минимум один раз как описывающий или идентифицирующий. В следующей таблице показаны возможные символы.

### Место установки

В этих полях находятся предшествующий знак и разделитель для "Место установки". Поля доступны, если вы определили структурный идентификатор в пределах структуры проекта как минимум один раз как описывающий или идентифицирующий. В следующей таблице показаны возможные символы.

Предшествующий знак |  Разделитель для подидентификатора

### Номер установки

В этом поле введите предшествующий знак для "Номера установки". При вводе недопустимых символов (пробелов и т. д.) отображается соответствующая всплывающая подсказка.

### Вид документа

В этом поле введите предшествующий знак для "Вид документа". По умолчанию установлен предшествующий знак "***&***". Вместо знака "&" в EPLAN можно выбрать также пробел.

### Опред. пользователем

В этом поле введите предшествующий знак и разделитель для произвольного, определяемого пользователем структурного идентификатора. При вводе недопустимых символов (пробелов и т. д.) отображается соответствующая всплывающая подсказка.

### Разделить также между цифрой и буквой

Если этот флажок установлен, то подструктуры разделяются при переходе от цифры к букве. Т. е. EPLAN интерпретирует замену цифры на букву внутри структурного идентификатора как подструктуру.

Если флажок снят, подструктуры будут представлены с помощью заданных разделителей для подыдентификаторов.

!!! example "Пример:"

    Если флажок установлен, идентификатор=A1.A2.A3отображается и рассматривается как=A1A2A3.

* * *

Групповое поле Маркировка продукции

!!! note "Замечание:"

    Настройки для маркировки продукции доступны только, если в свойствах проекта на вкладке Структура установлен флажок Расширенное обозначение ссылки.

Укажите в этом групповом поле, следует ли использовать маркировку продукции в качестве элемента структурирования и как должны быть представлены подструктуры маркировки продукции. Мы рекомендуем для оптимальной поддержки программы снабдить все устройства идентифицирующей маркировкой продукции ('-'). Дополнительные описательные критерии структурирования (аспекты) могут при необходимости улучшить структурирование. Путем структурирования проектов при помощи маркировки продукции можно полностью структурировать проектируемые машины по механической конструкции.

### Использовать для структурирования

Если этот флажок установлен, маркировка продукции — т. е. блок идентификаторов для устройства (предшествующий знак '-') используется в качестве дополнительного элемента структурирования. С помощью этой настройки достигается следующее:

* Предшествующий знак '-' отображается в полном ОУ всегда, т. е. даже в тех случаях, когда никакие другие блоки идентификаторов не используются.
* В диалоговом окне Полное ОУ таблица для ОУ скрывается, и маркировка продукции отображается в таблице Структурный идентификатор.
* Маркировка продукции отображается в управлении структурными идентификаторами.
* Маркировка продукции вкладывается также на точках разрыва.
* Вы можете использовать маркировку продукции для сегментов структуры в предварительном планировании. Для объектов планирования можно использовать маркировку продукции, даже если флажок снят.

Если флажок снят, маркировка продукции не используется в качестве элемента структурирования.

### Предшествующие знаки и разделители

В этих полях стоят предшествующие знаки и разделители для 'Маркировки продукции'. Поля доступны, если установлен флажок Использовать для структурирования. В следующей таблице показаны возможные символы.

Предшествующий знак |  Разделитель для подидентификатора
---|---
***-***  |  ***-***
|  ***.***

!!! note "Замечание:"

    Если в качестве разделителя для маркировки продукции выбрана точка, некоторые функции платформы EPLAN нельзя использовать в полном объеме, поскольку они несовместимы с использованием точки в качестве разделителя. При этом речь идет в том числе о функциях, использующих подсчетчики, и об обозначениях Fluid-техники, разделенных точкой.

### Разделить также между цифрой и буквой

Если этот флажок установлен, то подструктуры маркировки продукции разделяются при переходе от цифры к букве. Т. е. EPLAN интерпретирует замену цифры на букву внутри структурного идентификатора как подструктуру.

Если флажок снят, подструктуры будут представлены с помощью заданных разделителей для подыдентификаторов.

!!! example "Пример:"

    Если этот флажок установлен, ОУ-K01Q04S01внутренне раскладывается на подструктуры-K01,Q04иS01. При снятом флажке это ОУ отображается и рассматривается как-K01.Q04.S01.
