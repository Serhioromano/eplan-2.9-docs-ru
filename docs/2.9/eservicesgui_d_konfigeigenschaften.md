# Диалоговое окно "Конфигурировать свойства" (определенные пользователем свойства)

* Вы открыли проект. Данные проекта > Конфигурировать свойства.
* Сервисные программы > Изделие > Управление. Через меню ++Дополнительно++ > Настройки была открыта база данных изделий. [Дополнительно] > Конфигурировать свойства.

Создайте в этом диалоговом окне определенные пользователем свойства для проекта или изделия. Специфические для проекта свойства сохраняются в проекте и могут использоваться на различных объектах в проекте. Определенные пользователем свойства изделия сохраняются в каждой базе данных изделий и могут использоваться на изделиях. Эти определенные пользователем свойства можно присвоить изделию в базе данных изделий, используя вкладку Определенные пользователем свойства.

Обзор основных элементов диалогового окна:

В левой части диалогового окна определенные пользователем свойства в текущем проекте / в текущей базе данных изделий отображаются в представлении в виде дерева и представлении в виде списка.

!!! note "Замечание:"

    После присвоения определенных пользователем свойств одному или нескольким изделиям и выполнения синхронизации данных изделий свойства изделий также отображаются в конфигурации проекта. Такие определенные пользователем свойства можно распознать по невозможности их обработки в проекте и по записи "Изделие" в поле Присвоение.

### Фильтр

Введите в это поле текст, по которому должна выполняться фильтрация свойств. При этом поиск свойств выполняется как по идентифицирующему, так и по отображаемому имени.

Если необходимо вернуться к прежнему виду свойств, удалите текст из поля ввода. Для этого нажмите кнопку ![](images/all_delete_as.png){: .ui-icon } (Удалить).

### Отображать только "Больше не использовать"

Если этот флажок установлен, в представлении в виде дерева и представлении в виде списка отображаются только свойства, обозначенные как более не используемые. Для этих свойств установлен флажок Больше не использовать.

Кнопка |  Значение
---|---
![](images/all_new_as.png){: .ui-icon } (Создать) |  Открывает диалоговое окно Новое свойство, в котором можно создать свойство.
![](images/all_import_as.png){: .ui-icon } (Импортировать) |  Позволяет импортировать определенные пользователем свойства из файла формата XML.
![](images/all_export_as.png){: .ui-icon } (Экспортировать) |  Дает возможность экспортировать выделенные определенные пользователем свойства в файл формата XML. Свойства, обозначенные как более не используемые, не экспортируются.

Нажмите над полем на кнопку ![](images/all_new_as.png){: .ui-icon } (Создать), чтобы создать свойство. Затем в диалоговом окне Новое свойство введите однозначное идентифицирующее имя свойства. Рекомендуем начинать имя с идентификатора вашей фирмы. Идентифицирующее имя может содержать прописные и строчные буквы, цифры и специальные символы "., -, _, &". В имени должна быть как минимум одна точка, однако она не должна стоять в начале или конце имени. Нельзя использовать строку символов "EPLAN" в начале имени. Точки в имени расцениваются как подструктура и генерируют подслой в дереве.

!!! note "Замечание:"

    * Идентифицирующее имя должно быть однозначным в пределах одного проекта и не может быть изменено в дальнейшем.
    * При активированной настройке Отображать идентифицирующие номера в интерфейсе пользователя отображается идентифицирующее имя. Это соответствует отображению номеров свойств (ид.) в свойствах EPLAN.

### Всплывающее меню

Дополнительно здесь представлены следующие пункты всплывающего меню, специфические для данного диалогового окна:

Пункт меню |  Значение
---|---
Вставить в список результатов поиска |  Это возможно ***только*** при конфигурации проекта. Проверяет использование выделенных свойств в проекте и вносит найденные объекты в список результатов поиска. Благодаря этому с помощью пункта всплывающего меню Перейти к (графика) можно найти используемые свойства объектов, например условные обозначения, размещения изделий и т. д., в графике.

### Идентифицирующее имя

Здесь отображается однозначное имя, установленное при создании свойства. Свойством под этим именем можно управлять в EPLAN.

### Отображаемое имя

Здесь вводится имя, под которым будет отображаться свойство. Возможен многоязычный ввод.

!!! example "Пример:"

    Вы создаете свойство с идентифицирующим именем Идентификатор Фирмы.Мои Свойства.Свойство1и отображаемым именем Свойство 1. В структуре дерева в качестве верхнего узла отображается Идентификатор фирмы. Под ним находится узел Мои Свойства, а под ним — узел Свойство1. В проекте это свойство отображается под именем Свойство 1.

### Описание

Здесь вы можете ввести описание, которое будет отображаться для свойства в диалоговых окнах выбора. Возможен многоязычный ввод.

### Присвоение

Через раскрывающийся список можно задать, для каких объектов должно быть доступно это свойство. Присвоение можно задать ***только*** при создании свойства, позднее изменить его уже невозможно.

* Точка разрыва: Свойство доступно в точках разрыва.
* Ссылка изделия: Свойство доступно для ссылок изделий и таким образом отображается в диалоговом окне свойств главных функций во вкладке Изделие. Дополнительно автоматически на всех объектах, которые могут нести изделие, генерируется соответствующее индексированное свойство.
* Функция: Свойство доступно на любых функциях во всех разделах.
* Проект: Свойство доступно в свойствах проекта.
* Страница: Свойство доступно на страницах.
* Соединение: Свойство доступно на соединениях, в точках определения соединения, точках определения потенциала и точках определения сети.
* Предварительное планирование: Свойство можно использовать на сегментах в предварительном планировании, а также при создании определений сегмента.
* Изделие: свойство доступно на изделиях. Это присвоение возможно ***только*** при конфигурации в базе данных изделий.

### Также активировать для ссылок изделий

Установите этот флажок, если свойство должно быть доступно также как свойство ссылки изделия. Это возможно ***только*** при конфигурации в базе данных изделий.

### Тип поля

С помощью раскрывающегося списка задайте символы и формат, допустимые для определенных пользователем свойств. Например, возможны следующие типы полей: "Булево" (истина / ложь), "Десятичное число", "Многоязычный текст". Настройку можно задать ***только*** при создании свойства, позднее изменить ее уже невозможно.

!!! note "Замечание:"

    Если для определенного пользователем свойства необходимо ввести значения с единицами измерения, выберите тип поля "Значение с единицей измерения". Этот тип поля имеет свойство "одноязычного текста", за исключением изложенных ниже ситуаций.

* Если в качестве критерия фильтра для определенного пользователем свойства при фильтрации указывается значение с единицей измерения (например, "10 к В"), то в результатах поиска также отображаются свойства, для которых это значение было указано в другой единице измерения (например, "1000 В").
* Для значений с десятичными разрядами при представлении свойств в графическом редакторе отображаются не введенные разделители разрядов ("," или "."), а разделители разрядов, указанные для операционной системы.

### Перевести значение свойства

Доступно ***только*** для типа поля "Многоязычный текст". Установите этот флажок, если значение определенного пользователем свойства следует учитывать при автоматическом переводе проекта или автоматическом переводе в базе данных изделий.

### Помощь ввода

Выберите из раскрывающегося списка, какой вид ввода возможен для свойства:

* Свободный ввод: Можно вручную вводить любые значения, например текст, числа или числа с единицами измерения.
* Список выбора: Значения можно выбирать только из раскрывающегося списка. Возможные записи задаются в раскрывающемся списке в поле Список выбора значений по умолчанию.
* Свободный ввод со списком вариантов выбора: любые значения можно ввести вручную или выбрать из раскрывающегося списка. Возможные записи задаются в раскрывающемся списке в поле Список выбора значений по умолчанию.
* Саморасширяющийся список выбора: Значения можно выбрать из раскрывающегося списка или ввести вручную. Возможные записи задаются в раскрывающемся списке в поле Список выбора значений по умолчанию. Введенные вручную значения расширяют значения по умолчанию в раскрывающемся списке.

### Категория

Для просмотра в программе свойства подразделены на различные категории, т.е. на группы свойств, которые могут отображаться независимо друг от друга. Из раскрывающегося списка выберите категорию, которой вы хотите присвоить свойство.

### Список выбора значений по умолчанию

Если в поле Помощь ввода выбран один из параметров "Список выбора", "Свободный ввод со списком вариантов выбора" или "Саморасширяющийся список выбора", определите здесь возможные записи в раскрывающемся списке.

Нажмите на кнопку ![](images/all_new_as.png){: .ui-icon } (Создать) для создания новой строки в списке, после чего введите нужное значение. Возможен многоязычный ввод.
С помощью кнопки ![](images/all_delete_as.png){: .ui-icon } (Удалить) можно удалить ненужное значение.

### Больше не использовать

Если этот флажок установлен, свойство больше не отображается в проекте / на вкладке Определенные пользователем свойства базы данных изделий. С помощью пунктов меню ++Дополнительно++ > Убрать больше не используемые свойства эти свойства можно удалить.

!!! note "Замечание:"

    В проекте определенные пользователем свойства, которые используются в изделиях или других основных данных, невозможно удалить, пока в столбце Количество использований не отобразится значение "0", даже если параметр "Больше не использовать" включен. Для этого сначала необходимо отменить использование (убрать выбранные изделия, формы и т. д.) и удалить неиспользуемые изделия, формы и т. д. путем сжатия проекта. Только после этого соответствующие свойства можно удалить с помощью пункта меню Убрать больше не используемые свойства или повторного сжатия.
