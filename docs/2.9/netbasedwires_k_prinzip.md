# Сетевые соединения: Принцип

При помощи сетевых соединений можно проектировать схему соединений, не определяя сразу расположения соединений. Для этого можно определить внутри проекта сети, содержащие большое количество соединений. Последовательность соединений в сети не определяется (при использовании символов соединений без определения цели) символами подключения в схеме соединений, а должна устанавливаться позже в диалоговом окне "Свойства" соответствующей точки определения сети.

### Определение сетевых соединений

Сетевые соединения определяются через точку определения сети. В точке определения сети сохраняется, какие выводы устройств сети связаны друг с другом. Выводы устройств при этом идентифицируются через ид. вывода устройств, который уникален внутри сети. Ид. задаются автоматически, но их можно изменить вручную. При копировании сети эти ид. копируются и соединения, сохраненные у точки определения сети, остаются.

Соединения, созданные в точке определения сети, не сразу отображаются в навигаторе соединений. Соединения создаются из информации, сохраненной в точке определения сети, только при обновлении соединений, а затем отображаются и в навигаторе соединений.

Если вы обладаете лицензией на "Сетевую монтажную схему", то в диалоговом окне "Свойства" символа соединений с возможностью разветвления (тройники, перемычки, перекрестные соединения) отображается флажок Определение целей. Если этот флажок ***деактивирован***, для соответствующего символа соединений последовательность целей не используется и символ соединений обозначается как точка. В этом случае соединения не генерируются автоматически, но должны задаваться вручную.

Если флажок Определение целей установлен, то EPLAN создает соединения автоматически.

### Сетевые функции

Сеть заканчивается на функции, если она имеет разделение потенциалов, разделение сигналов или разделение сети. Стандартно большинство функций прерывается сетью. С помощью свойства Соединенный с сетью можно установить, чтобы сеть в данной функции продолжалась. Выводы устройств потенциалов всегда соединяются с сетью.

#### Клеммы и контакты штекеров в сети

Иногда требуется дальше провести сеть через клемму или контакт штекера. Соединения обоих выводов устройств клемм/контактов штекера должны являться частью той же сети. Это достигается тем, что свойство Соединенный с сетью присваивается клемме или контакту штекера.

!!! example "Пример:"

    Если клемме-X100присвоено свойство Соединенный с сетью, то лампы-H10и-H20находятся в той же сети и могут быть связаны друг с другом посредством сетевого соединения.

### Определения кабеля

Соединения могут определяться как кабельные соединения только через линию определения кабеля, если соединение не содержит символов соединения, для которых отключено определение цели.

!!! example "Пример:"

    Только соединение между-X1:3и-X1:4можно определить как кабельное соединение через линию определения кабеля-W2. Соединение в линии определения кабеля-W1не распознается как кабельное соединение, даже тогда, когда соединение междуX1:2иX1:1определено через точку определения сети.

### Составление отчетов по сетевым соединениям

Сетевые соединения можно выводить в отчетах в отсортированном виде (по сети и шлейфам). Для этого используются следующие свойства:

* Индекс сети: Идентифицирует сеть для сетевых соединений.
* Индекс шлейфа: Идентифицирует шлейф для сетевых соединений в пределах сети.
* Индекс соединения: Идентифицирует соединения для сетевых соединений в пределах шлейфа.

Свойство Тип управления может использоваться в качестве критерия фильтра в отчетах, чтобы выводить только соединения со значением 'Сетевые'.
