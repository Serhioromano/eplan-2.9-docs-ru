# Вкладка Общее

* Параметры > Настройки > Фирма > Сборка проводов > 'Машина' > Общее
* Сервисные программы > Данные изготовления > Сборка проводов > "Машина". Щелкните по кнопке ++...++ рядом с полем Схема. Выберите вкладку Общ.

В этой вкладке определяется путь файла и имя файла для экспорта данных провода на машины по сборке проводов.

Обзор основных элементов диалогового окна:

### Путь файла

Выберите каталог, в котором необходимо сохранить файлы экспорта. Нажав ++...++, откройте диалоговое окно Выбрать каталог и перейдите к нужному каталогу или создайте новый. С помощью пункта всплывающего меню Вставить переменную пути перейдите в диалоговое окно [Выбрать переменную пути](modaldialogsdb_d_pfadvariablen.md), из которого можно будет скопировать одну из доступных переменных пути.

### Имя файла

Вы можете присвоить имя сгенерированному файлу или, нажав ++...++, выбрать его в диалоговом окне Сохранить как. Расширение имени файла зависит от выбранного формата машины. Если в результате экспорта генерируются несколько файлов с различными расширениями имени файла, введенное здесь имя будет применяться ко всем файлам.

Некоторые имена файлов и / или расширения имен файлов устанавливаются в зависимости от производителя. В этом случае настроить можно только путь файла.

### Набор символов (только для формата "Общая спецификация проводов")

Данная настройка позволяет расширить набор символов для сгенерированного текстового файла, например, чтобы иметь возможность выводить обозначения выводов устройств со специальными символами или на восточноевропейских или азиатских языках. В раскрывающемся списке доступны следующие форматы:

* ANSI: Символы сохраняются в кодировке ANSI. Эту кодировку следует использовать, если вы продолжаете обрабатывать сгенерированный файл спецификации в приложениях Windows.
* ASCII: Символы сохраняются в кодировке ASCII. Эту кодировку следует использовать, если вы продолжаете обрабатывать сгенерированный файл спецификации в приложениях MS-DOS.
* UNICODE: Символы сохраняются в кодировке Юникод. При этом этот файл спецификации не зависит от платформы и операционной системы.

!!! note "Замечание:"

    Перед применением расширенного набора символов для экспорта общих спецификаций проводов необходимо проверить, смогут ли обработать этот набор конечные программы машин и принтеров. При возникновении проблем следует вернуться к использованию стандартной настройки "ANSI".
