# Диалоговое окно Настройки: Общие (проект, соединения)

Вы открыли проект. Параметры > Настройки > Проекты > "Имя проекта" > Соединения > Общее.

В этом диалоговом окне можно задать общие настройки соединений.

Обзор основных элементов диалогового окна:

### Источник и цель соединений определить из размещения

Если установлен этот флажок, то определение источника и цели соединений выполняется через ***графическую позицию*** подключенных функций. Эта настройка соответствует действиям в EPLAN версии 1.9 SP 1. В отчетах источники и цели соединений сортируются согласно их размещению в схеме соединений.

Если флажок не установлен, при определении источника и цели соединений сравниваются ***обозначения устройства*** подключенных функций. Для сравнения используется сортировка структурных идентификаторов в разделе управления структурными идентификаторами, в качестве исходного элемента применяется "меньшее" обозначение устройства.

!!! note "Замечание:"

    При данной настройке соединения кабелей не изменяются.

### Управление выводами мостовых перемычек в точке определения сети

Если данный флажок установлен, в диалоговом окне свойств точки определения сети на вкладке Соединения сети также отображаются ***Выводы устройства символов*** с типом вывода устройства 'Мостовая перемычка'. Дополнительные ***Выводы устройства функции*** для мостовых перемычек не отображаются.

Если флажок снят, выводы устройств мостовых перемычек на вкладке Соединения сети не отображаются.

!!! note "Замечание:"

    Сетевые соединения невозможно использовать вместе с оптимизацией перемычек распределенных клемм. Обе функции генерируют различные соединения, несовместимые друг с другом.

### Управление всеми соединениями сети в точке определения сети

Если флажок установлен, то для непосредственно соединенных сетевых функций (т. е. у функции активировано свойство Соединенный с сетью) цели не определяются автоматически. В этом случае соединения не генерируются автоматически, но должны задаваться вручную. Т. е. все соединения внутри сети (как соединения через символы соединения без определения цели, так и прямые соединения между сетевыми функциями) определяются и сохраняются в точке определения сети.

Если флажок снят, то для непосредственно соединенных сетевых функций цели определяются автоматически и автоматически генерируются соединения. Это настройка по умолчанию. Данные соединения отображаются в диалоговом окне Остальные соединения, но их невозможно удалить или изменить.

!!! note "Замечание:"

    * Если между сетевыми функциями находится символ соединения с определением цели (например, [Тройник](egedgui_d_tstueck.md)), то в каждом случае соединения определяются автоматически.
    * При удалении точки определения сети сетевые соединения становятся неопределенными.

Если установлены оба флажка (Управление выводами мостовых перемычек в точке определения сети и Управление всеми соединениями сети в точке определения сети), то все выводы устройства символов сети определяются через точку определения сети и заново соединяются проводами.

### Отменить неразмещенные сегменты маршрутизации топологии в трассе маршрутизации

Неразмещенные сегменты маршрутизации топологии появляются, например, при [генерировании функций топологии](cablinggui_h_strukturenverbinden.md) в точках маршрутизации структурного идентификатора.

Если этот флажок установлен, неразмещенные сегменты маршрутизации топологии скрываются в свойстве Топология: Трасса маршрутизации (ид. 20237) и, таким образом, не выводятся в отчетах.

### Вывести точки маршрутизации топологии с длиной в трассе маршрутизации

Если этот флажок установлен, точки маршрутизации топологии и точки маршрутизации структурного идентификатора топологии, на которых введена длина, выводятся вместе в свойстве Топология: Трасса маршрутизации (ид. 20237).

Если дополнительно активирована настройка Отменить неразмещенные сегменты маршрутизации топологии в трассе маршрутизации, в трассе маршрутизации выводятся только ***размещенные*** точки маршрутизации топологии и точки маршрутизации структурного идентификатора топологии.

Если флажок снят, в трассе маршрутизации выводятся только сегменты маршрутизации топологии.
