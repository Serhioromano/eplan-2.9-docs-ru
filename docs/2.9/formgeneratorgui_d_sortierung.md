В этом диалоговом окне вы создаете, обрабатываете схемы сортировки для отчетов и управляете ими. В схемах сортировки для отчетов задаются критерии сортировки, которые определяют, как сортируются данные проекта или изделий в отчетах.

Вы открыли проект. Сервисные программы > Отчеты > Генерировать. Щелкните по кнопке ![](images/all_new_as.png){: .ui-icon } (Создать). В диалоговом окне Определить отчет выделите [тип отчета](formgeneratorgui_k_auswertungstypen.md).Щелкните по кнопке ++OK++. Щелкните в диалоговом окне Настройки – <Тип отчета> рядом с полем Новая страница при изменении значения свойства по ++...++.

В этом диалоговом окне вы создаете, обрабатываете и управляете схемами разбивки на страницы для отчетов. В схемах разбивки на страницы задаются критерии разбиения страниц, которые определяют, когда в страницах отчета вставляется разбивка на страницы.

Вы открыли проект. Вы открыли навигатор спецификаций, отметили там несколько позиций и выбрали пункт всплывающего меню Нумеровать позиции. Щелкните в диалоговом окне Нумеровать позиции рядом с полем Новая нумерация при измененном свойстве по ++...++.

В этом диалоговом окне вы создаете, обрабатываете схемы нумерации для назначения номеров позиций и управляете ими. По такой схеме номера позиций могут назначаться с учетом структуры или страницы. Для этого необходимо задать критерии, которые будут определять начало новой нумерации с начального значения.

Обзор основных элементов диалогового окна:

### Схема

Выберите из раскрывающегося списка схему, которую требуется использовать или обработать. Для этого также воспользуйтесь соседней [панелью инструментов](schemeconfig_m_schemaschaltflaechen.md).

### Категория / свойство

В зависимости от диалогового окна определите в этом поле критерии сортировки, разбиения страниц или нумерации.

Чтобы задать ***критерии разбиения страниц***, щелкните по кнопке ![](images/all_new_as.png){: .ui-icon } (Создать) и последовательно выберите в диалоговом окне требуемые свойства.

!!! example "Пример:"

    Выберите последовательно свойства:Установка ОУ: буквенное обозначение Место установки.Сохраните схему. Если вы сейчас генерируете отчет, данные проекта или изделий выдаются отсортированными сначала по установке, затем по ОУ — Буквенное обозначение, и наконец, по месту установки.

!!! note "Замечание:"

    Обратите внимание, что для не относящихся к функции отчетов спецификации изделий, групповой спецификации изделий и списка обозначений устройств доступно свойство Определение функции (ID 20026) как критерий сортировки, и функции можно сортировать в алфавитном порядке.

Чтобы задать ***критерии разбиения страниц***, щелкните по ![](images/all_new_as.png){: .ui-icon } (Создать) и последовательно выберите в диалоговом окне требуемые свойства. Всегда, когда изменяется значение настройки выбранного свойства, при генерировании отчета создается новая страница.

!!! example "Пример:"

    Требуется сгенерировать список обозначений устройств. Ваш проект содержит для блока идентификаторов 'Установка' два структурных идентификатора:EB1FB1.Если вы сейчас для разбивки на страницы выберете свойство Установка, при генерации страниц отчета после последнегоустройства, присвоенного установкеEB1, будет вставлена разбивка на страницы. Первое устройство, присвоенное установкеFB1, выводится на новой странице отчетов, так какустановкаизменилась.

Чтобы задать ***критерии нумерации для назначения номеров позиций страниц***, щелкните по кнопке ![](images/all_new_as.png){: .ui-icon } (Создать) и последовательно выберите в диалоговом окне требуемые свойства.

!!! example "Пример:"

    В качестве критерия выберите свойство Установка. Сохраните схему. Если пронумеровать позиции сейчас, нумерация снова начнется с начального значения, как толькоструктурный идентификатордля установки будет изменен.
