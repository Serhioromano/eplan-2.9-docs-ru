# Определение искомых понятий

Искомые понятия в EPLAN ищут на следующих страницах:

* во всех объектах, расположенных на страницах;
* во всех пространствах листа размещаемых объектов;
* в объектах, размещенных и неразмещенных в свойствах;
* в данных страницы (за исключением имени страницы);
* в данных пространства листа (за исключением имени пространства листа);
* в данных проекта (за исключением имени проекта).

Поиск можно выполнять выборочно для всего проекта или только для определенных страниц. Кроме того, можно определить, в каких данных проекта следует искать нужное понятие.

Результаты поиска записываются в список результатов.

В качестве искомых понятий допустимы любые цепочки символов, которые также могут содержать [заполнители](searchandreplacegui_k_platzhalter.md) (напр., "*" и "?"). Также можно искать несколько взаимосвязанных понятий. Но поиск нескольких несвязанных понятий производить нельзя.

!!! example "Пример:"

    Если выполняется поиск "Управление электрошкафом", также будут найдены только тексты, в которых отдельные слова следуют в правильной последовательности. Понятия "Управление электрошкафом" или "Управление в электрошкафе" не будут найдены.

### Учет структурных элементов при поиске

Если в диалоговом окне Поиск в групповом поле Поиск в установлен флажок ОУ/обозначение, искомое понятие ищется во всем ОУ. Структурные элементы при этом не учитываются.

Только если в качестве поискового признака вводится МО вместе со своей структурой, структура учитывается. При этом слева от МО должна указываться всегда полная подструктура структуры проекта. Кроме того, нужно указывать и предшествующие знаки (=, + и т. п.). В элементах структуры заполнители не разрешены.

!!! example "Пример:"

    Допустим,структура проектасостоит из функционального присвоения, установки, места сборки и места установки.Допустимые искомые понятия, напр.:==AA=A++OO+O-K1, или=A++OO+O-K1, или++OO+O-K1, или+O-K1,но не=A-K1(неполная структураустройства, отсутствует указание места сборки и установки)или=A++O*+O-K1(заполнитель "*" не допускается).

В качестве результата поиска на просмотр выводится "видимое ОУ". С помощью функции Заменить можно изменять только видимые ОУ и только его элементы без структурных элементов. Если ОУ не выводится на просмотр на условном обозначении, то есть существует [Перенос ОУ](devicetaggui_k_bmkuebernahme.md), ничего не заменяется.

!!! note "Замечание:"

    При переносе ОУ видимое ОУ является пустым; поэтому в данном случае поиск ОУ не даст результата.
