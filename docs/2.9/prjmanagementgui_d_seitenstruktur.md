# Диалоговое окно Структура страницы

* Вы открыли проект. Страница > Навигатор. Выделите проект. Выберите пункт всплывающего меню Свойства. Выберите вкладку Структура. Рядом с полем Страницы щелкните ++...++.
* Вы выбрали базу данных управления проектами. Вы считали каталоги и проекты. Проект > Управление. Выделите проект. Выберите вкладку Структура. Рядом с полем Страницы щелкните ++...++.

В этом диалоговом окне вы можете создавать, обрабатывать схемы идентификаторов для маркировки и нумерации страниц и управлять схемами.

Обзор основных элементов диалогового окна:

### Схема

В этом поле находится имя текущей схемы идентификаторов.

Выберите из раскрывающегося списка схему, которую требуется использовать или обработать. Для этого также воспользуйтесь соседней [панелью инструментов](schemeconfig_m_schemaschaltflaechen.md).

### Идентификатор объекта

Идентификатор объекта требуется для структурирования страниц согласно стандарту DIN EN 61355-1. При этом через идентификатор объекта определенному объекту (например, функции, месту или продукту) присваивается документ (т.е. страница проекта).

!!! note "Замечание:"

    * Чтобы использовать идентификатор объекта для структурирования, в свойствах проекта должно быть также активировано расширенное обозначение ссылки (Флажок Расширенное обозначение ссылки на вкладке Структура).
    * Идентификатор объекта используется ***только*** вместе с блоком идентификаторов Вид документа. Комбинация идентификаторов объектов и других блоков идентификаторов невозможна. Если для использования идентификатора объекта установлен флажок Использовать для структурирования, таблица с блоками идентификаторов также становится недоступной (включая вид документа), а последовательность идентификаторов больше не может быть изменена.

При выставлении флажка Использовать для структурирования идентификатор объекта используется для структурирования проекта. Поведение при вводе и генерировании отчета по структуре страницы затем меняется, как описано в разделе "[Идентификация страниц согласно стандарту DIN EN 61355-1](projectstructure_k_objektkennzeichen.md)".

Если флажок снят, поведение при вводе и генерировании отчета по структуре страницы соответствует поведению в предыдущих версиях EPLAN (версия 2.3 или более ранние). В этом случае использование идентификатора объекта в качестве элемента структурирования недоступно.

### Таблица

В таблице установите, какие блоки идентификаторов являются идентифицирующими, описывающими или недоступными для структуры страницы.

Блоки идентификаторов отображаются в соответствии с заданной в таблице последовательностью в дереве навигатора страниц, в свойствах страницы и т. д. Используя кнопки со стрелками, можно изменять ***последовательность*** блоков идентификаторов.

* Структура: Здесь перечислены блоки идентификаторов, соответствующие структуре страницы. В соседнем столбце отображаются соответствующие предшествующие знаки.
* Значение: В этом столбце определите в раскрывающемся списке, является ли блок идентификаторов идентифицирующим, описывающим или он не должен быть доступен.
