# Расширения для обмена данными ПЛК в формате AutomationML

В этой версии реализованы следующие расширения для обмена данными ПЛК в формате AutomationML:
– Поддержка программ конфигурации ПЛК Rockwell Automation Studio 5000 и Siemens TIA Selection Tool
– Экспорт значений конфигурации, связанных с устройствами
– Поддержка текущей версии AutomationML 1.1.0.
Кроме того, реализованы улучшения при импорте данных ПЛК, благодаря чему обмен данными ПЛК теперь стал стабильнее.

### Поддержка других программ конфигурации ПЛК

Обмен данными ПЛК в формате AutomationML теперь также возможен со следующими программами конфигурации ПЛК:

* Rockwell Automation Studio 5000
* Siemens TIA Selection Tool

Для этого в диалоговом окне Экспортировать данные ПЛК в раскрывающемся списке поля Программа конфигурации ПЛК доступны новые записи.

### Экспорт значений конфигурации, связанных с устройствами

При обмене данными ПЛК в формате AutomationML теперь в файле экспорта можно также выводить значения конфигурации, связанные с устройствами. Для этого в диалоговом окне Настройки: Экспорт AutomationML доступен новый флажок Экспортировать значения конфигурации, связанные с устройствами. Для использования этой функции требуется дополнительная лицензия.

Значения конфигурации, связанные с устройствами, определяются в базе данных изделий как определенные пользователем свойства через диалоговое окно [Конфигурировать свойства](eservicesgui_d_konfigeigenschaften.md). Чтобы при обмене данными ПЛК эти свойства учитывались в формате AutomationML, в поле Категория необходимо ввести значение "Данные ПЛК", а в поле Присвоение — значение "Изделие" и установить флажок Также активировать для ссылок изделий. Префикс (строка символов до первой точки) в идентифицирующем имени этих свойств удаляется при экспорте и дополняется из данных соответствующего изделия при импорте.

### Другие улучшения

* В системах шин на базе Ethernet обмен данными ПЛК теперь ведет себя более "терпимо" в отношении обозначения штекера. Данные для обозначения штекера с или без "R" (например, "P1" и "P1R") рассматриваются при импорте как эквивалентные. В частности, это облегчает обмен данными ПЛК с Siemens TIA Portal V16.
* При присвоении изделий теперь в системах шин на базе Ethernet дополнительно учитывается количество выводов шины, если изделие было найдено многократно. Если в базе данных изделий есть два варианта изделия с разным количеством выводов шины, то при импорте посредством определения различий по количеству выводов шины для изделия можно выбрать подходящий вариант.

### Поддержка текущей версии AutomationML

Данный выпуск EPLAN поддерживает текущую версию AutomationML 1.1.0. Для этого в диалоговом окне Экспортировать данные ПЛК в поле Формат файла экспорта доступна запись "AutomationML V1.1.0".

В отличие от экспорта в формате "AutomationML V1.0.0" здесь, в частности, появились такие улучшения:

* Благодаря расширениям в файле AML теперь можно обмениваться более длинными обозначениями устройств.
* Расширения в списке присвоений улучшают обмен свободными символическими адресами.
* Теперь осуществляется обмен присвоениями карт ПЛК соответствующим ЦПУ. Это позволяет восстанавливать принадлежность карт ПЛК соответствующим ЦПУ при обмене данными ПЛК.
* Для свойства Система шин в раскрывающемся списке доступна новая запись PortToPort. Эта система шин может использоваться, если между выводами шины (подробное соединение портов) необходимо экспортировать только соединение, а не логическую сеть.
* Системы шин "DRIVE-CLiQ", "PortToPort" и "ET-Connection" теперь корректно экспортируются с подробным соединением портов.
* Обмен соединениями между каркасами осуществляется с подробным соединением портов.
* Теперь в системах шин осуществляется обмен адресами (отказоуст.).
* На устройствах можно поменять имя производителя. В EPLAN имя производителя сохраняется в данных ссылки изделия.
