# Компоновка электрошкафа 2D: Основы

Монтажные платы можно начертить в вертикальном и горизонтальном формате. Они снабжаются обозначением устройства, причем в качестве основы для стандартных функциональных элементов используется структура проекта. В графическом редакторе монтажная плата отображается с помощью специального черного ящика.

При оснащении монтажной платы несколькими выбранными изделиями можно указать выравнивание размещаемых ссылок изделий, причем выбранное выравнивание влияет на то, как будут размещены ссылки изделий по отношению друг к другу. Кроме того, можно настроить и выравнивание маркировок размещений изделий, данная настройка реализуется с помощью варианта угла размещения символа, а в случае размещенного изделия она может быть изменена для отдельных размещении свойств. При этом следует обратить внимание на то, что размещения изделий могут быть установлены только ***на*** монтажной плате; если они находится за ее пределами, выводятся соответствующие сообщения.

Размеры для ширины, высоты, глубины и интервала установки являются свойствами размещения изделия и сохраняются в базе данных изделий или в макросе изделия. Используя настройку в диалоговом окне Настройки: Компоновка электрошкафа 2D (всплывающее меню > Настройки) можно указать, каким образом должны копироваться эти данные.

С помощью свойств Функция доступна с видом представления "..." можно создавать фильтры, уменьшающие отображение. При фильтрации размещения изделия на основании такого свойства программа выполняет поиск функции с нужным видом представления, которой присвоено изделие. При установке фильтра, при котором эти свойства проверяются на настройку "Нет" (т. е. флажок снят), в диалоговом окне отображаются только такие функции / изделия, которые существуют только на монтажной плате.

!!! note "Замечание:"

    Обратите внимание, что в двухмерном навигаторе компоновки электрошкафа 2D (диалоговое окно Компоновка электрошкафа 2D — <Имя проекта>) фильтрация выполняется не по данным изделия, а по Данные ссылки изделия. Если, например, выполняется фильтрация по свойству Монтажная поверхность (ид. 22022), фильтр обращается к соответствующему свойству данных ссылок изделия (ид. 20918). Это свойство может отличаться от свойства изделия (ид. 22022), так как монтажная поверхность изделий основных данных и несохраненного изделия не обязательно должна быть идентичной монтажной поверхности ссылки изделия.

Кроме того, для каждой монтажной платы можно указать, должна ли быть сгенерирована ***Легенда страницы***, а если да, то какую форму необходимо использовать для этого. Если легенда не активирована, на символ изделия записывается полное ОУ. В ином случае все размещения изделий снабжаются порядковыми номерами, а соответствующая легенда может быть произвольно размещена рядом с монтажной платой.

!!! example "Пример:"

    В диалоговом окне Компоновка электрошкафа 2D - <Имя проекта> выводятся все ссылки изделий, имеющиеся в проекте. Во вкладке Дерево можно выбрать между двумя конфигурациями отображения (Всплывающее меню > Вид), в котором отображаются следующие данные (при их наличии):

* Структура проекта: Отображение статуса изделия, блоков идентификаторов (установка, место установки и т. д.), обозначения устройства и номера изделия.
* Структура проекта с идентификатором: Отображение имени монтажной платы (включая структурный идентификатор) ***или*** статуса изделия, блоков идентификаторов (установка, место установки и т. д.), обозначения устройства и номера изделия.

Статус изделия может иметь следующие значения:

Статус изделия |  Пояснение
---|---
Все изделия |  На этом слое древовидной структуры отображаются все изделия. Через пиктограмму (зеленый флажок) у номера изделия наглядно отображается, размещено ли уже изделие или нет. На один слой выше при помощи галочки и обозначения устройства подается сигнал, что все изделия устройства размещены.
Размещ. неверно |  Уже размещенный функциональный узел будет перемещен за пределы монтажной платы, после чего будет обновлен навигатор компоновки электрошкафа (например, посредством [F5]).
При "стандартном" размещении этот статус не генерируется, поскольку функциональный узел нельзя устанавливать таким образом. В случае ошибки размещение не выполняется.

Устройства сортируются на основании структурного идентификатора устройства. Устройства, которые по структурным идентификаторам не относятся ни к одной структуре проекта, сортируются в уровне структуры дерева 'Без структурных идентификаторов'. Отображение этого слоя структуры дерева возможно только в проектах со структурой идентификаторов (например, структура идентификаторов IEC), то есть не в проектах с общей нумерацией. Устройства, не имеющие ОУ, располагаются ниже уровня древовидной структуры 'Без структурных идентификаторов' на уровне 'Без ОУ'. Это действие аналогично представлению устройств без ОУ в других диалоговых окнах навигатора.

На вкладке Список эти данные не выводятся в иерархическом порядке, а объем отображаемых данных можно настроить с помощью пунктов меню Всплывающее меню > Конфигурировать представление.
