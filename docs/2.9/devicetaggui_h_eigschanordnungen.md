# Использовать определенный пользователем порядок свойств

Порядок свойств присваивает функции порядок размещенных текстов свойств и их настройки отображения. Настройки для образа контакта также сохраняются в порядке свойств.

Определенный пользователем порядок свойств сохраняется как специфический для проекта и присваивается варианту символа, для которого он был создан. Поэтому такой порядок свойств доступен только для функций с одинаковым вариантом символа. Для размещения изделия в пространстве листа порядки свойств присваиваются определению функции и, соответственно, доступны для размещения изделия с одинаковым определением функции.

Определенные пользователем порядки свойств можно создавать на условных обозначениях или непосредственно на 3D-размещении изделия либо при конфигурации в диалоговом окне конфигурации.

С помощью диалогового окна конфигурации [Конфигурировать порядки свойств](eservicesgui_d_konfigeigenschaftsanordnungen.md) выводится обзор всех порядков свойств, имеющихся в проекте. Это диалоговое окно позволяет экспортировать порядки свойств нескольких выбранных размещений изделий или условных обозначений и импортировать их в другой проект. Кроме того, можно проверять использование определенных пользователем порядков свойств и удалять неиспользованные порядки свойств.

Условие:

Вы открыли проект.

1. Выделите условное обозначение в графическом редакторе или функцию в навигаторе данных проекта. Или выделите размещение изделия в пространстве листа.
2. Выберите пункты меню Обработать > Свойства.
3. Выберите в диалоговом окне Свойства ++...++ вкладку Отображение.
4. Выберите из списка Порядок свойств запись "Опред. пользователем" и обработайте порядок свойств.
5. Используйте кнопки ![](images/all_new_as.png){: .ui-icon } (Создать) и ![](images/all_delete_as.png){: .ui-icon } (Удалить) для выбора дополнительных свойств или удаления выделенных свойств из списка.
6. Используйте кнопки ![](images/all_cut_as.png){: .ui-icon } (Вырезать) или ![](images/all_copy_as.png){: .ui-icon } (Копировать), чтобы вырезать или скопировать выделенные в списке свойства с их заданными свойствами отображения. В другом условном обозначении эти свойства вставляются с помощью кнопки ![](images/all_paste_as.png){: .ui-icon } (Вставить) под выделенным свойством в списке другого порядка свойств. Свойства, которые невозможно использовать на условном обозначении, не учитываются при вставке. (После этой операции порядок свойств на целевом условном обозначении устанавливается на "Определено пользователем".)
7. В списке свойств выделите свойство, настройки отображения которого вы желаете определить. При этом заданное значение в отдельных полях копируется из графы "Стандартное значение по умолчанию".
8. Обработайте свойства отображения в таблице [Свойство / Присвоение](devicetaggui_r_anzeigeeigenschaften.md).
9. Щелкните по кнопке ![](images/all_save_as.png){: .ui-icon } (Сохранить).
10. В диалоговом окне [Сохранить порядок свойств](devicetaggui_d_eigschanordnungspeichern.md) введите имя для порядка свойств или выберите имеющееся имя из раскрывающегося списка.
11. Установите флажок Использовать как стандарт, чтобы задать порядок свойств в качестве стандартного для всех дальнейших применений этого варианта символа (для функций) / этого определения функции (для размещений изделий).
12. Подтвердите ввод.

!!! info "Для сведения:"

    Порядок свойств сохраняется и присваивается варианту символа. При размещении изделий порядок свойств присваивается определению функции.

!!! info "Для сведения:"

    Если порядок свойств задан как стандартный, то он используется для соответствующего варианта символа по умолчанию, например при вставке символа. При размещении изделий порядок свойств используется по умолчанию для соответствующего определения функции.

!!! tip "Совет:"

    С помощью кнопок ![](images/all_export_as.png){: .ui-icon } (Экспортировать) и ![](images/all_import_as.png){: .ui-icon } (Импортировать) можно экспортировать сохраненный определенный пользователем порядок свойств варианта символа и импортировать его в другой проект.

Условие:

Вы открыли проект.

1. Выберите пункты меню Данные проекта > Конфигурировать порядки свойств.
2. Выделите в дереве уровень иерархии или уже имеющийся порядок свойств.
3. Нажмите в диалоговом окне Конфигурировать порядки свойств — <Имя проекта> кнопку ![](images/all_new_as.png){: .ui-icon } (Создать).

!!! info "Для сведения:"

    Если выбрано определение функции, вариант символа либо порядок свойств определения функции или варианта символа, открывается диалоговое окно Новый порядок свойств.

!!! info "Для сведения:"

    Если в дереве выбран более высокий уровень иерархии, выделенный контур используется как фильтр для диалогового окна Определения функций или Выбор символа, которое открывается далее. После выбора определения функции или символа также открывается диалоговое окно Новый порядок свойств.

4. В диалоговом окне Новый порядок свойств введите однозначное идентифицирующее имя.

!!! info "Для сведения:"

    Идентифицирующее имя затем уже нельзя изменить.

5. В поле Отображаемое имя введите имя, под которым порядок свойств будет отображаться в проекте. Возможен многоязычный ввод.
6. Установите флажок Стандарт, если порядок свойств следует применять по умолчанию во всех других случаях использования 3D-размещения изделия / варианта символа.
7. Чтобы выбрать свойства, на соответствующей вкладке Трехмерные функции, Условное обозначение и т. д. нажмите кнопку ![](images/all_new_as.png){: .ui-icon } (Создать).
8. В таблице Свойство / присвоение задайте настройки отображения для выбранных свойств.
9. Щелкните по кнопке ++OK++.

!!! info "Для сведения:"

    Порядок свойств сохраняется и присваивается варианту символа. При размещении изделий порядок свойств присваивается определению функции.

!!! info "Для сведения:"

    Если порядок свойств задан как стандартный, то он используется для соответствующего варианта символа по умолчанию, например при вставке символа. При размещении изделий порядок свойств используется по умолчанию для соответствующего определения функции.

!!! tip "Совет:"

    С помощью пунктов всплывающего меню Копировать / Вырезать и Вставить можно переносить порядки свойств с одного варианта символа или определения функции на другой вариант символа или определение функции. Свойства, которые невозможно использовать на условном обозначении или определении функции, не учитываются при вставке. Переносить порядки свойств между областями "Размещение изделия (3D)" и "Условное обозначение" невозможно.

Порядок свойств для условного обозначения или размещения изделия можно задать на вкладке Отображение в диалоговом окне 'Свойства' с помощью раскрывающегося списка Порядок свойств. В качестве альтернативы имеются следующие возможности:

* При табличной форме обработки функций можно отобразить столбец Порядок свойств и выбрать нужный вариант из раскрывающегося списка. Список содержит любой порядок свойств, доступный для соответствующего варианта символа.
* При создании схем для внешней обработки можно выбрать порядок свойств в диалоговом окне Настройки: Внешняя обработка как элемент формата, тем самым учитывая соответствующие данные при экспорте и импорте.
* В диалоговом окне Свойства объектов-заполнителей можно использовать свойство Порядок свойств (Ид. 19307) и с помощью переменной присваивать определенный пользователем порядок свойств в качестве наборов значений. Для этого проект должен содержать соответствующий порядок свойств.
* Уже имеющийся порядок свойств также можно присваивать с помощью определенной пользователем кнопки на панели инструментов. Для этого в диалоговом окне Настроить выберите XEsSetPropertyAction и используйте в качестве значения параметра PropertyId Ид. 19307 свойства Порядок свойств.

Условия:

* В Графическом редакторе открыта страница проекта.
* Вы создали минимум один определенный пользователем порядок свойств.

1. Выделите в графическом редакторе или в навигаторе данных проекта одну или несколько функций, которые используют определенный пользователем порядок свойств. Или выделите размещение изделия в пространстве листа.
2. Выберите пункты меню Обработать > Текст > Сбросить порядок свойств.

!!! info "Для сведения:"

    Порядок свойств для выделенных функций / размещенных изделий сбрасывается до стандартной настройки. Этой настройкой может быть либо настройка "Станд. указание", либо порядок свойств, заданный как стандартный.

!!! tip "Совет:"

    Стандартные значения по умолчанию также можно восстановить непосредственно в диалоговом окне "Свойства". На вкладке Отображение выберите из раскрывающегося списка Порядок свойств запись "Стандартное значение по умолчанию"; при этом все настройки будут заменены стандартными значениями.

!!! note "Замечание:"

    * При перезаписи определенного пользователем порядка свойств измененные настройки отображения автоматически переносятся на все функции / размещения изделия, использующие этот порядок свойств. Этот же принцип действует и для импорта порядков свойств. Если при импорте перезаписываются один или несколько порядков свойств, измененные настройки отображения также автоматически переносятся на все функции / размещения изделия, использующие эти порядки свойств.
    * При копировании и вставке страниц, пространств листа, функций или размещений изделия в другой проект или при вставке макросов определенные пользователем порядки свойств сравниваются. Имя порядка свойств на функции / размещении изделия сохраняется, если порядки свойств совпадают или если порядка свойств еще нет в проекте. Если одноименный порядок свойств уже существует в проекте и не совпадает с порядком свойств на функции / размещении изделия, настройка проекта [Применить порядок свойств из проекта](gedviewer_d_einstellungenprojektallgemein.md) определяет, какой порядок свойств использовать.
    * Нажмите кнопку ![](images/all_edit_as.png){: .ui-icon } (Обработать), чтобы открыть выбор свойств. Он позволяет заменить свойство в порядке свойств (например, свойство ОУ (видимое) на свойство ОУ (полное)).
