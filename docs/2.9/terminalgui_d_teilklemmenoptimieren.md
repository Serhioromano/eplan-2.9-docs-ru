# Диалоговое окно Оптимизация распределенных клемм

Вы открыли проект. Данные проекта > Клеммники > Оптимизация распределенных клемм.

В данном диалоговом окне задаются настройки для оптимизации распределенных клемм, например, какое изделие или определение функции присваивается различным типам клемм. Соединенные друг с другом распределенные клеммы можно объединять и сортировать по перемычкам.

Кроме того, при поддержке оптимизации распределенных клемм можно обозначать указанные выводы устройств распределенных клемм и объединять распределенные клеммы по полным функциям клемм. Для каждого вывода устройства, к которому подсоединен как минимум один провод (любая функция, перемычка и т. д. за исключением мостовых перемычек и внутренних перемычек), автоматически задается обозначение вывода устройства. Флажок Главная клемма автоматически устанавливается для первой распределенной клеммы функции клеммы (в соответствии с отображением в навигаторе клеммников).

Обзор основных элементов диалогового окна:

Групповое поле номера изделия

### Клемма... SH-клемма

В данных полях задается изделие, которое необходимо использовать для клемм, PE- / PEN-клемм, N-клемм и SH-клемм. С помощью [...] открывается диалоговое окно Выбор изделия, в котором можно выбрать клемму; номер изделия клеммы затем переносится в соответствующее поле.

Если в изделие введены шаблоны функций для клемм, то данные первого шаблона функций присваиваются оптимизируемым клеммам. При этом все свойства распределенной клеммы перезаписываются с соответствующими значениями из шаблона функции. Раннее заполненные поля также переписываются с пустыми значениями.

!!! note "Замечание:"

    Если при оптимизации распределенной клеммы вставляется изделие, в шаблонах функций которого находится обозначение клеммы, данное изделие не учитывается при оптимизации. В этом случае на экран выводится сообщение о том, что распределенная клемма не учитывалась во время оптимизации. Обозначения клеммы не учитываются, так как в противном случае при копировании маркировки клеммы произошло бы обобщение всех выбранных распределенных клемм в одну клемму.

### Группа (определение функции)

В этом поле задается группа функций, которая по умолчанию присваивается распределенным клеммам при вставке, если им невозможно присвоить изделие или в групповом поле Номер изделия не введен номер изделия. С помощью кнопки ++...++ открывается диалоговое окно Определения функций, в котором при необходимости можно выбрать другое определение функции.

### Сохранить имеющиеся обозначения клемм

Если установлен данный флажок, то имеющиеся обозначения распределенных клемм не изменяются. Если несколько клемм имеют одинаковые обозначения, автоматически задается индекс распределенной клеммы. При этом соединенные между собой распределенные клеммы с одинаковым ОУ и одинаковым обозначением 'собираются' до тех пор, пока число указанных выводов устройства не будет соответствовать числу выводов устройства выбранного выше изделия или выбранного выше определения функции. Для каждого вывода устройства изделия или определения функции необходимо найти указанный вывод устройства на распределенной клемме. 'Собранные' таким образом распределенные клеммы получают одинаковый ***Индекс распределенной клеммы*** и объединяются в одну функцию клеммы.

Если данный флажок снят, то обозначения клемм задаются заново в соответствии с настройками в онлайн-режиме нумерации. При этом соединенные между собой распределенные клеммы с одинаковым ОУ 'собираются' до тех пор, пока число указанных выводов устройства не будет соответствовать числу выводов устройства выбранного выше изделия или выбранного выше определения функций. Для каждого вывода устройства изделия или определения функции необходимо найти указанный вывод устройства на распределенной клемме. 'Собранные' таким образом распределенные клеммы получают одинаковое ***Обозначение клеммы*** и объединяются в одну функцию клеммы.

!!! example "Пример:"

    В схеме соединений восемь распределенных клемм обозначаются ОУ как-X1. В поле Группа (определение функции)выбрано определение функции с тремя выводами устройства, например Клеммы с мостовой перемычкой, 3 вывода.Если снят флажок Сохранить имеющиеся обозначения клемм, то распределенные клеммы собираются до тех пор, пока не будут доступны три указанных вывода устройства и данные распределенные клеммы объединяются.В данном примере образуются три функции клеммы (здесь выделено рамкой) с обозначениями клемм1,2и3.

### Сохранить имеющиеся обозначения выводов устройства

Если установлен данный флажок, то имеющиеся обозначения выводов устройств распределенных клемм не изменяются. Также не изменяются обозначения выводов устройства неуказанных выводов устройства или выводов устройства, к которым подсоединена только одна мостовая или одна внутренняя перемычка.

Если этот флажок снят, то обозначения выводов устройства, указанные на распределенных клеммах, сначала удаляются, а затем заполняются значениями из выбранного выше изделия или выбранного выше определения функции. Только указанные выводы устройств получают обозначение вывода устройства. Обозначения выводов устройства неуказанных выводов устройства или выводов устройства, к которым подсоединена только одна мостовая или одна внутренняя перемычка, убираются.

### Сохранить имеющийся индекс распределенной клеммы

!!! note "Замечание:"

    Эта настройка действует только, если установлен флажок Сохранить имеющиеся обозначения клемм.

Если этот флажок установлен, имеющиеся значения сохраняются для свойства Индекс распределенной клеммы. При этом не играет роли, введены ли данные значения вручную или заданы при прежней оптимизации распределенной клеммы автоматически.

Если флажок снят, индекс распределенной клеммы задается снова при оптимизации распределенных клемм.

### Разрешить объединение связанных между собой клемм

Если установлен данный флажок, то соединенные друг с другом клеммы пересортируются так, что после сортировки по перемычкам они будут расположены рядом друг с другом. Из этого по умолчанию исключаются PE- / PEN-клеммы, т. к. иногда в клеммнике они должны быть разделены.

Если флажок снят, то клеммы не пересортируются.

### Также PE-клеммы

Если данный флажок установлен, то PE- / PEN-клеммы учитываются при пересортировке соединенных друг с другом клемм.

Если флажок снят, PE- / PEN-клеммы не пересортируются.

### Объем

При помощи раскрывающегося списка определите, какие распределенные клеммы будут оптимизированы:

* Выделенные: Оптимизируются только содержащиеся в маркировке распределенные клеммы.
* Все из выбранных клеммников: Оптимизируются все распределенные клеммы выделенных клеммников.
* Все из всех клеммников: Оптимизируются все распределенные клеммы всех клеммников в проекте.
