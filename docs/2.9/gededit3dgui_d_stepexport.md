# Диалоговое окно Экспорт STEP / VRML

В навигаторе пространства листа выбрано пространство листа. Выберите пункты меню Пространство листа > Экспортировать > STEP / VRML.

В этом диалоговом окне устанавливаются настройки для экспорта трехмерных графических данных в формат STEP или VRML.

Обзор основных элементов диалогового окна:

### Источник

В этом поле выводится на просмотр источник экспорта. Если указан весь проект, то появляется его имя, а для отдельных пространств листа отображаются полные имена пространств листа (включая структурные идентификаторы: Функциональное присвоение, Установка и т. д.). Обработка записей в данном поле невозможна.

### Схема

Выберите из раскрывающегося списка схему, которая определяет настройки для экспорта STEP / VRML.

С помощью кнопки ++...++ можно открыть диалоговое окно Настройки: Экспорт STEP / VRML. В этом диалоговом окне указываются настройки для экспорта файлов STEP или VRML, после чего настройки сохраняются в виде схемы.

* * *

Групповое поле 'Средство экспорта'

### Носитель данных

При выборе этого средства экспорта файл STEP / VRML будет сохранен на определенный носитель данных. В поле Каталог вывода задается место хранения. Здесь предварительно выбран каталог, заданный в настройках пользователя для экспорта STEP / VRML.

Кнопка ++...++ открывает диалоговое окно Выбрать каталог для выбора любого каталога для экспорта.

С помощью пункта всплывающего меню Вставить переменную пути перейдите в диалоговое окно [Выбрать переменную пути](modaldialogsdb_d_pfadvariablen.md), из которого можно будет скопировать одну из доступных переменных пути.

### Эл. почта

При установке этого средства экспорта файл STEP / VRML будет сжат и отправлен по электронной почте. Из раскрывающегося списка Ограничение размера выберите максимальный размер для отдельных файлов, отправляемых по электронной почте. Допустимы значения от 1 до 10 MБ или запись 'Неогранич.'. EPLAN автоматически создаст то количество файлов для отправки по электронной почте, которое необходимо для отправки всех частей. Всем частям будет сопутствовать следующий текст: "<Имя файла> n/m". Здесь 'n' — это номер отдельного файла для отправки по электронной почте, а 'm' — общее количество файлов.

* * *

### Имя файла

При экспорте отдельного пространства листа допустим ввод любого имени файла. С помощью кнопки ++...++ также можно свободно задавать имя файла или выбирать имя уже имеющегося файла.

Если специальное имя файла не введено, экспортированному файлу присваивается имя в зависимости от настройки [Генерировать имена файлов](gededit3dgui_d_einstellungenstepexport.md).

С помощью пункта всплывающего меню Вставить переменную пути перейдите в диалоговое окно [Выбрать переменную пути](modaldialogsdb_d_pfadvariablen.md), из которого можно будет скопировать одну из доступных переменных пути.

При экспорте нескольких пространств листа невозможен свободный ввод имени файла. Будут файлы STEP / VRML генерироваться в заданном целевом каталоге с учетом порядкового номера, из имен пространств листа или из свойств, зависит от настроек выбранной схемы.

Если выбрано пространство листа, содержащее несколько электрошкафов, и при этом в диалоговом окне Настройки: Экспорт STEP / VRML активирована настройка Отдельный экспорт для каждого электрошкафа, задать имя файла невозможно. Поле Имя файла в этом случае деактивируется.

### Применить ко всему проекту

При установке данного флажка текущий выбор пространства листа распространяется на все пространства листа проекта.

Если флажок не установлен, экспортируются только выделенные пространства листа.

Если отдельный проект уже выделен (и при этом фильтры не установлены), флажок недоступен. В этом случае настройку больше невозможно изменить.
