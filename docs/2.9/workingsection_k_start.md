# Области обработки

Проекты можно разделять на области обработки. Это повышает наглядность при обработке проекта и облегчает одновременную обработку проектов несколькими пользователями. Критерии для разделения сохраняются в схемах фильтров.

Если проекты содержат блоки идентификаторов, разделение происходит на основе имеющихся в проекте структурных идентификаторов для блоков идентификаторов. При этом критерием для фильтра может служить любая комбинация этих структурных идентификаторов. Дополнительно можно выполнить фильтрацию страниц, пространств листа, функций и соединений по определенным разделам.

Для проектов с общей нумерацией возможно только разделение по разделам.

Возможно следующее:

* Выбрав схему, можно определить область обработки, в которой вы хотите работать.
* При использовании управления правами EPLAN области обработки может централизованно распределять администратор проекта. Если управление правами не используется, каждый отдельный пользователь может сам задавать область обработки, в которой он хочет работать.
* В навигаторах данных проекта отображаются только страницы, устройства и т. д., отвечающие критериям, заданным в схеме. Это позволяет увеличить наглядность проекта и повышает скорость работы.
* Область обработки можно временно распространить на весь проект.
