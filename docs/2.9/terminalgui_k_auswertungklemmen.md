# Отчеты по клеммам с отображением выводов устройств

В отчетах по клеммам с отображением выводов устройства указывается, какие цели подключены к каким клеммным выводам устройства. Для этой цели в спецификациях клеммников и в схемах подключения клемм предусмотрены элементы-заполнители Клеммный вывод устройства, внутренний и Клеммный вывод устройства, внешний, через которые выводятся в том числе соответствующие обозначения выводов устройства.

Свойство формы Отображать выводы устройства позволяет указать, что отчеты по клеммам должны составляться с учетом выводов устройств. Если данное свойство установлено, в спецификациях клеммников и схемах подключения клемм для каждой пары выводов (внешн./внутр.) устройства генерируется одна строка. При этом клеммы, имеющие более двух выводов устройства, автоматически распределяются на несколько строк. Выводы устройства без цели не выводятся.

На основе логической схемы выводов устройства клеммы определяется, какой из выводов является внешним или внутренним, а также в какой области на странице отчета этот вывод будет отображаться. По умолчанию EPLAN рассматривает функциональные элементы, находящиеся в зоне под клеммой, как внешние цели, а функциональные элементы, находящие в зоне над клеммой, как внутренние цели. (Например, для одной клеммы с четырьмя выводами устройства могут быть указаны три внешних вывода устройства и один — внутренний). Дополнительно выводятся также неподключенные выводы устройства клемм. Кроме того, перемычки между внутренним и внешним выводами устройства ***не*** рассматриваются и не отображаются как перемычки.

Если свойство форм Отображать выводы устройства снято, клеммы анализируются относительно цели. В отчетах относительно цели выводятся внешние и внутренние цели клемм. Для каждой цели с помощью элементов-заполнителей Клеммный вывод устройства, внутренний и Клеммный вывод устройства, внешний можно указать, на каком выводе клеммы подключена эта цель.

!!! note "Замечание:"

    Даже при отключенном свойстве формы Отображать выводы устройства перемычки между внутренним и внешним выводом устройства ***не*** рассматриваются как перемычки и не отображаются в отчете, пока в форме не будет использован один из следующих символов-заполнителей:

* Цель через вывод устройства
* Соединение через вывод устройства
* Соединение / кабель через вывод устройства.

!!! tip "Совет:"

    С помощью меню Вид > Внешние цели в графическом редакторе отображаются установленные внешние цели в виде стрелок. Это упрощает отслеживание цели в случае с клеммами.

### Вывод мостовых перемычек

В отчетах по клеммам с отображением выводов устройства указываются только выводы устройства без мостовых перемычек. Имеющиеся мостовые перемычки отображаются на соответствующих выводах устройства. Исходя из этого, мостовые перемычки выводятся в отчете только тогда, когда на той же стороне клеммы (внутренней / внешней) выводится также как минимум один вывод устройства без мостовой перемычки. Мостовые перемычки, которые связывают внутренний вывод устройства с внешним, ***не*** выводятся в отчетах по клеммам с отображением выводов устройства.

Вручную можно соединить мостовой перемычкой последний и предпоследний выводы устройства клеммы. Если все выводы устройства без мостовых перемычек находятся на одной стороне клеммы (например, на внешней), то на этой клемме отсутствует внутренняя сторона и, соответственно, возможность вывести там вывод устройства с мостовой перемычкой. Это означает:

* Если все выводы устройства без мостовых перемычек являются "внешними", то вывод устройства с мостовой перемычкой также должен быть "внешним".
* Если все выводы устройства без мостовых перемычек являются "внутренними", то вывод устройства с мостовой перемычкой также должен быть "внутренним".
* Если имеются и внешние, и внутренние выводы устройства без мостовых перемычек, то выводы устройства с мостовыми перемычками могут быть как "внутренними", так и "внешними".

В отчетах относительно цели выводятся все мостовые перемычки.

### Вывод прямых выводов устройства

Клеммы могут иметь прямые выводы устройства, для этого в логической схеме выводов устройства необходимо установить тип вывода устройства "Прямой вывод устройства". Такие прямые выводы устройства анализируются как обычные клеммные выводы, а обозначения выводов устройства выводятся в отчетах, связанных с выводами (например, в спецификации клеммника).

Исключение: для определений функций с контактами шины, например "PE-клемма с контактом шины, 2 вывода устройства", прямые выводы устройства (т. е. контакты шины) не выводятся.
