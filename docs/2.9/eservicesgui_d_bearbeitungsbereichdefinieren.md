# Диалоговое окно Определить области обработки / Диалоговое окно Области обработки определить как администратора проекта

* Вы открыли проект.Проект > Области обработки > Определить.
* Вы используете управление правами EPLAN и имеете право распределять области обработки в качестве администратора проекта. Вы открыли проект. Проект > Области обработки > Определить как администратор проекта.

В диалоговом окне Определить области обработки путем выбора схемы можно указать области обработки внутри проекта.

В диалоговом окне Области обработки определить как администратора проекта определите области обработки как схемы и присвойте этим областям обработки возможных обработчиков.

Обзор основных элементов диалогового окна:

### Таблица 'Определение'

В таблице отображены определенные для проекта области обработки и их текущие обработчики. Посредством выбора схемы вы устанавливаете область обработки.

* Статус: В этом столбце отображается пиктограмма, указывающая на то, отделены ли области обработки друг от друга или пересекаются. "Наложение" означает, что определенные объекты (например, страницы, пространства листа или сегменты предварительного планирования) имеются в нескольких областях обработки. Для генерирования частичных проектов области обработки не должны накладываться друг на друга. Зеленая галочка показывает, что область обработки не пересекается ни с какой другой. Желтый предупреждающий треугольник показывает, что части этой области обработки содержатся и в других областях.
* Схема: Здесь перечислены определенные для проекта области обработки. Для этого отображены имена схем, в которых определены критерии разделения.
* Описание: Здесь отображается описательный текст к соответствующей схеме.
* Ревизия: Если ревизия области обработки проведена, здесь отображается ее текущая версия.
* Все обработчики: В этом столбце отображаются другие пользователи, выбравшие область обработки в проекте. Если вы открыли диалоговое окно Области обработки определить как администратора проекта, столбец Все обработчики будет активирован. Здесь можно вносить или удалять новых обработчиков для соответствующей области обработки. При вводе нескольких обработчиков они разделяются запятой.
* Моя область обработки: В этом столбце можно определить область обработки, установив флажок для требуемой схемы. После этого в навигаторах данных проекта будут отображаться только страницы, устройства и т. д., соответствующие критериям, заданным в схеме.
* Заблокировано: Если область обработки заблокирована, здесь отображается причина блокировки. Это происходит, если область обработки была выгружена как частичный проект.

В диалоговом окне Определить области обработки как администратор проекта с помощью кнопки ![](images/all_edit_as.png){: .ui-icon } (Обработать) открывается диалоговое окно [Фильтр](modaldialogsdb_d_filternnach.md). Здесь можно обработать настройки для текущей схемы или создать новые схемы. В качестве критериев фильтра для структурных идентификаторов могут служить все свойства. Дополнительно можно использовать свойство Раздел (Области обработки), чтобы фильтровать страницы, пространства листа, функции и соединения по определенным разделам.

Кроме того, здесь доступны кнопки ![](images/all_import_as.png){: .ui-icon } (Импортировать) и ![](images/all_export_as.png){: .ui-icon } (Экспортировать) для импорта и экспорта схем в формате xml.

!!! note "Замечание:"

    * Даже пустой структурный идентификатор образует область обработки и может быть выбран для обработки.
    * Область обработки для раздела "Fluid-техника, общее" учитывает также отдельные разделы "Гидравлика", 'Пневматика', "Система охлаждения' и 'Смазка". Если вы определили такую схему, то в области обработки будут отображаться, например, страницы с разделом "Fluid-техника, общее", а также страницы, для которых настроены отдельные разделы "Гидравлика", "Пневматика" и т. д.
    * У страниц имеется дополнительная настройка раздела — 'Общее'. В разделе 'Общее' содержатся стандартные настройки, которые используются при генерации страниц отчетов. При фильтрации областей обработки по определенным разделам страницы с разделом "Общее" выводятся вместе с другими при указании любого раздела.

### Временно работать во всем проекте

Если этот флажок установлен, обработка временно распространяется на весь проект. Это целесообразно, например, при обновлении отчетов.

Если флажок не установлен, обработка ограничивается только определенной областью.
