# Импортировать в проект устройства и маршрутизируемые соединения: Принцип

Импорт устройств и маршрутизируемых соединений осуществляется в два отдельных шага, для которых используются различные пункты меню:

* Данные проекта > Устройство > Импортировать: Импортирует список обозначений устройств. В проекте появляются устройства с неразмещенными функциями. Эти импортированные устройства с одним или несколькими изделиями отображаются в навигаторе устройств или трехмерном навигаторе монтажных поверхностей и могут быть перемещены из навигаторов в пространство листа вручную путем перетаскивания мышью. Для этого изделие должно быть механическим, или присвоенное изделие должно иметь необходимую информацию (размеры, макрос).
* Данные проекта > Соединения > Импортировать > Маршрутизируемые соединения: Импортирует маршрутизируемые соединения. Возникают непроложенные маршрутизируемые соединения, если будут найдены их источник и цель. Непроложенные маршрутизируемые соединения можно заново маршрутизировать в навигаторе соединений через пункт всплывающего меню Маршрутизировать (пространство листа).

Правильное присвоение полей в источнике данных и в свойствах EPLAN играет важную роль для успешного импорта. Такое присвоение полей сохраняется на схеме. Предварительно определенная схема имеется, но в большинстве случаев она должна согласовываться с соответствующим источником данных.

### Присвоение полей как условие

Чтобы при импорте устройств можно было генерировать главные функции, соответствующему полю ввода в источнике данных необходимо присвоить свойство ОУ (полное) (ид. 20006).

Определение функции генерируемой главной функции задается при импорте путем присвоения изделия. Для этого следующему полю данных источника данных следует присвоить свойство Номер изделия [x] (ид. 20100 x). (Знак "x" после номера свойства обозначает здесь индекс.)

При этом для определения главной функции используется шаблон функции, указанный первым на изделии. Если шаблон функции не указан или активирована настройка проекта Учитывать макрос при генерировании устройств (в меню Параметры > Настройки > Проекты > "Имя проекта" > Устройство > Общ.), главная функция определяется посредством макроса, указанного на изделии.

Если при импорте не обнаружено подходящее изделие в базе данных изделий или из изделия не удается определить подходящую главную функцию из-за недостающего шаблона / макроса, можно напрямую присвоить определение функции. Например, этого можно добиться путем присвоения поля данных свойству Определение функции (ид. 20026).

В случае с идентичным ОУ клеммы и штекеры различаются свойством Обозначение клеммы / контакта штекера (ид. 20030). Поэтому для импорта этих функций полю данных, помимо свойства ОУ (полное) (ид. 20006), также необходимо присвоить свойство Обозначение клеммы / контакта штекера (ид. 20030).

Если клеммам и штекерам присваиваются изделия из базы данных изделий с помощью свойства Номер изделия [x] (ид. 20100 x), определения функций будут определяться из данных изделий по аналогии с устройствами. Клеммы с данными изделий генерируются при импорте в качестве главных клемм.

При импорте многоуровневых клемм путем присвоения полей можно указать уровень путем присвоения полей с помощью свойства Уровень (ид. 20034). В этом случае даже при импорте без данных изделия определяется, что речь идет о клемме. Второстепенные клеммы генерируются для многоуровневых клемм, если в источнике данных для этих клемм заданы значения для уровня, но отсутствуют номера изделий.

Чтобы при импорте данных соединений генерировались немаршрутизированные маршрутизируемые соединения, требуется наличие в проекте источника и цели соединения в качестве трехмерного размещения изделия. В присвоении полей этого импорта должны содержаться как минимум следующие свойства:

* Источник (ID 31019)
* Цель (ид. 310209).

Если дополнительно присвоить свойство ОУ (полное) (ид. 20006), при импорте автоматически генерируется кабельное соединение.

Возникшим немаршрутизированным маршрутизируемым и кабельным соединениям по умолчанию присваивается определение функции "Жила / провод". Путем следующего присвоения полей (напр., с помощью свойства Определение функции (ид. 20026) можно изменить это определение функции.

Часто бывает так, что номера изделий, присвоенные устройствам и соединениям во внешнем источнике, отсутствуют в базе данных изделий EPLAN. Чтобы в таком случае все же обеспечить присвоение изделий, можно использовать другие совпадающие свойства. Помимо номера изделия также можно присвоить следующие свойства:

* Импортировать устройства: Номер для заказа (ид. 20919) и / или Обозначение типа изделия (ид. 20200)
* Импортировать маршрутизируемые соединения: Цвет / номер соединения (ид. 31004) и / или Поперечное сечение / диаметр соединения (ид. 31002)

Для таких свойств во внешнем источнике данных должны предусматриваться соответствующие поля данных, которым затем присваиваются соответствующие свойства EPLAN.

### Присвоение изделий при синхронизации

Следующая процедура служит для присвоения изделий импортируемым устройствам и маршрутизируемым соединениям в проекте EPLAN.

* Если в базе данных изделий найдено изделие с таким же номером, это изделие автоматически присваивается.
* Если устройству или соединению присвоен номер изделия, отсутствующий в базе данных изделий, то этот номер изделия отображается жирным шрифтом в диалоговом окне Синхронизировать устройства или Синхронизировать маршрутизируемые соединения в столбце Номер изделия. В таком случае с помощью пункта всплывающего меню Присвоить изделие можно организовать поиск соответствующего изделия, у которого совпадали бы другие свойства, например Номер для заказа и / или Обозначение типа изделия.
* Если осуществить присвоение таким способом не удается, можно открыть выбор изделия в столбце Номер изделия при помощи кнопки ++...++ и вручную выбрать там изделие.
