# Обработка проекта в многопользовательском режиме

В системе EPLAN возможно для нескольких пользователей одновременно обрабатывать проект при помощи сети с разных станций. При этом в так называемом многопользовательском режиме могут возникнуть конфликты, если, например, несколько пользователей хотят обрабатывать одни и те же данные. если возникает случай конфликта, об этом сообщается пользователю в диалоговом окне.

### Сообщения в случае конфликта

если Вы хотите обработать данные проекта, которые уже обрабатываются другим пользователем, система EPLAN проинформирует Вас в диалоговом окне [Многопользовательский конфликт](xesmultiuserconflictgui_d_konflikte.md) о том, что это невозможно.

В этом диалоговом окне перечислены все пользователи, которые в данный момент обрабатывают эти же данные, с указанием имени, телефона, адреса электронной почты и названия рабочей станции. При помощи этой информации у Вас есть возможность попросить пользователя или пользователей по телефону или посредством электр. почты освободить используемые данные или сообщить Вам об этом по окончании их обработки.

Указанный выше случай конфликта обозначает, что Вы можете просмотреть данные, но эти данные заблокированы другим пользователем для обработки. После того как Вы закроете диалоговое окно Многопользовательский конфликт, в системе EPLAN для просмотра откроется соответствующее диалоговое окно "Свойства".

Как только первый пользователь закончил обработку, другие пользователи снова получают доступ к данным. Вы можете посмотреть измененные данные после обновления при помощи нажатия [F5]. При этом поля открытого диалогового окна "Свойства" снова будут разблокированы.

!!! note "Замечание:"

    * Обратите внимание, что для многопользовательского режима проект должен быть открыт в режиме обработки "Стандарт". Откройте проект в режиме "Эксклюзивный", тогда другие пользователи не смогут получить доступ к этому проекту.
    * Вы можете целесообразно использовать диалоговое окно Многопользовательский конфликт только в том случае, если каждый из пользователей, участвующий в многопользовательском режиме, указал соответствующие данные пользователя в "Настройках" (Параметры > Настройки > Пользователь > Отображение > Ид. пользователя/ адрес).

### Обработка для всего проекта в многопользовательском режиме

При некоторых рабочих шагах, когда изменяются данные для всего проекта, возможен доступ к проекту исключительно (эксклюзивно) только для одного пользователя. При этом речь идет, например, о следующих действиях.

* Пользователь обрабатывает свойства проекта.
* Пользователь выполняет изменения в пространстве листа.
* Пользователь открыл для проекта диалоговое окно "Управление структурными идентификаторами".
* Проект как раз сжимается, резервируется или восстанавливается пользователем.
* Пользователь выполняет синхронизацию основных данных в проекте, по всему проекту изменяет ссылки основных данных.

Система EPLAN не может в таком случае открыть соответствующий проект для многопользовательского режима и информирует других пользователей, которые хотели открыть проект, при помощи диалогового окна конфликта о том, как пользователь сейчас имеет доступ к проекту.

!!! note "Замечание:"

    При открытии одного проекта несколькими пользователями в многопользовательском режиме эти пользователи все же могут перейти в диалоговое окно ***Управление структурными идентификаторами***. Тем самым они могут изменять описания структурных идентификаторов также в многопользовательском режиме. В диалоговом окне Идентификатор пользователи могут одновременно работать с описаниями в столбцах Описание структуры 1-3 и Описание структуры дополнительного поля [1]-[10] по принципу "один структурный идентификатор — один пользователь". Другие функции обработки структурных идентификаторов в многопользовательском режиме недоступны, поэтому соответствующие поля и кнопки неактивны.

### Поведение с низкой вероятностью конфликта при генерации соединений

При обработке в в "Графическом редакторе" часто требуется обновленная информация о соединении. Для этого в меню Данные проекта имеется пункт меню Соединения > Обновить.

В многопользовательском режиме ***только*** тогда происходит конфликт, когда двумя пользователями одновременно обновляется соединение к одному и тому же проекту или к одной и той же странице. Пользователь, который начал процесс вторым, информируется системой EPLAN при помощи диалогового окна Многопользовательский конфликт. Для корректного просмотра его изменений соединения он должен затем повторно генерировать соединение.

Конфликты ***не*** возникают, например, в таких случаях:

* Один пользователь обновляет соединение для одной страницы (или для всего проекта), в то время как другой пользователь одновременно обрабатывает устройство, которое размещено на той же странице.
* Один пользователь обновляет соединение для одной или нескольких страниц проекта, в то время как другой пользователь как раз обновляет соединение для других страниц проекта.

### Отмена операций

В многопользовательском режиме каждый пользователь может отменить только собственные действия. Если после внесения изменений первым пользователем другой пользователь изменяет тот же самый объект, первый пользователь лишается возможности отменить свои действия.

Если первый пользователь все-таки пытается отменить свой рабочий шаг, он получает сообщение о том, что он не может отменить свои изменения, поскольку другой пользователь уже изменил объект далее.
