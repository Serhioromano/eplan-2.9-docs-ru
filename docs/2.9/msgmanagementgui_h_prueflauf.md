# Проверить проекты

Проекты можно проверять только ***в автономном режиме***. При проверке в автономном режиме можно проверять весь проект, любые элементы структуры проекта (напр., все места установки) или любые страницы проекта. При проверке всего проекта в автономном режиме дополнительно проверяются неразмещенные функции, точки разрыва, устройства, ссылки изделий и объекты предварительного планирования.

Условия:

* Вы открыли проект.
* Для контрольных прогонов созданы схемы, содержащие проверки, которые можно проводить в автономном режиме, и вид которых соответствует записи "Автономно".
* Вы открыли Навигатор страниц. В дереве навигатора страниц выделены проект, один или несколько узлов структуры проекта или определенные страницы проекта.

1. Выберите следующие пункты меню: Данные проекта > Сообщения > Выполнить контр. прогон
2. В диалоговом окне Выполнить контр. прогон выберите схему из раскрывающегося списка Настройки.
3. Если для проверки выбраны отдельные элементы структуры или страницы проекта либо диалоговое окно вызвано в графическом редакторе с помощью пункта всплывающего меню Повторить: Контрольный прогон, можно дополнительно настроить выполнение проверки для всего проекта. Для этого следует установить флаг Применить ко всему проекту.
4. Щелкните по кнопке ++OK++.

!!! info "Для сведения:"

    EPLAN проверяет проект в соответствии с настройками, заданными в схеме. Результаты контрольного прогона записываются в управление сообщениями. При этом убираются результаты предыдущего контрольного прогона.

!!! info "Для сведения:"

    Выводимые в ходе контрольного прогона сообщения (например, из-за невозможности выполнения некоторых проверок) перечисляются в диалоговом окне системных сообщений. Об этом информирует соответствующее примечание.

5. Выберите пункты меню Данные проекта > Сообщения > Управление.

!!! info "Для сведения:"

    Сообщения, сохраненные в базе данных сообщений, отображаются в диалоговом окне Управление сообщениями — <Имя проекта>.

!!! note "Замечание:"

    Количество результатов проверки может быть разным и зависит от того, какие элементы структуры проекта выбраны в навигаторе страниц для проверки в автономном режиме.

!!! example "Пример:"

    Если, например, в проверяемом проекте есть структура обозначений по установке и подчиненному месту установки и если выделить в навигаторе страницструктурный идентификатор***места установки***, то все страницы данного места установки будут проверены в соответствии с заданными параметрами.Если, напротив, в навигаторе страниц выделен структурный идентификатор***установки***, то будут проверены все относящиеся к ней места установки, включая подчиненные страницы.Если в навигаторе страниц выделен***Проект***, то в область проверки войдут все страницы подчиненных структурных элементов, при этом следует ожидать максимальное количество результатов проверки.Если выделена лишь одна***Страница***места установки в навигаторе страниц, проверена будет только эта страница. При этом количество результатов проверки будет соответственно меньше.
