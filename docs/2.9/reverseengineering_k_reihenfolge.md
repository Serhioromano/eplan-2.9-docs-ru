# Последовательность видов представлений при общей обработке и анализе

Если у функции есть несколько видов представлений, общие свойства которых содержат различные данные, на основе вида представления решается, какие свойства должны учитываться при общей обработке и составлении отчетов. Для этого в EPLAN существует заданная последовательность. Для [Последовательности обработки сгруппированных объектов](gededitgui_k_gruppierung.md) также существует заданная последовательность.

### Последовательность функций

Это последовательность видов представлений, в EPLAN учитываются общие свойства всех видов представлений функции в [Режим-Свойства (общие)](reverseengineering_k_modus.md):

* Многополюсный
* Обзор
* Парная перекрестная ссылка
* Однополюсный
* Функциональная схема автоматизации
* Топология
* Компоновка электрошкафа
* Компоновка электрошкафа подробно
* Трехмерный чертеж монтажных поверхностей
* Функциональный
* Обзор функций (Fluid-Техника).

EPLAN сначала учитывает свойства многополюсного вида представлений. Если функция не имеет многополюсного вида представления или в многополюсном виде представления соответствующее поле свойства пусто, EPLAN учитывает свойство функции обзора и т. д.

!!! note "Замечание:"

    Для свойств с типом поля "Булево (истина / ложь)" последовательность видов представления не имеет значения. Такое свойство (например, флажок Главная функция) считается активированным для функции во время общей обработки и генерации отчета, если оно активировано на любом виде представления.

!!! example "Пример:"

    Выбрана распределенная функция в следующих видах представлений:Вид представления Функциональный текст Многополюсный Заготовка в хранилище Обзор Заготовка на конвейере Парная перекрестная ссылка Заготовка на месте Однополюсный Поставить заготовку Функциональная схема автоматизации Заготовка Выделите функцию вида представления "Функциональная схема автоматизации" и выберите Всплывающее меню > Свойства (общие). В поле Функциональный текстотображается функциональный текст Заготовка в хранилище. Это функциональный текст многополюсной функции, т. к. сначала учитывается многополюсная функция.Измените функциональный текст. Функциональный текст синхронизируется во всех видах представления функции:Вид представления Функциональный текст Многополюсный Заготовка в хранилище Обзор Заготовка в хранилище Парная перекрестная ссылка Заготовка в хранилище Однополюсный Заготовка в хранилище Функциональная схема автоматизации Заготовка в хранилище

При общей обработке в других видах представлений свойства синхронизируются, если в соответствующих полях уже находятся записи. Пустые поля не заполняются.

!!! example "Пример:"

    Выбрана распределенная функция в следующих видах представлений:Вид представления Функциональный текст Многополюсный Обзор Заготовка на конвейере Парная перекрестная ссылка Заготовка на месте Однополюсный Поставить заготовку Функциональная схема автоматизации Заготовка Выделите функцию вида представления "Функциональная схема автоматизации" и выберите Всплывающее меню > Свойства (общие). В поле Функциональный текстотображается функциональный текст Заготовка на конвейере. Это функциональный текст функции обзора, т. к. многополюсная функция не обладает функциональным текстом, и EPLAN учтет функцию обзора в качестве следующегодействия.Измените функциональный текст. Функциональный текст синхронизируется во всех видах представления, но не в многополюсной функции, т. к. пустое поле не перезаписывется:Вид представления Функциональный текст Многополюсный Обзор Заготовка на конвейере Парная перекрестная ссылка Заготовка на конвейере Однополюсный Заготовка на конвейере Функциональная схема автоматизации Заготовка на конвейере

При ***Генерации отчета ***и ***Синхронизации*** функций, обладающих несколькими видами представлений, а также общие свойства которых содержат различные данные, свойства учитываются при одинаковой последовательности так, как при общей обработке.

### Последовательность соединений

Виды представления соединений учитываются при общей обработке, при генерации отчета и при синхронизации в следующей последовательности:

* Многополюсный
* Трехмерный чертеж монтажных поверхностей
* Однополюсный
* Функциональная схема автоматизации
* Топология.
