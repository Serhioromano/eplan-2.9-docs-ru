# Диалоговое окно "Настройки: Макросы"

Параметры > Настройки > Пользователь > Графическая обработка > Макросы

В этом диалоговом окне задаются специфические для пользователя настройки для графической обработки макросов.

Обзор основных элементов диалогового окна:

### Использовать графический выбор макроса

Если такой флажок активирован, во всех местах платформы EPLAN, где выбраны макросы, используется графический выбор макросов. В этом случае выполняется вставка макросов через пункты меню Вставить > Макрос окна / символа и Страница > Макрос страницы > Вставить размещение макросов в навигаторах данных проекта и выбор макросов в базе данных изделий. Пользователи "пролистывают" в этом диалоговом окне графического выбора макроса на основе сохраненных пиктограмм, символов или других графических файлов структуру каталогов и легко находят нужный макрос.

Если этот флажок отсутствует, диалоговое окно графического выбора макроса при вставке макроса не открывается.

### Учитывать при вставке вложенные рамки макросов

Эта функция имеет особое значение в том случае, если вы работаете с макросами, объединенными в вышестоящей рамке макроса. Эти т. н. "вложенные" макросы используются преимущественно при проектировании схем соединений Fluid.

Если флажок установлен, на всех функциях подчиненных макросов, объединенных в вышестоящей рамке макроса, остается первоначальное имя пути и файла макроса в поле свойства Макрос (ид. 20468).

Если флажок снят, имя вышестоящей рамки макроса вносится во все функции подчиненных макросов в свойство Макрос (ид. 20468).

!!! example "Пример:"

    Два макроса символа из каталога основных данных для макросов FESTO, 5/2-ходовый клапан с именем файлаP_V_0054.emsи цилиндр двойногодействияс именем файлаP_A_0017.EMSобъединены в общей рамке макроса с именем файлаV_Z_001.ema, поэтому речь идет о***вложенном***макросе. Сам вложенный макрос также сохраняется в каталоге макросовFESTO. На функциях подчиненных макросов, а также на вышестоящей рамке макроса для отображения выбрано свойство Макрос.Вложенный макрос два раза подряд вставляется на страницусхемы соединенийFluid. Сначала перед вставкой устанавливается флажок Учитывать при вставке вложенные рамки макросов******. После размещения макроса на функциях подчиненных макросов внутри вложенного макроса остаются имена пути и файла (см. рисунок ниже слева). Теперь флажок***снимается***, и еще раз вставляется вложенный макрос. В этот раз имя пути и файла вышестоящей рамки макросаV_Z_001.emaвносится на все функции подчиненных макросов в свойство Макрос(см. рисунок ниже справа).

!!! note "Замечание:"

    На функциях вы можете выполнить выбор изделия и при этом использовать имя подчиненного макроса в качестве автоматического фильтра. При этом на выбираемых изделиях нужно сохранить соответствующие имена файлов макросов.

Групповое поле "Открыть окно выбора набора значений объектов-заполнителей"

Объекты-заполнители, вставленные в макросы, могут содержать наборы значений. Укажите здесь отображается ли диалоговое окно выбора наборов значений при вставке таких макросов.

### При вставке макросов окна

Если этот флажок установлен, при вставке макросов окна или символа с объектами-заполнителями открывается диалоговое окно Выбрать набор значений — <Имя заполнителя>.

!!! note "Замечание:"

    Если эта настройка активирована, окно выбора набора значений открывается также при копировании и вставке объектов-заполнителей, а также макросов окна и символа с объектами-заполнителями.

### При вставке макросов страницы

Если этот флажок установлен, при вставке макросов страницы с объектами-заполнителями открывается диалоговое окно Выбрать набор значений — <Имя заполнителя>.

!!! note "Замечание:"

    Эта настройка не действует для копирования и вставки страниц проекта с объектами-заполнителями. При такой операции диалоговое окно выбора наборов значений ***не*** открывается.
