# Вкладка Общее (Автоматическое оснащение Rittal — Athex)

* Параметры > Настройки > Фирма > Автоматическое оснащение > Rittal — Athex. Выберите вкладку Общ..
* Сервисные программы > Данные изготовления > Автоматическое оснащение Rittal — Athex. Рядом с полем Машина нажмите ++...++. Выберите вкладку Общ..

На этой вкладке задаются общие настройки для экспорта данных, необходимых для управления системой автоматического оснащения Rittal — Athex для клеммников и последовательных принадлежностей клеммы.

Обзор основных элементов диалогового окна:

### Каталог вывода

Экспорт Rittal — Athex будет сгенерирован в виде файла XML. В этом поле укажите место хранения. В качестве целевых для экспорта данных можно указать любой дисковод и подкаталог.

С помощью пункта всплывающего меню Вставить переменную пути перейдите в диалоговое окно [Выбрать переменную пути](modaldialogsdb_d_pfadvariablen.md), из которого можно будет скопировать одну из доступных переменных пути.

### Генерировать имена файлов

В этом раскрывающемся списке задается стандартный порядок присвоения имен файлам при экспорте Rittal - Athex:

* Из непрерывного номера: при выборе этого параметра EPLAN по умолчанию присваивает файлам XML имена по образцу <непрерывный_номер>.XML. Если уже имеются файлы XML, EPLAN начнет нумерацию со следующего доступного числа.
* Из обозначения функц. элемента: При выборе этого параметра EPLAN по умолчанию присваивает файлам XML имена по образцу <Обозначение функц. элемента>.XML.
* Из свойств: если выбран этот параметр, имя файла составляется из свойств проекта, а также свободно определяемого разделителя. Используя кнопку ++...++, откройте диалоговое окно Формат: Свойство для конфигурации имени файла.

### Генерировать подкаталоги

В этом раскрывающемся списке определяется стандартный порядок создания подкаталогов при экспорте файлов Rittal — Athex:

* Нет: Если выбран данный параметр, при экспорте подкаталоги не генерируются.
* Из свойств: если выбран этот параметр, имя подкаталога составляется из свойств проекта, а также свободно определяемого разделителя. Используя кнопку ++...++, откройте диалоговое окно Формат: Свойство для конфигурации подкаталогов экспортируемых данных. Используйте здесь, например, элемент формата "Разделитель", чтобы сгенерировать структуру каталога с несколькими расположенными друг под другом подкаталогами.
