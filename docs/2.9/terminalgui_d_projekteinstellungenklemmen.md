# Диалоговое окно Настройки: Распределенные клеммы

Вы открыли проект. Параметры > Настройки > Проекты > "Имя проекта" > Устройство > Клеммы (распределенные клеммы).

В данном диалоговом окне определяются предварительные настройки для работы с распределенными клеммами.

Обзор основных элементов диалогового окна:

### Определение функции

В данном поле задается определение функций, по умолчанию присваиваемое распределенным клеммам при вставке. Кнопка ++...++ открывает диалоговое окно Определения функций и позволяет выбрать требуемые определения функций.

### Разместить по числу выводов

Здесь определяется минимальное количество выводов устройства с ***обозначениями выводов устройства***, которое должна иметь функция клеммы, чтобы при вставке устройств или при размещении клемм из навигатора клеммников размещались распределенные клеммы. Выводы устройств без обозначений выводов устройства при этом исключаются. Если число обозначений выводов устройства размещаемой клеммы меньше, чем указанное здесь значение, клемма размещается полностью.

### Автоматически связать родственные распределенные клеммы

Если установлен данный флажок, распределенные клеммы функции клеммы автоматически связываются, даже если между ними нет обозначенных соединений. Для этого при обновлении соединений между родственными распределенными клеммами (одинаковое определение функции, одинаковое ОУ, одинаковое обозначение клемм) генерируются внутренние перемычки. Данные автоматически сгенерированные внутренние перемычки соответственно располагаются на последнем выводе устройства функции клеммы. Для генерирования внутренних перемычек решающее значение имеет последовательность распределенных клемм в навигаторе клеммников.

Если флажок снят, то родственные распределенные клеммы на связываются автоматически. При контрольном прогоне выводится информация о распределенных клеммах, распознанных как родственные, но не связанных.

!!! note "Замечание:"

    Флажок Автоматически связать родственные распределенные клеммы должен быть установлен по умолчанию, чтобы обеспечить передачу потенциала внутри клеммы.

### Оптимизация перемычек

Если данный флажок установлен, то графически обозначенные соединения не используются при создании перемычек. Из графических соединений отображаются только распределенные клеммы, которые соединены друг под другом, но не видно, между какими отдельными клеммами имеются перемычки. При оптимизации перемычек (по возможности) вместо проволочных перемычек между клеммами генерируются мостовые перемычки. Также заново генерируются внутренние перемычки.

Мостовые перемычки, возникающие при оптимизации перемычек, всегда располагаются на первой распределенной клемме (с наименьшим обозначением вывода устройства) от последнего к предпоследнему выводу устройства клеммы. Тип вывода устройства, настроенный для последнего вывода устройства клеммы, определяет вид возникающей перемычки / соединения. Обычно последний вывод устройства должен иметь тип вывода устройства 'Мостовая перемычка'.

Если данный флажок снят, то при создании перемычек используются графически обозначенные соединения. Из всех соединений при обновлении соединений между распределенными клеммами генерируются соответствующие внутренние, мостовые и проволочные перемычки. В этом случае следует убедиться в том, что в схеме соединений обозначены необходимые соединения.

!!! example "Пример:"

    В данном примере три клеммы размещены в схеме соединений в качестве распределенных клемм и графически многократно связаны между собой. Клемма1связана с клеммой3три раза, и клемма2связана с клеммой3также три раза. Чтобы при обновлении соединений родственные распределенные клеммы связывались внутренними перемычками автоматически, установлен флажок Автоматически связать родственные распределенные клеммы.При оптимизации перемычек генерируются две мостовые перемычки:одна между клеммой1и клеммой2и другая между клеммой2и клеммой3.Кроме того родственные распределенные клеммы связываются автоматически сгенерированными внутренними перемычками:пять внутренних перемычек генерируются между распределенными клеммами1:aи1:b,2:aи2:b,2:bи2:c,3:aи3:b,3:bи3:c. Даже если между распределенными клеммами2:bи2:cнет графического соединения, они также распознаются как родственные (по критериям: одинаковое определение функции, одинаковое ОУ, одинаковое обозначение клеммы).

!!! note "Замечание:"

    Сетевые соединения невозможно использовать вместе с оптимизацией перемычек распределенных клемм. Обе функции генерируют различные соединения, несовместимые друг с другом.
