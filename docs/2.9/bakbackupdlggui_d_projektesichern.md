# Диалоговое окно Резервировать проекты / Настройки: Проекты

Вы открыли проект.

* Вы выделили проект в Навигаторе страниц. Проект > Резервировать > Проект.
* Вы настроили Управление проектами и выделили в дереве управления проектами, как минимум, один проект. Выберите пункт всплывающего меню Резервировать.
* Параметры > Настройки > Пользователь > Резервирование данных > Предварительные настройки > Проекты

В этом диалоговом окне задается способ и место резервирования выбранных проектов.

Обзор основных элементов диалогового окна:

### Выбранные проекты

В этом списке отображаются выбранные для резервирования проекты.

### Описание

В этом поле можно ввести информативный текст (напр., указание места хранения), который сохраняется вместе проектом и отображается в EPLAN при резервировании и восстановлении.

### Метод

В этом поле задается способ резервирования проекта. Настройки влияют на возможность дальнейшей обработки исходного проекта и на формат файлов резервирования.

* Дополнительно сохранить: проект резервируется на другом носителе данных. Могут быть выделены несколько проектов, которые будут резервированы последовательно. Выбрать одновременно несколько проектов можно в структуре управления проектами, представленного в виде дерева. Проект *.elk сохраняется в том же каталоге. Дополнительно в выбранном каталоге резервирования генерируется файл резервирования *.zw1.
* Выгрузить для внешней обработки: проект резервируется на другом носителе данных, а на исходный проект устанавливается защита от записи. Все данные выгруженного проекта остаются на исходном жестком диске. Из проекта *.elk в том же каталоге получается проект *.els. Дополнительно в выбранном каталоге резервирования генерируется файл резервирования *.zw1.
* Архивировать: проект резервируется на другом носителе данных, а исходный проект, за исключением информационного файла, удаляется с жесткого диска. Из проекта *.elk в том же каталоге получается проект *.ela. Дополнительно в выбранном каталоге резервирования генерируется файл резервирования *.zw1.

### Имя файла резервирования

Здесь необходимо указать имя файла резервирования. По умолчанию используется имя проекта. Нажав ++...++, откройте диалоговое окно Выбор файла резервирования и перейдите к требуемому каталогу или создайте новый.
В зависимости от того, находится ли проект непосредственно в каталоге проекта или в каталоге за пределами каталога проекта, будет автоматически предложено имя проекта. Если проект находится в подкаталоге каталога проекта, то первым в раскрывающемся списке предлагается имя проекта, а вторым — путь файла ниже каталога проекта (разделенный на пункты) и имя проекта.
Если резервируются несколько проектов, то это поле недоступно, и в качестве имени файла резервирования используется соответствующее имя проекта.
Имя файла, включая путь файла, должно содержать не более 260 символов.

### Носитель для резервирования

В данном поле указывается, будет ли проект сохранен на носитель данных (жесткий диск, сменный диск и т. д.) или отправлен по электронной почте. Имя файла задается автоматически при сохранении и его нельзя изменить.

* Носитель данных: при выборе этой опции проект резервируется на определенном носителе данных.
В поле Каталог для резервирования укажите место хранения. В качестве целевых для резервирования данных можно указать любой дисковод и подкаталог. Если в поле Имя файла резервирования выбран каталог, то этот каталог отображается здесь.
Нажав ++...++, откройте диалоговое окно Выбрать каталог и перейдите к нужному каталогу или создайте новый.
С помощью пункта всплывающего меню Вставить переменную пути перейдите в диалоговое окно [Выбрать переменную пути](modaldialogsdb_d_pfadvariablen.md), из которого можно будет скопировать одну из доступных переменных пути.
* Электронная почта: при выборе этой опции проект будет отправлен по электронной почте. Вы можете отправить по электронной почте за один раз только один проект. После запуска функции проект запаковывается посредством внутреннего архиватора, после чего запускается настроенный редактор электронной почты.
Из раскрывающегося списка Ограничение размера выберите максимальный размер для отдельных файлов, отправляемых по электронной почте. Вы можете установить значение от 1 до 10 Mб с шагом 1 Mб или выбрать запись "Неогранич.". EPLAN автоматически создаст то количество файлов для отправки по электронной почте, которое необходимо для отправки всех частей. Всем частям будет сопутствовать следующий текст: "<Имя проекта> n/m". При этом "n" — это номер отдельного файла для отправки по электронной почте, а "m" — общее количество файлов.

!!! note "Замечание:"

    Для обмена электронными сообщениями на вашем компьютере должен быть установлен и настроен клиент электронной почты, поддерживающий MAPI. По этой же причине не может применяться почтовое приложение Windows 8®.

### Параметры

здесь задается фактический объем резервирования проектов.

* Полностью резервировать проект: если этот флажок установлен, в каталоге проекта резервируются все файлы. Сюда также входят все файлы, содержащие избыточные данные и использующиеся только для быстрого доступа к файлам данных проекта. Файл резервирования будет несколько больше, чтобы быстрее восстановить проект.
Если этот флажок не установлен, резервироваться будут только действительно нужные данные. Файлы, использующиеся только для быстрого доступа к файлам проекта, не сохраняются; они повторно генерируются при восстановлении проекта.
В независимости от того, установлен флажок или нет, всегда резервируются также сохраненные основные данные. Внешние документы и графические файлы резервируются, только если установлен соответствующий флажок.
* Копировать данные, связанные со ссылками: если этот флажок установлен, в проекте сохраняются связанные внешние документы и графические файлы. Соответствующие файлы копируются в подкаталоги ...\<Имя проекта>\DOC\\*.* и ...\<Имя проекта>\Images\\*.*. Чтобы сохранить эти данные, должны быть установлены флажки Зарезервировать сохраненные внешние документы и / или Зарезервировать сохраненные графические файлы или флажки Полностью резервировать проект.
* Зарезервировать сохраненные внешние документы: если этот флажок установлен, сохраненные в проекте внешние документы, расположенные в подкаталоге ...\<Имя проекта>\DOC\\*.*, будут резервироваться вместе с проектом.
* Зарезервировать сохраненные графические файлы: если этот флажок установлен, сохраненные в проекте графические файлы, находящиеся в подкаталоге ...\<Имя проекта>\Images\\*.*, будут резервироваться вместе с проектом.

### [Связи]

Открывает диалоговое окно [Скопировать внешние файлы в проект](eservicesgui_d_insprojektkopieren.md); здесь в текущий проект можно скопировать графические файлы и внешние документы, на которые в проекте или в соответствующих основных данных имеются ссылки.

!!! note "Замечание:"

    EPLAN запоминает все настройки для соответствующего проекта и снова предлагает их при повторном процессе резервирования. Предварительные настройки, заданные в настройках пользователя, используются только при первом резервировании проекта, так как в этом случае специфические для проекта настройки пока отсутствуют.
