# Диалоговое окно Настройки: Кабель

Вы открыли проект. Параметры > Настройки > Проекты > "Имя проекта" > Устройство > Кабели.

В этом диалоговом окне задаются предварительные настройки для единицы измерения длины кабелей и последовательности присвоения кабельных соединений.

Обзор основных элементов диалогового окна:

### Кабельное соединение всегда сортировать по последовательности штекеров

Если флажок установлен, сортировка кабельных соединений между клеммами и штекерами всегда осуществляется по последовательности обозначений контактов штекера.

Если флажок снят, клеммы и штекеры равноправны, а сортировка кабельных соединений осуществляется по последовательности сортировки штекеров или клемм источника.

### Игнорировать подчиненные ОУ в случае вложенных целей

Данная настройка влияет как на нумерацию кабелей, так и на автоматическое генерирование кабелей и вывод кабелей в отчетах.

Если данный флажок установлен, подчиненные ОУ для вложенных устройств не будут выводиться как цели кабелей. В этом случае целями будут являться вышестоящие устройства (например, черные ящики). Отображение целей для кабельных соединений не изменяется, здесь в качестве целей всегда выводятся подчиненные ОУ.

Если данный флажок не установлен, в качестве целей кабелей выводятся подчиненные ОУ.

!!! example "Пример:"

    Двасоединенияпроходят между клеммами разных клеммников в двух черных ящиках-U1и-U2.***Случай 1: Флажок не установлен***В этом случае***клеммники***являются целями кабеля. При автоматическом генерировании кабелей генерируются два кабеля: один между клеммниками-U1-X1и-U2-X3и другой между клеммниками-U1-X2и-U2-X4.***Случай 2: Флажок установлен***В этом случае***черные ящики***являются целями кабеля. При автоматическом генерировании кабелей создается только кабель между черными ящиками-U1и-U2.

### Единица измерения длины кабеля

Выбранная здесь единица измерения используется при выводе схем кабеля, если при определении кабеля вручную в схеме соединений единица измерения для длины кабеля не внесена. В самой схеме соединений определение кабеля не изменяется. Обзор возможных единиц измерения приводится в соответствующем разделе "[Соединения: Единицы измерения длины и поперечного сечения](connectionbrowsergui_k_einheitenverbindungen.md)".

### Поперечные сечения с пробелом

Если этот флажок установлен, при комбинированном запросе поперечного сечения / диаметра и единицы измерения (например, с помощью свойства Кабель / Группа соединений: Поперечное сечение/диаметр с единицей измерения или подобного свойства) между значением и единицей измерения вставляется пробел.

### Длины с пробелом

Если этот флажок установлен, при комбинированном запросе длины кабеля и соединения и вывода единицы измерения (например, с помощью свойства Кабель / Группа соединений: Длина с единицей измерения или подобного свойства) между значением и единицей измерения вставляется пробел.

### Стандартный кабель

В этом поле можно задать кабель, который по умолчанию используется для переноса свойств жилы на соединения. С помощью кнопки ++...++ откройте диалоговое окно Выбор изделия и выберите там имеющийся кабель. Этот кабель автоматически сохраняется в проекте.

Если стандартный кабель задан, при черчении линии определения кабеля свойства жилы переносятся на соединения. При этом следует различать два случая:

* Если ни одно из соединений, пересекаемых линией определения кабеля, не принадлежит кабелю, соединениям присваиваются кабельные соединения ***стандартного кабеля***. Это происходит на основе идентифицирующих свойств, учитываемых также при автоматическом присвоении подходящих кабельных соединений. На соединения переносятся только свойства из шаблонов функций. В кабель не копируются ни изделие, ни тип кабеля, ни другие (неприсвоенные) шаблоны функций стандартного кабеля.
* Если кабелю принадлежит по меньшей мере одно из соединений, пересекаемых линией определения кабеля, то линия определения кабеля получает первое найденное ОУ кабеля, а новым добавляемым соединениям присваиваются свободные кабельные соединения уже имеющегося кабеля (по мере возможности).

Если стандартный кабель не задан, при черчении линии определения кабеля присваивания кабельных соединений не происходит.
