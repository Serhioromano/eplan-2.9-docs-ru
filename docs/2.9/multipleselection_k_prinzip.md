# Групповая обработка: Принцип

Принцип выполнения групповой обработки не имеет больших отличий для всех объектов. Сначала необходимо выделить обрабатываемые объекты. После этого необходимо вызвать диалоговое окно "Свойства". Затем можно изменить или синхронизировать общие свойства выделенных объектов.

Однако существуют некоторые специфические для объектов различия при групповой обработке проектов, страниц, условных обозначений, данных проекта и изделий:

### Проекты

Свойства проекта можно поблочно обрабатывать только в управлении проектами. Для этого соответствующие проекты необходимо выделить в дереве управления проектами. После этого свойства выделенных проектов сразу же выводятся на соответствующих вкладках, где их можно совместно изменять после активирования режима обработки.

### Страницы

Свойства страницы можно обрабатывать поблочно только для ***определенного*** проекта в навигаторе страниц. Для этого следует выбрать соответствующие страницы проекта в дереве или в списке. После вызова пункта всплывающего меню Свойства свойства выделенных страниц выводятся в отдельном диалоговом окне Свойства, где их можно совместно обработать для всех страниц сразу.

!!! note "Замечание:"

    В навигаторе страниц ***невозможно*** поблочно обрабатывать страницы, которые относятся к разным проектам!

### Условные обозначения

Условные обозначения схемы соединений имеют разные свойства. Устройства "общего назначения" имеют свойства, отличные от свойств, например, клемм или кабелей. Если же сравнить клеммы и кабели друг с другом, то и эти объекты имеют собственные, совершенно типичные для них свойства. Таким образом, условные обозначения — это такие объекты, которые –, если говорить об их свойствах, – можно объединить в один тип. Объекты одного типа всегда имеют одинаковые свойства.

Групповая обработка условных обозначений возможна только на текущей открытой странице схемы соединений. Для ее выполнения сначала следует выделить необходимые объекты. Если в выделение включены разные типы объектов, например устройства "общего назначения", клеммы и кабели, после выбора пункта меню Обработать > Свойства открывается диалоговое окно Выбор объекта. Выберите в этом диалоговом окне тот тип объекта, свойства которого требуется изменить поблочно. После выбора объекта откроется диалоговое окно Свойства ++...++ для выбранного типа объекта. Отличающиеся значения свойств помечаются EPLAN в этом диалоговом окне записью ***< ++...++ >*** в соответствующем поле. Вместо этих или других записей можно записать значения, которые требуется присвоить всем выделенным объектам. После завершения групповой обработки все выбранные объекты одного типа имеют одинаковые значения для обработанных свойств.

#### Независимые от функций свойства условных обозначений

EPLAN также позволяет одновременно изменять свойства разных типов объектов, то есть разных условных обозначений. Эта форма групповой обработки называется ***функционально-независимой групповой обработкой***.

При этом способе групповой обработки обрабатываются только свойства, которыми совместно обладают все различные условные обозначения, то есть свойства, независимые от функций. К примеру, таким образом можно совместно обработать функциональный текст, данные гравировки или технические параметры устройств общего назначения, клемм и кабелей.

Для обработки условных обозначений независимо от функций необходимо – так же, как при обычной групповой обработке – заранее выбрать область схемы соединений с условными обозначениями. Тогда после выбора через пункты меню Обработать > Обзор свойств открывается [Обзор свойств](multipleselection_k_start.md). В этом диалоговом окне затем можно совместно обрабатывать и унифицировать независимые от функций свойства выделенных условных обозначений посредством прилагаемой схемы для конфигурации свойств.

### Прочие объекты на страницах

К прочим объектам, которые можно поблочно обрабатывать на страницах, относятся тексты и графические элементы. С принципиальной точки зрения групповая обработка этих объектов мало чем отличается от групповой обработки условных обозначений. Выберите требуемые объекты и обработайте их свойства в общем диалоговом окне. Если выбраны объекты разных типов, перед групповой обработкой необходимо выбрать один тип объектов.

!!! note "Замечание:"

    Кроме того, поблочно можно обрабатывать и сгруппированные графические элементы.

### Данные проекта

Данные проекта можно просмотреть в отдельных окнах ( – навигаторах –), в которых можно настроить представление в виде дерева или списка для имеющихся в проекте функций и их свойств.

К таким окнам относятся навигатор страниц, навигатор устройств и другие навигаторы, например, диалоговые окна Клеммники — <Имя проекта> или Кабели — <Имя проекта>. В этих окнах можно выполнить групповую обработку ***зависимых от функций*** свойств. Для этого следует выделить соответствующие функции в дереве или списке требуемого окна.

Если в выделении находятся разные типы объектов, например устройства общего назначения, клеммы и кабели, здесь также открывается диалоговое окно Выбор объекта. Выберите в этом диалоговом окне тот тип объекта, свойства которого требуется изменить. После этого открывается диалоговое окно "Свойства" выбранного типа объекта.

!!! note "Замечание:"

    * Во всех навигаторах возможна групповая обработка зависимых от функций свойств ***только*** для текущего открытого проекта!
    * В навигаторе устройств можно поблочно обрабатывать размещенные и неразмещенные устройства текущего открытого проекта.

### Данные изделия

В базе данных изделий строки, содержащие различные значения, при одновременной обработке нескольких объектов (изделий, списков принадлежностей и т. д.) также обозначаются строкой символов ***<<…>>***. При разном количестве строк можно обрабатывать только те строки, которые соответствуют всем выбранным объектам. Если к выбранным объектам добавить новую строку, она будет сгенерирована за последним общим изделием.

!!! example "Пример:"

    Кроме того, возможно поблочное удаление всех строк, даже если эти строки присутствуют не во всех обработанных объектах.
