# Импортировать или синхронизировать списки присвоений

Списки присвоений можно импортировать в форматы Siemens *.SDF и *.SEQ или как файлы ASCII *.ASC. Кроме того, можно импортировать файлы с любыми расширениями имени файла, которые соответствуют необходимой структуре списков присвоений. Импорт всегда связан с ЦПУ. Вы можете импортировать данные или в имеющееся, или в новое ЦПУ.

!!! note "Замечание:"

    * Если вы осуществляете чистый ***импорт*** (т. е. без переписывания данных ПЛК в проекте), импортируйте данные в новое ЦПУ.
    * При импорте в уже имеющееся ЦПУ выполняется ***синхронизация*** данных.

### Импортировать список присвоений в имеющееся ЦПУ

1. Выберите следующие пункты меню: Данные проекта > ПЛК > Адреса / Списки присвоений
2. В диалоговом окне Адреса / Списки присвоения в поле Фильтр выберите фильтр, отображающий адреса только ***одного*** ЦПУ.
3. Выберите пункты меню ++Дополнительно++ > Импортировать список присвоений > Отображаемое ЦПУ.
4. Щелкните в диалоговом окне Импортировать / синхронизировать список присвоений в поле Имя файла по ++...++.
5. Выберите в диалоговом окне Выбрать файл список присвоений.

!!! note "Замечание:"

    Типы файлов, предлагаемые в диалоговом окне выбора файла, зависят от присвоенной ЦПУ схемы для специфических для ПЛК настроек. Эта схема отображается в верхней части диалогового окна Импортировать / синхронизировать список присвоений.

6. Щелкните по кнопке ++Открыть++.

7. В следующих полях диалогового окна [Импортировать / синхронизировать список присвоений](plcgui_d_importzuli.md) задайте язык, подходящий для списка присвоений, а также дополнительные настройки для импорта.
8. Щелкните по кнопке ++OK++.

!!! info "Для сведения:"

    Адреса из списка присвоений импортируются в проект согласно настройкам.

!!! info "Для сведения:"

    Имеющиеся адреса или символические адреса перезаписываются согласно настройке в поле Ссылочное значение.

!!! info "Для сведения:"

    Импортируются только адреса, соответствующие формату адреса для цифровых и аналоговых вводов и выводов. Если не установлен формат адресов (т. е. в диалоговом окне Настройки: Специфич. для ПЛК на вкладке Форматы адресов оба поля пустые), то импортируются все адреса.

!!! tip "Совет:"

    При помощи пункта всплывающего меню Создать фильтр для ЦПУ в таблице вы можете автоматически сгенерировать из данных выделенной строки фильтр для отображения адресов ПЛК. При этом в качестве критерия фильтра используются значения свойств Проект конфигураций (косвен.), Имя рабочей станции ПЛК (косвен.) и ЦПУ (косвен.).

### Импортировать список присвоений в новое ЦПУ

1. Выберите следующие пункты меню: Данные проекта > ПЛК > Адреса / списки присвоений
2. Выберите пункты меню ++Дополнительно++ > Импортировать список присвоений > Новое ЦПУ.
3. В диалоговом окне Новое ЦПУ в полях Проект конфигураций, Имя раб. станции и ЦПУ укажите нужные значения для нового ЦПУ.

!!! info "Для сведения:"

    В раскрывающихся списках отображаются все значения, которые вы вводили в других местах проекта для подобных функций.

4. В поле Специфические для ПЛК настройки выберите схему, задающую настройки для форматов адресов. С помощью кнопки ++...++ откройте диалоговое окно Настройки: Специфич. для ПЛК, в котором можно создать или обработать схему.
5. Щелкните по кнопке ++OK++.
6. Щелкните в диалоговом окне Импортировать / синхронизировать список присвоений в поле Имя файла по ++...++.
7. Выберите в диалоговом окне Выбрать файл список присвоений.

!!! note "Замечание:"

    Типы файлов, предлагаемые в диалоговом окне выбора файла, зависят от присвоенной ЦПУ схемы для специфических для ПЛК настроек. Эта схема отображается в верхней части диалогового окна Импортировать / синхронизировать список присвоений.

8. Щелкните по кнопке ++Открыть++.

9. В следующих полях диалогового окна [Импортировать / синхронизировать список присвоений](plcgui_d_importzuli.md) задайте язык, подходящий для списка присвоений, а также дополнительные настройки для импорта.
10. Щелкните по кнопке ++OK++.

!!! info "Для сведения:"

    Адреса из списка присвоений импортируются в проект согласно настройкам.

!!! info "Для сведения:"

    Импортируются только адреса, соответствующие формату адреса для цифровых и аналоговых вводов и выводов. Если не установлен формат адресов (т. е. в диалоговом окне Настройки: Специфич. для ПЛК на вкладке Форматы адресов оба поля пустые), то импортируются все адреса.
