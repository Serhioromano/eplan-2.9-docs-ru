# Редактор символов

Символы нужны, чтобы начертить схему соединений простым способом. Они содержат большое количество информации, такой как графические элементы, выводы устройства, привязка к символьной группе, логика и т.д. При этом связь между символом и функцией может выглядеть следующим образом:

* Символ ***не*** отображает функцию, например, угол, тройник.
* Символ отображает ***одну*** функцию, например, замыкающий контакт, размыкающий контакт.
* Символ отображает ***несколько*** функций, например, автомат защиты двигателя, трехфазный предохранитель.
* Символ отображает ***часть*** одной функции, например, вывод устройства, переменный контактный крючок.

Для управления символами имеются библиотеки символов. При этом в библиотеке символов могут содержаться сколько угодно символов. При обработке символа открывается и сохраняется не вся библиотека символов, а всегда только изменяемый символ. Это сокращает проблемы, которые могут возникнуть, если несколько пользователей одновременно обрабатывают одну библиотеку символов.

При работе с редактором символов следует учитывать некоторые особенности, отличающие редактор символов от графического редактора.

* Так как редактор символов не использует символы со ссылками, то при вставке символы раскрываются.
* Можно использовать только стандартные слои. Элементы, лежащие на свободных слоях, при вставке должны быть адаптированы соответствующим образом. Это особенно относится к макросам, DXF / DWG и буферу обмена.

Некоторые свойства символов можно предварительно задать в настройках проекта (Параметры > Настройки > Проекты). Чтобы их можно было использовать при создании символов, редактор символов должен иметь к ним доступ. По этой причине такие основные данные как символы могут обрабатываться только при открытом проекте.
