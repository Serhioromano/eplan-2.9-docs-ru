# Вкладка Технические данные

Сервисные программы > Изделие > Управление. В дереве или списке выделено одно изделие.

На этой вкладке указывается техническая информация, которая не зависит от выбранного типа изделия и группы продуктов (отдельная часть, кабели и т. д.). Эти данные можно указать ***один раз*** для каждого изделия.

Обзор основных элементов диалогового окна:

### Технические параметры

Введите здесь соответствующие значения. Так, например, для катушек контакторов имеется возможность перенести в технические параметры напряжение катушки и коммутационную способность.

Если для этого свойства на вкладке Шаблоны функций базы данных изделий введено значение, это значение отобразится здесь, а также скопируется при выборе изделия. В таком случае это поле нельзя обработать. При наличии нескольких шаблонов функций здесь отобразится значение первого "Главного шаблона". Главный шаблон — это первый шаблон функции на вкладке Шаблоны функций базы данных изделий, в определении функции которого установлено "Главная функция".

### Номер группы

Номер группы служит для различия отдельных групп изделий.

### Вид изделия

Виды изделий служат для группирования одинаковых изделий; так, напр., можно использовать группирование по силовым и вспомогательным контакторам с различными многими размыкающими и замыкающими контактами. Или, например, можно выделить в одну группу функциональные элементы, чувствительные к нагреву или вибрации. Просто выделите группируемые изделия и введите максимум 10 знаков в поле.

!!! example "Пример:"

    Создайте для чувствительных к нагреву функциональных элементов вид изделия "WE", для чувствительных к вибрации элементов вид изделия "VIB".

### Группа функций

Это поле служит в информативных целях и может, напр., использоваться для фильтрации при выборе изделий. Длина ввода ограничена 30 символами.

!!! example "Пример:"

    В схему соединений вставлен аварийный нажимной выключатель. Для его выбора можно разделить следующим образом:Группа функций: Переключатель Вид изделия: Кнопочный переключатель

### Макрос

Символы, представляющие собой различные варианты и положения коммутирующих элементов можно объединять в макросы, которые значительно ускоряют проектирование схемы соединений. Для размещения могут быть также использованы чертежи контуров, созданные в редакторе контуров как контуры типа "Выдавить контур".
В частности для модуля важно связывание с каким-нибудь макросом; если при сохранении модуля макрос не присвоен, будет выведено сообщение об этом.

С помощью пункта всплывающего меню Вставить переменную пути перейдите в диалоговое окно [Выбрать переменную пути](modaldialogsdb_d_pfadvariablen.md), из которого можно будет скопировать одну из доступных переменных пути.

!!! note "Замечание:"

    Соблюдайте наши [рекомендации по работы с 2D- и 3D-макросами](cabinetgui_k_3dmakros.md).

!!! tip "Совет:"

    Если какое-либо изделие с указанным таким образом макросом или чертежом контура вставить в список устройств, то данное изделие можно будет разместить в схеме соединений при помощи перетаскивания.

### Всплывающее меню

Всплывающее меню дает доступ, в зависимости от типа поля (например, дата, целое число, многоязычный), к пунктам меню, при помощи которых вы можете по необходимости, например, влиять на представление таблиц или обрабатывать значения в полях. Обзор пунктов этого всплывающего меню вы можете найти в разделе [Пункты всплывающего меню](userinterface_m_kontextmenu.md).
