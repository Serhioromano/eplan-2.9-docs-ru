# Вкладка ПЛК

Вы открыли проект.

* Вы сгенерировали объект планирования в навигаторе предварительного планирования или вставили его в Графическом редакторе. Выберите в диалоговом окне Свойства ++...++ вкладку ПЛК.
* Вы выделили объект планирования в навигаторе предварительного планирования или в Графическом редакторе. Обработать > Свойства (или Всплывающее меню > Свойства). Выберите вкладку ПЛК.

На этой вкладке запланируйте необходимые входы и выходы ПЛК для объекта планирования. Этим входам и выходам могут быть присвоены адреса ПЛК посредством выбора имеющихся адресов или создания новых. В качестве альтернативы с помощью свойства Символический адрес (отдельный элемент) можно сгенерировать однозначные символические адреса. Определенные на объекте планирования входы и выходы ПЛК можно вставить в список присвоений, который можно использовать для обмена данными с программами конфигурации ПЛК. Если вы присвоили вход или выход ПЛК объекта планирования выводу устройства ПЛК, нужно будет перенести адрес ПЛК и соответствующие данные на вывод устройства ПЛК.

Обзор основных элементов диалогового окна:

### Адреса ПЛК

В таблице имеются следующие столбцы:

* Строка: Непрерывный номер строки.
* Направление: Указывает на направление (вход, выход, не определено) адреса.
* Тип данных: Тип данных указывает на диапазон адресов. Возможными значениями являются, например, BOOL, BYTE, WORD и DWORD.
* Счетчик: Для входов ПЛК определите с помощью флажка, будет ли идти речь о счетчике. Данная информация служит для различия разных входов при автоматизации зданий.
* Проект конфигураций (косвен.): Введите здесь имя проекта конфигурации для блока ПЛК, к которому относится вывод устройства ПЛК, или выберите запись из раскрывающегося списка.
* Имя рабочей станции ПЛК (косвен.): Введите здесь имя станции для блока ПЛК, к которому относится вывод устройства ПЛК, или выберите запись из раскрывающегося списка.
* ЦПУ (косвен.): Введите здесь имя ЦПУ для блока ПЛК, к которому относится вывод устройства ПЛК, или выберите запись из раскрывающегося списка. ЦПУ однозначно идентифицируется путем ввода имени ЦПУ в формате [Проект конфигураций].[Имя станции].[Идентификатор ЦПУ].
* Символический адрес (отдельный элемент): На сегментах структуры и объектах планирования можно задать отдельный элемент символического адреса. Эти отдельные элементы тесно взаимодействуют друг с другом на входах и выходах ПЛК объектов планирования, начиная с верхнего узла, и отображаются в строчке Символический адрес (предварительное планирование). Подобным образом для входов и выходов ПЛК можно генерировать однозначные символические адреса.
* Символический адрес (предварительное планирование): Отображает комбинацию отдельных элементов символического адреса, которые были введены в объект планирования и вышестоящие сегменты. Содержание этого поля копируется в свойство Символический адрес (автоматически), если оно не заполнено.
* Адрес ПЛК: Реальный адрес ПЛК. Адрес можно ввести вручную или использовать. Если щелкнуть в ячейке по ++...++, откроется диалоговое окно Выбрать для использования имеющегося. Если выбрать здесь уже имеющийся адрес ПЛК и использовать его, остальные поля в строке таблицы автоматически заполнятся данными.
* Символический адрес: Здесь записывается автоматический символический адрес. Если щелкнуть в ячейке по ++...++, откроется диалоговое окно Выбрать для использования имеющегося. Если выбрать здесь уже имеющийся адрес ПЛК и использовать его, остальные поля в строке таблицы автоматически заполнятся данными.
* Функциональный текст (автоматически): Введите здесь функциональный текст для вывода устройства ПЛК.

!!! note "Замечание:"

    Записи в столбцах Проект конфигураций (косвен.), Имя рабочей станции ПЛК (косвен.), ЦПУ (косвен.) и Символический адрес идентифицируют входы и выходы ПЛК при синхронизации с выводами устройства ПЛК, при общей обработке и при генерации отчетов. (Однозначная идентификация ЦПУ выполняется на основании указанного проекта конфигурации, станции ПЛК и имени ЦПУ.)

### Панель инструментов

Кнопка |  Значение
---|---
![](images/all_rebook_as.png){: .ui-icon } (Возврат) |  Возвращает выделенный адрес ПЛК. Выделенная строка удаляется из таблицы, однако данные адреса ПЛК остаются в базе данных.
![](images/all_new_as.png){: .ui-icon } (Создать) |  Создает новую строку. При этом новая строка добавляется в конце списка.
Предварительной настройкой для новой строки является текущая выделенная, т. е. копируются ее данные по направлению, типу данных и т. д..
![](images/all_delete_as.png){: .ui-icon } (Удалить) |  Удаляет строку, в ячейке которой находится курсор, со всем содержимым, т. е. включая данные; при необходимости выполняется новая нумерация позиций. Возможен многократный выбор. Обратите внимание на то, что измененные данные записываются в базу данных только после того, как выполнен щелчок по кнопке ++Закрыть++ или ++Применить++.
![](images/all_arrowup_as.png){: .ui-icon } / ![](images/all_arrowdown_as.png){: .ui-icon } (Переместить вверх / вниз) |  Перемещает выделенные строки на соответственно одну запись вверх или вниз, если это возможно.

### Всплывающее меню

Всплывающее меню дает доступ, в зависимости от типа поля (например, дата, целое число, многоязычный), к пунктам меню, при помощи которых вы можете по необходимости, например, влиять на представление таблиц или обрабатывать значения в полях. Обзор пунктов этого всплывающего меню вы можете найти в разделе [Пункты всплывающего меню](userinterface_m_kontextmenu.md).
