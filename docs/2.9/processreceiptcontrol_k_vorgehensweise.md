# Нормоконтроль: Способ действий

Проверка проекта состоит из следующих шагов:

* [Генерировать исходный проект](revisionmgtgui_h_vergleichsprojekterzeugen.md): в качестве исходного проекта вы генерируете закрытый проект EPLAN с ревизиями (*.elr) из первоначального неизмененного проекта. В качестве исходных проектов можно также использовать стандартные проекты EPLAN (*.elk).
* Вы рассылаете свой первоначальный проект на обработку другим участникам проекта и получаете его назад в доработанном виде как проект с замечаниями. Вы также можете использовать проверку проектов, чтобы удостовериться, что ваш проект соответствует заранее установленному стандарту.
* [Создать схему сравнения настроек](processreceiptcontrol_h_vergleichsschema.md): укажите, какие специфические для проектов настройки в обоих проектах следует сравнить при помощи EPLAN, и сохраните эти требования в виде схемы.
* [Создать схему сравнения свойств](processreceiptcontrol_h_eigenschaftenvergleich.md): укажите, какие свойства в обоих проектах следует сравнить при помощи EPLAN, и сохраните эти требования в виде схемы.
* [Создать схему контрольных прогонов для проверки проекта](processreceiptcontrol_h_pruefschema.md): укажите, какие контрольные прогоны следует осуществлять при сравнении проектов в ходе проверки проекта в системе EPLAN, и сохраните эти требования в виде схемы.
* [Проверить проект](processreceiptcontrol_h_eingangskontrolvorber.md): проверьте, отвечает ли проект с замечаниями определенным вами требованиям.
* ***Отобразить результаты:*** после того как проект был проверен, EPLAN автоматически отображает результаты в диалоговом окне Проверка проекта: Результаты и управлении сообщениями. Таким образом, результаты находятся в проекте с замечаниями. Посредством функций копирования и вставки вы можете переносить результаты из управления сообщениями в другие приложения.
