# Диалоговое окно Настройки: Экспорт AutomationML

Вы открыли проект. Данные проекта > ПЛК > Экспортировать данные. В поле Формат файла экспорта вы выбрали формат экспорта AutomationML, а затем щелкнули [Параметры].

В этом диалоговом окне обозначьте степень детализации для файла экспорта в формате AutomationML.

Обзор основных элементов диалогового окна:

### Экспортировать подробное соединение портов

Эта настройка имеет значение только для следующих систем шин:

* [Системы шин на базе Ethernet](plcgui_k_spsanschluesse.md)
* DRIVE-CLiQ
* PortToPort
* ET-Connection
* Local-Bus: Расширение

Если этот флажок установлен, в файле экспорта подробно выводится соединение выводов шины. В дополнение к информации, связывающей блоки ПЛК друг с другом, отобразится вывод шины на первом блоке, который связан с выводом шины на втором блоке.

Если флажок снят, в файле экспорта выводятся только блоки ПЛК, которые связаны между собой. Информация о выводе шины, который связан с другим выводом шины, не отображается. Кроме того, соединения между каркасом и соответствующими расширяющими каркасами в этом случае не экспортируются.

### Экспортировать принадлежности

Если этот флажок установлен, предметы оборудования также выводятся в файле экспорта. В качестве принадлежностей учитываются все изделия, внесенные в диалоговое окно "Свойства" главной функции на вкладке Изделие в позициях с 2 по 50.

!!! note "Замечание:"

    Чтобы экспортировать предметы оборудования, обязательно укажите в базе данных изделий обозначение типа ПЛК для этих изделий.

Если флажок снят, предметы оборудования пропускаются.

### Экспортировать значения конфигурации, связанные с устройствами

Эта настройка доступна в формате экспорта AutomationML с версии V1.1.0. Если этот флажок установлен, значения конфигурации, связанные с устройствами, также выводятся в файле экспорта.

Если флажок снят или выделен серым, значения конфигурации, связанные с устройствами, во время обмена данными ПЛК игнорируются.

!!! note "Замечание:"

    Значения конфигурации, связанные с устройствами, определяются в базе данных изделий как определенные пользователем свойства через диалоговое окно [Конфигурировать свойства](eservicesgui_d_konfigeigenschaften.md). Чтобы при обмене данными ПЛК эти свойства учитывались в формате AutomationML, в поле Категория необходимо ввести значение "Данные ПЛК", а в поле Присвоение — значение "Изделие" и установить флажок Также активировать для ссылок изделий. Префикс (строка символов до первой точки) в идентифицирующем имени этих свойств удаляется при экспорте и дополняется из данных соответствующего изделия при импорте.
