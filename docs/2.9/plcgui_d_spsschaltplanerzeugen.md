# Диалоговое окно Генерировать схему соединений ПЛК

Вы открыли проект. Несколько выводов устройства ПЛК выделено в графическом редакторе, навигаторе ПЛК или в навигаторе устройств. Или одна или несколько страниц или проект выделены в навигаторе страниц. Данные проекта > ПЛК > Генерировать схему соединений.

В этом диалоговом окне выберите устройство ПЛК, которое затем должно быть автоматически размещено. Разместить при этом можно как неразмещенные функции ПЛК, так и те, что размещены только в определенных видах представлений. Если схема соединений ПЛК уже сгенерирована, автоматически размещенные функции ПЛК можно удалить и разместить заново. При открытии этого диалогового окна текущий проект полностью блокируется.

Обзор основных элементов диалогового окна:

### Таблица

В таблице отображаются устройства ПЛК, включенные в выборку. Устройство отображается всегда полностью, даже если в выборку включены только его части. Если выделен вывод устройства ПЛК, то будут отображены соответствующая карта ПЛК и все принадлежащие ей выводы. Каждая функция, однако, отображается в таблице только один раз, даже если в проекте она встречается на нескольких страницах (в различных видах представления). Отображаются также неразмещенные функции и шаблоны функций.

Таблица содержит следующие столбцы:

* Объекты: Здесь устройства ПЛК отображаются в форме древовидной структуры. На верхнем уровне находится ОУ. Разверните дерево, щелкнув по знаку ![](images/all_plussign_as.png){: .ui-icon }, чтобы отобразить блок ПЛК и все соответствующие ему выводы устройства.
* Файл макроса: Здесь отображается имя макроса, указанное на изделии, если функции присвоено соответствующее изделие. Щелкнув в строке по кнопке ++...++, можно открыть диалоговое окно Выбрать макрос и присвоить в нем макросы или заменить уже присвоенные. Когда выбран макрос с переменными и наборами значений, для переменных отображаются дополнительные столбцы.
* Статус: Во время присвоения макроса ПЛК выполняется проверка на наличие всех необходимых видов представлений макроса и их соответствие функциям устройства. Если макрос содержит все необходимые данные, отображается зеленая пиктограмма ![](images/all_statusok_as.png){: .ui-icon }. В строках макросов с недостающими данными отображаются желтые пиктограммы ![](images/all_statusnotok_as.png){: .ui-icon }. В случае отсутствующего или неподходящего макроса (в котором, например, содержится недостаточное количество выводов устройства) отображается красная пиктограмма ![](images/erroricon_error_systemmessages.png){: .ui-icon }.
* Функциональный текст: Здесь отображается функциональный текст, введенный для функции.
* Страница Однополюсный: Отображает имя страницы, на которой размещено однополюсное представление функции.
* Страница Многополюсный: Отображает имя страницы, на которой размещено многополюсное представление функции.
* Страница Обзор: Отображает имя страницы, на которой размещено обзорное представление функции (вариант макроса А).
* Страница Создание каркаса: Отображает имя страницы, на которой размещено обзорное представление функции для создания каркасов (вариант макроса Е). Отображаются имена только автоматически сгенерированных страниц.
* <Имя переменной>: Если присвоенный макрос содержит объекты-заполнители с переменными, то для каждой переменной будет отображен дополнительный столбец. При этом имя переменной отображается как заголовок столбца. Если в нескольких макросах содержатся переменные с одним и тем же именем, то для этого имени переменной будет отображен только один столбец. Через всплывающее меню в строках таблицы возможен выбор набора значений. Кроме того, ввести значение можно вручную.

### Панель инструментов

Кнопка |  Значение
---|---
![](images/all_new_as.png){: .ui-icon } (Создать) |  Открывает диалоговое окно для выбора дополнительных устройств ПЛК. При этом предлагаются все размещенные и неразмещенные устройства ПЛК проекта.
![](images/all_delete_as.png){: .ui-icon } (Удалить) |  Полностью убирает из списка выделенное устройство. Если, например, выделен только один вывод устройства, из списка будут убраны все остальные выводы и карта ПЛК.
![](images/all_export_as.png){: .ui-icon } (Экспортировать) |  Открывает диалоговое окно Сохранить как. Позволяет экспортировать конфигурацию диалогового окна.
![](images/all_import_as.png){: .ui-icon } (Импортировать) |  Открывает диалоговое окно Открыть. Позволяет импортировать сохраненнную конфигурацию диалогового окна.

### Всплывающее меню

Всплывающее меню дает доступ, в зависимости от типа поля (например, дата, целое число, многоязычный), к пунктам меню, при помощи которых вы можете по необходимости, например, влиять на представление таблиц или обрабатывать значения в полях. Обзор пунктов этого всплывающего меню вы можете найти в разделе [Пункты всплывающего меню](userinterface_m_kontextmenu.md).

Дополнительно здесь представлены следующие пункты всплывающего меню, специфические для данного диалогового окна:

Пункт меню |  Значение
---|---
Рассчитать страницы |  Позволяет в диалоговом окне Начальн. страницы ввести названия первых начальных страниц для всех активированных видов представлений. Остальные начальные страницы для других устройств рассчитываются и вносятся в таблицу автоматически.
Присвоить набор значений |  Открывает диалоговое окно Выбрать набор значений — <Имя заполнителя>, в котором можно присвоить набор значений переменным в выделенном макросе. Переменные предусмотрены не для присвоения значений функциям ПЛК, а только для ***дополнительных*** реализованных в макросе функций.
Очистить переменные |  Сортирует столбцы переменных в алфавитно-цифровом порядке; неиспользуемые столбцы переменных при этом удаляются.
Показать сообщения |  Открывает диалоговое окно Сообщения, в котором отображаются несоответствия между функциями устройства и присвоенным ему макросом (например, вид представления, вариант макроса, вывод устройства).
Свойства (общие) |  Открывает диалоговое окно Свойства (общие): ++...++. Позволяет обрабатывать свойства устройства.

### Генерировать однополюсное представление

Если этот флажок установлен, то генерируются однополюсные представления. На них размещаются макросы с видом представления "Однополюс.", начиная с варианта макроса А.

### Генерировать многополюсное представление

Если этот флажок установлен, то генерируются многополюсные представления. На них размещаются макросы с видом представления "Многополюс.", начиная с варианта макроса А.

### Генерировать обзорные представления

Если установлен этот флажок, то генерируются обзоры В/В. На этих обзорах размещаются макросы варианта А с видом представления "Обзор".

### Генерировать обзорные представления каркаса

Если установлен этот флажок, то генерируются обзоры для создания каркасов. На этих обзорах размещаются макросы варианта Е с видом представления "Обзор".

!!! note "Замечание:"

    Тип страниц, создаваемых при генерации различных представлений, зависит от типов страниц макросов. (При создании макроса автоматически сохраняется тип текущей страницы.) Так, например, при генерации обзорного представления макрос обзора может быть размещен на многополюсной странице, если первый размещенный макрос окна относится к типу страниц "Многопол. схема соедин."
