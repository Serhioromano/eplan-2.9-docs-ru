# Диалоговое окно Присвоение "определения функции к графике"

Вы открыли проект. Сервисные программы > Основные данные > Форма > Открыть > [Открыть]. Вы открыли диалоговое окно Свойства формы — <Имя формы> (Всплывающее меню > Свойства на форме в диалоговом окне навигатора Страницы — <Имя проекта>) и в поле Значение свойства Присвоение определения функции графике щелкнули кнопку ++...++.

В этом диалоговом окне укажите, для какого определения функции должна быть выведена определенная графика в области данных. Это свойство необходимо, чтобы в спецификации клеммника могла быть выведена (фиксированная) графика для клемм определенного типа клемм (конкретного определения функции).

Если после определения присвоения в данном диалоговом окне в форме разместить свойство Присвоение определения функции графике [n], то символ будет размещен в зависимости от определения выводимой функции.

Обзор основных элементов диалогового окна:

### Категория / Группа / Описание

Как только было выбрано определение функции, в этих полях выводятся соответствующие значения.

!!! example "Пример:"

    В качестве определения функции была выбрана 'проходная клемма, 4 цели, с мостовой перемычкой'; поля заполняются следующим образом:Категория:Клеммы Группа:Клеммы (2 вывода устройства)Описание:Проходная клемма с четырьмя целями и с возможностью установки мостовых перемычек.

### Определение функции

Щелкните по ++...++, чтобы в этом поле открыть диалоговое окно Определение функции и выбрать определение функции. Многократный выбор здесь ***невозможен***.

!!! note "Замечание:"

    Как вкладка Атрибуты, так и вкладка Данные соединения выбранного определения функции здесь ***не*** обрабатываются!

### Библиотека символов

При выборе символа в этом поле выводится библиотека символов.

### Имя символа

В это поле вводится символ, графика которого должна быть присвоена определению функции: Щелкните кнопку ++...++, чтобы открыть диалоговое окно Выбор символа и выбрать символ. Многократный выбор здесь ***невозможен***.

!!! note "Замечание:"

    Обратите внимание на то, что на просмотр выводятся только символы ***сохраненных*** библиотек символов. Если эта форма будет использоваться в проекте, в котором отсутствует или не сохранена соответствующая библиотека символов, то в отчете эти условные обозначения будут отсутствовать!

### Всплывающее меню

Всплывающее меню дает доступ, в зависимости от типа поля (например, дата, целое число, многоязычный), к пунктам меню, при помощи которых вы можете по необходимости, например, влиять на представление таблиц или обрабатывать значения в полях. Обзор пунктов этого всплывающего меню вы можете найти в разделе [Пункты всплывающего меню](userinterface_m_kontextmenu.md).
