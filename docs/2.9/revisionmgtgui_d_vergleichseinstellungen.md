# Диалоговое окно Настройки: Сравнение свойств проектов

Вы открыли проект. Вы создали исходный проект.

* Сервисные программы > Управление ревизиями > Сравнение свойств проектов > Сравнить проекты. В диалоговом окне Сравнить свойства проектов рядом с полем Настройки для сравнения свойств щелкните по ++...++.
* Параметры > Настройки > Фирма > Управление > Сравнение свойств проектов.
* Вы открыли управление проектами и выделили проект. [Дополнительно] > Проверить проект. Щелкните в диалоговом окне Проверить проект рядом с полем Сравнить свойства по ++...++.

В этом диалоговом окне задается следующее: сравнение каких свойств для каких типов объектов должно быть выполнено. По умолчанию для большинства типов объектов производится сравнение значений "Графического размещения", т. е. положения соответствующего объекта на странице. Эти настройки сохраняются в схеме.

Обзор основных элементов диалогового окна:

### Схема

Выберите из раскрывающегося списка схему, которую требуется использовать или обработать. Для этого также воспользуйтесь соседней [панелью инструментов](schemeconfig_m_schemaschaltflaechen.md).

### Категория

Выберите из раскрывающегося списка тип объекта, для которого требуется задать сравниваемые свойства. В зависимости от сделанного выбора (устройство, соединения и т. д.) в полях Не сравнивать и Сравнить будут перечислены различные свойства.

Выделите в поле Не сравнивать нужные свойства и переместите их с помощью кнопки со стрелкой в поле Сравнивать. Ненужные свойства можно переместить обратно в поле Не сравнивать. Настройки можно выполнить для каждого типа объектов отдельно и сохранить их все в выбранной выше схеме.

!!! example "Пример:"

    Предположим, необходимо найтиустройства, функциональный текст которых был изменен. Чтобы выполнить это, для типа объектов Устройствов поле Не сравниватьразверните структуру дерева под слоем "Данные", выделите свойства Функциональный тексти Функциональный текст (автоматически)и при помощи кнопки со стрелкой переместите их в поле Сравнить.

!!! note "Замечание:"

    * Чем больше свойств выбрано для сравнения, тем больше времени потребуется для сравнения.
    * При сравнении свойств с альтернативной идентификацией сравниваются только размещенные соединения. Свойства точек обозначения соединения при этом учитываются косвенно, так как эти свойства переносятся на соединения.
    При 'нормальном' сравнении свойств проектов сравниваются только неразмещенные соединения. Дополнительно размещенные соединения можно сравнить по свойствам точек обозначения соединений.
    * Проведите сравнение свойств для сегментов предварительного планирования; тогда при изменении в ***свойствах*** сегмента как сегмент в навигаторе предварительного планирования, так и размещение в графическом редакторе отобразятся с маркировкой изменения.
    Если, напротив, изменяется только ***графическое размещение*** сегмента (например, сегмент перемещен на странице), при сравнении свойств с одинаковой настройкой маркировку не получает ни соответствующий сегмент в навигаторе предварительного планирования, ни размещение.

### Учитывать удаленные и новые объекты

Если этот флажок установлен, удаленные и новые объекты типа объектов, указанного в поле Категория, будут рассматриваться как изменение. Эти объекты будут сравниваться и тогда, когда для сравнения не выделено никаких свойств (т. е. поле Сравнить пусто). Удаленные объекты вносятся в список результатов ревизии, но не отображаются посредством маркера ревизии в схеме соединений.

Если флажок не установлен, удаленные и новые объекты данного типа объектов игнорируются. Это позволяет ускорить процесс сравнения свойств.

!!! tip "Совет:"

    Для ускорения процесса сравнения свойств проектов рекомендуется сравнивать только действительно необходимые типы объектов. Все остальные типы объектов следует полностью исключить из сравнения. С этой целью свойства данных типов объектов не должны быть выбраны для сравнения; кроме того, флажок Учитывать удаленные и новые объекты должен быть снят.
