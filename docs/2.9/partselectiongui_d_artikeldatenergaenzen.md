# Диалоговое окно Дополнить данные изделия

* Вы открыли базу данных изделий. Сервисные программы > Изделие > Управление > [Дополнительно] > Настройки. В диалоговом окне Настройки: Изделие (пользователь) щелкните по кнопке ++...++ в поле Дополнить данные изделия через.
* Вы открыли проект и выделили главную функцию в графическом редакторе или навигаторе. Обработать > Свойства. Выберите вкладку Изделие и щелкните рядом с полем Источн. данных по ++...++. В диалоговом окне Настройки: Изделие (пользователь) щелкните по кнопке ++...++ в поле Дополнить данные изделия через.

### Дополнить данные изделия через

Если данные изделия находятся в разных базах данных, желательно, чтобы они считывались в "один проход" при выборе изделия. Ведь при выборе схемы для выбора изделия можно не только определить источник данных (поле Схема диалогового окна Настройки: Изделие (пользователь)), но и дополнительно выбрать другие источники данных, которые также необходимо определить в качестве схемы. Соответствующее присвоение выполняется в следующем поле:

Сначала щелкните по ![](images/all_new_as.png){: .ui-icon } (Создать), чтобы сгенерировать строку в списке привязок. После этого выберите из раскрывающегося списка схему, для которой будет действительна настройка, и установите флажок в столбце Генерировать изделия, если отсутствуют для каждой схемы. Вместе с номером изделия и вариантом, которые при создании новых записей данных получают значение Создать_<порядковый_номер>, свойство Группа функций (на вкладке Технические данные) также заполняется значением "Новое", что позволяет легко находить изделия, сгенерированные путем выбора изделий.

!!! example "Пример:"

    В поле Схемадиалогового окна Настройки: Изделие (пользователь)выбрана настройка "Стандарт", инастройки, определенные в этой схеме, учитываются при выборе изделия. В раскрывающемся списке Имя схемыполя Дополнить данные изделия черезвы выбрали обе схемы: "KundeA_SQL" и "KundeA_API"; если флажок Генерировать изделия, если отсутствуютустановлен, то соответствующие изделия генерируются в базе данных изделий.
