# Диалоговое окно Свойства символа — <Имя библиотеки>

* Вы открыли проект. Сервисные программы > Основные данные > Символ > Создать. если открыто несколько библиотек символов, то в диалоговом окне Выбор библиотеки символов выберите библиотеку символов, в которой должен быть создан символ. Выберите создаваемый вариант символа.
* Вы открыли проект и символ из библиотеки символов для обработки в редакторе символов. Выберите пункт всплывающего меню Свойства для символа в представлении в виде дерева или списка в диалоговом окне навигатора Страницы — <Имя проекта>.

Задайте в этом диалоговом окне свойства создаваемого или открытого символа.

Обзор основных элементов диалогового окна:

### Номер символа

Номер символа указывает на позицию символа в пределах библиотеки символов. Этот номер используется EPLAN только для внутренних, связанных с отчетами целей, и является уникальным для соответствующей библиотеки символов. С помощью раскрывающегося списка символу можно присвоить новый номер; показываются еще не распределенные номера.

### Имя символа

Имя символа обычно начинается с буквенного обозначения ОУ (предустановка) и может иметь любую длину. В этом поле Вы можете ввести соответствующие данные или выбрать имя из раскрывающегося списка; показываются все имена символов, имеющиеся в библиотеке.

### Определение функции

если необходимо изменить логику символа, то щелкните в этом поле по ++...++, чтобы открыть диалоговое окно Определения функций.

### Выводы устройства

Это поле доступно, если в поле Определение функции выбрано определение функции с дополнением 'различно' для различного количества выводов устройства символов.

### [Логика]

Открывает диалоговое окно Логическая схема выводов устройства. Показывает данные выводов устройства определения функции, которую вы присвоили символу через поле Определение функции. Позволяет подгонять данные вывода устройства к символу. Если вы выбираете другое определение функции, данные выводов устройства, связанные с символом, будут заменены данными выводов устройства другого определения функции и могут быть в свою очередь подогнаны к символу. При вставке символа на схему соединений используются специфические для символа данные выводов устройства, если они отличаются от данных выводов устройства определения функции на функции. Эта функциональность полезна, к примеру, если вам нужна специальная логика для определенных символов.

### Список Имя свойства / Значение

В зависимости от выбранной категории отображаются возможные свойства; некоторым из них, таким как "Автор", "Последний обработчик" и "Дата создания", уже присвоены значения, которые не могут быть изменены. Другим, как "Описание символа" и "Версия", может быть присвоено новое или изменено уже имеющееся значение.

### Всплывающее меню

Всплывающее меню дает доступ, в зависимости от типа поля (например, дата, целое число, многоязычный), к пунктам меню, при помощи которых вы можете по необходимости, например, влиять на представление таблиц или обрабатывать значения в полях. Обзор пунктов этого всплывающего меню вы можете найти в разделе [Пункты всплывающего меню](userinterface_m_kontextmenu.md).

!!! note "Замечание:"

    Свойства символа действительны для ***всех*** вариантов этого символа.
