# Предварительное планирование: Принцип

Структурирование в предварительном планировании осуществляется при помощи так называемых сегментов, которые подразделяются на сегменты структуры и объекты планирования.

### Сегменты структуры

Сегменты структуры задают часть структуры проекта. Тем самым предварительно определяются части ОУ. Один сегмент структуры может замещать один или несколько структурных идентификаторов (обычно установку) в детальном проектировании. Из структурных идентификаторов на более поздних этапах складывается структура проекта.

Сегменты структуры можно создавать и обрабатывать в навигаторе предварительного планирования. На сегменте структуры можно анализировать свойства для необходимых затрат (время, энергия и расходы) по подчиненным объектам планирования.

Сегменты структуры можно размещать в качестве обозначений местоположения на страницах типов страницы "Предварительное планирование" и "Функциональная схема автоматизации". На страницах предварительного планирования из графического расположения сегментов определяется структура ([Графическое предварительное планирование](planninggui_k_grafischevorplanung.md)). На страницах функциональной схемы автоматизации обозначение местоположения служит только для [отображения структуры](planningri_k_struktursegmente.md), для этого генерируется ссылка на сегмент структуры.

### Объекты планирования

Объекты планирования определяют часть машины или установки в предварительном планировании; они описывают устройство и имеют связь с функциями устройства. При этом устройство представляет функцию машины или установки (например, электродвигатель для ленточного конвейера). Каждый объект планирования может представлять только ***одно*** устройство.

На объектах планирования можно сохранять макросы. Такой макрос может содержать несколько устройств, так что объект планирования представляет несколько устройств. Однако объект планирования определяет через свои свойства только одно устройство — центральное устройство функциональности.

Объекты планирования можно создавать и обрабатывать в навигаторе предварительного планирования. Также объекты планирования можно создавать путем перетаскивания изделий мышью.

Объект планирования может описываться с разной степенью подробности:

* Через описание функции установки. Этот текст заносится в поле Описание диалогового окна свойств.
* Через дополнительные технические признаки. Для этого в поле Техническое описание диалогового окна свойств можно занести любые признаки и значения, например мощность, тип привода, давление, ток и т. д. Кроме того, для ввода технических данных можно использовать определенные пользователем свойства.
* Через шаблоны функции, относящиеся к тому же устройству.

* Через конкретные изделия, относящиеся к тому же устройству.

#### Разные структурные идентификаторы на объектах планирования

Каждый объект планирования может иметь структуру, отличающуюся от вышестоящих сегментов. По этой причине на объектах планирования предусмотрены свойства для отдельных элементов структурных идентификаторов. Таким образом, например, в свойство Место установки (отдельный элемент) (ид. 1228) можно ввести элемент структурного идентификатора для места установки. Общий структурный идентификатор для объекта планирования составляется в этом случае из собственного отдельного элемента и всех элементов вышестоящих сегментов структуры и объектов планирования.

Чтобы задать структуру, отличающуюся от вышестоящих сегментов структуры и объектов планирования, поставьте перед значением такого свойства знак "больше" (">"). Использование этого знака приводит к игнорированию вышестоящих структурных идентификаторов.

Использование этих свойств без применения знака "больше" позволит определить собственный дополнительный элемент структурного идентификатора для определенного блока идентификаторов (напр., места установки) на соответствующем объекте планирования. Таким образом, например, можно отличать разные места установки внутри измерительного контура.

#### Объекты планирования, соединения

В предварительном планировании в качестве дополнительных сегментов доступны т. н. "[Объекты планирования, соединения](planninggui_k_verbindungsvorplanung.md)" (объекты планирования, трубопроводы; объекты планирования, кабели). Объекты планирования (соединения) определяют, какие сегменты связаны между собой, и описывают трубопровод или кабель в предварительном планировании.

### Определения сегментов

Определение сегмента определяет вид сегмента (т. е. сегмента структуры или объекта планирования) так же, как определение функции устанавливает вид функции. При создании сегментов с помощью выбора определения сегмента задается вид соответствующего сегмента.

В проектах без определений сегментов при первом открывании (без защиты от записи), в зависимости от имеющейся лицензии, генерируются заранее заданные определения сегментов для предварительного планирования. По умолчанию имеются следующие определения сегментов:

* Общий сегмент структуры
* Общий объект планирования
* Измерительный контур
* Исполнительный механизм
* Функция измерения
* Функция потребителя
* Резервуары, общая информация
* Объект планирования (трубопровод), общая информация
* Объект планирования (кабель), общая информация.

### Макросы предварительного планирования

Любую структуру в навигаторе предварительного планирования можно сохранить в макросе предварительного планирования. Такой макрос представляет собой собственный тип файла с расширением имени файла *.emv. Благодаря использованию макросов предварительного планирования у вас есть возможность быстро и удобно составить предварительное планирование (без графического представления) из готовых подструктур.

### Макросы на объектах планирования

На объектах планирования можно сохранить файлы макросов с"обычным" оконным или страничным макросом (*.ema или *.emp). Такой макрос обычно содержит объекты-заполнители с переменными. Значения переменных можно выбрать в диалоговом окне "Свойства" объекта планирования, используя выбор набора значений, или изменить вручную. При размещении макроса выбранные в объекте планирования или указанные вручную значения копируются на размещенные функции.

Если на объекте планирования сохранен не макрос, а изделие, которому присвоен макрос, то для размещения функций используется макрос изделия.
