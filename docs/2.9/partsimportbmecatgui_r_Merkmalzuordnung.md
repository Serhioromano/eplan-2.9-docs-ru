# Вкладка Присвоение признака

Сервисные программы > Изделие > Управление > [Дополнительно] > Импортировать. В поле Тип файла выбрана запись для версии eCl@ss, а в поле Имя файла выбран XML-файл с наборами данных изделий и нажата кнопка ++OK++. В диалоговом окне Импорт eCl@ss нажмите кнопку ++...++ рядом с полем Настройки и выберите вкладку Присвоение признака.

На этой вкладке можно присваивать характеристики стандарта eCl@ss свойствам изделий базы данных изделий.

!!! note "Замечание:"

    Предварительно на вкладке Присвоение класса необходимо выбрать классы eCl@ss.

Обзор основных элементов диалогового окна:

### Панель инструментов

Кнопка |  Значение
---|---
![](images/all_new_as.png){: .ui-icon } (Создать) |  Открывает диалоговое окно Выбор признака. Позволяет выбрать одну или несколько характеристик стандарта eCl@ss. При этом из раскрывающегося списка Характеристики стандарта eCl@ss можно выбрать, должны ли использоваться характеристики стандарта eCl@ss типа "Advanced" (предварительно установлен) или "Basic". При этом "Advanced" обозначает структурированное дерево признаков, в то время как "Basic" — плоский список признаков. Фильтр позволяет ограничить количество отображаемых признаков. Для этого в поле Фильтр введите текст для поиска и щелкните по кнопке ![](images/all_search_as.png){: .ui-icon } (Поиск). При этом выполняется поиск имен признаков и их описания.
![](images/all_delete_as.png){: .ui-icon } (Удалить) |  Позволяет удалить одно или несколько присвоений признаков из схемы.

Таблица

Чтобы присвоить свойство изделия или определенное пользователем свойство выбранному признаку eCl@ss, в столбце Свойство изделия или Определенное пользователем свойство нажмите на кнопку ++...++; откроется диалоговое окно Выбор свойств, где вы сможете выбрать нужные свойства из списка доступных. Если свойство изделия уже назначено, определенное пользователем свойство не может быть назначено, и наоборот.

Количество присваиваемых свойств изделий или определенных пользователем свойств (одно или несколько) зависит от кардинальности признака eCl@ss; для этого см. тему [База данных изделий: Присвоение признаков eCl@ss свойствам изделий EPLAN](partsimportbmecatgui_k_mappingdetails.md).

В столбце Условие можно задать критерии фильтра, щелкнув кнопку ++...++ в поле и выбрав критерии в открывшемся диалоговом окне Фильтр. При этом с помощью кнопки ![](images/all_new_as.png){: .ui-icon } (Создать) в нем откроется диалоговое окно Выбор признака. Выберите здесь признак eCl@ss, для которого необходимо определить фильтр, и задайте условие (т. е. оператор и значение) в диалоговом окне Фильтр.

!!! example "Пример:"

    Представим, что вам необходимо импортировать пути к документам типа "Графика 2D" неизменно в одном и том же месте, например в свойстве изделия Внешний документ: Файл / гиперссылка 1, независимо от последовательности признаков в файле импорта. Затем для признака "Источник в качестве указания пути" задается свойство изделия Внешний документ: Файл / гиперссылка 1и создается условие "Тип документа = Графика 2D". (Оба признака "Источник в качестве указания пути" и "Тип документа" доступны в диалоговом окне Выбор признакав папке Дополнительная документация > Дополнительная документация.)

!!! note "Замечание:"

    * Характеристики стандарта eCl@ss могут присваиваться различным свойствам EPLAN, независимо от класса. Это объясняется тем, что характеристики стандарта eCl@ss имеют однозначное описание благодаря идентификатору и определению.
    * Одному свойству изделия EPLAN можно присвоить несколько характеристик стандарта eCl@ss. Это может приводить к конфликтам при импорте (если изделие имеет обе характеристики стандарта eCl@ss) и появлению системного сообщения. При этом приоритет будет у первого импортированного свойства.
    * Значения и характеристики стандарта eCl@ss, импортируемые в многоязычные поля, импортируются на языках каталога BMEcat (если имеются).
    * Единица измерения значений не добавляется только при импорте в произвольные свойства. В этом случае она переносится в поле Единица измерения вкладки Произвольные свойства. Для полей базы данных изделий с фиксированной единицей измерения в некоторых случаях необходимо проводить перерасчет.
    * Чтобы импортировать изделие, свойство Номер изделия должно быть всегда заполнено.
