# Управление клеммниками

Управление клеммниками осуществляется через ***Определение клеммника***. Определение клеммника идентифицирует клеммник и содержит все его релевантные данные, а также изделия колодки.

Чтобы определение клеммника можно было копировать, оно должно быть расположено на странице. Вы можете также создавать определение клеммника как неразмещенную функцию. Для нумерации клеммников используйте "стандартную" нумерацию ОУ.

### Отчеты

Для каждого клеммника в его определении можно указать форму, которая будет использоваться при выводе этого клеммника в схемы клемм и схемы подключения клемм. Можно подавлять вывод отдельных клеммников в спецификацию клемм.

### Строение клемников

Изделия клемм, установленных последовательно на клеммнике, можно сохранить в виде отдельных клемм. Для этого соответствующие клеммы должны быть определены как главные. В таком случае клеммник подразделяется на так называемые "самостоятельные области", причем каждая главная клемма определяет одну самостоятельную область.

Определения устройств, т. е. строения отдельных клемм, задаются в базе данных изделий. Для этого в изделиях клемм для главных клемм и соответствующих им второстепенных клемм вносятся шаблоны функций. Шаблон функции для главной клеммы должен находиться в самой верхней позиции.

#### Строение клеммника, как в предыдущих версиях

Если весь клеммник требуется определить таким же образом, как в версиях до EPLAN 2.0, т. е. только через определение клеммника, то он должен состоять только из ***одной*** самостоятельной области. В этом случае строение клеммника задается через определение устройства, сохраняемого в определении клеммника.

!!! note "Замечание:"

    Данный режим работы поддерживается программой только по причинам совместимости. Управление изделиями рекомендуется осуществлять на главных клеммах.

    Строение клеммника при помощи определения устройства можно задать только в том случае, если клеммник не содержит главных клемм или для него активировано свойство Основные клеммы не используются.
