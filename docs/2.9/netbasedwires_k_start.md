# Сетевые соединения

По умолчанию последовательность, в которой соединяются условные обозначения в EPLAN, определяется непосредственно при черчении соединений. Это происходит посредством выбора подходящего символа соединения, в котором логическая модель сохраняется для определения цели.

При помощи сетевых соединений можно проектировать схему соединений, не определяя сразу расположения соединений. При этом EPLAN запоминает ход соединений, независимо от графического представления на схеме соединений.

Возможно следующее:

* Внутри проекта можно смешанно использовать методы работы с определением цели и с сетевыми соединениями.
* Также внутри сети могут встречаться как соединения с определением цели, так и сетевые соединения.
* Символы соединений для сетевых соединений всегда обозначаются как точки соединений.
* Сетевые соединения определяются через точку определения сети. В точке определения сети сохраняется, какие выводы устройств сети связаны друг с другом.
* Соединения и их последовательности впоследствии обрабатываются без изменения чертежа.
* Сетевые соединения можно автоматически обозначать, как обычные соединения, через нумерацию соединений.
* Через точки разрыва сетевые соединения можно также переводить на другие страницы.
* Свойства сетевых соединений можно изменить с помощью "внешней обработки". Само соединение через "внешнюю обработку" изменить ***нельзя***.
* Линии определения кабеля не используются для сетевых соединений. Соединения под линией определения кабеля в сети с сетевыми соединениями не распознаются как кабельное соединение.
