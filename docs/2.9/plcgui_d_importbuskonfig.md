# Диалоговое окно Импортировать данные ПЛК

Вы открыли проект. Данные проекта > ПЛК > Импортировать данные.

В этом диалоговом окне импортируйте данные ПЛК из файла конфигурации ПЛК или любого файла XML. Такой файл можно генерировать при помощи внешней программы конфигурации ПЛК или предварительно экспортировать из EPLAN (и доработать). В зависимости от настроек данные при импорте переписываются или импортируются дополнительные новые функции.

Обзор основных элементов диалогового окна:

### Формат файла импорта

В раскрывающемся списке отображаются все доступные форматы программ конфигурации ПЛК. Выберите формат, в котором сохранен импортируемый файл конфигурации.

!!! note "Замечание:"

    * Если файл импорта имеет более старый формат, чем доступный в поле Формат файла импорта, то при некоторых обстоятельствах может произойти сбой во время обмена данными.
    * Для импорта данных ПЛК в формате AutomationML присвоение функций выполняется с помощью GUID. В этом случае параметры Синхронизация функций, Генерировать повторно все функции и Запустить генерацию схемы соединений ПЛК не применяются.

### Имя файла

В этом поле отображается имя файла конфигурации ПЛК. Через [...] можно открыть диалоговое окно выбора файла и выбрать там файл из любого каталога.

С помощью пункта всплывающего меню Вставить переменную пути перейдите в диалоговое окно [Выбрать переменную пути](modaldialogsdb_d_pfadvariablen.md), из которого можно будет скопировать одну из доступных переменных пути.

### Язык

В этом раскрывающемся списке представлены все языки, определенные в настройках перевода проекта. Выберите язык, на котором импортироваться данные ПЛК. Данные всегда выдаются на одном языке.

При [обмене данными ПЛК в формате AutomationML](plcgui_k_amlbusdatenaustausch.md) импортируются все языки. Установленный здесь язык определяет, на какой язык импортируется основной язык файла обмена.

### Синхронизация функций

При использовании этого параметра EPLAN сравнивает ОУ и обозначение типа функций в проекте с этими же параметрами в файле конфигурации. Данные ПЛК уже имеющихся функций переписываются при импорте соответствующими данными из файла конфигурации. Функции, еще отсутствующие в проекте, генерируются как неразмещенные функции.

При помощи этой опции можно переносить данные ПЛК также в другие (напр., скопированные) проекты.

!!! note "Замечание:"

    В EPLAN сетевые выводы систем шин на базе Ethernet совместимы друг с другом. При импорте таких сетевых выводов EPLAN автоматически распознает совместимую систему шин. Так, например, данные выводов устройства для системы шин Ethernet перезаписываются в проекте данными системы шин Profinet из файла импорта.

### Генерировать повторно все функции

При использовании этого параметра для всех данных ПЛК из файла конфигурации генерируются неразмещенные функции.

Таким образом, в новый проект можно ввести все необходимые функции ПЛК и впоследствии их использовать.

### Запустить генерацию схемы соединений ПЛК

Если этот флажок установлен, то после импорта будет автоматически открыто диалоговое окно [Генерировать схему соединений ПЛК](plcgui_d_spsschaltplanerzeugen.md). Там Вам предоставляется возможность автоматически разместить импортированные устройства.

### [Параметры]

Эта кнопка активна, если в поле Формат файла импорта выбрана одна из следующих записей:

* Стандартный формат обмена ПЛК (AutomationML)
* Mitsubishi iQ-Works (AutomationML)
* Siemens SIMATIC Step 7 TIA Portal (AutomationML).

В следующем диалоговом окне укажите, нужно ли перед импортом проверить недостающие изделия. Если в нем установлен флажок Проверить изделия перед импортом, перед импортом проверяется наличие изделий, используемых в файле импорта, в базе данных изделий. При этом изделия идентифицируются с помощью свойств Обозначение типа ПЛК или Описание устройства: Имя файла и Описание устройства: Индекс в файле. Если устройство содержит подустройства, соответствующие изделия дополнительно идентифицируются с помощью свойств Подустройство ПЛК x: На позиции / разъем и Подустройство ПЛК x: Обозначение типа ПЛК или Подустройство ПЛК x: Описание устройства: Индекс в файле.

Все ненайденные изделия отображаются в форме таблицы в окне сообщения. В столбце Обозначение типа ПЛК отображается либо обозначение типа ПЛК, либо файл описания устройства с индексом. Каждое недостающее изделие приводится только один раз, даже если встречается многократно в файле импорта. С помощью всплывающего меню можно скопировать записи и, например, вставить их в файл Excel. С помощью этого списка можно добавить изделия в базу данных изделий и снова выполнить импорт.
