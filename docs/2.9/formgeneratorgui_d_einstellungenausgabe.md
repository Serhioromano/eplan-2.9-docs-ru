# Диалоговое окно Настройки: Отображение / вывод

Вы открыли проект.

* Параметры > Настройки > Проекты > "Имя проекта" > Отчеты > Отображение / вывод.
* Сервисные программы > Отчеты > Генерировать > [Настройки] > Отображение / вывод.

В этом диалоговом окне установите настройки отображения и вывода для страниц отчетов.

Обзор основных элементов диалогового окна:

### Заменить идентичные тексты на

При выводе отчетов могут выводиться повторяющиеся идентичные тексты, например функциональные тексты. Такие идентичные тексты на страницах отчета можно заменить заменяющим символом. Задайте здесь этот заменяющий символ.

Тексты в отчетах, которые следует заменить, задаются во время обработки форм в редакторе форм при вставке или обработке текстов-заполнителей. По этой причине на вкладке [Размещение](formeditorgui_r_platzierung.md) для текстов-заполнителей предусмотрен флажок Заменить идентичный текст. Если флажок установлен, то повторяющиеся тексты, стоящие друг под другом, заменяются заданным здесь замещающим символом.

!!! example "Пример:"

    Задайте в качестве замещающего символа символ повторения".Главная функция Позиция Двигатель=AP+0/1.2"=AP+ST1/2.15Генератор=AP+0/1.1"=AP+ST1/2.12

### Заменить значения переменных на

Эта функция предназначена для отчета "Групповая спецификация изделий". При соединении функции и объекта-заполнителя в объекте-заполнителе можно управлять несколькими наборами значений и присвоить функции текущий набор значений. При генерации отчета функции значения текущего присвоенного набора значений выводятся по умолчанию. Когда генерируются отчеты таких функций, с помощью этой функциональности можно заменить значения текущего присвоенного набора значений на фиксированный текст. Для этого в форме групповой спецификации изделий необходимо активировать свойство формы Заменить значения переменных текстом. С помощью всплывающего меню можно ввести многострочный текст или перевести текст вручную.

!!! example "Пример:"

    Если в это поле ввести текст См. таблицу, тогда на том месте, на котором EPLAN обычно выводит значения текущего присвоенного набора значений, выводится текст См. таблицу. Благодаря этому можно, например, указать таблицу, которая отображает значения всех наборов значений.

!!! tip "Совет:"

    Свойство формы Заменить значения переменных текстом рекомендуется активировать только в случае необходимости. Если свойство формы активировано, программа выполняет поиск текстов заполнителей. В зависимости от размера проекта на эту операцию может потребоваться время.

### Стартовые страницы внутрисегментного смещения для группы выводимых данных

В это поле задается число, которое, начиная со второй группы вывода, прибавляется к (числовому) имени стартовых страниц. Благодаря этому остается место между отчетами, так что все отчеты одной группы могут следовать друг за другом.

Эта установка может быть использована только для числовых имен страниц. Дополнительно надо выбрать в поле Заполнить блок идентификаторов группой выводимых данных запись "Нет".

### Заполнить блок идентификаторов группой выводимых данных

Посредством раскрывающегося списка укажите, какой блок идентификаторов в указании стартовой страницы отчета должен быть перезаписан значением группы выводимых данных. Возможные значения: "Нет" или один из блоков идентификаторов из структуры проекта. Для этого блока идентификаторов генерируется для каждого значения группы выводимых данных один структурный идентификатор и отчет сортируется под этим структурным идентификатором в структуру страниц.

Эта настройка используется, если в диалоговом окне Сортировка страниц - <Тип отчета> в поле Группировать по / в выбрана опция "Группа вывода".

!!! example "Пример:"

    Откройте структуру проекта с блоками идентификаторов "Установка" и "Место установки" и выберите здесь запись "Место установки". В свойстве Группа выводаперенесите на разныеусловные обозначенияследующие значения:AB,CD,EF. При отчете название группы отчета записывается в виде указания стартовой страницы для места установки и для каждого значения группы вывода генерируется отдельный блок отчета. При этом генерируются места установкиAB,CD,EF, где и сортируются отчеты.

### Ограничить вывод кабелей

Если флажок не установлен, тогда все кабели всегда выводятся в страницах отчета. Если флажок установлен, то выводятся только те кабели, которые содержат минимальное число кабельных соединений. Минимальное число установите в поле Вывод, начиная с числа изделий.

### Определить источник / цель кабельных соединений через кабельные соединения

Если флажок установлен, источник и цель кабельных соединений соответствующего кабеля ***не*** выводятся. Если флажок снят, источник и цель кабельных соединений соответствующего кабеля выводятся.

### Ограничить вывод клеммников

Если флажок не установлен, тогда все клеммники всегда выводятся в страницах отчета. Если флажок установлен, тогда выводятся только те клеммники, которые содержат минимальное число клемм. Минимальное число установите в поле Вывод с числа клемм.

### Ограничить вывод штекеров

Если флажок не установлен, тогда все штекеры всегда выводятся в страницах отчета. Если флажок установлен, тогда выводятся только те штекеры, которые содержат минимальное число контактов штекеров. Минимальное число установите в поле Вывод с числа штырей.

### Использование для отдельных соединений

Если флажок установлен, отдельные соединения в спецификации клеммника или штекера могут выводиться с помощью заменяющего символа. Замещающий символ задается в поле Символы для отображения в таблице кабелей. Чтобы отдельные соединения выводились в спецификации клеммника или штекера, в диалоговом окне Свойства — текст-заполнитель на вкладке Размещение установить флажок Отобразить все соединения.

### Использование для кабельных соединений

Если флажок установлен, кабельные соединения в спецификации клеммника или спецификации штекера могут выводиться с помощью замещающего символа. Замещающий символ задается в поле Символы для отображения в таблице кабелей.

!!! example "Пример:"

    Если в качестве замещающего символа ввестиX, то все кабельные соединения в области данных таблицы кабелей будут представлены какX.
