# Схемы соединений (топология)

При маршрутизации соединений учитываются выводы устройства функции топологии или (если локальные выводы отсутствуют) данные вывода устройства из изделия. Выводы устройства могут быть определены на изделии на присвоенной схеме соединений или в шаблонах функций. Однако данные выводов устройства, например, позицию X, Y и Z или дополнительную длину можно записать только на схеме соединений, но не в шаблонах функции.

Если схема соединений или выводы устройства отсутствуют, выводы устройства при маршрутизации генерируются из многополюсных функций.

!!! note "Замечание:"

    Если значения для позиций X, Y и Z отсутствуют (поскольку они не записаны на схеме соединений или не определен вывод устройства) маршрутизация проводится для позиции вывода устройства (0,0,0). В этом случае необходимо указать дополнительную длину до вывода побольше, чтобы определенной длины до вывода устройства хватило для разводки. Для кабеля, однако, нельзя вычислить длину зачистки, она получает значение "0".

### Маршрутизация с локальной схемой соединений

Если имеется ***локальная*** схема соединений, поиск вывода устройства выполняется только локально. В этом случае выводы устройства из изделия не учитываются. Если вывод устройства не определен локально, он отсутствует. При маршрутизации генерируется недостающий вывод устройства, при этом маршрутизация осуществляется для позиции (0,0,0).

### Маршрутизация со схемой соединений из изделия

Если выводы устройства не определены локально, используется схема соединений из ***изделия***. При этом действуют следующие правила:

* Если вывод устройства определен в базе данных изделий (т.е. на вкладке Выводы устройства схемы соединений указаны значения в полях Обозначение вывода устройства и Обозначение штекера, а для клемм также в полях Уровень и Внутр./внешн. индекс), то используются указанные там значения.
* Если вывод устройства определен в базе данных изделий, но в полях Позиция X, Позиция Y и Позиция Z отсутствуют значения (т.е. 0,00 мм), то маршрутизация проводится для позиции (0,0,0).
* Если вывод устройства не определен на вкладке Выводы устройства, используются значения по умолчанию с вкладки Схема соединений. При этом маршрутизация выполняется для позиции (0,0,0).
* При расчете длины учитывается только ***одна*** дополнительная длина: если вывод устройства определен на вкладке Выводы устройства, в поле Дополнительная длина используется внесенное значение. В противном случае используется значение, указанное на вкладке Схема соединений в поле Дополнительная длина (стандарт).

### Маршрутизация без схемы соединений

Можно также маршрутизировать соединения к целям без схемы соединений. Однако для такой маршрутизации выводы устройства функции должны присутствовать на функциях топологии. Если необходимые выводы устройства отсутствуют, они автоматически генерируются перед маршрутизацией (если это возможно).

Если у маршрутизируемого соединения есть только одна функция топологии, необходимый вывод устройства функции генерируется на ней. Если имеется несколько подходящих функций топологии, но ни на одной или на обеих функциях есть подходящий вывод устройства функции, такое соединение невозможно маршрутизировать. В этом случае следует вручную обработать схему соединений.
