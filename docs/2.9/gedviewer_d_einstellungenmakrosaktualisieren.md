# Диалоговое окно Настройки: Обновить макросы

Вы открыли проект.

* Параметры > Настройки > Проекты > "Имя проекта" > Графическая обработка > Обновить макросы.
* Вы добавили макрос с рамкой макроса на страницу проекта. Для рамки макроса выберите пункт всплывающего меню Обновить макрос. Нажмите в диалоговом окне Обновить макросы рядом с полем Настройки на ++...++.
* В проекте есть подготовленные или вставленные макросы. Данные проекта > Макросы > Навигатор. Вы выделили в диалоговом окне Макросы — <Имя проекта> один или несколько макросов. Выберите пункт всплывающего меню Обновить макросы. Нажмите в диалоговом окне Обновить макросы рядом с полем Настройки на ++...++.

В этом диалоговом окне указываются настройки для обновления макросов, после чего настройки сохраняются в виде схемы.

Обзор основных элементов диалогового окна:

### Схема

Выберите из раскрывающегося списка схему, которую требуется использовать или обработать. Для этого также воспользуйтесь соседней [панелью инструментов](schemeconfig_m_schemaschaltflaechen.md).

### Предпочитать графическое выполнение присвоения

По умолчанию при обновлении заменяемая функция в расположенном макросе и заменяющая функция в файле макроса определяются на основе определения функции.

Если установлен этот флажок, обновляемые функции макросов определяются на основе позиции в макросе. Таким образом можно заменить условное обозначение на определенной позиции в макросе условным обозначением с другим определением функции (например, клемму на штекер). Обозначение устройства при этом сохраняется. Если на соответствующей позиции в макросе нет условного обозначения, используется следующее в графической последовательности условное обозначение с подходящим определением функции.

### Оставить только ОУ

По умолчанию при обновлении графическая информация (форматирование, порядок свойств и т. д.) переносится из файла макроса на вставленный макрос, а логическая информация не переносится и остается на вставленном макросе.

Если установлен этот флажок, при обновлении во вставленном макросе из файла макроса также переносится измененная логическая информация (обозначения выводов устройства, функциональные тексты, измененные символы, присвоенные изделия). Сохраняются только обозначения устройств, включая те, которые относятся к точкам разрыва.

### Заново обновить наборы значений

В большинстве случаев при обновлении макросов присвоенные наборы значений объектов-заполнителей остаются на вставленных макросах. Если это не так (например, потому что макрос содержит сильно измененную структуру или больше объектов-заполнителей), установите этот флажок. Таким образом вставленным макросам при обновлении заново присваиваются последние выбранные в проекте наборы значений.
