# Предварительное планирование: Нумерация ОУ

При связывании функций с объектом планирования или при перемещении объекта планирования вместе со связанными функциями внутри структур выполняется автоматическая нумерация ОУ функций. Это происходит при следующих действиях:

* размещение шаблона объекта планирования;
* присвоение объекта планирования функции;
* перемещение объекта планирования, которому присвоены функции, в навигаторе предварительного планирования;
* перемещение объекта планирования в графическом предварительном планировании в другое обозначение местоположения.

Нумерация связанных с объектом планирования функций выполняется в соответствии с настройками, которые действуют для нумерации в режиме подключения при вставке ***символов***. При этом не делается различий между функциями, размещенными как символ, и размещенными из макроса. Если макрос содержит другие функции, которые не связаны с объектом планирования, то эти функции нумеруются в соответствии с настройками нумерации в режиме подключения при вставке макросов.

Структурные идентификаторы функции составляются из структурных идентификаторов объекта планирования и целевой страницы.

Указанная на сегменте структуры маркировка продукта всегда принадлежит структуре идентификаторов. Если на нижестоящем объекте планирования не указана маркировка продукта, то она добавляется при нумерации.

!!! note "Замечание:"

    Нумерация ОУ функций при изменении структурного идентификатора объекта планирования или вышестоящих сегментов ***не выполняется*** автоматически. В этом случае требуется обновление детального планирования. Для этого в навигаторе предварительного планирования выберите пункт всплывающего меню Обновить детальное планирование, а в открывшемся диалоговом окне установите флажок Обновить обозначения устройства и символические адреса.

    В результате такого обновления детального планирования функции (или представления) устройства, которые ранее не имели ссылки на объект планирования, теперь получают эту ссылку. Для того чтобы это было возможно, устройство должно иметь только одну главную функцию.

!!! example "Пример:"

    В контакторе катушка связана с объектом планирования, а контакты не связаны. Выделите объект планирования в навигаторе предварительного планирования, выберите пункт всплывающего меню Обновить детальное планирование, а в открывшемся диалоговом окне установите флажок Обновить обозначения устройства и символические адреса. Впоследствии контакты также будут связаны с объектом планирования.Многополюсное представление электродвигателя связано с объектом планирования, а однополюсное представление — не связано. Выделите объект планирования в навигаторе предварительного планирования и обновите детальное планирование, как описано выше. Впоследствии однополюсное представление также будет связано с объектом планирования.

### Правила нумерации в режиме подключения

ОУ всегда должны быть уникальными. Если на одном объекте планирования в полной структуре указана маркировка продукции, сначала используется это ОУ. Если оно уже существует в целевой структуре, то есть могло бы появиться двойное ОУ, в этом случае добавляется нижестоящий счетчик или увеличивается существующий. При этом наибольший из существующих нижестоящих счетчиков увеличивается на 1.

Выводы устройства ПЛК, которые из-за (одинакового) символического адреса и ЦПУ принадлежат к одному адресу объекта планирования, не нумеруются. Но если вывод устройства ПЛК содержит ссылку на объект планирования, тогда его нумерация зависит от этого объекта планирования. (Выводы устройства ПЛК только тогда имеют ссылку на объект планирования, если в этом объекте планирования определена его главная функция, то есть блок ПЛК.)

Если в проект вставляются функции, которые не имеют ссылки на объект планирования, то нумерация этих функций не зависит от нумерации объектов планирования; если объекты планирования все же должны иметь присвоенные функции, то их ОУ учитывается.

!!! note "Замечание:"

    * Мы рекомендуем использовать маркировку продуктов без нижестоящего счетчика, чтобы нижестоящий счетчик оставался свободным для нумерации в режиме подключения.
    * Рекомендуется присваивать функциям только вопросительный знак и буквенное обозначение, но не счетчик. В этом случае для нумерации ОУ используется буквенное обозначение и генерируется новый счетчик, а также — если предусматривает формат — добавляется предшествующая цифра. Это буквенное обозначение имеет приоритет перед буквенным обозначением, которое указано в макросе.

Данные в наборе значений объекта заполнителя имеют приоритет перед значениями по умолчанию в объекте планирования. Значения по умолчанию на объекте планирования (без вопросительного знака "?") имеют приоритет перед заданным в проекте форматом нумерации.

При нумерации в режиме подключения сначала нумеруются функции, которые связаны с объектом планирования, а затем остальные функции.
