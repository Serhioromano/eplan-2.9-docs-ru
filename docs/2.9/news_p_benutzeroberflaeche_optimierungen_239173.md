# Дополнительная оптимизация для платформы EPLAN и ее продуктов

В текущей версии реализована дополнительная оптимизация для платформы EPLAN и ее продуктов:

### Платформа EPLAN

* ***Интерфейс пользователя:*** начальное диалоговое окно Выбрать объем меню, которое использовалось для определения объема отображаемых меню посредством режима пользования, удалено из интерфейса пользователя. Объем меню можно дополнительно ограничить с помощью управления правами EPLAN.
* ***Интерфейс пользователя:*** в диалоговом окне Выбор свойств (и в аналогичных диалоговых окнах) в нижней части для выделенного свойства отображается дополнительная информация об этом свойстве (идентификатор, обозначение и описание свойства).
* ***Интерфейс пользователя:*** теперь с помощью клавиши ++÷++ на цифровом блоке клавиатуры можно свернуть все уровни иерархии, которые открыты в развернутом дереве представления в виде дерева.
* ***Резервировании данных:*** теперь при ***восстановлении проектов*** также отображается описание проекта. Для этого диалоговое окно Восстановить проект в поле Проекты было дополнено новым столбцом Описание проекта. Отображение этой информации облегчает выбор резервированного проекта.
* ***Предварительный просмотр графики:*** если в окне предварительного просмотра графики дважды щелкнуть по странице, эта страница откроется в графическом редакторе. Чтобы открыть отобразившуюся в предварительном просмотре страницу в новом окне, во время двойного щелчка удерживайте нажатой клавишу ++Shift++.
* ***Предварительный просмотр графики:*** предварительный просмотр графики теперь также реагирует на выбор в диалоговом окне Сортировать точки разрыва и отображает соответствующую страницу с выделенной точкой разрыва.
* ***Навигаторы данных проекта:*** в представлениях в виде дерева отфильтрованных навигаторов выделение теперь сохраняется, даже если отключить фильтр.
* ***Настройки:*** цвет, выбранный и активированный для ***групп проектов*** в диалоговом окне [Настройки: Группы проектов (определение)](settingsmastergui_d_projektgruppendefinition.md), теперь отображается также для специфических для проекта настроек в диалоговом окне 'Настройки'. Если пользователь открывает несколько проектов, ему легче определить, для какого проекта он в данный момент выполняет настройки.
* ***Настройки:*** специфическая для пользователя настройка Отображать полосы прокрутки теперь по умолчанию отключена (путь меню: Параметры > Настройки > Пользователь > Представление > Общее). Таким образом, полосы прокрутки по умолчанию не будут отображаться в редакторах.
* ***Настройки:*** в диалоговом окне Настройки: Языки удалено лишнее групповое поле Справочная система (путь меню: Параметры > Настройки > Пользователь > Представление > Языки). При вызове ***справка*** открывается через интернет-соединение на соответствующем языке диалогового окна. В справочной системе с помощью кнопки можно переключаться между всеми доступными языками.
* ***Настройки:*** если для уже вставленного символа на вкладке Данные символа / функции открыть диалоговое окно Выбор символа, чтобы заменить его, то в этом диалоговом окне будет доступна настройка Заменить только символ. Чтобы предотвратить ошибочные изменения в определениях функций, эта настройка теперь включена по умолчанию.
* ***Соединения:*** диалоговое окно Перенести обозначения соединений открывается теперь даже в том случае, если ***обозначение соединения*** изменяется при непосредственной или табличной обработке. В этом диалоговом окне указывается, на какие другие соединения следует автоматически переносить измененное обозначение соединения. Чтобы это окно открылось, в настройках пользователя в диалоговом окне [Настройки: Символы соединения](gedviewer_d_einstellungenverbindungssymbole.md) должен быть активирован флажок Перенести обозначения соединений на периметр.
* ***Управление ревизиями:*** если теперь после генерирования первой ревизии для проекта деактивировать отображение ***маркеров ревизий*** посредством пунктов меню Вид > Маркеры ревизий, эта настройка сохранится при генерировании последующих ревизий. Маркеры ревизии включаются автоматически только при генерации первой ревизии.
* ***Редактор форм:*** в динамических формах для ***таблицы соединений*** теперь в окне выбора свойств для диалогового окна Изменение свойства можно также выбрать свойства блока. Это позволяет, например, выводить пустую строку в таблице соединений, если изменяется значение определенного свойства блока.

### EPLAN Electric P8

* ***Топология:*** свойство Топология: Фильтр соединений (ид. 20247) теперь отображается как столбец в представлении в виде списка навигатора топологии и при табличной обработке. Это помогает определить, для каких сегментов маршрутизации задан фильтр соединений, и обрабатывать фильтр в этих сегментах.
* ***Топология:*** фильтры соединений, установленные на сегментах маршрутизации, теперь также учитываются, когда для маршрутизируемого соединения задан маршрут по умолчанию (то есть, свойство Топология: Трасса маршрутизации - по умолчанию (ид. 31119) заполнено).

### EPLAN Pro Panel

* ***Трехмерный чертеж монтажных поверхностей:*** при размещении контура выдавливания в качестве шины, определенной пользователем, теперь из описания контура в пространстве листа автоматически генерируется описание функционального элемента. В свойствах контура это описание заносится в свойство Описание (форма, рамка, контур, профиль) (ид. 18011).
* ***Трехмерный чертеж монтажных поверхностей:*** при вставке монтажной платы флажок Встроить в структуру шкафа в опциях размещения теперь активирован по умолчанию. Это означает, что теперь монтажные платы при вставке в электрошкаф автоматически интегрируются в иерархию под электрошкафом, и нет необходимости открывать диалоговое окно Опции размещения.
* ***Трехмерный чертеж монтажных поверхностей:*** улучшены размещение электротехнических модулей в пространстве листа и последующая маршрутизация соединений для этих размещений изделий.
* ***Трехмерный чертеж монтажных поверхностей:*** Принадлежности компонентов с изменяемой длиной также теперь можно вставлять с изменяемой длиной. Таким образом теперь можно, например, с помощью опции выбора принадлежностей подобрать крышку для системы сборной шины и с изменяемой длиной разместить ее на системе сборной шины.
* ***Инструмент для монтажных работ:*** при перемещении инструмента для монтажных работ (точка монтажа, линия монтажа, точка захвата или исходная точка) в проекте макросов с одной монтажной поверхности на другую инструмент для монтажных работ теперь размещается непосредственно под новой монтажной поверхностью, если активирован параметр Захват монтажной поверхности.
* ***Навигатор пространства листа:*** теперь в навигаторе пространства листа можно точечно фильтровать по инструментам для монтажных работ (точкам монтажа, сборочным линиям и т. д.), которые используются в проекте. Для этого мы предлагаем предварительно определенную схему "инструментов для монтажных работ".
* ***Навигатор пространства листа:*** Объекты-заполнители теперь всегда отображаются в навигаторе пространства листа и могут быть оттуда удалены независимо от лицензии.
* ***Маршрутизация соединений:*** Края маршрутизируемых соединений теперь отображаются не черным, а соответствующим цветом для соединения. Это также улучшило отображение соединений в EPLAN Smart Wiring (при низком коэффициенте масштабирования).
* ***Основные данные для EPLAN Pro Panel:*** При импорте данных устройств теперь можно импортировать устройства (например, несущие шины, кабельные каналы и т. д.) с указанием длины, а затем вставлять их с соответствующей длиной в пространство листа.
* ***Основные данные для EPLAN Pro Panel:*** Контуры для "генерирования записей ЧУ-данных" теперь, как и контуры вдавливания, относятся к основным данным проекта и при размещении в пространстве листа включаются в проект. Эти контуры теперь также отображаются в диалоговом окне Синхронизация основных данных — <Имя проекта> и могут передаваться вместе с проектом.
* ***Основные данные для EPLAN Pro Panel:*** На размещении изделий в пространстве листа теперь можно хранить до 50 изделий. Таким образом, при импорте устройств все изделия с идентичным ОУ (например, электрошкаф и его принадлежности) переносятся на одно и то же размещение изделия.
* ***Данные изготовления "Rittal - Perforex LC / Secarex":*** При экспорте данных изготовления "Rittal - Perforex LC / Secarex" данные теперь генерируются для обработки отдельных частей. Благодаря этому съемные функциональные элементы электрошкафа (дверь, монтажная плата и т. д.) могут обрабатываться по отдельности центром лазерной обработки.
* ***Данные изготовления шаблона сверления:*** При экспорте данных изготовления шаблонов сверления данные теперь выводятся в единице измерения, которая установлена как отображаемая единица измерения. Это позволяет осуществлять экспорт в единице измерения "дюйм".
* ***Данные изготовления шаблона сверления:*** При экспорте данных изготовления шаблонов сверления теперь можно выводить различные типы монтажных отверстий разными цветами. Соответствующие цвета задаются в управлении слоями с помощью четырех новых слоев "EPLAN691", "EPLAN692", "EPLAN693" и "EPLAN694", которые доступны в управлении слоями под узлом "Графика > Шаблон сверления > Механическая обработка".
* ***Данные изготовления "CadCabel":*** При экспорте данных изготовления по данным сборки проводов для "CadCabel" теперь поддерживаются не только целочисленные обозначения соединений, но и обозначения соединений с десятичными или другими разделителями (например, "10.25" или "10/25").
* ***Управление сообщениями:*** С помощью ***нового контрольного прогона*** 026105 из класса сообщения 026 "Трехмерный чертеж монтажных поверхностей" можно проверить, имеются ли в проекте, пространстве листа и т. д. функциональные элементы с изменяемой длиной (кабельные каналы, несущие шины, сборные шины, профильные C-шины, определенные пользователем шины), имеющие одинаковое обозначение устройства.
* ***Управление сообщениями:*** С помощью контрольного прогона 026017 сравнивается структура идентификаторов на размещении изделия с соответствующей монтажной поверхностью. Это сообщение контрольного прогона теперь работает в соответствии с сообщением контрольного прогона 012008. В этом случае проверяется уже не вся структура идентификаторов, а только соответствие отдельных блоков идентификаторов. При этом отсутствующие блоки идентификаторов игнорируются.
* ***Управление сообщениями:*** Для экспорта данных изготовления через новый интерфейс на полностью автоматический станок для сборки проводов "Rittal — Wire Terminal WT" контрольный прогон 026072 был расширен двумя дополнительными критериями проверки. Сообщения теперь выводятся определенно для этого [нового интерфейса](news_p_propanel_wireterminalwt_256303.md), если отсутствует цвет соединения или было занижено минимальное поперечное сечение соединения.

### EPLAN Preplanning

* ***Навигатор шаблонов сегментов:*** в дереве навигатора шаблонов сегментов теперь также доступен пункт контекстного меню Конфигурировать представление. Вы можете изменить представление в виде дерева для шаблонов сегментов, классов труб и веществ, используя последующее диалоговое окно с тем же именем.
* ***Проверка предварительного планирования:*** проверка на многократные сегменты, структурные идентификаторы или идентифицирующие обозначения посредством ***контрольных прогонов*** 28001, 28002, 28003 и 28018 ранее была возможна, только если контрольные прогоны применялись ко всему проекту или совершался выбор в навигаторе предварительного планирования. В новой версии теперь также осуществляется проверка отдельной страницы, если на ней выделены соответствующие сегменты или объекты, связанные с предварительным планированием. Эта функция применяется к страницам предварительного планирования и функциональной схемы автоматизации, а также в связанных объектах страниц схемы соединений или в открытом пространстве листа. В рамках этого нововведения контрольные прогоны 28001 и 28002 были дополнены видом проверки "Предотвращение ошибок".
* ***Создание подробного планирования:*** при вырезании и вставке условных обозначений или фрагментов на функциональной схеме автоматизации связи с сегментами сохраняются в предварительном планировании. Теперь эта функция также доступна для вырезания и вставки в пространстве листа или на других страницах схемы соединений (типы страниц "Многопол. схема соедин.", "Однопол. схема соедин." и т. д.).
