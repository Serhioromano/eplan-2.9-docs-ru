# Принцип работы при вставке устройств

EPLAN при вставке устройства проверяет данные, введенные в изделие. В изделии имеются различные поля, позволяющие выполнять графические размещения. Для этих данных устанавливаются различные приоритеты:

* Изделие содержит ссылку на макрос (макрос символа или макрос окна).
* Изделие содержит определение устройства (шаблоны функций).
* Изделие содержит ссылку на графический макрос и размеры X / Y (имеет смысл только для размещений изделий).
* Изделие не содержит ни определение устройства, ни макрос.

Макрос имеет максимальный приоритет, за ним следуют шаблоны функций. Графический макрос с размерами используется исключительно для компоновки электрошкафа.

Новому устройству присваивается ОУ в зависимости от режима копирования. (Режим копирования настраивается в меню Параметры > Настройки > Проекты > "Имя проекта" > Устройство > Нумерация (online).)

Если устройство размещается на странице типа ***Компоновка электрошкафа***, то там создается новое размещение изделия (условное обозначение не создается).

### Определение главной функции

Чтобы можно было перенести изделие, следует в любом случае разместить главную функцию. При размещении устройства, не содержащего главной функции, первое размещенное условное обозначение автоматически становится главной функцией.

Многократный выбор изделий для размещения невозможен.

#### Макросы

Если макрос содержит ***только одну*** главную функцию, то изделие переносится на нее.

Если в макросе ***отсутствует*** главная функция, то главной функцией становится одно из условных обозначений. Для этого EPLAN сначала проверяет, какая из функций, содержащихся в макросе, была определена в определении функций как главная функция. Если таковая имеется, то она становится главной функцией. Если в действительности имеются только вспомогательные функции, то становится главной функцией графически первая функция.

Если макрос содержит ***несколько*** главных функций, то изделие переносится на первую подходящую главную функцию. Т. е. изделие присваивается первой главной функции макроса, которую оно может правильно взять на себя. Другие главные функции не изменяются. Если подходящей главной функции не обнаружено, сначала EPLAN ищет черный ящик. Если он отсутствует, то изделие переносится на графически первую главную функцию.
Таким образом, символы устройств обрабатываются в первую очередь, так как они, как правило, представляют устройство или модуль.

Если макрос содержит главную функцию, которой уже присвоено изделие, то сначала полностью убираются все изделия, а затем на главную функцию переносится выбранное изделие. При этом переносятся все технические данные, которые содержит изделие (технические параметры, обозначение вывода устройства, буквенное обозначение).

#### Шаблоны функций

Если изделие содержит шаблоны функций, то EPLAN при вставке устройства сначала предлагает функцию, которая представляет возможную главную функцию. После размещения главной функции последовательно размещаются остальные шаблоны функций. Вспомогательные функции размещаются на основании последовательности определений (из базы данных изделий). Если был установлен последний шаблон функции выбранного изделия, курсор перемещается на главную функцию и вы можете разместить устройство еще раз.

### Учет вариантов макросов при генерировании устройств

При генерировании нового устройства в навигаторах (с помощью пункта всплывающего меню Новое устройство) можно использовать функции из записанного на изделии макроса или из шаблонов функций, по выбору. Это зависит от настройки проекта Учитывать макрос при генерировании устройств (через меню Параметры > Настройки > Проекты > "Имя проекта" > Устройство > Общее).

Если эта настройка проекта ***отключена*** (это стандартная настройка), то из первой главной функции, записанной в ***шаблонах функции***, генерируется неразмещенное устройство. Если изделие содержит только один макрос, но не содержит шаблонов функции, то из первой найденной в макросе главной функции генерируется неразмещенное устройство.

Если настройка проекта ***включена***, то из содержащихся в ***макросе*** функций генерируются неразмещенные устройства. Точное поведение зависит от того, записаны ли для изделия дополнительно шаблоны функций.

* Если на изделии указан макрос, но отсутствуют шаблоны функций, неразмещенные функции создаются из одних только функций, содержащихся в первом варианте макроса. (При этом начинается поиск первого варианта макроса с видом представления "многополюсный".)
* Если на изделии помимо макроса указаны шаблоны функций, эти последние отображаются в навигаторах и на них накладываются соответствующие функции из вариантов макроса. Если хотя бы один шаблон функции подходит к варианту макроса, тогда и все остальные функции из этого варианта также генерируются в качестве неразмещенных.
При этом остальные варианты макросов используются по порядку до тех пор, пока накладываются свободные шаблоны функций. Если функции в текущем варианте макроса не подходят свободным шаблонам функций, процесс прерывается. Из остальных вариантов макросов больше не генерируются функции.

!!! example "Пример:"

    На изделии указаны макрос и шаблоны функций. В макросе имеются варианты А, В и С. Включена настройка проекта Учитывать макрос при генерировании устройств.При генерировании устройства на шаблоны функций накладываются сначала функции из варианта А. Если для оставшихся шаблонов функций отсутствуют подходящие функции в варианте макроса В, операция прерывается. В этом случае функции, содержащиеся в варианте макроса С, не учитываются.

### Учет вариантов макросов при вставке устройств

Если для изделия указаны как макрос, так и шаблоны функции, то при добавлении устройства в графическом редакторе (через меню Вставить > Устройство) учитываются также функции, содержащиеся в других вариантах этого макроса. Это зависит от настройки проекта Учитывать макрос при генерировании устройств.

Если настройка проекта ***деактивирована*** (это настройка по умолчанию), то при вставке устройства сначала предлагается первый вариант макроса, подходящий для типа страницы. Клавиша [Tab] позволяет "пролистать" варианты макроса и выбрать подходящий. Если на изделии отсутствуют шаблоны функций, размещаются функции выбранного в данный момент варианта макроса.
Если на изделии имеются шаблоны функций, размещаются все равно функции выбранного в данный момент варианта макроса. Остальные шаблоны, для которых невозможно найти подходящую функцию в выбранном варианте макроса, остаются свободными.

Если настройка проекта ***активирована***, а изделие содержит шаблоны функций, размещаются функции выбранного варианта макроса. Кроме того, если хотя бы один шаблон функции соответствует варианту макроса, из оставшихся вариантов макроса генерируются неразмещенные устройства. На шаблоны функций накладываются соответствующие функции из вариантов макроса, то есть как размещенные, так и неразмещенные функции получают данные из шаблонов функций. При этом остальные варианты макросов используются по порядку до тех пор, пока накладываются свободные шаблоны функций. Если функции в текущем варианте макроса не подходят свободным шаблонам функций, процесс прерывается. Из остальных вариантов макросов больше не генерируются функции.

Размещенный макрос можно вторично разместить при помощи курсора. При этом генерируется новое устройство.
