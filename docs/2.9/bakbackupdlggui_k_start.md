# Резервирование данных

Резервирование и передача проектов и основных данных являются важными элементами процесса проектирования. Резервирование данных позволяет резервировать и восстанавливать все сгенерированные данные. Резервирование данных проводится в следующих случаях:

* Резервирование внешних данных
* Архивирование данных на длительный срок
* Обмен файлами через дискету / компакт-диск / электронную почту / сеть / Internet.

Кроме резервирования данных доступны следующие функции:

* Резервирование одного или нескольких проектов со всеми соответствующими основными данными. При этом резервированному проекту может быть присвоен описательный текст, напр., указание места хранения.
* Резервирование отдельных областей основных данных (библиотеки символов, рамки, формы, шаблоны, данные изделий, словари).
* Сохранение резервированных проектов. В данном случае проверяется, изменились ли настройки проекта, существуют ли данные и можно ли их перезаписать.
* Упаковывание и распаковывание проектов в каталоге проекта; проекты, добавленные в архив в каталоге проекта, распаковываются при запуске.
* Архивирование проектов.
* Выгрузка проектов.
* Отправка проектов по электронной почте. Для этой цели проекты можно разбить на несколько небольших блоков данных, а потом снова соединить.
* Просмотр сообщений, которые генерируются в процессе резервирования или в ходе восстановления проекта.
