# Создание диаграмм GRAFCET

Диаграммы GRAFCET можно создавать на уже существующей или новой странице. Ниже описано создание диаграммы GRAFCET на новой странице. Порядок действий при создании диаграмм GRAFCET на уже существующих страницах аналогичен.

Условия:

* Вы открыли проект.
* Вы добавили в проект библиотеку символов OS_SYM_ESS.slk. (Необходимые настройки можно выполнить через пункты меню Параметры > Настройки > Проекты > "Имя проекта" > Управление > Библиотеки символов).
* Вы задали в настройках проекта специальный символ, который должен использоваться при размещении текстов свойств в блок-схемах рабочего процесса (например, OS3 для диаграмм GRAFCET).
* Вы вставили функции или устройства в проект и указали необходимые значения для свойств Функциональный текст, Статус [1] по Статус [10] и т. д.

!!! note "Замечание:"

    * Если вы создаете диаграмму GRAFCET, настройка проекта Функциональное размещение с графикой символов должна быть отключена.
    * Свойство Состояние [10] по умолчанию предназначено для ввода переменных для диаграмм GRAFCET. При использовании специального символа OS3 значение свойства Статус [10] размещается как текст переменной.

1. Создайте с помощью пунктов меню Страница > Создать страницу проекта.
2. Выберите в диалоговом окне Новая страница в поле Тип страницы запись "Обзор (I)" и наберите в поле Описание страниц текст описания для страницы проекта.
3. Введите в таблице в поле Сетка значение "3 мм".
4. Щелкните по кнопке ++OK++.

!!! info "Для сведения:"

    Генерируется новая страница проекта. Если настройки сетки и используемая форма подходят друг к другу, то точки сетки попадут ровно в середину определенных в форме ячеек таблицы. Это облегчает процесс размещения устройств и стадий рабочего процесса.

Заголовок таблицы переменных создается с помощью макроса. Отдельную строку таблицы можно вставить с помощью размещения текстов свойств из навигатора устройств.

1. Откройте страницу проекта для блок-схемы рабочего процесса, например, с помощью двойного щелчка мыши на странице проекта в навигаторе страниц.
2. Выберите пункт меню Вставить > Макрос окна / символа.
3. Выберите запись "Макрос окна (*.ema)" из раскрывающегося списка Тип файла диалогового окна Выбрать макрос.

!!! info "Для сведения:"

    В списке отображаются только доступные макросы окна.

4. Перейдите к каталогу макросов OS_ESS и выберите там макрос Table_GRAFCET.ema.
5. Щелкните по ++Открыть++, а затем разместите макрос с заголовком таблицы на нужном месте в диаграмме.
6. Выберите пункты меню Данные проекта > Устройство > Навигатор.
7. Выделите одну из функций в навигаторе устройств и выберите во всплывающем меню пункты Функциональное размещение > С графикой символов.

!!! info "Для сведения:"

    Специальный символ, заданный в настройках проекта, появится рядом с курсором. Если вы используете специальный символ OS3, то отобразится строка таблицы с текстами свойств функции (т. е. значение свойства Статус [10]).

8. Разместите графику (строку таблицы) на нужном месте в диаграмме.
9. Повторите описанные выше шаги для других устройств, которые должны быть представлены в таблице переменных.

Порядок рабочего процесса чертится с помощью графических элементов (например, ломаных линий). EPLAN поддерживает ускоренную вставку специальных элементов блок-схем рабочего процесса. Стрелки, сигнальные точки, условия 'И' и 'ИЛИ', шаги GRAFCET, действия GRAFCET и прохождения GRAFCET можно внести в блок-схему рабочего процесса с помощью соответствующих пунктов меню Вставить > Блок-схема рабочего процесса или панели инструментов Блок-схема рабочего процесса.

1. Если вам необходимо начертить ломаную линию, выберите пункты меню Вставить > Графика > Ломаная линия. (Пользуясь другими подпунктами меню Графика, вы можете чертить также прямоугольники, круги и т. д.)
2. Укажите по очереди точки ломаной линии и щелкните левой клавишей мыши, чтобы разместить каждую из этих точек.
3. Завершите операцию через [Пробел] или пункт всплывающего меню Закрыть ломаную линию.
4. Если вам необходимо вставить шаг, выберите пункты меню Вставить > Блок-схема рабочего процесса > GRAFCET > Шаг. (Через подпункты меню Шаг с прохождением, Действие, Действие с привязкой и Прохождение можно вставлять другие элементы GRAFCET.)

!!! info "Для сведения:"

    Шаг появится рядом с курсором.

5. Нажимая клавишу ++Tab++ несколько раз, выберите нужный вариант шага, например начальный шаг.
6. Разметите шаг в блок-схеме рабочего процесса.

!!! info "Для сведения:"

    После этого символ снова появится рядом с курсором, и вы сможете разместить следующий шаг.

7. Завершите действие, выбрав пункты меню Обработать > Прервать операцию или нажав клавишу ++Esc++.
8. Повторите описанные выше шаги для других элементов GRAFCET, которые следует разместить в блок-схеме рабочего процесса.

!!! tip "Совет:"

    * Специальные элементы блок-схемы рабочего процесса (например, косую черту, различные источники сигнала и т. д.) можно вставлять в качестве макроса окна из каталога макросов OS_ESS. Дополнительно макрос Arrows.ema позволяет вставлять предварительно подготовленные стрелки в диаграмму GRAFCET.
    * Звездочка (*) во вставленных элементах GRAFCET показывает, что здесь можно вводить тексты, например имена шагов, имена или условия прохождений или идентификатор действия. Различные объекты элементов GRAFCET объединяются в группу. Для непосредственного ввода отдельного текста без применения группировки удерживайте клавишу ++Shift++ и щелчком по соответствующей звездочке откройте диалоговое окно Свойства — текст.
    * Вариант C специального символа OS3 дает возможность размещать только видимое ОУ выделенного устройства. Вариант D позволяет разместить только переменную выделенного устройства. Таким образом, обозначение устройства или переменные можно вставлять и как тексты в управление процессом диаграммы GRAFCET.
