# Вкладка Схема соединений (Размещение изделия 3D)

Вы открыли проект. Навигатор пространства листа открыт, и открыто пространство листа. В пространстве листа находятся трехмерные размещения изделия. Вы дважды щелкнули по размещению изделия или выбрали пункты меню Обработать > Свойства. В диалоговом окне Свойства <Условное обозначение>: Размещение изделия (3D) выберите вкладку Схема соединений.

На этой вкладке отображаются данные выводов устройства, которые присваиваются размещению изделия через изделие или были определены графически. Данные выводов устройства могут быть определены на изделии в присвоенной схеме соединений или в шаблонах функций. Эти данные выводов устройства можно сохранять и изменять локально на размещении изделия.

Обзор основных элементов диалогового окна:

### Схема

Выберите из раскрывающегося списка схему, которую требуется использовать. Или щелкните по кнопке ++...++, чтобы в диалоговом окне Настройки: Конфигурация столбцов обработать схему или создать новую. Схема определяет, какие данные вывода устройства и в каком порядке будут отображаться. Так можно скрыть ненужные столбцы.

### Локальная схема соединений

Если этот флажок установлен, данные выводов устройства сохраняются локально на размещении изделия. Если изделию присвоена схема соединений или на изделии сохранены шаблоны функций, данные выводов устройства копируются, а защита от записи снимается. После этого данные выводов устройства можно обрабатывать в таблице. При синхронизации изделий изменения, вносимые вручную, остаются и не перезаписываются.

Если для какого-либо размещения изделия выводы устройства были определены графически, флажок устанавливается по умолчанию, а графически определенные выводы устройства отображаются в таблице.

Если флажок снят, используются данные выводов устройства, которые определены на изделии. В этом случае данные выводов устройства защищены от записи. Вносить изменения можно только в поля Обозначение вывода устройства, Обозначение штекера и Интерфейс шины: Имя, а для клемм — также в поля Уровень и Внутр./внешн. индекс. Эти "идентифицирующие" поля используются для идентификации вывода устройства в базе данных изделий: Если значения в этих полях для вывода устройства, определенного на изделии, и вывода устройства, имеющегося в размещении изделия, совпадают, речь идет об одном и том же выводе устройства.

### Таблица

В таблице в зависимости от выбранной схемы отображаются специфические данные выводов устройства, которые относятся к схеме соединений.

* Строка: Непрерывный номер строки.
* Статус: В этом столбце отображается, отличается ли схема соединений на размещении изделия от схемы соединений присвоенного изделия. Пиктограммы расшифровываются следующим образом:

Кнопка |  Значение
---|---
![](images/all_statusok_as.png){: .ui-icon } |  Обозначает вывод устройства, который копируется из базы данных изделий без изменений. На локальной схеме соединений все выводы устройства имеют этот идентификатор.
![](images/all_statusnotok_as.png){: .ui-icon } |  Обозначает вывод устройства, определенный для изделия, но отсутствующий в размещении изделия. Такой вывод устройства появляется, когда вы изменяете записи в "идентифицирующих полях" для какого-либо вывода устройства, скопированного из изделия. В этом случае первоначальный вывод устройства, определенный на изделии, отображается с таким идентификатором.
![](images/erroricon_error_systemmessages.png){: .ui-icon } |  Обозначает вывод устройства, определенный в размещении изделия, но отсутствующий в изделии. Такой вывод устройства появляется, когда вы добавляете новый вывод устройства или изменяете записи в 'идентифицирующих' полях для какого-либо вывода устройства, скопированного из изделия.
* Обозначение вывода устройства: Введите здесь имя позиции вывода устройства. По умолчанию (если не указана позиция) выводы устройства с четными обозначениями размещаются на функциональном элементе снизу, а выводы устройства с нечетными обозначениями — сверху.
* Обозначение штекера: Введите здесь ОУ штекера, с которым связан вывод устройства.
* Интерфейс шины: Имя: Для вывода шины введите здесь имя интерфейса шины.
* Уровень: Если вывод устройства представляет собой многоуровневую клемму, запишите сюда уровень.
* Внутренний / внешний индекс: Указывает для вывода устройства на схеме соединений, какой по счету внутренний или внешний вывод устройства он представляет.
* Позиция X / Позиция Y / Позиция Z: В обоих этих полях указывается позиция вывода устройства.
* Направление подсоединения: Выберите из раскрывающегося списка одно из возможных направлений вывода устройства "Автоматически", "Вверх" или "Вниз", "Влево" или "Вправо".
* Вектор X / Вектор Y / Вектор Z: В этих полях задаются соответствующие части длины направляющего вектора выводов устройства для направлений выводов устройства. Таким образом можно задать любое направление в пространстве. Значения задают преимущество, с которым вектор указывает в направлении соответствующей оси.

!!! example "Пример:"

    Оси X, Y и Z расположены под углом 90° друг к другу.Значение вектора 1 означает: параллельная ориентация по отношению к соответствующей оси.Значение вектора 0 означает: ось не влияет на выравнивание вектора.Значение вектора между 0 и 1 смещает влияние оси в диапазоне между 0° и 90° (0,5 = 45°, 0,25 = 22,5°). Также возможны отрицательные значения.Если у всех осей одно и то же значение вектора, вектор выравнивается точно в точке пересечения осей под углом 45° ко всем осям.Вектор Ввод значенийX = 1, Y = 0, Z = 0Определяет вектор параллельно оси Х.X = 0,5, Y = 0, Z = 1Определяет вектор, параллельный вектору, лежащему в слое плоскости осей Х-Z, под углом 26,5°, измеренным от оси Х.X= – 0,5, Y=0, Z=1Определяет вектор, параллельный вектору, лежащему в слое плоскости осей Х-Z, под углом 180°-26,5°=153,5°, измеренным от оси Х.X = 0, Y = 0,5, Z = 0,5Определяет вектор, параллельный вектору, лежащему в слое плоскости осей Y-Z, под углом 45° между осями Y и Z.

* Дополнительная длина: Введите здесь значение дополнительной длины позиции вывода устройства.
* Категория соединения: выберите из этого раскрывающегося списка значение по умолчанию, которое точнее определяет тип вывода устройства соединений, например винтовое подключение или пружинная клемма.
* Размер присоединения: В этом поле можно указать размер винта (например, M6) или накладки (например, 4,8x0,5) для винтовых подключений, резьбы, ниппелей и штекерных выводов.
* Мин. поперечное сечение / Макс. поперечное сечение: Укажите здесь значения для наименьшего и наибольшего поперечного сечения провода, которые можно подключить к выводу устройства.
* Макс. число соединений: Введите здесь максимальное число подключаемых проводов.
* Предписать двойную трубку: При помощи флажка укажите, следует ли использовать для вывода устройства двойные трубки, если к этому выводу устройства ведут два соединения.
* Мин. AWG / Макс. AWG: Укажите здесь значения для минимального и максимального поперечного сечения AWG, которые можно подключить к выводу устройства.
* Размер наконечника: инструмент для завинчивания (например, шуруповерт) состоит из привода и рабочей части. Привод — это компонент, который приводит в движение инструмент, например рукоять или вал двигателя. Рабочая часть — это компонент, который приводится в действие, например наконечник шуруповерта. Размер наконечника описывает габариты рабочей части (наконечника), например "3 x 0,5" = ширина 3 мм и толщина 0,5 мм. С помощью стандартной аббревиатуры можно указать тип исполнения рабочей части, например "PZ 1" — для крестообразного шлица Pozidriv или "TX 5" — для резьбовой головки Torx.
* Мин. момент затяжки / макс. момент затяжки: указывается в ньютон-метрах; описывает силу, с которой, например, затягивается винт, т. е. силу, которая передается с привода на рабочую часть.
* Длина зачистки: длина в миллиметрах, на которой машина для сборки проводов обрезает изоляцию.
* Интерфейс шины: имя интерфейса шины вывода устройства в схеме соединений.

### Всплывающее меню

Всплывающее меню дает доступ, в зависимости от типа поля (например, дата, целое число, многоязычный), к пунктам меню, при помощи которых вы можете по необходимости, например, влиять на представление таблиц или обрабатывать значения в полях. Обзор пунктов этого всплывающего меню вы можете найти в разделе [Пункты всплывающего меню](userinterface_m_kontextmenu.md).

Дополнительно здесь представлены следующие пункты всплывающего меню, специфические для данного диалогового окна:

Пункт меню |  Значение
---|---
Адаптировать схему соединений |  С помощью этого пункта всплывающего меню изменения на локальной схеме соединений, которая была присвоена размещению изделия через изделие, записываются обратно на схему соединений в базе данных изделий.
