# Системные сообщения

EPLAN документирует при старте и во время работы системы все возникающие события в форме сообщений, которые выводятся в диалоговом окне системных сообщений. Откройте диалоговое окно с помощью пунктов меню Сервисные программы > Системные сообщения.

### Номер сообщения

Каждому системному сообщению присвоен ***шестизначный*** номер сообщения, который состоит из трехзначного номера модуля и трехзначного текущего номера. Дополнительно к номеру сообщения выводятся дата и время возникновения сообщения.

### Категории сообщений

Существуют три различных категории сообщений, которые обозначены соответствующими пиктограммами. Список и значение отдельных категорий сообщений приведен в таблице ниже.

Пиктограмма |  Категория |  Значение
---|---|---
![](images/all_statusadditional_as.png){: .ui-icon } |  Замечание |  Вы получаете примечание, если функциональность выполняема. Программа работает безупречно.
![](images/warningicon_systemmessages.png){: .ui-icon } |  Предупреждение |  Вы получаете предупреждение, если определенная функциональность выполняема, но в последующем процессе проектирования могут возникнуть проблемы. Вы можете продолжать работу с программой, но как можно скорее должны понять причину вывода сообщения.
![](images/erroricon_error_systemmessages.png){: .ui-icon } |  Ошибка |  Вы получите сообщение об ошибке, если определенная функция более невыполнима, но другие части программы могут продолжить работу. Завершите работу с программой и запустите ее заново.
![](images/erroricon_systemmessages.png){: .ui-icon } |  Фатальная ошибка |  Если не выполняется какая-либо функция и программа находится в неопределенном состоянии, то возникает фатальная ошибка. Закончите работу с программой и запустите ее заново.

Такое системное сообщение совсем не означает, что вы больше не сможете работать дальше с программой. Например, каталог, к которому вы хотите обратиться, не находится больше на выбранном дисководе, или один из запрошенных вами наборов данных в данный момент используется другим пользователем.

Поэтому, пожалуйста, внимательно читайте описание, выдаваемое к каждому сообщению. В большинстве случаев вы сможете распознать проблему и быстро ее устранить.

### Отображение сообщений

Системные сообщения отображаются в форме списка с иерархической структурой. Каждый сеанс в списке визуализируется с помощью пиктограммы ![](images/sessionicon_systemmessages_as.png){: .ui-icon } и имеет номер сеанса и процесса. Дополнительно отображается имя обработчика и обозначение номера рабочей станции. Под пиктограммой появляются выводимые во время соответствующего сеанса системные сообщения. Список системных сообщений можно развернуть и свернуть, щелкнув по символам ![](images/all_plussign_as.png){: .ui-icon } и ![](images/all_minussign_as.png){: .ui-icon }.

Программа по умолчанию отображает системные сообщения текущего сеанса. С помощью флажка Отображать все сеансы можно расширить список системных сообщений и просмотреть в базе данных список сообщений из всех остальных сеансов.

Если отметить системное сообщение, то оно будет показано в отдельном текстовом поле, расположенном ниже списка. Текстовое поле отображает (независимо от установленного размера диалогового окна системных сообщений) соответствующее сообщение в полном объеме.

Системное сообщение может состоять из нескольких отдельных сообщений с иерархической структурой. Такие сообщения можно узнать в списке по ![](images/all_plussign_as.png){: .ui-icon } перед пиктограммой категории сообщения. При нажатии на этот символ будет отображен следующий уровень иерархии с нижерасположенными сообщениями (развернуть). С другой стороны, значок ![](images/all_minussign_as.png){: .ui-icon } говорит о том, что соответствующий уровень уже развернут. Щелкая по этому символу, вы снова сворачиваете уровень иерархии. Отмеченное сообщение появится в отдельном текстовом поле, расположенном под таблицей.

### База данных системных сообщений

Выводимые на экран системные сообщения всех сеансов автоматически сохраняются в базе данных под именем EplLog.eldb. Если объем заполнения базы данных превышает 30 Мбайт, то при повторном запуске EPLAN создается ее резервная копия под именем EplLog_bak.eldb. Одновременно создается новая база данных под именем EplLog.eldb. При переполнении этой базы данных операция повторится. Старый файл резервирования EplLog_bak.eldb будет удален.

Путь к файлу базы данных системных сообщений можно задать в настройках (выбрав Параметры > Настройки > Рабочая станция > Сообщения > Система).

!!! note "Замечание:"

    После того как путь к файлу базы данных будет изменен, необходимо перезапустить EPLAN.

### Выбор и экспорт базы данных

Выбор базы данных

В диалоговом окне системных сообщений можно получить доступ к любой требуемой базе данных системных сообщений и отобразить ее содержимое. Например, можно открывать резервные базы данных из предыдущих сеансов, которые перед этим были переименованы и архивированы. Для этого в диалоговом окне под кнопкой [Файл] используйте пункт меню Открыть.

Экспорт базы данных

Если необходимо получить подробную информацию о выводимых во время сеансов сообщениях, базу данных можно экспортировать из диалогового окна системных сообщений в файл формата *.csv. Для этого под кнопкой [Файл] используйте пункт меню Экспортировать. С помощью интегрированной программы конвертации EPLAN преобразует содержащуюся в базе данных информацию. Затем открыть файл экспорта для анализа ошибок можно с помощью любого приложения.

!!! note "Замечание:"

    Программа конвертации запускается после двойного щелчка по файлу базы данных *.eldb, выполняя конвертацию содержимого базы данных в файл формата *.csv и автоматически открывая приложение, выбранное для этого формата. Таким образом, приложение можно использовать в качестве программы просмотра системных сообщений, не открывая при этом EPLAN.

### Отображение ошибки в строке состояния

Если с момента последнего открытия диалогового окна системных сообщений ошибка возникает снова, в правой части строки состояния появляется пиктограмма ошибки ![](images/erroricon_error_systemmessages.png){: .ui-icon }. Двойной щелчок по пиктограмме откроет окно с системными сообщениями. После того, как вы прочитаете системное сообщение и закроете диалоговое окно, пиктограмма исчезнет.
