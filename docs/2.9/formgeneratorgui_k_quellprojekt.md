# Отчеты из исходных проектов

Данные проекта схемы соединений (так называемый 'исходный проект') можно вывести в качестве отчета в текущем открытом проекте, 'проекте отчета'. В шаблонах проекта отчета укажите, из какого исходного проекта взяты данные.
При последующем отчете исходный проект на момент составления отчета открыт для чтения на заднем плане, данные считываются и в проекте отчета генерируются отчеты. В это время другой пользователь может дальше работать в исходном проекте. Такой порядок работы позволяет также обновлять отчеты в проекте отчета.

Включение исходного проекта в шаблоны отчетов можно выполнить несколькими способами:

* В диалоговом окне Отчеты — <Имя проекта> создайте новые шаблоны на вкладке Шаблоны при помощи кнопки ![](images/all_new_as.png){: .ui-icon } (Создать). В следующем диалоговом окне Определить отчет для указания источника данных имеется поле Исходный проект. При помощи ++...++ откройте диалоговое окно выбора, в котором можно выбрать необходимый исходный проект.
* Для уже имеющихся шаблонов на вкладке Шаблоны доступно свойство Исходный проект. Чтобы ввести здесь исходный проект, щелкните в этом поле столбец Значение и перейдите посредством [...] в диалоговое окно выбора проекта.

В структуре дерева отчетов и шаблонов диалогового окна Отчеты — <Имя проекта> имена проектов отображаются в виде первого слоя древовидной структуры. Если исходный проект внесен в шаблон, то шаблон будет представлен под именем данного проекта. Имя текущего проекта всегда стоит на первом месте.

Чтобы определить для сгенерированной страницы отчета, из какого проекта взяты данные для отчета, в свойствах страницы содержится свойство Отчет: Исходный проект (ид. 11067). При генерации отчета сюда автоматически вносится имя исходного проекта.

Возможное применение — создание временных проектов отчетов для ежедневного использования. Посредством этой операции оценивается временное состояние проекта схемы соединений. Для этого выделите сначала в навигаторе страниц проект схемы соединений, который будет использоваться как исходный проект. Затем сгенерируйте проект отчета (например, с помощью пунктов меню Сервисные программы > Отчеты > Генерировать проект отчета). После этого данные заголовка выбранного проекта будут скопированы и будет сгенерирован проект отчета без страниц. Шаблон отчета нового проекта уже содержит корректный исходный проект. Затем генерируются отчеты.

Поскольку в проекте отчета выполняется временный отчет, проект схемы соединений не блокируется для последующей обработки. Текущее состояние и официальные отчеты и далее остаются в проекте схемы соединений.

!!! note "Замечание:"

    Если в шаблоны отчета выбранного проекта уже внесен исходный проект, то он сохраняется при генерации проекта отчета. Это важно в тех случаях, когда исходный проект одновременно является проектом отчета (например, если содержание проекта отчета необходимо отображать в исходном проекте).

В диалогом окне Генерировать проект отчета также можно выбрать существующий проект. Также таким способом можно перезаписать существующий проект отчета. После того как вы подтвердите запрос нажатием кнопки ++Да++, "старый" проект отчета будет заменен.

#### Отдельные шаблоны как основание проектов отчета

Также в отчетах можно генерировать проект отчета на основе выбранных шаблонов отчетов. Для этого в диалоговом окне Отчеты - <Имя проекта> на вкладке Шаблоны доступен пункт всплывающего меню Генерировать проект отчета. Генерированный при помощи следующего диалогового окна Генерировать проект отчета проект отчета содержит только шаблоны, которые были выбраны в исходном проекте и данные заголовка исходного проекта.
