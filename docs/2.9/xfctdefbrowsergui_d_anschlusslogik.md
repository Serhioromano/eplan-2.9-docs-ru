# Диалоговое окно Логическая схема выводов устройства

Вы открыли проект. Вы выделили функцию в графическом редакторе или в навигаторе. Обработать > Свойства. Выберите в диалоговом окне Свойства ++...++ вкладку Данные символа / функции, и щелкните по ++Логика++.

В этом диалоговом окне обработайте логическую модель функции, то есть свойства выводов устройства.

Обзор основных элементов диалогового окна:

В таблице для каждого вывода устройства ***функции*** выводится один столбец. Если записи в полях отличаются от стандартных значений исходного определения функции, они выделяются желтым цветом.

!!! note "Замечание:"

    В ***выводах устройства***, ***выводах устройства ПЛК*** и ***распределенных клеммах*** возможна передача логической информации на другие функции с одинаковым ОУ, например передача потенциалов и отслеживание цели. Это касается свойств Передать потенциал в, Потребитель / источник в и Отслеживание цели (ПЛК) в. Для [распределенных клемм](terminalgui_k_verteiltedarstellungklemmen.md) передача возможна только в пределах одной и той же функции клеммы (в многоуровневых клеммах, следовательно, в пределах одного уровня).

    Для такой ***ссылки вывода устройства*** введите на исходной функции в поле соответствующих свойств обозначение вывода устройства целевой функции (другого вывода устройства, вывода устройства ПЛК или вывода устройства распределенной клеммы). Ввод этой ссылки вывода устройства должен выполняться согласно следующему синтаксису:

    Обозначение вывода устройства целевой функции |  Ввод ссылки вывода устройства |  Пример (Исходная функция, 1 вывод устройства) |  Пример* (Исходная функция, 2 вывода устройства)
    ---|---|---|---
    С цифрами |  в "" |  "4" |  2,"4"
    С буквами |  без "" |  B |  2,B
    Целевая функция с различным обозначением штекера (например, X2) |  с двоеточием |  "X2:2" |  2,"X2:2"
    Исходная функция с различным обозначением штекера, целевая функция без различного обозначения штекера |  с двоеточием (без обозначения штекера) |  ":2" |  2,":2"

    *В случае с выводом устройства / выводом устройства ПЛК с несколькими выводами устройства в логической схеме выводов устройства по умолчанию указывается номер вывода устройства, используемый для внутренней передачи другому выводу устройства функции. Ссылку вывода устройства для целевой функции необходимо дополнительно ввести для этого номера вывода устройства (напр., 2).

    С помощью контрольного прогона 007027 можно определять неправильно введенные данные для ссылок выводов устройства в логической схеме выводов устройства.

    Поскольку клеммы не имеют обозначений штекера, то ввод, например "X2:2", в распределенных клеммах обрабатывается как простое обозначение вывода устройства.

### Тип вывода устройства

Выберите тип вывода устройства из раскрывающегося списка, возможен:

* Не определено: Вывод устройства может соединяться с любыми другими выводами устройства.
* Жила / провод: Соединение — это либо жила в кабеле (тогда должен быть активирован флажок Вывод кабеля), либо отдельное соединение (провод).
* Перемычка: Соединение осуществляется через перемычку, например силовые перемычки при Interbus-Inline или перемычки клеммы. (Соединения с типом вывода устройства "Жила/провод" образуют перемычки, если вы присваиваете свойство Перемычка через точку определения соединения.)
* Прямой вывод устройства: Прямое соединение между функциональными элементами, например если карта ПЛК вставлена в каркас, предохранитель — в шину или соединение штырей штекера и гнезд.
* Световод: Соединение может быть частью гибридного кабеля. В отчетах можно производить фильтрование по этому типу вывода устройства.
* Радиосвязь: В отчетах можно производить фильтрование по этому типу вывода устройства.
* Не электрич. общ.: Вывод устройства может соединяться с любыми другими неэлектрическими выводами устройства (например, трубами, шлангами, световодами и т. д.).
* Функциональный вывод устройства (ПО): Функциональные выводы устройства предназначены для создания логики программ ПЛК и не выводятся в отчетах.
* Fluid-техника: Из настроек проекта для соединения Fluid копируются данные о том, идет ли речь о шланге или о трубе.
* Внутр.: Соединение в пределах функциональных элементов.
* Технология производственных процессов: Соединение при технологии производственных процессов.
* Мостовая перемычка: Соединение в виде мостовой перемычки для реализации клемм с выводами для мостовых перемычек.

### Подключение кабеля

Установите этот флажок, если соединение должно находиться в кабеле. Эта настройка учитывается при автоматическом создании кабеля.

### Подвод давления/управляющий вывод устройства

Для Fluid-Техники. Выберите из раскрывающегося списка какую-либо запись. Для выбора доступны: "Не определено", "Рабочая линия", "Блок управления", "Труб. для слива масла", "Носитель процесса", "Теплоноситель и хладагент", "Запр. трубопровод" или "Абонентская линия".

### Передать потенциал в

Задайте (при помощи номера вывода устройства), к каким другим выводам устройства передается потенциал.

### Тип потенциала

Выберите из раскрывающегося списка тип потенциала. Для выбора доступны: "Не определено", "L", "N", "PE", "PEN", "+", "M", "-" или "SH".

### Потребитель / источник в

Задайте (при помощи номера вывода устройства), к какому другому выводу устройства внутренне подключается пользователь или генератор (источник).

### Вид сигнала

Выберите из раскрывающегося списка вид сигнала для вывода устройства. Для выбора доступны, например, "Не определено", "Датчик", "Исполнительный элемент", "Электропитание, "Энергоснабжение устройства", "Источник", "Дискретный вход", "Дискретный выход", "Аналоговый вход", "Аналоговый выход", "Источник шины", "Вход шины", "Выход шины", "Подвод давления" или "Вакуумный вывод".

### Отслеживание цели (ПЛК) в

Задайте (при помощи номера вывода устройства), к какому другому выводу устройства передается отслеживание цели. Указанная здесь последовательность для отслеживания цели учитывается при [автоматическом присвоении символического адреса](plcgui_k_symbolischeadressen.md).

### Число целей

Задайте число возможных целей / соединений, которые должен иметь вывод устройства.

### Число мостовых перемычек

Введите здесь для клемм число возможных мостовых перемычек.

### Внутр. / внешн.

Определите, интерпретируется вывод устройства как внутренний или внешний. Данная настройка влияет на подключение устройств (и определяет представление в пределах отчетов как схему подключения устройства).

### Возможна искробезопасность

Активировать флажок, чтобы вывод устройства был искробезопасным. Если позднее для функции будет настроено, что она искробезопасна, тогда будут искробезопасны все выводы устройства, для которых активирован этот флажок.

### Допускается аналогичное обозначение вывода устройства

Если установлен этот флажок, то для соответствующего вывода устройств в пределах устройства допускаются аналогичные обозначения устройств. Т.е. если устройство содержит несколько функций, то данные функции имеют аналогичное обозначение выводов устройства.

Если флажок не установлен, то соответствующие обозначения выводов устройства в пределах устройства должны быть уникальными.

!!! note "Замечание:"

    Если вы размещаете функции в нескольких видах представления, вам не следует устанавливать эту настройку. Определите каждую функцию однозначно. Иначе при обработке во всех представлениях возникнут проблемы.

### Вывод устройства символов

Здесь вы присваиваете выводам устройства функции выводы устройства символов. Обычно выводу устройства "1" функции присвоен и вывод устройства "1" символа и т.д. Через раскрывающиеся списки это присвоение можно изменить.

### Всплывающее меню

Всплывающее меню дает доступ, в зависимости от типа поля (например, дата, целое число, многоязычный), к пунктам меню, при помощи которых вы можете по необходимости, например, влиять на представление таблиц или обрабатывать значения в полях. Обзор пунктов этого всплывающего меню вы можете найти в разделе [Пункты всплывающего меню](userinterface_m_kontextmenu.md).

Дополнительно здесь представлены следующие пункты всплывающего меню, специфические для данного диалогового окна:

Пункт меню |  Значение
---|---
Сбросить на стандарт |  Восстанавливает в выбранных полях стандартные значения.
