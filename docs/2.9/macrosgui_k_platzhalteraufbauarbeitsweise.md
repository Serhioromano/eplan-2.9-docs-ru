# Объекты-заполнители: Структура и принцип работы

Для объектов-заполнителей можно определить переменные.

Переменная всегда начинается с символа <, далее следует имя переменной и затем символ > (например, <Функциональный текст>). Так как объект-заполнитель может содержать несколько переменных, такой метод записи необходим для разделения отдельных переменных, следующих друг за другом. После этого EPLaN проверяет введенные данные и при необходимости сообщает о возможных ошибках в формате.

Перед завершающим символом > в квадратных скобках при необходимости указывается число разрядов. В данном случае это то место, где вставляется новая строка (например, <Функциональный текст[24]>). После числа разрядов можно указать еще одну опцию, отделив ее запятой (например, <Функциональный текст[24,1]>), позволяющую различать следующие виды новой строки:

* Фиксированное начало строки после x символов (опция "0"): В этом случае после определенного числа символов выполняется перевод на новую строку, если, например, необходимо разделить зависимые понятия (записи).
* Новая строка после x символов с распознаванием конца слова (опция "1"): В данном случае выполняется попытка выполнить перевод на новую строку после определенного числа символов, если система распознала конец слова.

Перед, позади и между переменными можно вводить любые символы (например, A\<PLZH1\>:\<PLZH2\>B). Эти символы всегда применяются в том виде, в котором они введены, т. е. они ***не*** заменяются. Если в примере PLZH1 должен быть заменен на 11, а PLZH2 на 22, то в итоге получается A11:22B.

Для ввода символов < и > в тексте необходимо ввести их два раза. Только таким образом можно показать их отличие от начального и конечного символа переменной. ***Не*** разрешается вводить эти символы в сами имена переменных.

В EPLAN Engineering Configuration ***не*** использовать специальные символы в именах переменных. Рекомендуется использовать буквы и символы подчеркивания в качестве первых символов, а затем уже цифры.

***Слои*** идентифицируются в программе по собственному ид. номеру. Ид. номер слоев можно использовать, например, при присвоении слоев посредством объекта-заполнителя. Для стандартных слоев ид. номер отображается в имени слоя. Например, слой EPLAN100, График.Общее имеет ид. номер 100. Определенные пользователем слои получают ид. номера от 1000 по нарастанию. Ид. номер определенного пользователем слоя отображается при экспорте конфигурации слоев и при поиске значения записи A1426 для соответствующего слоя в файле конфигурации, например, A1426="1000".
