# Представление страниц проекта

В навигаторе страниц все пользователи EPLAN могут просматривать свойства страниц, открывать страницы и печатать их. Кроме того, в зависимости от полномочий пользователя можно обрабатывать страницы ***одного*** проекта, например, копировать страницы, удалять или изменять свойства страницы. Обратите внимание, что этими функциями обработки можно воспользоваться только в том случае, когда выделены страницы одного и того же проекта, так как несколько страниц из разных проектов нельзя обрабатывать одновременно.

При запуске EPLAN в навигаторе страниц выбирается последняя активная страница. Для этого имя страницы сохраняется для конкретного пользователя. Если эта страница отсутствует в открытом проекте, выбирается первая страница.

!!! tip "Совет:"

    Если ввести соответствующий номер страницы с помощью клавиатуры, курсор переходит на первую запись, которая соответствует введенному номеру. Этот способ рекомендуется использовать в случае объемных проектов, так как в этом случае можно выполнять поиск страниц по имени.

С помощью ***фильтров*** можно ограничить число просматриваемых страниц. Фильтры используются как для представления в виде дерева, так и для списка.

### Представление в виде дерева

В представлении в виде дерева страницы выводятся в иерархической последовательности согласно типу и идентификатору (например, установка, место установки и т. д.).

* Одновременно можно выводить несколько проектов с разной структурой.
* Возможен многократный выбор страниц.
* Для графических, логических страниц, а также страниц отчетов используются разные пиктограммы. Кроме того, отличающиеся пиктограммы применяются и для представления структурных идентификаторов.
* Открытые страницы выделяются полужирным шрифтом.
* Имя страницы можно обработать непосредственно в дереве.

#### Представление структуры страниц в дереве

Первый уровень в дереве образуют проекты. Под проектами выводятся страницы. По умолчанию они отображаются в соответствии с настроенной структурой.

При этом в представлении структуры дерева навигатора страниц по умолчанию отображаются все использованные структурные идентификаторы блоков идентификаторов, для которых в структуре страниц задан статус ***Идентифицирующий***. Описывающие и не используемые идентификаторы по умолчанию ***не*** отображаются.

В настройках проекта (Параметры > Настройки > Проекты > "Имя проекта" > Представление > Структура дерева (страницы)) можно изменять представление структуры дерева в навигаторе страниц. Таким образом можно показать или скрыть структурные идентификаторы одного или нескольких блоков идентификаторов и / или дополнительно отобразить тип страницы.

!!! note "Замечание:"

    Обратите внимание на то, что последовательность страниц в ***представлении в виде списка*** навигатора страниц по умолчанию является основной настройкой для вывода страниц (печать, экспорт PDF и т. д.) и для их перелистывания в графическом редакторе. Чтобы применить для этих различных функций отображаемую в представлении структуры дерева последовательность, необходимо активировать настройку проекта Использовать последовательность представления также для вывода страниц.

#### Отчет по типам страниц

Для представления в структуре дерева можно дополнительно выбрать тип страницы. По умолчанию он стоит в начале списка свойств в настройках проекта. Если установить соответствующий флажок, то страницы сортируются в древовидном представлении навигатора страниц в самом верхнем слое по типу. При этом соблюдается следующая последовательность:

* Титульный лист
* Содержание
* Однопол. схема соедин.
* Многопол. схема соедин.
* Схема соединений Fluid
* Обзор
* Обзор функций (Fluid-Техника)
* Компоновка электрошкафа
* Графика
* Внешний документ
* Функциональная схема автоматизации
* Отчеты (при алфавитно-числовой сортировке отдельных отчетов).

#### Отображение подидентификаторов

Для проектов, в которых используются структурные идентификаторы, возможен ввод подыдентификаторов. При этом главный идентификатор отделяется от подидентификатора точкой. Подидентификаторы всегда выводятся после главного идентификатора или стоящего впереди вспомогательного идентификатора.

В настройках проекта указывается, должны ли главные идентификаторы и подыдентификаторы выводиться в дереве на одном уровне или подыдентификаторы будут выводиться на нижестоящем уровне.

### Представление в виде списка

В представлении в виде списка все страницы выводятся друг под другом в заданной последовательности.

* Команды открытия, копирования, вызова диалогового окна "Свойства" и т. д. используются так же, как и в представлении в виде дерева.
* Возможен многократный выбор страниц.
* Последовательность строк (т. е. страниц) для любого столбца можно сортировать в восходящем и нисходящем порядке, для этого требуется щелкнуть по заголовку столбца.
* Некоторые поля, например Описание страниц, Номер чертежа и т. д., можно обработать непосредственно в списке.
* Для представления в виде списка навигатора страниц при необходимости можно определить [сортировку страниц вручную](pagebrowsergui_d_manuelleseitensortierung.md).

#### Конфигурация отображения

В списке как минимум всегда выводится та же информация, что и в дереве. Можно самостоятельно настроить, какую другую информацию следует вывести в столбцах и в какой последовательности она будет представлена. Возможен вывод любого свойства страницы. Так, например, для проектов небольшого и среднего размера в списке может быть выведена вся требующаяся информация страницы. Если объем информации в проектах большего размера слишком велик и отображение данных выполняется слишком медленно, выведите в списке только самую необходимую информацию, а для отображения информации по отдельным страницам откройте отдельное диалоговое окно с информацией по странице.
