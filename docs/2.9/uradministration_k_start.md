# Управление правами

Функция управления правами позволяет защитить данные от неавторизованного доступа. В рабочем режиме вы можете определить, будет ли EPLAN использоваться с функцией управления правами или без нее. При работе в EPLAN с включенной функцией управления правами необходимо выполнять следующие условия:

* При каждом запуске пользователь должен входить в систему под своим пользовательским именем и паролем.
* Альтернативная возможность — использовать для входа имя пользователя учетной записи Windows. Если вошедший в Windows (локально) пользователь зарегистрирован как пользователь EPLAN, вход в систему выполняется автоматически. Пользователю не нужно вводить свое имя и пароль в EPLAN.
* Администратор системы EPLAN может создавать пользователей и группы пользователей и присваивать им соответствующие права на обработку проектов. При этом он может блокировать пользователям доступ к определенным проектным данным и функциям их обработки. Соответствующие пункты меню становятся неактивны. Таким образом системный администратор предоставляет каждому пользователю права на выполнение определенных команд, которые необходимы ему для его работы.
* Кроме того, администратор системы EPLAN может блокировать диалоговые окна на интерфейсе пользователя. Хотя заблокированные окна и можно открыть, в них нельзя отрабатывать данные и документы.
* API-действия также могут быть выборочно присвоены или заблокированы.
* Администратор системы EPLAN может в любое время изменить или дополнить настройки.

При работе в сети действующие в корпоративной локальной сети права доступа всегда имеют приоритет перед внутренними правами для работы в EPLAN. Так, например, пользователь с правами "Только чтение" не сможет изменить основные данные изделия, хранящиеся на сервере базы данных изделий, даже если у него есть такие права в EPLAN.
