# Обмен данными файлов конфигурации ПЛК

Данными конфигурации блоков управления ПЛК систем шин можно обмениваться с различными программами конфигурации ПЛК. Обмен данными ПЛК основан на файле обмена (файл конфигурации ПЛК), который пишет одну программу обработки и считывает другую. Этим файлом легко могут обмениваться проектировщик EPLAN и программист ПЛК.

Программы конфигурации ПЛК и EPLAN имеют различный порядок представления данных конфигурации. Когда в EPLAN представляются подробности электротехнического оборудования, в программах конфигурации ПЛК используется логический порядок представления для программирования ПЛК. Например, EPLAN содержит выводы устройства ПЛК, которые служат для электропитания. Эти выводы устройства не требуются в программе конфигурации ПЛК. В программах конфигурации ПЛК, наоборот, содержится информация об интерфейсе, например информация о маршрутизаторе, которая не требуется в EPLAN. Данные, которые программа обработки не находит в файле обмена из-за того, что другая программа обработки не распознает их, добавляются при импорте.

EPLAN одинаково обрабатывает [системы шин на основе Ethernet](plcgui_k_spsanschluesse.md) при экспорте / импорте, т. к. они имеют одинаковую топологию аппаратного обеспечения.

В дополнение к данным устройств ПЛК во время обмена данными ПЛК также происходит обмен данными устройств, подключаемых к шине. Для обмена данными ПЛК необходимо указать проект конфигурации.

### Объем интерфейса

Как правило, меняются следующие данные конфигурации:

* Данные аппаратного обеспечения использованных устройств
Создание каркаса с информацией об изделии.
* Таблица символов (именуемая также как список присвоений, таблица переменных и пр.)
Присвоение символического адреса адресу аппаратного обеспечения.
* Структуры сети
Логический порядок представления (принадлежность устройств сетям) всегда заменяется. Экспорт / импорт точной разводки кабелей зависит от системы шин и используемого формата обмена.

Из-за различного порядка представления программ и текущих разработок в области аппаратного и программного обеспечения обмен данными не всегда может соответствовать всем требованиям. Например, это относится к пассивным функциональным элементам в жгуте шины и использованным системам полевой шины.

### Идентификация устройства

Идентификация устройств как в EPLAN, так и в программе конфигурации ПЛК выполняется либо с помощью обозначения типа ПЛК, либо путем указания файла описания устройства. С помощью этих свойств также выполняется [присвоение изделий](plcgui_k_importbuskonfig.md) при импорте файлов конфигурации ПЛК.

Каждая карта ПЛК может иметь только ***одну*** идентификацию устройства. Комбинация нескольких изделий посредством ввода нескольких обозначений типа ПЛК, файлов описания устройства или индексов не поддерживается.

Обозначение типа ПЛК

Обозначение типа ПЛК часто соответствует номеру для заказа в каталоге аппаратного обеспечения соответствующей программы конфигурации ПЛК, при этом следует учитывать регистр и позицию пробелов. В определенных [форматах импорта](plcgui_k_importbuskonfig.md) несколько изделий с одинаковым обозначением типа ПЛК различаются по свойству Описание объекта.

Файл описания устройства

Указание файла описания устройства может потребоваться, если в системе шин применяются устройства разных производителей, например устройство Beckhoff, подключенное к блоку управления Siemens.
Файл описания устройства содержит информацию об устройстве, которая отсутствует в стандартном каталоге аппаратного обеспечения производителя. Вы получаете файлы описания устройства (например, файлы GSD) от производителей используемых компонентов; эти файлы необходимо интегрировать в каталог стандартного аппаратного обеспечения программы конфигурации ПЛК.

Файл описания устройства указывается на карте ПЛК в свойстве Описание устройства: Имя файла. Вводится имя файла с расширением имени, но без указания пути файла. В дополнение к свойству Описание устройства: Имя файла необходимо указать свойство Описание объекта или — при обмене данными в формате AutomationML — Описание устройства: Индекс в файле. С помощью этих свойств устройство выбирается внутри файла при импорте файлов конфигурации ПЛК.

В свойстве Описание устройства: Имя файла следует сохранить не только имя файла GSD, но и другие записи, например идентификаторы устройства модулей CC-Link. Для этого непосредственно перед самим идентификатором устройства вводится префикс с ***двоеточием***, напр. CSP+:AJ65VBTCE2-8T. Эта запись будет экспортироваться без изменений. Для импорта это значение также должно содержаться в базе данных изделий, в противном случае устройство может быть не обнаружено при импорте.

Если запись не содержит префикс (без двоеточия) или префикс "GSD:", например GSD:SIEM8139.GSD, запись интерпретируется как файл описания устройства. В таком случае имя файла экспортируется вместе с индексом (при наличии).

!!! note "Замечание:"

    Если файл описания устройства указан, при обмене данными ПЛК свойство Обозначение типа ПЛК ***не*** экспортируется.
    При обмене данными ПЛК в формате AutomationML свойство Описание устройства: Имя файла экспортируется, только если ***не*** указано обозначение типа ПЛК.

### Рекомендации по проектированию

Начните с создания каркаса на однополюсной странице схемы соединений. В качестве главных функций используйте однополюсные блоки ПЛК.

* Путем создания каркаса проектируются данные аппаратного обеспечения используемых устройств. Все главные функции имеют однополюсное представление. При копировании этого представления содержащиеся в нем изделия не копируются.
* Отдельные функции (выводы устройства В/В, каналы, электропитание, выводы шины и т. д.) можно разместить из навигатора ПЛК с помощью перетаскивания мышью.
* Создание каркаса можно заменить с помощью программ конфигурирования ПЛК.

Свяжите однополюсные выводы шин узлов посредством автоматического соединения.

* Структуру сети можно наглядно задокументировать путем соединения однополюсных выводов шины.
* Структуру сети можно заменить с помощью программ конфигурирования ПЛК.

Разместите выводы устройства В/В с соответствующими датчиками / исполнительными элементами на многополюсных страницах схем соединений. Используйте побитное представление (т. е. по одному макросу на каждый канал) с функциональным группированием устройств. Отобразите на каждой странице по одному выводу устройства В/В с соответствующим датчиком / исполнительным элементом и при необходимости силовую электрическую цепь.

* Созданные макросы можно повторно использовать для идентичных функций машины.
* Выводы устройства В/В или каналы можно размещать или присваивать, перетаскивая их мышью из навигатора ПЛК.
* В навигаторе ПЛК или на странице обзора можно проверить загрузку.
