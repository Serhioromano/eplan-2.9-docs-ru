# Диалоговое окно Настройки: Общие (Проекты, Графическая обработка)

Вы открыли проект. Параметры > Настройки > Проекты > "Имя проекта" > Графическая обработка > Общее.

В этом диалоговом окне указываются настройки проекта, определяющие ***Представление*** ответвлений соединений, обозначений выводов устройства и т. д.

Обзор основных элементов диалогового окна:

### Отображать ответвления соединений

В этом групповом поле указывается, каким образом в схеме соединений будут представлены символы соединений:

* С определением цели: Для таких символов соединений, как тройники или перекрестные соединения, на экран выводятся "обычные" символы соединений, по которым видна последовательность целей соединений.
* Как точка: Символы таких соединений, как, например, тройники, перекрестные соединения и т. д., отображаются обычно в виде точек соединения.
* Как начерчено: Ответвления соединений отображаются так, как было настроено для соответствующего символа соединения при вставке.

!!! note "Замечание:"

    В пользовательских настройках (Параметры > Настройки > Пользователь > Графическая обработка > Символы соединения) установите настройки, которые позволяют ***начертить*** символы соединения.

### Отображать обозначения выводов устройства в повернутом виде

При создании символа можно создать два набора обозначений выводов устройства: один с неповернутыми и один с повернутыми на 90 градусов обозначениями выводов устройства. Например, такие обозначения вывода устройств используются для обработки согласно стандарту DIN. По умолчанию этот флажок снят и используются не повернутые обозначения вывода устройства. Установите этот флажок, чтобы использовать в проекте повернутые обозначения вывода устройства.

Эта настройка также влияет на свойство Обозначение клеммы / контакта штекера.

### Вставить также рамки макросов

Эта настройка доступна только в проектах схем соединений.

!!! note "Замечание:"

    Эта настройка действует только при условии, что при подготовке макросов в проекте макросов на соответствующей рамке макроса для настройки [Вставить также рамку макроса](macrosgui_r_makrokasteneinstellungen.md) выбран параметр "Из настроек проекта".

Установите этот флажок, если при вставке макроса окна и символа также требуется вставить рамку макроса. Эта настройка учитывается и при вставке устройства, если для него сохранен макрос. Рамки макросов содержат информацию о вставленных макросах, например, имя макроса и исходный проект, а также присвоение данных изделия. С помощью рамок макросов можно обновить вставленные макросы в проекте. При сжатии проекта рамки макросов можно снова удалить.

Если этот флажок не установлен, вставляются только отдельные составные части макроса. В этом случае при добавлении прибора не учитывается присвоение данных изделия рамке макроса!

### Вставить ссылающиеся макросы

Эта настройка доступна только в ***проектах макросов***.

Если этот флажок установлен, макросы с методом использования "Определяющий" вставляются в проект макросов с методом использования "Ссылающийся" и указанным исходным проектом. Макросы можно обновлять, но на самой рамке макроса можно внести только некоторые изменения.

Если этот флажок снят, такие макросы вставляются в проект макросов с методом использования "Определяющий" и без указанного исходного проекта.

### Инвертировать последовательность поиска для выводов устройства ПЛК

В случае выводов устройства ПЛК и выводов устройств поиск функциональных текстов зон сначала выполняется против направления вывода устройства, а затем (если в этом направлении ничего не было найдено) поиск производится в направлении вывода устройства.
Установите этот флажок, чтобы инвертировать последовательность поиска, то есть чтобы поиск начинался в направлении вывода устройства.

### Использовать альтернативный порядок свойств

В символах по ГОСТу (государственному стандарту Российской Федерации) обозначение устройства должно размещаться справа или справа вверху от графики символа.

Многочисленные символы библиотеки символов SPECIAL, позволяющие отобразить ОУ, отличаются соответствующими порядками свойств. Для переключения между различными порядками свойств (например Станд. знач. по умолч. (слева вверху, снаружи, 0°) -> Станд. знач. по умолч. (справа вверху, снаружи, 0°, (альтернативно))) при работе с этими специальными символами предусмотрен данный флажок:

Если он установлен, то для вставленных специальных символов проекта на вкладке Отображение свойству Порядок свойств присваивается "Стандартное значение по умолчанию (альтернативное)" по стандарту ГОСТ. Для тех символов из библиотеки, которые не имеют соответствующего порядка свойств, установка проходит без изменений — они сохраняют исходный стандартный порядок свойств.

### Распространить функциональный текст зоны на всю зону

Если установлен этот флажок, то поиск функционального текста зоны выполняется в пределах границ определенного рамкой зоны. В этом случае функциональный текст зоны переносится в свойство условного обозначения Функциональный текст (автоматически), если он находится в любом месте в пределах данной зоны. Необходимо, чтобы точка вставки функционального текста зоны находилась точно под / над точкой вставки схемы соединений. Если в зоне содержится несколько функциональных текстов зон, то в условное обозначение будет скопирован первый найденный текст. Для этого ведется поиск по всей зоне — одинаково для всех функций — слева направо и снизу вверх.

Если флажок не установлен, то функциональные тексты зон переносятся только тогда в свойство Функциональный текст (автоматически), когда точка вставки условного знака и точка вставки функционального текста зоны находятся точно друг над другом.

!!! note "Замечание:"

    Эта настройка выполняется только для проекта схемы соединений, но не для проекта макросов.

### Чертить обозначения местоположения с адаптацией

Если этот флажок установлен, в зависимости от положения текстов свойств левый верхний угол у обозначений местоположения чертится как углубление, и там размещается обозначение устройства. Чтобы углубление отображалось, тексты свойств следует располагать таким образом, как если бы они находились ***вверху слева*** и ***внутри*** исходного прямоугольника обозначения местоположения. Если вы, например, выберете порядок свойств "Слева вверху, внутри, 0°", то для ОУ и всех прикрепленных к нему свойств будет отображаться углубление. В качестве альтернативы можно выбрать аналогичный порядок свойств или вручную переместить тексты свойств.

Если этот флажок снят, то обозначения местоположений будут отображаться в виде обычных прямоугольников.

!!! example "Пример:"

    На следующем рисунке схематично показаны два углубления на обозначениях местоположения.А. C порядком свойств "Слева вверху, внутри, 0°"B. С порядком свойств, определенным пользователем: тексты свойств были перемещены вручную.

!!! note "Замечание:"

    В порядках свойств обозначений местоположения (и других устройств) для отображения и размещения обозначения устройства по умолчанию используется свойство ОУ (видимое) (Ид. 20010). Для обозначения местоположений с углублениями в качестве альтернативы можно использовать следующие свойства:

* ОУ (полное) (ид. 20006)
* ОУ (идентифицирующее) (ид. 20005).

### Применить порядок свойств из проекта

Определенные пользователем порядки свойств сохраняются как специфические для проекта и присваиваются варианту символа, для которого они были созданы. При копировании и вставке страниц или функций в другой проект или при вставке макросов сравниваются определенные пользователем порядки свойств. Имя порядка свойств для функции сохраняется, если порядки свойств совпадают или если порядка свойств еще нет в проекте. В этом случае генерируется порядок свойств.

Если одноименный порядок свойств уже существует в проекте и не совпадает с порядком свойств для функции, то эта настройка определяет, какой порядок свойств используется:

Если этот флажок установлен, то порядок свойств переносится из проекта на вставленную функцию, а имя порядка свойств сохранится на функции.

Если этот флажок снят, порядок свойств сохранится на вставленной функции, а имя изменится на "Определено пользователем".

!!! note "Замечание:"

    В пространстве листа вышеописанное для функций поведение действительно для размещения изделия. Для размещения изделия в пространстве листа порядки свойств присваиваются определению функции и, соответственно, доступны для размещения изделия с одинаковым определением функции.

### Не использовать новые конфигурации свойств из макросов

При вставке и копировании макросов определенные пользователем свойства, сохраненные в макросах, также вставляются в проект вместе со своими конфигурациями.

Если этот флажок установлен, то новым определенным пользователем свойствам в [диалоговом окне конфигурации](eservicesgui_d_konfigeigenschaften.md) будет присвоена настройка Больше не использовать. После этого данные свойства не будут отображаться в соответствующих объектах в проекте — они удаляются при повторном сжатии проекта.

Если этот флажок снят, то новые определенные пользователем свойства остаются доступными в проекте и отображаются в соответствующих объектах.

!!! note "Замечание:"

    Новые определенные пользователем свойства можно также вставить с помощью других основных данных, например форм, рамок и т. д. Эти свойства также учитываются при настройке.

### Рамка текста

Здесь задаются соответствующие предварительные настройки для рамки текста (отдельно для ширины и высоты). В диалоговом окне "Свойства" для текста эти предварительные настройки можно будет скопировать.

### Обзор модели

Здесь задается цвет фона для оттенения вида модели, для этого необходимо щелкнуть по ++...++ и открыть диалоговое окно [Выбор цвета](mf_d_farbauswahl.md).

### Мин. размер шрифта для проверки

Укажите минимальный допустимый размер шрифта. Это значение используется при проверке на наличие слишком мелких текстов. Слишком мелкий шрифт может также возникнуть в результате подгона текстов под блок выравнивания (напр., при отображении нескольких языков после перевода).
