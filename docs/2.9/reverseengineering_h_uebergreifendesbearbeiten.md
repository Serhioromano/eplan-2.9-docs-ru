# Общая обработка свойств на страницах отчетов

С помощью функциональности ***Общая обработка*** возможно выполнение изменений на страницах отчетов, которые немедленно применяются на всех логических страницах и во всех представлениях.

!!! note "Замечание:"

    * Если на странице отчета изменить данные объекта, на самой странице будут обновлены строка или столбец, в которых находится данный объект. Обратите внимание, что другие страницы отчетов, на которых, возможно, также находятся данные этого объекта, не изменяются. При необходимости эти отчеты следует дополнительно обновить.
    * На страницах отчетов типа Перечень форм, Перечень рамок, Обзор ревизий и Перечень символов общая обработка ***невозможна***.
    * Отчеты, которые были сгенерированы в другом проекте, теряют свои ссылки на первоначальные исходные объекты. Поэтому общая обработка на таких страницах также ***невозможна***.
    * Некоторые записи на страницах отчетов не имеют ссылок на объект. Для таких записей невозможно открыть диалоговое окно для обработки, при двойном щелчке по такой записи всегда открывается только диалоговое окно Свойства - текст заполнитель.

Условие:

Открыта страница отчета.

1. Выделите требуемый текст заполнитель на странице отчета.
2. Выберите Всплывающее меню > Свойства (общие).

!!! info "Для сведения:"

    Открывается диалоговое окно обработки, связанное с данным исходным объектом.

3. Выполните требуемые изменения в соответствующем диалоговом окне.
4. Щелкните по кнопке ++OK++.

!!! info "Для сведения:"

    Изменение немедленно применяется на странице отчета. Также автоматически выполняется обновление данных исходного объекта на всех логических страницах и во всех представлениях.

!!! example "Пример:"

    При просмотре открытого списка обозначений устройств было обнаружено, что плавкий предохранитель имеет неверное ОУ-F25. Установите курсор на обозначениеустройстваэтойфункциии выберитевсплывающее меню > Свойства (общие).Открывается диалоговое окно Свойства (общие): Устройство общего назначения. Щелкните на вкладке Плавкий предохранитель (устройство)в поле Полное ОУи, например, измените счетчикобозначенияустройства на3. После закрытия диалогового окна свойств нажатием кнопки ++OK++ новое ОУ будет немедленно выведено в списке обозначений устройств. Кроме того, на всех других логических страницах, связанных с данной функцией (например, многополюсная схема соединений) также будет выполнено обновление этого обозначения устройства.Вспецификацииизделий проекта для этого плавкого предохранителя продолжает оставаться неверное ОУ. Обновите эту страницу, например, вручную. Для этого выделите спецификацию изделий в навигаторе страниц и затем выберите Сервисные программы > Отчеты > Обновить. Теперь и в спецификации изделий имеется правильное ОУ.

Если в меню Параметры активирован пункт Свойства (общие), то при двойном щелчке по тексту заполнителю также можно открыть соответствующее диалоговое окно обработки исходного объекта. Если этот параметр не активирован, при двойном щелчке открывается диалоговое окно Свойства - текст заполнитель.

!!! example "Пример:"

    Открыта страница содержания проекта и требуется изменить для страницысхемы соединенийее описание Электропитание. Для этого в меню Параметрынеобходимо активировать режим Свойства (общие). Затем дважды щелкнуть в странице отчета по изменяемому тексту. Открывается диалоговое окно Свойства страницы. После изменения текста в поле Описание страницына Электроснабжение станции 1следует нажать кнопку ++OK++. В содержании для этой страницы будет выведен новый текст, в навигаторе страниц и в рамке страницы соединений также немедленно будет выполнено обновление изменения.

!!! tip "Совет:"

    EPLAN можно настроить таким образом, чтобы имеющиеся страницы отчетов автоматически обновлялись при открытии. Для этого следует активировать настройку пользователя Обновить отчеты при открытии страниц (Параметры > Настройки > Пользователь > Отображение > Общ.).
