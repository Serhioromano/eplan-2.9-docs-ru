# Работа, ориентированная на канал

Данные ПЛК могут отображаться в древовидной структуре навигатора ПЛК с ориентацией на канал. Все относящиеся к каналу выводы устройства ПЛК объединяются и отображаются под каналом (пиктограмма ![](images/plcgui_plcchannel_as.png){: .ui-icon }). Т. е. под обозначением канала отображаются выводы устройства В/В и соответствующие источники питания. Пиктограмма ![](images/plcgui_plcchannelwithsafty_as.png){: .ui-icon } для канала отображает, находится ли под ним релевантный для безопасности вывод устройства. Дополнительно установленная пиктограмма ![](images/navigatorgui_nonplaced_as.png){: .ui-icon } отображает, содержит ли канал неразмещенные выводы устройства В/В. Выводы устройств ПЛК, у которых нет ни внесенного вручную, ни автоматически определенного обозначения канала, упорядочиваются согласно ОУ.

Каналы можно вставлять в схему соединений с ориентацией на устройство или функцию. Оба принципа работы описаны ниже.

### Вставка каналов с ориентацией на устройства

При ориентированном на устройства режиме работы выберите канал в навигаторе ПЛК (в качестве части устройства) и разместите его.

#### Создание карты ПЛК

Сначала создайте в навигаторе ПЛК карту ПЛК (через пункт всплывающего меню Новое устройство). Если выбранное изделие обладает только шаблонами функции, но не макросами, то генерируется блок ПЛК, содержащий только шаблоны функций. Если изделие обладает только макросами, шаблоны функций наслаиваются через многополюсные макросы из внесенного в изделие файла макроса. Для присвоения функций шаблонам функций используется один вариант макроса за другим, пока не останется ни одного варианта макроса, содержащего подходящие выводы устройства ПЛК. Остальные функции, содержащиеся в используемых вариантах макроса, отображаются в навигаторе как неразмещенные функции. В этом случае блок ПЛК имеет также неразмещенные функции.

!!! note "Замечание:"

    Для ориентированного на устройства режима работы целесообразно использовать изделия, содержащие макросы изделий, так как эти макросы можно непосредственно размещать.

#### Размещение макроса

После выделения необходимого канала в навигаторе ПЛК разместите соответствующий макрос (с помощью пункта всплывающего меню Разместить, клавиши ++Backspace++ и последующего выбора вида представления в диалоговом окне Разместить устройство). При этом указанный на изделии макрос будет размещен, при его наличии, в противном случае откроется выбор макроса. Шаблоны функций устройств накладываются посредством функций макроса.

!!! note "Замечание:"

    Непосредственно перед размещением нажмите клавишу ++Shift++, чтобы запустить операцию размещения в режиме "Отдельные функции".

### Вставка каналов с ориентацией на функцию

При ориентированном на функцию режиме работы сначала разместите макрос, содержащий каналы ПЛК, а затем присвойте каналам неразмещенные функции или шаблоны функций (посредством операций Присвоить или Использование имеющегося).

#### Создание карты ПЛК

Сначала создайте в навигаторе ПЛК карту ПЛК (через пункт всплывающего меню Новое устройство). Если выбранное изделие обладает только шаблонами функции, но не макросами, то генерируется блок ПЛК, содержащий только шаблоны функций. Если изделие обладает только макросами, шаблоны функций наслаиваются через многополюсные макросы из внесенного в изделие файла макроса.

!!! note "Замечание:"

    Если функции необходимо ***присвоить по отдельности***, целесообразно использовать изделия, которые не содержат макросы изделия, чтобы не создавались наложенные шаблоны функций. При присвоении учитывается копирование ОУ. При присвоении отдельной функции может случиться, что остальные, еще не присвоенные выводы устройств ПЛК копируют ОУ этой функции. Данные выводы устройств ПЛК под каналом отображаются дважды: как неразмещенная функция с перенесенным ОУ (пиктограмма ![](images/navigatorgui_nonplaced_as.png){: .ui-icon }![](images/navigatorgui_function_as.png){: .ui-icon }) и как размещенная наложенная функция (пиктограмма ![](images/navigatorgui_functionstate_as.png){: .ui-icon }). Сразу после присвоения всех устройств ПЛК отобразятся только размещенные функции, которые наложены на шаблоны функций.

#### Размещение макроса

При ориентированном на функцию режиме работы вставьте многополюсные макросы или макросы страницы в схему соединений. Они служат в качестве целевых функций для присвоения. Макросы у функций ПЛК должны содержать только выводы устройств ПЛК (без блоков ПЛК), так как при присвоении из ориентированного на канал вида присваиваются только выводы устройств ПЛК. Если макросы, тем не менее, содержат блоки ПЛК, то они не могут иметь изделие, так как они сами представляют устройства, т. е. их выводы устройств ПЛК автоматически уже присвоены.

!!! tip "Совет:"

    При размещении макроса используйте режим Нумеровать с маркером "?", чтобы обозначить целевые функции ПЛК. Все остальные функции, содержащиеся в макросе (проводка выводов устройства) с помощью нумерации ОУ будут повторно пронумерованы.

#### Присвоение функций

После размещения макроса неразмещенные функции или шаблоны функций карты ПЛК присваиваются условному обозначению. Для этого выделите канал в навигаторе ПЛК и выберите пункт всплывающего меню Присвоить. Все функции выводов устройства ПЛК, относящиеся к выделенному каналу, отдельно присваиваются размещенным в схеме соединений выводам устройств ПЛК.

Как только канал через присвоение получает размещенный многополюсный ***Вывод устройства В/В***, канал в навигаторе ПЛК больше не обозначается дополнительной, установленной перед каналом пиктограммой ![](images/navigatorgui_nonplaced_as.png){: .ui-icon } для неразмещенных устройств. Кроме того, в навигаторе ПЛК пиктограмма ![](images/navigatorgui_function_as.png){: .ui-icon } больше не отображает целевые функции в схеме соединений, так как исходная функция выводов устройств ПЛК заменяется на неразмещенную функцию.

!!! note "Замечание:"

    При присвоении проверяется, относится ли неразмещенная функция или шаблон функций (исходная функция) к аналогичной группе функций как вывод устройства ПЛК в схеме соединений (целевая функция). Если группы функций исходной и целевой функции не совпадают, отображается сообщение.

!!! tip "Совет:"

    * ***Одновременное присвоение*** всех функций / шаблонов функций, относящихся к каналу, возможно через многократный выбор целевых функций. Для этого после выбора пункта меню Присвоить щелкните по схеме соединений и растяните рамку вокруг требуемого вывода устройства ПЛК. Существует другой способ, согласно которому перед присвоением необходимо сгруппировать выводы устройств ПЛК в схеме соединений (на канал группы).
    Неразмещенные функции или шаблоны функций присваиваются в том порядке, в котором они отображены в навигаторе ПЛК. На схеме соединений выполняется поиск подходящих целевых функций согласно их графической последовательности.
    * Можно присвоить функцию посредством Drag & Drop, удерживая нажатой комбинацию клавиш [Ctrl] \+ [Shift], перетащить требуемую функцию из навигатора ПЛК на целевую функцию в схеме соединений.

#### Учет вида представления

Если выбор в навигаторе ПЛК содержит только один вид представления для исходной функции, подходящей целевой функции (с аналогичным определением функции), он присваивается независимо от вида представления. Целевая функция сохраняет свой вид представления.

Если в навигаторе ПЛК выбраны несколько видов представления определенной исходной функции, присваивается функция, вид представления которой аналогичен подходящей целевой функции. Если при подходящих исходной и целевой функциях (с аналогичным определением функции) нет аналогичного вида представления, выбирается подходящая исходная функция согласно последовательности при общей обработке.

Вид представления функции, который прикреплен к курсору, отображается через дополнительный текст у курсора. Клавиша [Tab] позволяет переключаться между двумя видами представления.

!!! tip "Совет:"

    Альтернативой операции присвоения может быть операция Использ. имеющихся выводов устройства ПЛК поблочно для упорядочивания выводов устройства ПЛК. Для этого сначала выделите целевые функции на схеме соединений и затем выберите пункты меню Данные проекта > ПЛК > Использ. имеющихся выводов устройства поблочно.
