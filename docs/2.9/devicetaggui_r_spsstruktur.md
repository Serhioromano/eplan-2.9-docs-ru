# Вкладка Данные структуры ПЛК

Вы открыли проект.

* В графическом редакторе уже выделен блок ПЛК (главная функция). Обработать > Свойства. Выберите в диалоговом окне Свойства ++...++ вкладку Данные структуры ПЛК.
* Вы выделили блок ПЛК (главная функция) в навигаторе устройств или в навигаторе ПЛК. Пункт всплывающего меню Свойства или Свойства (общие). Выберите в диалоговом окне Свойства ++...++ вкладку Данные структуры ПЛК.

На этой вкладке задаются свойства, которые имеет вся карта ПЛК внутри системы шин. Кроме того, здесь задается присвоение карты ПЛК каркасу.

В режиме обработки ***Свойства (общие)*** вы изменяете свойства сразу для всех представлений функции, в этом случае во вкладке дополнительно отображается "(Устройство)".

Обзор основных элементов диалогового окна:

### Проект конфигураций

Введите здесь имя проекта конфигурации для блока ПЛК или выберите запись из раскрывающегося списка. В списке отображаются все значения, которые вы вводили в других местах проекта для подобных функций.

### Имя станции

Введите здесь имя рабочей станции для блока ПЛК или выберите запись из раскрывающегося списка. В списке отображаются все значения, которые вы вводили в других местах проекта для подобных функций. Имя станции служит элементом группировки для элементов сети; обычно оно задается в зависимости от физического расположения (электрошкаф, блок управления и т. д.) элемента сети.

### Каркас

Если блок ПЛК представляет собой каркас или первичную станцию, то введите здесь обозначение каркаса.
Каркасы расширения необходимо соединить с главным каркасом через выводы шины.

Для обмена данными ПЛК в формате AutomationML допускаются числовые значения, больше или равные 0. Нумерация каркаса на каждой рабочей станции начинается с 0 ("Главный каркас"), нумерация осуществляется непрерывно и должна быть уникальной в пределах одной рабочей станции. Следовательно, расширяющие каркасы имеют значения больше 0.

!!! note "Замечание:"

    ***Первичная станция*** характеризуется тем, что сама является каркасом и для нее установлен флажок Шинный интерфейс / первичная станция или флажок ЦПУ.

### Карта ПЛК вставлена в каркас

Если карта ПЛК вмонтирована в каркас или присоединена к первичной станции, введите здесь обозначение каркаса. Если карта ПЛК вставлена в каркас и сама является каркасом, необходимо также заполнить поле Каркас.

### На позиции / разъем

Для карты ПЛК, которая вставлена в каркас, в первичную станцию или присоединена к первичной станции, введите здесь номер позиции разъема.

[Групповое поле "Свойства"](devicetaggui_r_gruppenfeldeigenschaften.md)

Для карт ПЛК доступны, в частности, следующие свойства:

### ЦПУ

Установите этот флажок, если блок ПЛК представляет ЦПУ.

### Специфические для ПЛК настройки

Здесь отображается схема для специфических для ПЛК настроек, которая присвоена ЦПУ. Запись "Из настроек" показывает, что используется схема, заданная в настройках проекта.

Если флажок ЦПУ установлен, это поле доступно для обработки, и вы можете выбрать другую имеющуюся в наличии схему из раскрывающегося списка.

### ЦПУ: Имя

С помощью индексированного свойства ЦПУ: Имя [1-128] присвойте блоку ПЛК одно или несколько ЦПУ. Все адреса ПЛК с одинаковой спецификацией ЦПУ относятся к одному списку присвоений. Указание различных ЦПУ позволяет управлять несколькими контроллерами ПЛК в одном проекте. Так, могут быть задействованы, например, входы одного ЦПУ и выходы другого ЦПУ.

* Если блок ПЛК сам является ЦПУ, введите его имя в свойстве ЦПУ: Имя [1]. ЦПУ может иметь только одно имя, поэтому свойства для других имен ЦПУ выделяются серым цветом, если установлен флажок ЦПУ.
* Введите имя соответствующего ЦПУ на блоках ПЛК, представляющих карты входа или выхода, а не ЦПУ. Путем ввода имен ЦПУ укажите, какие ЦПУ могут обрабатывать сигналы карты ПЛК. Правильное указание имен ЦПУ на блоках ПЛК необходимо также для того, чтобы список присвоений экспортировался полностью.

!!! note "Замечание:"

    Имя ЦПУ должно быть уникальным внутри проекта EPLAN. На блоках ПЛК нужно всегда вводить полное имя ЦПУ в форме [Проект конфигураций].[Имя станции].[Идентификатор ЦПУ], например Siemens SIMATIC S7.Station 300.1.
    Поскольку в полном имени ЦПУ в качестве разделителя используется точка, значения для проекта конфигураций, имени рабочей станции и идентификатора ЦПУ не должны содержать точек.

### Шинный интерфейс / первичная станция

установите этот флажок, если текущая функция является шинным интерфейсом.

При обмене данными ПЛК этот флажок также используется для обозначения первичной станции. В таком случае необходимо заполнить поле Каркас для соответствующей карты ПЛК.

### Электропитание

установите этот флажок, если текущая функция является электропитанием.

### Шинный ответвитель

Установите этот флажок, чтобы обозначить блок ПЛК как шинный ответвитель

### Карта ПЛК вставлена в первичную станцию

Установите этот флажок для карт ПЛК, которые непосредственно вставлены или интегрированы в первичную станцию. Внесите в свойстве На позиции / разъем соответствующую позицию на первичной станции.

Для карт ПЛК, расположенных рядом с первичной станцией, флажок не устанавливается. Внесите в свойстве На позиции / разъем соответствующую позицию рядом с первичной станцией.

### Система шин

Выберите из раскрывающегося списка систему шин, например Profibus, Ethernet, ASI и т. д.

!!! note "Замечание:"

    Особенности системы шин "Local-Bus: Расширение", "DRIVE-CLiQ", "PortToPort" и "ET-Connection":

* Эти системы шин не образуют сеть.
* Для этих систем шин экспортируется только соединение между выводами шины (подробное соединение портов).
* Свойство Интерфейс шины: Имя (ид. 20447) должно быть одинаковым на всех соответствующих выводах шины устройства, пустая запись здесь также возможна.
* Адрес шины не требуется. Соответствующий контрольный прогон [004037](msg/messages_p_004037.md) можно подавить, активировав свойство Игнорировать недостающий адрес шины (ид. 20412) для этого вывода шины.

### Физическая сеть: Адрес шины / номер позиции

Занесите сюда адрес шины, который имеет данный блок ПЛК в присоединенной системе шин.

### Идентификатор сортировки (позиция в каркасе)

Идентификатор сортировки определяет последовательность карт ПЛК на каркасе. Обычно карты ПЛК сортируются соответственно значению свойства На позиции / разъем. Кроме того, карты ПЛК можно сортировать по указанному здесь идентификатору сортировки. Это целесообразно в первую очередь тогда, когда каркас содержит карты ПЛК, для которых введены одинаковые номера позиций в свойстве На позиции / разъем или для которых не указано номеров позиций. Если вы изменили последовательность карт ПЛК в диалоговом окне Обработать каркас, карты ПЛК получают идентификатор сортировки и сортируются в соответствии с этим идентификатором.
