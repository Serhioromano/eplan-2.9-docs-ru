# Отображение маркеров ревизии

Маркеры ревизии генерируются автоматически и отображаются в проекте. При этом возможны два принципа работы:

* если вы создали новую ревизию и продолжаете работать в проекте ревизии, изменения выделяются автоматически ("***Отслеживание изменений***").
* После сравнения свойств двух проектов изменения заносятся в список результатов ревизии и автоматически обозначаются в проекте. Если вы удалили эти обозначения, маркеры ревизий можно повторно сгенерировать из списка результатов, чтобы в проекте снова отображались изменения. По умолчанию результаты ревизии перезаписываются при каждом новом сравнении проектов.

Изменения ревизии в ***схеме соединений*** отображаются графическими маркерами. В настройках проекта (в меню Параметры > Настройки > Проекты > "Имя проекта" > Управление > Ревизия (графическое представление)) в диалоговом окне Настройки: Ревизия задайте внешний вид и размер этой графики.

Изменения в ***отчетах*** также обозначаются графическими маркерами. Графически выделяются измененные объекты на соответствующих страницах отчета.

Изменения в ***свойствах*** отображаются дополнительно с помощью текста маркировки, введенного в свойство Маркер ревизии (из отслеживания изменений) или Маркер ревизии (из сравнения свойств). Дополнительно измененный объект отображается с помощью графического маркера.

При отслеживании изменений этот текст создается автоматически, но его можно изменить. Задайте также форматирование в настройках проекта в диалоговом окне Настройки: Ревизия (отслеживание изменений).

При сравнении проекта укажите этот текст в диалоговом окне Сравнить свойства проектов.

!!! tip "Совет:"

    Если при копировании и вставке фрагментов схемы соединений, страниц или пространств листа, а также при вставке макросов вместе с ними не должны копироваться и вставляться в целевой проект ***ни*** данные ревизии, ***ни*** маркеры ревизии, деактивируйте пользовательскую настройку Сохранить данные и маркеры ревизии при вставке.

### Маркер удаления

Если в пункте Параметры > Настройки > Проекты > 'Имя проекта' > Управление > Ревизия (Отслеживание изменений) установлен флажок При удалении сгенерировать маркер удаления и перечислить удаленные страницы, удаленные объекты (условные обозначения, тексты и т. д.) на страницах проекта для сравнения будут отмечены маркером ревизии, а также специальным знаком — маркером удаления. Для маркера ревизии используется цвет, установленный для удаленных объектов. Маркер удаления находится на том же самом месте, где прежде находился объект, и располагает диалоговым окном 'Свойства'.

При сравнении свойств проектов также можно создавать маркеры удаления. Для этого в диалоговом окне Сравнить свойства проектов необходимо установить флажок Сгенерировать маркер удаления для удаленных объектов.

### Отображение удаленных страниц

Если флажок При удалении сгенерировать маркер удаления и перечислить удаленные страницы установлен, то при удалении страниц проекта ревизии сведения об удаленных страницах сохраняются. К соответствующему списку можно перейти из диалогового окна Удаленные страницы. В данное диалоговое окно можно попасть посредством пункта меню Сервисные программы > Управление ревизиями > Отслеживание изменений > Удаленные страницы. Представленные здесь данные не могут быть изменены.

Удаленные страницы также отображаются в обзоре ревизий: Если свойство формы Вывод ревизий с удаленными страницами (ид. 13088) активировано, удаленные страницы будут выведены в обзоре ревизии. Кроме того, для свойства Вид вывода ревизии (Ид. 13106) нужно выбрать параметр 'Страницы'.
