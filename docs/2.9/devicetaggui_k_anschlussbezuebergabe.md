# Перенос обозначения вывода устройства

Обозначения выводов устройства можно присваивать условному обозначению различными способами:

* По умолчанию обозначения выводов устройства копируются из определения функции. Определения функций содержат список возможных обозначений выводов устройства, при этом первая запись служит в качестве предварительно установленного значения. В условном обозначении можно выбрать запись из этого списка или вручную перезаписать обозначения выводов устройства. Разделители между выводами устройства можно генерировать, нажав ++Ctrl++ \+ [Ввод].
* Если Вы определили устройства и задали для них обозначения выводов устройства, то эти имеющиеся обозначения выводов устройства используются при размещении функции.
* При выборе устройства (или выборе изделия) перезаписываются обозначения выводов устройства, введенные в схеме соединений. Если в настройках для выбора устройства, в части критериев выбора установлен флажок Обозначение вывода устройства, то обозначения выводов устройства, введенные в схеме соединений, остаются без изменений.
* Обозначения выводов устройств, как и при [переносе ОУ](devicetaggui_k_bmkuebernahme.md), можно скопировать с одной из функций, лежащих в направлении поиска. Необходимым условием для этого является установка в настройках проекта флажка Копировать обозначение вывода устройства (Параметры > Настройки > Проекты > "Имя проекта" > Управление > Совместимость).

### Перенос из направления поиска

В функции можно вводить не только собственные обозначения выводов устройства, но и все обозначения выводов устройства "последующих условных обозначений", которые копируют ОУ от этой функции и находятся в схеме соединений справа от этой функции (или под ней). Так можно в условном обозначении вводить обозначения выводов устройства для всей серии условных обозначений. В последующих условных обозначениях можно выбрать из списка возможные обозначения выводов устройства. Возможные комбинации обозначений выводов устройства определяются на основании обозначений выводов устройства, предварительно настроенных во время создания символа.

Следующие условные обозначения подвергнутся влиянию только в том случае, если обозначений выводов устройства введено больше, чем нужно для текущего условного обозначения.

В диалоговом окне "Свойства" в поле "Обозначение вывода устройства" текущие обозначения выводов устройства всегда выводятся первыми в списке, после них выводятся возможные обозначения выводов устройства последующих условных обозначений. Для всех пустых обозначений выводов устройства отображается разделитель, благодаря чему видно, сколько обозначений выводов устройства ожидается.

Для следующих функций перенос обозначения вывода устройства ***не*** применяется:

* Кабели
* Кабельные соединения
* Черные ящики
* Клеммы
* Контакты штекеров.
