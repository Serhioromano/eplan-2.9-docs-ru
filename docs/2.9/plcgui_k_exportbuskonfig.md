# Экспорт файлов конфигурации ПЛК

### Подготовка к экспорту в проекте EPLAN

Условия для экспорта данных аппаратного обеспечения:

* Конфигурация ПЛК определяется с помощью проекта конфигураций. Проект конфигураций должен быть указан во всех связанных блоках ПЛК и может быть указан в выводах шины (поле Проект конфигураций на вкладках Данные структуры ПЛК и Данные шины в диалоговом окне "Свойства").
* На блоках ПЛК, представляющих главные функции, заполнено свойство Обозначение типа ПЛК или свойство Описание устройства: Имя файла. С помощью этих свойств выполняется [идентификация устройства](plcgui_k_austauschbuskonfig.md).
* Для блоков ПЛК, представляющих ЦПУ, установлен флажок ЦПУ и указано свойство ЦПУ: Имя (вкладка Данные структуры ПЛК в диалоговом окне 'Свойства').
* Выводы устройства ПЛК категорий "Источник питания", "Питание модуля", "Общее" и "2 вывода устройства" не заменяются.

Условия для экспорта структур сети:

* Выводы шины представлены однополюсно и связаны между собой. При обмене данными ПЛК в формате AutomationML учитываются также многополюсные, соединенные друг с другом выводы шин.
* На однополюсных или многополюсных выводах шины на вкладке Данные шины диалогового окна 'Свойства' заполнены поля, относящиеся к обмену данными ПЛК (Физическая сеть: Имя до Система шин и, при известных условиях, Логическая сеть: Вывод шины является главным).
* Для систем шин на базе Ethernet заполнено свойство Интерфейс шины: Имя.
* На вкладке Вывод устройства ПЛК диалогового окна "Свойства" заполнено свойство Обозначение штекера.

!!! tip "Совет:"

    * Логическая сеть — это подмножество физической сети. Следовательно, все выводы шины логической сети должны быть присвоены одной и той же физической сети. С помощью функции корректировки проекта и настройки Синхронизировать имена физических сетей можно перенести имя физической сети с главного шины на соответствующие подчиненные шины в одной и той же логической сети.
    * Для безошибочного экспорта структур сети все выводы шины физической сети должны отображаться в однополюсном или многополюсном представлении, и они должны быть соединены между собой. С помощью контрольных прогонов 004103 и 004104 можно проверить, содержат ли главный и подчиненный шины одинаковое значение свойства Физическая сеть: Имя, и все ли выводы шины физической сети связаны между собой.

Условия для экспорта выводов устройства ПЛК (вход / выход) выглядят следующим образом:

* Для вывода устройства ПЛК в проекте конфигураций предусмотрено ЦПУ. Имя ЦПУ в формате [Проект конфигураций].[Имя станции].[Идентификатор ЦПУ] должно быть однозначным для всего проекта.
* Вывод устройства ПЛК имеет символический адрес. В рамках одного ЦПУ символические адреса должны задаваться однозначно.
* Тип данных (BOOL, BYTE и т. д.) указан на выводе устройства ПЛК. С помощью функции корректировки проекта и настройки Установить типы данных по идентификатору можно автоматически установить тип данных, если соответствующее поле пустое. Имеющиеся значения не перезаписываются.

Условия для экспорта каркасов выглядят следующим образом:

* Поле Каркас заполнено (вкладка Данные структуры ПЛК в диалоговом окне свойств блока ПЛК).
* Для узлов на каркасе заполнены поля Карта ПЛК вставлена в каркас и На позиции / разъем.
* В случае с каркасом или первичной станцией поле На позиции / разъем остается незаполненным.
* EPLAN исходит из того, что устройства на одном и том же каркасе связаны шиной задней стенки. Она не проектируется отдельно.

!!! tip "Совет:"

    * Для эффективной обработки и проверки данных ПЛК используйте табличную обработку. Для различных вариантов использования есть предварительно определенные схемы, которые можно настроить в соответствии с вашими требованиями.
    * Перед экспортом проверьте наличие необходимых данных в проекте. Для этого предусмотрены контрольные прогоны класса сообщения 004 "ПЛК / шина". Для выбранных производителей ПЛК в диалоговом окне Настройки: Сообщения и контр. прогоны есть предварительно определенные схемы.
    * Перед экспортом проверьте, нет ли противоречащих друг другу данных ПЛК в разных видах представления одного вывода шины. Для этого используйте контрольный прогон 007010. Чтобы сохранить только данные в однополюсном представлении, используйте сжатие проектов с настройкой сжатия "Убрать данные проекта > Данные шины на неоднополюсных выводах шины".
