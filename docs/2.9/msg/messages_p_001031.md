# P001031: Распределенная клемма не была учтена при оптимизации. \<x\>

### Причина

Невозможность учета распределенной клеммы при оптимизации может быть вызвана несколькими причинами:

* Распределенная клемма имеет неизвестные обозначения выводов устройства. Например, если оптимизируемая распределенная клемма имеет определенные обозначения выводов устройств, которые следует сохранить после оптимизации. Для этого в диалоговом окне Оптимизация распределенных клемм установлен флажок Сохранить имеющиеся обозначения выводов устройства, но впоследствии для оптимизации были выбраны изделия клеммы с установленными посредством определения функций обозначениями выводов устройств.
* Распределенная клемма имеет больше выводов устройств, чем определение функции вновь выбранного изделия клеммы. Например, если для оптимизации выбрано изделие, определение функции которого содержит меньшее количество выводов устройств, чем оптимизируемая распределенная клемма.
* Новое определение функции недействительно. Например, если для оптимизации соответствующей распределенной клеммы выбрано изделие, отличное от изделия клеммы.

Разные причины выводятся в тексте сообщения с заполнителем \<x\>.

### Решение

1. Установите местоположение данной распределенной клеммы на схеме соединений, пользуясь функциональностью Перейти к (графика), которую можно найти во всплывающем меню диалогового окна Управление сообщениями.
2. Выберите для каждого из трех случаев изделие клеммы с подходящим определением функции и подходящими обозначениями выводов устройств.

!!! note "Замечание:"

    Относящиеся к модулю сообщения выводятся при оптимизации распределенных клемм. При выполнении нового контрольного прогона (любой настроенной схемы) его можно удалить из диалогового окна управления сообщениями.
