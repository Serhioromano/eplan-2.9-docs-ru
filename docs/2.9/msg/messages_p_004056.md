# P004056: Изделие не найдено: Обозначение типа ПЛК '\<x\>' или файл описания устройства '\<y\>' / индекс '\<z\>'! При импорте '\<u\>', возможно, произошла ошибка

### Причина

При импорте файлов конфигурации ПЛК возникают главные функции, на которых вводится изделие, если подходящее изделие содержится в базе данных изделий. В конкретном случае не удалось найти подходящее изделие в базе данных изделий.

Изделие считается подходящим, если свойство Описание устройства: Имя файла в файле конфигурации совпадает с одноименным свойством на изделии. Если файл описания устройства не указан, сравнивается свойство Обозначение типа ПЛК. Если файл описания устройства указан, для форматов AutomationML, т. е. Siemens SIMATIC STEP 7 TIA Portal (AutomationML) и Mitsubishi iQ-Works (AutomationML), дополнительно сравнивается свойство Описание устройства: Индекс в файле. В заполнителях от \<x\> до \<z\> сообщения выводятся значения этих свойств, если они внесены в файл конфигурации. В заполнителе \<u\> выводится видимое ОУ блока ПЛК.

### Решение

1. Создайте соответствующее изделие в базе данных изделий. Следите, чтобы на этом изделии для свойства Описание устройства: Имя файла, свойства Описание устройства: Индекс в файле или свойства Обозначение типа ПЛК было введено то же значение, что и в импортируемом файле конфигурации.
2. Затем повторите импорт ПЛК.

!!! note "Замечание:"

    Это относящееся к модулю сообщение можно удалить при выполнении нового контрольного прогона (любой настроенной схемы) из диалогового окна управления сообщениями.
