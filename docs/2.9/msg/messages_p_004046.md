# P004046: При копировании данных ПЛК для '\<x\>' возник конфликт

### Причина

Начиная с версии EPLAN 2.4 все данные структуры ПЛК можно найти только в главной функции блока ПЛК. При копировании проекта или размещении макросов старых версий данные (значения свойств) вспомогательных функций собираются и переносятся на главную функцию. В случае неудачной попытки на экране отобразится сообщение.

### Решение

1. Проверьте данные на главной функции и на вспомогательных функциях.
2. Временно обозначьте вспомогательную функцию как главную, чтобы отобразить вкладку Данные структуры ПЛК.
3. Скопируйте значения свойств в блок ПЛК, который впоследствии должен стать главной функцией.
4. Затем выберите пункты меню Проект > Организовать > Сжать, чтобы очистить проект.
5. Выберите в поле Настройки диалогового окна Сжать проект схему "Убрать избыточные данные проекта" и перейдите с помощью [...] к настройкам для сжимаемых данных.
6. Установите флажок Убрать данные проекта и в нижестоящей ветке дополнительно установите флажок Удалить данные структуры ПЛК во вспомогательных функциях ПЛК.
7. Запустите сжатие проекта, нажав кнопку ++OK++ для всех диалоговых окон Настройки.
8. Затем запустите новый контрольный прогон.
