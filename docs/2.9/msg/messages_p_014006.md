# P014006: Свойство 'Метод использования' неправильно настроено в рамке макроса: \<x\>

### Причина

Неверная настройка свойства Метод использования в рамке макроса. Текст с причиной ошибки выводится в заполнителе \<x\> сообщения.

Причина |  Дополнительные сведения |  Решение
---|---|---
Применяется метод использования 'Не задан' |  Рамка макроса с методом использования "Не задан" учитывается как при автоматическом генерировании макросов (в проектах макросов), так и при обновлении макроса. Это имеет смысл только в специальных решениях для автоматизации. Вложенные макросы создаются в направлении изнутри наружу. Поэтому методы использования следует указывать (снаружи внутрь) следующим образом: "Определяющий", "Ссылающийся", "Нижестоящий". Настройка "Определяющий" имеет смысл только в проектах макросов. |  Откорректируйте метод использования.
Применяется метод использования 'Определяющий' в проекте схемы соединений |  Метод использования "Определяющий" применяется в проектах макросов, чтобы указать, что рамку макроса следует учитывать при автоматической генерации макросов. Эта функция недоступна в проектах схем соединений, следовательно этот метод использования не имеет смысла в проектах схем соединений. |  Установите метод использования "Ссылающийся", если макрос следует учитывать при автоматическом обновлении. Если это не требуется, установите метод использования "Нижестоящий".
Применяется метод использования 'Определяющий' внутри окружающей рамки макроса, которая является ссылающейся или нижестоящей |  Вложенные макросы создаются в направлении изнутри наружу. Поэтому методы использования следует указывать (снаружи внутрь) следующим образом: "Определяющий", "Ссылающийся", "Нижестоящий". Настройка "Определяющий" имеет смысл только в проектах макросов. |  Откорректируйте метод использования.
Применяется метод использования 'Ссылающийся' внутри окружающей рамки макроса, которая является ссылающейся или нижестоящей |  Вложенные макросы создаются в направлении изнутри наружу. Поэтому методы использования следует указывать (снаружи внутрь) следующим образом: "Определяющий", "Ссылающийся", "Нижестоящий". Настройка "Определяющий" имеет смысл только в проектах макросов. |  Откорректируйте метод использования.
