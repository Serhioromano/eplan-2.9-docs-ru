# P013045: В импортируемых данных есть недействительное изделие

### Причина

В импортируемом файле (формата XML) для импорта спецификаций было изменено имя поля P_ARTICLEREF_IDENTNAME в тэге <device_P_ARTICLEREF_IDENTNAME>.

!!! example "Пример:"

    Тэг<deviceARTICLEREF_TEST_IDENTNAME>был переименован.

### Решение

1. При выводе сообщения следите за тем, чтобы в диалоговом окне Управление сообщениями также выводилось сообщение [P013048: Поле с именем \<x\> неизвестно и поэтому игнорируется](messages_p_013048.md). Это сообщение дополнительно показывает недостающее имя поля на месте заполнителя \<x\>.
2. Откройте используемый импортируемый файл в текстовом редакторе или редакторе файлов XML.
3. Найдите имя поля в тэге "device", указанное в сообщении P013048, и исправьте его, как указано в разделе "Причина".
4. Сохраните импортируемый файл и импортируйте его еще раз.
5. Затем запустите новый контрольный прогон.

Более подробную информацию и рекомендации вы найдете в разделе [Импорт спецификаций](../partslistgui_h_stuecklistenimportieren.md).

!!! note "Замечание:"

    Это относящееся к модулю сообщение можно удалить при выполнении нового контрольного прогона (любой настроенной схемы) из диалогового окна управления сообщениями.
