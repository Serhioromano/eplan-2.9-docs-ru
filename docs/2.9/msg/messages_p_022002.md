# P022002: Невозможно найти символ \<x\>, вариант \<y\> в библиотеке символов \<z\>

### Причина

В исходном проекте из символов определенной библиотеки символов \<z\> был создан макрос. После этого макрос был вставлен в целевой проект (текущий проект), в котором также сохранена библиотека символов, использованная при создании макроса. Однако перед сохранением символ \<x\> в определенном варианте \<y\> был удален из библиотеки символов, хотя этот символ является частью вставленного макроса.

!!! note "Замечание:"

    Отсутствующие символы выводятся в схеме соединений целевого проекта в виде красного символа "x".

### Решение

Заново создайте отсутствующий символ на основе имеющегося варианта и сохраните его в сохраненную библиотеку символов. Заново вставьте макрос.

Затем запустите новый контрольный прогон.

!!! tip "Совет:"

    Используйте функциональность синхронизации для основных данных, чтобы обновить библиотеку символов, сохраненную в целевом проекте. Для этого выберите пункты меню Сервисные программы > Основные данные > Синхронизировать текущий проект. Выделите соответствующую библиотеку символов в области основных данных диалогового окна Синхронизация основных данных — <Имя проекта> и щелкните по кнопке со стрелкой, указывающей влево. Выводится сообщение, что основные данные проекта обновлены.
