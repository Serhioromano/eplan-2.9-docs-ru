# P004106: Адрес ПЛК '\<x\>' не находится в диапазоне адресов соответствующей карты ПЛК

### Причина

На выводе устройства ПЛК определен адрес вне диапазона адресов соответствующей карты ПЛК.

Выводы устройства ПЛК с определением функции "Вывод устройства ПЛК, внутр., общ." не проверяются этим контрольным прогоном, поскольку они служат только для отображения свободных символических адресов.

### Решение

1. Установите местоположение вывода устройства ПЛК на схеме соединений с помощью функции Перейти к (графика), которую можно найти во всплывающем меню диалогового окна Управление сообщениями.
2. С помощью пункта всплывающего меню Перейти к (обратный эквивалент) вывода устройства ПЛК перейдите к соответствующему блоку ПЛК на карте ПЛК.
3. Вызовите диалоговое окно "Свойства" блока ПЛК.
4. В поле Начальный адрес карты ПЛК и Устройство ПЛК: Длина данных (...) определите значения, внесенные в список свойств на первой вкладке.
5. Измените адрес ПЛК на выводе устройства ПЛК так, чтобы он находился в пределах диапазона адресов карты ПЛК, либо соответствующим образом скорректируйте начальный адрес или длину данных в блоке ПЛК.
6. Подтвердите ввод.
7. Затем запустите новый контрольный прогон.

!!! note "Замечание:"

    Этот контрольный прогон проверяет, находится ли адрес вывода устройства ПЛК в пределах диапазона адресов соответствующей карты (от = "начальный адрес", до = "начальный адрес + длина данных").

    Для выполнения контрольного прогона необходимо соблюдать такие условия:

* Вывод устройства ПЛК имеет одно из определений функции "Вывод устройства ПЛК, аналоговый выход", "Вывод устройства ПЛК, аналоговый вход", "Вывод устройства ПЛК, дискретный выход", "Вывод устройства ПЛК, дискретный вход" или "Вывод устройства ПЛК, многофункциональный" с соответствующей логикой. Идентификатор адресации для входов и выходов не является решающим для проверки.
* Адрес ПЛК на выводе устройства ПЛК не должен быть пустым.
* Для вывода устройства ПЛК существует соответствующий блок ПЛК со свойствами Начальный адрес карты ПЛК и Устройство ПЛК: Длина данных. В соответствующие поля необходимо внести значения, и оба значения должны соответствовать типу В/В и индексу проверяемого вывода устройства ПЛК.
* Значение свойства ЦПУ: Имя (ид. 20253) не должно быть пустым. Эта запись определяет используемую схему адресации.
* В схеме адресации, относящейся к ЦПУ, в специфических для ПЛК настройках необходимо указать счетчик со значением конфигурации "Начальный адрес карты ПЛК".

    Кроме того, для контрольного прогона действуют следующие условия:

* Без указания схемы адресации диапазон адресов (до = "начальный адрес + длина данных") рассчитать невозможно.
* Длина данных указывается независимо от используемой схемы адресации как десятичное значение в ***битах***.
* Длину данных можно сохранить в базе данных изделий и оттуда копировать.
