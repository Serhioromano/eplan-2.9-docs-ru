# P013048: Поле с именем '\<x\>' неизвестно и поэтому игнорируется

### Причина

При импорте спецификации обнаружено недопустимое имя поля. Возможно, имя поля в файле импорта было изменено вручную, причем произошла ошибка печати. Как следствие, после импорта соответствующие данные игнорируются и не отображаются как свойство устройства в проекте.

### Решение

Откройте файл импорта в текстовом или XML редакторе и выполните поиск по символьной строке, заданной в диалоговом окне управления сообщениями под сообщением "013048".

Сравните строку символов с именем поля, указанным в справке в разделе [Спецификации: Поля в файле экспорта](../partslistgui_k_felderexportdatei.md). При обнаружении неверного имени поля исправьте его. Сохраните файл и заново импортируйте спецификацию.

!!! example "Пример:"

    Имя полясвойствасодержит ошибку печати. В имени поляP_ARTICLE_MANUFACTURE="Siemens" на конце имени поля отсутствует "R".Поэтому невозможно присвоить производителя. Правильное присвоениеP_ARTICLE_MANUFACTURER="Siemens".

!!! note "Замечание:"

    Это относящееся к модулю сообщение можно удалить при выполнении нового контрольного прогона (любой настроенной схемы) из диалогового окна управления сообщениями.
