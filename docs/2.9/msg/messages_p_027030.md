# P027030: Распределенный сегмент маршрутизации топологии недоступен попарно

### Причина

Два распределенных сегмента маршрутизации топологии с одинаковым ОУ вместе составляют один сегмент маршрутизации, а не пару. Причина заключается в том, что оба распределенных сегмента маршрутизации топологии одновременно являются главными или вспомогательными функциями.

### Решение

1. Установите местоположение первого распределенного сегмента маршрутизации топологии в сети соединенных сегментов, пользуясь функцией Перейти к (графика), которую можно найти во всплывающем меню диалогового окна Управление сообщениями.
2. Откройте диалоговое окно Свойства функции топологии и проверьте во вкладке Распределенный сегмент маршрутизации, установлен ли флажок Главная функция.
3. Повторите эту процедуру для второго распределенного сегмента маршрутизации топологии, также отображаемого в диалоговом окне Управление сообщениями.
4. Затем установите для одного распределенного сегмента маршрутизации топологии значение главной функции, а для другого — вспомогательной, чтобы создать пару распределенных сегментов маршрутизации топологии.
5. Подтвердите изменения, нажав кнопку ++OK++.
6. Затем запустите новый контрольный прогон.
