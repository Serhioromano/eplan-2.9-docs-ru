# P007023: Данные главной функции или главной клеммы не совпадают с данными изделия: \<x\>

### Причина

При проектировании схемы соединений был вставлен прибор в качестве главной функции или главной клеммы, поэтому возникшее таким образом устройство переняло сохраненные на приборе данные изделия, например значение свойства Технические параметры. Однако впоследствии это значение было изменено вручную в диалоговом окне Свойства ++...++ устройства, из-за чего совпадение с исходными данными изделия теперь отсутствует.

Для главной клеммы, которая представлена в распределенном виде, проверяется вся функция клеммы. В этом случае данные, введенные на отдельных распределенных клеммах одной и той же функции клеммы, объединяются.

### Решение

1. Найдите устройство в проекте, используя функциональность Перейти к (графика).
2. Откройте диалоговое окно Свойства ++...++ устройства.
3. Переместите вкладку Изделие на передний план и еще раз присвойте главной функции устройства исходные данные изделия, используя выбор изделия.
4. В отобразившемся диалоговом окне Конфликт выберите для измененного и выделенного желтым цветом свойства параметр Перезаписать.
5. Подтвердите ввод.
6. Затем запустите новый контрольный прогон.

!!! note "Замечание:"

    Относительно этой проверки у вас есть возможность настроить вид проверки "Предотвращение ошибок", если вы хотите запретить изменение исходных данных изделия на устройстве вручную. В этом случае вы получите сообщение, в котором будет указан текст контрольного прогона, страница, ОУ и позиция устройства. После этого изменения, внесенные вручную, будут отменены.
