# P027017: Топология - Схема соединений не определена

### Причина

На функции топологии не определена схема соединений.

То есть отсутствует локальная схема соединений или схема соединений, сохраненная на используемом изделии.

### Решение

1. Установите местоположение функции топологии в проекте, пользуясь функциональностью Перейти к (графика), которую можно найти во всплывающем меню диалогового окна Управление сообщениями .
2. Откройте диалоговое окно Свойства ++...++ функции топологии.
3. Переместите вкладку Изделие на передний план и проверьте сначала, есть ли у данной функции топологии изделие и записана ли для него схема соединений. В противном случае, если при маршрутизации для функции топологии будет использоваться схема соединений, определенная на изделии, действуйте согласно указаниям в разделе 'Возможность 1'.
4. Если изделие на функции топологии отсутствует, проверьте на вкладке Схема соединений, занесены ли в таблицу обозначения выводов устройств, которые идут от выводов устройства функции или свободно определены и еще не перенесены в локальную схему соединений. В противном случае, если при маршрутизации будет использоваться локальная схема соединений функции топологии, действуйте согласно указаниям в разделе 'Возможность 2'.

#### Возможность 1

При маршрутизации для функции топологии должна использоваться схема соединений, определенная на изделии. Для этого выберите в базе данных изделий одно изделие, на котором сохранена схема соединений. При этом рекомендуется действовать следующим образом.

1. Если диалоговое окно 'Свойства' функции топологии еще открыто, закройте его.
2. Выберите пункты меню Сервисные программы > Изделие > Управление и перейдите в дереве диалогового окна База данных изделий к изделию, которое вы хотите присвоить функции топологии.
3. Выведите на передний план вкладку Технические данные и щелкните рядом с полем Схема соединений по кнопке ++...++.
4. В открывающемся диалоговом окне Выбор схемы соединений выделите имя необходимой схемы соединений и щелкните кнопку ++OK++.

!!! info "Для сведения:"

    Имя будет скопировано в поле Схема соединений на вкладке Технические данные.

5. Закройте базу данных изделий, сохраните измененные данные и обновите базу данных изделий.
6. Еще раз откройте диалоговое окно Свойства ++...++ функции топологии.
7. Переместите вкладку Изделие на передний план и присвойте функции топологии выбранное изделие со схемой соединений из базы данных изделий.
8. Перейдите на вкладку Схема соединений и выберите из всплывающего меню пункт Обновить схему соединений.

!!! info "Для сведения:"

    Будет считана схема соединений из изделия.

!!! info "Для сведения:"

    Если функция топологии уже имеет выводы устройства, они перечисляются в таблице и помечаются пиктограммой ![](../images/erroricon_error_systemmessages.png){: .ui-icon }.

!!! info "Для сведения:"

    Если обозначения выводов устройства считанной, сохраненной на изделии, схемы соединений отличаются от таковых на функции топологии, они помечаются в таблице пиктограммой ![](../images/all_statusnotok_as.png){: .ui-icon }.

9. Удалите на схеме соединений функции топологии все выводы устройства, которые не относятся к схеме соединений изделия, и нажмите ++OK++.
10. Перенесите обозначения выводов устройства на локальную схему соединений функции топологии, установив флажок Локальная схема соединений.

!!! info "Для сведения:"

    В столбце 'Статус' обозначения выводов устройства будут помечены пиктограммой ![](../images/all_statusok_as.png){: .ui-icon }.

11. Закройте диалоговое окно "Свойства", нажав ++OK++, и повторно запустите контрольный прогон.

!!! note "Замечание:"

    Если схема соединений в базе данных изделий отсутствует, ее нужно будет создать позднее. Выполните для этого следующие действия:

1. Выделите уровень иерархии Схема соединений в дереве базы данных изделий и выберите пункт всплывающего меню Создать.
2. Определите необходимые данные выводов на вкладках Схема соединений, Выводы устройства и / или Атрибуты, введите имя схемы соединений и щелкните по кнопке ++Применить++.
3. Выделите в дереве изделие и присвойте ему схему соединений, как описано выше.

    ![](../images/arrow.png){: .ui-icon } Присвоение выполняется также для размещения изделия в пространстве листа.

#### Возможность 2

При маршрутизации должна использоваться схема соединений, записанная локально на функции топологии. При этом рекомендуется действовать следующим образом.

1. Выведите на передний план вкладку Схема соединений.

!!! info "Для сведения:"

    Если функция топологии уже имеет выводы устройства, они перечисляются в таблице и помечаются пиктограммой ![](../images/erroricon_error_systemmessages.png){: .ui-icon }.

2. Перенесите обозначения выводов устройства на локальную схему соединений функции топологии, установив флажок Локальная схема соединений.

!!! info "Для сведения:"

    В столбце 'Статус' обозначения выводов устройства будут помечены пиктограммой ![](../images/all_statusok_as.png){: .ui-icon }.

3. Закройте диалоговое окно 'Свойства', нажав ++OK++, и повторно запустите контрольный прогон.

!!! note "Замечание:"

    У вас есть возможность использовать на схеме соединений в том числе и свободно определенные обозначения выводов устройства. Выполните для этого следующие действия:

1. Добавьте в таблицу новые строки в соответстви с количеством выводов устройства.
2. Укажите нужные обозначения выводов устройства.
3. Удалите все ненужные обозначения выводов устройства и скопируйте данные в локальную схему соединений, установив соответствующий флажок.
