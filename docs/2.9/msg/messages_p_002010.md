# P002010: Неверное образование мостов. перемычки

### Причина

#### Возможность 1

Два контакта штекера соединены перемычкой с помощью углов. Кроме того, соединение, возникшее между контактами штекера, имеет точку определения соединения, свойство которого Определение функции настроено на "мостовую перемычку". Однако в случае свойств для логической схемы контактов штекера у всех выводов устройства функций, свойство Число мостовых перемычек имеет значение "0". Отсюда результирует противоречие с точки зрения образования мостовой перемычки.

#### Возможность 2

Два контакта штекера соединены перемычкой без использования углов. Перемычка создается с помощью свойства Мостовая перемычка, при этом для первого контакта штекера выбирается настройка "Начало перемычки", а для второго контакта штекера - настройка "Конец перемычки". Однако в случае свойств для логической схемы контактов штекера у всех выводов устройства функций, свойство Число мостовых перемычек имеет значение "0". Отсюда результирует противоречие с точки зрения образования мостовой перемычки.

### Решение

#### Для возможности 1

Увеличьте значение для числа мостовых перемычек на обоих контактах штекера. Щелкните дважды в схеме соединений на соответствующий контакт штекера и выберите в диалоговом окне Свойства ++...++ вкладку Данные символа / функции. Щелкните в групповом поле Данные функции по кнопке ++Логика++. Увеличьте значение свойства для одного или обоих выводов устройства функции Число мостовых перемычек минимум на "1".

Или измените определение функции на точке определения соединения. (Например, выберите настройку "Соединение, общее")

#### Для возможности 2

Увеличьте значение для числа мостовых перемычек на обоих контактах штекера (см. возможность 1).

Обновите соединения и затем повторно запустите контрольный прогон.
