# P022017: В проекте имеются превышения длины поля в базе данных: \<x\>

### Причина

Управление многочисленными идентификаторами, номерами и обозначениями в проектах EPLAN, например обозначениями устройств, осуществляется в различных базах данных. В отдельных случаях ввод очень длинных последовательностей букв и / или цифр в одной из баз данных может привести к усечению информации и, как следствие, к неверным результатам.

### Решение

Реорганизуйте проект.

1. Выберите пункты меню Проект > Организовать > Реорганизовать.
2. Установите в диалоговом окне Реорганизовать проект флажок Расширенный режим, нажмите ++Да++, и дождитесь завершения процесса реорганизации.
3. После этого запустите новый контрольный прогон для ***всего*** проекта.

!!! note "Замечание:"

    Нам часто задают вопросы о стандартной длине поля идентифицирующих элементов обозначения устройства.

    В этой связи EPLAN поддерживает следующее количество знаков:

* 16 знаков для предшествующих цифр
* 16 знаков для буквенного обозначения
* 64 знака для счетчика
* 16 знаков для подсчетчика.

    Поэтому ради наглядности и во избежание ошибок присваивайте как можно более короткие обозначения устройств.
