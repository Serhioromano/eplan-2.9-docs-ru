# P001012: Неверное образование мостов. перемычки

### Причина

На схеме соединений обнаружена ошибочно созданная мостовая перемычка. Это сообщение выдается по ряду причин, которые разъясняются в следующих примерах:

#### Возможность 1

В схеме соединений в качестве мостовой перемычки определено соединение между разными уровнями многоуровневой клеммы.

!!! example "Пример:"

    Между обеими клеммами-X1:1и-X1:2существует соединение, определенное как мостовая перемычка. Обе клеммы расположены на***разных***уровнях***многоуровневой клеммы***. Создание мостовой перемычки в данном случае невозможно.

#### Возможность 2

Вы определили соединение между клеммами в схеме соединений как мостовую перемычку, однако не указали количества мостовых перемычек на одной или обеих клеммах.

!!! example "Пример:"

    Между обеими клеммами-X1:1и-X1:2существует соединение, определенное как мостовая перемычка (см. рисунок в примере в разделе "Возможность 1"). Обе клеммы являются "стандартными" проходными клеммами с двумя возможностями вывода устройств. В диалоговом окне Логическая схема выводовустройствадля первого вывода устройствафункциина обеих клеммах***не***введено значения или введен0длясвойства Число мостовых перемычек.

### Решение

#### Для возможности 1

1. В схеме соединений дважды нажмите на точку определения соединения между клеммами.
2. Выберите в диалоговом окне Свойства ++...++ вкладку Точка определения соединения.
3. Щелкните рядом с полем Определение функции по кнопке ++...++ и выберите в диалоговом окне Определения функций вместо записи "Мостовая перемычка" запись "Проволочная перемычка".
4. Обновите соединения и затем повторно запустите контрольный прогон.

#### Для возможности 2

1. Для связанных друг с другом клемм вызовите диалоговое окно Свойства.
2. Перейдите на вкладку Данные символа/функции и, нажав кнопку ++Логика++, откройте диалоговое окно Логическая схема выводов устройства.
3. В поле Число мостовых перемычек для соответствующих выводов устройства функции введите значение, которое больше "0" (например, в разделе "Возможность 2" на первом выводе устройства функции указана 1).
4. Затем запустите новый контрольный прогон.
