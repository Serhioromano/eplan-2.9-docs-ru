# P001033: Невозможно генерировать созданную вручную мостовую перемычку. \<x\>

### Причины

Если не удается генерировать созданную вручную мостовую перемычку, следует рассмотреть следующие возможные причины, отображенные в заполнителе \<x\>:

#### 1\. Не удалось выполнить анализ информации о мостовой перемычке \<y\>.

Определенное на клемме свойство Вручную созданные мостовые перемычки (внутр.) или Вручную созданные мостовые перемычки (внешн.) не содержит действительных значений \<y\>. Возможно, был введен или добавлен неиспользуемый символ.

#### 2\. На конце перемычки отсутствует клемма.

Определенное на клемме свойство Вручную созданные мостовые перемычки (внутр.) или Вручную созданные мостовые перемычки (внешн.) содержит значение, которое приводит к слишком длинной мостовой перемычке. Поэтому EPLAN не находит клемму на конце перемычки. Возможно, значение для начала перемычки и величина шага были расширены вручную без предоставления соответствующей клеммы на конце мостовой перемычки. Также могла быть удалена задействованная в существующей мостовой перемычке последняя клемма.

#### 3\. Клемма \<y\> не снабжена внутренним выводом мостовой перемычки.

На клемме определено свойство Вручную созданные мостовые перемычки (внутр.). Согласно этому свойству указанная клемма должна быть соединена перемычкой. Однако создание мостовой перемычки невозможно, так как клемма не снабжена внутренним выводом мостовой перемычки. Это происходит, например, когда при существующей созданной вручную внутренней мостовой перемычке определение функции на задействованной клемме меняется таким образом, что клемма остается без внутреннего вывода мостовой перемычки.

#### 4\. Клемма \<y\> не снабжена внешним выводом мостовой перемычки.

На клемме определено свойство Вручную созданные мостовые перемычки (внешн.). Согласно этому свойству указанная клемма должна быть соединена перемычкой. Однако создание мостовой перемычки невозможно, так как клемма не снабжена внешним выводом мостовой перемычки. Это происходит, например, когда при существующей созданной вручную внешней мостовой перемычке определение функции на задействованной клемме меняется таким образом, что клемма остается без внешнего вывода мостовой перемычки.

### Решение

Сначала для соответствующей причины установите местоположение данной клеммы на схеме соединений, пользуясь функциональностью Перейти к (графика), которую Вы можете найти во всплывающем меню диалогового окна Управление сообщениями.

Все последующие действия, касающиеся различных опций сообщений, описаны ниже.

#### К п. 1.: Не удалось выполнить анализ информации о мостовой перемычке \<y\>.

Условие:

Вы определили местоположение соответствующей клеммы на схеме соединений и отметили его.

1. Выберите пункт всплывающего меню Свойства.
2. В диалоговом окне свойств клеммы вынесите на передний план первую вкладку.
3. Удалите свойство Вручную созданные мостовые перемычки (внутр.) или Вручную созданные мостовые перемычки (внешн.) из таблицы свойств.
4. Щелкните по кнопке ++OK++.
5. Выберите пункт меню Данные проекта > Клеммники > Обработать.
6. Выделите в диалоговом окне Обработать клеммник клеммы, которые вы хотите вручную соединить перемычками.

7. Щелкните на панели инструментов по кнопке ![](../images/terminalgui_createsaddlejumpint_as.png){: .ui-icon } (Генерировать созданную вручную мостовую перемычку (внутр.)) или по кнопке ![](../images/terminalgui_createsaddlejumpext_as.png){: .ui-icon } (Генерировать созданную вручную мостовую перемычку (внешн.)).

!!! info "Для сведения:"

    Требуемая мостовая перемычка будет сгенерирована.

8. Щелкните по кнопке ++OK++.

!!! info "Для сведения:"

    На клемме свойство Вручную созданные мостовые перемычки (внутр.) или Вручную созданные мостовые перемычки (внешн.) будет заново определено и заполнено правильной информацией.

9. Затем запустите новый контрольный прогон.

#### К п. 2.: На конце перемычки отсутствует клемма.

Выполните действия, описанные в разделе ***К п. 1.: Не удалось выполнить анализ информации о мостовой перемычке \<y\>***.

#### К п. 3.: Клемма \<y\> не снабжена внутренним выводом мостовой перемычки.

Условие:

Вы определили местоположение соответствующей клеммы на схеме соединений и отметили его.

1. Выберите пункт всплывающего меню Свойства.
2. В диалоговом окне свойств клеммы вынесите на передний план вкладку Данные символа / функции.
3. Щелкните в групповом поле Данные функции (логика) рядом с полем Определение на ++...++.
4. В открывшемся диалоговом окне выберите в качестве определения функции клемму, снабженную выводом мостовой перемычки.
5. Щелкните по кнопке ++OK++.
6. Щелкните в диалоговом окне свойств по кнопке ++Логика++.
7. Присвойте одному из выводов мостовой перемычки в свойстве Внутр. / Внешн. значение "Внутр.".
8. Примените изменения, нажав кнопку ++OK++.
9. Запустите новый контрольный прогон.

#### К п. 4.: Клемма \<y\> не снабжена внешним выводом мостовой перемычки.

Выполните действия, описанные в разделе ***К п. 3.: Клемма \<y\> не снабжена внутренним выводом мостовой перемычки***. Но одному из выводов мостовой перемычки в свойстве Внутр. / Внешн. присвойте значение "Внешн.".
