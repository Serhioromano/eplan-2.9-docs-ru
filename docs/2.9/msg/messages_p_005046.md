# P005046: Неполная сеть

### Причина

#### Возможность 1

Не все выводы устройства сети соединены друг с другом.

!!! example "Пример:"

    СетьN1в схеме соединений, на первый взгляд, соединяет 5 резервирований (начиная с-F1и до-F5) с потенциаломL1.Однако из свойств сети следует, что выводустройства-F5:1в шлейфе ни с чем не соединен.Шлейф Вывод устройства Источник ЦельL1L1-F1:1-F1:1-F1:1-F2:1-F2:1-F2:1-F3:1-F3:1-F3:1-F4:1-F4:1

#### Возможность 2

Два шлейфа одной сети не соединены друг с другом.

!!! example "Пример:"

    СетьN1в схеме соединений, на первый взгляд, соединяет 5 резервирований (начиная с-F1и до-F5) с потенциаломL1.Однако из свойств сети следует, что выводы устройства имеются в двух различных шлейфах:шлейф 1Вывод устройства Источник Цель-F4:1-F4:1-F3:1-F3:1-F3:1-F2:1-F2:1-F2:1-F1:1-F1:1шлейф 2Вывод устройства Источник ЦельL1L1-F5:1-F5:1Если бы точкисоединениясети были заменены символами соединения с указанием цели, мы имели бы следующую картину:

#### Возможность 3

На схеме соединений было удалено соединение в одной из сетей. Из-за этого между шлейфами возникает зазор. Кроме того, в этом случае появляется сообщение [P005047: Точка определения сети содержит неполные соединения](messages_p_005047.md).

#### Возможность 4

В процессе копирования на схеме соединений в одной из сетей было размещено дополнительное соединение, которое еще не присвоено ни одному шлейфу. Кроме того, в этом случае появляется сообщение [P005050: Двойной идентификатор для сетевых соединений в пределах сети (\<x\>)](messages_p_005050.md).

### Решение

#### Для возможности 1

!!! example "Пример:"

    В соответсвии с примером из "Возможности 1" действуйте следующим образом:Установите местоположение соответствующейточки определения сетина схеме соединений, пользуясь функцией Перейти к (графика), которую можно найти во всплывающем меню диалогового окна Управление сообщениями.Вызовите диалоговое окно Свойства ++...++ точки определения сетиN1.Выведите на передний план вкладку Соединения сети.В таблице Выводы устройствадважды щелкните по выводу устройства-F5:1.Вывод устройства-F5:1будет скопирован в таблицу Соединенияи записан под горизонтальной линией первого шлейфа. Тем самым начинается новый шлейф.Выделите вывод устройства-F5:1под горизонтальной линией и переместите его, щелкнув по кнопке(Переместить выше) в уже закрытый к этому моменту шлейф.Закройте диалоговое окно Свойства ++...++ точки определения сети, нажав ++OK++, и повторно запустите контрольный прогон.

#### Для возможности 2

Соедините оба разделенные шлейфа.

!!! example "Пример:"

    В соответсвии с примером из "Возможности 2" действуйте следующим образом:Установите местоположение соответствующей точки определения сети на схеме соединений, пользуясь функцией Перейти к (графика), которую можно найти во всплывающем меню диалогового окна Управление сообщениями.Вызовите диалоговое окно Свойства ++...++ точки определения сетиN1.Выведите на передний план вкладку Соединения сети.В таблице Выводы устройствадважды щелкните по выводу устройства-F4:1.Вывод устройства-F4:1будет скопирован в таблицу Соединенияи записан под горизонтальной линией второго шлейфа. Генерируется новый шлейф.Щелкните по(Завершить шлейф), чтобы закрыть шлейф.Появится горизонтальная линия под выводом устройства-F4:1в таблице Соединения.Выделите вывод устройства-F4:1в третьем шлейфе и переместите его, щелкнув по кнопке(Переместить выше), во второй шлейф.Третий шлейф при этом будет расформирован и вывод устройства-F5:1соединен с выводом устройства-F4:1.Закройте диалоговое окно Свойства ++...++ точки определения сети, нажав ++OK++, и повторно запустите контрольный прогон.

#### Для возможности 3

Подробное описание вы найдете в справке, в разделе [P005047: Точка определения сети содержит неполные соединения](messages_p_005047.md).

#### Для возможности 4

Подробное описание вы найдете в справке, в разделе [P005050: Двойной идентификатор для сетевых соединений в пределах сети (\<x\>)](messages_p_005050.md).

* * *

!!! note "Замечание:"

    EPLAN предоставляет возможность автоматического соединения родственных распределенных клемм (пункты меню Параметры > Настройки > Проекты > "Имя проекта" > Устройство > Клеммы > Настройки (распределенные клеммы)) даже в случае отсутствия обозначенных соединений между распределенными клеммами. Эта функция действует также в случае, когда точка определения сети определяет соединения. Чтобы при проектировании с распределенными клеммами избежать появления сообщения "P005046: Неполная сеть", как минимум один из многократно встречающихся выводов распределенных клемм должен быть соединен с сетью.
