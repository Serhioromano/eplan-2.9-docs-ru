# Области обработки: Принцип

Разделение на области обработки осуществляется при помощи схем фильтров. В качестве критериев фильтра для структурных идентификаторов могут служить все свойства. Любые комбинации имеющихся в проекте структурных идентификаторов могут использоваться как критерии фильтра. Дополнительно можно использовать свойство Раздел (области обработки), чтобы фильтровать страницы, пространства листа, функции и соединения по определенным разделам.

Несколько критериев фильтров можно связать при помощи логических операторов. Тем самым можно выполнять фильтрацию одновременно по определенным структурным идентификаторам и по определенному разделу.

Для значений критериев фильтра можно использовать знаки-заполнители (такие как '*' и '?'). Это дает возможность, например, объединить в одном выражении фильтра структурные идентификаторы с глубже подключенными подструктурами.

### Открытие проектов с активированным выбором областей обработки

Если в проекте активирован выбор областей обработки, то при открытии проекта сразу появится диалоговое окно Определить области обработки.

Если над одним проектом по сети работают несколько пользователей, то в столбце Все обработчики отображаются области обработки, определенные другим пользователем в этом проекте. Открыл ли другой пользователь проект, вы узнать не сможете.

Если программа запускается заново и один из автоматически открываемых при этом проектов разделен на области обработки, выводится соответствующее сообщение.

### Внесение изменений в проект в целом

В платформе EPLAN есть операции обработки (например, нумерация устройств), которые вы можете выполнять не только для актуального выбора, но и распространить на проект в целом. В соответствующих диалоговых окнах для этого необходимо установить флажок Применить ко всему проекту.

Если в каком-то проекте вы работаете с определенными областями обработки, то, несмотря на активную настройку, такие операции обработки действуют не для всего проекта, а только для вашей области обработки.

Если вы все же хотите выполнить обработку всего проекта, необходимо временно распространить вашу область обработки на весь проект (например, выбрав пункты меню Проект > Области обработки > Весь проект). После этого в навигаторах данных проекта будет снова отображаться проект в целом. После завершения операции завершите временную обработку в проекте в целом, для чего еще раз выберите данный пункт меню.

!!! tip "Совет:"

    С помощью всплывающего меню Перейти к (графика) можно перейти к объекту (странице / пространству листа), который не принадлежит вашей области обработки. Таким способом можно, например, со страницы отчета перейти на соответствующую страницу схемы соединений и выполнить здесь соответствующие изменения, даже если страница схемы соединений не относится к вашей области обработки.

#### Обновление соединений

Выполненное в ручном или автоматическом режиме обновление соединений влияет только на актуальную область обработки. При этом соединение относится к области обработки, если цель данного соединения относится к данной области обработки.

Обратите внимание на то, что неактуальные соединения могут привести к неверным отчетам (например, в схемах подключений). Для того чтобы выполнить обновление соединений для проекта в целом, необходимо сначала временно расширить область обработки на проект в целом.

### Отчет по областям обработки

Для анализа областей обработки доступен пункт меню Сервисные программы > Отчеты > Генерация отчета по проекту. С помощью данного пункта меню выполняется полный или частичный анализ всех отчетов в вашей области обработки.

Все страницы отчетов, находящиеся вне собственной области обработки, невозможно удалить, создать или обновить. Это относится также к обзорам модели.

Блоки отчетов обновляются полностью либо вообще не обновляются. Чтобы блок отчетов обновился, его начальная страница должна лежать в вашей области обработки.

Если вы хотите проанализировать проект в целом, сначала вам необходимо временно расширить свою область обработки на проект в целом.

Мы рекомендуем выводить отчеты по вашей области обработки в отдельный проект отчета. Отчеты проекта в целом можно выводить в следующий проект отчета, либо они могут содержаться в самом проекте и создаваться и обновляться, например, ночью.

### Отображение областей обработки на многопользовательском мониторе

При работе в многопользовательском режиме с помощью многопользовательского монитора можно узнать, какие области обработки в ***открытом ***проекте выбрал другой пользователь.

В таблице Мои проекты, открытые в многопользовательском режиме в столбце Область отображаются структурные идентификаторы, определенные другим пользователем в качестве области обработки в данном проекте. Если другой пользователь работает в общем проекте временно, то в столбце Область появится запись 'без ограничений'.
