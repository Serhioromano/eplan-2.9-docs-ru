# Диалоговое окно Поиск

Вы открыли проект. Поиск > Поиск.

В этом диалоговом окне введите искомое понятие и определите, в каких данных проекта будет вестись его поиск.

Обзор основных элементов диалогового окна:

### Фильтр для функций

С помощью фильтра для ***функций*** поиск по функциям можно ограничить определенным видом представления или определением функции.

В этом раскрывающемся списке отображаются все доступные фильтры. Выбранный фильтр активируется автоматически. Запись "- Не активировано -" отключает фильтр и приводит к тому, что данные отображаются в неотфильтрованном виде. С помощью кнопки ++...++ откройте диалоговое окно [Фильтр](modaldialogsdb_d_filternnach.md). Здесь можно создать, обработать, удалить, скопировать, экспортировать, импортировать фильтр и управлять им.

Во всплывающем меню раскрывающегося списка Фильтр содержатся следующие записи:

* Выключить: Этот пункт меню доступен, если фильтр установлен: Сбрасывает настройку фильтра до записи "- Не активировано -".
* Активировать <Имя фильтра>: Этот пункт меню доступен, если для настройки фильтра установлено значение "- Не активировано -": Повторно активирует последний активный фильтр.

Таким образом можно быстро переключаться между неотфильтрованным и отфильтрованным в соответствии с требованиями пользователя представлениями.

### Поиск

В этом поле вводите поисковый признак. Использование [заполнителей](searchandreplacegui_k_platzhalter.md) (напр., * или ?) возможно.

С помощью пункта всплывающего меню Специальный символ также можно найти требуемые специальные символы.

### Учитывать регистр

Если этот флажок установлен, находятся только те тексты, регистр которых совпадает с регистром поискового признака. Если вы ввели, например, поисковый признак "Двигатель", находятся тексты "Двигатель" и "Автомат защиты двигателя", но не "Двигатель постоянного тока".

### Искать только весь текст

Если этот флажок установлен, используется только полный поисковый признак. Если вы ввели, например, поисковый признак "Двигатель", находится только текст "Двигатель", но не "Автомат защиты двигателя" или "Двигатель постоянного тока".

* * *

Групповое поле "Поиск в"

В групповом поле Поиск в определите, в каких данных проекта программа должна производить поиск искомого понятия.

### ОУ / обозначение

Если этот флажок установлен, искомое понятие ищется в следующих объектах:

* ***Устройства общего назначения*** (ОУ на общих функциях, черных ящиках и блоках ПЛК как в схеме соединений, так и на страницах обзора)
* ***Клеммы*** ('ОУ: обозначение клеммы', напр. -X1:12, на клеммах и текстах определения клеммы)
* ***Штекеры*** ('ОУ: номер штыря / гнезда', напр. -X1:12, на штекерах и текстах определения штекера)
* ***Выводы устройства*** ('ОУ: номер вывода устройства', напр. -A1:12, на выводах устройств, потенциалов и ПЛК на страницах схемы соединений и обзора)
* ***Точки разрыва*** (обозначение точки разрыва, напр. -L1)
* ***Кабели*** (ОУ кабеля, напр. -W1)
* ***Соединения*** (обозначения соединений / обозначение жил, например 0012. Поиск по неразмещенным соединениям не ведется, так как поиск ведется только по точкам обозначения соединений.)
* ***Потенциалы*** (обозначение потенциалов)
* ***Адреса ПЛК*** (например, A2.0).

В качестве результата поиска на просмотр выводится "видимое ОУ". С помощью команды Заменить можно изменять только видимые ОУ и только его элементы без структурных элементов. Если ОУ не выводится на просмотр на условном обозначении, то есть существует [Перенос ОУ](devicetaggui_k_bmkuebernahme.md), ничего не заменяется.

!!! note "Замечание:"

    * Поиск всегда ведется только во всем ОУ, а не в структурных элементах. При переносе ОУ видимое ОУ является пустым; поэтому в данном случае поиск ОУ не даст результата.
    * Только если в качестве поискового признака вводится ОУ вместе со своей структурой, структура учитывается! При этом в направлении поиска от ОУ должна указываться всегда полная подструктура структуры проекта. Кроме того, нужно указывать и предшествующие знаки (=, + и т. п.). В элементах структуры заполнители не разрешены.
    * Быстрее всего поиск выполняется при вводе полного ОУ без заполнителей и при настройке Искать только весь текст. При этом флажок Все свойства всех условных обозначений необходимо ***снять***. В этом случае система ищет по индексу базы данных.

### Все свойства всех условных обозначений

Если установлен этот флажок, поиск искомого понятия выполняется во всех свойствах условных обозначений за исключением ОУ и текстов. (Поиск в ОУ и текстах включается отдельно.) В данном случае учитываются объекты-заполнители и рамки макроса, причем в рамках макроса выполняется поиск по данным "имя", "версия", "источник / ссылка" и "описание", а в объектах-заполнителях - по данным "имя".

При этом поиск следующих свойств ***не*** выполняется:

* ОУ, включая все элементы, такие как, установка, место установки, предшествующая цифра, счетчик и т. д.
* Обозначение клеммы
* Номер штыря штекера / гнезда
* Обозначение вывода устройства
* Описание вывода устройства
* Обозначение точки разрыва
* ОУ кабеля
* Обозначение соединения
* Обозначение потенциала.

### Тексты

Если этот флажок установлен, искомое понятие ищется в следующих текстах:

* Функциональные тексты зон
* Общие (простые) тексты
* Специальные тексты на страницах вывода.

Поиск ведется на всех языках. По каждому языку, на котором велся поиск, имеется один результат поиска.

### Объекты-заполнители

Если этот флажок установлен, поиск осуществляется среди объектов-заполнителей (как среди переменных, так и среди наборов значений).

Если заменить имя набора значений, одновременно будет заменена и запись в свойстве Объект-заполнитель: Последний выбранный набор значений.

### Предварительное планирование

Если установлен этот флажок, то поиск искомого понятия будет выполняться в свойствах сегментов структуры и объектов планирования.

### Свойства страницы

Если этот флажок установлен, искомое понятие ищется в свойствах страницы (напр., описание страницы, номер чертежа и т. д.). При этом учитывается свойство страниц Полное имя страницы.

### Свойства проекта

Если этот флажок установлен, искомое понятие ищется в свойствах проекта (напр., имя проекта, место монтажа и т. д.).

* * *

Групповое поле "Поиск на"

В этом групповом поле указываются типы страниц, на которых необходимо вести поиск поискового признака. Имейте в виду, что найти поисковой признак можно только в том случае, если как минимум один из выбранных здесь типов страниц совпадает с типом страницы выделенных страниц. Если, например, выделено несколько ***страниц отчетов*** и выделены только флажки Логич. страницы и Граф. страницы, поиск может не дать результатов.

### Неразмещенные устройства

Установите этот флажок, чтобы искать искомое понятие в неразмещенных функциях.
Если флажок не установлен, неразмещенные устройства при поиске не учитываются.

### Пространство листа

Установите этот флажок, чтобы искать искомое понятие в пространстве листа.
Если флажок не установлен, пространство листа при поиске не учитывается.

### Скрытые опции

Установите этот флажок, чтобы искать искомое понятие в скрытых опциях проекта.
Если флажок не установлен, скрытые опции проекта при поиске не учитываются.
