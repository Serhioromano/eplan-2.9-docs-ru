# Восстановить проекты

Проекты можно восстанавливать из Навигатора страниц и из окна управления проектами.

По умолчанию все основные данные считываются в связи с проектом, после чего сравниваются с пулом основных данных. При выявлении различий следует определить используемые данные.

1. Из Навигатора страниц выберите пункты меню Проект > Восстановить > Проект, или из окна Управление проектами выберите ++Организовать++ > Восстановить.
2. В диалоговом окне Восстановить проект в поле Каталог для резервирования с помощью кнопки ++...++ выберите каталог, в котором располагаются резервированные проекты.

!!! info "Для сведения:"

    В списке Проекты отображаются резервированные проекты, содержащиеся в выбранном каталоге.

!!! info "Для сведения:"

    Если в каталоге содержатся разделенные файлы резервирования, отображается только файл *.000.zw1, остальные файлы резервирования, относящиеся к этому же проекту (*.001.zw1, *.002.zw1 и т. п.), ***не*** отображаются.

3. Выделите проект для восстановления. Возможен многократный выбор.
4. В поле Целевой каталог с помощью кнопки ++...++ выберите целевой каталог, в который должно быть скопировано резервирование данных.
5. Чтобы не перезаписывать уже существующий проект, измените имя, отображаемое в поле Имя проекта при выборе единичного проекта.
6. Подтвердите ввод.

!!! info "Для сведения:"

    Данные выгружаемого файла будут скопированы в указанный каталог проектов. При выделении сразу нескольких проектов они будут восстановлены по очереди.

При восстановлении EPLAN проверяет, существует ли конкретный проект и можно ли его перезаписать.

При этом следует обратить внимание на следующие различия:

* Проект есть, но не открыт в данный момент. В данном случае при восстановлении появляется сообщение и Вы можете решить, нужно ли перезаписать проект.
* Проект есть, и его открыл сам текущий пользователь. В данном случае при восстановлении появляется сообщение и Вы можете решить, нужно ли перезаписать проект. Если Вы подтвердите восстановление, то проект, открытый в данный момент, закроется, будет перезаписан "старыми" данными и открыт снова. Обратите внимание, что при перезаписи текущие изменения могут быть утеряны.
* Проект есть, и его открыл другой пользователь. В данном случае при восстановлении появится сообщение, а процесс будет прерван.

!!! note "Замечание:"

    * Сообщения, возникающие при резервировании проекта, сохраняются как системные сообщения; просмотреть их можно через Сервисные программы > Системные сообщения.
    * При сохранении проектов производится проверка на изменения настроек проекта. При выявлении различий на экране отображаются сравнительные сообщения, которые сохраняются в Управлении сообщениями (Данные проекта > Сообщения> Управление).
