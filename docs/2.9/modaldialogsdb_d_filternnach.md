# Диалоговое окно Фильтр

Открыто диалоговое окно, для которого можно задать фильтр. Щелкните по ++...++ рядом с раскрывающимся списком Фильтр.

В этом диалоговом окне определите критерии фильтра для фильтра ***по полю***. Он ведет поиск только на определенных полях записей данных (напр., для поля "Номер типа"), при этом проверяется, совпадает ли все содержимое поля с искомым текстом. При этом критерии фильтра могут сохраняться в виде схем, так что ими можно в любой момент воспользоваться снова.

Обзор основных элементов диалогового окна:

### Схема

Выберите из раскрывающегося списка схему, которую требуется использовать или обработать. Для этого также воспользуйтесь соседней [панелью инструментов](schemeconfig_m_schemaschaltflaechen.md).

* * *

Таблица

В таблице определяются критерии фильтра для свойств объекта. Отдельные строки стандартно связаны логическим оператором И.

### Критерий

В этом поле выбирается свойство, для которого требуется определить критерий фильтра. С помощью кнопки ++...++ откройте диалоговое окно выбора, в котором свойства расположены в дереве.

С помощью записи ИЛИ создайте строку для операции ИЛИ.

### Оператор

В этом столбце выберите оператор сравнения, чтобы соединить свойство со значением.

!!! example "Пример:"

    Чтобы выбрать из списка все неразмещенныефункции, критерий Главная функцияс оператором "=" и значениемследует связать с критерием Размещениес оператором "=" и пустой записью в поле Значение.Активн.Отрицат.Критерий Оператор Значение Главная функция=Размещение=

!!! note "Замечание:"

    * Обратите внимание на тот факт, что операторы в текстовых полях, например, в поле "Напряжение", будут обработаны неверно, так как в текстовых полях сравниваются не значения, а тексты.
    * Последовательность введенных критериев, как правило, не имеет значения; однако некоторая беспорядочность при вводе может вызывать ошибки (например, пропущена закрывающая скобка) и таким образом приводить к неверным результатам фильтрации.

### Значение

Здесь занесите значение для сравнения, с которым должно сравниваться значение выбранного свойства. Возможно использование [заполнителей](searchandreplacegui_k_platzhalter.md) типа '*' и '?'. Ввод нескольких значений на одной строке (с разделением точкой с запятой) расценивается как операция ИЛИ.

В случае с полями, которые представляют каталоги, можно вручную ввести замещающий символ "*"; в этом случае в выбор войдут все подкаталоги.

### Язык

Выберите из этого раскрывающегося списка язык поиска. Предварительная настройка — "Все языки". При выборе определенного языка поиск выполняется только на нем; при вводе без настройки языка поиск выполняется на всех языках.

### Быстрый ввод

Это поле появится, если из навигатора данных проекта вызвать диалоговое окно Фильтр.

Если этот флажок установлен, поле Значение: <Свойство> деблокируется для навигатора, как только будет выбрана соответствующая схема фильтра.

Чтобы ввести в поле Значение: <Свойство> новое значение, щелкните по полю, введите значение и нажмите ++Ввод++. Представление в навигаторе будет соответствующим образом скорректировано. В схему фильтра указанное здесь значение будет скопировано в качестве значения критерия фильтра.
