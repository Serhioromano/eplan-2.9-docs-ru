# Диалоговое окно Настройки: Изделие (пользователь)

* Вы открыли базу данных изделий. Сервисные программы > Изделие > Управление > [Дополнительно] > Настройки.
* Вы открыли проект и выделили главную функцию в графическом редакторе или навигаторе. Обработать > Свойства. Выберите вкладку Изделие и щелкните рядом с полем Источн. данных по ++...++.
* Параметры > Настройки > Пользователь > Управление > Изделия.

В этом диалоговом окне устанавливаются определенные пользователем настройки для работы с ***выбором изделия и базой данных изделий***.

Обзор основных элементов диалогового окна:

### Выбор изделия с выбором схемы

Если этот флажок установлен, то при каждом выборе изделия открывается новое диалоговое окно выбора, в котором из раскрывающегося списка выбирается схема и посредством [...] можно просматривать / обрабатывать настройки. Это позволяет, например, заполнять список устройств данными из разных источников данных.
Если флажок не установлен (настройка по умолчанию), выбора не происходит.

### Схема

Выберите из раскрывающегося списка схему, которую требуется использовать или обработать. Для этого также воспользуйтесь соседней [панелью инструментов](schemeconfig_m_schemaschaltflaechen.md).

### Access

Выберите этот параметр, чтобы выбрать файл базы данных *.mdb с помощью кнопки ++...++ в интерактивном режиме.

С помощью пункта всплывающего меню Вставить переменную пути перейдите в диалоговое окно [Выбрать переменную пути](modaldialogsdb_d_pfadvariablen.md), из которого можно будет скопировать одну из доступных переменных пути.

Или щелкните по кнопке ![](images/all_new_as.png){: .ui-icon } (Создать), чтобы в следующем диалоговом окне сгенерировать новую базу данных.

Текущая база данных затем показывается в строке заголовка диалогового окна База данных изделий.

### Сервер SQL

Выберите этот параметр и щелкните мышкой по кнопке ++...++, чтобы в диалоговом окне [Сервер SQL — Настройка](modaldialogs_d_sqlserver.md) определить настройки для использования сервера SQL.
Или щелкните по кнопке ![](images/all_new_as.png){: .ui-icon } (Создать), чтобы в следующем диалоговом окне сгенерировать новую базу данных.
Или выберите требуемую базу данных из раскрывающегося списка 15 последних использованных баз данных SLQ, при этом последние использованные базы находятся в верхней позиции. При открытии / генерации новой базы данных, которая отсутствует в списке, при необходимости из списка удаляется база данных, находящаяся на самой нижней позиции. С помощью пункта всплывающего меню Удалить записи можно полностью очистить список, в том числе базу данных, использующуюся в настоящий момент.

Текущий выбранный сервер затем отображается в строке заголовка диалогового окна База данных изделий.

### Использовать полнотекстовый поиск сервера SQL

Если этот флажок установлен, в базе данных изделий для поля Полнотекст. фильтр используется полнотекстовый поиск, интегрированный на сервере SQL. Индекс поиска генерируется при первом использовании полнотекстового поиска и самостоятельно обновляется сервером SQL при изменении данных изделий. Поскольку обновление проходит на сервере SQL, EPLAN не требуется дополнительная мощность компьютера, что позволяет ускорить поиск.

!!! note "Замечание:"

    * При использовании SQL Server Express необходимо установить и сконфигурировать на сервере SQL модуль "Advanced Services", чтобы полнотекстовый поиск мог выполняться в фоновом режиме.
    * В отличие от "нормального" полнотекстового поиска, здесь находятся только поисковые понятия, стоящие в начале слова. Так, например, через параметр поиска INTER вы найдете изделия с обозначением INTERBUS, а введя BUS – нет.
    * Чтобы искать числа, заканчивающиеся цифрой "0", необходимо заключить число в кавычки, иначе SQL идентифицирует символы не как символьную строку, а как диапазон значений. (Например, при полнотекстовом поиске значение "21.022000" будет идентифицировано как число, а не как символьная строка, поэтому будет выполняться поиск "21.022*", а не "21.022000*".)

Если флажок снят, при работе с сервером SQL используется тот же полнотекстовый поиск, что и для базы данных изделий Access.

### API

С помощью кнопки ++...++ откройте диалоговое окно Настройки: Выбор API изделия. Укажите, какие свойства изделий из внешней базы данных используются, а какие сохраняются в проект.

### Дополнить данные изделия через

Если технические данные изделия, специфические для EPLAN, должны дополняться коммерческими данными из подсоединенных систем, таких как PMS, то добиться этого можно с помощью таблицы из диалогового окна [Дополнить данные изделия](partselectiongui_d_artikeldatenergaenzen.md), открываемого с помощью кнопки ++...++. При выборе схемы выбора изделия вы можете определить не только источник данных (поле Схема данного диалогового окна), но и выбрать дополнительные источники данных, которые также необходимо определить как схему.

### Конфигурация дерева

Выберите из этого раскрывающегося списка схему, которая должна использоваться для представления в виде дерева в базе данных изделий и в выборе изделия. Или щелкните по кнопке ++...++, чтобы в диалоговом окне Конфигурация дерева обработать схему или создать новую.
Если необходимы различные представления структуры дерева в выборе изделия и в базе данных изделий (например, одно для проектирования и одно для обработки основных данных), вы можете создать для этого в настройках изделия несколько схем с различной конфигурацией дерева и через выбор схемы быстро переходить из одного представления структуры дерева в другое.

### Развернуть узел

При установке этого флажка разворачиваются узлы, и устройству присваиваются номера изделия отдельных составных частей узла.
При снятии флажка устройству присваивается номер изделия узла.

По умолчанию флажок снят. Потом при создании отчетов можно решить, разворачиваются ли узлы и, напр., должны ли появиться в спецификации номера изделия и заказа узла или отдельного изделия.

### До степени

Если установлен флажок Развернуть узел, укажите до какой глубины вложения должны разворачиваться узлы.

### Разрешено изменение при выборе

Если этот флажок установлен, то при выборе изделия на вкладке Изделие диалогового окна свойств условных обозначений можно изменять изделия, имеющиеся в базе данных изделий, и создавать новые изделия. В случае с устаревшей базой данных также поступит запрос на обновление при выборе изделий.

Если флажок не установлен, то изделия можно только выбирать и невозможно создавать или обрабатывать.

### Объект-заполнитель / Рамка макроса: Копировать номер изделия без синхронизации данных изделия

Если этот флажок установлен, то при присвоении набора значений с помощью объекта-заполнителя / рамки макроса диалоговое окно Конфликт не отображается и данные из базы данных изделий не копируются.

Если флажок снят, то диалоговое окно Конфликт отображается и можно задать, какие данные необходимо оставить без изменений, а какие данные необходимо перезаписать новыми.
