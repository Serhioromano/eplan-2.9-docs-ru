# Вкладка Блок ПЛК

Вы открыли проект.

* В графическом редакторе уже выделен блок ПЛК. Обработать > Свойства. Выберите в диалоговом окне Свойства ++...++ вкладку Блок ПЛК.
* Вы выделили блок ПЛК в навигаторе устройств или в одном из навигаторов ПЛК. Пункт всплывающего меню Свойства или Свойства (общие). Выберите в диалоговом окне Свойства ++...++ вкладку Блок ПЛК.

На этой вкладке указываются свойства выделенного блока ПЛК. (У главной функции определяются свойства для всей карты ПЛК.) Помимо свойств, которые имеют все функции, здесь также представлены другие специфические для ПЛК свойства.

В режиме обработки ***Свойства (общие)*** вы изменяете свойства сразу для всех представлений функции, в этом случае во вкладке дополнительно отображается "(Устройство)".

Обзор основных элементов диалогового окна:

### Видимое ОУ

Видимое ОУ будет показано в графическом редакторе рядом с условным обозначением. Если вы определили в настройках проекта, что устройство должно автоматически нумероваться при вставке символов, здесь вам будет автоматически предложено ОУ. Вы можете сохранить или переписать вручную это ОУ.

Кроме того, можно использовать [всплывающее меню или кнопку ++...++](devicetaggui_r_feldsichtbaresbmk.md), чтобы cгенерировать или выбрать ОУ.

### Полное ОУ

В этом поле отображается полное ОУ (т. е. структурный идентификатор текущего окружения + видимое ОУ) видимого ОУ / функционального элемента. С помощью кнопки ++...++ откройте диалоговое окно Полное ОУ. Здесь можно обработать структурный идентификатор и, например, выбрать другое место установки.

### Технические параметры

Введите здесь технические параметры блока ПЛК.

### Функциональный текст

Укажите здесь функциональный текст для функции / устройства.

Независимо от введенного здесь вручную функционального текста существуют другие (определяемые автоматически) функциональные тексты, которые находятся в таблице:

* Функциональный текст (автоматически): Отображает содержание введенного вручную функционального текста или, при его отсутствии, функциональный текст зоны. Если вы изменяете функциональный текст в режиме обработки ***Свойства (общие)***, его содержание переносится обратно в ту функцию, в которой функциональный текст был первоначально задан. При отображении этого свойства на странице проекта разрывы строк удаляются.
* Функциональный текст (общий): отображает (автоматический) функциональный текст функции или, при его отсутствии, (автоматический) функциональный текст соответствующей главной функции.

### Текст гравировки

Введите здесь текст гравировки для блока ПЛК или выберите запись из раскрывающегося списка. В списке отображаются все значения, которые вы вводили в других местах проекта для подобных функций.

### Место монтажа (описыв.)

Введите здесь описываемое место монтажа для блока ПЛК или выберите запись из раскрывающегося списка. В списке отображаются все значения, которые вы вводили в других местах проекта для подобных функций.

### Главная функция

установите этот флажок, если текущая функция является главной.
При использовании имеющегося ОУ, для которого уже задана главная функция, при использовании уже размещенной главной функции, а также при размещении уже размещенной главной функции данное свойство будет автоматически отключено на целевой функции, то есть данный флажок будет снят.

!!! note "Замечание:"

    эта настройка автоматически синхронизируется с одноименным флажком на вкладке Данные символа / функции.

* * *

[Групповое поле "Свойства"](devicetaggui_r_gruppenfeldeigenschaften.md)

Для блока ПЛК доступны, в частности, следующие свойства:

* Описание объекта: Задайте здесь описательный текст, напр. номер типа абонента шины.
* ЦПУ: Имя: Здесь через точку с запятой отображаются имена ЦПУ, введенные на вкладке Данные структуры ПЛК в свойствах от ЦПУ: Имя [1] до ЦПУ: Имя [128]. Отображение выполняется без пробелов, т. е. в списке отсутствуют пустые записи.

* Начальный адрес карты ПЛК: введенное здесь значение задает начальное значение для диапазона адресов карты ПЛК. В то же время адресация входов или выходов этой карты начинается с начального значения, введенного здесь. Кроме того, это значение можно использовать в качестве [значения конфигурации](eservicesgui_d_formatzaehler.md) для переадресации выводов устройства ПЛК. Для карты, имеющей как входы, так и выходы, здесь указывается начальное значение для входов.

!!! example "Пример:"

    Формат SiemensДля программируемых контроллеров в формате Siemens задайте, например, в качестве начального адреса для карты входа наименьший байтовый адрес этой карты ПЛК. Если вы ввели значение "4", при автоматической адресации у первого адреса будет обозначение "E4.0". Идентификатор (здесь: "E") зависит от используемойсхемы.Формат MitsubishiДля программируемых контроллеров в формате Mitsubishi задайте, например, в качестве начального адреса для карты выхода наименьший байтовый адрес этой карты ПЛК. Если вы ввели значение "8", при автоматической адресации у первого адреса будет обозначение "Y8". Идентификатор (здесь: "Y") зависит от используемой схемы.

* Устройство ПЛК: Длина данных (входы): это свойство вместе с начальным адресом используется для определения диапазона адресов карты ПЛК. Длина данных указывает количество битов следующего свободного адреса на следующей карте ПЛК. Введите здесь количество битов для входов карты ПЛК. Для карты цифрового выхода ПЛК или карты ПЛК, которая имеет как вход, так и выход, в свойстве Устройство ПЛК: Длина данных (выходы) введите число битов для выходов.

!!! note "Замечание:"

    * Для карт ПЛК, которые имеют ***только входы*** или ***только выходы***, задайте начальный адрес с помощью свойства Начальный адрес карты ПЛК. То есть, в зависимости от карты ПЛК, это свойство будет действительно либо для входов, либо для выходов. Для карт ПЛК, имеющих ***входы и выходы***, укажите в этом свойстве значение для входов.
    * Для карт ПЛК, имеющих ***как входы, так и выходы***, с помощью свойства Начальный адрес 2 карты ПЛК можно задать отдельный начальный адрес выходов. Чтобы это свойство анализировалось при адресации, флажок Раздельная область адресов для входов и выходов в настройках, относящихся к ПЛК, должен быть ***снят***.
    * Если для карты ПЛК задано свойство Начальный адрес 2 карты ПЛК, при экспорте данных ПЛК значения свойств Начальный адрес карты ПЛК и Начальный адрес 2 карты ПЛК записываются в файл экспорта даже при отсутствии выводов устройства ПЛК для соответствующего диапазона адресов.

!!! note "Замечание:"

    Для обмена данными в формате SIEMENS STEP 7 Classic доступны свойства Диапазон адресов (SIEMENS STEP 7 Classic) и Диапазон адресов 2 (SIEMENS STEP 7 Classic). В этих свойствах введите размер диапазона адресов в блоке управления ПЛК, присваиваемый картой, например "4". В зависимости от используемой карты ПЛК здесь требуется указать биты или байты. С помощью свойства Диапазон адресов 2 (SIEMENS STEP 7 Classic) можно задать отдельный размер для диапазона адресов выходов. Использование свойств для диапазона адресов аналогично свойствам для начальных адресов, и вышеуказанные замечания для начальных адресов также применяются к этим свойствам.
