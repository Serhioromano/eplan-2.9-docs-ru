# Диалоговое окно Конфигурировать определения сегментов — <Имя проекта>

Вы открыли проект. Данные проекта > Предварительное планирование > Конфигурировать определения сегмента.

В данном диалоговом окне можно создавать пользовательские определения сегмента. Эти определения сегментов сохраняются как специфические для проекта и могут использоваться в проекте точно так же, как предварительно определенные определения сегментов. Для удобства ориентации имя проекта также отображается в строке заголовка диалогового окна.

При вставке макросов и при копировании их в другой проект пользовательские определения сегментов вставляются вместе с ними. При отсутствии идентифицирующего имени в целевом проекте определение сегмента будет создано в нем и обозначено как более не используемое.

!!! note "Замечание:"

    Обратите внимание, что конфигурация определений сегмента в многопользовательском режиме невозможна.

Обзор основных элементов диалогового окна:

В левой части диалогового окна в представлении в виде дерева отображаются имеющиеся в текущем проекте определения сегментов. В качестве самого верхнего уровня иерархии отображаются определения базовых сегментов, под ними определения сегментов с их отображаемыми именами. Если вы еще не создали собственные определения сегментов, отображаться будут только заранее заданные. Возможен многократный выбор определений сегментов. Некоторые функции (напр., создание новых определений сегментов или конфигурация свойств сегмента) недоступны при многократном выборе и в таком случае блокируются.

Доступны следующие определения базовых сегментов:

* Сегмент структуры: Здесь можно создавать определения сегментов для сегментов структуры. Предварительно настроено определение сегмента "Общий сегмент структуры".
* Объект планирования: Здесь можно создавать определения сегментов для объектов планирования. Предварительно задано определение сегмента "Общий объект планирования".
* Технологический контур: Здесь можно создавать определения сегментов для технологических контуров. Предварительно заданы определения сегментов "Измерительный контур" и "Исполнительный механизм".
* Функция ТК: Здесь можно создавать определения сегментов для функций ТК. Предварительно настроены определения сегментов "Функция измерения" и "Функция потребителя".
* Резервуар: Здесь можно создавать определения сегментов для резервуаров. Предварительно настроено определение сегмента "Резервуары, общая информация".
* Объект планирования, трубопровод: Здесь можно создавать определения сегментов для объектов планирования (трубопроводов). Предварительно настроено определение сегмента "Объект планирования (трубопровод), общая информация".
* Объект планирования, кабель: Здесь можно создавать определения сегментов для объектов планирования (кабелей). Предварительно настроено определение сегмента "Объект планирования (кабель), общая информация".
* Другие шаблоны: Под этим уровнем иерархии можно создать предварительные определения для шаблонов классов трубы и веществ. Предварительно заданы "Класс трубы, общая информация" и "Вещество, общая информация".

!!! note "Замечание:"

    В сравнении с другими определениями сегментов в случае с предварительными определениями для шаблонов классов трубы и веществ предусмотрены ***только некоторые*** настройки. Кроме настройки Отображать только "Больше не использовать", на вкладке Конфигурация предусмотрены только настройка Отображаемое имя, флажок Больше не использовать, а также выбор определенных пользователем пиктограмм с помощью кнопки ++Пиктограмма++. Кроме того, предварительным определениям для шаблонов классов трубы и веществ на вкладке Свойства сегмента можно присвоить определенные пользователем свойства.

### Отображать только "Больше не использовать"

Если этот флажок установлен, в представлении структуры дерева будут отображаться только сегменты, обозначенные как более не используемые. Для этих сегментов на вкладке Конфигурация установлен флажок Больше не использовать.

Вкладка конфигурация

### Идентифицирующее имя

Здесь отображается уникальное имя, установленное при создании определения сегмента. Под этим именем можно управлять определением сегмента в EPLAN.

### Отображаемое имя

Здесь вводится имя, под которым будет отображаться определение сегмента в проекте. Возможен многоязычный ввод.

!!! example "Пример:"

    Вы создаете под определением базового сегмента "Объект планирования" определение сегмента с идентифицирующим именем Идентификатор Фирмы.Мои Объекты Планирования.Объект Планирования1и отображаемым именем Объект планирования 1. В структуре дерева в качестве верхнего узла отображается имя определения базового сегмента Объект планирования. Под ним в качестве другого узла отображаются определение сегмента под своим отображаемым именем Объект планирования 1. В проекте данное определение сегмента также отображается под именем Объект планирования 1.

### Формат отображения навигатора

С помощью кнопки ++...++ откройте диалоговое окно Формат. Укажите здесь, какие свойства должны отображаться на сегменте в навигаторе предварительного планирования, которому присвоено это определение сегмента. С помощью доступных элементов формата можно отобразить информацию о сегменте, а также косвенные свойства вышестоящих / нижестоящих сегментов. Так можно отобразить, например, имя шаблона сегмента, номер записанного на сегменте изделия, ОУ присвоенной функции и т. д.

### Возможность вставки в

Укажите здесь, под какими другими определениями сегментов может быть вставлено это определение сегмента. С помощью кнопки ++...++ откройте диалоговое окно Выбрать определения сегментов. Установите здесь соответствующие флажки. Если несколько однотипных определений сегментов могут вкладываться друг под другом, необходимо установить также флажок для текущего определения сегмента. Чтобы новое определение сегмента было доступно там для выбора, необходимо прежде сохранить его, нажав на ++Применить++.

### Предшествующий знак

Введите символ, который должен отображаться в свойстве Полное обозначение в качестве обозначения сегмента с данным определением. Если это поле пусто, по умолчанию используется пробел.

### Предшествующий знак также в начале

Если этот флажок установлен, предшествующий знак сегмента с данным определением ***всегда*** отображается в полном обозначении. Т. е. предшествующий знак отображается даже тогда, когда сегмент образует верхний уровень иерархии в структуре дерева, и его обозначение стоит в начале полного обозначения.

Если флажок снят, предшествующий знак бывает скрыт, если соответствующий сегмент стоит в начале полного обозначения. Это настройка по умолчанию.

### Больше не использовать

Если этот флажок установлен, определение сегмента в проекте защищено от записи. При сжатии проекта такие определения сегментов можно удалить, если они не используются в проекте.

!!! note "Замечание:"

    * При вставке и копировании макросов вставляются также сохраненные в макросе пользовательские определения сегментов. При этом в проект могут снова быть вставлены определения сегментов, которые ранее были помечены как более не используемые и удалены. Таким определениям сегментов автоматически присваивается настройка Больше не использовать, так что при повторном сжатии проекта они снова будут удалены. Определения сегментов, используемые в проекте, не удаляются.
    * Этот контрольный прогон позволяет найти сегменты, имеющие определение сегмента, обозначенное как более не используемое.

### [Пиктограмма]

При нажатии этой кнопки открывается диалоговое окно Выбор пиктограммы. В зависимости от выбранного определения базового сегмента там предлагаются на выбор несколько предварительно определенных пиктограмм. В качестве альтернативы можно выбрать в поле Определено пользователем собственную пиктограмму. С помощью расположенной рядом кнопки ++...++ откройте диалоговое окно Выбрать пиктограмму. Перейдите по структуре каталога, чтобы выбрать нужную пиктограмму, которая должна отображаться в навигаторе предварительного планирования на этом сегменте. Вы можете использовать так файлы пиктограмм, так и графические файлы JPEG и BMP.

Вкладка Свойства сегмента

В таблице определению сегмента присваиваются ***определенные пользователем*** свойства, которые должны быть доступны для сегментов с этим определением сегмента. Значения свойств можно задать заранее. Таблица содержит следующие столбцы:

* Строка: Непрерывный номер строки.
* Свойство (отображаемое имя): Здесь отображается имя выбранного свойства. У определенных пользователем, свободных свойств оно является отображаемым именем.
* Защита от записи: Здесь задается, может ли значение свойства быть изменено (запись "Нет") или должно быть защищено от записи (запись "Да").
* Копировать: Если флажок установлен, то при копировании внутри проекта значения соответствующих определенных пользователем свойств копируются вместе с сегментом.
* Значение по умолчанию: Занесите сюда значение, которое должно по умолчанию быть присвоено этому свойству. В зависимости от соответствующего свойства значение можно задать вручную или выбрать из списка при помощи кнопки ++...++.

!!! note "Замечание:"

    Чтобы здесь можно было выбрать определенные пользователем свойства, они должны присутствовать в проекте. Для этого предусмотрены две возможности:

* В диалоговом окне Конфигурировать свойства можно создавать свои, определенные пользователем свойства.
* При копировании данных проектов с базой данных PPE свойства из EPLAN PPE преобразуются в определенные пользователем свойства для предварительного планирования.
* При использовании специальных шаблонов проекта, например для автоматизации зданий, вам по умолчанию доступны определенные пользователем свойства.

Используя кнопки со стрелками над таблицей, можно задать последовательность свойств. Заданная здесь последовательность учитывается также при выводе свойств в отчетах.

Вкладка Нумерация / доступность

На этой вкладке можно задать настройки нумерации обозначений и структурных идентификаторов для определений сегмента. Кроме того, можно сконфигурировать защиту определений сегментов от записи и для всех сегментов указать, какие вкладки будут доступны для ввода данных на сегментах.

### Нумерация обозначений

В следующих полях этого группового поля для разных определений сегмента можно задать отдельные настройки по умолчанию для нумерации обозначений. Эта конфигурация определяет, как будут нумероваться сегменты при вставке.

### Начальное значение

Ввод в это поле показывает, с какого значения начинается нумерация.

### Мин. число разрядов

Значение в этом поле выдает результат, каково минимальное используемое число разрядов. Это значение относится только к разрядам до запятой. Если после запятой в результате присутствует меньше цифр, чем минимальное число разрядов, свободные разряды заполняются заполнителем, указанным в поле Заполнитель.

### Величина шага

Значение в этом поле определяет интервал счета при нумерации.

### Заполнитель

Символ в этом поле используется как заполнитель. Отсутствующие места заполняются этим символом. То, сколько разрядов заполняется, зависит от значения, введенного в поле Мин. число разрядов.

### Защита от записи и нумерация для структурного идентификатора

С помощью защиты от записи определения сегментов структуры конфигурируются так, что для каждого слоя структуры в навигаторе предварительного планирования можно вставлять и обрабатывать только структурные идентификаторы блока идентификаторов (например, установка, место установки и т. д.).

!!! example "Пример:"

    В проекте используютсяблоки идентификаторов"Установка", "Место установки" и "Номер установки" и, в том числе, слой структуры "Завод". Для определения сегментов этого сегмента структуры в таблице Защита от записи и нумерация для структурного идентификаторавозле блоков идентификаторов "Место установки" и "Номер установки" установлен флажок Защита от записи. Вследствие этого пользователь предварительного планирования в сегментах структуры слоя структуры "Завод" получает возможность вводить для блока идентификаторов "Установка" толькоструктурный идентификатор.

Для других определений сегмента можно конфигурировать защиту от записи для отдельных элементов структурных идентификаторов (например, свойство Установка (отдельный элемент) на объектах планирования, технологических контурах и т. д.).

Таблица содержит следующие столбцы:

* Структура: Здесь перечислены блоки идентификаторов, соответствующие структуре проекта.
* Защита от записи: Здесь для выбранного определения сегмента можно задать, разрешено ли изменять структурные идентификаторы для соответствующего блока идентификаторов (флажок снят), или же эти структурные идентификаторы защищены от записи (флажок установлен).
Если для определения сегмента включена защита от записи, в диалоговых окнах 'Свойства' этих сегментов блоки идентификаторов и отдельные элементы структурных идентификаторов будут отображаться, но не будут доступны для обработки.
* Нумеровать: Доступно, только если защита от записи выключена. Здесь определяется, как для структурных идентификаторов соответствующего блока идентификаторов следует проводить нумерацию (не нумеровать, по порядку, как обозначение). При выборе записи "По порядку" счетчики структурных идентификаторов увеличиваются по возрастанию. При выборе записи "Как обозначение" структурные идентификаторы нумеруются в соответствии с настройками в групповом поле Нумерация обозначений.

Настройки защиты от записи и нумерации доступны не только для структурных идентификаторов сегментов, но и для свойства Символический адрес (отдельный элемент).

### Доступные данные

В этой таблице можно сконфигурировать определения сегмента таким образом, что на сегментах будут отображаться только вкладки для необходимых данных. Таким образом, например, для объектов планирования, не имеющих макросов или ПЛК-входов, вкладки ПЛК и Макрос будут скрыты.

Таблица содержит следующие столбцы:

* Данные: Здесь перечислены вкладки, доступные для ввода данных на сегментах.
* Отображение: Здесь вы устанавливаете для выбранных определений сегментов, какие вкладки должны отображаться для ввода данных на сегментах.

Если вкладки для соответствующего определения сегмента принципиально недоступны, соответствующие строки будут неактивны.

!!! note "Замечание:"

    Если во вкладке на сегменте содержатся данные, вкладка отображается и в том случае, если в таблице Доступные данные отключено отображение.
