# Диалоговое окно Синхронизация основных данных — <Имя проекта>

Вы открыли проект, основные данные которого должны быть синхронизированы; вы ***не*** открыли редактор для обработки основных данных (редактор рамок, редактор форм и т. д.). Сервисные программы > Основные данные > Синхронизировать текущий проект.

В этом диалоговом окне установите, какие данные и в каком направлении должны быть синхронизированы. Учтите при этом, что на основные данные (например, на библиотеку символов) ссылки не производятся. Они сохраняются в проекте, а также копируются. Тем самым, осуществляется сохранение в ***проекте*** использованных основных данных. В зависимости от настроек проекта в меню Параметры > Настройки > Проекты > "Имя проекта" > Управление > Общее, Параметры > Настройки > Проекты > "Имя проекта" > Управление > Библиотеки символов и Параметры > Настройки > Проекты > "Имя проекта" > Отчеты > Вывод в страницах вы можете при необходимости вручную задать синхронизацию специфичных для проекта основных данных с относящимися к системе данными.

Обзор основных элементов диалогового окна:

### Основные данные проекта

Список основных данных, используемых в проекте, содержит следующие записи:

* Статус: Сведения о статусе, например "Более новый" или "Идентичный"
* Имя: Имя файла объекта основных данных, без расширения имени файла
* Тип: Тип основных данных (библиотека символов, форма для титульной страницы, рамка и т. д.)
* Дата изменения: Дата последнего изменения объекта основных данных в проекте.

Записи сортируются по столбцам в алфавитно-числовом порядке, при этом сортировка по умолчанию производится по записи Имя. Щелкните столбец Тип, в результате все записи отображаются отсортированными по типу основных данных.

### Каталоги

Данный раскрывающийся список позволяет выбрать различные каталоги для синхронизации основных данных. Чтобы обеспечить эту возможность, различные настройки каталогов необходимо предварительно сохранить в виде схемы. Это может пригодиться при использовании различных каталогов основных данных для различных фирм / клиентов. Щелкните по кнопке ++...++, чтобы открыть диалоговое окно Настройки: Каталоги и создать новую схему для каталогов.

### Основные данные системы

Список системных основных данных содержит следующие записи:

* Статус: Сведения о статусе, например "Не сохранено" или "Идентичный"
* Имя: Имя файла объекта основных данных, без расширения имени файла
* Тип: Тип основных данных (библиотека символов, форма для титульной страницы, рамка и т. д.)
* Дата изменения: Дата последнего изменения объекта основных данных в проекте в соответствующем редакторе (редакторе форм, редакторе символов, редакторе рамок и т. д.).

Сортировка соответствует сортировке в меню Основные данные проекта.

### Кнопки со стрелками

Используйте клавиши со стрелками для копирования в проект основных данных системы или для применения отдельных основных данных проекта в системные основные данные. Многократный выбор возможен в обоих списках.

EPLAN привлекает внимание в окне указаний на то, сколько данных проектов / системных основных данных были обновлены; если данные проектов и системные основные данные идентичны, об этом также выдается информация.

### Фильтр

В этом раскрывающемся списке отображаются все доступные фильтры. Выбранный фильтр активируется автоматически. Пункт "-Не активировано-" отключает фильтр и приводит к тому, что данные отображаются в неотфильтрованном виде.

* Отобразить различия: Выберите этот фильтр, чтобы отображались только различные данные как основных данных проекта, так и основных данных системы, независимо от того, какие данные более ранние или поздние.
* Основные данные проекта: Выберите этот фильтр, чтобы отображались только те основные данные проекта, которые являются более ранними, чем соответствующие основные данные системы.
* Основные данные системы: Выберите этот фильтр, чтобы отображались только те основные данные системы, которые являются более ранними, чем соответствующие основные данные проекта.

### [Обновить]

Пункт меню |  Значение
---|---
Проект |  Заменяет устаревшие основные данные проекта на новые системные основные данные.
Система |  Заменяет устаревшие системные основные данные на новые основные данные проекта.
Сжатые библиотеки символов в проекте  |  Заменяет сжатые библиотеки символов в основных данных проекта библиотеками символов из основных данных системы.

!!! note "Замечание:"

    * Если при синхронизации новая версия объектов основных данных заменяется на более старую (идентификация в записи по дате изменения), выводится соответствующее указание, и вы можете прервать обновление.
    * При сравнении основных данных допускаются имена файлов с прописными и строчными буквами, но обрабатываются они одинаково. То есть имя файла ABC.XYZ идентично с abc.xyz.
    * При обновлении ***библиотек символов*** среди прочего проверяется, совместимы ли файлы (например относительно обозначений выводов устройства); если это не так, процесс прерывается с соответствующим сообщением.
