# Вкладка Соединение

Вы открыли проект. Данные проекта > Соединения > Навигатор. Выделите соединение и выберите во всплывающем меню пункт Свойства. Выберите в диалоговом окне Свойства ++...++ вкладку Соединение.

На этой вкладке обрабатываются свойства соединения.

Обзор основных элементов диалогового окна:

### Обозначение соединения

При нумерации соединений сюда автоматически вставляется обозначение соединения. Однако обозначение можно вставить и вручную.

С помощью пункта всплывающего меню Определить новое обозначение можно автоматически определить ближайшее свободное обозначение соединения. Тогда на основе настроек для нумерации соединений будет определено следующее свободное обозначение соединения и записано здесь.

### Описание

Здесь можно ввести дополнительное описание соединения, это свойство не является идентифицирующим и используется только в информационных целях.

* * *

Групповое поле Кабель / Группа соединений

### Видимое ОУ

Если соединение представляет собой кабельное соединение или кабель, то здесь отображается ОУ кабеля. Для шлангов отображается ОУ группы соединений.

Кнопка ++...++ позволяет открыть диалоговое окно Выбор ОУ, если в таблице в групповом поле Свойства для свойства Соединение: Принадлежность настроено значение "Кабель"; в противном случае откроется диалоговое окно Использовать имеющиеся соединения.

Видимое ОУ будет показано в графическом редакторе рядом с условным обозначением. Если вы определили в настройках проекта, что устройство должно автоматически нумероваться при вставке символов, здесь вам будет автоматически предложено ОУ. Вы можете сохранить или переписать вручную это ОУ.

Кроме того, можно использовать [всплывающее меню или кнопку ++...++](devicetaggui_r_feldsichtbaresbmk.md), чтобы cгенерировать или выбрать ОУ.

### Полное ОУ

В этом поле отображается полное ОУ (т. е. структурный идентификатор текущего окружения + видимое ОУ) видимого ОУ / функционального элемента. С помощью кнопки ++...++ откройте диалоговое окно Полное ОУ. Здесь можно обработать структурный идентификатор и, например, выбрать другое место установки.

### Цвет / номер

Введите здесь цвет или номер соединения, это свойство идентифицирует это соединение. Данное поле автоматически заполняется с помощью функции нумерации соединений. Щелкните ++...++, чтобы открыть диалоговое окно Цвета соединений и выбрать цвет из списка.

### Парный индекс

Для кабельных соединений парной скрутки кабеля или спаренных шлангов одного комплекта задается парный индекс, которому принадлежат оба кабельных соединения или шланга. Таким образом, он идентифицирует пару кабельных соединений или пару шлангов. Это значение может быть введено вручную или автоматически заполнено при выборе кабеля или выборе изделия.

Парный индекс служит в качестве идентифицирующего свойства для автоматического присвоения соединениям кабельных соединений или шлангов и может использоваться как критерий для выбора кабеля или комплекта через выбор устройства.

* * *

### Поперечное сечение / диаметр:Единица измерения сечения / диаметра

Поперечное сечение или диаметр соединения копируется из базы данных изделий и могут быть использованы в качестве критерия отбора при выборе устройств. Щелкните по ++...++, чтобы открыть диалоговое окно Поперечные сечения / диаметры и выбрать сечение / диаметр, включая соответствующую единицу, из предложенного списка.

Если в поле Поперечное сечение / диаметр вы выбрали предварительно определенное поперечное сечение / диаметр, то в поле Единица измерения сечения / диаметра автоматически появится соответствующая единица. С помощью раскрывающегося списка возможен выбор другой единицы измерения. Обзор возможных единиц измерения приводится в соответствующем разделе "[Соединения: Единицы измерения длины и поперечного сечения](connectionbrowsergui_k_einheitenverbindungen.md)".

### Вид представления

Вид представления соединений зависит от вида представления целей этих соединений. Его, однако, можно явно определить в точке определения соединения либо на самом соединении. При этом данная настройка перезаписывает вид представления, полученный от целей соединения.

При изменении вида представления изменяется и оценка соединения. Для неразмещенных соединений могут использоваться только виды представления "Многополюс." и "Однополюс."; другие виды представления доступны только для размещенных соединений. Соединения (как и условные обозначения) могут быть [графическими и внешними](devicetaggui_k_grafischeschaltzeichen.md). Выберите требуемую запись из раскрывающегося списка:

* Не определено: При этой настройке вид представления автоматически определяется по целям соединения.
* Многополюсный: Определяет соответствующее соединение как многополюсное. При проведении оценки соединения можно отфильтровать по этому критерию.
* Однополюсный: Определяет соответствующее соединение как однополюсное. При проведении оценки соединения можно отфильтровать по этому критерию.
* Функциональная схема автоматизации: Используется для соединений, размещенных на функциональной схеме автоматизации. К таким относятся, например, соединения между технологическими резервуарами.
* Обзор функций (Fluid-Техника): Используется для соединений, которые размещаются в обзорах функций для Fluid-техники. Такие соединения ***не*** анализируются в отчетах.
* Внешн.: При этой настройке во время прогонов отчета и контрольных прогонов соответствующее соединение ***не*** анализируется напрямую. Внешние соединения анализируются только косвенно, то есть они учитываются, например, в схемах подключения схем, но не учитываются в таблицах соединений. При нумерации соединений внешние соединения не обозначаются.
* Графика: При этой настройке во время прогонов отчета и контрольных прогонов соответствующее соединение ***не*** анализируется. При нумерации соединений графические соединения не обозначаются.

!!! note "Замечание:"

    Цели анализируются посредством внешних и логических соединений. Это значит, что не от самой цели, а от соединения с этой целью зависит, будет ли цель выводиться в отчете. Если соединение с целью графическое, то об этой цели отчет не формируется.

### Определение функции

С помощью кнопки ++...++ откройте диалоговое окно Определения функций. Определения функций там уже отфильтрованы, поэтому выводятся только определения соединений. Выберите требуемое определение функции.

Это поле синхронизируется с полем Определение, находящимся на вкладке Данные символа / функции.
