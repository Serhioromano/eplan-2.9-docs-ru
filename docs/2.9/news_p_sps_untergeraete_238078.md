# Расширения для подустройств ПЛК

Некоторые устройства ПЛК состоят из нескольких интегрированных узлов, каждый из которых имеет собственные диапазоны адресов и начальные адреса. Такие интегрированные модули в устройстве ПЛК могут отображаться в EPLAN подустройствами ПЛК — для этого доступно до двенадцати подустройств ПЛК. В этой версии расширено управление подустройствами ПЛК. Это означает, что теперь вы также можете обмениваться подустройствами ПЛК, которые управляются как отдельные устройства в программе конфигурации ПЛК. Кроме того, теперь стало возможным присваивать выводы шин подустройству ПЛК в дополнение к соединениям ввода-вывода.

### Идентификация устройства для подустройств ПЛК

Теперь на подустройствах ПЛК можно вводить идентификацию устройства. Поэтому для каждого подустройства ПЛК доступны два новых свойства:

* Подустройство ПЛК x: Обозначение типа ПЛК: обозначение типа ПЛК для подустройства ПЛК x карты ПЛК.
* Подустройство ПЛК: Описание устройства: Индекс в файле: ссылка на устройство в файле описания устройства, который введен на блоке ПЛК (главном устройстве) в свойстве Описание устройства: Имя файла.

![](images/benefit.png)Эффект:

Благодаря идентификации устройства при обмене данными ПЛК в формате AutomationML возможен также обмен данными подустройств ПЛК, которые в программе конфигурации ПЛК управляются как самостоятельные устройства.

Эти свойства доступны в диалоговом окне "Свойства" блоков ПЛК на вкладке Блок ПЛК и в базе данных изделий на вкладке Свойства. При выборе изделия или устройства те значения, которые сохранены в шаблонах функций изделий, переносятся на функции. Данные устройства для главного устройства и связанных подустройств ПЛК настраиваются производителем в базе данных изделий один раз для каждого устройства.

!!! note "Замечание:"

    Идентификация устройства должна выполняться аналогичным образом для главного устройства и связанных с ним подустройств ПЛК. Если, например, на блоке ПЛК указано обозначение типа ПЛК, то подустройства ПЛК также должны определяться с использованием обозначения типа ПЛК.

### Единые свойства для всех подустройств ПЛК

В рамках этого расширения свойства подустройств ПЛК были стандартизированы, а начальные адреса и длина данных первого подустройства ПЛК были отделены от начальных адресов и длины данных главного устройства. Для подустройства ПЛК 1 эти данные больше не указываются на главном устройстве, а, как и в других подустройствах ПЛК, через отдельные свойства. Для этого доступны такие новые свойства:

* Подустройство ПЛК 1: Начальный адрес (входы)
* Подустройство ПЛК 1: Начальный адрес (выходы)
* Подустройство ПЛК 1: Длина данных (входы)
* Подустройство ПЛК 1: Длина данных (выходы)

### Групповое отображение подустройств ПЛК в базе данных изделий

В базе данных изделий на вкладке Свойства можно вводить данные для подустройств ПЛК. Под узлом ПЛК свойства для отдельных подустройств ПЛК теперь отображаются в сгруппированном виде. Благодаря этому сразу можно увидеть присвоение свойств отдельным подустройствам ПЛК.

### Выводы шин на подустройствах ПЛК

Свойство Подустройство ПЛК: Индекс (ид. 20384) теперь доступно также на выводах шин. Ранее это свойство было доступно только для соединений ввода/вывода. Это свойство используется для указания того, какому подустройству ПЛК принадлежит вывод устройства ПЛК. Так, например, значение индекса "3" означает, что вывод устройства ПЛК относится к подустройству ПЛК 3. Эти данные требуются при автоматической адресации. Свойство также может указывать в шаблонах функций изделий и переносить на выводы устройства ПЛК при выборе устройства или при вставке устройств.
