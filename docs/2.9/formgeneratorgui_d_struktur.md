# Диалоговое окно <Тип отчета>

Вы открыли проект.

* Сервисные программы > Отчеты > Генерировать. Щелкните по кнопке ![](images/all_new_as.png){: .ui-icon } (Создать). Выделите в диалоговом окне определить отчет [тип отчета](formgeneratorgui_k_auswertungstypen.md). Щелкните по кнопке ++OK++.
* Сервисные программы > Отчеты > Генерировать. Выберите вкладку Шаблоны. Щелкните возле свойства Начальная страница блока отчетов в столбце Значение и по ++...++..

В этом диалоговом окне указывается нач.страница для каждого блока отчетов. Генерирование отчета в одном или нескольких блоках отчетов зависит от вида [Сортировка страниц отчетов](formgeneratorgui_k_seitensortierung.md), которую вы задали в настройках. Если вы генерируете содержание, сортировка страниц Общая генерирует, например, только один блок отчетов, т. е. общее содержание для всей документации схемы соединений, в то время как сортировка страниц Установка для каждой установки генерирует блок отчетов, т. е. отдельное содержание для каждой установки. Или вы копируете начальную страницу, которую предлагает EPLAN для блока отчетов, или задаете другую начальную страницу. Нач.страницу можно указать, выбрав имеющиеся структурные идентификаторы структуры страниц или создав новый структурный идентификатор.

Обзор основных элементов диалогового окна:

### Структурный идентификатор

В данной таблице отображаются структурные идентификаторы. Их можно создать или выбрать как начальную страницу для текущего блока отчетов. Чтобы создать структурный идентификатор, введите его прямо в поле. Для выбора имеющегося структурного идентификатора, щелкните на ++...++, и выберите в выведенном диалоговом окне структурный идентификатор.

### Имя страницы

В этом поле находится имя страницы, которое предлагает EPLAN для страницы отчетов.

### Добавить в конце

Данная функция влияет на отчеты, генерируемые из шаблонов отчетов. Флажок доступен, если в диалоговом окне Отчеты на вкладке Шаблоны выделен шаблон отчета и рядом со свойством Начальная страница блока отчетов в столбце Значение нажато [...]. Если флажок установлен, то поле Имя страницы пустое и недоступное. При составлении отчетов страницы отчетов сортируются в введенных структурных идентификаторах. Если в уровне структурного идентификатора уже находятся страницы отчетов, то все новые страницы отчетов сортируются после последней уже существующей страницы. Не используются имена страниц, оставленные свободными для резервных страниц между имеющимися страницами.

### Доп. поле номера листа

В этом поле находится номер листа, который предлагает EPLAN для страницы отчетов.

### Автоматическое описание страницы

Если установлен этот флажок, то для описания страницы используется формат, который установлен в форме для свойства Формат для автоматического описания страниц. Если этот флажок не установлен, то в поле Описание страницы вы можете ввести описывающий текст.

### Описание страницы

Это поле доступно, если флажок Автоматическое описание страницы не установлен. В этом поле введите описывающий текст для страницы.

### Навигатор страниц

В этом поле выводится структура страницы со всеми имеющимися страницами проекта. Тем самым можно задать нач.страницу страницы отчета. При выделении уровня структуры страниц структурные идентификаторы уровня структуры страниц автоматически заносятся в верхние поля. Нач. страница начинается на этом уровне структуры страниц.

### Применить начало для всех структурных идентификаторов

Эта функция доступна только если отчет создается в нескольких [блоках отчетов](formgeneratorgui_k_auswertungsbloecke.md). Для каждого блока отчетов необходимо задать нач. страницу. Если этот флажок установлен, не обязательно задавать нач. страницу для каждого блока отчетов или подтвердить предложенные EPLAN страницы по отдельности. Нач. страница для всех дальнейших блоков отчетов EPLAN задается автоматически.
